颜色科学:RGB / CIE / HSV / Lab

同一个"红",相机怎么记、显示器怎么发光、印刷机怎么调墨、算法怎么分割、科学家怎么量色差——五个场景,催生了五套坐标系。这一节把它们摆到一起,讲清"为什么一个颜色需要这么多种说法"。

直觉类比

同一个地点,可以用经纬度(全球统一、绝对)、门牌号(人用、方便找)、也可以用快递分拣码(机器用、方便处理)来描述。它们指的是同一个点,只是为不同用途服务。颜色空间就是颜色的这几套"地址系统":RGB 是设备发光的物理配方,HSV 是人调色的手感,Lab 是测量色差的标准尺,CIE XYZ 则是那个"全球统一的经纬度"——所有转换都要经过它中转。

1.颜色从哪来:光谱与人眼

颜色不是物体的固有属性,而是光与人眼共同作用的结果。可见光是波长在 400 nm ~ 700 nm 的电磁波;每段波长上有多少能量,画成曲线就是光谱能量分布 E(λ)(SPD)。

关键在人眼这一侧:视网膜上有三种视锥细胞,分别对红、绿、蓝最敏感。任何复杂的光谱进到眼里,都被压缩成三个数(三种视锥的响应量)。这就是"三色理论"——也是为什么显示器只用 RGB 三种光就能骗过眼睛,重现几乎所有颜色。

这解释了一切颜色空间的根:颜色本质是三维的,所以所有颜色空间都是三个分量(CMYK 的 K 是为省墨额外加的)。

2.四套核心坐标系

RGB(加性颜色模型)
红、绿、蓝三种光相加混合,越加越亮,全开为白。用于相机采集、屏幕显示、图像存储。是"发光"设备的物理配方——但它是设备相关的:同一组 RGB 值在不同屏幕上颜色略有差异。
CIE XYZ(标准基色空间)
为解决 RGB 配色曲线出现负值(现实中无法"减去一束光")的问题,CIE 用线性变换造出三个虚拟原色 X、Y、Z,使配色函数全为正。其中 Y = 亮度(等于光视效率曲线 V(λ)),X、Z 负责描述色彩本身。设备无关的国际标准;是 RGB↔Lab 转换的必经中转站。
HSV(色相/饱和度/明度)
H 色相(0~360°,红黄绿…绕一圈)、S 饱和度(0~1,鲜艳↔灰)、V 明度(0~1,亮↔暗)。贴近人"描述颜色"的直觉:想调更鲜就加 S,想调更亮就加 V。图像分割、OpenCV 算法常用。
L*a*b*(CIELAB)
L* 亮度(0 黑~100 白)、a*(绿−到红+)、b*(蓝−到黄+)。按韦伯定理设计,使得数值上等距 ≈ 视觉上等距。视觉均匀:两点欧氏距离≈人眼感受的色差。色差计算、印刷质检的标准。

色度图与色域(高频考点)。CIE 把 XYZ 归一化投影成二维色度图(马蹄形),边界是纯光谱色的光谱轨迹,白点在 (1/3, 1/3)。色域(Gamut)指某设备/颜色空间能准确表示的颜色子集——在色度图上是一个三角形(RGB 三顶点围出)。落在三角形外的颜色叫超色域(out-of-gamut),设备无法显示,只能沿"该色→白点"连线与色域边界的交点做近似。这就是为什么照片打印出来常和屏幕上不完全一样:印刷的 CMYK 色域 ≠ 屏幕的 RGB 色域。

3.横向对比:各颜色空间为何而生

不要死记参数,抓住每一行的"核心用途"——那就是它存在的理由。

颜色空间定义 / 通道典型用途优点 / 缺点
RGB加性;R 红 / G 绿 / B 蓝(0~1 或 0~255)相机、屏幕显示、图像存储+ 直连硬件、通用入口
− 设备相关;不贴近人感知
CIE XYZ虚拟原色;Y=亮度,X/Z 描述色彩颜色标准、跨空间中转+ 设备无关的绝对标准
− 不直观,人不会直接用它调色
HSVH 色相° / S 饱和度 / V 明度调色、图像分割、算法(OpenCV)+ 符合"调颜色"直觉、易分离色相
− 非视觉均匀;设备相关
L*a*b*L 亮度 / a 绿-红 / b 蓝-黄色差计算、色彩度量、印刷质检+ 视觉均匀,距离≈色差;设备无关
− 计算复杂,不能直接驱动硬件
CMY(K)减性;C=−R,M=−G,Y=−B,K 黑打印机、印刷、墨水+ 匹配"油墨吸收光"的物理;K 省墨省钱
− 色域比 RGB 窄,屏↔纸有色差

加性 vs 减性(易混):RGB 是加性(光相加,越加越亮,叠满是白);CMY 是减性(油墨从白光里"减掉"某色,越涂越暗,叠满近黑)。青(C)吸收红、品红(M)吸收绿、黄(Y)吸收蓝——所以 C=−R、M=−G、Y=−B。理论上 C+M+Y 叠出黑,但油墨叠黑既贵又发污,故印刷额外加一支纯黑墨 K,即 CMYK。

4.一个能算的转换:RGB ↔ CMY

RGB→CMY 是课上唯一给出闭式公式的转换(在归一化到 [0,1] 的前提下),简单到可以直接考:

例题:把 RGB 转成 CMY

某像素归一化后 R=0.2, G=0.6, B=1.0(偏蓝),求其 CMY 值。

  1. 套公式 C = 1 − R → C = 1 − 0.2 = 0.8
  2. M = 1 − G → M = 1 − 0.6 = 0.4
  3. Y = 1 − B → Y = 1 − 1.0 = 0.0
结果:CMY = (0.8, 0.4, 0.0)。意义自洽——该色很蓝,而蓝=白−黄,所以黄墨 Y=0(不需要吸收蓝);同时青墨 C 最高,因为要大量吸收红。反向转换同理:R=1−C 等。

注意:HSV、Lab 与 RGB 的转换不是这种一行公式(Lab 还必须经 XYZ 中转),期末一般不要求手算,只考概念辨析和这条 CMY 公式。

自测题 1

下列关于颜色空间的说法,错误的是?

  1. RGB 是加性模型,是设备相关的
  2. CIE XYZ 是设备无关的标准,常作为不同颜色空间转换的中转
  3. CMY 是减性模型,C、M、Y 分别对应 −R、−G、−B
  4. HSV 是视觉均匀空间,两点欧氏距离等于人眼感知的色差
答案:D

视觉均匀是 L*a*b*(CIELAB) 的特性(按韦伯定理设计,距离≈色差),不是 HSV。HSV 只是贴近人"调色"的直觉,并不满足"数值等距=视觉等距"。A、B、C 均正确:RGB 加性且设备相关,XYZ 是中转标准,CMY 减性且 =−R/−G/−B。
套路:"视觉均匀 / 色差计算"这类关键词,几乎总是指向 Lab。

自测题 2

照片在屏幕上看很鲜艳,打印出来却发暗、偏色。用色域(gamut)概念解释,最恰当的是?

  1. 打印机分辨率不够,像素太少
  2. CMYK 印刷色域比 RGB 屏幕色域窄,部分屏幕颜色超出印刷色域,只能近似
  3. Lab 空间不支持打印
  4. 光谱轨迹是马蹄形,所以颜色会失真
答案:B

色域是某设备能准确表示的颜色子集。屏幕的 RGB 色域与打印机的 CMYK 色域在色度图上是两个不同大小的三角形,CMYK 通常更窄。屏幕上那些落在 RGB 色域内、却在 CMYK 色域之外的鲜艳色就是超色域颜色,打印时只能沿"该色→白点"连线映射到色域边界做近似,于是发暗偏色。A 是分辨率问题(与色域无关),D 只描述了色度图形状但没解释色差成因。
