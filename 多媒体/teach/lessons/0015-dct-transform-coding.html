DCT 变换编码

DCT 本身一分信息都不丢——它只是换了个"坐标系"。真正的魔法在于:换过去之后,几乎全部能量都挤到了左上角那几个数上,后续压缩才有得压。

直觉类比

想象你要打包一整衣柜的衣服。衣服本身没变(无损),但你先把它们按季节分类叠好——厚外套一摞、夏装一摞。分类后你会发现:90% 的体积集中在几摞常穿的衣服上,那些几乎不穿的可以真空压缩甚至扔掉。DCT 就是这个"分类"步骤:它不丢信息,只是把图像块里"变化缓慢的大块能量"和"变化剧烈的细节能量"分开,让绝大部分能量集中到少数几个低频系数上。真正"扔衣服"的是下一步的量化。

1.关键定义

变换编码 (Transform Coding)
先把信号从一个域(空间/像素域)可逆地变换到另一个域(频率域),使能量集中、样本间相关性被去除,再对变换系数做量化与熵编码的压缩框架。流程:变换 → 量化 → 熵编码。变换这一步本身不压缩、不丢信息。
DCT (离散余弦变换)
把一组像素值表示成一系列不同频率余弦波的加权和,权重就是 DCT 系数。它是 JPEG 与 MPEG 视频的核心。DCT 是线性、正交、完全可逆的变换,配套的逆变换叫 IDCT。
DC 系数 (直流系数)
频率为 0(u=0)的那个系数 F(0),代表整个块的平均亮度——余弦波"零频率"就是一条水平直线。它通常是块里数值最大、最重要的系数。
AC 系数 (交流系数)
频率大于 0(u>0)的所有系数,描述块内亮度怎样起伏变化:低频 AC 管大致明暗轮廓,高频 AC 管边缘与细节。

2.为什么要"变换后再编码"

这是本讲最容易被问到"为什么"的地方。分三层理解,一步比一步深:

层次问题DCT 给的答案
1 去相关相邻像素高度相似(空间冗余),直接编码是在重复存储同样的信息。DCT 去除样本间相关性,把冗余显式地暴露出来。
2 能量集中信息散布在 64 个像素上,无从取舍。DCT 把绝大部分能量集中到左上角前几个系数,其余系数几乎都在 0 附近。
3 便于取舍要压缩就必须敢丢东西,但不能乱丢。能量集中后,那些接近 0 的小系数一量化就归零,几乎不影响重建质量——人眼对高频不敏感。

换句话说:直接对像素量化,丢哪个都心疼,因为每个像素都携带差不多的信息。变换之后,信息被"重新分配":少数系数扛起了几乎全部能量,剩下一大片可以放心量化掉。这就是"先变换、后编码"的全部动机——用一次无损的坐标变换,换来后续压缩的"下手空间"。

务必记牢(与 0014 呼应)

DCT 本身无损。它是线性正交变换,IDCT 能把系数精确还原成原始像素——理论上不损失任何信息。JPEG 的失真全部来自量化那一步(把系数除以量化表并取整,取整就回不去了)。所以考试里若问"JPEG 在哪一步产生损失",答案永远是量化,绝不是 DCT。

3.1D DCT 公式与 DC 系数

对 N=8 个样本,一维 DCT 把输入 f(i) 变成新序列 F(u):

F(u) = C(u) · Σi=0N-1 f(i) · cos[(2i+1)uπ / 2N]

其中系数 C(u) 保证变换正交:C(0)=√(1/N),C(u)=√(2/N)(u>0)。当 N=8 时分母 2N=16,正是课件里那个 (2i+1)uπ/16。

把 u=0 代进去:此时 cos(0)=1,求和退化为把 8 个样本直接相加。于是

F(0) = C(0) · Σ f(i) = √(1/8) · (8 × 均值) = √8 · 均值

这就证明了:DC 系数正比于该块所有样本的平均值。"直流"这个名字也由此而来——零频率的余弦就是一条常数直线,对应的正是"平均水平"。

例题 · 一个纯直流块的 DCT

设一行 8 个像素全都相等:f = [100,100,100,100,100,100,100,100](一块毫无变化的平坦区域)。求它的 1D DCT 系数,并解释"能量集中"。

  1. 算 DC 系数 F(0)。 u=0 时余弦项全为 1,故 F(0)=C(0)·Σf(i)=√(1/8)·(100×8)=800/√8=800/2.828≈282.8。也等于 √8×均值=√8×100,与公式一致。
  2. 算 AC 系数 F(u), u>0。 由于所有 f(i) 相等(记作常数 k),F(u)=C(u)·k·Σicos[(2i+1)uπ/16]。对每个 u>0,这 8 个余弦值正负对称、恰好相加为 0。
  3. 得到结果向量。 F = [282.8, 0, 0, 0, 0, 0, 0, 0]。
  4. 解读能量集中。 输入的全部"能量"原本摊在 8 个数上,变换后100% 集中到了单一的 DC 系数,7 个 AC 系数全为 0。后续只需存 1 个数而非 8 个——这正是压缩得以发生的根源。
结论:平坦块 → 只有 DC 非零,能量全在低频。真实图像块虽不会这么极端,但"大部分能量挤在左上角少数系数、其余接近 0"是普遍规律(课件中 8×8 块的 DCT 系数直方图显示绝大多数系数都在 0 附近)。而且注意:这一步没丢任何信息,对 F 做 IDCT 能精确还原出原来的 8 个 100。
自测题

关于"为什么 JPEG 要先做 DCT 再编码",下列说法正确的是?

  1. DCT 会直接压缩数据量,是 JPEG 主要的信息损失来源。
  2. DCT 本身无损,它把能量集中到少数低频系数,为后续量化"敢于丢弃小系数"创造了条件。
  3. DCT 把图像从频率域变回空间域,方便人眼观看。
  4. DCT 的作用是给每个像素单独编号,和压缩无关。
答案:B

B 正确。DCT 是线性可逆(正交)变换,本身不丢信息;它的价值在于去相关 + 能量集中——把大部分能量堆到左上角少数系数,使那些接近 0 的高频系数在量化时归零而几乎不损害画质。
A 错:DCT 不压缩、不丢信息,损失来自量化(呼应 0014)。
C 错:DCT 是空间域 → 频率域,方向反了,而且人眼看的是重建后的空间域图像。
D 错:DCT 恰恰打破"逐像素"的表示,换成频率系数表示,正是为压缩服务。
解题套路:凡把"损失"归给 DCT 的选项一律排除;凡强调"无损变换 + 能量集中"的就是正解。
