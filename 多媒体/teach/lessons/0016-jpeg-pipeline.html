JPEG 完整流水线

这是那道 15 分的大计算题。前几节我们已把 RGB→YCbCr、DCT、量化拆开练过;本节把最后三步——之字扫描、DC 差分、AC 游程——一次串成完整答卷,给你一套照抄就能得分的解题模板。

直觉类比

量化后的 8×8 系数块像一张左上角写满字、右下角几乎空白的表格:能量都堆在左上(低频),右下角全是 0。若逐行读,0 会被一堆非零数打散;而之字(Zig-zag)扫描是沿对角线来回走,把"从最低频到最高频"顺序拉直——于是所有 0 自然滑到队尾,连成一条长长的零尾巴。后面的游程编码就靠这条尾巴"一句话交代完",这正是 JPEG 省空间的关键一招。

1.JPEG 五步流水线

把整条压缩链背下来,大题第一问常直接考"写出 JPEG 编码步骤"。真题的计算重心在第 4、5 步,但前三步的名词不能漏。

步操作做了什么 / 为什么
1RGB → YCbCr + 色度二次采样分离亮度 Y 与色度 Cb/Cr;人眼对色度不敏感,4:2:0 先扔掉 3/4 色度信息。
2分 8×8 块做 2D DCT把空间像素变到频域,能量集中到少数低频系数(左上角)。
3量化 F̂(u,v)=round(F(u,v)/Q(u,v))除以量化表并取整,高频被压成 0——JPEG 有损、压缩的主来源。
4之字扫描 + 游程编码Zig-zag 拉直系数;DC 用 DPCM 差分,AC 用 (run,level) 游程。本节重点。
5熵编码(Huffman)对 DC 差分和 AC 游程符号做变长编码,进一步无损压缩。

记忆口诀:换色 → 变频 → 量化 → 扫描游程 → 熵编码。前三步"有损",第 4、5 步"无损"。

2.四个关键定义

Zig-zag 之字扫描
沿对角线来回,把 8×8 系数块按频率由低到高展开成 64 个数的一维序列。目的:让左上低频集中在前、右下零系数集中在后,拉长零游程,喂给游程编码。
DC 系数与 AC 系数
DC = F(0,0),块的平均亮度(直流分量);AC = 其余 63 个系数(交流分量)。二者分开编码。
DC 的 DPCM 差分
相邻块 DC 值接近,故只存差值 d = DC本块 − DC前块(首块 d = DC 本身)。Differential PCM,差分脉冲编码,用小数代替大数。
AC 的游程编码 RLC
把 AC 序列写成一串 (RUNLENGTH, VALUE):VALUE 前面有几个连续 0(run),VALUE 是该非零值。末尾若全为 0,用 (0,0)=EOB(块结束)一次收尾。

3.为什么先做之字扫描

量化后的块长这样(下面例题会用到它):能量堆在左上,右下大片 0。假如逐行读,第一行读完非零值后马上撞上 0,下一行又冒出非零值,0 被切成好几段短游程——游程编码几乎占不到便宜。

之字扫描换了个走法:它按"离左上角的对角线距离"由近及远地遍历,而对角线距离恰好对应 DCT 的频率高低。于是低频非零值全排在前面,一旦进入高频区就是连续的 0,直到块尾。一条长零尾巴用一个 EOB 就能表达完毕——这就是 Zig-zag 存在的唯一目的:把二维的"零分布"重排成一维的"零聚集",最大化零游程长度。

记住扫描的头几步走向:(0,0)→(0,1)→(1,0)→(2,0)→(1,1)→(0,2)→(0,3)→…,像沿反斜线来回折返。这个顺序背下来,考场上照着块填一维序列即可。

例题 · 15 分真题还原 · JPEG 第 4~5 步全流程

某图像块经 DCT 与量化后得到下面的 8×8 量化系数块。已知前一个块的 DC 系数为 38。请:①做之字扫描得一维序列;②对 DC 做 DPCM 差分;③对 AC 做游程编码 (run, level);④说明如何对 AC 结果做 Huffman 熵编码。

405-100000
-32000000
00000000
-10000000
00000000
00000000
00000000
00000000

红 = DC(0,0);深色格 = 非零 AC;浅色 0 是被量化压掉的高频。

  1. 之字扫描,展开成一维。 沿 Zig-zag 顺序取值:(0,0)=40 → (0,1)=5 → (1,0)=-3 → (2,0)=0 → (1,1)=2 → (0,2)=-1 → (0,3)=0 → (1,2)=0 → (2,1)=0 → (3,0)=-1,其后位置全是 0。
    40, 5, -3, 0, 2, -1, 0, 0, 0, -1, 0, 0, …, 0  (共 64 个,后 54 个全 0)
    第 1 个是 DC=40,后面 63 个是 AC。注意零已被扫描聚拢到尾部,这正是之字扫描的功劳。
    序号0(DC)12345678910~63
    值405-302-1000-1全 0
  2. DC 用 DPCM 差分。 本块 DC = 40,前块 DC = 38,故差值 d = 40 − 38 = 2。JPEG 只把这个 2 送去编码,而不是 40——相邻块亮度接近,差值远小于原值,更省位数。
    DC 编码值 = 2(若为首块则直接编码 40)。
  3. AC 用游程编码 (run, level)。 对 AC 序列 5, -3, 0, 2, -1, 0, 0, 0, -1, 0…0 从左到右扫,记录每个非零值前有几个连续 0:
    非零值前导 0 个数 (run)(run, level)
    50(0, 5)
    -30(0, -3)
    21(1, 2)
    -10(0, -1)
    -13(3, -1)
    —(其后全 0)—(0, 0) = EOB
    AC 游程结果:(0,5) (0,-3) (1,2) (0,-1) (3,-1) (0,0)
    尾部 54 个 0 一个 EOB (0,0) 全部交代完——这就是零尾巴带来的压缩红利。
  4. 对 AC 结果做 Huffman 熵编码。 每个 AC 符号进一步写成 (RUNLENGTH, SIZE, AMPLITUDE):SIZE = 表示 |level| 所需的位数,AMPLITUDE 是该值的二进制码(负数取正数的反码,以 0 开头即为负)。
    (run, level)SIZEAMPLITUDE(RUN, SIZE, AMP)
    (0, 5)3101(0, 3, 101)
    (0, -3)200(0, 2, 00)
    (1, 2)210(1, 2, 10)
    (0, -1)10(0, 1, 0)
    (3, -1)10(3, 1, 0)
    (0, 0) EOB——(0, 0)
    规则:把 (RUNLENGTH, SIZE) 拼成一个字节(各 4 位)去查 Huffman 表得变长码,AMPLITUDE 直接附在后面原样发送(不再 Huffman)。DC 那边则用 (SIZE, AMPLITUDE):d=2 → SIZE=2、AMP=10 → (2, 10),SIZE 查 DC 专用 Huffman 表。
    最终码流 = DC 的 Huffman 码 + 各 AC 的 Huffman 码,即完成 JPEG 一个块的全部编码。
自测题

某 8×8 块之字扫描后的一维序列为(第一个数为 DC):
28, 3, 0, 0, -2, 0, 0, 0, 0, 4, 0, 0, …, 0(其后全为 0)。
请写出对 AC 部分的游程编码 (run, level),并以 EOB 收尾。下列哪一项正确?

  1. (0,3) (2,-2) (4,4) (0,0)
  2. (1,3) (2,-2) (4,4) EOB
  3. (0,3) (3,-2) (5,4) (0,0)
  4. (0,3) (2,-2) (4,4) (1,0)
答案:A (0,3) (2,-2) (4,4) (0,0)

去掉首位 DC=28,AC 序列为 3, 0, 0, -2, 0, 0, 0, 0, 4, 0…0。逐个非零值数它前面的 0:
• 3:前面 0 个 0 → (0, 3)
• -2:前面 2 个 0 → (2, -2)
• 4:前面 4 个 0 → (4, 4)
• 其后全 0 → (0, 0) = EOB
B 把第一个 run 数错(3 前面没有 0,不是 1);C 把中间零个数多算了;D 结尾必须是 (0,0),不是 (1,0)。
易错提醒:run 计的是该非零值之前紧邻的连续 0 个数,不是从块头累计;EOB 恒为 (0,0)。
