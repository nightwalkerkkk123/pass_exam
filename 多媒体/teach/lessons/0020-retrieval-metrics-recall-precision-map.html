图像检索指标:Recall / Precision / AP / mAP

检索系统吐出一串排序结果,好不好用两把尺子量:查得准不准(Precision)、该找的漏没漏(Recall)。而 AP 与 mAP,是把"排序好不好"压成一个数字的关键计算题——真题 15 分就藏在这里。

直觉类比

你用一张鞋子照片去检索,系统返回 10 张图。Precision(查准率)问的是:"你捞上来的这一网,有几条是真鱼?"——返回结果里相关的比例。Recall(查全率)问的是:"池子里所有的鱼,你捞上来几成?"——所有相关图里被找到的比例。
但一个好的检索系统,不只要"捞得准",还要把真鱼排在前面。AP 就是奖励"相关结果靠前"的分数,mAP 则是把多个查询的 AP 平均,给整个系统打一个总分。

1.核心定义

Precision 查准率
返回结果中真正相关的比例。公式 P = 相关且被检出 / 被检出总数。回答"我给你的东西里有多少是对的"。
Recall 查全率(召回率)
所有相关文档中被成功检出的比例。公式 R = 相关且被检出 / 相关文档总数。回答"该找的我找到了多少"。
AP 平均精度 (Average Precision)
单个查询的评分。在每一个"相关文档出现的位置"上算一次 Precision,再把这些 Precision 求平均。AP = (Σ 各相关位置处的 P) / 相关文档总数。相关结果越靠前,AP 越高。
mAP 平均精度均值 (mean Average Precision)
整个系统的评分。把所有查询各自的 AP 再求平均。mAP = (Σ 各查询的 AP) / 查询个数。是图像检索最常用的综合指标。

2.为什么要 AP,而不只是 P 和 R

单看一个 Precision 数字有个盲区:它不关心相关结果排在第几位。返回 [相关, 不相关, 不相关, 相关] 和 [不相关, 相关, 相关, 不相关] 可能有相同的总 Precision,但前者把相关的排在首位,用户体验完全不同。

指标关心什么范围粒度
Precision结果准不准0~1某个截断点
Recall漏没漏0~1某个截断点
AP相关结果是否靠前0~1单个查询
mAP系统整体表现0~1全部查询

核心记忆:AP 只在"相关文档出现的那些位置"上取 Precision 来平均,分母是相关文档总数——这一句是全部计算的钥匙。相关的排得越靠前,前排位置的 Precision 越高,AP 就越大。

3.逐位置计算的模板

拿到一列排好序的检索结果,按位置从上往下扫,维护两个计数器:

  • 被检出总数 = 当前扫到的位置号(rank)。
  • 相关且被检出 = 到目前为止遇到的相关文档个数。

在每一行都能算出 P = 相关数 / rank 和 R = 相关数 / 相关总数。但算 AP 时,只挑"该行是相关文档"的那几个 P 值相加,除以相关总数。

例题 · 从排序结果算 AP 与 mAP(核心,15 分)

查询 1:用一张查询图检索,数据库中与它相关的图共 5 张。系统返回前 10 个结果,标注如下(✔=相关,✗=不相关):

位置 rank12345678910
相关?✔✗✔✔✗✗✔✗✗✔
  1. 标出相关文档所在位置。从表看:相关文档出现在 rank = 1、3、4、7、10,共 5 个——正好等于相关总数 5,说明 5 张都被检出了。
  2. 逐位置累计"相关数",在相关位置算 Precision。 P = 累计相关数 / rank:
    rank相关?累计相关数Precision = 相关/rankRecall = 相关/5
    1✔11/1 = 1.0001/5 = 0.20

    3✔22/3 = 0.6672/5 = 0.40
    4✔33/4 = 0.7503/5 = 0.60
    5✗33/5 = 0.600(不计入)3/5 = 0.60
    6✗33/6 = 0.500(不计入)3/5 = 0.60
    7✔44/7 = 0.5714/5 = 0.80
    8✗44/8 = 0.500(不计入)4/5 = 0.80
    9✗44/9 = 0.444(不计入)4/5 = 0.80
    10✔55/10 = 0.5005/5 = 1.00
  3. 只把相关位置(加粗行)的 Precision 相加。 1.000 + 0.667 + 0.750 + 0.571 + 0.500 = 3.488。
  4. 除以相关文档总数 5,得到查询 1 的 AP。 AP₁ = 3.488 / 5 = 0.698。
    查询 1 的 AP₁ ≈ 0.698。(注意:分母是相关总数 5,不是返回结果数 10。)
  5. 查询 2 同法算 AP₂。设查询 2 相关文档共 3 张,返回结果中相关文档出现在 rank = 1、2、4:
    相关位置 rank累计相关数Precision
    111/1 = 1.000
    222/2 = 1.000
    433/4 = 0.750
    AP₂ = (1.000 + 1.000 + 0.750) / 3 = 2.750 / 3 ≈ 0.917。
  6. 把两个查询的 AP 求平均,得到 mAP。 mAP = (AP₁ + AP₂) / 2 = (0.698 + 0.917) / 2 = 1.615 / 2 ≈ 0.807。
    系统 mAP ≈ 0.807。查询 2 因为相关结果几乎全排在最前,AP 高达 0.917,把整体拉了上来——这正是 AP 奖励"靠前"的体现。

4.易错点清单

  • AP 分母搞错:分母永远是相关文档总数,不是返回条数,也不是相关位置的个数(除非相关文档全被检出,二者才相等)。
  • 把不相关位置的 P 也加进去:AP 只累加"相关位置"处的 Precision。不相关行的 P 只用来理解曲线,不进求和。
  • 没被检出的相关文档:若某个相关文档没出现在返回列表里,它对应的 Precision 记为 0,但分母仍是相关总数——这会拉低 AP。
  • mAP 和 AP 混用:AP 是单查询、mAP 是多查询平均。题目问哪个,分母就对应哪个。
自测题 · 算 AP

某查询相关文档共 4 张。系统返回前 8 个结果,相关文档出现在 rank = 2、3、5、8(其余为不相关)。该查询的 AP 最接近:

  1. 0.500
  2. 0.567
  3. 0.633
  4. 0.750
答案:B ≈ 0.567

只在相关位置算 Precision:
rank 2 → 累计相关 1,P = 1/2 = 0.500
rank 3 → 累计相关 2,P = 2/3 = 0.667
rank 5 → 累计相关 3,P = 3/5 = 0.600
rank 8 → 累计相关 4,P = 4/8 = 0.500
求和 0.500 + 0.667 + 0.600 + 0.500 = 2.267,除以相关总数 4:AP = 2.267 / 4 ≈ 0.567。
陷阱:D 选项 0.750 是"只看 rank 3 那个值"或误用分母的结果;A 0.500 是漏加了 rank 3 的贡献。记住:相关位置逐个算 P,求和再除以相关总数 4。
