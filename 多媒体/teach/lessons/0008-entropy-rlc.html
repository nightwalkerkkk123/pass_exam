信息熵与游程编码 RLC

压缩的本质,是把"你早就猜得到的东西"少写几个字。这一节先量化"意外程度"——信息熵,再看最朴素的一种去冗余手段:把连着重复的符号折叠成"个数 + 符号"。

直觉类比

天气预报说"明天太阳照常升起",你几乎没获得任何信息——因为它本来就必然发生。可要是说"明天下陨石",这条消息信息量极大,因为它太意外了。越不可能发生的事,一旦发生,携带的信息量越大——这就是自信息。把一个信源里所有符号的自信息按概率加权平均,就得到信息熵:它衡量"平均每个符号有多让人意外",也正好是无损压缩每个符号平均最少要花的比特数。

1.四个核心定义

冗余 (Redundancy)
数据中"可以被预测、因而无需重复存储"的部分。压缩的本质就是去除冗余:时间冗余(视频帧间)、空间冗余(相邻像素)、频谱冗余(颜色分量间)、心理视觉冗余(人眼不敏感的细节)。RLC 攻击的是"连续重复"这种最直白的空间/时间冗余。
自信息 (Self-information)
单个符号 sᵢ 携带的信息量,定义为 I(sᵢ) = −log₂ pᵢ,单位 bit。概率 pᵢ 越小,自信息越大;必然事件 p=1 时自信息为 0。
信息熵 (Entropy)
信源 S 中每个符号的平均信息量:η = Σ pᵢ · I(sᵢ) = −Σ pᵢ · log₂ pᵢ,单位 bit/symbol。等概率分布时熵最大;某符号概率为 1 时熵为 0。
游程编码 (Run-Length Coding, RLC)
一种无损压缩:把一串连续重复的符号,替换成"重复次数 + 符号"这一个二元组。适合大片相同的数据(如传真、纯色区域)。

2.自信息随概率怎么变

把 I = −log₂ p 代几个值进去,直觉马上清晰——概率减半,自信息就多 1 bit:

概率 p自信息 −log₂ p (bit)含义
10必然事件,毫无信息
1/21一次公平抛硬币
1/42更少见,信息更多
1/83越稀有越"值钱"
→ 0→ ∞几乎不可能,一旦发生信息量爆炸

3.为什么熵是无损压缩的下界

熵不是随便定义的数字,它是一道物理下限:任何无损编码,平均每个符号的码长 l̄ 都不可能低于熵 η,即 η ≤ l̄。

道理在于:要想缩短平均码长,就得给高概率符号短码、低概率符号长码。但码字不能随便短——任意无损前缀码受 Kraft 不等式约束 Σ 2^(−lᵢ) ≤ 1。在这个约束下让加权平均码长 Σ pᵢ·lᵢ 最小,最优解恰好是 lᵢ = −log₂ pᵢ,此时平均码长正好等于熵。

可现实里码长必须是整数(Huffman 就是逐符号给整数位码),没法精确取到 −log₂ pᵢ 这种小数,所以只能 l̄ ≥ η,并且最优时满足 η ≤ l̄ < η + 1。这也解释了后面为什么算术编码(对整条消息编码、能逼近熵)常常胜过 Huffman。一句话:熵 = 消除每个符号平均不确定性所需的最少比特数。

例题 A · 计算信息熵

信源 S = {a, b, c},概率 P = {0.5, 0.25, 0.25}。求熵 η。

  1. 写出公式:η = −Σ pᵢ · log₂ pᵢ。
  2. 逐项算自信息:−log₂ 0.5 = 1,−log₂ 0.25 = 2,−log₂ 0.25 = 2。
  3. 按概率加权求和:η = 0.5×1 + 0.25×2 + 0.25×2。
  4. 算出:η = 0.5 + 0.5 + 0.5 = 1.5。
结果: η = 1.5 bit/symbol。作对比:若用等长定长码,3 个符号要 ⌈log₂ 3⌉ = 2 bit/symbol——熵告诉我们理论上平均只需 1.5 bit,还有压缩空间。
例题 B · 游程编码 RLC

对字符串 aaabbc 做 RLC 编码,并算压缩效果。

  1. 从左到右扫描,统计每段连续相同符号的长度:aaa(3 个 a)、bb(2 个 b)、c(1 个 c)。
  2. 把每段写成 (次数, 符号):(3,a) (2,b) (1,c)。
  3. 对比长度:原始 6 个符号 → 编码后 3 个二元组。
结果: aaabbc → (3,a)(2,b)(1,c)。
提醒:RLC 只在重复多时才划算。若是 abcabc 这种毫无连续重复的串,RLC 反而会把 6 个符号变成 6 个二元组,越"压"越大——这正是它的边界情况。
自测题 1

字母表 S = {a, b, c, d},概率 P = {1/2, 1/4, 1/8, 1/8}。该信源的熵 η 是多少 bit/symbol?

  1. 2.00
  2. 1.75
  3. 1.50
  4. 2.25
答案:B 1.75

自信息分别为 −log₂(1/2)=1、−log₂(1/4)=2、−log₂(1/8)=3、−log₂(1/8)=3。
加权求和:η = 1/2×1 + 1/4×2 + 1/8×3 + 1/8×3 = 0.5 + 0.5 + 0.375 + 0.375 = 1.75。
解题套路:凡是 1/2ⁿ 这种概率,自信息就是整数 n,不用查对数表,心算即可。

自测题 2

对字符串 WWWWWBBWBBB 做游程编码,正确结果是?

  1. (5,W)(2,B)(1,W)(3,B)
  2. (4,W)(2,B)(1,W)(3,B)
  3. (5,W)(3,B)(3,B)
  4. (W,5)(B,2)(W,1)(B,3)
答案:A (5,W)(2,B)(1,W)(3,B)

分段:WWWWW=5 个 W、BB=2 个 B、W=1 个 W、BBB=3 个 B。合起来 (5,W)(2,B)(1,W)(3,B)。
B 项:数错了,W 是 4 个不是 5 个;C 项:漏掉了中间那个单独的 W,把两段 B 错误合并;D 项:本题约定"次数在前、符号在后",顺序写反。
易错点:中间夹的单个符号也必须单独成一段,不能和前后同类符号合并。
