## 2. 横向知识串联

### 2.1 主题【压缩】 串联 L06 / L07 / L08 / L09 / L10

| 讲次 | 关键概念 | 公式 / 步骤 | 与其他讲次的关联 |
|------|----------|-------------|------------------|
| L06 | 熵 / Huffman / 算术编码 | `η = -Σ pi·log2(pi)`, `η ≤ l̄ < η+1` | L08 JPEG 用 Huffman，L07 率失真给压缩理论下界 |
| L07 | 量化 / DCT / PSNR | `F̂(u,v) = round(F/Q)`, `PSNR = 10·log10(x²peak/MSE)` | 量化是 L08 JPEG 主要失真源；DCT 是 L08/L10 共用工具 |
| L08 | JPEG 流水线 | 分块 → DCT → 量化 → ZigZag → DPCM(DC)/RLC(AC) → Huffman | 是 L06 (熵编码) + L07 (DCT+量化) 的工程实例 |
| L09 | 时间冗余 / 运动补偿 / MAD | `MAD = (1/N²)Σ|T-R|`, MV 搜索复杂度 | 是 L10 视频标准的前置；用 L08 的 I 帧作参考 |
| L10 | H.261 / MPEG I/P/B 帧 | I:P:B ≈ 7:1 / 20:1 / 50:1 | 综合 L08 帧内 + L09 帧间预测 |

| 讲次 | 关键概念 | 公式 / 步骤 | 与其他讲次的关联 |

|------|----------|-------------|------------------|

## 3. 终极易错清单 (Top 30)

按易错程度排序（前 10 为"必扣分"，后 20 为"易丢分"）。

### 3.1 必扣分 TOP 10

| # | 易错点 | 正确 vs 错误 | 出现讲次 | 避坑提示 |
|---|--------|--------------|----------|----------|
| 1 | **8-bit 灰度图数据量** | 正确：像素数 × 1 字节 (B)；错误：× 8 bit (混淆 b 与 B) | L02 | 单位永远先换算到 Byte，再 × 像素数 |
| 2 | **YCbCr 4:2:0 子采样后总像素数** | 正确：每 2×2 块 = Y(4)+Cb(1)+Cr(1) = 6 字节；平均 1.5 B/px；错误：仍按 3 B/px 算 | L03/L04/L08 | 4:2:0 = 水平+垂直都 2:1 降采样 |
| 3 | **量化是 JPEG 有损的真正来源** | 正确：DCT 本身无损；错误：以为是 DCT 丢信息 | L07/L08 | "DCT 可逆，量化不可逆"——口诀 |
| 4 | **PSNR 公式的对数底与系数** | 正确：`10·log10(x²peak/MSE)`；错误：`20·log10`（电压系数）、`ln`（自然对数） | L07 | 功率用 10，电压用 20；永远是 10·log10 |
| 5 | **MPEG-4 不等于 H.264** | 正确：H.264 = MPEG-4 Part 10 (AVC)；MPEG-4 是协议族；错误：把 MPEG-4 Visual 当成 H.264 | L10 | 记住对应表：H.262=MPEG-2，H.264=MPEG-4 AVC，H.265=MPEG-H HEVC |
| 6 | **H.261 的宏块大小** | 正确：亮度 N=16，色度 4:2:0 下 N=8；错误：16×16 色度（混淆色度不降采样） | L09/L10 | 宏块 = 16×16 Y + 8×8 Cb + 8×8 Cr（4:2:0） |
| 7 | **CNN 卷积输出尺寸公式中 padding 的"双边"性** | 正确：`(N + 2·padding - F)/stride + 1`；错误：只 + padding（单边） | L12 | 公式中的 2 表示左右（或上下）各填充 padding |
| 8 | **Harris 角点不具有尺度不变性** | 正确：Harris 单尺度；错误：以为 Harris 抗缩放；Harris-Laplace 才有 | L11 | 抗尺度的检测子：SIFT/SURF/MSER/Harris-Laplace |
| 9 | **H.261 P 帧的参考帧是"已解码的帧"** | 正确：参考帧必须用已解码（重建）的帧；错误：以为是原始帧 | L09/L10 | DPCM 类系统共性——防 drift |
| 10 | **Huffman 平均码长 ≥ 熵** | 正确：`η ≤ l̄ < η+1`；错误：以为 `l̄ = η`（仅当所有 `-log2(pi)` 整数时才相等） | L06 | 整数约束 + Kraft 不等式 → 平均码长必 ≥ 熵 |

### 3.2 易丢分 11-30

| # | 易错点 | 正确 vs 错误 | 出现讲次 | 避坑提示 |
|---|--------|--------------|----------|----------|
| 11 | **灰度公式 Y = 0.299R + 0.587G + 0.114B 的 G 最大** | G 权重 0.587 最大（人眼对绿最敏感）；不是平均 1/3 | L02/L03 | BT.601 标准，反映人眼视锥细胞敏感度 |
| 12 | **Nyquist 假频公式** | 假频 = \|f_sample − f_true\|（采样率 < 2·f_max 时）；不是简单的 f_true − f_sample | L05 | 注意绝对值 |
| 13 | **PCM / DPCM / DM 的关系** | DM 是 DPCM 的极端简化（1-bit 方向量化）；不是简单的"差分" | L05 | DM = DPCM 的 1-bit 版本，ADPCM = DPCM 的自适应版本 |
| 14 | **H.264 比 MPEG-2 节省 50% 码率** | 不是 100%，是约 50%（同画质下） | L10 | 记住"半码率" |
| 15 | **MSE / SNR / PSNR 单位** | SNR/PSNR 单位都是 dB，且都用 10·log10（功率）；不是 20·log10（那是电压/电流） | L07 | "功率用 10" |
| 16 | **ZigZag 扫描顺序** | 从 (0,0) 到 (7,7)，低频→高频，Z 字形；不是行优先或列优先 | L08 | 手画一遍：0,0→0,1→1,0→2,0→1,1→0,2 |
| 17 | **RLC 中 "run" 是之前零的个数** | run 表示"这个非零值前有多少个零"；不是这个值之后的零 | L08 | 例子：(0,6)(0,-1)(0,-1)(1,-1)(3,-1)(2,1)(0,0) |
| 18 | **qf 与质量的关系** | qf 大 → scaling 小 → Q 小 → 量化细 → 质量高；qf=50 时 scaling=1 | L08 | "qf 大 = 质量高" |
| 19 | **4:2:0 与 4:1:1 的几何差异** | 两者都是 1.5 B/px，但 4:2:0 二维各 1/2，4:1:1 仅水平 1/4 | L04 | "4:2:0 二维半、4:1:1 水平 1/4" |
| 20 | **SIFT 描述子维度** | 4×4 分块 × 8 方向 = 128 维；不是 4×8=32 维，也不是 128 byte | L11 | 维度 = 分块数 × 方向数 |
| 21 | **MSER 是唯一具有仿射不变性的检测子** | MSER：Yes；SIFT/SURF/Harris-Laplace：No（仅尺度不变） | L11 | "唯一抗仿射" |
| 22 | **ResNet 解决的"深度退化"问题** | 不是梯度消失，而是训练误差随深度增大而增大 | L12 | 退化 ≠ 梯度消失 |
| 23 | **CNN 池化层无参数** | 池化是固定运算（Max/Avg）；卷积才有可学习权重 | L12 | 池化 = 零参数 |
| 24 | **Dropout 训练 vs 测试** | 训练时随机失活；测试时全部使用（inverted dropout 训练时已除以 (1-p)） | L12 | 测试关闭 |
| 25 | **YUV 与 YCbCr 的区别** | YCbCr 是 YUV 的 scale+offset 数字版本；不是完全等同 | L03 | "YCbCr 是 YUV 的数字版" |
| 26 | **24-bit RGB vs 32-bit RGBA** | 32-bit = 24-bit RGB + 8-bit alpha 通道；不是更精细的 RGB | L02 | 多 1 字节 alpha |
| 27 | **H.263 引入半像素精度** | 半像素 + 双线性插值是 H.263 起，不是 H.261 | L09 | "H.263 起" |
| 28 | **运动估计的复杂度对比** | 顺序 O(p²N²)、对数 O(log p·N²)、分层最少；不是反过来 | L09 | "分层最省" |
| 29 | **B 帧不能被其他帧作为参考** | 传统 MPEG-1/2/4 中 B 帧不作为参考（H.264 起才支持 B 参考） | L10 | B 帧 = 不可参考 |
| 30 | **Y 与 luminance 的等价性** | Y 表亮度（luminance），与 luminance 等价；不要混淆 brightness（视觉感受） | L03 | "Y = luminance" |

## 4. 模拟期末试卷 (Mock Exam · 100 分)

### 4.1 选择题（每题 1 分，共 10 分）

**【选-1】** 一幅 1920×1080 的 24-bit RGB 图像，转为 YCbCr 4:2:0 后每像素字节数为？
- A. 3 B/px
- B. 2 B/px
- C. 1.5 B/px
- D. 1 B/px
- **答案：** C
- **解析：** 4:2:0 每 2×2 块 = Y(4)+Cb(1)+Cr(1) = 6 B / 4 = 1.5 B/px。
- **涉及：** L03/L04/L08

**【选-2】** 22.05 kHz 声音，按奈奎斯特定理最低采样率应为？
- A. 22.05 kHz
- B. 33.075 kHz
- C. 44.1 kHz
- D. 88.2 kHz
- **答案：** C
- **解析：** f_sample ≥ 2×f_max = 2×22.05 = 44.1 kHz（CD 标准）。
- **涉及：** L05

**【选-3】** Huffman 编码满足 η ≤ l̄ < η+1，若某信源概率为 {1/2, 1/4, 1/8, 1/8}，其熵为？
- A. 1.5 bit/symbol
- B. 1.75 bit/symbol
- C. 2.0 bit/symbol
- D. 2.5 bit/symbol
- **答案：** B
- **解析：** η = 1/2·1 + 1/4·2 + 1/8·3 + 1/8·3 = 0.5+0.5+0.375+0.375 = 1.75。
- **涉及：** L06

**【选-4】** JPEG 标准中，有损压缩的主要来源是？
- A. RGB→YCbCr 变换
- B. 8×8 DCT 变换
- C. 均匀标量量化
- D. Huffman 编码
- **答案：** C
- **解析：** DCT/RGB变换/Huffman 都可逆；只有量化 `F̂ = round(F/Q)` 不可逆。
- **涉及：** L07/L08

**【选-5】** H.261 中亮度宏块大小 N_Y 与 4:2:0 下色度宏块大小 N_C 分别是？
- A. N_Y=8, N_C=8
- B. N_Y=16, N_C=16
- C. N_Y=16, N_C=8
- D. N_Y=32, N_C=16
- **答案：** C
- **解析：** 亮度 16×16；4:2:0 色度水平/垂直各 2:1 → 8×8。
- **涉及：** L09/L10

**【选-6】** 关于 SIFT 检测子使用 DoG 近似 LoG 的原因，正确的是？
- A. DoG 精度更高
- B. DoG 计算更简单
- C. DoG 具有仿射不变性而 LoG 没有
- D. DoG 在离散像素上更容易实现
- **答案：** B
- **解析：** DoG = 两个高斯图像相减，无需二阶微分，计算量大幅降低。
- **涉及：** L11

**【选-7】** 对 32×32 输入用 10 个 5×5 卷积核、stride=1、padding=2，输出特征图尺寸？
- A. 32×32×10
- B. 28×28×10
- C. 30×30×10
- D. 32×32×5
- **答案：** A
- **解析：** (32 + 2×2 − 5)/1 + 1 = 32；深度 = 卷积核数 = 10。
- **涉及：** L12

**【选-8】** 已知 qf=25，按 JPEG 公式 scaling_factor = ?
- A. 0.5
- B. 1.0
- C. 2.0
- D. 3.0
- **答案：** C
- **解析：** qf<50 时 scaling = 50/qf = 50/25 = 2.0。
- **涉及：** L08

**【选-9】** ResNet 残差网络主要解决的问题是？
- A. 梯度消失
- B. 深度退化（训练误差随深度增加而增大）
- C. 过拟合
- D. 数据不足
- **答案：** B
- **解析：** 残差结构让 F(x)=0 成为恒等映射，易学，可训练超深网络；不是解决梯度消失。
- **涉及：** L12

**【选-10】** MPEG-1 I/P/B 帧的典型压缩比约为？
- A. 7:1 / 20:1 / 50:1
- B. 10:1 / 15:1 / 20:1
- C. 5:1 / 10:1 / 15:1
- D. 都是 7:1
- **答案：** A
- **解析：** I 帧类似 JPEG (7:1)，P 帧 20:1，B 帧 50:1（双向预测最高）。
- **涉及：** L10

### 4.3 计算题（每题 6 分，共 30 分）

**【计-1】** 一幅 256×256 的 8-bit 灰度图像，压缩前后 MSE = 100，求 PSNR (dB)。
- **答案：** PSNR ≈ 28.13 dB
- **解析：** x_peak = 255, x²_peak = 65025；PSNR = 10·log10(65025/100) = 10·log10(650.25) ≈ 28.13 dB。
- **涉及：** L07

**【计-2】** 一段 1920×1080、30fps、4:2:0 的 HD 视频，求：
(1) 单帧数据量；
(2) 1 秒数据量 (MB)；
(3) 与原始 RGB (3 B/px) 相比压缩比。
- **答案：**
(1) 1920×1080×1.5 ≈ 3.11 MB
(2) 3.11 × 30 ≈ 93.3 MB/s
(3) RGB 原始 = 1920×1080×3×30 ≈ 186.6 MB/s；压缩比 = 186.6/93.3 = 2:1
- **解析：** 4:2:0 子采样已经带来 2:1 的视觉无损压缩。
- **涉及：** L04

**【计-3】** 用 Huffman 编码对符号 A=15, B=7, C=6, D=6, E=5（共 39 字符）编码，求：
(1) Huffman 平均码长 l̄；
(2) 理想熵 η；
(3) 与 8-bit 定长编码的压缩比 CR。
- **答案：**
(1) l̄ = (1×15 + 3×7 + 3×6 + 3×6 + 3×5)/39 = 87/39 ≈ 2.23 bit/symbol
(2) η = -Σ pi·log2(pi) ≈ 2.19 bit/symbol
(3) CR = 8/2.23 ≈ 3.59
- **解析：** Huffman 树构造使 A 用 1 bit，其他用 3 bit。
- **涉及：** L06

**【计-4】** 对一个 224×224×3 的输入图像做卷积：第一层用 64 个 3×3 卷积核 (stride=1, padding=1)；第二层用 128 个 3×3 卷积核 (stride=1, padding=1)；第三层用 2×2 MaxPool (stride=2)。求每层输出尺寸和参数量。
- **答案：**
- C1 输出 = (224+2-3)/1+1 = 224 → **224×224×64**；参数 = (3×3×3+1)×64 = **1,792**
- C2 输出 = (224+2-3)/1+1 = 224 → **224×224×128**；参数 = (3×3×64+1)×128 = **73,856**
- P3 输出 = (224-2)/2+1 = 112 → **112×112×128**；参数 = **0**
- **解析：** 池化层无参数；每个卷积核深度 = 输入通道数。
- **涉及：** L12

**【计-5】** 一段 MPEG-1 视频采用 IBBPBBPBBP 序列（10 帧：1 I、3 P、6 B），I/P/B 压缩比为 7:1/20:1/50:1。求：
(1) 每 10 帧的"加权平均大小"；
(2) 平均压缩比。
- **答案：**
(1) 设原始 1 帧大小 = 1；则 I = 1/7, P = 1/20, B = 1/50；
总大小 = 1/7 + 3/20 + 6/50 = 0.1429 + 0.15 + 0.12 = 0.4129
(2) 平均压缩比 = 10/0.4129 ≈ **24.2:1**（与讲义 27:1 量级一致）
- **解析：** 帧大小与压缩比成反比；平均压缩比 = 总原始大小 / 总压缩后大小。
- **涉及：** L10

### 4.4 简答题（每题 5 分，共 20 分）

**【简-1】** 简述 RGB 不适合直接压缩而 YCbCr 适合压缩的 3 个原因。
- **答题要点：**
1. **人眼特性**：对亮度 (Y) 变化敏感，对色度 (Cb/Cr) 不敏感 → 可对色度粗处理；
2. **解相关**：YCbCr 各通道相关性远低于 RGB → 压缩效率高；
3. **可分离压缩**：可对 Cb/Cr 下采样 (4:2:0、4:2:2) 节省带宽，视觉影响小；
4. **兼容电视系统**：源自模拟电视（PAL 用 YUV、NTSC 用 YIQ），数字时代统一为 YCbCr。
- **涉及：** L03/L04/L08

**【简-2】** 简述 Huffman 编码的构造步骤，并解释为什么 Huffman 码满足"唯一前缀性质"。
- **答题要点：**
1. 把所有符号按频次从大到小排序成表；
2. 取出频次最小的两个节点作为左右子树，新节点频次 = 两子节点之和；
3. 新节点插入表中保持有序；
4. 重复 2-3 直至表只剩一个节点（根）；
5. 对每条边赋 0（左）/ 1（右），从根到叶的路径即码字。
- **唯一前缀性原因**：码字对应二叉树的叶子节点，叶子无子女，因此任何码字都不是其他码字的前缀（前缀性），保证解码时即时匹配无歧义。
- **涉及：** L06

**【简-3】** 简述 JPEG 编码的 5 个主要步骤，并指出哪一步是不可逆的，为什么。
- **答题要点：**
1. **RGB→YCbCr + 4:2:0 子采样**（可逆 / 视觉无损）：颜色空间解相关；
2. **分块 8×8 + 2D DCT**（无损可逆）：把空间域变换到频率域，能量集中；
3. **均匀标量量化 `F̂(u,v) = round(F(u,v)/Q(u,v))`**（**不可逆**，是有损主因）：除以步长并取整，丢失精度；
4. **ZigZag 扫描 + DPCM(DC)/RLC(AC)**（无损）：把零聚在一起便于编码；
5. **Huffman 熵编码**（无损）：去除统计冗余。
- **涉及：** L07/L08

**【简-4】** 试比较 AlexNet、VGG、GoogleNet、ResNet 四种典型 CNN 的核心创新与演进动机。
- **答题要点：**

| 网络 | 年份 | 层数 | 核心创新 | 演进动机 |
|------|------|------|----------|----------|
| AlexNet | 2012 | 8 | ReLU + Dropout + Data Aug + 双 GPU | 开启深度学习时代（top-5 错误率从 ~26% → 16%） |
| VGG | 2014 | 16-19 | 多个 3×3 小卷积核替代大卷积核 | 证明"变深"提升性能（top-5 16% → 7.3%） |
| GoogleNet | 2014 | 22 | Inception 多尺度模块 + 1×1 降维 + 辅助分类器 | 解决 VGG 参数爆炸（138M → 5M，1/12 AlexNet） |
| ResNet | 2015 | 152 | 残差连接 `y=F(x)+x` 解决深度退化 | 突破深度瓶颈（top-5 6.7% → 3.57%） |

- **演进主线**：深度↑ → 参数↓ → 解决退化 → 性能持续提升。
- **涉及：** L12

### 4.5 综合应用题（每题 10 分，共 20 分）

**【综-1】** 给定一个远程视频监控系统的需求：分辨率 640×480、25 fps、低码率 (< 64 kbps)、网络丢包率高、运动缓慢。请设计压缩方案，要求说明：
(1) 色度采样方案及理由；
(2) 帧类型选择及 I 帧间隔；
(3) 运动估计方法选择及理由；
(4) 估算原始数据量与压缩后数据量。

- **答题要点：**
1. **色度采样**：4:2:0（1.5 B/px），适合监控场景（人眼对色度不敏感 + 节省 50% 带宽）。
2. **帧类型**：I 帧 + P 帧（不用 B 帧——增加编码延迟且监控不要求双向预测）。I 帧间隔约 1 秒（抗丢包漂移）。
3. **运动估计**：**分层 (Hierarchical) 搜索**——复杂度最低（0.51×10⁹ vs 顺序 29.89×10⁹ OPS），适合缓慢运动场景，节省算力。
4. **数据量估算**：
   - 原始：640×480×1.5×25 ≈ 11.5 MB/s ≈ 92 Mbps；
   - 压缩后 64 kbps；
   - 压缩比 ≈ 92 Mbps / 64 kbps ≈ **1440:1**（需 H.264/H.265 强压缩）；
   - 实际方案：可采用 H.264 Baseline + 分层搜索 + 适度 GOP，2-3 Mbps 即可达监控可用画质。
- **涉及：** L04/L09/L10

**【综-2】** 设计一个基于内容的商品图像检索系统（百万级商品库），要求用户上传一张商品照片，系统在 1 秒内返回相似商品。请描述：
(1) 离线建库流程；
(2) 在线检索流程；
(3) 关键技术挑战与解决方案。

- **答题要点：**
1. **离线建库流程**：
   - 商品库图像预处理（去噪、统一尺寸如 256×256）；
   - 特征提取：局部特征 (SIFT/SURF) + 描述子（128 维）；
   - 特征聚合：Bag-of-Words (BoW) 聚类 (K-means, K=1000) → 视觉词袋；
   - 高维索引：倒排索引（基于视觉词）+ LSH 哈希；
   - 存储特征库与索引。
2. **在线检索流程**：
   - 用户上传查询图 → 预处理 → 提取 SIFT 特征；
   - 特征量化：SIFT 描述子映射到最近视觉词（NN 搜索）；
   - 查询向量构造：直方图表示；
   - 倒排索引查询 → 候选集；
   - 重排序：RANSAC 几何一致性验证；
   - 返回 top-K 最相似商品。
3. **关键技术挑战**：
   - **语义鸿沟**：相同商品外观差异大 → 用 CNN 深度特征替代 SIFT（迁移学习）；
   - **高维索引**：128 维 + 百万库，KNN 太慢 → 用 ANN (近似最近邻) / LSH；
   - **可扩展性**：商品更新时增量构建索引；
   - **效率 vs 精度**：BoW + 倒排索引牺牲部分精度换速度。
- **涉及：** L11

## 5. 最后一周冲刺清单

### 5.1 五个"必须背下来"的核心公式/数字

| # | 公式/数字 | 含义 |
|---|-----------|------|
| 1 | **PSNR = 10·log10(255²/MSE)** (dB) | 8-bit 灰度图失真度量；MSE=100 → PSNR≈28 dB |
| 2 | **SQNR = 6.02 × N** (dB) | N 位均匀量化信噪比；16 bit ≈ 96 dB (CD 音质) |
| 3 | **f_sample ≥ 2·f_max** | Nyquist 定理；假频 = \|f_sample − f_true\| |
| 4 | **CR = B0/B1** | 压缩比；η ≤ l̄ < η+1（Huffman 界） |
| 5 | **输出尺寸 = (N + 2·padding − F)/stride + 1** | CNN 卷积输出公式；参数 = (F·F·C + 1)·K |

### 5.2 五个"必须能推导"的过程

| # | 过程 | 关键步骤 |
|---|------|----------|
| 1 | **JPEG 编码流程** | RGB→YCbCr(4:2:0) → 8×8 DCT → 量化 (`F̂=round(F/Q)`) → ZigZag → DC 用 DPCM / AC 用 RLC → Huffman |
| 2 | **Huffman 编码步骤** | 频次排序 → 取最小两个合并 → 新节点插入 → 重复直到根 → 左 0 右 1 赋码 |
| 3 | **帧间预测（运动补偿）原理** | 当前帧宏块 → 在参考帧 (2p+1)² 窗口搜索 → 找 MV 使 MAD 最小 → 用 MV 取预测块 → 编码残差 |
| 4 | **H.261 P 帧编码** | 宏块 → ME 找 MV → MC 取预测 → 残差 DCT→量化→ZigZag→Huffman；参考帧必须是已解码帧 |
| 5 | **算术编码步骤** | low=0, high=1, range=1 → 每符号缩窄区间 `[low+range·rL, low+range·rH)` → 输出 `low ≤ code < high` 的最短二进制 |

### 5.3 五个"必须能对比"的对比表

> 详细对比见第 2 部分"横向知识串联"。

| # | 对比主题 | 关键差异 |
|---|----------|----------|
| 1 | **压缩主题** | 无损 (Huffman/LZW) vs 有损 (JPEG/MPEG)；JPEG vs JPEG2000；H.261 vs H.264 |
| 2 | **颜色空间** | RGB vs HSV vs YCbCr；YUV vs YCbCr（数字版） |
| 3 | **质量度量** | SNR vs PSNR；MSE vs MAD；距离 vs 相似度 |
| 4 | **预测方式** | 帧内预测 vs 帧间预测；前向 vs 双向；DPCM vs ADPCM |
| 5 | **特征提取** | 全局特征 vs 局部特征；Harris vs SIFT/SURF/MSER；手工 vs CNN |
