II.5 个必须背下来的核心公式

下面 5 张卡片是期末最容易出计算题的公式。卡片内容一字不差背到能写出为止。

公式 1 · PSNR(峰值信噪比)

PSNR = 10·log10(255² / MSE)
8-bit 灰度;MSE=100 → PSNR ≈ 28 dB;≥ 40 dB 视觉无损

公式 2 · SQNR(量化信噪比)

SQNR = 6.02 × N (dB)
16 bit ≈ 96 dB(CD 音质);每 +1 bit ≈ +6 dB

公式 3 · Nyquist(奈奎斯特采样)

f_sample ≥ 2·f_max
假频 = |f_sample − f_true|;电话 8 kHz 可还原 4 kHz

公式 4 · 压缩比

CR = B0 / B1
Huffman 界:η ≤ l̄ < η+1(平均码长 ≥ 熵)

公式 5 · CNN 输出尺寸

(N + 2·padding − F) / stride + 1
参数 = (F·F·C + 1)·K;N=输入边长,F=卷积核

III.5 个必须能推导的过程

流程题是综合应用题的重头戏。下面 5 个过程能默写到能讲明白为什么这么做的程度。

过程 1 · JPEG 编码

RGB → YCbCr(4:2:0 子采样) → 8×8 分块 → DCT → 量化 F̂=round(F/Q) → ZigZag → DC 用 DPCM、AC 用 RLC → Huffman 编码

过程 2 · Huffman 编码

频次排序 → 取最小两个合并为新节点 → 插入新节点 → 重复直到根 → 左 0 右 1 赋码;平均码长满足 η ≤ l̄ < η+1

过程 3 · 帧间预测 (MC)

当前宏块 → 参考帧 (2p+1)² 窗口搜索 → 计算 MAD → 找 MV 使 MAD 最小 → 取预测 → 编码残差 d = T − R

过程 4 · H.261 P 帧编码

宏块 16×16 → ME 找 MV → MC 取预测 → 残差 4 个 8×8 Y + 2 个 8×8 色度 DCT → 量化 → ZigZag → Huffman;参考帧必须是已解码帧

过程 5 · 算术编码

low=0, high=1, range=1 → 每符号缩窄区间 range·=p, low+=range_start → 输出满足 low ≤ code < high 的最短二进制

IV.5 个必须能对比的主题

对比题常以"维度 × 优劣"两栏形式出。下面 5 张对比表是高频考点。

对比 1 · 压缩主题

质量完全可还原有失真,PSNR 度量

V.5 个必须能举例的实例

抽象概念记不住就记典型数字。下面 5 个数字钩子是高频出现的"必举例"。

例 1 · Lena 512² JPEG

原始 512×512×1 = 256 KB;典型 JPEG 10-20:1 → 12-25 KB;qf=75 时约 30-50 KB

例 2 · H.264 vs MPEG-2

同画质下 H.264 码率约为 MPEG-2 的 50%(节省一半码率)

例 3 · 1080p@30fps 4:2:0

单帧 1920×1080×1.5 ≈ 3.11 MB;每秒 93 MB (≈744 Mbps);H.264 压缩到 2-5 Mbps

例 4 · 经典 CNN 参数对比

AlexNet 60M → VGG 138M → GoogleNet 5M(1/12) → ResNet 25M

例 5 · 电话 8 kHz / 8 bit

8 kHz × 8 bit = 64 kbps = DS0 标准;SQNR ≈ 48 dB 满足语音可懂度

VI.5 大心法 (Five Mental Models)

复习方法论。心法不对,再多时间也事倍功半。下面 5 条是考场答题的总指挥原则。

心法 1 · 算为王

L02/03/04/05/06/07/08/12 全部必出计算题。练习时 先列公式 → 代入数字 → 验算单位(b vs B, KB vs KiB, Mbps vs MB/s)。

心法 2 · 流为纲

JPEG 5 步、H.261 I/P 帧、CNN 三大思想、SIFT 5 步——任何"流程题"按 分块 → 变换 → 量化 → 编码 四步即可得分。

心法 3 · 比为剑

高频对比题(Huffman vs 算术、YUV vs YCbCr、I vs P vs B、AlexNet vs VGG vs GoogleNet vs ResNet)按 维度 × 优劣 两栏对比表答。

心法 4 · 错为镜

TOP 30 易错清单 考前过 3 遍,尤其是必扣分 TOP 10(数据量单位、PSNR 系数、量化是 JPEG 主因、Huffman 界、H.261 宏块等)。

心法 5 · 例为锚

抽象概念记不住就记典型数字——Lena 512×512、MPEG 7:1/20:1/50:1、SQNR ≈ 6N dB、H.264 比 MPEG-2 省 50% 码率。数字是记忆的钩子。

VII.答题卡格式建议

格式规范 = 多拿 5-10 分。下面是各类题型的答题模板。

1 · 选择题 / 填空题(每题 1-2 分)

  • 时间预算:每题 30-60 秒,5 分钟一组做完填答题卡
  • 答题格式:用 2B 铅笔涂黑,确认题号对应
  • 策略:先易后难,不会的标星号回头再想,不空题

2 · 计算题(每题 5-10 分,通常 2-3 道)

  1. 列已知:写出题目给出的符号和数值(用 P = ? 形式)
  2. 列公式:写出核心公式,标注每个符号的含义
  3. 代数字:代入数值,每一步写明单位
  4. 得结果:写明数值 + 单位,保留 2-3 位有效数字
  5. 验算:用数量级检查合理性(128 KB vs 1.28 GB)

3 · 简答题(每题 5-8 分,通常 3-4 道)

  • 答题结构:定义 + 特点 + 应用 + 优缺点 / 异同(对比题)
  • 长度:200-400 字,3-5 个要点,每个要点 1-2 句
  • 加分技巧:能用公式/数字/图示表达的绝不只写字

4 · 论述题 / 综合应用题(每题 10-15 分,通常 1-2 道)

  1. 总-分-总:开头一句概括核心论点,中间分点论述,结尾总结
  2. 跨讲次串联:综合题最爱考"压缩/颜色/质量/预测/变换"七主题
  3. 举数字:每个抽象概念配一个具体数字(心法 5)
  4. 画流程图:时间充裕时画 ASCII 流程图(JPEG 5 步、CNN 三大思想)

5 · 时间分配建议(100 分卷 · 90 分钟)

题型题数分值总时长单题时长
选择 / 填空15-2030 分20 分钟60-90 秒
计算题2-320 分25 分钟8-12 分钟
简答题3-425 分25 分钟6-8 分钟
综合应用1-225 分20 分钟10-15 分钟

口诀:填空选择题 25%、计算 25%、简答 25%、综合 25%——各类题型分配均衡。
