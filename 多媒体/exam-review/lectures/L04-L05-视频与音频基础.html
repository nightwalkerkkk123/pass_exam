L04 视频基本概念 + L05 数字音频基础 · 7 + 8 考点

I.考点清单 (Exam Point Checklist)

L04 视频基础 — 7 大考点

L04-考点 1

模拟视频信号与扫描方式

核心
模拟视频 f(t) 是关于时间的连续信号;亮度关于电压单调。两种扫描方式 = 逐行 (progressive) + 隔行 (interlaced)。
关键
NTSC 30fps / 60场/秒 / 33.3ms/帧;PAL 25fps / 50场/秒 / 40ms/帧;每场 20ms。
易错
隔行扫描是 "先奇后偶",而不是把一帧拆成两半独立时间均匀画。PAL "25fps" 是帧率,场率是 50。
频率
高频
L04-考点 2

帧 vs 场 (Frame vs Field)

核心
帧 = 一幅完整的图像;场 = 隔行扫描中奇数场或偶数场(只含一半扫描线)。人眼靠视觉暂留把两场合成一帧。
关键
1 帧 = 2 场(NTSC/PAL 隔行)。
易错
HDTV 为避免运动锯齿 (jaggies) 改用 progressive scan (逐行),而不是用更快的隔行。
频率
高频
L04-考点 3

色度二次采样 J:a:b 体系

核心
人对色度 (Cr/Cb) 的分辨力弱于亮度 (Y),可降采样色度节约带宽。
关键
J=水平基准(4)。4:2:2 = 2 Bytes/pixel;4:1:1 = 1.5 Bytes/pixel(水平 1/4);4:2:0 = 1.5 Bytes/pixel(水平/垂直各 1/2)。
易错
4:2:0 的 "0" 表示第二行不单独采样(复用第一行),并不是全 0;4:2:2 水平半采样但垂直全采样。
频率
高频 · 必考计算
L04-考点 4

HDTV / UHDTV 分辨率与数据量

核心
HD 4:3 → 16:9;SD 720×480;2K-HD 1920×1080;4K UHD 3840×2160;8K UHD 7680×4320。
公式
每秒数据量 = 宽 × 高 × 帧率 × 每像素字节数(4:2:0 为 1.5)。
易错
1080p30 在 4:2:0 下 1 秒是 1920×1080×30×1.5 ≈ 93 MB,不是 RGB 的 186 MB。
频率
高频 · 必考计算
L04-考点 5

数字视频优势

核心
可存储、易编辑(直接访问帧)、重复录制无质量损失、易加密、噪声容忍。
易错
模拟视频的 "非线性编辑很难",但数字视频容易;数字 ≠ 压缩。
频率
中频
L04-考点 6

模拟视频接口对比

核心
分量视频 (3 路 YCC/RGB) > S-Video (Y+C) > 复合视频 (Y+C 混合)。
易错
VGA 是模拟分量 RGBHV (5 路),不是数字接口。
频率
中频 · 简答/对比
L04-考点 7

数字显示接口 & 3D 显示

核心
DVI 早期数字、HDMI 音视频一体化、DP 带宽最高且免费。3D = 单目线索(阴影/纹理/遮挡)+ 双目线索(视差)。
易错
HDMI 向后兼容 DVI;DP 通过有源适配器可兼容 VGA/DVI。
频率
中频

L05 数字音频基础 — 8 大考点

L05-考点 1

声音的三要素 (Amplitude / Frequency / Timbre)

核心
振幅→响度 (Loudness);频率→音调 (Pitch);波形/频谱→音色 (Timbre)。复音 = 基频 + 多次谐波叠加。
易错
纯音 ≠ 复合音;噪音是非周期 (aperiodic) 的随机频率。
频率
高频
L05-考点 2

音频 vs 语音频率范围

关键
音频 (Audio) 20Hz – 20kHz;语音 (Speech) 85Hz – 1.1kHz;语音带宽远小于音频。
易错
1.1k 单位是 Hz,不是 kHz;语音之所以能用低采样率编码 (8kHz) 就是因为这个 1.1k 上限。
频率
高频
L05-考点 3

数字化过程 = 采样 + 量化

核心
采样 = 连续时间 → 离散时间;量化 = 连续幅值 → 离散幅值。两个维度都要数字化。
易错
不要把 "采样" 与 "量化" 搞混。常见采样率 8 kHz(电话)~ 48 kHz(专业音频)。
频率
高频
L05-考点 4

奈奎斯特定理 (Nyquist Theorem)

公式
f_sample ≥ 2 × f_max。临界 2f_max 称为奈奎斯特频率;超过(不满足)时出现混叠 (aliasing),产生假频。
关键
22.05 kHz 声音 → 必须 ≥ 44.1 kHz 采样;33.075 kHz 采样真频 22.05 kHz → 假频 = 33.075 - 22.05 = 11.025 kHz。
易错
假频 = |f_sample - f_true|(当 f_sample < 2 f_max 且接近 f_sample 整数分之一时)。
频率
高频 · 必考计算
L05-考点 5

信噪比 SNR / SQNR (dB)

公式
SNR = 10·log10(P_s/P_n) = 20·log10(V_s/V_n);SQNR = 6.02·N (N 位量化)。
关键
每加 1 bit ≈ 6 dB;8 bit = 电话质量 (~48 dB);16 bit = CD 质量 (~96 dB);12 bit 已足够。
易错
"功率—10,电压—20" 系数;1 dB = 1/10 bel。
频率
高频 · 必考计算
L05-考点 6

PCM / DPCM / DM 编码

核心
PCM (Pulse Code Modulation) = 脉冲编码调制,原始采样直接量化;DPCM = 差分编码,传输差值;DM = Delta Modulation,是 DPCM 的极端简化版(1-bit 量化差值)。
易错
DM 不是简单 "差分",而是 1-bit 表示增减方向;ADPCM 是 DPCM 的自适应版本。
频率
高频
L05-考点 7

无损预测编码 (DPCM) — SU/SD 控制码

核心
差分使样本直方图更尖锐 (peak),便于熵编码。当差值过大时用 SU (+32) / SD (-32) 控制码 + 小差值表示。
关键
差值 100 = SU, SU, SU, 4(即 3×32 + 4 = 100)。
频率
中频
L05-考点 8

dB 音量等级与音频幅度范围

关键
0 dB 听力阈值;60 dB 日常对话;120 dB 不舒服;140 dB 痛苦;160 dB 鼓膜受损。
频率
低频 · 送分题

L04 视频基础 — 习题

L04-Q1单选 关于模拟视频扫描方式,下列说法正确的是:

  • A. 逐行扫描是先扫描偶数行再扫描奇数行
  • B. PAL 标准采用 30fps、60场/秒
  • C. 隔行扫描通过在一帧内分奇偶两场扫描,依赖人眼视觉暂留合成完整帧
  • D. NTSC 标准下每帧扫描时长是 20ms
显示答案与解析
答案 C 解析 逐行是按顺序逐行扫描;PAL 是 25fps / 50场;NTSC 每帧 33.3ms。隔行的核心是 "奇场 1/3/5... + 偶场 2/4/6..." 两场合成一帧,靠视觉暂留。

L04-Q2单选 帧 (frame) 与场 (field) 的关系,下列正确的是:

  • A. 一帧 = 一场
  • B. 一帧 = 二场(奇场 + 偶场),仅用于隔行扫描
  • C. 一场 = 二帧
  • D. 场是 3D 视频的单位
显示答案与解析
答案 B 解析 隔行扫描中,奇数行构成的图称为奇场,偶数行构成的图称为偶场,两场合成为一帧。逐行扫描无场概念。

L04-Q3单选 4:2:2 色度二次采样中,每像素占多少字节?

  • A. 3 字节
  • B. 2 字节
  • C. 1.5 字节
  • D. 1 字节
显示答案与解析
答案 B 解析 4:2:2 指 8 像素有 Y=8 字节,Cb/Cr 各 4 字节,共 16 字节 / 8 像素 = 2 Bytes/pixel。4:4:4 (无降采样) 才是 3 Bytes/pixel。

L04-Q4单选 4:2:0 与 4:1:1 相比,下列说法正确的是:

  • A. 4:2:0 带宽利用率比 4:1:1 高
  • B. 4:2:0 水平分辨率是原来的 1/4,与 4:1:1 相同
  • C. 4:2:0 水平/垂直分辨率均为原来的 1/2,4:1:1 仅水平降为 1/4
  • D. 二者完全相同
显示答案与解析
答案 C 解析 4:2:0 二维都降为 1/2(第二行复用第一行色度);4:1:1 只水平降为 1/4,垂直仍全分辨率。两者均为 1.5 Bytes/pixel,但几何分布不同。

L04-Q5多选 以下属于数字显示接口的有:

  • B. HDMI
  • C. DisplayPort
  • D. VGA

答案 A, B, C 解析 VGA 是模拟分量接口 (RGBHV)。DVI、HDMI、DP 都是数字接口,HDMI/DP 同时传音频信号。

L04-Q6计算题 一段 720×480 分辨率、25 fps、采用 4:2:0 色度采样的数字视频片段,长度为 5 分钟,请计算其原始数据量(以 MB 为单位)。

显示答案与解析
答案
- 每像素字节数:4:2:0 → 1.5 Bytes/pixel
- 每帧数据量:720 × 480 × 1.5 = 518,400 Bytes ≈ 0.494 MB (按 1 MB = 2²⁰ Bytes,约 0.494 MiB;若按 10⁶ ≈ 0.518 MB)
- 每秒数据量:0.494 × 25 ≈ 12.36 MB/s (二进) 或 12.96 MB/s (十进)
- 5 分钟 = 300 s:约 0.494 × 25 × 300 ≈ 3,705 MB ≈ 3.62 GB (二进) 或 3.89 GB (十进)
解析 用公式 数据量 = 宽 × 高 × 帧率 × 时长 × 1.5,注意单位和单位换算。课堂 SD 数据给的是 ~15.5 MB/s (即约十进制),与本题结果一致量级。

L04-Q7计算题 一段 4K UHDTV 视频,分辨率 3840×2160,30fps,4:2:0 格式。求:
(1) 单帧数据量;
(2) 1 秒数据量 (MB);
(3) 相对 1080p 视频的倍数。

显示答案与解析
答案
(1) 单帧:3840 × 2160 × 1.5 = 12,441,600 Bytes ≈ 11.87 MB
(2) 1 秒:11.87 × 30 ≈ 356 MB/s(接近讲义的 ~372 MB/s,因为 60fps 时才能达到 372 MB/s;讲义表中 30fps 是近似值)
(3) 4K 像素是 1080p 的 4 倍 → 单帧 4 倍、相同 fps 下码率 4 倍
解析 4:2:0 压缩比 1.5 B/px;4K 像素 = 3840×2160 = 8,294,400,是 1920×1080 = 2,073,600 的 4 倍。

L04-Q8简答 简述为什么 HDTV 选择逐行扫描 (progressive scan) 而非隔行扫描 (interlaced scan)?这对视频数据量有何影响?

显示答案与解析
答案要点
1. 隔行扫描锯齿 (jaggies):快速运动物体横跨两场时,由于两场时间不同,合成帧产生边缘锯齿;逐行扫描同一帧内所有行同一时刻,无此问题。
2. 隔行扫描对现代 LCD/OLED 显示器不友好(显示设备按逐行刷新)。
3. HDTV 同时增加可视域宽度(4:3 → 16:9),需要更高空间分辨率。
4. 数据量影响:在同样帧率下,逐行需一次送出所有行,瞬时带宽更高;但视频流总码率 = 帧率 × 单帧数据,二者若帧率相同 (如 30p vs 60i),码率相同;现代高清更常用 60p 真正高于隔行。
解析 1080p30 (逐行) 与 1080i60 (隔行) 帧率定义略有差异:p 用逐行帧数;i 用场数映射到 60 场/秒。

L04-Q9简答 比较分量视频、S-Video、复合视频三种模拟接口在画质、线缆数量、串扰三方面的差异。

显示答案与解析
答案要点
1. 分量视频:3 路独立 YCC/RGB,线缆 3 根,画质最高,无亮色串扰,同步严格,成本高、专业级。
2. S-Video:1 根线缆两路铜芯(Y + 复合 C),亮度与色度分开,画质中等,串扰极少。
3. 复合视频:1 根线缆 Y+C 混合(色度副载波放高频),画质最低,亮色串扰严重,成本最低。
4. 整体趋势:分离越多 → 画质越好、串扰越少、成本越高。

L04-Q10综合应用 设计一种将现有 NTSC 模拟电视信号转为可在 IP 网络上传输的数字流方案,需要说明:分辨率选择、色度采样、帧率、码率、显示端接口。请给出关键参数选择理由。

显示答案与解析
答案要点
1. 分辨率选择:NTSC 原始 720×480 是 SD 标清;考虑向后兼容 + 带宽,选 720×480 即可;若面向高清可升 1920×1080。
2. 色度采样:从兼容性最好选 4:2:0,1.5 Bytes/pixel。专业制作用 4:2:2。
3. 帧率:保持与 NTSC 兼容 30fps 或 29.97fps (drop-frame)。
4. 码率:
  - SD 720×480@30fps, 4:2:0 → 720×480×30×1.5 ≈ 15.5 MB/s ≈ 124 Mbps 原始
  - 高清 1920×1080@30fps, 4:2:0 → ~93 MB/s ≈ 744 Mbps 原始
  - 实际传输需 H.264/H.265 压缩至 2–10 Mbps 量级。
5. 显示端接口:现代电视/显示器多用 HDMI 2.0 (4K@60Hz) 或 DP 1.4 (8K@60Hz);老设备兼容 DVI。
6. 设计折衷:在 "画质 / 带宽 / 兼容性" 三者间权衡;本题 SD 方案总原始码率 ~124 Mbps,需要至少 ~3 倍压缩才能在 100M 以太网舒适传输。

L05-Q3单选 一个频率为 22.05 kHz 的声音,根据奈奎斯特定理,最低采样率应为:

L05-Q7计算题 一段单声道音频,采样率 8 kHz,每个样本 8 bit 量化,时长 10 秒。求:
(1) 每秒数据量;
(2) 总数据量;
(3) 与 44.1 kHz/16 bit 立体声 CD 相比,码率差距多少倍?

显示答案与解析
答案
(1) 8 kHz × 8 bit = 64 kbps = 8 KB/s
(2) 10 × 8 KB = 80 KB
(3) CD 立体声:44.1 kHz × 16 bit × 2 = 1411.2 kbps,本题 64 kbps,差距约 22 倍
解析 公式 码率 = 采样率 × 每样本 bit × 通道数。"码率差距" 主要由采样率、量化位数、通道数三方面决定,立体声比单声道也是 2 倍差距来源之一。

L05-Q8计算题 一段最高频率为 8 kHz 的音频信号:
(1) 求奈奎斯特采样率;
(2) 若实际采样率 = 12 kHz,求假频;
(3) 若改用 16 bit 量化,求理论 SQNR。

显示答案与解析
答案
(1) f_Nyquist = 2 × 8 = 16 kHz
(2) 假频 = |12 − 8| = 4 kHz (因为 12 < 16,违反 Nyquist)
(3) SQNR = 6.02 × 16 ≈ 96 dB
解析 当采样率 < 2 f_max 时,假频 = |f_sample − f_true|;16 bit CD 质量约 96 dB。

L05-Q9简答 解释 PCM、DPCM、DM 三者的关系。DM 与 DPCM 相比有什么主要优势与代价?

显示答案与解析
答案要点
1. PCM (Pulse Code Modulation):直接对每个采样值量化(不做差分),是基础编码。
2. DPCM (Differential PCM):对相邻样本差值量化,由于差值小、方差小,量化更精细。
3. DM (Delta Modulation):DPCM 的极端简化,每样本只用 1 bit 表示 "上升"或"下降"一个固定步长。
4. 优势:DM 硬件简单、码率低 (1 bit/sample),适合实时 AD 转换和语音低速编码。
5. 代价:DM 在快速变化信号上 "斜率过载" (slope overload),在缓慢变化区易 "颗粒噪声" (granular noise);只能用于语音类缓变信号。

L05-Q10简答 + 设计 为什么电话系统可以使用 8 kHz 采样、8 bit 量化(即 64 kbps)即可实现可懂语音通话?结合奈奎斯特定理与 SQNR 公式解释。

显示答案与解析
答案要点
1. 语音 (speech) 频率范围 85 Hz – 1.1 kHz,远小于音频 (20 Hz – 20 kHz);奈奎斯特要求 8 kHz 采样即可覆盖 4 kHz 内全部信息(留有余量到 4 kHz)。
2. 8 bit 量化 → SQNR ≈ 48 dB,对语音可懂度足够(人耳对语音信噪比要求 ~30–40 dB 即可懂)。
3. 综合:码率 = 8 kHz × 8 bit = 64 kbps,是单信道 PSTN 数字电话的标准速率 (DS0 = 64 kbps)。
4. 设计验证:若用 16 kHz/16 bit 量化,仍是高质量电话;现代 VoIP 普遍用 8 kHz/16 bit 或 16 kHz/16 bit 进一步提升音质。
