I.考点清单 (Exam Point Checklist)

第二讲 (L02) 考点

L02-考点 1

像素与图像分辨率

核心
像素 (Pixel) = picture element,图像的最小离散单位;图像分辨率 = 像素总数
关键
标准图 Lena 256×256(灰度),Baboon 512×512(彩色)
易错
分辨率 ≠ 屏幕尺寸;屏幕 DPI 是物理密度
频率
高频
L02-考点 2

灰度图像与 Gray 转换公式

核心
8-bit 灰度图每个像素取值 0-255;Pillow 用 convert('L') 转灰度
公式
Gray = 0.299·R + 0.587·G + 0.114·B(ITU-R BT.601)
易错
系数非平均 1/3;G 通道权重最高(人眼对绿最敏感)
频率
高频 · 必考计算
L02-考点 3

二值图像

核心
每个像素 1 bit (0/1);用于简单图形/文字
公式
阈值化 point(lambda x: 255 if x > 127 else 0)
易错
阈值 127 不是固定值,Otsu 可自适应选取
频率
中频
L02-考点 4

位平面 (Bitplane) 分解

核心
8-bit 图像可分解为 8 个 1-bit 位平面
公式
bitplane_k = (pixel >> k) & 1
易错
高阶位(高 bit)决定图像结构;低阶位(低 bit)含噪声/细节
应用
图像压缩、加密隐写、去噪、信息分析
频率
高频
L02-考点 5

24-bit 真彩色图像

核心
R(8-bit) + G(8-bit) + B(8-bit) 共 3 字节/像素;支持 256³ ≈ 16.8M 色
关键
640×480 的 24-bit 图 ≈ 900 KB (640×480×3/1024)
易错
32-bit 图 = 24-bit + 8-bit 特效通道 (alpha)
频率
高频
L02-考点 6

直方图 (Histogram)

核心
统计每个灰度值出现的像素数;8-bit 直方图长度 256
公式
np.histogram / np.bincount;Bin 范围 [0, 255]
易错
直方图相同 ≠ 图像相同;可反映对比度、亮度
频率
高频
L02-考点 7

8-bit 伪彩色与颜色查找表 (CLUT)

核心
每个像素只存 8-bit 索引;调色板 256 项;适合颜色数少的图
公式
8-bit color = R(3-bit) + G(3-bit) + B(2-bit) = 8-bit
易错
3:3:2 分箱导致"边缘伪影";调色板可基于直方图选最频 256 色
频率
高频
L02-考点 8

图像数据量计算

核心
宽×高×每像素 bit 数 = 总 bit;÷8 = Byte;÷1024 = KB
公式
size = W × H × B / 8 (Bytes)
易错
单位混淆 (b vs B, Kb vs KB);24-bit 灰度是 8-bit 灰度的 3 倍
频率
高频 · 必考计算
L02-考点 9

Dithering(抖动 / 网板打印)

核心
用 1-bit 打印机表现灰度图;用空间分辨率换强度分辨率
公式
有序抖动 O(x,y) = 1 if I(x,y) > D(i,j) else 0,i = x mod n,j = y mod n
关键
4×4 Bayer 抖动矩阵 D(值 0-15)
易错
强度大的区域 on dot 多;常用于半色调打印
频率
中频

L03-考点 3

色度匹配函数与 CIE XYZ

核心
CIE RGB 匹配函数 r̄(λ), ḡ(λ), b̄(λ) 存在负 lobe;引入 XYZ 虚拟原色使其全非负

易错

频率

CIE 色度图与色域 (Gamut)

白点 (1/3, 1/3);光谱轨迹 (spectrum locus);色域 = 设备可表示颜色子集
关键
超色域 (out-of-gamut) 颜色 = 该颜色到白点的连线与设备色域边界的交点
易错
色域是三角形但色度空间是马蹄形
频率
中频
L03-考点 5

RGB 加色模型

核心
红+绿+蓝光叠加;设备相关;用于屏幕显示
关键
每通道 8-bit,组合 16.8M 色
易错
RGB 是 device-dependent;不同设备同一 RGB 值不同
频率
高频
L03-考点 6

Gamma 校正 (γ 校正)

核心
CRT 发光强度 ∝ 电压^γ(γ ≈ 2.2);故存储时要 inverse gamma 预校正
公式
显示亮度 = (V)^(1/γ),存储编码 = V^(1/γ)
易错
γ 校正用于显示;也用于图像处理(如增强对比度)
频率
高频
L03-考点 7

CMY(K) 减色模型

核心
印刷用;C = 1-R, M = 1-G, Y = 1-B;K = 黑(节省墨水)
公式
C = 1 - R, M = 1 - G, Y = 1 - B(归一化)
易错
CMY 混合得不到真黑,故加 K
频率
中频
L03-考点 8

HSV / HSL 模型

核心
H (色调 0-360°)、S (饱和度 0-1)、V/L (明度/亮度 0-1)
关键
调色用 HSV,设计用 HSL
易错
HSV 是 hexcone 模型,HSL 是双六棱锥;二者不同
频率
高频
L03-考点 9

CIELAB (L*a*b*) 色彩模型

核心
L* = 亮度 (0-100);a* = 红(-)绿(+),b* = 黄(-)蓝(+)
关键
视觉均匀 (Weber 定律) — 同样数值变化 ≈ 同样感知差异
易错
L*a*b* 不是色域,是感知空间
频率
高频
L03-考点 10

YUV / YIQ / YCbCr 视频颜色模型

核心
Y = 亮度 (luma),U/V 或 Cb/Cr = 色度 (chroma)
关键
Y: 16-235;Cb/Cr: 16-240(数字域,TV 标准)
应用
PAL/SECAM 用 YUV;北美用 YIQ;JPEG/MPEG 用 YCbCr
易错
YCbCr ≠ YUV;YCbCr 是 YUV 的 scale+offset 版本
频率
高频

  • A. 2.25 MB
  • B. 2.34 MB
  • C. 768 KB
  • D. 7.5 MB
显示答案与解析
答案 A 解析 1024×768×3 = 2,359,296 Bytes ≈ 2.25 MB (÷1024²)。注意 1024×768×24/8 = 2,359,296 B = 2.25 MiB。

L02-Q2单选 关于 Gray 转换公式 Y = 0.299R + 0.587G + 0.114B,下列说法正确的是:

  • A. 三个系数相等
  • B. R 通道权重最大
  • C. G 通道权重最大,因为人眼对绿色最敏感
  • D. B 通道权重最大
显示答案与解析
答案 C 解析 系数 0.587 (G) > 0.299 (R) > 0.114 (B),反映了人眼视锥细胞对绿光最敏感的事实,对应 ITU-R BT.601 标准。

L02-Q3多选 关于 8-bit 位平面分解,下列说法正确的有:

  • A. 8-bit 灰度图可分解为 8 个 1-bit 位平面
  • B. 高阶位平面包含图像整体结构信息
  • C. 低阶位平面主要包含噪声和细节
  • D. 位平面分解常用于水印嵌入和压缩
显示答案与解析
答案 A, B, C, D 解析 位平面 k 由像素的第 k 位 (bit) 组成;MSB (高阶) 决定结构,LSB (低阶) 多为噪声;可应用于压缩、隐写、去噪。

(1) 存储为 24-bit RGB 时的总字节数 (KB);
(2) 转换为 8-bit 灰度图后的总字节数 (KB);

L02-Q6简答 简述 24-bit 彩色图像转换为 8-bit 伪彩色图像的两种方法及其优缺点。

显示答案与解析
答案要点
1. 均匀分箱法 (Uniform Quantization):R 取 3-bit, G 取 3-bit, B 取 2-bit,每像素 8-bit。优点:实时、无需训练;缺点:颜色分布不均,出现"边缘伪影"。
2. 流行色法 (Popularity / Median-cut):基于直方图选 256 个最频繁颜色作为调色板 (CLUT)。优点:视觉效果好;缺点:需要统计计算、可能丢失稀有颜色。
解析 两种方法都把每像素颜色降到 8-bit 索引,通过查找表 (LUT) 还原 24-bit 真彩色。

L02-Q7论述 Lena 图像为何成为图像处理领域的"标准测试图"?为什么后来受到批评?

显示答案与解析
答案要点
1. 广泛使用的原因:包含平滑区、纹理区、边缘、细节、阴影等多种特征;中等复杂度 (256×256) 适合早期算法;来源 Playboy 杂志 (1972)。
2. 批评点:(a) 来源不当,性别歧视争议;(b) 不够现代,复杂场景代表性差;(c) 主流期刊已弃用,转向 BSDS500、COCO 等数据集。
3. 新趋势:现代 benchmark 使用 Kodak、BSD500、Set5/Set14、Urban100 等。

L02-Q8综合应用 给定一张 640×480 的彩色照片,要求将其用于黑白喷墨打印机打印。请设计完整的处理流程,并估算压缩前后的数据量。

显示答案与解析
答案要点
1. 彩色 → 灰度:用 Gray = 0.299R + 0.587G + 0.114B,数据量 640×480×1 = 307,200 B ≈ 300 KB。
2. 灰度 → 二值 (Dithering):用 4×4 Bayer 抖动或 Floyd-Steinberg 误差扩散;数据量 640×480/8 = 38,400 B ≈ 37.5 KB。
3. 数据量比较:原图 900 KB → 灰度 300 KB → 二值 37.5 KB(压缩 24 倍)。
4. 效果对比:直接阈值化(无 dithering)会丢失中间灰度细节;dithering 用空间分辨率换强度分辨率,保留视觉层次。

L03-Q6简答 比较 HSV 与 HSL 颜色模型的异同。

显示答案与解析
答案要点
- 相同:都用 H (色相 0-360°) + S (饱和度 0-1) + 第三分量 (V 或 L 0-1);都从 RGB 转换;适合交互式调色。
- 不同:
  • HSV (Value):hexcone 六棱锥模型;V = max(R,G,B);饱和时颜色纯。
  • HSL (Lightness):bi-hexcone 双六棱锥模型;L = (max+min)/2;纯白纯黑都可达。
- 应用:HSV 用于图像分割/算法 (OpenCV);HSL 用于 UI 设计/CSS/PS。

L03-Q7论述 详细说明为什么数字视频和 JPEG 压缩普遍使用 YCbCr 而不是 RGB。

显示答案与解析
答案要点
1. 人眼特性:对亮度 (Y) 变化敏感,对色度 (Cb/Cr) 不敏感。
2. 可分离压缩:可对 Cb/Cr 下采样 (4:2:0、4:2:2) 节省带宽,视觉影响小。
3. 兼容电视系统:源自模拟电视 (PAL 用 YUV、NTSC 用 YIQ),数字时代统一为 YCbCr。
4. 亮度色度独立:便于黑白电视兼容;亮度信号可独立传输。
5. 范围标准化:Y: 16-235, Cb/Cr: 16-240,给同步和滤波留 headroom。

L03-Q8综合应用 给定一张 1920×1080 的 RGB 彩色图像,分别计算在 RGB 和 YCbCr 4:2:0 子采样下的存储容量(每像素 8-bit)。

显示答案与解析
答案
1. RGB (24-bit):1920×1080×3 = 6,220,800 B ≈ 5.93 MB
2. YCbCr 4:2:0:每 2×2 像素块共用 Y(4)+Cb(1)+Cr(1) = 6 字节,平均每像素 1.5 字节。
  总字节数 = 1920×1080×1.5 = 3,110,400 B ≈ 2.97 MB
3. 压缩比:约 2:1,且视觉无明显损失
解析 4:2:0 表示色度在水平和垂直方向都减半采样,是 JPEG、MPEG、HEVC 的默认子采样格式。

L03-Q9简答 CIE RGB 匹配函数为何要引入 XYZ 虚拟原色?

显示答案与解析
答案要点
1. 负 lobe 问题:CIE RGB 匹配函数 r̄(λ) 在某些波长出现负值,物理上需"负光强",不直观。
2. 虚拟原色 XYZ:通过 3×3 线性变换,使匹配函数 x̄(λ), ȳ(λ), z̄(λ) 全部非负。
3. 约束条件:ȳ(λ) ≡ V(λ) (光视效率曲线),便于把 Y 直接作为亮度。
4. 意义:建立设备无关的标准基色空间,所有颜色空间转换以 XYZ 为中转。

L03-Q10论述 详细分析 Gamma 校正的原理、作用与典型应用。

显示答案与解析
答案要点
1. 原理:CRT 显示的发光强度 L ∝ V^γ(γ ≈ 2.2),即电压与亮度非线性。
2. 校正方法:存储图像时预校正 s = V^(1/γ),使显示还原后 L = (V^(1/γ))^γ = V,线性还原。
3. 作用:
  • 视觉感知:人眼对暗部变化敏感,γ 校正匹配 Weber-Fechner 定律。
  • 编码效率:用有限 bit 数更好地分配亮度信息,避免暗部 banding。
4. 典型 γ 值:CRT 2.2;Mac OS 1.8;sRGB 2.2;印刷用 1.8 (L*).
5. 应用场景:图像显示、图像处理(gamma 调整增强对比度)、HDR 编码、视频传输。
