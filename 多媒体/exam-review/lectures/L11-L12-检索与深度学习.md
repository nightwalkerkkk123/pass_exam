## 第 1 部分：考点清单 (Exam Point Checklist)

### L11 基于内容的多媒体检索（CBIR）

- **【L11-考点1】语义鸿沟与意图鸿沟**
  - 核心概念：低层视觉特征与高层语义之间的"语义鸿沟 (Semantic Gap)"；文本标签难以覆盖图像内容与查询意图的"意图鸿沟 (Intention Gap)"。
  - 关键对比：传统基于文本的检索 vs 基于内容的检索 (CBIR, Query by Example)。
  - 易错点：混淆"语义鸿沟"和"意图鸿沟"——前者是低层特征↔高层语义，后者是用户查询↔内容描述。
  - 考频预判：中频。

- **【L11-考点2】CBIR 系统框架**
  - 核心概念：离线库图像→特征提取→索引；在线查询图像→特征提取→匹配→结果。三大关键技术：特征提取、特征匹配、高维索引。
  - 关键流程：QBE (Query By Example) 是 CBIR 的核心交互模式。
  - 易错点：忽略"高维索引"环节的重要性——它是支持大规模库检索的工程关键。
  - 考频预判：中频。

- **【L11-考点3】全局特征（颜色/纹理/形状）**
  - 核心概念：颜色直方图、纹理统计量、形状描述子；计算简单但难以应对遮挡、裁剪等攻击，缺乏不变性。
  - 关键弱点：对几何变换（旋转、缩放、裁剪）敏感；丢失空间分布信息。
  - 易错点：颜色直方图对光照敏感（不同光照下颜色分布变化大）；对"内容相同但颜色不同"的图像区分力差。
  - 考频预判：中频。

- **【L11-考点4】局部特征检测子分类与 Harris 角点**
  - 核心概念：局部特征应满足鲁棒性、区分性、提取高效性。检测子分为：基于角点 (Harris/FAST)、基于斑点 (SIFT/SURF)、基于区域 (MSER) 三类。
  - 关键公式：Harris 角点响应函数 `R = det(M) - k·trace(M)²`，k = 0.04~0.06；M 是 2×2 图像微分矩阵，由 `Ix², IxIy, Iy²` 高斯加权得到。
  - 关键分类（基于特征值 λ₁, λ₂）：两者都小→Flat；一大一小→Edge；都大且相近→Corner。
  - 易错点：Harris **不具有尺度不变性**；对模糊和噪声敏感；k 的取值是经验常数。
  - 考频预判：高频。

- **【L11-考点5】FAST 角点检测**
  - 核心思想：圆环上若有连续 12 个像素比中心像素更亮（或更暗），则该点为角点（实际使用加速剪枝先测 1/9/5/13 号像素）。
  - 优缺点：算法简单、速度极快（几毫秒）；但角点定义过严、误检率高、不具有尺度不变性。
  - 易错点：FAST 的"12 点连续"是定义门槛，硬件加速剪枝并非定义。
  - 考频预判：低频。

- **【L11-考点6】尺度空间与 Harris-Laplace**
  - 核心概念：图像多尺度描述两种方法——图像金字塔（平滑+降采样）与高斯尺度空间（卷积可变尺度高斯核）。特征尺度 = 响应值最大的尺度。
  - 关键流程：Harris-Laplace 先生成高斯尺度空间并计算各尺度角点响应，再在尺度空间搜索特征尺度（拉普拉斯响应极大值）。
  - 优缺点：具有尺度不变性，对旋转/对比度不敏感；但一个局部结构常对应多个特征尺度，多尺度计算耗时。
  - 易错点：高斯尺度空间 ≠ 图像金字塔——前者图片尺寸不变，后者尺寸逐层减半。
  - 考频预判：中频。

- **【L11-考点7】SIFT 检测子（DoG 金字塔）**
  - 核心思想：Lowe 1999/2004；在 DoG (Difference-of-Gaussian) 尺度空间同时检测二维空间与尺度空间的局部极值作为特征点。
  - 关键步骤：①构建高斯金字塔→组内逐层相减得到 DoG 金字塔；②每个像素与 26 邻域比较找极值；③曲线拟合精确定位；④Hessian 矩阵删除边缘响应点。
  - 为什么用 DoG：DoG 是规范化 LoG (Laplacian-of-Gaussian) 的近似，LoG 各向同性具有真正的尺度不变性，用 DoG 近似减少计算量。
  - 优缺点：鲁棒性高（旋转+尺度不变）、独特性好、定位准确；但实时性不高（1~2 秒）、不具有仿射不变性、特征数约 3000。
  - 易错点：DoG ≈ LoG 但 ≠ LoG；SIFT 不抗仿射；26 邻域是"立方体邻域"（同层 8 + 上下层各 9）。
  - 考频预判：**高频**。

- **【L11-考点8】SURF 加速鲁棒特征**
  - 核心思想：基于 Hessian 矩阵行列式建立多尺度金字塔；用二阶导数近似模板+积分图加速；金字塔构建时**图片大小不变，只改变高斯核尺度**。
  - 优缺点：速度是 SIFT 的 3~5 倍（描述子环节可达 3~7 倍）；具有旋转+尺度不变性；但不抗仿射、光照变化鲁棒性略弱于 SIFT。
  - 易错点：SURF 金字塔构建方式与 SIFT 不同（图片尺寸不变 vs 逐层降采样）。
  - 考频预判：中频。

- **【L11-考点9】MSER（最大稳定极值区域）**
  - 核心思想：Matas 2002；地形学分水岭概念；对图像按灰度排序（Binsort, O(n)），用 Component Tree 记录连通区域随阈值的生长，在宽阈值范围内形状稳定的区域即 MSER。
  - 优缺点：**唯一具有仿射不变性**的区域检测子；提取快速、重复率高；但对图像模糊敏感，检测到的常是灰度一致的连通区域（信息量少）。
  - 易错点：MSER 是"检测子 + 区域"，而 SIFT/SURF 是"检测子 + 描述子"组合。
  - 考频预判：中频。

- **【L11-考点10】SIFT 描述子（128 维向量）**
  - 提取步骤：①像素梯度计算（模+方向）；②局部区块主方向统计（直方图峰值，可能多个主方向）；③坐标轴旋转到主方向+4×4 分块每块 8 方向梯度幅值→**4×4×8=128 维**；④特征归一化去光照影响。
  - PCA-SIFT：在 39×39 邻域内取 3042 维梯度向量，用离线训练得到的 K 维 PCA 投影矩阵降至 K 维（通常 20~36）。
  - 优缺点：128 维区分性强、旋转不变；缺点是维数高、存储匹配代价大。
  - 易错点：4×4 是**分块数**，8 是**方向 bin 数**；PCA-SIFT 不是把 128 维降到 K 维，而是把 3042 维原始梯度降到 K 维。
  - 考频预判：高频。

- **【L11-考点11】距离度量与性质**
  - 核心概念：CBIR 是相似性检索，必须定义距离函数 `d(x,y)`；理想度量应满足：①非负性 d(x,y)≥0；②同一性 d(x,x)=0；③对称性 d(x,y)=d(y,x)；④三角不等式 d(x,k)+d(k,y)≥d(x,y)。
  - 常用度量：欧氏距离（L2）、曼哈顿距离（L1）、余弦距离、马氏距离、汉明距离（二进制描述子）。
  - 易错点：相似性≠距离，需要从距离转换（如高斯核 e^(-d²/2σ²)）；不同特征类型选用不同度量。
  - 考频预判：中频。

- **【L11-考点12】群组匹配（空间上下文）**
  - 核心概念：单点特征匹配不稳定，需要群组/几何校验；方法包括 KNN Vote、RANSAC、WGC (Weak Geometric Consistency)、Bundle of features。
  - 易错点：RANSAC 适合估计几何变换矩阵；KNN Vote 用最近邻投票做空间投票。
  - 考频预判：低频。

- **【L11-考点13】高维索引（简述）**
  - 核心概念：倒排索引（基于量化词袋）、树型索引（KD-Tree / R-Tree）、哈希索引（LSH 局部敏感哈希）。
  - 易错点：当维度 > 16 时 KD-Tree 退化为线性扫描，需要近似最近邻 (ANN) 方法。
  - 考频预判：低频（仅作选择题概念）。

### L12 深度学习与卷积神经网络

- **【L12-考点1】深度学习三部曲**
  - 核心概念：①搭建网络结构；②找到合适损失函数；③找到合适优化函数更新参数。
  - 关键概念：损失函数衡量预测与真实吻合度；优化函数通过反向传播更新权重 W。
  - 易错点：把"损失函数"和"优化函数"混淆——前者衡量吻合度，后者更新参数。
  - 考频预判：中频。

- **【L12-考点2】常用损失函数**
  - 分类损失：交叉熵损失 (Cross Entropy Loss)、Hinge Loss `l(y,f(x)) = max(0, 1-y·f(x))`（SVM 风格）。
  - 回归损失：均方误差 MSE、L1 损失（平均绝对值误差）。
  - 易错点：分类≠回归——损失函数选择与任务类型严格对应；Hinge Loss 与 SVM 紧密相关。
  - 考频预判：中频。

- **【L12-考点3】全连接网络处理图像的瓶颈**
  - 核心概念：图像展平后输入全连接网络，权重矩阵参数过多→过拟合。
  - 例：1000×1000 RGB 图像 = 3×10⁶ 输入维度，第一层若有 1000 个神经元，仅一层就需要 3×10⁹ 个权重。
  - 易错点：参数爆炸≠计算慢，关键是过拟合。
  - 考频预判：中频。

- **【L12-考点4】CNN 三大设计思想**
  - 局部关联：每个神经元只感受局部区域（局部感受野），符合图像的空间局部性。
  - 参数共享：同一卷积核在整张图上滑动共享权重，大幅减少参数。
  - 多卷积核：使用多个不同卷积核提取不同特征（如边缘、纹理、颜色）。
  - 易错点：CNN 三大优点——局部关联+参数共享+层级特征表达。
  - 考频预判：高频。

- **【L12-考点5】卷积层关键超参数与输出尺寸**
  - 关键术语：input / kernel(filter) / weights / receptive field / activation map(feature map) / padding / depth(channel) / stride。
  - 输出尺寸公式：
    - 无 padding：`output = (N - F) / stride + 1`
    - 有 padding：`output = (N + 2·padding - F) / stride + 1`
  - 卷积示例：5×5 输入 + 3×3 卷积核 + stride 1 → 3×3 输出（无 padding：(5-3)/1+1=3）。
  - 参数量：`(F × F + 1) × K`（含 1 个 bias），其中 K 为卷积核个数。
  - 易错点：padding 是**单边**还是**双边**——公式中 `+2·padding` 表示左右（或上下）各填充 padding 行/列；输出尺寸必须为整数（否则需要调整参数）。
  - 考频预判：**高频**（必考计算题）。

- **【L12-考点6】池化层（Pooling）**
  - 核心作用：在保留主要特征的同时减少参数和计算量、防止过拟合、提高泛化能力。
  - 类型：Max Pooling（取最大值，更常用）、Average Pooling（取平均值）。
  - 位置：通常位于卷积层之间、全连接层之间。
  - 易错点：池化层**没有可学习参数**；只是固定运算（与卷积层不同）。
  - 考频预判：中频。

- **【L12-考点7】全连接层（FC）**
  - 核心概念：两层之间所有神经元都有权重连接；通常位于 CNN 尾部；**参数量通常最大**（VGG 138M 参数大多在 FC 层）。
  - 易错点：ResNet/GoogleNet 已逐步去掉 FC 层（除输出层外），改用全局平均池化 (GAP)。
  - 考频预判：中频。

- **【L12-考点8】AlexNet（2012 ImageNet 冠军）**
  - 历史意义：top-5 错误率比上一年下降约 10 个百分点，开启深度学习时代。
  - 结构：8 层 (C1→C2→C3→C4→C5→F1→F2→F3)，5 卷积 + 3 全连接。
  - 成功关键：①百万级 ImageNet 数据；②ReLU 激活函数（解决梯度消失）；③Dropout + Data Augmentation 防过拟合；④双 GPU 并行训练。
  - 网络细节：前两次"卷积-ReLU-池化"，第三/四次"卷积-ReLU"无池化，第五次"卷积-ReLU-池化"，第六/七次"FC-ReLU-Dropout"，第八次"FC-Softmax"。
  - 易错点：ReLU 的关键贡献是"在正区间不会饱和"，解决 sigmoid/tanh 的梯度消失。
  - 考频预判：高频。

- **【L12-考点9】VGG（2014）**
  - 核心特征：更深的网络（16-19 层 vs AlexNet 8 层）；多个 3×3 小卷积核替代大卷积核（两个 3×3 堆叠 = 一个 5×5 感受野，但参数更少+非线性更多）。
  - 性能：ILSVRC top-5 错误率 11.7% → 7.3%。
  - 参数量：138M，**主要分布在全连接层**。
  - 易错点：VGG 简单但参数巨大，正是后续 GoogleNet/ResNet 改进的动机。
  - 考频预判：中频。

- **【L12-考点10】GoogleNet / Inception（2014 冠军）**
  - 性能：top-5 错误率 11.7% → 6.7%。
  - 架构特点：22 层带参数层（含 pooling 共 27 层），独立成块约 100 个，**参数量仅 AlexNet 的 1/12**，除输出层外**没有 FC 层**。
  - Inception 模块演进：
    - Naïve Inception：多卷积核（1×1/3×3/5×5/池化）并联增加特征多样性。
    - Inception V2：插入 1×1 卷积核进行降维（瓶颈层），解决计算复杂度过高问题。
    - Inception V3：用小卷积核替代大卷积核（如 5×5 → 两个 3×3），进一步降参数量并增加非线性。
  - 辅助分类器：解决深度网络的梯度消失问题。
  - 易错点：1×1 卷积的核心作用是**降维/通道变换**，不是提取空间特征。
  - 考频预判：高频。

- **【L12-考点11】ResNet 残差网络（2015 冠军）**
  - 性能：top-5 错误率 6.7% → 3.57%，深度 152 层。
  - 残差思想：`H(x) = F(x) + x`，让网络学习残差 F(x) 而非直接学习映射 H(x)，突出微小变化；解决深度网络的"退化问题"（不是过拟合，是训练误差随深度增大而增大）。
  - 架构特点：除输出层外**没有其他全连接层**（用全局平均池化 GAP）。
  - 易错点：残差网络解决的不是梯度消失，而是**深度退化**问题；跳连接使得"什么都不做"也容易（学 F(x)=0）。
  - 考频预判：**高频**。

- **【L12-考点12】CNN 典型网络演进对比**
  - 网络层数：LeNet(5) → AlexNet(8) → VGG(16-19) → GoogleNet(22) → ResNet(152) → Xception(2017)。
  - 演进主线：变深 → 减少参数 → 解决退化。
  - 易错点：Xception 是 Inception V3 的极端版（深度可分离卷积）。
  - 考频预判：中频。

- **【L12-考点13】数据增强、Dropout、正则化**
  - Data Augmentation：平移、翻转、对称；随机 crop（如 256×256 → 224×224）；水平翻转；RGB 通道强度高斯扰动 (PCA Color Augmentation)。
  - Dropout：训练时随机关闭部分神经元（典型比例 0.5）；测试时使用所有神经元；本质是隐式的模型集成。
  - ReLU：`f(x) = max(0, x)`；正区间不会饱和，计算快、收敛远快于 sigmoid；缺点是负区间梯度为 0（Dead ReLU）。
  - 易错点：Dropout 只在训练时启用，测试时关闭；ReLU 不是零中心化。
  - 考频预判：中频。

- **【L12-考点14】深度学习开源框架对比**
  - TensorFlow：Google Brain，数据流图（节点=运算，边=Tensor），静态图，节点间异步、节点内并行；适合生产部署。
  - PyTorch：Facebook，Torch + Python，**动态图（Define-by-Run）**，相比 TF 1.x 最大优势；学术主流。
  - Caffe：UC Berkeley 贾扬清，layer-based，适合传统 CNN，配置简单但灵活度低。
  - 易错点：TF 2.x 已默认动态执行（Eager Execution），与 PyTorch 接近；区分 TF 1.x 和 TF 2.x。
  - 考频预判：低频（概念题）。

## 第 2 部分：习题 (Exercises)

### L11 习题

**【L11-Q1】(单选)** 下列哪一项最准确地描述了 CBIR 系统需要克服的"语义鸿沟 (Semantic Gap)"？
- A. 不同图像采集设备造成的像素差异
- B. 图像底层视觉特征（如颜色、纹理）与高层人类语义之间的差异
- C. 用户查询关键词与数据库索引词之间的差异
- D. 图像压缩前后视觉质量的差异

**答案：** B

**解析：** 语义鸿沟 (Semantic Gap) 指图像底层视觉特征与高层人类理解的概念（如"一只狗在草地上跑"）之间的鸿沟；C 选项对应的是"意图鸿沟"。

---

**【L11-Q2】(单选)** 关于 Harris 角点响应函数 R = det(M) - k·trace(M)² 的描述，错误的是？
- A. k 是经验常数，取值范围 0.04~0.06
- B. M 是 2×2 图像微分矩阵，由 Ix², IxIy, Iy² 高斯加权得到
- C. 当 λ₁, λ₂ 都较大且相近时，R 为较大正值，对应角点

**答案：** D

**【L11-Q6】(简答)** 简述 SIFT 检测子的主要步骤，并解释为什么 DoG 可以近似 LoG。

**答题要点：**
1. **SIFT 步骤**：
   - Step 1：构建高斯金字塔（多分辨率金字塔+多层高斯滤波）；
   - Step 2：组内逐层相减得到 DoG 金字塔；
   - Step 3：每个像素与 26 邻域（立方体邻域：同层 8 个 + 上下层各 9 个）比较 DoG 响应值找极值；
   - Step 4：对 DoG 函数进行曲线拟合，实现关键点精确定位；
   - Step 5：利用 Hessian 矩阵判断主曲率，删除边缘响应点。
2. **DoG 近似 LoG**：
   - LoG (Laplacian-of-Gaussian) 算子具有各向同性的斑点响应特性，是真正的尺度归一化算子；
   - DoG 是两个相邻尺度高斯核的差分；
   - 由热扩散方程可得：∂G/∂σ ≈ σ·∇²G，即 LoG 可用相邻尺度高斯差分近似；
   - 因此 DoG 是规范化 LoG 的近似，但计算量仅为 LoG 的一小部分（无需对图像做二阶微分）。

**答题要点：**

| 检测子 | 光照 | 旋转 | 尺度 | 仿射 | 适合场景 |
|--------|------|------|------|------|---------|

| MSER | Yes | Yes | Yes | Yes | 文字检测、显著区域提取、要求仿射不变的应用 |

MSER 因具有仿射不变性，特别适合**视角变化大**的场景（如文字识别、宽基线匹配）。

**【L11-Q8】(综合应用)** 设计一个基于内容的商品图像检索系统，要求用户上传一张商品照片，系统在百万级商品库中找到相同或相似的商品。请描述系统的离线建库流程和在线检索流程，并讨论关键技术挑战。

**答题要点：**

**离线建库流程**：
1. 商品库图像预处理（去噪、统一尺寸如 256×256）；
2. 特征提取：提取全局特征（颜色直方图、纹理 GLCM）和局部特征（SIFT/SURF 检测子+描述子）；
3. 特征聚合：对局部特征采用 Bag-of-Words (BoW) 模型聚类（如 K-means, K=1000）形成视觉词袋；每张图转化为视觉词频直方图；
4. 高维索引：使用倒排索引（基于视觉词）+ 哈希索引 (LSH)；
5. 存储特征库与索引。

**在线检索流程**：
1. 用户上传查询图像，预处理后提取同样的全局+局部特征；
2. 特征量化：将 SIFT 描述子映射到最近的视觉词（NN 搜索）；
3. 查询向量构造：直方图表示；
4. 倒排索引查询：返回候选集；
5. 重排序：用空间校验（RANSAC）做几何一致性验证，过滤误匹配；
6. 返回 top-K 最相似商品。

**关键技术挑战**：
- **语义鸿沟**：相同语义商品外观差异大（不同角度/光照/遮挡），需要鲁棒特征；
- **高维索引**：SIFT 128 维 + 百万库，精确 KNN 太慢，需要 ANN 近似最近邻；
- **可扩展性**：商品类别更新时索引需要增量构建；
- **效率 vs 精度**：BoW + 倒排索引牺牲部分精度换取速度。

**【L11-Q9】(论述)** 论述全局特征与局部特征在 CBIR 中的优缺点，并解释为什么现代 CBIR 系统通常采用"局部特征 + 视觉词袋"的组合策略。

**答题要点：**

**全局特征**：颜色直方图、GLCM 纹理、形状描述子等
- 优点：计算简单、维度低、检索速度快；
- 缺点：对遮挡、裁剪、几何变换敏感；丢失空间分布信息；难以应对"语义相似但外观差异"的图像。

**局部特征**：SIFT/SURF 关键点 + 描述子
- 优点：对旋转、尺度、光照变化鲁棒；对遮挡、裁剪有一定容忍度（少量局部区域仍可匹配）；
- 缺点：单图特征数多（3000+），存储和匹配代价大；缺乏全局空间结构。

**为什么采用"局部特征 + 视觉词袋"**：
1. 利用局部特征的鲁棒性应对几何/光照变化；
2. 通过 BoW 聚类将不定数量的局部特征转化为固定维度的直方图，便于索引和比较；
3. 借鉴文本检索中 TF-IDF + 倒排索引的成熟框架；
4. 局部特征的空间几何信息可通过 RANSAC 等做后验校验，避免完全丢失结构信息。

**【L12-Q7】(简答)** 简述 ReLU 激活函数的优缺点，并解释为什么 AlexNet 使用 ReLU 而不是 sigmoid。

**答题要点：**

**ReLU 函数**：`f(x) = max(0, x)`

**优点**：
1. 正区间梯度恒为 1，**不存在梯度消失**问题；
2. 计算简单（只需比较和取最大），速度远快于 sigmoid 的指数运算；
3. 收敛速度远快于 sigmoid/tanh（实验证明训练速度快约 6 倍）；
4. 稀疏激活：负区间输出为 0，使网络具有稀疏性。

**缺点**：
1. 负区间梯度为 0，可能出现"Dead ReLU"（神经元永远不被激活）；
2. 输出不是零中心化，可能影响收敛；
3. 对学习率敏感，过大易导致大量神经元死亡。

**AlexNet 选择 ReLU 的原因**：
- 解决 sigmoid 在深层网络中的**梯度消失**问题，使得训练更深的网络成为可能；
- 收敛速度快，可大幅缩短训练时间；
- 计算简单，适合 GPU 并行。

**【L12-Q8】(简答)** 什么是 Dropout？它在训练和测试阶段的行为有什么区别？为什么 Dropout 能防止过拟合？

**答题要点：**

**Dropout 定义**：训练时以概率 p（通常 0.5）随机将神经元输出置零，使其不参与前向传播和反向传播。

**训练阶段**：每个 mini-batch 随机丢弃部分神经元，相当于在训练不同的子网络。

**测试阶段**：使用所有神经元，但需要按保留概率 (1-p) 对权重进行缩放（inverted dropout 实现时已在训练时除以 (1-p)，测试时无需缩放）。

**为什么能防止过拟合**：
1. **隐式的模型集成**：每次训练相当于训练一个不同的子网络，最终模型相当于大量子网络的集成；
2. **减少神经元共适应**：防止某些神经元只在其他特定神经元存在时才有效，迫使每个神经元学习更鲁棒的特征；
3. **类似正则化**：等价于在损失函数中加入 L2 正则项；
4. **增加噪声**：训练过程的随机性提升泛化能力。

## 第 3 部分：Cheat Sheet

### 关键公式

| 公式 | 说明 |
|------|------|
| `R = det(M) - k·trace(M)²` | Harris 角点响应函数，k = 0.04~0.06 |
| `E(u,v) ≈ [u,v] M [u,v]ᵀ` | Harris 角点灰度变化的双线性近似 |
| `(N - F)/stride + 1` | 卷积输出尺寸（无 padding） |
| `(N + 2·padding - F)/stride + 1` | 卷积输出尺寸（有 padding） |
| `(F × F × C + 1) × K` | 卷积层参数量（F 核大小, C 输入通道, K 核个数, +1 为 bias） |
| `CrossEntropy = -Σ y·log(p)` | 多分类交叉熵损失 |
| `Hinge: l(y,f(x)) = max(0, 1-y·f(x))` | SVM 风格 Hinge Loss |
| `MSE = (1/n)·Σ(y-ŷ)²` | 均方误差（回归损失） |
| `ReLU: f(x) = max(0, x)` | ReLU 激活函数（解决梯度消失） |
| `d(x,y) ≥ 0, d(x,x)=0, d(x,y)=d(y,x), d(x,k)+d(k,y)≥d(x,y)` | 距离函数四性质（非负/同一/对称/三角不等式） |

### 流程图 / 步骤列表

**SIFT 检测子步骤**：
1. 构建高斯金字塔（多分辨率金字塔 + 多层高斯滤波）
2. 组内逐层相减 → DoG 金字塔
3. 26 邻域比较找局部极值
4. DoG 曲线拟合精确定位关键点
5. Hessian 矩阵删除边缘响应点

**SIFT 描述子步骤**：
1. 计算像素梯度（模+方向）
2. 局部区块主方向统计（直方图峰值）
3. 坐标轴旋转到主方向
4. 4×4 分块 × 8 方向梯度直方图 → 128 维
5. 特征归一化

**AlexNet 网络结构**：
C1 → C2 → C3 → C4 → C5 → F6 → F7 → F8
（5 conv + 3 FC，详细：CR-P, CR-P, CR, CR, CR-P, FR-D, FR-D, FS）

**ResNet 残差块**：
- 主路径：`x → Conv → BN → ReLU → Conv → BN`
- 跳连接：`y = F(x) + x`
- 输出：`y → ReLU`

**深度学习三部曲**：
1. 搭建网络结构（CNN/RNN/Transformer）
2. 选择损失函数（Cross Entropy / MSE / Hinge）
3. 选择优化函数（SGD / Adam / RMSprop），反向传播更新参数

**CBIR 系统流程**：
- 离线：库图像 → 特征提取 → 索引
- 在线：查询图像 → 特征提取 → 特征匹配 → 检索结果

### 易混淆点

| 易混淆 | 区别 |
|--------|------|
| **Harris vs Harris-Laplace** | Harris 单尺度，不抗尺度；Harris-Laplace 在尺度空间搜索特征尺度，抗尺度变化 |
| **SIFT 检测子 vs SIFT 描述子** | 检测子 = 在 DoG 尺度空间找关键点位置；描述子 = 在关键点邻域提取 128 维梯度直方图向量 |
| **SIFT vs SURF** | SURF 金字塔图片大小不变，只改高斯核尺度；SIFT 逐层降采样。SURF 速度更快但光照鲁棒性略弱 |
| **DoG vs LoG** | LoG 是规范化 Laplacian；DoG 是相邻高斯差分；DoG 是 LoG 的近似但计算更简单 |
| **全局特征 vs 局部特征** | 全局特征 = 整图统计（颜色直方图），简单但缺不变性；局部特征 = 关键点描述子（SIFT），鲁棒但复杂 |
| **卷积层 vs 全连接层** | 卷积 = 局部连接 + 参数共享，参数少；全连接 = 全连接，参数多（VGG 138M 大部分在 FC） |
| **卷积 vs 池化** | 卷积有可学习参数（卷积核权重）；池化无参数（Max/Avg 固定运算） |
| **MaxPool vs AvgPool** | Max 取局部最大值，保留纹理；Avg 取局部平均，平滑特征 |
| **ReLU vs sigmoid** | ReLU 正区间不饱和，无梯度消失；sigmoid 两端饱和易梯度消失 |
| **Dropout 训练 vs 测试** | 训练时随机失活部分神经元；测试时使用全部（需缩放权重或 inverted dropout） |
| **Inception V2 vs V3** | V2 = 引入 1×1 卷积降维；V3 = 用小卷积核替代大卷积核（如 5×5 → 两个 3×3） |
| **梯度消失 vs 深度退化** | 梯度消失 = 反向传播梯度趋零，无法训练；深度退化 = 训练误差随深度增加而增大（ResNet 解决的问题） |
| **语义鸿沟 vs 意图鸿沟** | 语义鸿沟 = 低层特征↔高层概念；意图鸿沟 = 用户查询↔内容文本描述 |
| **距离 vs 相似度** | 距离越小越相似；可转换 d → s = e^(-d²/2σ²) 或 s = 1/(1+d) |
| **CBIR vs 文本检索** | CBIR 是相似性检索（用距离度量）；文本检索是精确匹配（用倒排索引） |
| **静态图 vs 动态图** | TF 1.x 静态图（先定义后运行，性能好但调试难）；PyTorch/TF 2.x 动态图（Define-by-Run，调试直观） |
