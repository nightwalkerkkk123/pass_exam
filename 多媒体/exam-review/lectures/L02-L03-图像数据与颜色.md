## 第 1 部分：考点清单 (Exam Point Checklist)

### 第二讲 (L02) 考点

- **【L02-考点1】像素与图像分辨率**
  - 核心概念：像素 (Pixel) = picture element，图像的最小离散单位；图像分辨率 = 像素总数
  - 关键数字/公式：标准图 Lena 256×256（灰度），Baboon 512×512（彩色）
  - 易错点：分辨率 ≠ 屏幕尺寸；屏幕 DPI 是物理密度
  - 考频预判：高频

- **【L02-考点2】灰度图像与 Gray 转换公式**
  - 核心概念：8-bit 灰度图每个像素取值 0-255；Pillow 用 `convert('L')` 转灰度
  - 关键数字/公式：`Gray = 0.299·R + 0.587·G + 0.114·B`（ITU-R BT.601）
  - 易错点：系数非平均 1/3；G 通道权重最高（人眼对绿最敏感）
  - 考频预判：高频（必考计算）

- **【L02-考点3】二值图像**
  - 核心概念：每个像素 1 bit (0/1)；用于简单图形/文字
  - 关键公式：阈值化 `point(lambda x: 255 if x > 127 else 0)`
  - 易错点：阈值 127 不是固定值，Otsu 可自适应选取
  - 考频预判：中频

- **【L02-考点4】位平面 (Bitplane) 分解**
  - 核心概念：8-bit 图像可分解为 8 个 1-bit 位平面
  - 关键数字/公式：bitplane_k = (pixel >> k) & 1
  - 易错点：高阶位 (高 bit) 决定图像结构；低阶位 (低 bit) 含噪声/细节
  - 应用：图像压缩、加密隐写、去噪、信息分析
  - 考频预判：高频

- **【L02-考点5】24-bit 真彩色图像**
  - 核心概念：R(8-bit) + G(8-bit) + B(8-bit) 共 3 字节/像素；支持 256³ ≈ 16.8M 色
  - 关键数字/公式：640×480 的 24-bit 图 ≈ 900KB (640×480×3/1024)
  - 易错点：32-bit 图 = 24-bit + 8-bit 特效通道 (alpha)
  - 考频预判：高频

- **【L02-考点6】直方图 (Histogram)**
  - 核心概念：统计每个灰度值出现的像素数；8-bit 直方图长度 256
  - 关键公式：`np.histogram / np.bincount`；Bin 范围 [0, 255]
  - 易错点：直方图相同 ≠ 图像相同；可反映对比度、亮度
  - 考频预判：高频

- **【L02-考点7】8-bit 伪彩色与颜色查找表 (CLUT)**
  - 核心概念：每个像素只存 8-bit 索引；调色板 256 项；适合颜色数少的图
  - 关键数字/公式：8-bit color = R(3-bit) + G(3-bit) + B(2-bit) = 8-bit
  - 易错点：3:3:2 分箱导致"边缘伪影"；调色板可基于直方图选最频 256 色
  - 考频预判：高频

- **【L02-考点8】图像数据量计算**
  - 核心概念：宽×高×每像素 bit 数 = 总 bit；÷8 = Byte；÷1024 = KB
  - 关键公式：`size = W × H × B / 8` (Bytes)
  - 易错点：单位混淆 (b vs B, Kb vs KB)；24-bit 灰度是 8-bit 灰度的 3 倍
  - 考频预判：高频（必考计算）

- **【L02-考点9】Dithering（抖动 / 网板打印）**
  - 核心概念：用 1-bit 打印机表现灰度图；用空间分辨率换强度分辨率
  - 关键公式：有序抖动 `O(x,y) = 1 if I(x,y) > D(i,j) else 0`，i=x mod n, j=y mod n
  - 关键矩阵：4×4 Bayer 抖动矩阵 D（值 0-15）
  - 易错点：强度大的区域 on dot 多；常用于半色调打印
  - 考频预判：中频

### 第三讲 (L03) 考点

- **【L03-考点1】光与可见光谱**
  - 核心概念：光 = 电磁波；可见光范围 400-700 nm
  - 关键数字：nm = 10⁻⁹ m；SPD E(λ) 表示光谱能量分布
  - 易错点：波长 λ 单位容易写错；V(λ) 在 555 nm 处最高（黄绿最亮）
  - 考频预判：中频

- **【L03-考点2】人眼视觉模型**
  - 核心概念：视网膜由柱状细胞 (rod) + 三种视锥细胞 (cone: R/G/B) 组成
  - 关键概念：柱状细胞负责低照度黑白视觉；视锥细胞负责彩色
  - 关键公式：光谱灵敏度向量 q(λ) = (qR(λ), qG(λ), qB(λ))ᵀ
  - 易错点：眼睛三种"颜色接收器"敏感度不同
  - 考频预判：中频

- **【L03-考点3】色度匹配函数与 CIE XYZ**
  - 核心概念：CIE RGB 匹配函数 r̄(λ), ḡ(λ), b̄(λ) 存在负 lobe；引入 XYZ 虚拟原色使其全非负
  - 关键公式：3×3 线性变换 (r̄,ḡ,b̄) → (x̄,ȳ,z̄)，其中 ȳ(λ) = V(λ)
  - 易错点：Y 表亮度，X 和 Z 描述色相+饱和度
  - 考频预判：中频

- **【L03-考点4】CIE 色度图与色域 (Gamut)**
  - 核心概念：白点 (1/3, 1/3)；光谱轨迹 (spectrum locus)；色域 = 设备可表示颜色子集
  - 关键概念：超色域 (out-of-gamut) 颜色 = 该颜色到白点的连线与设备色域边界的交点
  - 易错点：色域是三角形但色度空间是马蹄形
  - 考频预判：中频

- **【L03-考点5】RGB 加色模型**
  - 核心概念：红+绿+蓝光叠加；设备相关；用于屏幕显示
  - 关键数字：每通道 8-bit，组合 16.8M 色
  - 易错点：RGB 是 device-dependent；不同设备同一 RGB 值不同
  - 考频预判：高频

- **【L03-考点6】Gamma 校正 (γ 校正)**
  - 核心概念：CRT 发光强度 ∝ 电压^γ（γ ≈ 2.2）；故存储时要 inverse gamma 预校正
  - 关键公式：显示亮度 = (V)^(1/γ)，存储编码 = V^(1/γ)
  - 易错点：γ 校正用于显示；也用于图像处理（如增强对比度）
  - 考频预判：高频

- **【L03-考点7】CMY(K) 减色模型**
  - 核心概念：印刷用；C = -R, M = -G, Y = -B；K = 黑（节省墨水）
  - 关键公式：`C = 1 - R, M = 1 - G, Y = 1 - B`（归一化）
  - 易错点：CMY 混合得不到真黑，故加 K
  - 考频预判：中频

- **【L03-考点8】HSV / HSL 模型**
  - 核心概念：H (色调 0-360°)、S (饱和度 0-1)、V/L (明度/亮度 0-1)
  - 关键用法：调色用 HSV，设计用 HSL
  - 易错点：HSV 是 hexcone 模型，HSL 是双六棱锥；二者不同
  - 考频预判：高频

- **【L03-考点9】CIELAB (L\*a\*b\*) 色彩模型**
  - 核心概念：L\* = 亮度 (0-100)；a\* = 红(-)绿(+)，b\* = 黄(-)蓝(+)
  - 关键特性：视觉均匀 (Weber 定律) —— 同样数值变化 ≈ 同样感知差异
  - 易错点：L\*a\*b\* 不是色域，是感知空间
  - 考频预判：高频

- **【L03-考点10】YUV / YIQ / YCbCr 视频颜色模型**
  - 核心概念：Y = 亮度 (luma)，U/V 或 Cb/Cr = 色度 (chroma)
  - 关键数字：Y: 16-235；Cb/Cr: 16-240（数字域，TV 标准）
  - 关键应用：PAL/SECAM 用 YUV；北美用 YIQ；JPEG/MPEG 用 YCbCr
  - 易错点：YCbCr ≠ YUV；YCbCr 是 YUV 的 scale+offset 版本
  - 考频预判：高频

**【L02-Q1】(单选)** 一幅 1024×768 的 24-bit 真彩色图像，未压缩时其存储容量约为：
- A. 2.25 MB
- B. 2.34 MB

**答案：** A

**【L02-Q2】(单选)** 关于 Gray 转换公式 `Y = 0.299R + 0.587G + 0.114B`，下列说法正确的是：
- A. 三个系数相等
- B. R 通道权重最大
- C. G 通道权重最大，因为人眼对绿色最敏感
- D. B 通道权重最大，因为蓝色视觉敏感度低需补偿

**答案：** C
**解析：** 系数 0.587 (G) > 0.299 (R) > 0.114 (B)，反映了人眼视锥细胞对绿光最敏感的事实，对应 ITU-R BT.601 标准。

**【L02-Q6】(简答)** 简述 24-bit 彩色图像转换为 8-bit 伪彩色图像的两种方法及其优缺点。

**答案要点：**
1. **均匀分箱法 (Uniform Quantization)**：R 取 3-bit, G 取 3-bit, B 取 2-bit，每像素 8-bit。优点：实时、无需训练；缺点：颜色分布不均，出现"边缘伪影"。
2. **流行色法 (Popularity / Median-cut)**：基于直方图选 256 个最频繁颜色作为调色板 (CLUT)。优点：视觉效果好；缺点：需要统计计算、可能丢失稀有颜色。

**解析：** 两种方法都把每像素颜色降到 8-bit 索引，通过查找表 (LUT) 还原 24-bit 真彩色；调色板大小固定为 256。

**【L02-Q7】(论述)** Lena 图像为何成为图像处理领域的"标准测试图"？为什么后来受到批评？

**答案要点：**
1. **广泛使用的原因**：包含平滑区、纹理区、边缘、细节、阴影等多种特征；中等复杂度 (256×256) 适合早期算法；来源 Playboy 杂志 (1972)。
2. **批评点**：(a) 来源不当，性别歧视争议；(b) 不够现代，复杂场景代表性差；(c) 主流期刊 (IEEE 等) 已弃用，转向 BSDS500、COCO 等数据集。
3. **新趋势**：现代 benchmark 使用 Kodak、BSD500、Set5/Set14、Urban100 等。

**解析：** 标准测试图选图原则：内容多样、复杂度适中、可复现。Lena 是历史选择，现代评测需更公允。

**【L02-Q8】(综合应用)** 给定一张 640×480 的彩色照片，要求将其用于黑白喷墨打印机打印。请设计完整的处理流程，并估算压缩前后的数据量。

**答案要点：**
1. **彩色 → 灰度**：用 `Gray = 0.299R + 0.587G + 0.114B`，数据量 640×480×1 = 307,200 B ≈ 300 KB。
2. **灰度 → 二值 (Dithering)**：用 4×4 Bayer 抖动或 Floyd-Steinberg 误差扩散；数据量 640×480/8 = 38,400 B ≈ 37.5 KB。
3. **数据量比较**：原图 900 KB → 灰度 300 KB → 二值 37.5 KB (压缩 24 倍)。
4. **效果对比**：直接阈值化 (无 dithering) 会丢失中间灰度细节；dithering 用空间分辨率换强度分辨率，保留视觉层次。

**解析：** 此流程在传真、报纸印刷、低成本打印场景常见。

**【L03-Q5】(计算题)** 已知一像素 RGB 值 (R=200, G=100, B=50)（归一化到 [0,1] 时 R=0.784, G=0.392, B=0.196），求：
(1) 对应的 CMY 值；

**【L03-Q6】(简答)** 比较 HSV 与 HSL 颜色模型的异同。

**答案要点：**
- **相同**：都用 H (色相 0-360°) + S (饱和度 0-1) + 第三分量 (V 或 L 0-1)；都从 RGB 转换；适合交互式调色。
- **不同**：
  - HSV (Value)：hexcone 六棱锥模型；V = max(R,G,B)；饱和时颜色纯。
  - HSL (Lightness)：bi-hexcone 双六棱锥模型；L = (max+min)/2；纯白纯黑都可达。
- **应用**：HSV 用于图像分割/算法 (OpenCV)；HSL 用于 UI 设计/CSS/PS。

**解析：** 二者都解决 RGB "调色调不直观"的问题，区别在第三维的语义。

**【L03-Q7】(论述)** 详细说明为什么数字视频和 JPEG 压缩普遍使用 YCbCr 而不是 RGB。

**答案要点：**
1. **人眼特性**：对亮度 (Y) 变化敏感，对色度 (Cb/Cr) 不敏感。
2. **可分离压缩**：可对 Cb/Cr 下采样 (4:2:0、4:2:2) 节省带宽，视觉影响小。
3. **兼容电视系统**：源自模拟电视 (PAL 用 YUV、NTSC 用 YIQ)，数字时代统一为 YCbCr。
4. **亮度色度独立**：便于黑白电视兼容；亮度信号可独立传输。
5. **范围标准化**：Y: 16-235, Cb/Cr: 16-240，给同步和滤波留 headroom。

**解析：** YCbCr 是 JPEG/MPEG/H.264 标准的颜色空间基础。

**【L03-Q8】(综合应用)** 给定一张 1920×1080 的 RGB 彩色图像，分别计算在 RGB 和 YCbCr 4:2:0 子采样下的存储容量（每像素 8-bit）。

**答案：**
1. **RGB (24-bit)**：1920×1080×3 = 6,220,800 B ≈ 5.93 MB
2. **YCbCr 4:2:0**：每 2×2 像素块共用 Y(4)+Cb(1)+Cr(1) = 6 字节，平均每像素 1.5 字节。
   - 总字节数 = 1920×1080×1.5 = 3,110,400 B ≈ 2.97 MB
3. **压缩比**：约 2:1，且视觉无明显损失

**解析：** 4:2:0 表示色度在水平和垂直方向都减半采样，是 JPEG、MPEG、HEVC 的默认子采样格式。

---

**【L03-Q9】(简答)** CIE RGB 匹配函数为何要引入 XYZ 虚拟原色？

**答案要点：**
1. **负 lobe 问题**：CIE RGB 匹配函数 r̄(λ) 在某些波长出现负值，物理上需"负光强"，不直观。
2. **虚拟原色 XYZ**：通过 3×3 线性变换，使匹配函数 x̄(λ), ȳ(λ), z̄(λ) 全部非负。
3. **约束条件**：ȳ(λ) ≡ V(λ) (光视效率曲线)，便于把 Y 直接作为亮度。
4. **意义**：建立设备无关的标准基色空间，所有颜色空间转换以 XYZ 为中转。

**解析：** 这是颜色科学标准化的关键步骤，使所有 RGB 设备都能映射到统一基准。

---

**【L03-Q10】(论述)** 详细分析 Gamma 校正的原理、作用与典型应用。

**答案要点：**
1. **原理**：CRT 显示的发光强度 L ∝ V^γ（γ ≈ 2.2），即电压与亮度非线性。
2. **校正方法**：存储图像时预校正 s = V^(1/γ)，使显示还原后 L = (V^(1/γ))^γ = V，线性还原。
3. **作用**：
   - **视觉感知**：人眼对暗部变化敏感，γ 校正匹配 Weber-Fechner 定律。
   - **编码效率**：用有限 bit 数更好地分配亮度信息，避免暗部 banding。
4. **典型 γ 值**：CRT 2.2；Mac OS 1.8；sRGB 2.2；印刷用 1.8 (L*).
5. **应用场景**：图像显示、图像处理 (gamma 调整增强对比度)、HDR 编码、视频传输。

**解析：** Gamma 校正是数字图像管线的"必修课"，缺失会导致图像偏暗或细节丢失。

### 流程图 / 步骤列表

**24-bit RGB → 8-bit 伪彩色（CLUT）步骤：**
1. 统计 RGB 立方体直方图
2. 量化分箱 (R:3-bit, G:3-bit, B:2-bit) OR 选 256 个最频色
3. 建立调色板 (LUT) 256 项 × 24-bit
4. 每个像素 → 索引 (8-bit)
5. 解码时查表还原 24-bit

**Dithering 流程：**
1. 选择 Bayer 矩阵 (4×4 或 8×8)
2. 对每个像素 (x, y)：
   - i = x mod n, j = y mod n
   - if I(x,y) > D(i,j): O(x,y) = 1 (on)
   - else: O(x,y) = 0 (off)

**RGB → YCbCr (BT.601) 步骤：**
1. Y  = 0.299·R + 0.587·G + 0.114·B
2. Cb = 128 + (-0.169·R - 0.331·G + 0.500·B)
3. Cr = 128 + (0.500·R - 0.419·G - 0.081·B)
4. 范围裁剪到 [16, 235] / [16, 240]
