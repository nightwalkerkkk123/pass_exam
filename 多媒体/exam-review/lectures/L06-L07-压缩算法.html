L06 无损压缩(熵/Huffman/Shannon-Fano/算术/LZW/RLC) + L07 有损压缩(失真/PSNR/率失真/量化/DCT/IDCT) · 10 + 10 考点

I.考点清单 (Exam Point Checklist)

第六讲 (L06) 无损压缩算法

L06-考点 1

压缩的必要性与冗余类型

核心
压缩 = 去除冗余；四类冗余：时间冗余 (Temporal, 帧间)、空间冗余 (Spatial, 相邻像素)、频谱冗余 (Spectral, 颜色/亮度分量相关)、心理视觉冗余 (Psycho-visual)。
数字
512×512 24-bit 彩色图 = 0.75 MB；4032×3024 24-bit ≈ 34 MB；HDTV 1920×1080@25fps 真彩 ≈ 8.7 GB/min。
易错
时间冗余是帧与帧之间（视频），空间冗余是像素与像素之间（图像内）。
频率
高频
L06-考点 2

无损 vs 有损压缩

核心
无损解压后得到原数据完全拷贝（信息无丢失）；有损解压得到"近似"（信息丢失）。
示例
无损 — Shannon-Fano、Huffman、算术编码、LZW；有损 — JPEG、MPEG。
易错
无损压缩率一般较小，有损压缩率大；不能说"无损一定好"，要看应用场景（图像归档 vs 视频流媒体）。
频率
高频
L06-考点 3

压缩比 (Compression Ratio)

核心
压缩前比特数 / 压缩后比特数。
公式
CR = B0 / B1，B0 = 压缩前比特数，B1 = 压缩后比特数。CR > 1 表示压缩，CR = 1 表示无变化。
易错
是 B0/B1 不是 B1/B0；压缩比和压缩效率不同（效率 = (B0-B1)/B0）。
频率
高频 · 必考计算
L06-考点 4

信息熵 (Entropy)

核心
源 S 中每个符号包含的平均信息量；单位 bit/symbol。
公式
自信息 I(si) = -log2(pi)；熵 η = Σ pi · I(si) = -Σ pi·log2(pi)。
易错
等概率分布时熵最大；熵是无损压缩的理论下限（编码平均长度 l̄ ≥ η）。
频率
高频 · 必考计算
L06-考点 5

Huffman 编码

核心
1952 年 Huffman 提出；JPEG/MPEG/传真机均采用；自底向上构造二叉树。
步骤
① 按频次排序；② 取出频次最低两个节点合并为父节点（频次相加）；③ 重复直至只剩一个根；④ 左 0 右 1（或反过来）赋码字。
性质
唯一前缀码（prefix-free，任意码字不是另一码字的前缀）→ 解码无歧义；最优性 η ≤ l̄ < η + 1。
易错
Huffman 对给定概率模型是最优，但若概率估计不准则整体未必最优；不能处理动态概率（除非自适应 Huffman）。
频率
高频 · 必考构造题
L06-考点 6

Shannon-Fano 编码

核心
自顶向下；将符号集按频次排序后递归二分使两部分频次和近似相等。
易错
Shannon-Fano 不保证最优，可能产生非最优码（虽然满足前缀性）；Huffman 一定优于 Shannon-Fano（在同一概率模型下）。
频率
中频
L06-考点 7

算术编码 (Arithmetic Coding)

核心
把整条消息视为一个单位，输出 [0,1) 区间内的一个实数；通常优于 Huffman。
公式
每读入一个符号 s，将当前区间 [low, high) 缩窄为 low + range·Range_low(s), low + range·Range_high(s)，其中 range = high - low。
易错
整条消息→一个码字（不是每个符号→一个码字）；输出码字满足 low ≤ codeword < high；需要终止符 $ 标识结束。
频率
高频
L06-考点 8

游程编码 (Run-Length Coding, RLC)

核心
用 (count, symbol) 替代连续重复符号；最简单无损压缩。
示例
WWWWBWWBWWWWWWB → (4,W)(1,B)(2,W)(1,B)(6,W)(1,B)。
易错
只对重复多的数据有效；图像里相邻像素常相近但不一定相等，常配合量化或预测使用。
频率
中频
L06-考点 9

LZW 字典编码

核心
固定长码字（区别于 Huffman 的变长）；把新字符串加入字典并输出其索引。
数字
GIF 字典大小 4096 (2^12)；初始字典 2^l0 个条目；码长 l ∈ [l0, lmax]。
示例
ABABBABCABABBA 初始 A=1, B=2, C=3，输出码 1 2 4 5 2 3 4 6 1，压缩比 14/9 ≈ 1.56。
易错
编/解码器动态构建相同字典；压缩比定义可能与前面不同（原文/编码后字符数比）。
频率
中频
L06-考点 10

熵编码 (Entropy Coding) 总结

核心
所有熵编码都是无损；本质是让高概率符号用短码、低概率符号用长码；理论下限是熵 η。
对比
Huffman（整数码字，对字符逐个编码）vs 算术编码（区间实数，对整条消息编码，逼近熵）。
易错
熵编码 ≠ 压缩全部，只是压缩流水线的一步（前端需先做变换/预测去除相关性）。
频率
高频

第七讲 (L07) 有损压缩算法

L07-考点 1

有损压缩的目的与定位

核心
有损是"手段"，目的是获取高压缩率；解压后是"近似"而非原数据。
易错
有损 ≠ 失真越大越好——需要在率失真 (Rate-Distortion) 曲线上找平衡。
频率
中频
L07-考点 2

失真度量 (Distortion Measures)

核心
衡量压缩前后差异的指标。
公式
均方差 MSE σ²d = (1/N) · Σ(xi - yi)²；信噪比 SNR = 10·log10(σ²x / σ²d) (dB)；峰值信噪比 PSNR = 10·log10(x²peak / σ²d) (dB，8-bit 灰度 x_peak = 255)。
易错
SNR 分母是噪声方差（不是信号方差）；PSNR 对 8-bit 信号 x_peak=255；单位都是 dB（10 倍对数，不是 20 倍！）。
频率
高频 · 必考计算
L07-考点 3

率失真理论 (Rate-Distortion)

核心
R(D) 函数 — 给定可容忍失真 D，能编码源数据的最低比特率 R。
思想
率失真曲线下凸；D=0 时 R = 熵（无损极限）；R=0 时 D 最大。
易错
R(D) 是理论下限，实际编码只能接近，不能达到（除非理想情况）。
频率
中频
L07-考点 4

量化 (Quantization) 基础

核心
有损压缩"信息丢失"的主要来源；将连续取值映射到离散集合。
示例
原 0~255 → 8 个量化级 {16, 48, 80, 112, 144, 176, 208, 240}，间距 32。
易错
量化 ≠ 采样（采样是时间/空间离散化，量化是取值离散化）。
频率
高频 · 必考简答+计算
L07-考点 5

标量量化 vs 向量量化

核心
均匀标量量化：等间距划分区间。非均匀标量量化：不等间距（Lloyd-Max、压扩量化）。向量量化 (VQ)：把 n 维向量映射到码本中的码向量；编/解码器共享码本。
易错
VQ 通常优于标量量化（利用分量间相关性），但码本设计（训练）代价高。
频率
中频
L07-考点 6

变换编码 (Transform Coding) 思想

核心
把数据从空间域变换到频率域；少数低频系数携带大部分能量，其余可粗量化或丢弃。
示例
X = {100, 101, 100, 99, 100, 100, 100, 100} 做差分变换 T: d1 = x1, di = xi - xi-1 → Y = {100, 1, -1, -1, 1, 0, 0, 0}，能量集中。
易错
变换本身无损，"损失"来自对变换系数的量化。
频率
高频
L07-考点 7

DCT (离散余弦变换)

核心
JPEG/MPEG Video/MPEG Audio 的"心脏"；与 DFT 相比 DCT 只有实数，无相位偏移问题，能量更集中。
公式
1D DCT：F(u) = C(u) · Σ f(i) · cos[(2i+1)uπ/(2N)]；C(0)=√(1/N), C(u>0)=√(2/N)。
关键
2D DCT 可分离为两次 1D DCT，迭代从 8×8 → 8+8 步。
易错
DCT 不是无损的——不丢信息是因为 IDCT 可逆，但量化步骤才丢信息。
频率
高频 · 必考
L07-考点 8

DCT 系数的物理意义：DC vs AC

核心
DC 分量 (u=0)：信号的平均强度/亮度；图像中即"整体灰度"。AC 分量 (u>0)：周期性变化的频率分量；图像中对应边缘/纹理细节。
易错
图像低频 = 平滑区域/整体亮度；高频 = 边缘/细节 → 高频系数小则可粗量化（人眼不敏感）。
频率
高频
L07-考点 9

DCT 在图像压缩中的应用流程

步骤
① 读灰度图；② 8×8 分块；③ 对每块做 2D DCT；④ 保留左上 k×k（低频），其余置 0；⑤ IDCT 重建。
易错
分块大小常用 8×8（权衡：太大→边界效应，太小→压缩率低）；低频保留多少是质量与压缩率的权衡。
频率
高频 · 综合题
L07-考点 10

傅里叶变换 vs DCT

核心
FT 复数输出，有相位；DCT 实数输出，更紧凑的能量聚集；FT 用于 MPEG Audio，DCT 用于 JPEG/MPEG Video。
易错
DCT 边界处理更优（无 Gibbs 现象明显）；K-L 变换理论最优但无快速算法，DCT 是次优但有快速算法。
频率
中频

L06-Q2单选 字母表 S = {a, b, c, d}，概率 P = {1/2, 1/4, 1/8, 1/8}，则该信源的熵 η 为多少 bit/symbol？

  • B. 1.75
  • C. 2.0
  • D. 2.5
显示答案与解析
答案 B 解析 η = -Σ pi·log2(pi) = -(1/2·1 + 1/4·2 + 1/8·3 + 1/8·3) = 0.5 + 0.5 + 0.375 + 0.375 = 1.75 bit/symbol。

L06-Q3多选 关于 Huffman 编码，下列说法正确的是？

  • A. 是一种变长编码
  • B. 满足唯一前缀性质，码字之间无歧义
  • C. 平均码长一定等于熵
  • D. 对于给定概率模型是最优的
显示答案与解析
答案 A, B, D 解析 Huffman 平均码长满足 η ≤ l̄ < η+1，等于熵仅在所有 -log2(pi) 都是整数时才成立，所以 C 错。

L06-Q4计算题 字符串 "ACABADADEAABBAAAEDCACDEAAABCDBBEDCBACAE"（39 个字符），符号频次：A=15, B=7, C=6, D=6, E=5。求 Huffman 编码的平均码长（单位：bit/symbol）以及与理想熵的差。

显示答案与解析
答案 平均码长 l̄ = 2.23 bit/symbol，理想熵 η = 2.19 bit/symbol，差 0.04 bit/symbol。 解析 由讲义示例：(1×15 + 3×7 + 3×6 + 3×6 + 3×5) / 39 = (15+21+18+18+15)/39 = 87/39 ≈ 2.23。理想熵 η = -Σ pi·log2(pi) = 2.19。满足 η ≤ l̄ < η+1。

L06-Q5计算题 一幅 512×512 的 8-bit 灰度图像（每像素 1 字节，原始数据量 256 KB = 2,097,152 bit）。使用 Huffman 编码后平均码长 2.5 bit/pixel。求压缩比 CR。

显示答案与解析

L06-Q7简答题 简述 Huffman 编码的构造步骤，并说明为什么 Huffman 码满足"唯一前缀性质"？

显示答案与解析
答案要点
1. 把所有符号按出现频率从大到小（或从小到大）排成一个有序表。
2. 取出频率最小的两个符号作为左右子树，新节点的频率为两子节点之和。
3. 把新节点重新插入有序表保持有序。
4. 重复 2-3 直到表中只剩一个节点（根）。
5. 对每条边赋 0（左）/ 1（右），从根到叶的路径即码字。
唯一前缀性的原因：码字对应二叉树的叶子节点，叶子节点无子女，因此任何码字都不是其他码字的前缀（前缀性）。这保证了解码时可以即时匹配（遇到码字立即停止）而不会产生歧义。

L06-Q8简答题 比较 Huffman 编码和算术编码。算术编码为什么通常能优于 Huffman？

显示答案与解析
答案要点
  • Huffman：每个符号独立编码为一个整数比特码字；码长受整数限制，最坏情况下平均码长 ≥ η。
  • 算术编码：把整条消息视为一个单位，输出 [0,1) 区间内的一个实数；理论上可以让码长任意接近熵 η。
  • 算术编码更优的原因：① 对整条消息编码，去除了符号间的整数比特限制；② 当某些符号概率使 -log2(pi) 为非整数时，Huffman 平均码长只能 ≥ ⌈-log2(pi)⌉ 而算术编码可以分配非整数比特；③ 算术编码天然支持自适应概率更新。
  • 代价：实现复杂、需要浮点运算（早期）、专利问题（已过期）。

L06-Q9综合应用题 给定字符串 S = "AAAAAAAAAAABBBBBCCCCCCCDDDDDDDDE"（10A+5B+7C+8D+1E，共 31 字符）。要求：
(1) 构造 Huffman 树并给出每个符号的码字；
(2) 计算平均码长 l̄；
(3) 计算压缩比 CR（对比 5-bit 定长编码）。

显示答案与解析
答案

(1) Huffman 树构造（频次 E=1, B=5, C=7, D=8, A=10）：
- 取 E(1), B(5) 合并 → 节点 EB(6)；表序: C(7), D(8), A(10), EB(6)
  • 重新排序：D(8), A(10), C(7), EB(6)
  • 取 EB(6), C(7) 合并 → 节点 EB-C(13)；表序: A(10), EB-C(13), D(8)
  • 重新排序：EB-C(13), D(8), A(10)
  • 取 D(8), A(10) 合并 → 节点 D-A(18)；表序: EB-C(13), D-A(18)
  • 合并 → 根(31)

码字（左 0 右 1）：
- A = 11 (2 bit)
- D = 10 (2 bit)
- C = 00 (2 bit)
- B = 010 (3 bit)
- E = 011 (3 bit)

(2) l̄ = (10×2 + 8×2 + 7×2 + 5×3 + 1×3) / 31 = (20+16+14+15+3)/31 = 68/31 ≈ 2.19 bit/symbol

(3) 5-bit 定长：31 × 5 = 155 bit；Huffman 后：68 bit；CR = 155/68 ≈ 2.28。

解析 Huffman 构造的关键是每次取频次最小的两个节点合并。注意：本题结果中 A、C、D 都是 2 bit，频次最高的 A 反而没拿到最短的码（因为 Huffman 考虑的是整体最优，而不是简单地让最高频最短）。

L06-Q10简答题 解释什么是"熵"，并说明为什么无损压缩的平均码长一定 ≥ 熵。

显示答案与解析
答案要点
  • 熵：源 S 中每个符号所包含的平均信息量，定义为 η = -Σ pi·log2(pi)，单位 bit/symbol。
  • 物理意义：熵是消除一个符号平均不确定性所需的最少比特数；即无损压缩理论下界。
  • 为什么 l̄ ≥ η：考虑任意无损编码方案，对每个符号 si 分配长度 li 的码字。Kraft 不等式给出 Σ 2^(-li) ≤ 1，在该约束下，使加权平均 Σ pi·li 最小化的最优解正好是 li = -log2(pi)（即 Shannon 编码）。此时平均码长恰好是熵 η。实际编码若不能实现非整数长度（如 Huffman 用整数），则 l̄ 只能 ≥ η，且最优时满足 η ≤ l̄ < η+1。

L07-Q1单选 下列哪个指标最适合用于衡量有损压缩重建图像与原图像的像素级差异？

  • A. 压缩比
  • B. 熵
  • C. PSNR
  • D. 帧率
显示答案与解析
答案 C 解析 PSNR（峰值信噪比）以 dB 为单位衡量重建信号与原信号的失真，是有损图像/视频压缩的标准评价指标。压缩比衡量压缩效率，熵衡量信息量，帧率衡量时间分辨率。

L07-Q2单选 在 JPEG 编码中，量化步骤的"信息损失"主要是因为？

  • A. DCT 变换不是可逆的
  • B. 量化把连续/多值映射到离散/少值集合
  • C. Huffman 编码使用了整数码字
  • D. 图像被分成了 8×8 块
显示答案与解析
答案 B 解析 DCT 本身可逆（IDCT 可完美恢复），Huffman 是无损的，8×8 分块本身也不丢信息。真正的信息丢失来自量化——它把系数除以量化步长并取整，导致精度下降且不可逆。

L07-Q3多选 关于 DCT 的说法正确的是？

  • A. DCT 是 JPEG 的核心
  • B. DCT 的输出都是实数
  • C. DCT 的 DC 分量代表信号的平均能量
  • D. DCT 与 DFT 相比能量更集中
显示答案与解析
答案 A, B, C, D 解析 四项都正确。DCT 在 JPEG 中将空间域变换到频率域；输出实数；DC 分量（u=0）正比于信号均值；相比复数 DFT，DCT 边界更平滑，能量更集中。

L07-Q4计算题 一幅 256×256 的 8-bit 灰度图像，原图与重建图的 MSE = 100。求 PSNR（dB）。

显示答案与解析
答案 PSNR ≈ 28.13 dB 解析 对 8-bit 灰度图，x_peak = 255，x²_peak = 255² = 65025。PSNR = 10·log10(65025/100) = 10·log10(650.25) ≈ 10 × 2.813 ≈ 28.13 dB。常用经验：MSE=100 → PSNR≈28dB。

L07-Q5计算题 一段音频信号 σ²x = 10000（信号方差），压缩后引入噪声方差 σ²d = 100。求 SNR（dB）。若信号峰值为 32000（16-bit 满量程），求 PSNR（dB）。

显示答案与解析
答案 SNR = 20 dB，PSNR ≈ 70.1 dB 解析 SNR = 10·log10(σ²x/σ²d) = 10·log10(10000/100) = 10·log10(100) = 20 dB。PSNR = 10·log10(32000²/100) = 10·log10(1.024×10^7/100) = 10·log10(102400) ≈ 50.1 + 20 = 70.1 dB。

L07-Q6计算题 均匀标量量化。原 8-bit 灰度（0~255），用 8 个量化级 {16, 48, 80, 112, 144, 176, 208, 240} 表示。求量化步长 Δ，以及将原值 100 量化到哪个级。

显示答案与解析
答案 Δ = 32；100 量化到 112 解析 8 个量化级间距 (240-16)/7 = 32（注意步长 = (max - min)/(N-1)，因为 N 个值有 N-1 个间隔）。100 所在区间：80 ≤ 100 < 112（按中心点划分），或按距离最近原则：|100-80|=20, |100-112|=12，距离 112 更近 → 量化到 112。

L07-Q7简答题 为什么 DCT 比 DFT 更适合图像压缩？说明 3 个理由。

显示答案与解析
答案要点
1. 实数 vs 复数：DCT 输出全为实数，无需处理相位信息；DFT 输出复数，需额外存储相位，效率低。
2. 能量集中：DCT（特别是 Type-II DCT）的能量聚集能力优于 DFT，少数低频系数即可携带图像绝大部分能量，便于后续粗量化高频系数。
3. 边界连续性：DCT 隐含偶对称延拓，图像块边界处更平滑，避免 DFT 在边界处产生的 Gibbs 现象（振铃效应）。
4. 快速算法：有成熟的快速 DCT 算法（如 Loeffler 算法），计算复杂度 O(N log N)，可分块（如 8×8）应用。

L07-Q8简答题 简述"率失真理论"的核心思想。R(D) 曲线有什么特点？

显示答案与解析
答案要点
  • 核心思想：在允许一定失真 D 的条件下，对源进行编码所需的最小比特率 R 称为率失真函数 R(D)。它给出了"率"（压缩率）与"失真"（重建误差）之间的理论下界。
  • R(D) 曲线特点：
    • 单调递减：D 越大，可允许的 R 越小；
    • 下凸（凸向下）：斜率递减（每多允许一点失真，节省的比特越来越少）；
    • 当 D=0（无损）时，R(0) ≥ η（熵）；
    • 当 R=0 时，D 达到最大值（无任何信息）。
  • 意义：指导编码器设计——给定目标比特率，能算出理论最低失真；或反之。

L07-Q9综合应用题 设计一个简化的图像有损压缩流程（类似 JPEG 但简化）：
输入：8-bit 灰度图，分块 8×8。
要求：
(1) 画出编码流水线（4-5 个主要步骤）；
(2) 说明每步的目的（无损还是有损）；
(3) 在哪一步信息损失最大？为什么？

显示答案与解析
答案

(1) 流水线：

  1. 分块：把图像分成 8×8 子块（无损）；
  2. DCT：对每个 8×8 块做 2D DCT，得到 8×8 频域系数（无损，IDCT 可逆）；
  3. 量化：用量化表（如保留低频、丢弃高频，或对每个频率除以量化步长并取整）——这是主要信息损失来源；
  4. Zig-Zag 扫描 + 行程编码：把 8×8 系数按 Z 字形排成一维序列，便于出现连续零（无损）；
  5. Huffman 编码：对扫描结果做熵编码（无损）。

(2) 每步目的：

  • 分块：便于局部处理；
  • DCT：能量集中，把空间冗余转换为频率表示；
  • 量化：去除心理视觉不敏感的细节，是压缩率与质量的主要权衡点；
  • 扫描+RLC：把零集中，便于压缩；
  • Huffman：进一步去除统计冗余。

(3) 量化步骤信息损失最大。原因：除以步长取整是不可逆运算，把接近但不相等的系数映射到同一量化级，必然丢失精度。DCT 本身完全可逆，分块无损，Huffman 完全可逆，只有量化真正改变了数据。

L07-Q10简答题 解释向量量化 (VQ) 的基本原理，并比较 VQ 与标量量化的优缺点。

显示答案与解析
答案要点
  • 基本原理：VQ 把输入向量（n 维）作为一个整体，用"码本"（codebook，一组 n 维"码向量"）中最接近的码向量表示。编码器输出码向量的索引（log₂(码本大小) bit），解码器根据索引查表恢复。
  • VQ 优点：
    • 利用分量间相关性（联合量化），理论最优（在等长条件下，VQ 可逼近率失真下界，标量量化不能）；
    • 对非均匀分布更鲁棒。
  • VQ 缺点：
    • 码本设计复杂（LBG 算法训练，需迭代）；
    • 搜索复杂度高（最近邻搜索）；
    • 码本需随训练数据更新，编/解码器必须共享同一码本。
  • 标量量化：简单、易实现，但不能利用分量间相关性。

JPEG 编码 7 步 (简化版)

1. 分块 (8×8)
2. DCT 变换
3. 量化 (量化表)
4. Zig-Zag 扫描
5. 行程编码 (RLC)
6. Huffman / 算术编码
7. 输出码流

Huffman 编码步骤

1. 初始化：所有符号按频次排序成表
2. 重复直至表剩一个：
   - 取频次最小两个节点作为左右子树
   - 新节点频次 = 两子节点之和
   - 新节点插入表中保持有序
3. 对每条边赋 0/1 (左/右)
4. 根到叶的路径 = 码字

算术编码步骤

1. 初始化：low=0, high=1, range=1
2. 对每个符号 s：
   - low2  = low + range · Range_low(s)
   - high2 = low + range · Range_high(s)
   - 更新 low, high, range
3. 遇到终止符 $ 后输出满足 low ≤ code < high 的最短二进制码

LZW 编码步骤 (核心)

1. 初始化字典：A=1, B=2, ..., Z=26 (256 项基础)
2. 读入字符 w；若 w+下一字符 c 在字典中，w ← w+c 继续
3. 若不在字典中：输出 w 的索引，将 w+c 加入字典，w ← c
4. 重复至输入结束；输出 w 的索引
