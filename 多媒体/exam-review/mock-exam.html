I.一、选择题(每题 1 分,共 10 分)

Q1单选 一幅 1920×1080 的 24-bit RGB 图像,转为 YCbCr 4:2:0 后每像素字节数为?

  • A. 3 B/px
  • B. 2 B/px
  • C. 1.5 B/px
  • D. 1 B/px
显示答案与解析
答案 C 解析 4:2:0 每 2×2 块 = Y(4)+Cb(1)+Cr(1) = 6 B / 4 = 1.5 B/px。 涉及 L03/L04/L08

Q2单选 22.05 kHz 声音,按奈奎斯特定理最低采样率应为?

  • A. 22.05 kHz
  • B. 33.075 kHz
  • C. 44.1 kHz
  • D. 88.2 kHz
显示答案与解析
答案 C 解析 f_sample ≥ 2×f_max = 2×22.05 = 44.1 kHz(CD 标准)。 涉及 L05

Q3单选 Huffman 编码满足 η ≤ l̄ < η+1,若某信源概率为 {1/2, 1/4, 1/8, 1/8},其熵为?

  • A. 1.5 bit/symbol
  • B. 1.75 bit/symbol
  • C. 2.0 bit/symbol
  • D. 2.5 bit/symbol
显示答案与解析
答案 B 解析 η = 1/2·1 + 1/4·2 + 1/8·3 + 1/8·3 = 0.5 + 0.5 + 0.375 + 0.375 = 1.75。 涉及 L06

Q4单选 JPEG 标准中,有损压缩的主要来源是?

  • A. RGB→YCbCr 变换
  • B. 8×8 DCT 变换
  • C. 均匀标量量化
  • D. Huffman 编码
显示答案与解析
答案 C 解析 DCT/RGB变换/Huffman 都可逆;只有量化 F̂ = round(F/Q) 不可逆。 涉及 L07/L08

Q5单选 H.261 中亮度宏块大小 N_Y 与 4:2:0 下色度宏块大小 N_C 分别是?

  • A. N_Y=8, N_C=8
  • B. N_Y=16, N_C=16
  • C. N_Y=16, N_C=8
  • D. N_Y=32, N_C=16
显示答案与解析
答案 C 解析 亮度 16×16;4:2:0 色度水平/垂直各 2:1 → 8×8。 涉及 L09/L10

Q6单选 关于 SIFT 检测子使用 DoG 近似 LoG 的原因,正确的是?

  • A. DoG 精度更高
  • B. DoG 计算更简单
  • C. DoG 具有仿射不变性而 LoG 没有
  • D. DoG 在离散像素上更容易实现
显示答案与解析
答案 B 解析 DoG = 两个高斯图像相减,无需二阶微分,计算量大幅降低。 涉及 L11

Q7单选 对 32×32 输入用 10 个 5×5 卷积核、stride=1、padding=2,输出特征图尺寸?

  • A. 32×32×10
  • B. 28×28×10
  • C. 30×30×10
  • D. 32×32×5
显示答案与解析
答案 A 解析 (32 + 2×2 − 5)/1 + 1 = 32;深度 = 卷积核数 = 10。 涉及 L12

Q8单选 已知 qf=25,按 JPEG 公式 scaling_factor = ?

  • A. 0.5
  • B. 1.0
  • C. 2.0
  • D. 3.0
显示答案与解析
答案 C 解析 qf<50 时 scaling = 50/qf = 50/25 = 2.0。 涉及 L08

Q9单选 ResNet 残差网络主要解决的问题是?

  • A. 梯度消失
  • B. 深度退化(训练误差随深度增加而增大)
  • C. 过拟合
  • D. 数据不足
显示答案与解析
答案 B 解析 残差结构让 F(x)=0 成为恒等映射,易学,可训练超深网络;不是解决梯度消失。 涉及 L12

Q10单选 MPEG-1 I/P/B 帧的典型压缩比约为?

  • A. 7:1 / 20:1 / 50:1
  • B. 10:1 / 15:1 / 20:1
  • C. 5:1 / 10:1 / 15:1
  • D. 都是 7:1

答案 A 解析 I 帧类似 JPEG (7:1),P 帧 20:1,B 帧 50:1(双向预测最高)。 涉及 L10

III.三、计算题(每题 6 分,共 30 分)

C1计算题 一幅 256×256 的 8-bit 灰度图像,压缩前后 MSE = 100,求 PSNR (dB)。

显示答案与解析
答案 PSNR ≈ 28.13 dB 解析 x_peak = 255, x²_peak = 65025;PSNR = 10·log10(65025/100) = 10·log10(650.25) ≈ 28.13 dB。 涉及 L07

C2计算题 一段 1920×1080、30fps、4:2:0 的 HD 视频,求:
(1) 单帧数据量;
(2) 1 秒数据量 (MB);
(3) 与原始 RGB (3 B/px) 相比压缩比。

显示答案与解析
答案
(1) 1920×1080×1.5 ≈ 3.11 MB
(2) 3.11 × 30 ≈ 93.3 MB/s
(3) RGB 原始 = 1920×1080×3×30 ≈ 186.6 MB/s;压缩比 = 186.6/93.3 = 2:1 解析 4:2:0 子采样已经带来 2:1 的视觉无损压缩。 涉及 L04

C3计算题 用 Huffman 编码对符号 A=15, B=7, C=6, D=6, E=5(共 39 字符)编码,求:
(1) Huffman 平均码长 l̄;
(2) 理想熵 η;
(3) 与 8-bit 定长编码的压缩比 CR。

显示答案与解析
答案
(1) l̄ = (1×15 + 3×7 + 3×6 + 3×6 + 3×5)/39 = 87/39 ≈ 2.23 bit/symbol
(2) η = −Σ pi·log2(pi) ≈ 2.19 bit/symbol
(3) CR = 8/2.23 ≈ 3.59 解析 Huffman 树构造使 A 用 1 bit,其他用 3 bit。 涉及 L06

C4计算题 对一个 224×224×3 的输入图像做卷积:第一层用 64 个 3×3 卷积核 (stride=1, padding=1);第二层用 128 个 3×3 卷积核 (stride=1, padding=1);第三层用 2×2 MaxPool (stride=2)。求每层输出尺寸和参数量。

显示答案与解析
答案
C1 输出 = (224+2−3)/1+1 = 224 → 224×224×64;参数 = (3×3×3+1)×64 = 1,792
C2 输出 = (224+2−3)/1+1 = 224 → 224×224×128;参数 = (3×3×64+1)×128 = 73,856
P3 输出 = (224−2)/2+1 = 112 → 112×112×128;参数 = 0 解析 池化层无参数;每个卷积核深度 = 输入通道数。 涉及 L12

C5计算题 一段 MPEG-1 视频采用 IBBPBBPBBP 序列(10 帧:1 I、3 P、6 B),I/P/B 压缩比为 7:1/20:1/50:1。求:
(1) 每 10 帧的"加权平均大小";
(2) 平均压缩比。

显示答案与解析
答案
(1) 设原始 1 帧大小 = 1;则 I = 1/7, P = 1/20, B = 1/50;
总大小 = 1/7 + 3/20 + 6/50 = 0.1429 + 0.15 + 0.12 = 0.4129
(2) 平均压缩比 = 10/0.4129 ≈ 24.2:1(与讲义 27:1 量级一致) 解析 帧大小与压缩比成反比;平均压缩比 = 总原始大小 / 总压缩后大小。 涉及 L10

IV.四、简答题(每题 5 分,共 20 分)

S1简答 简述 RGB 不适合直接压缩而 YCbCr 适合压缩的 3 个原因。

显示答案与解析
答题要点
1. 人眼特性:对亮度 (Y) 变化敏感,对色度 (Cb/Cr) 不敏感 → 可对色度粗处理;
2. 解相关:YCbCr 各通道相关性远低于 RGB → 压缩效率高;
3. 可分离压缩:可对 Cb/Cr 下采样 (4:2:0、4:2:2) 节省带宽,视觉影响小;
4. 兼容电视系统:源自模拟电视(PAL 用 YUV、NTSC 用 YIQ),数字时代统一为 YCbCr。 涉及 L03/L04/L08

S2简答 简述 Huffman 编码的构造步骤,并解释为什么 Huffman 码满足"唯一前缀性质"。

显示答案与解析
答题要点
1. 把所有符号按频次从大到小排序成表;
2. 取出频次最小的两个节点作为左右子树,新节点频次 = 两子节点之和;
3. 新节点插入表中保持有序;
4. 重复 2-3 直至表只剩一个节点(根);
5. 对每条边赋 0(左)/ 1(右),从根到叶的路径即码字。
唯一前缀性原因:码字对应二叉树的叶子节点,叶子无子女,因此任何码字都不是其他码字的前缀(前缀性),保证解码时即时匹配无歧义。 涉及 L06

S3简答 简述 JPEG 编码的 5 个主要步骤,并指出哪一步是不可逆的,为什么。

显示答案与解析
答题要点
1. RGB→YCbCr + 4:2:0 子采样(可逆 / 视觉无损):颜色空间解相关;
2. 分块 8×8 + 2D DCT(无损可逆):把空间域变换到频率域,能量集中;
3. 均匀标量量化 F̂(u,v) = round(F(u,v)/Q(u,v))(不可逆,是有损主因):除以步长并取整,丢失精度;
4. ZigZag 扫描 + DPCM(DC)/RLC(AC)(无损):把零聚在一起便于编码;
5. Huffman 熵编码(无损):去除统计冗余。 涉及 L07/L08

S4简答 试比较 AlexNet、VGG、GoogleNet、ResNet 四种典型 CNN 的核心创新与演进动机。

显示答案与解析
答题要点
网络年份层数核心创新演进动机
AlexNet20128ReLU + Dropout + Data Aug + 双 GPU开启深度学习时代(top-5 错误率从 ~26% → 16%)
VGG201416-19多个 3×3 小卷积核替代大卷积核证明"变深"提升性能(top-5 16% → 7.3%)
GoogleNet201422Inception 多尺度模块 + 1×1 降维 + 辅助分类器解决 VGG 参数爆炸(138M → 5M,1/12 AlexNet)
ResNet2015152残差连接 y=F(x)+x 解决深度退化突破深度瓶颈(top-5 6.7% → 3.57%)
演进主线:深度↑ → 参数↓ → 解决退化 → 性能持续提升。 涉及 L12

V.五、综合应用题(每题 10 分,共 20 分)

A1综合应用 给定一个远程视频监控系统的需求:分辨率 640×480、25 fps、低码率 (< 64 kbps)、网络丢包率高、运动缓慢。请设计压缩方案,要求说明:
(1) 色度采样方案及理由;
(2) 帧类型选择及 I 帧间隔;
(3) 运动估计方法选择及理由;
(4) 估算原始数据量与压缩后数据量。

显示答案与解析
答题要点
1. 色度采样:4:2:0(1.5 B/px),适合监控场景(人眼对色度不敏感 + 节省 50% 带宽)。
2. 帧类型:I 帧 + P 帧(不用 B 帧——增加编码延迟且监控不要求双向预测)。I 帧间隔约 1 秒(抗丢包漂移)。
3. 运动估计:分层 (Hierarchical) 搜索——复杂度最低(0.51×10⁹ vs 顺序 29.89×10⁹ OPS),适合缓慢运动场景,节省算力。
4. 数据量估算:
  • 原始:640×480×1.5×25 ≈ 11.5 MB/s ≈ 92 Mbps;
  • 压缩后 64 kbps;
  • 压缩比 ≈ 92 Mbps / 64 kbps ≈ 1440:1(需 H.264/H.265 强压缩);
  • 实际方案:可采用 H.264 Baseline + 分层搜索 + 适度 GOP,2-3 Mbps 即可达监控可用画质。
涉及 L04/L09/L10

A2综合应用 设计一个基于内容的商品图像检索系统(百万级商品库),要求用户上传一张商品照片,系统在 1 秒内返回相似商品。请描述:
(1) 离线建库流程;
(2) 在线检索流程;
(3) 关键技术挑战与解决方案。

显示答案与解析
答题要点
1. 离线建库流程:
  • 商品库图像预处理(去噪、统一尺寸如 256×256);
  • 特征提取:局部特征 (SIFT/SURF) + 描述子(128 维);
  • 特征聚合:Bag-of-Words (BoW) 聚类 (K-means, K=1000) → 视觉词袋;
  • 高维索引:倒排索引(基于视觉词)+ LSH 哈希;
  • 存储特征库与索引。
2. 在线检索流程:
  • 用户上传查询图 → 预处理 → 提取 SIFT 特征;
  • 特征量化:SIFT 描述子映射到最近视觉词(NN 搜索);
  • 查询向量构造:直方图表示;
  • 倒排索引查询 → 候选集;
  • 重排序:RANSAC 几何一致性验证;
  • 返回 top-K 最相似商品。
3. 关键技术挑战:
  • 语义鸿沟:相同商品外观差异大 → 用 CNN 深度特征替代 SIFT(迁移学习);
  • 高维索引:128 维 + 百万库,KNN 太慢 → 用 ANN (近似最近邻) / LSH;
  • 可扩展性:商品更新时增量构建索引;
  • 效率 vs 精度:BoW + 倒排索引牺牲部分精度换速度。
涉及 L11

VII.七、考试时间建议 (90 分钟)

阶段题型建议时长占总时长策略建议
第一阶段选择 + 填空20 分钟22%10 题选 + 5 题填 = 15 题,平均 1.3 分钟/题。选择题先做会的,标记难的后回头;填空题计算要准(数字常考)。
第二阶段计算题30 分钟33%5 题 × 6 分 = 30 分,平均 6 分钟/题。计算题过程分 60%,所以必须写公式 → 代入 → 单位,即使最终算错也能拿过程分。
第三阶段简答题25 分钟28%4 题 × 5 分 = 20 分,平均 6 分钟/题。简答题按要点答,每点 1-2 句解释,避免"只写关键词无解释"。
第四阶段综合应用题15 分钟17%2 题 × 10 分 = 20 分,平均 7.5 分钟/题。先写"总体方案"再分点答关键参数,最后做数据量估算。
合计—90 分钟100%前 80 分钟答完,留 10 分钟检查单位、公式、计算

答题策略心法

  • 先易后难:选择题先扫一遍,把拿不准的标记跳过,做完其他题再回头。
  • 过程分必拿:计算题哪怕最终答案算错,公式 + 代入正确也有 60% 分。
  • 单位常考:b vs B、KB vs KiB、dB 是否带单位,这些易错点占 4-6 分。
  • 关键词保命:简答题"解相关"、"心理视觉冗余"、"双向预测"等术语必写。
  • 预留 10 分钟:检查 PSNR 公式是 10·log10 还是 20·log10,卷积公式 padding 是 2p 还是 p。
