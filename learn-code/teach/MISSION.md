# Mission: 按操作系统章节讲解 xv6 内核代码

## Why
老师要求**按操作系统教材的章节划分**来讲解 xv6 的源码。目标不是泛泛了解操作系统,
而是能对着 `xv6-riscv-20230207/` 里的真实代码,按官方 xv6 教材(rev3)的章节顺序
把每一块代码"是什么、在哪一章、为什么这样写"讲清楚——既能应对课程/考核,也能真正读懂内核。

## Success looks like
- 给定 xv6 的任一源文件(如 `proc.c`、`vm.c`、`trap.c`),能说出它属于教材哪一章、负责哪个子系统。
- 能按章节顺序复述一条完整的执行路径(例如:用户程序调用 `write` → 陷入内核 → 系统调用分发 → 返回)。
- 能向他人(或在考卷上)用本工作区的统一术语解释一个机制,并指到对应代码行。

## Constraints
- **起点:刚入门**。C 能读,但操作系统核心概念(进程、虚拟内存、系统调用、陷阱)基本是新的。
- 每节课必须短、能快速完成——先建立"读得懂"的信心,再逐步加难度。
- 讲解必须锚定**真实存在的本地代码行**,不能停留在抽象描述。

## Out of scope(暂不追)
- RISC-V 汇编的逐指令精读(只在必要时点到为止)。
- 自己动手写新功能 / 做 6.1810 的 lab(目标是"读懂并讲清楚",不是"改内核")。
- 与 xv6 无关的现代 Linux 内核细节。
