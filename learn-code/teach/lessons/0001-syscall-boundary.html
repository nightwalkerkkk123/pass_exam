Lesson 0001 · 第 1 章 操作系统接口

系统调用边界
—— 从 19 行的 echo.c 看懂用户态与内核态

🎯 对应使命:这是"按章节讲解 xv6 代码"的第一站。第 1 章讲的是 操作系统对外提供什么接口——而最小的入口,就是一个用户程序如何"请内核帮忙"。
这节课你将能做到一件事: 拿到任意一个 xv6 用户程序,一眼指出哪几行是 普通计算(程序自己就能做),哪几行是 系统调用(必须求内核做)——并说清楚这条"边界"为什么存在。

1 · 一个事实:程序自己什么都"碰不到"

你写的程序运行时是一个进程(process)。进程被关在一个盒子里:它只能算数、读写 自己的内存。它不能直接往屏幕写字、不能直接读磁盘、不能直接碰任何硬件。 这些能力被操作系统内核(kernel)独占。

为什么这么设计?因为硬件只有一份,而进程有很多个。如果谁都能直接写磁盘,一个有 bug 的程序就能 毁掉所有人的数据。于是内核运行在一个特权级别(内核态),普通程序运行在 受限级别(用户态),中间隔着一道硬件强制的墙。

进程想做"碰硬件"的事,唯一的办法是礼貌地敲门——发起一次系统调用(system call)。 系统调用是内核对外开的、数量有限的几扇门。

📖 依据:xv6 book (rev3) 第 1 章 Operating system interfaces — book-riscv-rev3.pdf。

2 · 看真东西:user/echo.c

这是 xv6 里 echo 命令的全部源码(本地路径 ../xv6-riscv-20230207/user/echo.c)。它做的事极简单:把命令行参数原样打印出来。 我用颜色标出了两类东西——橙色 = 系统调用(穿墙), 蓝色 = 普通计算(在盒子里): 

5int
6main(int argc, char *argv[])
7{
8  int i;
10  for(i = 1; i < argc; i++){
11    write(1, argv[i], strlen(argv[i]));
12    if(i + 1 < argc){
13      write(1, " ", 1);
14    } else {
15      write(1, "\n", 1);
16    }
17  }
18  exit(0);
19}
行代码哪一类?为什么
10for(...)普通计算循环、比较、加法——CPU 在盒子里就能算
11strlen(argv[i])普通计算只是数自己内存里的字符,不碰硬件
11,13,15write(1, ...)系统调用要把字节送到屏幕——硬件,必须求内核
18exit(0)系统调用"销毁这个进程"只有内核能做

关键观察:strlen 和 write 写法长得一样(都是 名字(参数)),但 strlen 只是个普通函数,write 却会陷入内核。 从源码表面看不出区别——区别在于 write 背后会触发一条特殊的 CPU 指令,把控制权交给内核。 那扇"门"具体怎么开,是第 4 章「陷阱与系统调用」的内容,我们到那一章再拆。

🔎 想确认 write 是系统调用?它和其它系统调用一起声明在 ../xv6-riscv-20230207/user/user.h 里,而 strlen 是普通库函数,实现在 user/ulib.c。
