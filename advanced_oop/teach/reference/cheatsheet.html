第 2 章 · 创建和销毁对象 条款 1–9

#条款优先级核心反例 / 一句话
1静态工厂方法代替构造器高5 大优点：有名字 / 可复用 / 可返子类 / 按参数变 / 服务提供者框架
2Builder 处理多参数高≥4 个参数时用 Builder；避免重叠构造器 / JavaBeans 半构造
3私有构造器 / enum 强化 Singleton中单元素 enum 是最佳实现
4私有构造器禁止实例化低工具类：私有构造器 + 抛 AssertionError
5依赖注入中资源（连接、工厂）通过构造器 / setter 注入，不用 static
6避免创建不必要对象高复用 Boolean.TRUE；避免 String s = new String("a")；注意自动装箱
7消除过期对象引用中Stack pop 后清空 elements[--size] = null；内存泄漏三源：过期引用 / 缓存 / 监听器
8避免 finalizer 和 cleaner低时机不确定、性能差、安全漏洞；用 AutoCloseable 替代
9try-with-resources 代替 try-finally低实现 AutoCloseable；更短、更安全、异常不被吞

第 3 章 · 所有对象通用方法 条款 10–14

#条款优先级核心反例 / 一句话
10覆盖 equals 遵守通用约定高五性质：自反 / 对称 / 传递 / 一致 / 非 null；CaseInsensitiveString 破对称
11覆盖 equals 必覆盖 hashCode高equal ⇒ hashCode 相等；反之不要求；Objects.hash 性能差（可变参数装箱）
12始终覆盖 toString中默认 ClassName@hash 难调试；包含所有关注信息
13谨慎覆盖 clone低优先考虑拷贝构造器 / 拷贝工厂；clone 约定脆弱
14考虑实现 Comparable中实现 compareTo 让对象可排序；用 Comparator 静态方法代替复杂实现

第 4 章 · 类和接口 条款 15–22

#条款优先级核心反例 / 一句话
15可访问性最小化高private → 默认 → protected → public；公有类禁止公有可变域
16公有类用访问方法高不要 public double x；用 getX/setX 维持封装
17最小化可变性高5 步：private final + 类 final + 防御性拷贝 + 无 setter + "with" 返新对象
18复合优于继承高跨包继承破封装；用"包装类"持有父类实例（"装饰者"前身）
19继承设计 + 文档中继承必须文档化 @implSpec；构造器禁止调用可重写方法
20接口优于抽象类中接口允许多实现；用 default 给演进留余地
21为后代设计接口低用 default 减少破坏性变更；不强迫实现无关方法
22接口只用于定义类型中常量接口是反模式；用 final class + 私有构造器放常量

第 5 章 · 泛型 条款 23–33

#条款优先级核心反例 / 一句话
23不要在新代码中使用原始类型高用 List<?> 或 List<Object> 不用 List；混用丢类型安全
24消除 unchecked 警告中每条警告代表可能的 ClassCastException；用 @SuppressWarnings 必须加注释
25列表优先于数组中数组协变、具体化；泛型不变、擦除——类型安全差太多
26优先考虑泛型中自己写类型化的容器；不要在 API 里漏类型
27优先考虑泛型方法中静态工具方法泛型化：public static <E> Set<E> union(...)
28优先考虑列表而非泛型数组高new List<E>[] 非法；数组记住元素类型，超出泛型能力
29优先考虑类型安全的异构容器低Map<Class<?>, Object> 模式；过线可不深究
30用 var 引导读者低类型明显时用 var；读者看右边更省力
31用限定通配符增加 API 灵活性低PECS：Producer-Extends / Consumer-Super；过线可放
32谨慎组合泛型与可变参数低@SafeVarargs 仅在确实安全时用
33考虑类型安全的异构 map（见 29）低合并条目；过线忽略

第 6 章 · 枚举和注解 条款 34–41

#条款优先级核心反例 / 一句话
34用 enum 代替 int 常量高int 常量无类型安全；enum 是完整类：可加字段/方法/toString
35用实例域代替序数中不要 ordinal() 当数组下标；自己存数字
36用 EnumSet 代替位域中text.applyStyles(Style.BOLD | Style.ITALIC) → EnumSet.of(...)
37用 EnumMap 代替序数索引低不用 array[enum.ordinal()]，用 EnumMap
38用接口模拟可扩展枚举低enum 不能继承；用接口 + enum 实现
39注解优先于命名模式中不要 setTestMethod() 这种命名；用 @Test 注解
40坚持使用 @Override高编译器替你查重写错误；不写 @Override 错过 N 多 bug
41用标记接口定义类型低标记接口（无方法的）能进入类型系统；标记注解不能

第 7 章 · Lambda 和 Stream 条款 42–48

#条款优先级核心反例 / 一句话
42Lambda 优先于匿名类中函数式接口实现 → 用 Lambda 更短；除非 ≥3 行
43方法引用优先于 Lambda中str -> str.length() → String::length，更清晰
44坚持使用标准函数式接口中优先 Function / Predicate / Supplier / Consumer，不要自己造
45谨慎使用 Stream高无状态 / 短路 / 顺序：滥用会变慢、变难读
46优先选择 Stream 中无副作用的函数中forEach 写副作用 = 走回头路；终端用 collect/reduce
47Stream 优先返回 Collection低返回 List / Set 给调用者更大灵活性
48谨慎并行 Stream中并行只在 ArrayList + 廉价运算 + 多核才有意义；默认别用

第 8 章 · 方法 条款 49–56

#条款优先级核心反例 / 一句话
49检查参数有效性中方法开头 Objects.checkIndex(i, len) 等；早失败
50必要时做保护性拷贝中构造器接收可变参数（Date）→ 防御性拷贝
51谨慎设计方法签名低短名 / 通用 / 不过载太多
52慎用重载高重载按编译时类型选；运行期类型无关；null 调重载会歧义
53慎用可变参数中每次调都分配数组；性能敏感场景先看固定重载
54返回空数组 / 集合而不是 null中返回 Collections.emptyList() 代替 null
55谨慎返回 Optional低结果可能缺失时；不要用作 null 的镜像
56为所有导出的 API 编写文档注释中Javadoc 描述契约；{@code} / @param / @throws 不可少

优先级定义：高 12 条、中 18 条、低 26 条。高+中 = 过线型必背 30 条。
