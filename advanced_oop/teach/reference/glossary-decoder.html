第 2 章 · 创建和销毁对象 条款 1–9

名词考频一句人话（它其实就是）
静态工厂方法static factory高不用 new，改用一个方法把对象递给你。Boolean.valueOf(true) 就是。5 大优点：有名字 / 可复用 / 可返子类 / 按参数返不同类型 / 服务提供者框架。
构建器 · Builderbuilder高参数太多（≥4 个）时，像点奶茶一样 .加料().加料().build()，替代又长又乱的"重叠构造器"和半成品的 JavaBeans 写法。
Singleton 单例singleton中全程序只允许造出"独此一个"的对象。最佳实现是单元素枚举 enum X { INSTANCE }。
依赖注入dependency injection中类需要的"工具"（数据库连接、工厂等）从外面传进来（构造器/setter），而不是自己 new——这样能替换、能测试。
自动装箱autoboxing高int（基本类型）与 Integer（对象）之间偷偷自动转换。循环里误用会悄悄造一堆对象、变慢。
过期引用 · 内存泄漏obsolete reference中该丢的对象还被一个引用攥着，GC 回收不了 → 内存泄漏。经典题：自己写的 Stack，pop 后要 elements[--size]=null。三大来源：过期引用 / 缓存 / 监听器。
祛魅："工厂""构建器""注入"听着像设计模式黑话，其实都在回答同一个朴素问题——"造这个对象的活儿，该谁干、怎么干最不容易出错"。

第 3 章 · 所有对象通用的方法 条款 10–14

名词考频一句人话
equals 五大约定equals contract高自定义"两个对象算不算相等"时必须守的 5 条规矩：自反 / 对称 / 传递 / 一致 / 非 null。高频反例：大小写不敏感的字符串会破坏对称性。
hashCode 契约hashCode contract高只要改了 equals，必须同时改 hashCode，否则放进 HashMap 会找不到。口诀：equal 的对象，hashCode 必相等（反过来不要求）。
toString中让对象能被人读懂地打印出来。不覆盖默认是 类名@十六进制，没法调试。
Comparable · compareTocomparable中给对象装上"可排序"开关：实现 compareTo，就能 Collections.sort。复杂排序用 Comparator 静态方法拼。
重点关注（你的薄弱区）：条款 10 与 11 是一对，必须连着记——改 equals 不改 hashCode = 集合里丢数据。这是你之前标"模糊"的地方，建议优先做一遍代码题验证。

第 4 章 · 类和接口 条款 15–25

名词考频一句人话
可访问性最小化 · 信息隐藏information hiding高能 private 就别 public，把内部藏起来——就是"封装"。顺序：private → 默认 → protected → public。
访问方法 vs 公有域accessor methods高别写 public double x，改成 private 字段 + getX/setX，把修改入口收成唯一一个、好加检查。
最小化可变性 · 不可变类immutability高对象造好就永不改（如 String）。5 步：字段 private final + 类 final + 无 setter + 防御性拷贝 + 改就返回新对象。
复合优先于继承 · 包装类composition over inheritance高想复用别人的类，别去 extends 它（跨包继承会破坏封装），而是把它当字段持有（装饰者模式的前身）。经典题：InstrumentedHashSet。
接口优于抽象类interface vs abstract中优先 interface：一个类能实现多个接口、只能继承一个父类；用 default 方法给演进留余地。
标签类 → 类层次tagged class低一个类里塞个 type 字段 + 一堆 if(type==…) 是反模式，应拆成父类 + 多个子类。

第 5 章 · 泛型 条款 26–33 · 名词最密，最该祛魅

名词考频一句人话
泛型generics高尖括号 List<String>，给容器贴个"只准装 String"的标签，编译期就帮你挡住放错类型。
原始类型raw type高不写尖括号的老写法 List，丢掉类型安全。条款 23：新代码别用，要写就用 List<?>。
泛型擦除type erasure高编译完，<String> 这个尖括号被擦掉；运行时 List<String> 和 List<Integer> 是同一个类。Java 泛型一切"坑"的根源。
协变 vs 不可变covariant / invariant中数组协变（String[] 能当 Object[] 用 → 运行时埋雷）；泛型不可变，List<String> 不能当 List<Object> → 更安全。所以"列表优先于数组"。
有界通配符 · PECSbounded wildcard高? extends T（生产者，只读出）/ ? super T（消费者，只写入）。口诀 PECS = Producer-Extends, Consumer-Super。期末高频简答。
类型安全异构容器heterogeneous container低用 Class<T> 当键，让一个容器每个键各自带类型安全（Favorites 类）。能看懂即可。
祛魅（这一簇的钥匙）："擦除""协变""不可变"是一根链条——正因为运行时类型被擦掉，Java 才不敢让泛型协变（否则埋的雷运行时才炸）。理解这一句，第 5 章一半的对比题自动会做。

第 6 章 · 枚举和注解 条款 34–39

名词考频一句人话
枚举enum中一组固定常量的专用类型，enum Day { MON, TUE }，比用 int 当常量安全得多。
EnumSet / EnumMap低专为枚举优化的集合/字典：EnumSet 代替"位域"，EnumMap 代替"用序数当下标"的数组。
注解annotation中贴在代码上的"标签"，如 @Override，给编译器或框架看。条款 39：注解优于靠方法命名约定（如 testXxx）。

第 7 章 · Lambda 和 Stream 条款 42–47

名词考频一句人话
Lambdalambda高把"一小段函数"当参数传，x -> x + 1，替代啰嗦的匿名类。条款 42：Lambda 优先于匿名类。
函数接口functional interface中只有一个抽象方法的接口（如 Runnable、Comparator）。Lambda 本质就是它的简写。优先用标准库自带的。
方法引用method reference中Lambda 的再简写：x -> x.length() 可写成 String::length。
Streamstream中数据"流水线"：list.stream().filter(…).map(…).collect(…)，链式处理集合。条款 46：流里的函数应无副作用。
