第 2 章 · 对象创建 · 条款 1
关于静态工厂方法，相对公有构造器的优点，下列哪一项不是其优点？
答案：D。静态工厂方法的五大优点是"有名称 / 可复用 / 可返回子类 / 可按参数变化 / 便于服务提供者框架"。运行期性能不在其论点里——它和 new 在 JVM 层是同等开销，区别在 API 设计与可表达性。
第 2 章 · 对象创建 · 条款 2
一个类有 6 个成员变量，其中 2 个必填、4 个可选。最适合的构造方式是？
答案：C。条款 2 明确：参数 ≥ 4 个时 Builder 是首选。重叠构造器可读性崩溃；JavaBeans 模式把对象置于"半构造"不一致状态，破坏不可变性。Builder 既清晰又安全。
第 3 章 · 通用方法 · 条款 10
下面两个 equals 实现违反了哪条通用约定？
// CaseInsensitiveString.java
public boolean equals(Object o) {
    if (o instanceof CaseInsensitiveString)
        return s.equalsIgnoreCase(((CaseInsensitiveString)o).s);
    if (o instanceof String)                 // 关键
        return s.equalsIgnoreCase((String)o);
    return false;
}
答案：B。cis.equals(str) 可能为 true，但反向 str.equals(cis)（String.equals 不知道 CaseInsensitiveString 的存在）必定为 false。对称性破。条款 10 经典反例。
第 3 章 · 通用方法 · 条款 11
关于 equals 与 hashCode 的关系，下列说法正确的是？
答案：A。这是 hashCode 契约的"必要"方向。B 错（hash 碰撞不需要 equals 相等）。D 错（Objects.hash 用可变参数数组，触发数组分配+装箱，性能反而差，条款 11 明确不推荐）。C 错（不重写 equals 时 hashCode 不重写也合法）。
第 4 章 · 类与接口 · 条款 15
关于类和成员的可访问性，条款 15 的核心原则是？
答案：B。条款 15 的核心是"尽可能小"——信息隐藏解耦、独立开发测试、提高可重用。A 错（protected 是 API 的一部分，比 private 难撤回）。C 错（公有类禁止公有可变域）。D 错（测试应通过 API 触达，不该开洞）。
第 4 章 · 类与接口 · 条款 17
设计不可变类时，下列做法不必要？
答案：D。setter 与不可变类根本对立。不可变类应：私有 final 域、类 final、构造器中防御性拷贝可变组件、提供"with"方法返回新对象而非修改自身。setter 是可变类的标志。
第 5 章 · 泛型 · 条款 23
关于 Java 泛型，下列说法正确的是？
答案：B。A 错（类型擦除后是同一个 ArrayList）。C 错（条款 28：泛型数组创建非法，会在编译期产生 unchecked warning，推荐用 List<List<String>> 替代）。D 错（必须用包装类 Integer，且条款 61 提示优先基本类型避免自动装箱）。B 是泛型不变性的关键——协变通过 ? extends 通配符显式表达。
第 7 章 · Lambda 与 Stream · 条款 45
下列哪个是 Stream 的终端操作（terminal operation）？
答案：D。filter / map / sorted 都是中间操作（intermediate）——返回 Stream<T>，可以链式串联。终端操作触发流水线执行并产生非流结果（void / Optional / 集合 / 数值等）。collect、forEach、reduce、count 都是终端。
