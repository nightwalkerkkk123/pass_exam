第 2 章 · 创建和销毁对象

条款 1静态工厂方法代替构造器第 2 章
用 public static T xxx() 代替 new T(...)，给"创建"起一个有意义的名字。
5 大优点：有名字 / 可复用同一实例 / 可返子类 / 可按参数切换实现 / 服务提供者框架。
条款 2Builder 处理多参数第 2 章
参数 ≥ 4 个时，用链式 .setA(...).setB(...).build() 代替一长串构造器或 setter。
重叠构造器不可读、JavaBeans 模式半构造不一致；Builder 既清晰又支持不可变。
条款 5依赖注入（DI）第 2 章
资源（数据库连接、工厂、客户端）通过构造器 / setter 传入类，不在内部 new 出来。
类内部 static 持有资源 → 测试时无法替换、复用困难。
条款 6避免创建不必要对象第 2 章
能复用就复用（Boolean.TRUE、常量池、Pattern.compile 缓存）。
new String("a") 永远多余；自动装箱在循环里悄悄变慢。
条款 7消除过期对象引用第 2 章
类自己管理内存（如栈）时，出栈后手动把引用置 null，让 GC 释放。
内存泄漏三源：过期引用 / 缓存未清 / 监听器未注销。

第 3 章 · 通用方法

条款 10覆盖 equals 遵守通用约定第 3 章
五性质：自反 / 对称 / 传递 / 一致 / 非 null。任一破，整个集合行为就坏。
CaseInsensitiveString 与 String 互相 equals 但反向不等 → 破对称。
条款 11覆盖 equals 必覆盖 hashCode第 3 章
equal ⇒ hashCode 相等；反之不必。HashMap / HashSet 都靠这条。
只覆盖 equals 不覆盖 hashCode → 对象从 HashMap 里找不到自己。
条款 12始终覆盖 toString第 3 章
让对象打印出"信息丰富、易读"的字符串，调试 / 日志都受益。
默认 PhoneNumber@163b91 难调试；好的 toString 应包含关键域。

第 4 章 · 类和接口

条款 15可访问性最小化第 4 章
每个类 / 成员尽可能小：private → 默认 → protected → public。信息隐藏解耦。
公有类的公有可变域是设计错误；protected 是 API 的一部分，撤回代价高。
条款 16公有类用访问方法第 4 章
公有类的字段不能公开，必须 private + 公有 getX/setX。
暴露 public double x → 外部可绕过任何不变量约束。
条款 17最小化可变性第 4 章
不可变类：5 步法——private final 域、类 final、构造器防御性拷贝可变组件、无 setter、提供 "with" 返新对象。
setter + 公开可变字段 = 可变类，不是不可变；String / Integer 是范例。
条款 18复合优于继承第 4 章
跨包继承会破封装（父类修改影响子类）；改用"持有父类实例"的方式扩展功能。
继承 Properties 调 setProperty → 静默插入键值（Hashtable.putAll 的副作用）。
条款 20接口优于抽象类第 4 章
类型定义用 interface；需要共实现时再考虑 default 方法或抽象类。
Java 只支持单继承 → 抽象类被占位后无法再扩展；接口允许多实现。
条款 22接口只用于定义类型第 4 章
接口用来声明"能做什么"，不要拿它当"常量容器"。
常量接口（interface PhysicalConstants { ... }）是反模式。

第 5 章 · 泛型

条款 23不要在新代码中使用原始类型第 5 章
原始类型（List）会绕过泛型类型检查；必须用 List<String> 或 List<?>。
List raw = new ArrayList(); raw.add(1); 编译通过，运行期 ClassCastException。
条款 25列表优先于数组第 5 章
数组协变且具体化，泛型不变且擦除——混用出错。多数情况用 List 更安全。
Object[] 数组不能装 List<String>[]，泛型有界通配符可补。
条款 27优先考虑泛型方法第 5 章
静态工具方法应泛型化：public static <E> Set<E> union(Set<? extends E> a, Set<? extends E> b)。
返回 Set（原始）= 类型不安全且必须加 @SuppressWarnings。
条款 28列表优先于泛型数组第 5 章
数组记着元素类型（具体化），泛型擦除后记不住——new List<E>[10] 编译期就非法。
需要二维结构时用 List<List<String>> 而非 List<String>[]。

第 6 章 · 枚举和注解

条款 34用 enum 代替 int 常量第 6 章
Java enum 是完整类，可以加字段、方法、构造函数；编译期类型安全。
public static final int APPLE_FUJI = 1; 会被传成普通 int，无类型保护。
条款 40坚持使用 @Override第 6 章
覆盖方法时加 @Override，编译器替你查重写错误（签名拼错 / 重载混淆）。
把 equals(Object) 写成 equals(PhoneNumber) → 编译能过但根本没重写。

第 7 章 · Lambda 和 Stream

条款 42Lambda 优先于匿名类第 7 章
函数式接口（单抽象方法）实现时用 Lambda (x) -> ...，比匿名类短得多。
3 行以上的逻辑、抽象类、需要 this/super 引用 → 仍用匿名类。
条款 43方法引用优先于 Lambda第 7 章
已经存在的方法用 Class::method 引用，比 Lambda 更短更清晰。
str -> str.length() → String::length。
条款 44坚持使用标准函数式接口第 7 章
优先 Function / Predicate / Supplier / Consumer / Runnable 等已有接口。
自己造 MyPredicate = 重复造轮子，别人读起来也累。
条款 45谨慎使用 Stream第 7 章
流式 API：中间操作（filter/map/sorted）+ 终端操作（collect/forEach/reduce/count）。
滥用 Stream 会更难读、更慢；简单循环有时更直白。
条款 46优先选择 Stream 中无副作用的函数第 7 章
Stream 操作应只产生结果，不改外部状态。终端用 collect/reduce 收口。
stream.forEach(list::add) 偷偷改外部 = 反模式（条款 46 经典吐槽）。

第 8 章 · 方法

条款 49检查参数有效性第 8 章
方法开头用 Objects.checkIndex(i, len) / requireNonNull 等校验参数；早失败。
不在方法内检查 → 错误延迟到很远的调用栈，难定位。
条款 50必要时做保护性拷贝第 8 章
构造器 / 方法接收可变参数（如 Date、数组）时，先 new Date(param.getTime()) 拷贝再存。
不拷贝 → 外部修改了 Date，你的对象状态就悄悄变了。
条款 52慎用重载第 8 章
重载按编译时类型选方法，跟运行期类型无关。多重载 + 自动装箱 + 可变参数 = 灾难。
List<Integer> list; list.remove(0) 删的是索引 0 而不是值 0（remove(int) 与 remove(Object) 重载）。
条款 54返回空数组 / 集合而不是 null第 8 章
无结果时返回 Collections.emptyList() / new T[0]，不要返回 null。
返回 null → 调用方必须先判空，否则 NPE；API 易用性差。
条款 56为所有导出的 API 编写文档注释第 8 章
public 方法用 Javadoc：/** ... */ 描述契约，@param @return @throws 不可省。
没有 Javadoc → 调用者看源代码猜行为；维护期易改坏。
