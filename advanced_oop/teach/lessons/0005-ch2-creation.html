主题简介

第 2 章是 Effective Java 的开篇——"如何造对象、造多少、什么时候不要造"。这一章 5 条高频条款在生产代码里出现频率极高：

  • 条款 1：static 工厂方法代替 public 构造器——5 大优点（名字 / 复用 / 子类 / 按参数变 / 服务框架）
  • 条款 2：Builder 处理 ≥4 个参数——为什么不要重叠构造器、为什么不要 JavaBeans
  • 条款 5：依赖注入——资源（连接 / 工厂）通过构造器注入，不要用 static 字段持有
  • 条款 6：避免创建不必要对象——Boolean.TRUE 复用、自动装箱在循环里变慢
  • 条款 7：消除过期对象引用——自管内存的类（Stack / 缓存）要主动置 null

本节是 LR-0002 标记的"集中薄弱区"——ch2 全部 5 条你均标"不认识"。
