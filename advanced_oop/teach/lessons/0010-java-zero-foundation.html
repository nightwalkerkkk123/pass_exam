为什么这一节要先讲?

LR-0001/0002 显示你「零词汇」——但零词汇不等于零概念。你脑子里大概率已经有这些画面,只是不知道 Java 里叫啥名字、写在哪个符号上。这节课只做一件事:把 5 个最底层的词钉死。后面所有条款(条款 1 静态工厂、条款 2 Builder、条款 10 equals ……)都是在这 5 个词上长出来的。

本节不背任何 Java 语法,只用生活类比。你只需要能"用自己的话复述",不用记代码。

1. 程序 = 写在纸上的菜谱

想象你去厨房,墙上贴着一张菜谱:

红烧肉菜谱:
  1. 五花肉切块
  2. 加酱油、料酒、糖
  3. 小火炖 1 小时

这张菜谱只是文字——它本身不能让你吃到肉。它需要有人真的去厨房做一遍。

程序(program) 就是这种"写在磁盘上的菜谱":一串给计算机看的文字/符号,本身什么也不做。你双击 QQ.exe,那个 .exe 文件就是程序。

2. 进程 = 真的在做菜

你按菜谱开始炖肉——这一次做菜的过程就是进程(process)。

关键点:

  • 菜谱 1 份(程序 1 份),但你同时炖 2 锅肉 → 这是2 个进程在跑同一个程序。
  • 每个进程有自己的:炖锅(内存)、灶火开关(寄存器状态)、计时器(程序计数器)。
  • 进程是活的、正在发生的;程序是死的、躺在磁盘上的。
口诀:程序 = 菜谱(静态),进程 = 正在做菜(动态)。1 个程序可以跑出 N 个进程(双开 QQ = 1 程序 2 进程)。

3. 类 = 菜谱里的"红烧肉"模板

你现在不只想做红烧肉,还想做糖醋里脊、麻婆豆腐。每道菜都要一张菜谱——每张菜谱就是 1 个"类"(class)。

类就是 Java 里的"模板/图纸",告诉计算机"以后照这个规格造东西":

// "红烧肉"模板(Java 写法,先不要被符号吓到)
class RedBraisedPork {
    String meat;        // 用哪块肉
    int    minutes;     // 炖几分钟
}

类本身不是肉——它只是一张规格单。规格单不顶饱。

4. 对象 = 按规格做出来的那锅肉

你拿着"红烧肉模板"去厨房——真的做出一锅,这一锅就是一个对象(object)。

关键点:

  • 类(模板) → 决定"对象长啥样"
  • 对象(实例) → 真的在内存里占地方的那锅肉
  • 1 个类可以造出 N 个对象——同样一份红烧肉模板,可以做 5 锅给 5 桌客人
口诀:类 = 模具,对象 = 模具压出来的成品。

5. 引用 = "我手上拿着 3 号锅的取餐牌"

你厨房里有 5 锅红烧肉,你自己记不住哪锅是哪锅——你给每锅肉贴个取餐牌。

取餐牌不是肉——它只是写着"3 号锅"的纸条。你撕掉纸条,锅还在;3 号锅倒掉,纸条指向的"肉"也消失了。

在 Java 里,这个取餐牌叫引用(reference):

RedBraisedPork a = new RedBraisedPork();   // 做了 1 锅,拿到 1 张取餐牌 a
RedBraisedPork b = a;                       // 把 a 那张牌复印一张给 b
// 现在 a 和 b 都指向"同一锅肉"
a.meat = "五花肉";  // 用 a 改,肉变了
System.out.println(b.meat);  // 印出 "五花肉"——因为是同一锅

关键点:

  • 引用是"地址条",对象是"那锅肉"。
  • 两个引用可以指向同一个对象(同一个取餐牌,两人都能取)。
  • 没有引用指向的对象,Java 会在某个时间点自动回收(锅没人取了,后厨清理掉)。
口诀:引用 = 地址条;对象 = 地址条指向的"那锅肉"。地址条不是肉,撕掉地址条,肉可能被后厨清掉。

6. 把 5 个词串成一张图

磁盘上的 RedBraisedPork.class    →  类(模板)
        ↓ Java 读到内存
new RedBraisedPork()              →  在堆里造出 1 个对象
        ↓ 同时给你
RedBraisedPork a = ...            →  a 是引用(地址条),指向那个对象
        ↓ 双击图标
操作系统创建 1 个进程             →  这个进程里跑着这段代码

8. 4 道选择题(不打分,只为再过一遍)

概念 1 · 程序 vs 进程
桌上有 1 张红烧肉菜谱,你同时炖了 3 锅。请问:几个程序,几个进程?
答案:B。 1 张菜谱 = 1 个程序(静态);3 锅在炖 = 3 个进程(动态)。OpenOffice 双开文档,1 个程序文件,2 个进程,道理一样。
概念 2 · 类 vs 对象
关于类和对象,哪个说法不对?
答案:C。 类(模板)必须先写出来,才能照着做对象。C 反了。A/B/D 都对(对象在堆里占空间,类的元信息在方法区,常驻)。
概念 3 · 引用
关于引用,哪个说法最准?
答案:A。 B 错(不是同一个东西);C 错(可以多个引用指向同一对象);D 错(引用一多,只是多了"取餐牌",肉还是那锅肉)。
概念 4 · 综合
运行下面代码后,印出什么?
RedBraisedPork a = new RedBraisedPork();
a.meat = "五花肉";
RedBraisedPork b = a;
a = null;                      // 把 a 这张牌扔了
System.out.println(b.meat);    // 用 b 这张牌看
答案:D。 a 和 b 之前都指向同一锅肉;a 被撕掉(b = a 之后 b 自己也是一张牌),b 那张牌还在,肉没被清掉。这是"引用 vs 对象"分得清不清的经典题——A/C 是把"扔掉 a"当成"扔掉对象"的常见错。
