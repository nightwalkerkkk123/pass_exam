# Working Notes

## 用户偏好
- **目标档位**：~~应试 + 真会用~~ → **过线型**（v2 起，参见 MISSION.md）
- **时间档位**：1 周内（紧急）——单 lesson 控制在 5–10 分钟
- **基础水平**：诊断后确认**零基础**——30 条核心条款仅 4 条"认识"，且 10/11 出现模式匹配风险
- **语言**：中文（lesson 文本与对话默认中文，代码标识符与英文术语保留英文）
- **不允许的捷径**：不假装"已会"来跳过 lesson；不联网杜撰资料

## 教学策略调整（v2 后）
- 主题聚类优先于按条款顺序——零基础下"4 条同章"比"4 条不同章"更高效
- 每个 cluster 后必须有"代码反例识别"题验证真假理解，不能停留在自评按钮
- 警惕"认名不认意"——单选项按钮证据太弱，必须用代码题交叉验证

## 工作约定
- lesson 编号严格递增；不重写已发 lesson，仅通过后续 learning-record 修正理解
- 浏览器可打开是硬性要求——每写完一个 lesson HTML 后必须自检一遍浏览器渲染
- 引用条款一律用"条款 N"格式（与 `Effective_Java_期末复习资料.md` 对齐），方便 user 双向定位
- 诊断课的结果会作为第一条 learning-record 之外的 ZPD 锚点，**不写到 learning-records**（因为它是观察而非学习），而是作为 lesson 2 起的选课依据
