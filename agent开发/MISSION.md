# Mission: Agent 开发（面向服务端工程师的 AI 工程转型）

## Why
用户是后端工程师，正在向 AI 工程方向转型，且手上有生产/准生产的 Agent 项目可以落地所学。目标不是"了解 Agent"，而是获得**可迁移到任何 Agent 系统的生产级工程判断力**——能设计、评估并长期运营一个 Agent 服务。

## Success looks like
- 能为一个真实 Agent 系统设计 eval 集与自动化评测流水线，并说清"裁判可信度"如何保证
- 能画出一个 Agent 的 context 组装策略（带什么、淘汰什么、放哪里），并解释每个决策对质量/成本/缓存的影响
- 能按"不可靠调用方"的假设设计工具边界：幂等、超时、重试、错误信息的可读性
- 能对生产账单做归因：token 花在哪、prompt cache 命中率为何、模型路由怎么设
- 面试或方案评审中，能用服务端工程语言向后端团队解释以上全部决策

## Constraints
- 学习者已有扎实的服务端工程背景（session/缓存/RPC/CI），课程应**借力类比、同时明确标出类比失效点**
- 四大主题（评估 / context 管理 / 工具调用 / 成本与延迟）全部覆盖，以 context 管理为主轴展开
- 有真实项目环境，课程偏实操：每课应产出能直接用在项目上的判断或代码模式
- 中文授课，技术术语保留英文

## Out of scope
- 模型训练、微调、RLHF——只做推理侧工程
- 具体某个 Agent 框架的 API 细节（LangChain/LlamaIndex 等）——学的是框架无关的工程原理
- Prompt 写作技巧本身——只在影响系统行为时讨论
