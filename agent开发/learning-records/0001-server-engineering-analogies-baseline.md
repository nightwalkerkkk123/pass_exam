# 已有知识基线：用服务端工程类比理解 Agent 四支柱

学习者（后端工程师）在开课前自发给出四段类比：eval≈新的CI、context管理≈状态管理+缓存淘汰、工具调用≈RPC边界设计、token≈新的计费单位，并总结"Agent 里大量问题本质是服务端工程问题"。类比方向全部正确，说明服务端心智模型可以直接借力，无需从零建立框架。

**Evidence**: 开课陈述为学习者原创，四个类比均能对应到正确的工程关切（回归监控、状态淘汰、幂等重试、成本路由）。

**Implications**:
- 教学起点不是"什么是 Agent"，而是每个类比的**失效点**：eval 的统计性与裁判可信度、context 的位置敏感性（lost in the middle）、工具描述作为面向模型的 API 文档、prompt cache 的前缀匹配语义。
- 学习者尚未意识到四支柱经由 context 组织相互耦合（质量/成本/评估可复现性），这是第一课的核心目标。
