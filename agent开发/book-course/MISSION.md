# Mission: 精读《深入理解 AI Agent》，把 Agent 设计从「感觉驱动」变成「原则驱动」

## Why
学习者是正在向 AI 工程转型的后端工程师，手上有生产/准生产的 Agent 项目。这门课的目标不是"读完一本书"，而是把李博杰《深入理解 AI Agent：设计原理与工程实践》里那套**能穿越模型迭代周期的架构判断力**，内化成自己的默认思维——以后拿到任何 Agent 系统、排查任何 Agent 故障、评审任何 Agent 方案，都能一眼看出它在核心公式 `Agent = LLM + 上下文 + 工具` 的哪一格出了问题。

## Success looks like
- 能把任意一个 Agent 产品拆进核心公式的三格（大脑/眼睛/手脚），并说清每一格的关键设计决策与取舍
- 能解释上下文工程的地基：消息结构、KV Cache 前缀不变性、Skills 渐进披露、压缩 vs context rot
- 能按"感知/执行/协作/事件触发/用户沟通"五类给工具归类，并说清执行工具为什么要多层安全、Coding Agent 为什么是"元能力"
- 能为一个 Agent 建起可复现、防泄漏的评估体系，分清"模型替换实验"与"消融实验"，读懂 Pass@k vs Pass^k
- 能讲清后训练两条主线（SFT 记忆、RL 泛化；数据与环境比算法更重要），知道何时该 SFT、何时才轮到 RL
- 能判断"多 Agent 何时真的优于单 Agent"——用"是否引入生成时不存在的新信息"这一条判据
- **动手**：至少把书里 3–5 个配套实验（开源仓库 `chapterN/`）亲手跑一遍，把直觉从"读懂"升级到"做过"

## Constraints
- 以**书**为唯一真实源（`原始资料/ai-agent-book/book/*.md` + 开源仓库配套代码）；不靠 parametric 记忆补充或"纠正"原文
- 中文授课，技术术语保留英文（LLM、context、RAG、KV Cache、SFT、RL、MCP 等）
- 术语纪律：严格区分「思考(reasoning)」与「推理(inference)」，按书的约定用词
- 每课一个窄主题、图文并茂、可独立打开阅读、带自测题；不把内容一次性讲满
- 与本工作区另两门课（`from-zero/` 入门、`lessons/` 四支柱）**互补而非重复**：那两门落到用户自己的 `mino_server` 代码，这门落到书里的原则与开源实验

## Out of scope
- 追逐某个具体框架的 API 细节（LangChain / LlamaIndex 等）——学的是框架无关的架构原则
- 把书当 Demo 教程"跑通即止"——要的是"为什么这样设计"，不是"照抄能跑"
- 第 7 章后训练里的重数学推导——书本身也只做直觉级解释，本课同此边界
