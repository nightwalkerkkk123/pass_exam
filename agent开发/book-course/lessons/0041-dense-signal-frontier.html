被浪费的反馈：稀疏奖励的困境，与两条把它救回来的前沿路

这节课只押一件事：当前 RL 样本效率低，根子在于它只从最终的成败结果里学，却把环境每一步给出的丰富反馈全浪费了。把这条困境讲透，你就能看懂本章第 7.10 与 7.12 节两条前沿思路——RLVP（验证路径惩罚）和 On-Policy Distillation（在轨蒸馏）——为什么它们本质上是同一件事：把被纯结果奖励扔掉的信息，重新变回模型能学的信号。这两条路，正是全章收尾时点名"最有希望的两个方向"。

一、是什么：稀疏奖励的困境

先看一个书里反复用到的场景——Pine AI 打电话帮用户改套餐。用二元奖励（成功 = 1，失败 = 0）训练：第一次忘收集账号，reward = 0；第二次忘信用卡后四位，reward = 0；第三次遗漏账单地址，reward = 0……经过 100 次尝试才偶然成功一次。

问题的根源，书里引用 Silver 与 Sutton《Welcome to the Era of Experience》说得很直白：当前 RL 方法只能从最终的成败结果中学习，却无法从环境给出的丰富反馈中学习。客服明明说了"需要信用卡后四位"，人类听一次就记住了；但 RL 只看到最终结果"失败"，根本不知道为什么失败。更糟的是——10 步流程里，即使前 9 步完美、只有第 9 步出错，它拿到的信号也只是"整个任务失败了"，无从得知到底哪一步坏了事。

二、为什么是"浪费"：model-free 只认最终标量

书里把根子说得更技术一点：RL 样本效率低有多重原因（高方差、稀疏奖励、在轨数据难以复用），其中一个重要根源在于主流策略梯度方法的 model-free（无模型）特性——它不建模环境动态，也难以直接利用单次反馈里的丰富信息。于是环境每次交互返回的错误原因、缺失字段、正确流程提示，大部分都被浪费了。那个打电话的例子里，客服明确告知"需要信用卡后四位来验证身份"，但 model-free RL 只能从 0/1 的最终信号学习，无法直接用上这句话，只能靠数百次随机探索偶然撞对。

作者把两条破局思路并排摆出来：一条是把环境反馈里被浪费的信息重新变成可学习的奖励（写进奖励函数）——这是 RLVP；另一条是让每一步的训练信号更密集（不只在终点拿一个标量）——这是 On-Policy Distillation。下面分别拆开。

三、破局一：RLVP —— 用"组内方差"解释惩罚为何永远有效

RLVP 的配方一句话：奖励结果，惩罚路径（reward the outcome, penalize the path）。总奖励写成两部分：

它为什么有效？书里给了一个统一解释——组内方差（within-group variance）。回忆 图7-13 的 GRPO：它不训价值网络，而是对同一个 prompt 采一组 rollout，用每条"相对组内平均"的好坏当优势。这里有个数学事实：GRPO 的优势本质就是组内方差——如果一组里每条 rollout 拿到的奖励完全一样，方差为零、优势全为零，这组样本贡献不出任何梯度、白跑了。

只用结果奖励时，零方差死局恰好在训练一头一尾必然发生：

  • 全败组（训练早期）：任务太难，一组 rollout 全失败，O 全是 0 → 方差为零 → 没梯度。训练早期几乎全是这种组，大量昂贵采样被白白浪费。
  • 全胜组（训练后期）：任务快学会了，一组 rollout 全成功，O 全是 1 → 方差同样为零 → 没梯度。

也就是说，纯结果奖励在成功率的两个极端都是"瞎的"。社区以前的做法是把这些零方差的组直接丢掉（DAPO 的 dynamic sampling）。RLVP 换了个问法：与其丢掉，不如问——什么样的密集信号能在这里补回缺失的方差？答案分成"永远可达"和"有条件可达"两半：

  • 可验证的惩罚，永远能补回方差。 哪怕一组 rollout 全失败，它们"失败得规不规矩"通常各不相同——有的跑了破坏性命令、有的没有。惩罚一加，全败组内部立刻有了差异，梯度就活了。因为坏动作总是便宜可查，惩罚是"普遍可用的那一半"。
  • 可验证的进展奖励（Partial Credit），只在"进展可达"时补回方差。 一组里有的多通过两个测试、有的多证出一个引理，就有了进展差异；但若每条 rollout 的进展都卡在零（软件修复里没人能让任何隐藏测试通过），进展信号处处为零，还是零方差——它是"可达性门控的那一半"。
密集信号只在它能补回结果奖励所缺的组内方差时才有用——惩罚永远满足，进展奖励只在部分成功可达时满足。 ——《深入理解 AI Agent》第 7 章 · 验证路径惩罚（RLVP）

实验 7-14 给了硬数据：TerminalBench 上每局违规次数从纯结果奖励的 3.71 降到 0.66（约 6 倍），而成功率基本持平——说明"守规"几乎是免费拿到的；miniF2F 代数题上达到 0.9 成功率所需迭代数从 7.0 降到 4.4（4B 模型，30B 上更明显）；链式文件操作里"全败组"比例从 65% 降到 8%。而 RLVP 与本章反复出现的 RLVR 只差一个字母，恰好互补：RLVR 验证结果，RLVP 额外验证过程。

四、破局二：On-Policy Distillation —— "在轨 + 稠密"两个短板一起补

另一条路不动奖励函数，而是让每一步的信号本身更密集。要理解它，先看 SFT 和 RL 各自的一个致命短板：

  • SFT 的短板 · Learner-Sampler Mismatch：数据由"采样者"（教师/人类）生成，"学习者"只被动模仿正确路径；自己上场一旦走到训练数据里没出现过的偏差状态，就不知道怎么回正轨，小错累积成大错。根源是"训练时谁在走"和"部署时谁在走"不是同一个分布。
  • RL 的短板 · 信号太稀疏：让学生自己走（在轨）解决了分布不匹配，但每条轨迹走到头只拿一个成败标量，中间每步该怎么改还得靠成百上千次试错慢慢反推。

On-Policy Distillation 把两者的优点合起来：让学生自己生成轨迹（On-Policy，解决分布不匹配），同时让一个更强的教师模型对学生走的每一步逐 token 打分（Dense Signal，解决信号稀疏）。 教师不只判断这一步对不对，而是直接给出"当前位置下一个 token 各种选择该占多大概率"的完整分布，学生最小化自己分布与教师分布的 KL 散度。相比只有最终成败的二元信号，这种逐 token 的分布对齐，密集了不止一个数量级。

效果书里给得很实：在数学等任务上，达到同等性能所需的训练步数只要纯 RL 的约 1/10；它还顺带缓解过拟合（每次轨迹都不同、教师针对具体轨迹给反馈，学到的是通用策略而非背答案），数据复用率大幅提升。在多轮 Agent 场景里价值尤其大——成败信号出现在最末端、既稀疏又滞后，逐 token 的教师分布恰好补上中间每一步缺失的指引。作者与合作者在 8B 小模型的"时间感"后训练里做过一次干净验证：DPO 与四种 RL 配方轮番失败（各踩中一个前面讲过的失败模式），换成 On-Policy Distillation（冻结的 Qwen3-32B 教师逐 token 给分布）训练平滑收敛，四种条件下通过率一律比同源 SFT 基线高出 23 到 47 个百分点。

误区一：把"必须先 SFT 后 RL"当普适规律。 它只在"较小基模 + 严格结构化输出"下成立——V-IRL 那类多轮任务里不经 SFT 初始化，基础模型连结构化 JSON 都产不出来，RL 无从训起；但强基模（如 R1-Zero）能直接 RL。别把一条工程经验背成铁律。

误区二（本课最关键）：以为 On-Policy Distillation 的"蒸馏"就是普通蒸馏。 普通（离轨）蒸馏是教师先生成一批轨迹、学生照着模仿；On-Policy Distillation 的关键区别是轨迹由学生自己采样（在轨），教师只在学生真实走出的每一步上给逐 token 目标分布。丢掉"在轨"这两个字，你就把它降级回了 SFT，也就说不清它凭什么能同时补上两个短板。它还有个前提：必须有一个足够真实的仿真环境让学生自由探索——否则学生走到教师也没见过的偏差状态，教师打分同样不可靠。

本节两条路都能在开源仓库里动手跑。RLVP 对应 实验 7-14（chapter7/retool 演示的是相邻的工具 RL 思路：SFT 预热 + 基于 veRL 改造 PPO，把代码解释器沙盒整合进思考循环、对 <interpreter> 反馈 token 做损失屏蔽，AIME2024 从 25% 提升到 67%）。要亲手体会"组内方差救回全败组"，也可以看 实验 7-13 的 chapter7/SimpleVLA-RL——它用动态采样过滤全成功/全失败组来确保稳定梯度，正是同一枚硬币的另一面。完整仓库：github.com/bojieli/ai-agent-book。

五、关键定义

验证路径惩罚（RLVP, RL with Verified Penalty）
在结果奖励 O 之外加一路确定性规则引擎给出的路径信号 Φ（R = O + β·Φ）：违规动作扣分、合规/可达进展动作加分——"奖励结果、惩罚路径"。避免与 RLVR 混淆：RLVR 验证结果，RLVP 额外验证过程，两者互补叠加。
可验证奖励强化学习（RLVR, RL with Verifiable Rewards）
奖励来自规则验证器（测试是否通过、答案是否正确）而非学习到的奖励模型的 RL；此时"奖励模型"退化成一段确定性代码。避免与 RLHF（奖励来自人类偏好训练的 RM）混淆，也别和只差一字母的 RLVP 混。
信用分配（credit assignment）
多步序列中判断到底哪一步对最终成败贡献最大的问题；由折扣因子、GAE（PPO）、组内相对比较（GRPO）、turn-level 分摊等机制承担。避免与"奖励设计"画等号——它是奖励延迟下的归因难题。
在轨蒸馏（On-Policy Distillation）
让学生自己生成轨迹（在轨），更强教师对每一步逐 token 给出目标概率分布（稠密），学生最小化与教师分布的 KL 的方法；是"在轨 + 稠密信号"的组合。避免与普通（离轨）蒸馏或 CoT 蒸馏混淆——关键区别是轨迹由学生而非教师采样。

六、自测题

Q1：GRPO 里为什么"全败组"和"全胜组"都学不到东西？RLVP 的可验证惩罚为什么能救回全败组？

GRPO 的优势本质是组内方差——一组 rollout 奖励完全相同时方差为零、优势全为零、贡献不出梯度。全败组（O 全 0）和全胜组（O 全 1）都是零方差死局。RLVP 加一路可验证惩罚：哪怕一组全失败，它们"失败得规不规矩"通常各不相同（有的跑了破坏性命令、有的没有），惩罚一加组内立刻有了差异，梯度就活了——所以惩罚是"永远可达"的那半个解。

Q2：把三种后训练方法放到"在轨/离轨 × 稠密/稀疏"的坐标里，SFT、RL、On-Policy Distillation 各落在哪一格？为什么说 On-Policy Distillation 同时补上了另两者的短板？

SFT = 离轨 + 稠密（有分布不匹配，即 Learner-Sampler Mismatch）；RL = 在轨 + 稀疏（信号太稀疏，中间每步靠试错反推）；On-Policy Distillation = 在轨 + 稠密。它让学生自己采样轨迹（在轨，解决 SFT 的分布不匹配），同时让更强教师对每一步逐 token 给出目标分布（稠密，解决 RL 的信号稀疏），所以两个短板一起补上。代价是必须有足够真实的仿真环境，否则学生走到偏差状态时教师打分不可靠。

本课唯一要带走的判断：RL 卡住的根子往往不是奖励函数设计得不够巧，而是信号本身不够密——纯结果奖励把环境里"要先验证身份""这条命令有破坏性""又证出一步"这类明确、可机器判定的反馈全浪费了。RLVP（把被浪费的信息写进奖励，用组内方差解释惩罚为何永远可达）与 On-Policy Distillation（让每一步都拿到教师的逐 token 分布）是两条互补的回收路，也是全章点名"最有希望的两个方向"。
