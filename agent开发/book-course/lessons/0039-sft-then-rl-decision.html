先形后神：什么时候做 SFT，什么时候才轮到 RL

上一课把「SFT 记忆、RL 泛化」的原理讲透了；这一课回答那个真正落地时会咬到你的问题——面对一个具体任务，到底该用哪一个？本课唯一要拿下的东西，是一套可以照着走的决策顺序：先问要不要后训练，再先 SFT，SFT 顶不住才加又贵又不稳的 RL。它对应第 7 章「何时选择 SFT，何时选择 RL」一节，是全章两条主线（SFT 记忆 / RL 泛化、数据环境比算法重要）落到工程动作上的收口。

一、先把三阶段的顺序摆正：为什么 SFT 一定在 RL 之前

作者在开篇给了一张地图：一个现代模型的能力是分三阶段炼成的——预训练（读万卷书，积累知识）、SFT（老师手把手教标准解法，模仿示范）、RL（自己下场做题、根据对错反复打磨，试错提升）。三者不是三选一，而是一条流水线：先读书，再看示范，最后实战。

那为什么 SFT 必须排在 RL 之前？答案藏在 RL 的工作方式里。RL 不看标准答案，而是让模型自己生成回答、再根据好坏给奖惩。可要判断好坏，首先得能把模型的输出解析出来——如果任务要求吐一段 JSON 或一次工具调用，而模型给的是一团格式混乱的文本，奖励函数连「成功还是失败」都判断不了，RL 也就无从学起。

所以 SFT 在这里扮演「先把话说利索」的角色：用少量示范让输出格式稳定、能被可靠解析，RL 才有一个能打分的起点。作者借中国画的说法把它凝成四个字——先形后神：SFT 先把「形」（格式、结构）立起来，RL 再追「神」（策略、泛化）。反过来先 RL 后 SFT 行不通：没有稳定的输出，奖励信号就是一片噪声。

RL 能自己长出"神"（策略与推理能力），但"形"（格式与可读性）还是靠 SFT 立得又快又稳。 ——《深入理解 AI Agent》第 7 章

二、决策顺序：三步走，别一上来就 RL

这是本课的骨头。作者把实操决策明确拆成三步，一步一步往下沉，能在上一步解决就绝不进下一步——因为越往下越贵、越不稳。

逐格说清每一步为什么在这里：

  • ① 需要后训练吗？ 如果靠 Harness 工程（优化 prompt、工具设计、上下文管理）就能解决，就不要训练模型。作者一句话点破：「大多数 Agent 应用落在这里。」训练是重武器，能不动权重就别动。
  • ② 先试 SFT。 一旦确定要训练，先上 SFT——它专治「固化格式/协议/风格」：JSON schema、API 调用格式、术语用法、流程习惯、语气长度。它成本低、见效快，通常几千到几万条高质量样例就见效。但注意边界：SFT 不适合注入大量事实性知识（「知道什么」），那要交给继续预训练或 RAG。
  • ③④ SFT 不够，才加 RL。 RL 适用于需要泛化到新场景、探索最优策略（专家示范本身不一定最优）、或标注成本过高、无法为每条路径都给示范的情况。它能学到可迁移策略，代价是样本效率低（常是 SFT 的几十到上百倍）、训练不稳、易震荡。

三、临界信号：什么时候「再加示范也没用」

三步里最难拿捏的是第 ③ 格——到底什么时候判定「SFT 到头了」？作者给了一个可操作的临界点判据：

当「无论怎么增加示范数据，新场景的表现仍然上不去」时，就是该转向 RL 的临界点。关键在于：问题的根源不在示范数量，而在 SFT 的优化目标本身。

为什么根源不是数据量？因为 SFT 的优化目标是最大化标注答案的概率（极大似然）——它只会把示范映射记得更牢，天花板就是示范者的水平，一份 60 分老师标注的数据训不出 90 分的学生。你再堆一万条同分布的示范，也只是把同一个记忆钉得更死，对分布外（OOD）场景毫无帮助。要泛化，必须换优化目标——从「像不像标准答案」换成「结果好不好」，这正是 RL 干的事。所以这不是「数据不够」的问题，是「方法用错」的问题。

SFT 适用于格式固化（JSON 输出、对话风格）、拥有高质量专家示范、训练与部署环境高度一致的场景；RL 必须介入的，恰恰是部署环境与训练环境存在系统性差异时（比如训练时卡牌 J/Q/K 都算 10，部署时变成 11/12/13——规则变了）。把这两句对照着记，第 ③ 格就有了判断依据。

四、边界与易错：「必须先 SFT 后 RL」不是普适律

初学者读完「先形后神」，最容易把「必须先 SFT 后 RL」当成放之四海皆准的铁律——这是错的。作者反复强调：这个结论有边界条件，它只在「较小基础模型 + 严格结构化输出要求」的设定下成立（实验 7-11 里，Llama-3.2-Vision-11B 这个量级不经 SFT 直接 RL 会完全失败，因为基础模型产不出结构化输出、奖励根本没法算）。

但如果基础模型足够强，它可能一上来就能产出够格的输出，从而跳过 SFT 直接 RL。DeepSeek-R1-Zero 就是证据：强基模直接 RL 成功，自行涌现出反思与长链思考。代价是输出可读性差、中英文混杂，所以 DeepSeek 最终还是在 R1 里加回了「冷启动 SFT」，把「形」重新立稳。R1 从 Zero 到冷启动的这段往返，正是「先形后神」最好的注脚——但也说明：顺序是配方，不是定律，随基模强弱而变。

五、宏观背景：这套顺序为什么值钱

因为它把一个昂贵的决策前置成一串便宜的追问。RL 是全章篇幅最大、最需要工程功力、也最烧钱最不稳的一步；工业界最反直觉也最值钱的一条经验是——很多场景下，只要 SFT 的数据质量到位，你根本不需要做 RL。这套三步顺序的价值，就是逼你在掏出重武器之前，先老老实实问两遍「更便宜的办法试过了吗」。它和全章主线二（数据和环境比算法更重要）是同一种智慧：把注意力从「调哪个算法」拉回到「这一步到底该不该做、数据够不够好」。下一课会正面展开这条判断。

六、关键定义

监督微调（SFT, Supervised Fine-Tuning）
用几千到几万条「问题—标准回答」示范对继续做「预测下一个词」、且只在回答 token 上算损失的后训练方法，把指令遵循、格式、风格、流程等协议性知识固化进参数。避免泛称为「微调」或等同于「训练」；它特指有标注输入-输出对、带 loss masking 的方法。
损失屏蔽（loss masking）
计算 SFT 损失时把不希望模型学习的 token（问题部分）屏蔽掉，只对模型自己生成的回答 token 回传梯度——这是 SFT 在工程上与预训练唯一实质性的区别。避免与注意力掩码（attention mask）混为一谈，两者作用位置不同。
先形后神
借中国画术语：SFT 先把「形」（格式、结构）立准，RL 再追「神」（策略、泛化）。对应最稳健的「先 SFT 后 RL」两阶段配方——但只在较小基模+严格结构化输出下是必需顺序。
寻峰与覆盖式（mode-seeking vs mass-covering）
两种分布拟合性格：极大似然（SFT）是 mass-covering，努力覆盖示范里所有模式、雨露均沾；带 KL 约束的策略优化（RL）是 mode-seeking，只集中到少数高奖励峰、赢者通吃。避免把 mode-seeking 简单等同于「过拟合」。

七、自测题

Q1：什么信号出现时，说明「再加示范数据也没用、该转向 RL 了」？为什么根源不在数据量？

当「无论怎么增加示范数据，新场景表现仍上不去」时。因为根源不在示范数量，而在 SFT 的优化目标本身——它只会更牢地记忆示范映射，天花板是示范者水平，无法泛化到分布外场景；泛化要靠 RL 换优化目标（从「像不像标准答案」换成「结果好不好」）。

Q2：「任何任务都必须先 SFT 再 RL」这句话对吗？在什么条件下它成立，什么反例推翻了它的普适性？

不完全对，它不是普适规律。它只在「较小基础模型 + 严格结构化输出要求」的设定下成立（如实验 7-11 中 Llama-3.2-Vision-11B 不经 SFT 直接 RL 会完全失败，因为奖励无法计算）。反例是 DeepSeek-R1-Zero：足够强的基础模型可以跳过 SFT、直接 RL 成功，自行涌现反思与长链思考——代价是可读性差、中英文混杂，所以 R1 后来又加回了「冷启动 SFT」把「形」立稳。

想亲手把「预训练 → SFT → RL」这条流水线完整走一遍？看配套项目 chapter7/MiniMind-pretrain/（实验 7-3）：在消费级 GPU 上从头训一个约 100M 的 MiniMind 2，走完 tokenizer → 预训练 → SFT → 偏好优化（DPO）全流程，约 14 小时、34 美元。你会亲眼看到各阶段的分工——预训练后能答事实但格式乱，SFT 后指令遵循与格式显著改善，偏好优化再减少事实错误与不自然表达。开源仓库：github.com/bojieli/ai-agent-book。
