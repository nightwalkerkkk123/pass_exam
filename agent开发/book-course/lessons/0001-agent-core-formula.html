Agent 核心公式：大脑 + 眼睛 + 手脚

整本书只押一句话：Agent = LLM + 上下文 + 工具。这节课的唯一目标，是让你把这句公式刻进肌肉记忆，并且立刻知道它对应的身体隐喻——大脑、眼睛、手脚——好在哪、又在哪里会漏水。后面九十多页、十章内容，全都是这三个词的展开。先把地基夯实。

一、公式：三个词，缺一不可

作者在引言里把话说得很满：“本书的核心公式只有一句话。”不是“大概由这几部分组成”，而是一个等式——少任何一项，它就不再是一个 Agent。

Agent = LLM + 上下文 + 工具
图0-1 · 三者缺一不可

为什么“缺一不可”不是修辞？把任意一项抽掉，剩下的东西都还有名字，但都不是 Agent：

  • 只有 LLM，没有上下文、没有工具 → 那是一个聊天机器人：能说话，但看不到世界、也动不了手。
  • 有 LLM + 工具，但不把工具执行的结果喂回上下文 → 它会盲目地一遍遍做同一件事，因为它“看不到”自己刚才做了什么（这一点第 0007 课会用消融实验坐实）。
  • 有 LLM + 上下文，但没有工具 → 它能想得头头是道，却对现实世界零影响力。

二、身体隐喻：把公式变成直觉

公式好记，但抽象。作者给了一个一秒就懂的隐喻——把 Agent 想象成一个身体：

这三者不是并排摆着不动，而是转成一个循环：大脑读眼睛看到的信息 → 决定动哪只手脚 → 手脚改变世界 → 世界的新状态又回到眼睛里。Agent 干活的全过程，就是这个圈一圈圈转，直到任务完成。

三、“眼睛”这个类比，在这里漏水

身体隐喻好用，但作者本人在引言里就主动打了个补丁——而这个补丁，恰恰是初学者最容易忽略、后面最常栽跟头的地方。

把“上下文 = 眼睛”理解成“Agent 看到的环境信息”，是不够的，甚至会误导你。作者原话：上下文“不仅包含环境信息和对话历史，还包含工具定义等内容”——也就是说，Agent “看到”的信息里，也包括“它有哪些手脚可用”。

所以别把上下文窄化成“对话历史”或“prompt”。它的准确定义是一句更狠的话：上下文 = 模型能感知到的一切信息。手脚长什么样、能干什么，也是被眼睛看见的——这就是为什么第二章会说“上下文决定 Agent 的能力上限”。

这个隐喻旨在传达核心直觉：上下文是模型能感知到的一切信息。 ——《深入理解 AI Agent》引言

记住这条边界，你就能提前看懂一件后面反复出现的事：工具定义写得好不好、放在上下文的哪个位置，直接决定模型选不选得对工具。因为工具定义本身，就是“眼睛看到的东西”。

四、为什么值得为一个公式单开一课（宏观背景）

因为它是全书的坐标系。作者的野心不是教你跑通一个 Demo，而是把 Agent 设计“从感觉驱动变成原则驱动”。当下一代模型、下一个热门框架出现时，你需要一个不随版本翻篇的框架去安放它——而这个框架就是三个问题：它能看到什么（上下文）、能做什么（工具）、大脑本身有多强（LLM）。

作者还借强化学习之父 Richard Sutton 的话把 Agent 抬到了宇宙演化的高度：宇宙从尘埃→恒星→生命→智能体，走了四个阶段。生物进化是盲目的（随机变异、自然选择，多数生物并不理解自己怎么运作）；而 Agent 是一种全新的存在——它能通过生成代码实现自举（bootstrap）和自我进化，“就像一个程序员编写了另一个程序员，新程序员又能继续编写下一个”。这一句是第 5 章（代码是元能力）和第 8 章（自我进化）的总伏笔。

本课唯一要带走的判断：任何一个 Agent，无论多花哨，都能拆成三个问题——大脑（LLM）多强、眼睛（上下文）看到了什么、手脚（工具）能做什么。而“眼睛”看到的，不只是环境，还包括自己有哪些手脚。以后读任何 Agent 系统、排查任何 Agent 故障，先把它套进这三个格子。

五、关键定义

Agent 核心公式（Agent = LLM + 上下文 + 工具）
全书主线：Agent 由大语言模型（大脑，思考决策）、上下文（眼睛，能看到的信息）、工具（手脚，能做的事）三者构成，缺一不可。对应图0-1。
上下文（Context）
模型能感知到的一切信息——不止环境信息与对话历史，还包含工具定义等内容。对应强化学习里的观察空间（Observation Space）。避免把它窄化为“对话历史”或“prompt”。
工具（Tools）
决定 Agent 能做什么事（搜索网页、调 API、操作数据库……）的能力集合。对应强化学习里的动作空间（Action Space）。避免把它混同为普通的函数调用。
自举（bootstrap）
Agent 通过生成代码实现自我进化——像一个程序员写出了另一个程序员，新程序员又能继续写下一个。避免与编译器 bootstrap 的狭义含义混淆。

六、自测题

Q1：Agent 核心公式的三个组成部分是什么？它们分别对应身体的哪个部位？

LLM + 上下文 + 工具。LLM = 大脑（思考、决策）；上下文 = 眼睛（决定能看到什么信息）；工具 = 手脚（决定能做什么事）。三者缺一不可。

Q2：有人说“上下文就是 Agent 看到的环境信息和对话历史”。按本书的定义，这句话缺了什么关键的一块？为什么这一块很重要？

缺了工具定义。上下文的准确定义是“模型能感知到的一切信息”，其中也包含“Agent 有哪些手脚可用”——工具长什么样、能干什么，同样是被“眼睛”看见的。这一块重要，是因为它直接解释了后面反复出现的结论：工具定义写得好不好、放在上下文哪个位置，会决定模型能否选对工具。把上下文窄化成“对话历史/prompt”，就会漏掉这条因果。
