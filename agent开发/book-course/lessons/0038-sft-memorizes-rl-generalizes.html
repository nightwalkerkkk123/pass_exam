SFT 记忆、RL 泛化：一副 J/Q/K 卡牌讲透两种优化目标

本课只押一件事：SFT 学到的是“见到这种输入就吐那种输出”的固定映射（记忆），RL 学到的是“怎样的过程能得到好结果”的可迁移策略（泛化）。这个分野不是口号，而是能测量的现象——第 7 章用一副卡牌游戏 GeneralPoints 把它量化了出来：训练时 J/Q/K 都当 10，测试时 J 变成 11，SFT 当场崩、RL 反而更稳。把这一节吃透，你就拿到了整章“为什么要 SFT、什么时候必须上 RL”的判断根基。

一、是什么：同一副牌，两种“想要”

作者在 7.1 节把话挑明——SFT 和 RL 的一切差异，都源于一件事：优化目标不同。不是算法花哨程度不同，不是数据量不同，是它们各自“想要什么”根本不一样。

SFT → 像不像标准答案（极大似然）
RL → 结果好不好（期望奖励）

用书里的卡牌例子把它落地。任务像“24 点”：用四张牌上的数字、每个恰用一次、加减乘除凑出 24。训练时约定 J/Q/K 都计为 10。

  • SFT 优化“像不像标准答案”。给定问题，只有唯一一条“对”的输出——那条示范答案。模型被拉着去逼近它，学到的是“看到这种输入，就吐那种输出”的固定映射。于是它记忆：记死“见到 J/Q/K 用 10”。
  • RL 优化“结果好不好”。给定问题，任何能拿高奖励的输出都算好——不止一条。模型自己探索多条路径，哪条结果好就强化哪条，学到的是“怎样的过程能凑出目标”这套策略。于是它泛化。

二、为什么这样设计：优化目标决定了学到什么

要真正理解“为什么”，得先接受书里第一个反直觉的事实：SFT 在数学上和预训练是同一个任务——都是“预测下一个词”、最小化同一个损失函数，只差两点：数据换成了“问题→理想回答”的示范对，以及计算损失时只在回答 token 上回传梯度（loss masking，损失屏蔽）。

正因为 SFT 的目标就是让标注回答里每个 token 的概率尽可能高，“SFT 记忆”就顺理成章了：它被训练成尽量一字不差地复现示范。这在格式固定的任务上样本效率极高（几千条就见效），但能力边界也被钉死在示范数据上——示范里没有的情况它没学过，示范里的答案一旦不再适用（环境变了），它还是照背不误。

RL 则完全不看标准答案，而是让模型自己生成、再按结果好坏给奖惩。因为“任何拿到高奖励的输出都算好”，模型被逼着去找的不是“那一条答案”，而是“能产出好答案的那套过程”。过程是可迁移的，答案不是——这就是泛化的来源。作者在 7.1 节把这句话讲透后，用一张对照表收束：

SFT 学的是“看到这种输入，就输出那种答案”的映射……RL 学的则是“怎样的计算过程能得到正确答案”这一更通用的策略：J 变成 11 时，RL 模型会用同样的策略重新计算，而不是套用记忆中的答案。 ——《深入理解 AI Agent》第 7 章 · 实验 7-11 小结

三、把现象量化：GeneralPoints 实验的数字

光讲道理不够，作者搬出了 GeneralPoints（Chu 等人 2025，《SFT Memorizes, RL Generalizes》）这组对照实验（实验 7-11，架构见图7-12）。设计干净到只留一个变量：同一任务、同一基础模型（Llama-3.2-Vision-11B）、同一计算预算，唯一区别是训练方法用 SFT 还是 RL（RL 部分用带价值网络的 PPO）。都只用单一规则（J/Q/K=10）的数据训练，然后在分布内（ID）与分布外（OOD）两套测试集上评估。

  • 规则变体（GP-L / GP-VL）：训练 J/Q/K=10，测试改成 11/12/13——数字组合是训练没见过的，考“规则泛化”。
  • 视觉变体（GP-VL）：训练用黑色花色（♠♣），测试用红色花色（♥♦），考“视觉外观泛化”。

还有一个值得记的补充结论：验证迭代次数越多，泛化越好——10 次带来 +5.99%，1 次只有 +0.48%。作者据此判断：思考时的计算扩展，是 RL 泛化的关键之一。

四、边界与易错：别把这套结论用歪

误区一：以为 SFT 是和预训练“完全不同的新方法”。不是。作者反复强调——SFT 在数学上就是预训练那个“预测下一个词”，只差换了数据、加了 loss masking。把它想成“另一种训练”，你就会误以为记忆是它的 bug；其实记忆恰恰是“最大化标注答案概率”这个目标的直接后果，是特性不是缺陷。

误区二：把“必须先 SFT 后 RL”当成宇宙规律。书里说得很死：这条只在“较小基础模型 + 严格结构化输出”的设定下成立。本实验里 Llama-3.2-Vision-11B 这个量级不经 SFT 直接 RL 会完全失败——不是效果差，而是基础模型产不出结构化输出、奖励根本无从计算。但足够强的基模（DeepSeek-R1-Zero）能直接 RL 成功、自行涌现反思与长链思考。把边界条件一并记住，才不会张冠李戴。

误区三：想用 SFT/RL 去“记住大量事实”。后训练只擅长固化“怎么用知识”的协议性知识（格式、风格、流程），要灌大量事实性知识该找 RAG 或继续预训练。

五、宏观背景：为什么这一节是全章地基

因为“SFT 记忆、RL 泛化”是第 7 章的两条主线之一，后面所有实验都在反复坐实它。作者由此提炼出“先形后神”的训练范式——借中国画术语：SFT 先把“形”（格式、结构）画准，RL 再追“神”（策略、泛化）。两者不是竞争而是先后：没有稳定的输出格式，RL 的奖励信号就是一片噪声，压根算不出来。这一判断会在第 0039 课展开成“到底什么时候该 SFT、什么时候才轮到 RL”的决策框架。

本课唯一要带走的判断：SFT 优化“像不像标准答案”，学到固定映射，所以记忆、分布一漂移就崩；RL 优化“结果好不好”，学到可迁移策略，所以泛化、换了环境还能重算。同一副牌、同一预算，J 从 10 变 11，就足以让 SFT 掉 8.1% 而 RL 涨 3.5%。以后判断一个后训练该用哪种方法，先问一句：部署环境会不会和训练时不一样？

六、关键定义

监督微调（SFT, Supervised Fine-Tuning）
用几千到几万条“问题—标准回答”示范对继续做“预测下一个词”、且只在回答 token 上算损失的后训练方法，把指令遵循、格式、风格、流程等协议性知识固化进参数。避免泛称为“微调”或等同于“训练”——它特指有标注输入-输出对、带 loss masking 的方法。
损失屏蔽（loss masking）
计算 SFT/工具 RL 损失时把不希望模型学习的 token（问题部分、环境返回的反馈 token）屏蔽掉，只对模型自己生成的回答 token 回传梯度。避免与注意力掩码（attention mask）混为一谈，两者作用位置不同。
寻峰与覆盖式（mode-seeking vs mass-covering）
两种概率分布拟合性格：极大似然（SFT）是 mass-covering，努力覆盖示范里所有模式、雨露均沾；反向 KL 的策略优化（RL）是 mode-seeking，只集中到少数几个高奖励峰、赢者通吃。避免把 mode-seeking 简单等同于“过拟合”，它是分布拟合方向的固有性质。

七、自测题

Q1：训练时 J/Q/K 都当 10，测试时 J 变成 11。为什么 SFT 模型会出错而 RL 模型能答对？

SFT 优化“像不像标准答案”，学到的是固定映射“见到 J/Q/K 就当 10”，J 变 11 时它照旧用 10，所以出错；RL 优化“结果好不好”，学到的是“怎样的计算过程能凑出目标”这套可迁移策略，J 变 11 时用同一策略重算一遍，所以泛化。

Q2：在 GeneralPoints 这组实验里，如果不先做 SFT、直接对基础模型做 RL，会发生什么？这能不能说明“任何模型都必须先 SFT”？

会完全失败：Llama-3.2-Vision-11B 这个量级的基础模型产不出结构化输出，奖励函数根本无法计算，RL 也就无从学起。但这不是普适规律——书里特别强调，这只是“较小基础模型 + 严格结构化输出”这个特定设定下的结论。足够强的基模（如 DeepSeek-R1-Zero）可以跳过 SFT 直接 RL 成功。

GeneralPoints（实验 7-11）本身没有单独的配套目录，但同源研究里的视觉导航版本可以亲手跑：实验 7-12 对应仓库 chapter7/SpatialReasoning/——基于 V-IRL 真实城市街景的视觉-语言导航 RL（带价值网络的 PPO + 过程奖励），在跨城市的视觉 OOD 上把 RL 从 16.7% 拉到 77.8%，把“SFT 记忆、RL 泛化”在多轮、多模态任务上又验证了一遍。完整仓库见 github.com/bojieli/ai-agent-book。
