评估到底在评什么：模型替换实验 vs 消融实验

这一课只带你拿下一件事：评估的对象不是模型，而是「模型 + Harness」这个组合体。想清楚这一点，你就会用两种正交的实验去定位瓶颈——模型替换（固定 Harness、只换模型）判断瓶颈在模型还是在 Harness，消融（关掉某个组件）判断 Harness 里哪个部件在拖后腿。这是第 6 章「Agent 评估」把评估结果转化为改进行动的第一把钥匙，对应本章开篇（chapter6.md）那句关键论断。

一、评估的对象是一个组合体，不是模型

作者在本章一上来就点了一句很重的话：「评估的对象不应只是模型，而应是模型与 Harness 的组合体。」为什么？因为同一个模型放进不同的 Harness，表现能差到天上地下——书里明确说，「一些团队仅通过优化 Harness 就显著提升了同一模型在终端类任务上的表现」。

这句话的直接后果是：当你的 Agent 分数不好看时，「换个更强的模型」往往不是正确答案。真正的问题可能出在提示词、工具设计、反馈循环——也就是 Harness 的某个组件上。所以一套合格的评估体系，必须能把两类本质不同的问题分开：

二、两个动作，不要张冠李戴

书里把两种方法讲得刀切一样清楚，它们操作的对象不同，回答的问题也不同：

  • 消融实验（Ablation）：关闭 Harness 的某个组件（思考模式、上下文压缩、自动记忆……），看整体性能掉多少 → 回答「Harness 内部哪个部件最重要」。
  • 模型替换实验（Model Swap）：固定 Harness、只换模型（换更强或更弱的），看分数波动幅度 → 回答「瓶颈到底在模型还是在 Harness」。

四、为什么这套方法值钱：一个真实的 2×2 案例

五、边界与最容易踩的坑

坑一：把模型替换和消融当成一回事。初学者常常一看到「做个实验定位问题」就随手关组件，却以为自己在判断「要不要换模型」。记住操作对象：消融关组件（定位 Harness 内部），模型替换换模型（区分模型 vs Harness）。用错了，你会关掉一堆组件、得出一堆「这个特性有用」的结论，却始终没回答「到底该不该升级模型」这个最初的问题。

坑二：把一次运行的分差当真、看到 3% 就切模型。书里算得很实：100 个用例、成功率 70%，二项分布标准误约 √(0.7×0.3/100)≈4.6%，95% 置信区间约 70%±9 个百分点。把两个成功率当独立来比，差值的标准误还要再乘 √2、约 6.5%。所以「新模型 73% vs 旧模型 70%」这 3 个百分点完全落在噪声带宽里——据此切换模型，「与掷硬币差别不大」。正确做法：每个配置跑 3-5 次取均值；同一批任务上用配对分析（McNemar 检验的思路，只看一对一错的用例），它扣掉了「题目难易」这个共同噪声，比「两个独立成功率相减」灵敏得多。分差小于噪声带宽时，不做切换决策。

本课唯一要带走的判断：你评的从来不是模型，而是「模型 + Harness」组合体。分数不好，先别急着换模型——用模型替换（固定 Harness、只换模型）判断瓶颈在模型还是 Harness：换强不涨 → 瓶颈在 Harness，换弱大跌 → 瓶颈在模型；确认锅在 Harness 后，再用消融（逐个关组件）定位是哪个部件。而且任何分差，都要先量一量它有没有超出噪声带宽。

六、关键定义

模型替换实验（Model Swap）
固定 Harness、只更换更强或更弱的模型，观察分数变化幅度，用以区分性能瓶颈在模型能力还是在 Harness 设计。避免与消融实验混用——消融是关闭 Harness 组件，模型替换是换模型。
消融实验（Ablation Study）
逐一「摘除」某个组件（思考模式、上下文压缩、自动记忆等），看整体性能掉多少，从而判断该组件的真实贡献；用来回答「某个特性是真的改善了体验，还是只是感觉有用」。
二项分布标准误（Standard Error）
刻画成功率因抽样随机而波动的幅度，值越大说明该成功率越不可靠。n 个用例上成功率 p 时约为 √(p(1-p)/n)；95% 置信区间约 p±2 个标准误。避免把落在噪声带宽内的分差当成真实提升。
配对分析（Paired Analysis / McNemar）
同一批任务上对比两个配置的默认正确做法：逐题比较胜负、只看结果不同的用例，扣除「题目本身难易」这一共同噪声源，因此比「两个独立成功率相减」灵敏得多。

七、自测题

Q1：你的 Agent 分数不理想，换上业界最强模型分数几乎没涨，这说明瓶颈最可能在哪里？该做消融还是模型替换来进一步确认？

瓶颈大概率在 Harness（提示词/工具/反馈循环）而非模型能力——因为换强模型不涨，说明模型不是限制因素。此时应做消融实验：逐一关闭 Harness 的某个组件，看整体性能掉多少，定位是哪个部件在拖后腿。模型替换已经完成了它的使命（区分瓶颈在模型还是 Harness）。

Q2：你把新模型和旧模型各在 100 个用例上跑了一次，成功率 73% vs 70%。能据此判定新模型更好、直接全量切换吗？

不能。100 个用例、成功率 70% 时二项标准误约 4.6%，95% 置信区间约 70%±9 个百分点；把两个成功率当独立比较，差值标准误约 6.5%。3 个百分点完全落在噪声带宽之内，据此切换「与掷硬币差别不大」。正确做法是每个配置跑 3-5 次取均值，并在同一批任务上做配对分析（McNemar），确认差异超出噪声后再决策——分差小于噪声带宽时不做切换。

想把「读报告 → 提假设 → 跑 2×2 → 做决策」的完整闭环亲手走一遍，做实验 6-10（对应仓库 chapter6/android-world/）：以 AndroidWorld 上 t3a Agent 的真实评估报告为起点，交叉分析逐任务表格与能力标签矩阵，构建三层改进假设，正是本课那个 H5/H6 案例的出处。要顺带理解配对比较背后的排名机制，可做实验 6-6 / 6-8（对应仓库 chapter6/elo-leaderboard/）：用 Chatbot Arena 的真实盲选投票数据从零实现 Elo 与 Bradley-Terry 模型。整本书实验都在 github.com/bojieli/ai-agent-book。
