Rubric 四准则与一票否决：给开放式任务打分的红线

开放式任务（写报告、处理投诉、创意内容）没有标准答案可比对，只能让评判模型照着 Rubric（评分标准）打分。这节课只押一件事：Scale AI 的 Rubric 四准则，尤其是其中最反直觉的一招——把幻觉、严重安全违规设成 一票否决项（veto），再把抽象标准改写成自包含、可验证的具体行为。这两手合起来，是本章第 6 章“LLM-as-a-Judge”一节对抗奖励作弊的核心武器。

一、先看一张表：Rubric 把一次轨迹拆成可检查的维度

注意最后一行和前三行类型不同：前三个是 1–4 分的分级维度，幻觉检测却只有“通过 / 否决”。书里点明了原因：幻觉与质量是正交的——一个流畅、详尽、礼貌的回答如果包含虚假事实，对用户的伤害远大于一个简短但准确的回答。所以它不该被摊进加权平均里，而要单独设成一道红线。

二、Rubric 四准则（Scale AI，“Rubrics as Rewards”）

为了系统性地对抗评判偏差和作弊，书里给出一套四准则。把它记成“专家、覆盖、权重、自包含”四个词：

逐条拆开看它们各自在防什么：

  • ① 基于专家指导——必须反映领域知识、抓住核心事实和推理步骤。缺乏专业基础的 Rubric 只能捕捉“语言流畅度”这类表面特征（比如医疗问答，要写进诊断标准和必须避免的医学错误）。
  • ② 全面覆盖——涵盖事实准确性、逻辑连贯性、完整性、安全性；而且不只定义正面标准，还要明确陷阱（Pitfall），即高风险的常见错误（如推荐未经验证的疗法）。
  • ③ 标准重要性权重——分成必要项（Essential）、重要项、可选项、陷阱项，并支持一票否决（Veto）。这是本课的核心，下一节展开。
  • ④ 自包含评估——每个评价项独立可操作，不依赖评价者的领域知识。要避免“回应展示了深刻理解”这种抽象标准，改成“引用了至少两个权威理论并准确解释如何支持结论”这种可验证标准。

三、一票否决：为什么它无论如何都直接判零

准则③里最狠的一招是 veto。它不参与加权平均，逻辑很简单粗暴：某个否决维度（幻觉、严重安全违规）一旦触发，无论其他维度多满分，总分直接归零。

书里给了一个符合四准则的完整 Rubric 作为范本（用户记忆 Agent，问题是“我女儿的儿科医生是谁？”，答案要跨两次对话关联：第一次提到“女儿叫 Lily”，第二次提到“带 Lily 去看了 Dr. Chen”）。它的四个维度权重是这样标的：

事实正确性=essential   信息完整性=important   思考正确性=important   幻觉检测=veto
幻觉检测：一旦触发，总分归零

而“好的 Rubric vs 坏的 Rubric”的分水岭，就在准则④：每个评分档都写成可验证的具体行为（“准确回答 Dr. Chen”），而不是“展示了对记忆的深刻理解”这种无法客观判定的描述。否决项则把底线钉死：即使其他维度全部满分，一旦出现幻觉就直接判零。

四、更深的“为什么”：这是在对抗古德哈特定律

为什么要这么费劲地防作弊？因为一旦 Rubric 变成 Agent 的优化目标，它就会退化。书里点破了这条铁律——古德哈特定律：当一个度量指标变成优化目标时，它就不再是好的度量指标。Agent 越是在某个评分系统上训练或调优，就越倾向于钻这个系统的漏洞，而非真正提升能力。

所以 Rubric 设计要主动防范 奖励作弊（Reward Hacking）——即 Agent 找到了获取高分的“捷径”却没有真正完成任务。具体手段就是明确惩罚：幻觉、讨好用户、关键词堆砌、回避棘手问题。veto 只是其中最硬的一档；此外还要为每个维度定义客观可验证的评分档次，配上具体示例和边界案例帮助区分模糊情况。

初学者最容易想错的地方：把否决项当成“权重特别大的一个维度”，用加权平均去算——“质量三项都满分，幻觉扣一点，平均下来还挺高”。这恰恰是 Rubric 想堵死的思路。

否决项根本不参与加权。它是一道逻辑闸门，输出只有“通过 / 否决”两态，触发即清零。理由是幻觉与质量正交——它衡量的不是“答得好不好”，而是“能不能信”。一个既流畅又详尽却夹带假信息的回答，比一个简短但句句属实的回答更危险。如果你用加权平均，等于给了模型一条捷径：把其他维度堆到极高，就能稀释掉造假的代价——这正是奖励作弊。安全与合规违规同理，书里对它们也采取零容忍：一次严重违规即否决整体评价。

这正是古德哈特定律所说的：当一个度量指标变成优化目标时，它就不再是好的度量指标。 ——《深入理解 AI Agent》第 6 章
本课唯一要带走的判断：Rubric 四准则里，真正救命的是准则③长出的一票否决和准则④要求的自包含可验证。幻觉/安全违规不打分、只设闸门，触发即清零；每个评分档必须写成能被客观核对的具体行为。做到这两点，你的评分系统才不会被“流畅但造假”的回答骗过，也才扛得住 Agent 在上面反复调优时的钻空子。

五、关键定义

Rubric（评分标准 / 评分准则）
量化 Agent 表现的评分依据，可以是二元的（通过/不通过）、连续的（0–100 分）或多维的（分别给准确性、效率、安全性打分）。开放式任务无标准答案，靠它让 LLM-as-a-Judge 逐维度评判。
一票否决机制（Veto）
Rubric 中的否决项设计：某维度（如幻觉、严重安全违规）一旦触发，无论其他维度多优秀都直接把总分判零。因为该维度与质量正交且伤害极大。区别于普通的分级评分维度，否决项不参与加权。
奖励作弊（Reward Hacking）
Agent 找到了获取高分的“捷径”却没有真正完成任务（如关键词堆砌、讨好用户、回避棘手问题）。Rubric 需明确惩罚这些行为，veto 是最硬的一道防线。
古德哈特定律（Goodhart's Law）
当一个度量指标变成优化目标时，它就不再是好的度量指标；Agent 越在某评分系统上调优，越倾向钻系统漏洞而非真正提升能力。这是所有防作弊设计的根本动因。

六、自测题

Q1：一个客服回答流畅、详尽、礼貌，四个质量维度都接近满分，但编造了一个对话中不存在的就诊日期。用符合四准则的 Rubric 打分，总分应该是多少？为什么这样设计？

总分归零。因为幻觉是 veto 否决项——它与质量正交，一个流畅详尽的回答若含虚假事实，对用户的伤害远大于简短但准确的回答。否决机制无论其他维度多优秀都直接判零，同时能防范关键词堆砌、讨好用户等奖励作弊（古德哈特定律：度量变成目标就不再是好度量）。

Q2：按准则④（自包含评估），下面哪条评分档是“坏”的，应该怎么改？——“该维度得 4 分：回应展示了对记忆的深刻理解”。

这是坏的评分档：“展示了深刻理解”是抽象标准，依赖评价者的主观判断，无法客观核对。按准则④要改成独立可操作、可验证的具体行为，例如书里范本里的写法——“准确回答 Dr. Chen，且关联到女儿 Lily”。判定标准写得越像“能勾选的清单”，不同评判者给出的分就越一致，也越难被钻空子。
