三阶段炼成：预训练、SFT、RL 各在做什么

这节课只带你拿下一张地图：一个现代模型的能力，是分预训练 → SFT → RL 三步、按流水线炼成的，每一步用的数据、优化目标、代价各不相同。这张图在第 7 章开篇的“预训练、SFT、RL：三阶段全景”一节（对应表7-1），是读懂整章后训练技术的钥匙——后面所有花哨的算法，都能往这三个格子里对号入座。先把地基这张图刻进脑子。

一、是什么：三步流水线，不是三选一

作者在本章开篇把话说得很直白：一个现代模型的能力，是分三个阶段炼成的，“这三个阶段环环相扣，缺一不可”。

  • 预训练（Pre-training）：在海量互联网文本上做“预测下一个词”，学会语言规律、世界知识和基本推理。像“读完了图书馆里所有书”——博学，但还不会好好回答问题。最贵的一步（数百万～数千万美元），是能力的地基。
  • SFT（监督微调）：用几千到几万条“问题—标准回答”示范对，教会模型“该用什么格式、风格、流程来回答”。把博学的模型变成听得懂指令、输出规整的助手。便宜、快、稳，几乎所有部署模型都要经过这一步。
  • RL（强化学习）：不再给标准答案，让模型自己去试，把做得好的行为概率调高、做得差的调低。让它在没见过的情况下也能做出合理决策。篇幅最大、最需要工程功力，常是 SFT 的几十～上百倍代价。

关键：三者的关系不是三选一，而是流水线——先读书，再看示范，最后实战。

二、为什么：SFT 其实是“换了数据的预训练”

这是本章第一个要打通的认知，也解释了为什么 SFT 那么便宜：SFT 在数学上和预训练是同一个任务——都是“预测下一个词”、最小化同一个损失函数。很多初学者以为 SFT 是一种全新方法，其实不是。它和预训练只差两点：

  • 数据不同。 预训练用原始互联网文本（无结构、什么都有）；SFT 用人工精心准备的“输入—输出”对，格式统一为“用户提问 → 理想回答”。
  • 损失只算在“回答”上（loss masking，损失屏蔽）。 一条 SFT 样本含问题和回答两部分；我们不希望模型学“怎么提问”，只想它学“怎么回答”，所以算损失时把问题部分的 token 屏蔽掉，只对回答部分回传梯度。这是 SFT 在工程上与预训练唯一实质性的区别。

理解了这一点，“SFT 记忆”就顺理成章：它的优化目标是让标注回答里每个 token 的概率尽可能高——说白了就是“把这条标准答案背下来”。所以 SFT 固化的是“格式、风格、流程”这类协议性知识（该怎么说、怎么做），而不是大量事实性知识（知道什么）。

三、为什么必须先 SFT 后 RL，而不是反过来

预训练排最前没争议——没有语言和知识的地基，后面无从谈起。真正要解释的是：为什么 SFT 要在 RL 之前？

答案藏在 RL 的工作方式里。RL 不看标准答案，而是让模型自己生成回答，再按好坏给奖惩。可要判断好坏，首先得能把输出解析出来：如果任务要一段 JSON 或一次工具调用，模型却吐出一团格式混乱的文本，奖励函数根本无从算起（连“成功还是失败”都判断不了），RL 也就无从学起。

所以 SFT 扮演“先把话说利索”的角色：用少量示范让输出格式稳定、能被可靠解析，RL 才有一个能打分的起点。借用中国画的说法——SFT 先把“形”（格式、结构）立起来，RL 再追求“神”（策略、泛化），即先形后神。

反过来先 RL 后 SFT 行不通——没有稳定的输出，奖励信号就是一片噪声。 ——《深入理解 AI Agent》第 7 章

四、边界与最容易想错的地方

误区一：把 SFT 当成一种和预训练完全不同的新方法。 初学者常觉得“微调”是另起炉灶的新技术。其实 SFT 在数学上就是预训练那个“预测下一个词”，只差数据和 loss masking 两点。想不清这一点，就理解不了“SFT 为什么倾向记忆”——因为它的目标本来就是“把标准答案背下来”。

误区二：把“必须先 SFT 后 RL”当成普适铁律。 这个顺序只在“较小基础模型 + 严格结构化输出”的设定下成立（实验 7-11 里 Llama-3.2-Vision-11B 这个量级不经 SFT 直接 RL 会完全失败）。但若基础模型足够强，它一上来就能产出够格的输出，从而跳过 SFT——DeepSeek-R1-Zero 就证明了强基模可以直接 RL 成功，自行涌现出反思与长链思考。代价是输出可读性差、中英文混杂，所以 DeepSeek 最终仍在 R1 里加回“冷启动 SFT”，把“形”重新立稳。R1 从 Zero 到冷启动的往返，正是“先形后神”的最好注脚。

五、关键定义

监督微调（SFT, Supervised Fine-Tuning）
用几千到几万条“问题—标准回答”示范对继续做“预测下一个词”、且只在回答 token 上算损失的后训练方法，把指令遵循、格式、风格、流程等协议性知识固化进参数。避免泛称为“微调”或等同于“训练”；它特指有标注输入-输出对、带 loss masking 的方法。
损失屏蔽（loss masking）
计算 SFT/工具 RL 损失时，把不希望模型学习的 token（问题部分、环境返回的反馈 token）屏蔽掉，只对模型自己生成的回答 token 回传梯度。避免与注意力掩码（attention mask）混为一谈，两者作用位置不同。
寻峰与覆盖式（mode-seeking vs mass-covering）
两种概率分布拟合性格：极大似然（SFT）是 mass-covering，努力覆盖示范里所有模式、雨露均沾；反向 KL 的策略优化（RL）是 mode-seeking，只集中到少数几个高奖励峰、赢者通吃。这解释了为什么经过 RL 的模型回答更“笃定”。避免把 mode-seeking 简单等同于“过拟合”，它是分布拟合方向的固有性质。

六、自测题

Q1：为什么说 SFT 在数学上和预训练“是同一个任务”？它俩到底只差哪两点？

因为两者都是“预测下一个词”、最小化同一个损失函数。只差两点：(1) 数据不同——预训练用原始互联网文本，SFT 用人工准备的“输入—输出”示范对；(2) 损失屏蔽——SFT 计算损失时屏蔽掉问题部分的 token，只对回答部分回传梯度。

Q2：“必须先 SFT 后 RL”这个顺序，成立的前提是什么？什么情况下可以跳过 SFT 直接 RL？

它在“较小基础模型 + 严格结构化输出”的设定下成立——因为 RL 要靠奖励函数打分，得先有能被可靠解析的稳定输出，SFT 负责把“形”（格式）立起来，否则奖励信号是一片噪声。但当基础模型足够强时，它一上来就能产出够格的输出，可以跳过 SFT 直接 RL——DeepSeek-R1-Zero 即是证明，自行涌现出反思与长链思考；代价是输出可读性差、中英文混杂，所以 R1 又加回了“冷启动 SFT”。

配套实验 7-3（对应仓库 chapter7/MiniMind-pretrain/）：在消费级 GPU 上从头训一个 100M 的 MiniMind 2，走完 tokenizer → 预训练 → SFT → RLHF(DPO) 全流程——正好把这张三阶段地图亲手跑一遍（约 14 小时 / 34 美元）；顺带演示 QK Norm + Muon 优化器带来的算法改进威力。开源仓库总入口：github.com/bojieli/ai-agent-book。

本课唯一要带走的判断：模型能力分三步炼成——预训练读万卷书（贵、给知识地基）、SFT 看示范（便宜、固化格式与协议、倾向记忆）、RL 下场做题（贵、学可迁移策略、倾向泛化）。SFT 本质就是“换了数据、只在回答上算损失”的预训练；“先形后神、先 SFT 后 RL”是稳健配方，但对强基模并非铁律。以后读任何后训练技术，先问它落在这三格的哪一格。
