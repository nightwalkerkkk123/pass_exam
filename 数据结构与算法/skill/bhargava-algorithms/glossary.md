# 《算法图解》术语表 (Glossary)

按术语拼音（英文/缩写按字母）排序，括号标注出现章节。

**背包问题 (Knapsack Problem)** — 在背包容量约束下选择一组不可分割的商品，使总价值最大化的组合优化问题。(ch09)
**编辑距离 (Levenshtein Distance)** — 衡量两字符串相似程度、可用动态规划求出的指标，用于拼写检查、查重等场景。(ch09)
**并行管理开销与负载均衡** — 多核加速非线性的两大现实阻力：任务拆分/合并耗时，以及任务难度不均导致部分核心空闲。(ch11)
**常量时间 (O(1))** — 不管数据规模多大，操作所需时间都相同，是散列表平均情况下的复杂度。(ch05)
**冲突 (collision)** — 两个不同的键被散列函数映射到了数组的同一个位置。(ch05)
**大O表示法 (Big O Notation)** — 描述算法运行时间随输入规模增长速度（而非绝对秒数）的记号，通常基于最糟情况。(ch01)
**大O表示法中的常量** — 大O描述增长趋势时省略的固定系数，阶数相同时常量差异决定实际速度（如快排常量小于合并排序）。(ch04)
**单向散列** — SHA 类函数可正向算出散列值，但无法从散列值反推原始输入，是密码安全存储的基础。(ch11)
**调用栈 (call stack)** — 计算机用来管理函数调用的栈结构，每次调用分配一块内存（栈帧）保存局部变量。(ch03)
**递归 (Recursion)** — 函数直接或间接调用自身来解决问题的编程技巧。(ch03)
**递归条件 (recursive case)** — 问题尚未达到基线条件时，用于缩小问题规模并再次调用自身的分支逻辑。(ch03, ch04)
**递推公式/状态转移方程** — 描述如何用相邻已知单元格的值计算当前单元格值的规则，因问题而异，没有通用形式。(ch09)
**动态规划 (Dynamic Programming)** — 将问题拆解为彼此独立的离散子问题，用网格记录并组合子问题的解，求解约束条件下最优值的方法。(ch09)
**动态规划网格 (DP grid)** — 行列分别对应商品/字符与约束的离散取值，单元格存放对应子问题最优解的表格。(ch09)
**队列 (Queue)** — 先进先出（FIFO）的数据结构，只支持入队和出队，是实现 BFS 的核心结构。(ch06)
**对数时间 (Logarithmic Time)** — 操作数以 log2n 增长，记为 O(log n)；对数是幂运算的逆运算。(ch01)
**二分查找 (Binary Search)** — 在有序列表中每次取中间元素比较、并排除一半剩余范围的查找算法，运行时间为 O(log n)。(ch01)
**分类 (classification)** — 把样本分到已有的类别中，即"编组"。(ch10)
**分区 (partitioning)** — 把数组按是否小于/大于基准值分成两个子数组的操作，是快速排序每层递归的核心工作。(ch04)
**父节点 (parent)** — 记录到达某节点的最优路径上紧邻的前一个节点，用于算法结束后回溯出完整路径。(ch07)
**负权边 (negative-weight edge)** — 权重为负的边，存在时会使狄克斯特拉算法的核心假设失效，导致结果错误。(ch07)
**概率型数据结构** — 用牺牲绝对精确性换取巨大空间/时间节省的数据结构，如布隆过滤器、HyperLogLog。(ch11)
**公钥与私钥** — Diffie-Hellman/RSA 类非对称加密机制，公钥公开用于加密，私钥保密用于解密。(ch11)
**归纳证明 (inductive proof)** — 通过证明基线条件成立、且从 n 成立可推出 n+1 也成立，证明算法对任意规模输入都有效的方法。(ch04)
**环 (cycle)** — 从某节点出发经过若干边又回到该节点；正权重的环只会增加总开销，不会出现在最短路径中。(ch07)
**缓存 (cache)** — 把已计算/请求过的结果保存起来避免重复计算，通常存储在散列表中。(ch05)
**回归 (regression)** — 预测一个数值型结果，例如评分或销量。(ch10)
**键与值 (key / value)** — 散列表存储的基本单元，键经散列函数计算出索引，值存放在该索引处。(ch05)
**简单查找 (Simple Search)** — 从头到尾逐个检查元素，最坏情况下需检查全部 n 个元素，运行时间为 O(n)。(ch01)
**加权图 (weighted graph)** — 边带有权重的图，需用狄克斯特拉算法（而非广度优先搜索）求最短路径。(ch07)
**阶乘时间 / 阶乘函数 (Factorial Time)** — 操作数（或方案数）按 n 的阶乘增长，是最快的增速类别，典型例子是旅行商问题暴力枚举（约 n! 条路线）。(ch01, ch08)
**集合 (Set)** — 类似列表但不允许重复元素的数据结构，支持并集、交集、差集运算。(ch08)
**近似算法 (Approximation Algorithm)** — 精确解耗时过长时，用来快速得到足够接近最优解的算法，评价标准是速度与接近程度。(ch08)
**机器学习 (machine learning)** — 让计算机通过数据自动变得更聪明的技术领域，KNN 是其入门算法之一。(ch10)
**基线条件 (base case)** — 递归函数中不再继续调用自身、直接返回结果的终止情形（处理数组时通常是空数组或单元素数组），用于避免无限递归。(ch03, ch04)
**基准值 (pivot)** — 快速排序中从数组里选出的参照元素，其余元素据此分为小于和大于两组。(ch04)
**局部最优解 (Local Optimum)** — 只考虑当前这一步能拿到的最好选择，不考虑对后续步骤的全局影响。(ch08)
**局部敏感 vs 局部不敏感** — SHA 追求输入微小变化导致散列值剧变（防破解）；Simhash 追求输入微小变化只带来散列值微小变化（便于比较相似度）。(ch11)
**距离公式（欧几里得距离）** — 形如 sqrt(Σ(xi−yi)²) 的公式，可推广到任意维度，用来量化两组数字之间的相似程度。(ch10)
**开销 (cost)** — 算法运行过程中，从起点到某节点当前已知的最小总权重，未知时设为无穷大。(ch07)
**K最近邻算法 (KNN)** — 通过比较新样本与已知样本的距离、参考最近的 K 个邻居来完成分类或回归的算法。(ch10)
**链表 (Linked List)** — 元素分散存放、每个元素记录下一个元素地址的数据结构，插入删除代价低。(ch02)
**邻居 (Neighbor)** — 与某节点直接相连的节点，只有一步之遥的节点才算邻居。(ch06)
**离散子问题 (discrete/independent subproblem)** — 子问题互不依赖，是动态规划能够正确工作的前提条件。(ch09)
**旅行商问题 (Traveling Salesman Problem)** — 求访问 n 个城市一次并返回的最短路径问题，暴力枚举所有排列的运行时间为 O(n!)。(ch01)
**map 函数** — 对数组每个元素独立施加同一操作，得到等长新数组，天然可并行分发。(ch11)
**幂集 (Power Set)** — 一个集合所有子集构成的集合，大小为 2 的 n 次方，是集合覆盖问题精确解法需枚举的对象。(ch08)
**NP完全问题 (NP-complete Problem)** — 以难解著称的一类问题，如旅行商问题和集合覆盖问题，目前没有已知的可快速求解算法。(ch08)
**OCR（光学字符识别）** — 让计算机从图片中自动识别文字/数字的技术，核心思路与 KNN 一致，只是特征更复杂。(ch10)
**平均情况 (average case)** — 算法在随机/典型输入下的性能表现，快速排序的平均情况是 O(n log n)。(ch04)
**全局最优解 (Global Optimum)** — 整个问题在所有可能方案中真正的最优解。(ch08)
**权重 (weight)** — 加权图中每条边关联的数字，代表通过该边需要付出的代价（时间、金钱等）。(ch07)
**reduce 函数** — 把一个数组归并为单一值，如求和。(ch11)
**散列表 (hash table)** — 由数组和散列函数组合而成的数据结构，用键映射到值，也称散列映射、映射、字典、关联数组。(ch05)
**散列函数 (hash function)** — 将任意输入映射为一个数字（索引）的函数，是散列表的核心组件。(ch05)
**树 (Tree)** — 一种特殊的图，其中没有指向上级的回指边，例如家谱。(ch06)
**顺序访问 (Sequential Access)** — 必须从第一个元素开始、沿链接逐个读取才能到达目标元素的方式，链表只能这样访问。(ch02)
**数组 (Array)** — 元素在内存中连续存放的数据结构，支持通过索引直接定位任意元素。(ch02)
**随机访问 (Random Access)** — 可以直接跳转到指定位置读取元素的访问方式，是数组的强项。(ch02)
**索引 (Index)** — 数组中元素的位置编号，从 0 开始计数，而非从 1 开始。(ch02)
**特征抽取 (feature extraction)** — 把物品（水果、用户、数字图像等）转换为一组可比较数字（坐标）的过程，是使用 KNN 的前提。(ch10)
**填装因子 (load factor)** — 散列表中已被占用的位置数除以总位置数，衡量散列表的拥挤/性能风险程度。(ch05)
**调整长度 (resizing)** — 当填装因子过高时创建更大（通常两倍）的新数组，并重新插入所有元素。(ch05)
**图 (Graph)** — 由节点（node）和边（edge）组成的结构，用于建模事物之间的连接关系。(ch06)
**尾递归 (tail recursion)** — 一种可被部分语言优化以避免栈持续增长的高级递归技巧。(ch03)
**误报与漏报** — 布隆过滤器只可能误报（说存在实际不存在），绝不会漏报（说不存在就真的不存在）。(ch11)
**无向图 (Undirected Graph)** — 边不带箭头、关系双向对称的图，例如 ross - rachel 表示互相约会。(ch06)
**线性时间 (Linear Time)** — 操作数与输入规模 n 成正比增长，记为 O(n)。(ch01)
**选择排序 (Selection Sort)** — 反复从未排序部分找出最小元素并移入新数组，直至排序完成的算法，时间复杂度 O(n²)。(ch02)
**训练 (training)** — 机器学习算法在正式使用前，先浏览大量已知数据、提取特征的准备过程。(ch10)
**已处理节点 (processed)** — 一旦某节点被处理，算法就假设不存在到它的更便宜路径，之后不再重新考虑——这是贪心假设的核心，也是负权边会破坏它的原因。(ch07)
**有向图 (Directed Graph)** — 边带箭头、关系单向的图，例如 rama→adit 表示 rama 欠 adit 钱但反过来不成立。(ch06)
**有向无环图 (DAG)** — 狄克斯特拉算法本质上要求的图结构；无向图因每条边都等价于一个环需格外小心。(ch07)
**预留空间 (over-provisioning)** — 为数组预先申请比当前所需更多的位置以延缓整体搬迁，是权宜之计而非根本解法。(ch02)
**栈 (Stack)** — 只支持压入（push）和弹出（pop）两种操作的后进先出（LIFO）数据结构。(ch03, ch06)
**栈溢出 (stack overflow)** — 调用栈过高、占用内存过多导致内存耗尽的风险，是深层递归的主要代价。(ch03)
**最长公共子串 (Longest Common Substring)** — 两字符串中都存在的连续字符序列的最大长度。(ch09)
**最长公共子序列 (LCS)** — 两字符串中都包含、但不要求连续排列的字符序列的最大长度。(ch09)
**最短路径问题 (Shortest-Path Problem)** — 寻找从节点 A 到节点 B 所需经过边数最少的路径的问题，BFS 是其经典解法。(ch06)
**最糟情况 (Worst Case)** — 算法在最不利输入下所需操作数的上界（大O默认给出的保证）；如快速排序在已排序数组+固定选首元素为基准值时退化为 O(n²)。(ch01, ch04)
