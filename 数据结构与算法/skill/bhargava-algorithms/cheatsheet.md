# 算法图解 · 考试速查表

## 1. 复杂度总表（跨全书，考前必背）

| 结构/算法 | 平均 | 最坏 | 备注 |
|---|---|---|---|
| 简单查找 | O(n) | O(n) | 无序 |
| 二分查找 | O(log n) | O(log n) | 需已排序 |
| 数组·按下标读 | O(1) | O(1) | 公式定址 |
| 数组·插入/删除 | O(n) | O(n) | 需搬家；末尾插入摊销O(1) |
| 链表·查找/读取 | O(n) | O(n) | 需顺着走 |
| 链表·插入/删除 | O(1) | O(1) | 前提：位置已知，否则先查找是O(n) |
| 选择排序 | O(n²) | O(n²) | 嵌套循环 |
| 快速排序 | O(n log n) | O(n²) | 随机基准防最坏；常量比归并小 |
| 归并排序 | O(n log n) | O(n log n) | 稳定但需O(n)额外空间 |
| 散列表 增/删/查 | O(1) | O(n) | 填装因子>0.7需扩容 |
| BFS | O(V+E) | O(V+E) | 仅限无权图 |
| Dijkstra（简单实现） | O(V²) | O(V²) | 仅限非负权图 |
| 贪婪·集合覆盖近似 | O(n²) | O(n²) | 非最优但够快 |
| DP·0/1背包 | O(n·W) | O(n·W) | n件物品×W容量网格 |
| KNN（暴力） | O(n) | O(n) | n=样本数，每次查询 |
| 二叉查找树 | O(log n) | O(n) | 退化成链表时最坏 |
| 集合覆盖精确解 | — | O(2ⁿ) | 枚举幂集，NP完全 |
| 旅行商穷举 | — | O(n!) | NP完全 |

增速排序：`O(log n) < O(n) < O(n log n) < O(n²) < O(n!)`（大O=最坏情况操作数增速，非秒数）

## 2. 决策速查（当X用Y，因为Z）

| 场景 | 选用 | 原因 |
|---|---|---|
| 有序数据查找 | 二分查找 | n越大较简单查找差距越悬殊 |
| 频繁随机访问/下标读 vs 频繁任意位置增删 | 数组 vs 链表 | 前者地址公式O(1)，后者只改指针O(1) |
| 数组"预留空位" | 仍是权宜之计 | 超额后照样整体搬家 |
| 嵌套深度不确定的遍历 | 递归(调用栈隐式记账)，基线设为空/单元素数组，先写基线再写递归条件；层数太深则改循环 | 否则栈溢出；循环通常比递归性能更优 |
| 输入接近有序 | 快排随机选基准 | 固定选首元素→一侧恒空→O(n²) |
| 键→值查找/去重/缓存 | 散列表 | 平均O(1) |
| 散列分布集中/非确定 | 换均匀且确定的散列函数 | 否则退化成链表或直接失效 |
| 图求"最少几步/有无路径" | BFS，名单用队列非栈，查过即标记 | 逐层扩展先到即最短；防重复入队死循环 |
| 带正权重求最小总代价 | Dijkstra | — |
| 图含负权边 | 禁用Dijkstra，改贝尔曼-福特 | 已处理节点不重考虑，负权可事后更优 |
| 需具体最短路径 | 靠parents回溯 | 非正向遍历直接得出 |
| 可分解为独立选择步骤/规模呈阶乘指数增长 | 先贪婪探路/怀疑NP完全 | 快速拿基准解；转近似算法 |
| 背包类资源+价值最大化 | 慎用贪婪 | 先查反例(音响vs笔记本+吉他) |
| 可归约为集合覆盖/旅行商 | 判定NP完全 | — |
| 约束下求最优+子问题独立 | DP网格法 | 暴力O(2ⁿ)→多项式级 |
| 资源可分割(连续量) | 优先贪婪非DP | DP只能二元"拿/不拿" |
| 子问题相互依赖 | 禁用DP | 正确性依赖子问题独立 |
| 最长公共子串 | 看网格最大值 | 未必末尾对齐 |
| 最长公共子序列 | 看网格最后一格,max(左,上) | 允许不连续 |
| 分类/预测新样本 | 先抽特征再KNN比较K个近邻 | 特征须相关，否则先修正 |
| 打分尺度不一致 | 余弦相似度非欧氏距离 | — |
| 数据到百万级/变量近乎随机(股票) | 重估K值/别指望KNN | 小K不适用大数据；纯随机数据ML也救不了 |
| 频繁插入且要保持有序可查 | 二叉查找树非有序数组 | 数组插入需重排 |
| 单机处理不动的海量数据 | MapReduce/分布式 | 横向扩展优于纵向优化 |
| 只需存在性判断/近似计数+省内存 | 布隆过滤器/HyperLogLog | 牺牲精确换空间 |
| 密码散列/判相似度非精确匹配 | SHA-2/SHA-3/bcrypt；Simhash非SHA | 禁SHA-0/1(已破)；二者局部敏感性目标相反 |

## 3. 决策树：选数据结构/算法

```
查找？
├─ 已排序 → 二分查找 O(log n)
└─ 未排序 → 简单查找 O(n)

存数据？
├─ 常按下标随机读 → 数组
├─ 常任意位置增删 → 链表
├─ 常按键查/去重/缓存 → 散列表
└─ 常插入又要保持有序 → 二叉查找树

图论？
├─ 无权，求最少边数 → BFS
├─ 正权，求最小总代价 → Dijkstra
└─ 含负权边 → 贝尔曼-福特

最优化？
├─ 资源连续可分 → 贪婪算法
├─ 离散不可分+子问题独立 → 动态规划
├─ 子问题相互依赖 → 都不适用，另寻他法
└─ 需枚举全部排列/子集(疑NP完全) → 贪婪近似算法

分类/预测？
├─ 特征有代表性 → KNN（尺度不一→余弦相似度）
└─ 数据量大 → 重估K；纯随机变量 → 别用ML

数据量爆炸？单机放不下→MapReduce；只判存在性/计数→布隆过滤器/HyperLogLog
```

## 4. 硬阈值 / 经验法则

| 阈值 | 动作 |
|---|---|
| 散列表填装因子 > 0.7 | 扩容（通常翻倍） |
| 集合覆盖候选 5~10 个 | 精确枚举幂集(2ⁿ)仍可行 |
| 候选集合数量增大 | 换O(n²)贪婪近似 |
| 快排基准选择 | 必须随机，不可固定选首/尾元素 |
| 递归基线条件 | 空数组/单元素数组，先写此项 |

## 5. 一眼识别信号（Fast Tells）

- "负权/折扣券倒找钱" → 贝尔曼-福特，非Dijkstra
- "从XX中挑集合覆盖所有YY" → 集合覆盖，候选多→贪婪近似
- "两串连续重合" vs "可跳跃对齐" → 子串(网格最大值) vs 子序列(网格右下角)
- "打分范围不一致(1-5 vs 1-100)" → 换余弦相似度
- "看似要试遍所有排列/子集" → NP完全信号，别死磕精确解
- "固定选首/尾元素当基准" → 快排最坏情况隐患
