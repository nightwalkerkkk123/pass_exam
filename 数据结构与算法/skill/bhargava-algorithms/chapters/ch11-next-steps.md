# 第11章：接下来如何做

## 核心思想 (Core Idea)
本书没有把算法世界讲完，只是打了个地基；这一章用"是什么 + 何时用"的速览方式，把书中未展开的10个真实算法领域指个路，鼓励读者按兴趣挑一个深挖下去。

## 引入的框架与算法 (Frameworks Introduced)

- **二叉查找树（Binary Search Tree, BST）**：当数据需要频繁插入又要保持有序可查找时使用（例如用户名注册系统）。原理：对每个节点，左子树全部更小、右子树全部更大，查找时不断与当前节点比较、决定往左还是往右，逼近目标。平衡时查找平均 O(log n)，但插入、删除比有序数组快得多，代价是不支持"取第 k 个元素"这种随机访问。
- **红黑树 / B树 / 堆 / 伸展树**：都是 BST 的变种或扩展，用于保证树不退化成链表（如红黑树自平衡）或适配数据库存储（B树）。书中只点名，不展开。
- **反向索引（Inverted Index）**：搜索引擎的核心结构。用于根据关键词快速定位包含它的网页。原理：建一个散列表，键是单词，值是包含该单词的页面集合；查询时直接查表取交集/并集，无需扫描全部网页。
- **傅里叶变换（Fourier Transform）**：当需要把一段信号（声音、图像、地震波、DNA序列）分解成不同频率成分时使用。原理：输入一段混合信号，输出它由哪些频率的分量构成，从而可以针对性地保留或删除某些频率（如 MP3 删除人耳不敏感的音符、JPG 压缩图像、Shazam 识别歌曲）。
- **并行算法（Parallel Algorithm）**：当单核速度提升到瓶颈、需要利用多核硬件时使用。原理：把任务拆给多个核心同时执行，比如并行快速排序理论上可把排序降到 O(n)。但存在两个現实阻力：管理开销（拆分和合并任务本身耗时）与负载均衡（任务难度不均导致有的核心空闲）。
- **MapReduce**：当数据量大到单机数据库处理不了（数十亿行）时使用的分布式算法，代表工具是 Hadoop。原理分两步：map 函数把同一操作施加到数组每个元素上（可分发到上百台机器并行做，如批量下载URL）；reduce 函数把一个数组归并成一个值（如求和）。二者组合让海量数据的查询从数小时降到几分钟。
- **布隆过滤器（Bloom Filter）**：当需要判断"某元素是否在一个超大集合里"、且能容忍偶尔出错、但必须省内存时使用（如 Reddit 判重、Google 判断网页是否已抓取、bit.ly 判断恶意网址）。原理：用远小于散列表的存储空间给出一个概率性答案——可能"误报"（说存在但实际不存在），但绝不会"漏报"（说不存在就一定不存在）。
- **HyperLogLog**：与布隆过滤器同类的概率型算法，用于近似计算一个集合中不同元素的数量（如统计某天有多少种不同的搜索词），只求"八九不离十"而不要求精确，换取远小于精确计数所需的内存。
- **SHA 算法（Secure Hash Algorithm）**：当需要比较超大文件是否相同、或验证密码而不存储明文时使用。原理：对字符串生成一段散列值，任何不同输入生成不同散列值，且是单向的——能从原字符串算出散列值，但无法从散列值反推原字符串。当前应优先用 SHA-2、SHA-3（SHA-0、SHA-1已发现缺陷），密码存储更推荐 bcrypt。
- **Simhash（局部敏感散列）**：当需要判断两段内容"有多相似"而不是"是否完全相同"时使用（如查重、判断论文抄袭、检测上传内容是否侵权）。原理：与 SHA 追求"改一个字符结果面目全非"相反，Simhash 让相似输入产生相似的散列值，通过比较散列值的接近程度衡量内容相似度。
- **Diffie-Hellman 密钥交换**：当双方需要在不安全信道（如互联网）上协商加密而无需提前当面约定密钥时使用。原理：使用一对公钥、私钥——公钥可公开发布，任何人用它加密消息；只有持有对应私钥的人才能解密，从根本上解决了"如何安全共享加密算法"的老问题。RSA 是其广泛使用的替代/延伸。
- **线性规划（Linear Programming）**：当需要在若干资源约束条件下让某个目标（利润、支持票数等）最大化时使用。原理：把目标和约束都写成线性不等式/等式，用 Simplex 算法求解最优解；图算法（最短路、最大流等）都可以看作线性规划的特例，线性规划是更宽泛的最优化框架。

## 关键概念 (Key Concepts)
- **概率型数据结构**：像布隆过滤器、HyperLogLog 这样，用牺牲绝对精确性换取巨大空间/时间节省的数据结构。
- **误报 vs 漏报**：布隆过滤器只可能误报（说有其实没有），绝不会漏报（说没有就真没有），这个非对称性是它可用的关键。
- **单向散列（One-way Hash）**：SHA 这类函数可以正向算出散列值，但无法通过散列值反推原始输入，这是密码安全存储的基础。
- **局部敏感 vs 局部不敏感**：SHA 追求输入微小变化导致散列值剧变（防破解）；Simhash 反过来追求输入微小变化只带来散列值微小变化（便于比较相似度）。
- **map 与 reduce**：map 是"一对一"变换（数组到数组，每个元素独立处理，可并行），reduce 是"多对一"聚合（数组到单个值）。
- **并行开销与负载均衡**：多核不会带来线性加速，一是因为任务拆分/结果合并本身耗时，二是因为任务难度不均会导致部分核心空闲等待。
- **公钥与私钥**：Diffie-Hellman/RSA 类非对称加密的核心机制，公钥公开用于加密，私钥保密用于解密。

## 思维模型 (Mental Models)
- 当"数据要不断插入又要保持有序可查"时，用二叉查找树而不是有序数组（数组插入需要重新排序，代价太高）。
- 当"数据量大到单机/单一算法处理不动"时，把并行算法、MapReduce、分布式系统当作横向扩展（scale out）的思路来理解，而不是继续在单机上抠常数优化。
- 当"只需要近似正确的答案、但要求极致省空间"时，把布隆过滤器、HyperLogLog 当作用"可控的不确定性"换取效率的工具。
- 把 SHA 和 Simhash 当作一对相反的散列设计目标来记：一个追求"差之毫厘、失之千里"用于安全校验，一个追求"差之毫厘、也只差毫厘"用于相似度比较。

## 常见陷阱 (Anti-patterns)
- 误以为多加一倍核心数就能获得双倍速度——书中明确指出并行管理开销和负载不均会侵蚀理论加速比，速度提升从不是线性的。
- 误以为布隆过滤器和 HyperLogLog 的用法可以互换——前者判断"元素是否存在"，后者估算"不同元素的数量"，两者解决的问题不同。
- 用普通散列表存储密码明文对应关系而非只存 SHA 散列值，一旦数据库泄露密码就直接暴露；正确做法是只比较散列值。
- 继续使用 SHA-0/SHA-1 计算密码散列——书中提到这两代已被发现存在缺陷，应使用 SHA-2/SHA-3 或更专门的 bcrypt。

## 代码示例 (Code Examples)
```python
>>> arr1 = [1, 2, 3, 4, 5]
>>> arr2 = map(lambda x: 2 * x, arr1)
[2, 4, 6, 8, 10]
```
这段代码演示 MapReduce 中的 map：对数组每个元素独立套用同一个函数（这里是翻倍），得到一个等长的新数组；这种"逐元素独立处理"的特性正是它能被自动分发到多台机器并行执行的原因。

```python
>>> arr1 = [1, 2, 3, 4, 5]
>>> reduce(lambda x,y: x+y, arr1)
15
```
这段代码演示 reduce：把整个数组归并成一个单一的值（这里是求和），与 map 的"一对一"相对，reduce 是"多对一"。

## 参考表格 (Reference Tables)
| 数据结构/算法 | 解决的问题 | 关键代价/限制 |
|---|---|---|
| 有序数组 | 查找快（O(log n)，二分查找） | 插入慢，需要重新排序 |
| 二叉查找树（平衡时） | 查找、插入、删除都约 O(log n) | 不支持随机访问；不平衡时退化到 O(n) |
| 散列表 | 精确判断元素是否存在，O(1) | 需要存储全部键，占用空间大 |
| 布隆过滤器 | 近似判断元素是否存在，省空间 | 可能误报，但绝不漏报 |
| HyperLogLog | 近似统计不同元素个数，省空间 | 结果是近似值，非精确计数 |
| SHA | 比较文件/密码是否相同，且单向不可逆 | 局部不敏感，微小输入差异导致散列值巨变 |
| Simhash | 比较两段内容相似度 | 局部敏感，专为"相似"而非"相同"设计 |

## 讲解示例 (Worked Example)
以 Diffie-Hellman 想解决的问题为例：假设用最原始的替换加密——a→1，b→2，以此类推——给朋友发消息"4,15,7"，对方按同一映射表还原出"d,o,g"。这个方案的致命伤是：加密算法本身（也就是那张映射表）必须提前双方都知道，而协商这张表的过程（邮件、见面）一旦被截获，消息就形同明文；即使算法定期更换，破解者只需暴力尝试少数几种映射（比如把 a=1 换成 a=2）就能撞出正确解码，例如面对密文"9,6,13,13,16 24,16,19,13,5"，用 a=1 得到乱码，换成 a=2 立刻还原成合理单词。Diffie-Hellman 用公钥/私钥这对不对称密钥绕开了这个死结：公钥可以随意公开（贴网站、发邮件都行），任何人都能用它加密消息，但只有持有对应私钥的一方才能解密——双方从此不再需要面对面协商同一套密码规则，也让暴力破解变得极端困难。

## 学习要点 (Key Takeaways)
- 二叉查找树用插入/删除的速度优势换取"不能随机访问"和"可能失衡退化为 O(n)"的代价，平衡时查找为 O(log n)。
- 反向索引是搜索引擎的核心结构：单词→页面集合的散列表。
- 傅里叶变换的本质是把信号分解为频率分量，MP3、JPG、Shazam、地震预测、DNA分析背后都是它。
- 并行算法不是线性加速：管理开销（拆分/合并）与负载均衡（任务难度不均）是两大现实阻力。
- MapReduce = map（逐元素并行变换）+ reduce（归并为单值），是处理数十亿行级数据的分布式方案，代表工具 Hadoop。
- 布隆过滤器"只可能误报、绝不漏报"，HyperLogLog"近似计数"，二者都是用可控的不确定性换取海量数据下的空间效率。
- SHA 是单向且局部不敏感的散列（用于密码/文件校验，优先选 SHA-2/SHA-3 或 bcrypt）；Simhash 反过来是局部敏感的散列（用于相似度检测，如查重、反抄袭）。
- Diffie-Hellman/RSA 用公钥加密、私钥解密解决了"无需事先协商密钥"的加密难题；线性规划是比图算法更宽泛的最优化框架，图问题只是它的一个子集。
