# 第10章：K最近邻算法

## 核心思想 (Core Idea)
判断一个新样本该归为哪一类、或该赋予它什么数值，最简单有效的办法之一是"看看离它最近的邻居们是什么样子"——这就是K最近邻算法（K-Nearest Neighbors, KNN），也是本书从经典算法迈入机器学习领域的第一步。

## 引入的框架与算法 (Frameworks Introduced)

**K最近邻算法（K-Nearest Neighbors, KNN）**
- 何时用：需要对物品分类（如判断水果是橙子还是柚子、识别数字图像）或预测一个数值（如预测用户对电影的评分、预测面包店当天销量）时，尤其是"物以类聚"直觉成立的场景。
- 如何工作：
  1. **特征抽取**：把每个待比较的物品（水果、用户、数字图像……）转换成一组可比较的数字，即坐标。
  2. **计算距离**：用距离公式（书中给出的是毕达哥拉斯/欧几里得距离公式）计算新样本与所有已知样本之间的距离，数字越少表示越像。
  3. **挑出K个邻居**：找出距离最近的K个已知样本。K不是固定为5，可以是2、10，甚至10000——这正是算法名为"K"最近邻而非"5最近邻"的原因。
  4. **给出结论**：
     - 做**分类（classification）**时，看这K个邻居里哪个类别的数量最多，新样本就归为哪一类；
     - 做**回归（regression）**时，把这K个邻居的数值求平均，作为预测结果。

**朴素贝叶斯分类器（Naive Bayes classifier）**
- 何时用：应用领域与KNN相似，书中以垃圾邮件过滤为例——判断一封邮件是否为垃圾邮件。
- 如何工作：统计邮件正文中每个单词在历史垃圾邮件中出现的概率，综合这些概率计算出该邮件是垃圾邮件的整体概率。它基于概率而非距离，是与KNN思路不同的另一种简单分类算法。

**余弦相似度（cosine similarity）**——书中仅点到为止，未展开算法细节
- 何时用：当参与比较的对象存在"打分尺度不一致"的问题时（例如同样喜欢一部电影，有人打5星、有人打4星，但两人品味其实很接近）。
- 核心区别：欧氏距离公式比较的是两点之间的直线距离，余弦相似度比较的是两个向量的**夹角**，因此对"整体偏好方向是否一致"更敏感，不会因为打分尺度不同而误判两人不相邻。

## 关键概念 (Key Concepts)
- **K最近邻算法（KNN）**：通过比较新样本与已知样本的距离、参考最近的K个邻居来完成分类或回归的算法。
- **特征抽取（feature extraction）**：把物品转换为一组可比较数字（坐标）的过程，是使用KNN的前提。
- **距离公式（欧几里得距离）**：形如 sqrt(Σ(xi−yi)²) 的公式，即使涉及很多维度（如5个特征、5维空间）依然适用，用来量化两组数字的相似程度。
- **分类（classification）**：把样本分到已有的类别中，即"编组"。
- **回归（regression）**：预测一个数值型结果，例如评分或销量。
- **机器学习（machine learning）**：让计算机通过数据自动变得更"聪明"的技术领域，KNN是入门这一领域的领路算法。
- **训练（training）**：机器学习算法在正式使用前，先浏览大量已知数据、提取特征的过程；OCR和朴素贝叶斯分类器都包含这一步。
- **OCR（光学字符识别，optical character recognition）**：让计算机从图片中自动识别文字/数字的技术，其核心思路（提取特征、找最近邻居）与判断水果的例子一致，只是特征（线段、点、曲线）复杂得多。

## 思维模型 (Mental Models)
- 当需要判断"这个新东西该归到哪一类"或"该给它一个什么数值"时，用KNN：先把它翻译成数字特征，再算距离，最后看邻居中的多数类别或数值平均。
- 把**特征抽取**当作"翻译"来理解：水果、用户喜好、数字图像，这些原本无法直接比较的东西，只有先翻译成同一套坐标系里的数字，才谈得上"计算相似度"。
- 当K的取值发生变化时，把K当作"你愿意听取多少个邻居意见"的旋钮：K太小容易被个别异常邻居带偏，K太大又会把不相关的邻居也纳入平均，选多大合适没有统一答案，取决于数据规模（练习中提到：用户数以百万计时，K=5可能就显得太少）。
- 当距离公式给出的"相似"结果与直觉不符时（比如打分尺度不同导致品味接近的人被判定为不相邻），要意识到欧氏距离不是唯一选择，换成比较方向而非距离的余弦相似度可能更贴近真实相似度。

## 常见陷阱 (Anti-patterns)
- **特征选择不当**：如果选来比较的特征与目标不相关（用对小猫图片的喜好来判断电影推荐是否合拍）或过于片面（只让用户给同一系列的三部电影打分，无法体现其真实的电影品味），距离算出来的"相似"没有意义。挑选特征的两个原则是：与目标紧密相关、且不偏不倚（不能只覆盖某一类电影/情况）。
- **忽视评分尺度差异**：不同人打分的宽严标准不同（有人喜欢的都打5分，有人只有特别喜欢才打5分），若直接套用距离公式，品味实际相同的两人可能被判定为不相邻，这是练习10.1提出的问题，解决方向是加权或改用余弦相似度。
- **认为所有邻居的意见同等重要**：现实中某些"意见领袖"的评分应该占更大权重（练习10.2），单纯的距离最近K个邻居未必是最优方案，需要按场景调整算法。
- **迷信历史数据能预测一切**：股票市场这类受变量太多、近乎随机的现象，即便应用机器学习和精心挑选特征，也可能无法找到可靠的预测方法——机器学习不是万能药。

## 讲解示例 (Worked Example)

**示例一：橙子还是柚子（分类的直觉起点）**
把水果按"个头"和"颜色"画在一张图上：一般规律是柚子比橙子个头更大、颜色更红。碰到一个特征模糊的水果时，办法不是死记规则，而是看它最近的3个邻居——如果这3个邻居里橙子比柚子多，就把它判定为橙子。这正是最朴素的KNN分类（K=3）。

**示例二：Netflix电影推荐（特征抽取 + 距离计算）**
把每个用户对5类电影的喜好程度记为5个数字（一个5维坐标）。Priyanka和Justin都喜欢爱情片、讨厌恐怖片，两人的距离算出来是1；Morpheus喜欢动作片、讨厌爱情片，与Priyanka的距离算出来是24。距离越小代表品味越接近，因此判定Justin是Priyanka更合适的"邻居"——只要是Justin喜欢的电影，就可以推荐给Priyanka。

**示例三：预测电影评分（回归）**
要预测Priyanka会给电影《Pitch Perfect》打多少分，先找出离她最近的5个人（Justin、JC、Joey、Lance、Chris），把这5个人对该片的评分取平均，得到4.2分，这就是回归给出的预测值。

**示例四：面包店销量预测（回归 + K=4）**
面包店用"天气指数（1~5）""是否周末/节假日（0或1）""是否有活动（0或1）"三个特征描述"每一天"，并保存了历史销量数据。对于"今天"（周末、天气不错），计算它与历史各天的距离，找出最近的4个邻居A、B、D、E，把这4天的销量取平均，得到218.75，这就是今天预计要烤的面包数量（K=4的回归）。

## 学习要点 (Key Takeaways)
- KNN有两大用途：**分类**（把样本编组）和**回归**（预测一个数值），二者的区别仅在于最后一步是"取多数类别"还是"取平均值"。
- KNN效果的上限取决于**特征抽取**质量：特征必须与目标紧密相关、且覆盖全面不偏倚，否则距离计算再精确也没有意义。
- 距离公式（欧氏距离）在特征维度增加时依然适用（书中从2维水果特征扩展到5维用户特征，公式不变），但当评分尺度不一致时会失真，此时应考虑余弦相似度。
- K是一个需要根据数据规模权衡的参数，没有固定公式；数据量越大，可能需要相应调大K，反之亦然。
- KNN是很多更复杂机器学习应用（OCR、语音识别、人脸识别）的思想基石：本质仍是"提取特征→找最近邻居"，只是特征提取的复杂度大幅提升。
- 朴素贝叶斯分类器是与KNN应用领域相近但原理不同（基于概率而非距离）的分类算法，典型场景是垃圾邮件过滤。
- 机器学习并非万能：变量繁多、近乎随机的问题（如股票市场涨跌）即使套用KNN等算法也未必能得到可靠预测，选择合适的应用场景同样重要。
