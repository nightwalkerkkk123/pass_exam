# 第4章：快速排序

## 核心思想 (Core Idea)
分而治之（D&C）是一种通用的解题思路——先找最简单的基线条件，再想办法把问题不断缩小规模直到符合基线条件；快速排序（Quicksort）正是这种思路在排序问题上的典范应用，其平均性能优异但依赖基准值的选择。

## 引入的框架与算法 (Frameworks Introduced)

**分而治之（Divide and Conquer，D&C）**
- 何时用：面对一个不知如何下手的问题（递归、分割类问题）时，作为通用解题策略。
- 工作原理：
  1. 找出基线条件（baseline case）——最简单、无需再分解的情形。
  2. 不断将原问题分解或缩小规模，直到符合基线条件为止。
- 书中用"均匀分割土地为最大方块"的例子说明：先看两边长度是否成整数倍（基线条件），若不是，则划出当前能容纳的最大方块，把剩下的小块地当作新的子问题递归处理（本质上是欧几里得算法求最大公约数）。

**快速排序（Quicksort）**
- 何时用：通用场景下的高效排序，C 语言标准库的 `qsort` 即用此算法实现。
- 工作原理：
  1. 基线条件：数组为空或只有一个元素，直接返回，视为"已排序"。
  2. 从数组中选取一个元素作为基准值（pivot），初学时可简单选第一个元素。
  3. 分区（partitioning）：将其余元素分为两个子数组——小于等于基准值的 `less` 和大于基准值的 `greater`。
  4. 对 `less` 和 `greater` 递归调用快速排序。
  5. 合并结果：`quicksort(less) + [pivot] + quicksort(greater)`，得到有序数组。

## 关键概念 (Key Concepts)
- **基线条件（base case）**：递归函数中最简单、不再需要递归的终止情形；处理数组的递归函数中，基线条件通常是空数组或只含一个元素的数组。
- **递归条件（recursive case）**：问题尚未达到基线条件时，用于缩小问题规模并调用自身的分支逻辑。
- **基准值（pivot）**：快速排序中从数组里选出用来分区的参照元素，其余元素据此被分为两组。
- **分区（partitioning）**：把数组按"是否小于/大于基准值"分成两个子数组的操作，是快速排序每一层递归的核心工作。
- **归纳证明（inductive proof）**：一种证明算法对任意规模输入都成立的方法，同样分基线条件和归纳（递归）条件两步，例如"若快排对 n 个元素管用，则对 n+1 个元素也管用"。
- **平均情况 / 最糟情况（average case / worst case）**：算法性能会因输入数据和参数选择（如基准值）不同而波动，需分别评估；快速排序的最糟情况是 O(n²)，平均情况是 O(n log n)。
- **大 O 表示法中的常量**：大 O 只描述增长趋势，会忽略执行每步操作所需的固定时间（常量 c）；当两个算法的大 O 相同时（如快排与合并排序都是 O(n log n)），常量差异会真正影响实际速度。

## 思维模型 (Mental Models)
- 当面对一个"不知道如何下手"的递归型问题时，用 D&C：先问自己"最简单、根本不用处理的情形是什么"，再问"如何把当前问题变成一个规模更小、但结构相同的子问题"。
- 把递归函数的正确性证明当作爬梯子来理解：只要证明"站在第一级"（基线条件）和"能从任意一级爬到下一级"（归纳条件），就能证明能爬到任意高度——这就是归纳证明。
- 当比较两个大 O 相同的算法时，别忘了看常量：大 O 只是"增长趋势"的抽象，常量小的算法在同阶复杂度下往往更快（快排常量小于合并排序，所以更受欢迎）。
- 把快速排序的调用栈高度当作"运气"的体现：基准值选得好（接近中位数）→ 每次均分数组 → 栈高 O(log n) → 整体 O(n log n)；基准值选得差（如已排序数组每次选第一个元素）→ 一边子数组总为空 → 栈高 O(n) → 整体退化为 O(n²)。

## 常见陷阱 (Anti-patterns)
- 固定选择数组第一个元素作为基准值：如果输入数组本身已经有序（或接近有序），每次分区都会得到一个空的子数组和一个几乎不变规模的子数组，调用栈深度退化为 O(n)，导致最糟情况 O(n²)。应随机选择基准值以使平均情况更可靠地体现为 O(n log n)。
- 忽略大 O 表示法中的常量差异：认为两个算法只要大 O 阶数相同就"速度一样"，从而在快速排序和合并排序之间随意选择——实际上快速排序的常量更小，通常更快。
- 编写递归函数时忘记设置基线条件，或基线条件设置得不是"最简单"的情形（例如数组为空/单元素），容易导致无限递归或逻辑复杂化。

## 代码示例 (Code Examples)
```python
def quicksort(array):
  if len(array) < 2:
    return array
  else:
    pivot = array[0]
    less = [i for i in array[1:] if i <= pivot]
    greater = [i for i in array[1:] if i > pivot]
    return quicksort(less) + [pivot] + quicksort(greater)
print quicksort([10, 5, 2, 3])
```
这是快速排序的完整实现：基线条件（长度小于 2 直接返回）、选取第一个元素为基准值、用列表推导式完成分区，再递归合并 `less + [pivot] + greater`，体现了 D&C 三步走（基线条件→分解→合并）的完整结构。

## 讲解示例 (Worked Example)
以数组 `[33, 15, 10]` 为例手动追踪快速排序：
1. 选基准值 `33`（数组第一个元素）。
2. 分区：`less = [15, 10]`（均 ≤ 33），`greater = []`（无元素 > 33）。
3. 对 `less = [15, 10]` 递归：选基准值 `15`，分区得 `less2 = [10]`，`greater2 = []`；两者都已是基线条件（长度 ≤ 1），直接返回。合并：`quicksort([10]) + [15] + quicksort([]) = [10] + [15] + [] = [10, 15]`。
4. 对 `greater = []` 递归：长度为 0，触发基线条件，直接返回 `[]`。
5. 顶层合并：`quicksort([15, 10]) + [33] + quicksort([]) = [10, 15] + [33] + [] = [10, 15, 33]`。

再对比调用栈高度的两种极端情形（书中用 8 个元素的数组说明）：若每次都选到能把数组均分成两半的基准值，调用栈只需约 `log₂8 = 3` 层，每层处理量之和为 O(n)，总时间 O(n) × O(log n) = O(n log n)（最佳/平均情况）；若数组已排序且总选第一个元素作基准值，每次分区都产生一个空子数组，调用栈深度退化为 O(n) 层，总时间 O(n) × O(n) = O(n²)（最糟情况）。

## 学习要点 (Key Takeaways)
- D&C 两步法：先定基线条件，再想办法缩小问题规模；处理数组的递归函数，基线条件通常是空数组或单元素数组。
- 快速排序三步：选基准值 → 分区（小于/大于基准值两个子数组）→ 对子数组递归排序后与基准值合并。
- 复杂度结论：平均情况 O(n log n)，最糟情况 O(n²)；性能高度依赖基准值选择，随机选择基准值可使平均情况更可靠。
- 大 O 相同不等于速度相同：快速排序因常量比合并排序小，实践中通常更快，这是"常量有时候事关重大"的典型例子。
- 调用栈高度即递归的"层数"：基准值越接近中位数，栈越矮（O(log n)）；越接近极值，栈越高（最糟 O(n)）。
- 归纳证明（基线条件 + 归纳条件）是理解和证明递归算法（包括快速排序）对任意规模输入都成立的思维工具。
