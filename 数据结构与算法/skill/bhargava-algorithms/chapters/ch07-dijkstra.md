# 第7章：狄克斯特拉算法

## 核心思想 (Core Idea)
当图的每条边都带有权重（比如时间、金钱）时，广度优先搜索找到的"段数最少"路径未必是"代价最小"路径；狄克斯特拉算法（Dijkstra's algorithm）通过贪心地反复选择"当前最便宜的未处理节点"并更新其邻居开销，求出加权图中从起点到各节点的总权重最小路径——但前提是所有边的权重必须为正。

## 引入的框架与算法 (Frameworks Introduced)

**狄克斯特拉算法（Dijkstra's algorithm）**
- 适用场景：图是**加权图**（weighted graph，边带有数字权重），且不含负权边；本质上要求图是有向无环图（directed acyclic graph，DAG），无向图因为每条边都等价于一个环需格外小心。
- 求解目标：从起点到图中每个节点（含终点）的总权重最小路径。
- 执行步骤：
  1. 在所有**未处理**节点中，找出开销（从起点到它的当前已知最小代价）最低的节点。
  2. 检查该节点的每个邻居：如果"经该节点前往邻居"的开销比邻居当前记录的开销更低，就更新邻居的开销，并把该节点记为邻居的父节点。
  3. 将该节点标记为已处理；重复步骤1、2，直到图中所有节点都处理过。
  4. 从终点开始，沿父节点链条反向回溯，即可还原出最终的最短路径。

**贝尔曼-福特算法（Bellman-Ford algorithm）**（本章仅提及，不展开实现）
- 适用场景：图中存在负权边，此时狄克斯特拉算法会给出错误答案。
- 原理：本章未详细介绍，只指出它是处理负权图最短路径问题的替代算法。

## 关键概念 (Key Concepts)
- **权重（weight）**：加权图中每条边关联的数字，代表通过这条边需要付出的代价（时间、金钱等）。
- **加权图（weighted graph）/ 非加权图（unweighted graph）**：边带权重的图用狄克斯特拉算法求最短路径，不带权重的图用广度优先搜索。
- **开销（cost）**：算法运行过程中，从起点到某节点当前已知的最小总权重，初始时未知节点设为无穷大（`float("inf")`）。
- **父节点（parent）**：记录到达某节点的最优路径上紧邻的前一个节点，用于算法结束后回溯出完整路径。
- **已处理节点（processed）**：一旦某节点被处理，就意味着算法确信不存在到它的更便宜路径，之后不再重新考虑它——这是贪心策略的核心假设，也是负权边会破坏该假设的原因。
- **负权边（negative-weight edge）**：权重为负的边，一旦存在，"处理过的节点不可能再变便宜"这一假设就会失效，导致狄克斯特拉算法给出错误结果。
- **环（cycle）**：从某节点出发经过若干边又回到该节点；只要环的权重之和为正，绕环走就必然增加总开销，因此最短路径不会包含这样的环。

## 思维模型 (Mental Models)
- 当图没有权重（或只关心边数）时，用广度优先搜索找段数最少的路径；当图有权重时，用狄克斯特拉算法找总权重最小的路径。
- 把"找出最便宜的未处理节点"当作一条不断向外扩张、确定下来就不再收缩的边界来理解：节点一旦被纳入边界（被处理），就代表"没有更便宜的路径能到它了"。
- 把负权边当作专门戳破这个边界假设的陷阱来理解——它可能在"边界已经画定之后"制造出一条更便宜的路径，而算法却不会回头检查。
- 把 `costs` 表和 `parents` 表当作算法运行时的两份"状态快照"：`costs` 只回答"目前最便宜是多少"，`parents` 才负责回答"具体怎么走"，最终路径要靠回溯父节点链条才能拼出来，而不是在正向计算过程中直接得到。

## 常见陷阱 (Anti-patterns)
- **对含负权边的图使用狄克斯特拉算法**：书中"换黑胶唱片"的例子表明，一旦某节点（如海报）被标记为已处理，算法就不再重新考虑它，但负权边完全可能在后面制造出一条更便宜的路径抵达它，导致最终选出的不是真正最优路径。正确做法是改用贝尔曼-福特算法。
- **把"最短路径"狭义地理解成物理距离最短**：本章的换钢琴例子说明，"最短路径"泛指任何要最小化的度量指标（如额外支付的费用），不要被字面意思限制住。

## 代码示例 (Code Examples)
```python
graph = {}
graph["start"] = {"a": 6, "b": 2}
graph["a"] = {"fin": 1}
graph["b"] = {"a": 3, "fin": 5}
graph["fin"] = {}

infinity = float("inf")
costs = {"a": 6, "b": 2, "fin": infinity}
parents = {"a": "start", "b": "start", "fin": None}
processed = []

def find_lowest_cost_node(costs):
    lowest_cost = float("inf")
    lowest_cost_node = None
    for node in costs:
        cost = costs[node]
        if cost < lowest_cost and node not in processed:
            lowest_cost = cost
            lowest_cost_node = node
    return lowest_cost_node

node = find_lowest_cost_node(costs)
while node is not None:
    cost = costs[node]
    neighbors = graph[node]
    for n in neighbors.keys():
        new_cost = cost + neighbors[n]
        if costs[n] > new_cost:
            costs[n] = new_cost
            parents[n] = node
    processed.append(node)
    node = find_lowest_cost_node(costs)
```
这段代码用三个散列表（`graph` 存邻居及边权重、`costs` 存当前最小开销、`parents` 存最优路径上的前驱节点）加一个 `processed` 列表实现狄克斯特拉算法完整流程：`find_lowest_cost_node` 每次线性扫描未处理节点找出开销最低者，外层 `while` 循环则不断"取最便宜节点→更新邻居开销→标记已处理"，直至没有可处理的节点为止。

## 参考表格 (Reference Tables)
| 算法 | 适用图类型 | 求解目标 | 权重限制 |
|---|---|---|---|
| 广度优先搜索（第6章） | 非加权图 | 边数最少的路径 | 无（不区分权重） |
| 狄克斯特拉算法 | 加权图（近似DAG） | 总权重最小的路径 | 仅限正权重 |
| 贝尔曼-福特算法（本章仅提及） | 含负权边的加权图 | 总权重最小的路径 | 允许负权重 |

## 讲解示例 (Worked Example)
以本章反复使用的示例图为例：起点 `start` 到节点 A 需要6分钟、到节点 B 需要2分钟；B 到 A 需要3分钟、B 到终点 `fin` 需要5分钟；A 到 `fin` 需要1分钟。目标：求 `start` 到 `fin` 耗时最短的路径。

初始状态：`costs = {A:6, B:2, fin:∞}`，`parents = {A:start, B:start, fin:None}`，`processed = []`。

| 步骤 | 处理的节点 | 更新后 costs[A] | 更新后 costs[B] | 更新后 costs[fin] | 说明 |
|---|---|---|---|---|---|
| 0（初始） | — | 6 | 2 | ∞ | 只知道起点直接邻居的开销 |
| 1 | B（最便宜，2分钟） | 5（经B：2+3<6，父节点改为B） | 2 | 7（经B：2+5<∞，父节点设为B） | B的两个邻居都被刷新 |
| 2 | A（次便宜，5分钟） | 5 | 2 | 6（经A：5+1<7，父节点改为A） | 找到更短的到终点路径 |
| 3 | fin（最便宜，6分钟，无邻居） | 5 | 2 | 6 | 所有节点处理完毕，算法结束 |

最终从 `parents` 表反向回溯：`fin` 的父节点是 A，A 的父节点是 B，B 的父节点是 `start`。于是最短路径为 `start → B → A → fin`，总耗时6分钟——比广度优先搜索得到的"段数最少但耗时7分钟"的路径更优，恰好印证了"段数少不等于总权重小"这一核心区别。

（书中另有一个更贴近现实决策的例子：Rama 想用一本乐谱换到一架钢琴，图上的节点是各人愿意交换的物品，边的权重是交换时需要额外支付的费用。用狄克斯特拉算法沿"乐谱→黑胶唱片→架子鼓→钢琴"这条路径，Rama 只需额外支付35美元，比经吉他中转更省钱——这说明"最短路径"完全可以指代任意要最小化的指标，而不局限于时间或距离。）

## 学习要点 (Key Takeaways)
- 狄克斯特拉算法四步骤必须记牢：①找最便宜的未处理节点 ②更新其邻居开销（并同步更新父节点） ③重复直到所有节点处理完 ④沿父节点回溯得到最终路径。
- 判断该用广度优先搜索还是狄克斯特拉算法的标准很简单：图不带权重（或只关心边数）用前者；图带权重、求总代价最小用后者。
- 狄克斯特拉算法**仅在所有边权重为正时**才正确；一旦出现负权边，必须改用贝尔曼-福特算法，否则可能选出错误（更长/更贵）的路径。
- 实现依赖三个散列表（`graph`、`costs`、`parents`）加一个 `processed` 列表；`find_lowest_cost_node` 每轮对未处理节点做线性扫描，是最朴素的实现方式（面试/考试中可提及用优先队列/堆可优化这一步）。
- "最短路径"是广义概念，可以指时间、金钱或任何需要最小化的度量指标，不要局限于物理距离。
- 图中只要存在正权重的环，绕环走一定会增加总开销，因此最短路径不会主动绕环。
