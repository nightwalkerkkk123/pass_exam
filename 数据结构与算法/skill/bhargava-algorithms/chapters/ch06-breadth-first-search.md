# 第6章：广度优先搜索

## 核心思想 (Core Idea)
广度优先搜索（Breadth-First Search，BFS）是一种按"距离起点由近及远"逐层扩展的图搜索算法，只要严格按加入顺序（先入先出）检查节点，就能同时回答"A到B有没有路径"和"A到B的最短路径是几步"这两类问题。

## 引入的框架与算法 (Frameworks Introduced)

**广度优先搜索（Breadth-First Search，BFS）**
- 适用场景：图（graph）上的最短路径问题（shortest-path problem），尤其是"边的权重都相同/不计权重、只关心步数"的场景，例如社交网络中找最近的目标人物、地图上找换乘最少的公交路线。
- 工作步骤：
  1. 创建一个队列（queue），把起点的所有邻居（neighbor）入队。
  2. 只要队列不为空，就取出队首元素（出队）。
  3. 判断该元素是否满足目标条件；若满足，返回结果。
  4. 若不满足，把它的所有邻居加入队尾，并把当前元素标记为"已检查"。
  5. 重复 2-4，直到找到目标或队列为空（说明不存在路径）。
- 关键约束：必须使用队列而非其他结构来维护"待检查名单"，且每个节点只能检查一次，否则要么得不到最短路径，要么可能陷入无限循环。

**拓扑排序（Topological Sort）**
- 适用场景：任务之间存在依赖关系（例如"刷牙"必须在"吃早餐"之前）的图，需要生成一个满足所有依赖的可行顺序。
- 工作原理：本章仅作概念引入——若任务A依赖任务B，则在结果列表中A必须排在B之后；据此可以从依赖图直接读出一个可行的执行顺序（详细算法未展开，只给出练习与直觉）。

## 关键概念 (Key Concepts)
- **图（Graph）**：由节点（node）和边（edge）组成的结构，用于建模事物之间的连接关系。
- **节点与边（Node & Edge）**：节点是图中的实体（如人、地点），边是连接两个节点的关系。
- **邻居（Neighbor）**：与某节点直接相连的节点；只有一步之遥的节点才算邻居。
- **有向图（Directed Graph）**：边带箭头，关系单向，例如"rama→adit"表示 rama 欠 adit 钱，但反过来不成立。
- **无向图（Undirected Graph）**：边不带箭头，关系双向，例如"ross - rachel"表示两人互相约会。
- **队列（Queue）**：一种先进先出（First In First Out，FIFO）的数据结构，只支持入队和出队两种操作，不能随机访问。
- **栈（Stack）**：一种后进先出（Last In First Out，LIFO）的数据结构，与队列相对。
- **最短路径问题（Shortest-Path Problem）**：寻找从节点A到节点B所需经过边数最少的路径的问题。
- **树（Tree）**：一种特殊的图，其中没有指向"上级"（回指）的边，例如家谱中边只从子节点指向父母。

## 思维模型 (Mental Models)
- 当问题涉及"关系网络中找最近的目标"或"最少换乘/最少步数"时，用广度优先搜索：先建图，再逐层扩散搜索。
- 把队列当作现实中的排队来理解：先来的人先被服务（先入队的节点先被检查），这保证了"先检查一度关系，再检查二度关系"的顺序，从而保证找到的是最短路径。
- 把"已检查名单"当作防止重复劳动和死循环的安全网来理解：一个节点可能通过多条边被多次加入队列（如例子中的 Peggy 是 Alice 和 Bob 的共同好友），但只应检查一次。
- 把有向图和无向图的区别理解为"单行道 vs 双行道"：有向图的邻居关系不对称（Anuj 是 Bob 的邻居，但 Bob 不是 Anuj 的邻居），无向图的邻居关系总是对称的。

## 常见陷阱 (Anti-patterns)
- **不用队列、随意顺序检查**：如果不按加入顺序检查（比如先检查后加入的二度关系 Anuj，再检查先加入的一度关系 Claire），即使两人都符合条件，找到的也不是关系最近（最短路径）的那个，结果不可靠。
- **不标记已检查节点**：如果检查完一个节点后不将其记录为"已检查"，且该节点会重新把邻居加回队列，就可能在两个（或多个）节点间反复相互添加，形成无限循环，程序永不终止。
- **认为图中路径条数最少必然等于所需时间最少**：段数最少的路径只是边数最少，不代表现实中"最快"（下一章会引入加权图来处理这个问题）。

## 代码示例 (Code Examples)
```python
def search(name):
    search_queue = deque()
    search_queue += graph[name]
    searched = []
    while search_queue:
        person = search_queue.popleft()
        if not person in searched:
            if person_is_seller(person):
                print person + " is a mango seller!"
                return True
            else:
                search_queue += graph[person]
                searched.append(person)
    return False
search("you")
```
这段代码是本章广度优先搜索的完整实现：用双端队列（deque）维护待检查名单，用 `searched` 列表记录已检查过的人以避免重复检查和无限循环，逐层扩展直到找到目标或队列耗尽。

## 讲解示例 (Worked Example)
书中用"芒果销售商"场景演示 BFS：假设你想知道自己的 Facebook 好友网络里有没有芒果销售商。

1. 建图：`graph["you"] = ["alice", "bob", "claire"]`，`graph["bob"] = ["anuj", "peggy"]`，`graph["alice"] = ["peggy"]`，`graph["claire"] = ["thom", "jonny"]`，其余人（anuj、peggy、thom、jonny）没有邻居（数组为空）。
2. 初始化搜索队列为你的一度好友：`[alice, bob, claire]`。
3. 依次出队检查：alice 不是销售商（判断规则是"名字以 m 结尾"），把她的邻居 peggy 加入队尾；bob 不是销售商，把 anuj、peggy 加入队尾（此时 peggy 被加入了两次）；claire 不是销售商，把 thom、jonny 加入队尾。
4. 继续处理二度关系：anuj（以 j 结尾，不是）、peggy（第一次出现，检查后标记为已检查，加入 searched）、peggy（第二次出现，但已在 searched 中，直接跳过，不重复检查）、thom（以 m 结尾，是芒果销售商！）——找到后返回 True，搜索结束。
5. 全程严格按队列顺序（先一度关系、后二度关系）检查，如果 thom 和某个一度好友都符合条件，最终返回的一定是关系最近（最先加入队列）的那个，这正体现了 BFS 保证"最短路径"的原理。

## 学习要点 (Key Takeaways)
- BFS 能回答两类问题："A 到 B 是否有路径"和"A 到 B 的最短路径是多少"，二者用同一套算法，只是判断目标条件不同。
- 必须用队列（FIFO）维护待检查节点；用栈（LIFO）无法保证按"距离由近及远"的顺序检查，会破坏最短路径的正确性。
- 检查过的节点要记录（如 `searched` 列表），防止重复检查导致的性能浪费和可能的无限循环。
- Python 中常用散列表（dict）实现图：键是节点，值是该节点的邻居列表；散列表本身无序，键值对添加顺序不影响图的结构和搜索结果。
- BFS 的运行时间是 O(V + E)（V 为顶点/节点数，E 为边数）：每条边都可能被遍历一次（O(边数)），每个节点入队/出队耗时固定（O(1)，共 O(人数)）。
- 有向图与无向图影响"邻居"是否对称，建图时需先明确图的方向性，否则会漏掉或多算边。
- 拓扑排序可用于给有依赖关系的任务生成可行顺序，是图应用的另一分支，本章只做概念引入，未给出具体实现代码。
