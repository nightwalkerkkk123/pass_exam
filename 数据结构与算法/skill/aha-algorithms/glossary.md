# 《啊哈！算法》术语表

按术语拼音首字母排序，括号内为出现的章节。

**边（Edge）** — 连接两个顶点的线，可带权值表示距离或代价。(ch05)

**并查集 (Union-Find)** — 维护森林式分组结构，支持合并集合与查询代表元（根）；Kruskal 用它判断两点是否已连通。(ch07, ch08)

**book 数组（标记数组）** — 用 0/1 数组记录某元素/位置/顶点是否已出现或访问过，把逐一枚举查找的 O(n) 优化为 O(1) 判断，DFS/BFS 中防止重复访问。(ch02, ch03, ch04, ch05)

**单源最短路径** — 指定一个源点到其余所有顶点的最短路径，Dijkstra/Bellman-Ford/SPFA 求解目标。(ch06)

**顶点（Vertex）** — 图中的基本节点，例如一个城市。(ch05)

**堆 (Heap)** — 满足父结点值不大于（最小堆）或不小于（最大堆）子结点值的完全二叉树。(ch07)

**队列 (Queue/FIFO)** — 先进先出的线性结构，只在队首删除、队尾插入，用 head/tail 两下标实现 O(1) 操作，也是 BFS 的核心结构，保证按距离分层扩展。(ch02, ch04, ch05)

**多数元素/主元素问题 (Majority Element Problem)** — 长度为 n 的序列中找出出现次数严格超过 n/2 的元素的问题。(ch09)

**多源最短路径** — 任意两点之间的最短路径，Floyd-Warshall 求解目标。(ch06)

**二分思想** — 快速排序依赖的核心思想，每轮把问题规模大致折半处理。(ch01)

**二分图 (Bipartite Graph)** — 顶点可分成两个集合、每条边两端各属一个集合、集合内部无边的图。(ch08)

**方向数组 (next数组)** — 用二维数组打包上下左右四个方向的坐标偏移，配合循环枚举扩展方向。(ch04)

**负权边** — 权值为负数的边，会破坏 Dijkstra 贪心假设的正确性。(ch06)

**负权回路（负权环）** — 环路总权值为负，使图不存在最短路径（可无限绕环变小）。(ch06)

**割边/桥 (Bridge)** — 删除后使连通图变为不连通的边。(ch08)

**割点/割顶 (Cut Vertex)** — 删除后使连通图变为不连通的顶点。(ch08)

**归位** — 一个数被排序过程放到最终有序序列的正确位置后不再移动。(ch01)

**估计值与确定值** — dis 数组中的路程值先是可能被更新的估计值，一旦证明无法再变小即成为确定值。(ch06)

**head / tail** — 队列的两个下标变量，head 指队首元素，tail 指队尾下一位置，head==tail 时队列为空。(ch02)

**回溯 (Backtrack/Backtracking)** — DFS 从递归深处返回上一层前撤销当前标记，恢复现场使其他分支能继续正确探索。(ch04, ch05)

**降维（消元）优化** — 当变量间存在确定性运算关系（如 C=A+B）时，只枚举独立变量，其余直接算出，降低一个维度复杂度。(ch03)

**剪枝/约束判断** — 判断某扩展方向是否合法（越界、障碍物、已访问），不合法则跳过。(ch04)

**结构体 (struct)** — 把相关变量打包成新类型的 C 语言机制，用 `.` 访问变量成员，`->` 访问指针所指对象成员。(ch02)

**静态链表 / 数组模拟链表** — 不用指针，改用 data[] 存值、right[] 存下一结点下标（0代表NULL）实现链表等价逻辑。(ch02)

**计数抵消 (Cancellation)** — 把一个多数票和一个非多数票视为相互抵消的一对，抵消后多数元素的相对优势不变。(ch09)

**计数数组/类桶排序 (Counting Array)** — 开一个以数值为下标的数组累加出现次数，最大值所在下标即为众数，O(N+M)。(ch09)

**基准数 (pivot)** — 快速排序每轮用来分割序列的参照数，通常取当前区间最左边的元素。(ch01)

**可达性问题（Reachability）** — 候选解在网格/图上是否真的可以从起点到达，纯数值枚举默认不考虑此约束。(ch03)

**空间换时间** — 用远大于待排序数个数的存储空间（桶）换取接近线性时间复杂度的权衡策略。(ch01)

**快速选择 (Quickselect)** — 借用快速排序的一次划分操作，只递归进入包含目标位置的一侧，期望线性时间找出第k大/小元素。(ch09)

**连通性 (Connectivity)** — 网格或图中两点之间是否存在合法路径可相互到达。(ch04)

**邻接表** — 用 first/next 数组模拟链表存图的方式，空间 O(M)，适合稀疏图。(ch06)

**邻接矩阵（Adjacency Matrix）** — 用二维数组 e[i][j] 记录顶点 i 到 j 是否有边或边权，无向图对称。(ch05)

**离散化 (Discretization)** — 把跨度很大、稀疏分布的数值重新映射为连续小整数下标，以便使用下标类算法。(ch09)

**low值 (low-link value)** — 某顶点不经过其 DFS 树父亲、仅靠树边+回边所能回溯到的最早时间戳。(ch08)

**路径压缩 (Path Compression)** — 并查集查找根结点时，将沿途结点父指针直接指向最终根，加速后续查询。(ch07)

**枚举范围上界分析** — 写循环前先从题目约束反推每个变量的最大可能取值，避免无意义地枚举过大范围。(ch03)

**枚举算法（穷举法）** — 把问题所有可能的候选解一一列出并代入验证的算法思想，正确性有保证但效率取决于候选空间大小。(ch03)

**摩尔投票法 (Boyer-Moore Voting Algorithm)** — 用一个候选值和一个计数器，遇同加一、遇异减一，O(N)时间O(1)空间内找出多数元素。(ch09)

**匹配 (Matching)** — 二分图中一组两两不共享端点的边集合。(ch08)

**全排列（Full Permutation）** — n个不同元素的所有可能顺序排列，共 n! 种。(ch03)

**哨兵 i / 哨兵 j** — 快速排序中从序列两端向中间探测的两个下标变量，j 找小于基准数的数，i 找大于基准数的数。(ch01)

**深度 (Depth)** — 从根结点到某结点经过的层数，根结点本身记为第1层。(ch07)

**时间戳** — DFS 遍历中记录每个顶点被访问先后次序的编号。(ch05, ch08)

**时间复杂度的常数忽略** — 比较算法优劣时只看最高阶项，忽略较小常数系数和低阶项。(ch01)

**树 (Tree)** — 任意两个结点间有且仅有一条路径的无向图，等价于不含回路的连通无向图。(ch07)

**松弛（Relax）** — 用一条新路径尝试更新当前已知的最短路程估计值，若更短则替换，是最短路径算法的统一基本操作。(ch06)

**趟 (pass)** — 冒泡排序中完整走一遍相邻比较与交换的过程，每一趟固定归位一个数。(ch01)

**贪心策略** — Dijkstra 每轮选择当前不会再变短的最优点纳入确定集合，依赖边权非负这一前提。(ch06)

**top** — 栈顶下标变量，top==0 表示栈为空。(ch02)

**完全二叉树 (Complete Binary Tree)** — 除最后一层外每层都满，且最后一层结点必须从左向右连续排列的二叉树。(ch07)

**位置n/2定位法** — 若已知某数出现次数超过50%，排序后它必然位于第n/2位，只需对该位置做一次计数确认。(ch09)

**向上调整 (Siftup)** — 新元素插入到数组末尾后，与父结点比较并沿路径持续上浮，直到满足堆性质，O(log N)。(ch07)

**向下调整 (Siftdown)** — 堆顶或某结点值变化后，与较小/较大的子结点交换并沿路径持续下沉，直到重新满足堆性质，O(log N)。(ch07)

**优先队列 (Priority Queue)** — 支持插入元素与取出最值元素两种操作的抽象数据结构，堆是其经典高效实现。(ch07)

**增广路 (Augmenting Path)** — 一条起点和终点都是未匹配点的路径，沿路径翻转匹配可使匹配数加1。(ch08)

**栈 (Stack/LIFO)** — 后进先出的线性结构，插入删除都在栈顶，用单个下标 top 实现。(ch02)

**正无穷表示** — 代码中用足够大的常数（如99999999）表示两顶点不可达，简化分支逻辑。(ch05)

**指针 (pointer) / malloc** — 指针存储内存地址（*p 取值，&a 取地址）；malloc 运行时动态申请内存，返回需强转的 void* 指针。(ch02)

**状态/决策点 (step)** — 搜索树上的一个节点，代表目前已经做出的部分选择。(ch04)

**最小生成树 (MST)** — 无向连通图中用 n-1 条边连接全部 n 个顶点、且边权和最小的子图。(ch08)
