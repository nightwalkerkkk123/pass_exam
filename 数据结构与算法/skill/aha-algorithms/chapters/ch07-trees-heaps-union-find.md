# 第7章：神奇的树

## 核心思想 (Core Idea)

树是"不包含回路的连通无向图"，正是这个"无环"特性衍生出唯一路径、n-1条边等一系列好性质；本章沿着"树 → 二叉树 → 堆 → 并查集"这条线，展示同一种数据结构如何在优先队列（堆）和分组合并（并查集）这两类完全不同的问题里发挥作用。

## 引入的框架与算法 (Frameworks Introduced)

### 1. 树（Tree）与二叉树（Binary Tree）

**是什么**：树是任意两个结点间有且仅有一条路径的无向图，等价于"不包含回路的连通无向图"。二叉树是特殊的树，每个结点最多有两个子树（左儿子、右儿子），递归定义为：要么为空，要么由根结点+左子树+右子树组成，且左右子树本身也是二叉树。

**何时用**：凡是具有层级、包含关系的数据都天然适合用树表达，例如文件目录、家族族谱、公司组织架构、赛事晋级图。

**核心性质**（书中直接给出）：
1. 树中任意两个结点有且仅有唯一一条路径连通；
2. n 个结点的树恰好有 n-1 条边；
3. 在树中任加一条边必构成回路。

术语速记：根结点（无父亲）、叶结点（无子结点）、内部结点（非根非叶）、深度（根为第 1 层，逐层往下加 1）。

**满二叉树（Full/Perfect Binary Tree）与完全二叉树（Complete Binary Tree）**：满二叉树要求每个内部结点都有两个儿子（等价于所有叶子深度相同），深度为 h 的满二叉树恰有 2^h − 1 个结点；完全二叉树则允许最后一层不满，但必须从左到右连续排列、不能有"右边有而左边缺"的情况。

**数组存储的神奇规律**：完全二叉树可以直接用一维数组存储，无需指针。若父结点编号为 k，则左儿子编号为 2k，右儿子编号为 2k+1；反过来，若儿子编号为 x，其父结点编号为 x/2（C 语言整数除法自动向下取整）。含 N 个结点的完全二叉树高度约为 log₂N。这条规律是后面"堆"的存储基础。

### 2. 堆（Heap）——优先队列的实现

**是什么**：堆是一种特殊的完全二叉树。最小堆要求每个父结点的值都不大于其子结点；最大堆则相反，父结点值都不小于子结点。

**何时用**：需要反复"取最值 + 插入新元素"的场景——即优先队列（Priority Queue）。若用普通数组线性扫描找最值，单次操作是 O(N)；若做 N 次这样的操作则退化为 O(N²)。堆能把单次操作压到 O(log N)。

**怎么工作（核心操作）**：
- **向下调整 siftdown(i)**：某结点的值变大（如堆顶被替换）导致不再满足堆性质时，将它与左右儿子中较小者比较，若比儿子大就交换，并沿该路径持续向下调整，直到满足堆性质或到达叶结点为止。时间复杂度 O(log N)。
- **向上调整 siftup(i)**：插入新元素时，先把它放到数组末尾，再与其父结点比较，若比父结点小就交换，持续向上直到不再违反堆性质或到达堆顶。时间复杂度 O(log N)。
- **建堆（Build Heap）**：法一是从空堆开始逐个插入并 siftup，整体 O(N log N)；法二（更优）是把 N 个数直接放入数组形成完全二叉树，然后从最后一个非叶结点（编号 N/2）开始倒着向前，对每个结点做一次 siftdown，整体只需 O(N)。
- **堆排序（Heap Sort）**：建好堆后，每次把堆顶（当前最值）与末尾元素交换、堆大小减一、再对新堆顶 siftdown，重复直到堆为空，即可得到有序序列，整体时间复杂度 O(N log N)。

### 3. 并查集（Union-Find）

**是什么**：一种维护"不相交集合（分组）"的数据结构，也称不相交集（Disjoint Set）。逻辑上是维护一片森林——初始时每个元素各自成一棵只有一个结点的树，之后根据"两个元素属于同一组"的信息逐步把树合并成更大的树。

**何时用**：需要频繁判断"两个元素是否已在同一分组"并支持"合并两个分组"的场景，例如判断连通性、分组归类（本章例子：判断团伙数）、后续章节最小生成树中的 Kruskal 算法。

**怎么工作**：
- 用数组 f[i] 记录 i 的"上级"（父结点），初始化 f[i] = i（每人是自己的老大）。
- **查找 getf(v)**：递归地沿父指针往上找，直到找到 f[v] == v 的祖先（即这个集合的代表元/根）。
- **路径压缩（Path Compression）**：在 getf 递归返回时，顺手把沿途经过的每个结点的父指针直接指向最终找到的根，写法是 `f[v] = getf(f[v]); return f[v];`。这样不增加当次查找的复杂度，却让后续查找显著变快（结点被压平到根的直接子结点）。
- **合并 merge(v,u)**：分别 getf(v)、getf(u) 得到两个根 t1、t2；若 t1 ≠ t2，说明二者尚不在同一集合，令 f[t2] = t1（书中称为"靠左"法则，右边的根归顺到左边的根下）。

## 关键概念 (Key Concepts)

- **树（Tree）**：不包含回路的连通无向图，任意两点间路径唯一。
- **深度（Depth）**：从根结点到该结点经过的层数，根本身为第 1 层。
- **完全二叉树（Complete Binary Tree）**：除最后一层外每层都满，且最后一层结点从左向右连续排列。
- **堆（Heap）**：满足"父结点 ≤（或 ≥）所有子结点"的完全二叉树，分最小堆/最大堆。
- **优先队列（Priority Queue）**：支持"插入元素"与"取出最值元素"两种操作的抽象数据结构，堆是其经典实现。
- **向下调整/向上调整（Siftdown/Siftup）**：堆维持自身性质的两种局部修复操作，均为 O(log N)。
- **并查集（Union-Find）/不相交集（Disjoint Set）**：维护分组归属并支持合并、查询的数据结构。
- **路径压缩（Path Compression）**：并查集查找时顺带把沿途结点直接挂到根下，加速后续查询的常见优化。

## 思维模型 (Mental Models)

- 当需要反复"拿走当前最大/最小值并塞入新值"时，用**堆**而不是排序数组或线性扫描——堆把这类操作的代价从 O(N) 摊薄到 O(log N)。
- 把完全二叉树当作"可以直接用数组下标算亲属关系的树"来理解：下标 k 的儿子是 2k、2k+1，父亲是 k/2，不需要额外的指针字段。
- 当问题的本质是"不断出现的两两归属关系，最终要问'是否同组'或'一共几组'"时，用**并查集**而不是每次都做一次连通性搜索（如 DFS/BFS）——这是并查集与图遍历类算法的分工边界。
- 把并查集的"查根"过程当作"擒贼先擒王"：只有找到最终的祖先（王），才能判断两人是否同伙；中间的"小 BOSS"不能说明问题。

## 常见陷阱 (Anti-patterns)

- 误以为只要判断两个结点的直接父结点相同就算同组——必须一路找到根（最终祖先）才能下结论，中间层级会随合并顺序变化。
- 建堆时逐个插入再 siftup，虽然正确但是 O(N log N)，不是本章推荐的最优建堆方式；应优先采用"从最后一个非叶结点倒序 siftdown"的 O(N) 方法。
- 完全二叉树的数组下标若从 0 开始存储，2k/2k+1 的公式会变化（需相应调整为 2k+1/2k+2），书中代码统一从下标 1 开始存，混用两套下标体系是常见出错点。

## 代码示例 (Code Examples)

```c
// 并查集：查找（带路径压缩）与合并
int f[1000];

void init(int n) {
    for (int i = 1; i <= n; i++) f[i] = i;   // 初始时每个人的老大是自己
}

int getf(int v) {                            // 找祖宗（递归+路径压缩）
    if (f[v] == v) return v;
    f[v] = getf(f[v]);                       // 沿途结点直接挂到根下
    return f[v];
}

void merge(int v, int u) {                   // 合并两个集合
    int t1 = getf(v), t2 = getf(u);
    if (t1 != t2) f[t2] = t1;                // “靠左”法则：右边归顺左边
}
```
这段代码演示了并查集最核心的两步——递归查根时顺带做路径压缩，以及合并时只需改动两个根结点的父指针，整体近似 O(1) 均摊代价。

## 讲解示例 (Worked Example)

**堆的例子**：14 个数 99、5、36、7、22、17、46、12、2、19、25、28、1、92，要求反复"取最小值+插入新值"。若逐次线性扫描，1 亿次操作大约需要 1 亿的平方次运算（约 115 天，按每秒 10 亿次估算）；改用最小堆后，每次取最小只需 O(log N)，1 亿次操作总计约 27 亿次运算，仅需 2.7 秒——这正是本章反复强调的"算法的伟大"。建堆时从数组末尾的最后一个非叶结点（第 N/2 个）开始，逐个向前对每个结点做 siftdown，只需两行代码 `for (i = n/2; i >= 1; i--) siftdown(i);` 即可在 O(N) 内把无序数组整理成合法的最小堆。

**并查集的例子（"解密犯罪团伙"）**：警方掌握 10 个强盗中若干对"是同伙"的线索（如 1-2、3-4、5-2、4-6、2-6、8-7、9-7、1-6、2-4）。初始时 f[i]=i，每人各自为政。逐条处理线索，每次找到两人各自的"最高领导人"（getf），若不同则合并（靠左法则、根指向根）。处理到"5 号与 2 号是同伙"时会出现有趣情形：2 号的老板是 1 号，5 号的老板是自己，"靠左"要求右边（5 号一系）归顺左边（1 号一系），于是把 f[1] 也改为 5——这体现了"必须找祖宗、不能只看直接父结点"的关键点。全部线索处理完毕后，数组中满足 f[i]==i 的下标只剩 5、9、10 三个，说明恰好存在 3 个独立犯罪团伙：以 5 为首的团伙（含 1、2、3、4、5、6 号）、以 9 为首的团伙（含 7、8、9 号）、以及仅 10 号一人的团伙。

## 学习要点 (Key Takeaways)

- 完全二叉树可用一维数组存储：父 k → 左儿子 2k、右儿子 2k+1；儿子 x → 父 x/2（整数除法）。
- 堆的两个基本操作 siftdown（自上而下修复）和 siftup（自下而上修复）都是 O(log N)；建堆首选自底向上 O(N) 方法，而不是逐个插入的 O(N log N) 方法。
- 优先队列的经典实现就是堆；堆排序整体复杂度 O(N log N)，与快速排序同阶但无需额外递归栈的最坏情况顾虑。
- 求数列中第 k 大/小的数，可用大小为 k 的堆（求第 k 大用最小堆、求第 k 小用最大堆），复杂度 O(N log k)，优于全排序。
- 并查集解决"分组归属+合并"问题：查找必须递归到根（祖先），路径压缩能把后续查找摊薄到近似 O(1)，是接下来学习最小生成树（Kruskal 算法）判环的必备前置知识。
- Dijkstra 最短路径算法可以用堆优化，将复杂度从 O(N²) 降到 O((M+N) log N)，这是堆在图论算法中的重要应用，为后续章节埋下伏笔。
