# 第5章：图的遍历

## 核心思想 (Core Idea)
图（Graph）把"顶点+边"这种更自由的关系结构统一存进邻接矩阵后，深度优先搜索（DFS）和广度优先搜索（BFS）就是同一件事——"不漏、不重地走遍所有顶点"——的两种走法：DFS 一条路走到黑再回溯，BFS 一圈一圈向外扩散；选哪种走法，取决于你要的是"路径本身"还是"最少步数"。

## 引入的框架与算法 (Frameworks Introduced)

### 图的基本定义与邻接矩阵表示法（Adjacency Matrix）
- **是什么**：图由顶点（vertex，图中的小圆点）和边（edge，连接顶点的线）组成。若边有方向，称有向图（Directed Graph）；若边无方向（i 能到 j，j 也能到 i），称无向图（Undirected Graph），此时邻接矩阵沿主对角线对称。
- **怎么存**：用二维数组 `e[i][j]` 表示顶点 i 到顶点 j 是否有边（或边的权值），自己到自己（i==j）通常设为 0，不存在的边设为一个很大的数（书中用 99999999 代表正无穷）表示不可达。
- **何时用**：顶点数不太大、需要频繁判断"i 到 j 是否有边"时，邻接矩阵是最直观的入门存储方式（代价是空间 O(n²)，稠密图友好，稀疏图浪费空间——书中提到后面章节会讲邻接表）。

### 深度优先搜索（Depth-First Search, DFS）作为图的遍历
- **何时用**：需要系统地"走遍"图中所有顶点、枚举所有路径，或者问题天然是"选择—递归—回溯"的结构（如后面求最短路径的暴力解法）。
- **怎么做**：从一个未访问过的顶点出发，沿着一条边走到相邻的未访问顶点；到达新顶点后，重复同样的过程继续往下走；当当前顶点已经没有可走的未访问邻居时，回溯（backtrack）到上一个顶点，换一条边继续尝试；直到所有顶点都被访问过。用一个 `book` 数组标记某顶点是否已访问过，防止重复访问和死循环。

### 广度优先搜索（Breadth-First Search, BFS）作为图的遍历
- **何时用**：所有边"等价"（权值相同）时，求从起点到终点的最少步数/最少经过边数，BFS 是最自然的选择，因为它按"距起点的层数"逐层扩展。
- **怎么做**：起始顶点先入队（queue）并标记已访问；只要队列不空，就取出队首顶点，把它所有未访问过的相邻顶点标记为已访问并依次入队，然后处理下一个队首顶点，如此循环，直到队列为空或找到目标顶点。因为是逐层扩散，第一次到达某顶点时所走的步数一定是最少的。

### 用 DFS 暴力求最短路径（对应本章"城市地图"一节）
- **何时用**：图不大、只是想直观理解"最短路径"这个概念，且愿意接受指数级枚举所有路径的代价（本章尚未讲 Floyd/Dijkstra/Bellman-Ford 这些真正高效的最短路径算法，那是下一章内容）。
- **怎么做**：DFS 递归地尝试所有从起点出发、不重复经过已在路径中的顶点的走法；每走一步累加当前路程 `dis`；一旦到达终点就用全局变量 `min` 更新已发现的最小路程；回溯时记得把 `book[j]` 重新置 0（取消标记），否则会漏掉其他分支；可加剪枝——若 `dis` 已经超过当前 `min`，立即返回，不再往下搜。

### 用 BFS 求最少中转次数（对应本章"最少转机"一节）
- **何时用**：所有边权重相同（这里是"一条航线算一次转机"），只关心最少经过几条边到达终点，而不关心具体走了哪条路径。
- **怎么做**：给队列中每个元素额外附带一个字段（书中用结构体 `struct note{x, s}`，x 是城市编号，s 是到达该城市已转机的次数）；每次扩展新顶点时，令其转机次数等于当前顶点转机次数 +1；一旦扩展到目标城市就立刻停止，此时该顶点携带的次数即为最少转机数。

## 关键概念 (Key Concepts)
- **顶点（Vertex）**：图中的基本节点，例如一个城市。
- **边（Edge）**：连接两个顶点的线，可以带权值（如路程），也可以只表示"是否连通"。
- **有向图 / 无向图**：边是否有方向；无向图的邻接矩阵一定对称。
- **邻接矩阵（Adjacency Matrix）**：用二维数组 `e[i][j]` 记录顶点间是否有边/边权的存储方式。
- **book 数组**：DFS/BFS 中用来标记"某顶点是否已访问过"的布尔数组，是避免重复访问和死循环的关键。
- **回溯（Backtracking）**：DFS 在枚举路径时，退出某个分支前要撤销对该分支的标记（如 `book[j]=0`），才能让其他分支正确探索。
- **时间戳**：DFS 遍历中，每个顶点被访问到的先后次序编号，直观展示了 DFS 的访问顺序。
- **队列（Queue）**：BFS 依赖的先进先出结构，用 `head`、`tail` 两个下标模拟，`head<tail` 时循环处理队首元素。
- **正无穷的表示**：代码里用一个足够大的常数（如 99999999）表示"不可达"，避免用特殊值处理带来的分支复杂度。

## 思维模型 (Mental Models)
- 把 DFS 当作"一条路走到黑，走不通就退回上一步再换一条路"来理解——它天然适合用递归实现，函数调用栈本身就在帮你记录"回溯到哪里"。
- 把 BFS 当作"石头丢进水里泛起的一圈圈涟漪"来理解——每一层都是"再多走一步能到达的新地方"，所以第一次摸到目标的那一层就是最少步数。
- 当问题是"边权相同、求最少步数/最少中转"时，用 BFS；当问题是"要枚举路径、结合权重做穷举或者天然递归"时，用 DFS。
- 把邻接矩阵 `e[i][j]` 当作一张"i 行 j 列"的查询表来理解：判断"i 到 j 是否有边"永远是 O(1) 的查表操作，代价是无论图多稀疏都要开 n×n 的空间。

## 常见陷阱 (Anti-patterns)
- **DFS 忘记回溯**：递归深入某个分支、标记 `book[j]=1` 后，如果返回上一层前忘记把它改回 `book[j]=0`，会导致其他分支误以为该顶点"已经走过"，从而漏掉本该存在的路径——书中原文特意强调了这一步。
- **BFS 忘记 `head++`**：处理完当前队首顶点的所有邻居后，如果忘记把 `head` 往后移一位，会导致同一个顶点被反复处理，陷入死循环——书中代码注释里专门用"千万不要忘记"提醒。
- **把 DFS 暴力枚举当成"最短路径算法"来用在大图上**：这种做法本质是枚举所有路径，时间复杂度随顶点数指数级增长，只适合本章教学演示的小规模例子；真正处理大规模最短路径问题要用下一章的 Floyd/Dijkstra/Bellman-Ford。
- **有向图和无向图的初始化搞混**：无向图要同时给 `e[a][b]` 和 `e[b][a]` 赋值，有向图只赋一侧；混淆会导致图的连通关系与题意不符。

## 代码示例 (Code Examples)
```c
// 图的深度优先遍历（邻接矩阵表示法），book 数组防止重复访问
#include <stdio.h>
int book[101], sum, n, e[101][101];

void dfs(int cur) // cur 是当前所在的顶点编号
{
    int i;
    printf("%d ", cur);
    sum++;                 // 每访问一个顶点，sum 加 1
    if (sum == n) return;   // 所有顶点都已访问过，直接返回

    for (i = 1; i <= n; i++) // 从 1 号顶点到 n 号顶点依次尝试
    {
        // 判断 cur 到顶点 i 是否有边，且顶点 i 尚未访问过
        if (e[cur][i] == 1 && book[i] == 0)
        {
            book[i] = 1;   // 标记顶点 i 已访问
            dfs(i);        // 从顶点 i 继续深入遍历
        }
    }
    return;
}
```
这段代码展示了 DFS 的核心骨架：标记当前顶点→枚举所有邻居→对未访问的邻居递归深入→自然回溯（函数返回后循环体继续尝试下一个邻居），是本章后续"求最短路径""求连通性"等所有 DFS 应用的共同底座。

## 参考表格 (Reference Tables)
| 维度 | DFS（深度优先） | BFS（广度优先） |
|---|---|---|
| 底层结构 | 递归调用栈（或显式栈） | 队列（queue） |
| 访问顺序 | 一条路走到底再回溯 | 按距起点的层数逐层扩展 |
| 典型应用（本章） | 枚举所有路径、暴力求最短路程 | 边权相同时求最少步数/最少中转 |
| 是否保证第一次到达即最优 | 否（需要遍历完所有路径取最小值） | 是（对无权图/等权图成立） |
| 空间/时间特点 | 递归深度可能很深；枚举路径可指数级 | 每个顶点、每条边最多入队/扩展一次，线性于 O(n+m) |

## 讲解示例 (Worked Example)
本章"城市地图"一节：小哼要从 1 号城市（自己家）走到 5 号城市（小哈家），已知 5 个城市、8 条单向公路，用邻接矩阵 `e` 存储路程，不可达记为无穷大。用 DFS 从 1 号城市出发递归尝试所有路径，走到 5 号城市（`cur==n`）就用当前累计路程 `dis` 更新全局最小值 `min`，每走一步前标记 `book`，回溯时取消标记。书中枚举出的全部路径是：
- 1→2→3→4→5，路程 14
- 1→3→2→5，路程 9
- 1→5，路程 10

三条路径比较后，最短路径是 **1→3→2→5，长度 9**。之后书中把同一张地图改成无向图重新计算，此时因为边可以双向走，反而找到了更短的 1→3→5（长度 7）——这说明"图是有向还是无向"直接决定了可达路径集合，进而影响最短路径的结果。

本章"最少转机"一节则是同一套邻接矩阵换成 BFS：小哼和小哈要从 1 号城市飞到 5 号城市，边权全部视为 1（一条航线=一次转机）。BFS 从 1 号城市入队开始，逐层扩展：第一层能到达 2、3 号城市；从 2 号城市能再扩展到 3、4 号城市（3 号已入队，只需新入队 4 号）；从 3 号城市能扩展到 4、5 号城市（4 号已入队，只需新入队 5 号）——一旦 5 号城市入队，算法立刻停止，此时它携带的转机次数就是全局最少值。这体现了 BFS"逐层扩散、第一次抵达即最优"的核心性质。

## 学习要点 (Key Takeaways)
- DFS 和 BFS 都能"遍历"图的全部顶点，但访问顺序完全不同：DFS 靠递归/栈"深入到底再回头"，BFS 靠队列"一层一层往外扩"。
- 求"最少步数/最少中转"这类边权相同的最短路径问题，优先用 BFS，因为它天然保证第一次到达目标就是最优解，不需要像 DFS 那样枚举完所有路径再比较。
- DFS 求最短路径（本章的城市地图解法）是暴力穷举，指数级复杂度，仅用于教学演示；真正的高效最短路径算法（Floyd、Dijkstra、Bellman-Ford 等）留到下一章。
- DFS 回溯时一定要撤销标记（`book[j]=0`），BFS 扩展完队首后一定要 `head++`，这两处是本章代码里最容易出错、也是书中反复强调的两个细节。
- 有向图 vs 无向图会直接改变可达关系和最短路径结果，初始化邻接矩阵时要根据题意决定是否对称赋值。
- 邻接矩阵是图的入门存储法，判断边是否存在是 O(1)，但空间是 O(n²)，图很大或很稀疏时应改用邻接表（书中后续章节会介绍）。
