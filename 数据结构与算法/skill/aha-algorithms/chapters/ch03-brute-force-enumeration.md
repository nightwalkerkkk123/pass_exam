# 第3章：枚举！很暴力

## 核心思想 (Core Idea)
枚举（穷举）算法的本质是：把问题所有可能的候选解一一列出并逐个代入验证，用"暴力"换正确性；但暴力不等于蛮干——本章反复强调的真正技巧是**分析候选范围的上界**与**用变量间的运算关系降低枚举维度**，从而把不可接受的时间复杂度压到可接受的范围。

## 引入的框架与算法 (Frameworks Introduced)

### 第1节 坑爹的奥数——枚举算法入门
枚举算法又叫穷举算法：把每一个可能的取值都试一遍，用条件判断筛出满足要求的解。

- 用于最简单的情形：口3×6528=3口×8256，两个方框填同一数字，只需 `for(i=1;i<=9;i++)` 枚举一遍即可。
- 稍复杂的版本：口口口+口口口=口口口，1~9 各用一次拼成加法等式，共多少种组合？朴素做法是九层嵌套循环枚举 a~i 九个数字，再用一长串 `&&` 判断两两互不相等，外加等式是否成立；这类代码又长又难维护。
- 改进做法：引入**标记法（book 数组）**——用一维数组 `a[1..9]` 存放九个变量，用 `book[1..9]` 标记 1~9 各数字是否被用过。每轮枚举后统计 `book` 数组里 1 的个数，若恰为 9 则说明九个数字互不相同，不必再写九个变量两两比较的判断式。因为 `173+286=459` 与 `286+173=459` 被视为同一组合，最终答案要对 `total` 除以 2。

### 第2节 炸弹人——网格候选点枚举 + 方向扫描
问题：给定一张由墙 `#`、空地 `.`、敌人 `G` 组成的地图，只有一枚超远程炸弹，求放在哪个空地能消灭最多敌人。

- 做法：用二维字符数组存地图；双重循环枚举每一个空地作为候选放置点；对每个候选点分别向上、下、左、右四个方向扫描，直到碰到墙 `#` 为止，边扫描边对遇到的 `G` 计数；四个方向计数之和即该点可消灭的敌人数；全局维护最大值 `map` 及对应坐标 `(p,q)`。
- 特别提醒：书中坐标 `(x,y)` 指"第 x 行第 y 列"，不是数学坐标系里的 x 轴、y 轴，容易搞混。
- 该算法有一个隐藏缺陷：它只按几何直线和墙壁判断"能否炸到"，却没有检查小人能否**走到**这个空地——如果把地图里某处墙改成空地，纯枚举算法可能选中一个小人根本走不到的位置。这是本章故意留下的伏笔，作者明确说"将在第4章讨论"，指向下一章的 BFS/DFS 连通性判断。

### 第3节 火柴棍等式——枚举范围的分析与降维
问题：用 m 根火柴棍拼出形如 A+B=C 的等式（0~9 每个数字的火柴棍拼法固定；加号、等号各占 2 根；A+B=C 与 B+A=C 视为不同解），一共有多少种拼法？

- 关键第一步：分析枚举上界。m 最多 24 根，减去加号等号占用的 4 根，剩余最多 20 根火柴棍；而数字 1 只需 2 根火柴棍最省，20 根最多拼出 10 个 1，所以 A、B、C 中任何一个数都不会超过 11111。这给出了枚举的具体范围 0~11111，而不是漫无边际地枚举。
- 朴素做法：三层循环分别枚举 A、B、C，判断三者火柴棍总数是否等于 `m-4`。时间复杂度约 O(m³)（m≈11112），实测要跑 1000 多秒，不可行。
- **降维优化**：C 完全可以由 A+B 算出来，不需要单独枚举！只枚举 A、B 两层循环，C=A+B 直接算出，判断三者火柴棍数之和是否等于 `m-4`。这一步把复杂度从 O(m³) 降到 O(m²)，运行时间从 1000 多秒降到约 1 秒。这是本章"暴力枚举也需要仔细分析问题"的核心示范。
- 配合一个 `fun(x)`（或称 `matchCost(x)`）函数：用长度为 10 的数组记录 0~9 每个数字需要的火柴棍根数，对 x 逐位取余、整除，把每一位的火柴棍数累加起来，返回总数。

### 第4节 数的全排列——嵌套循环枚举的极限
问题：求 1~n 的全排列（n=3 时为 123 的全排列，n=9 时为 123456789 的全排列）。

- 做法：n 层嵌套 for 循环，每层枚举一位，配合一长串 `&&` 判断各层变量两两不相等，满足则输出一种排列。n=3、n=4 时代码尚可接受，但 n=9 时需要写九层嵌套循环加 36 个两两不等判断，明显不现实。
- 本节故意留下"终极问题"：输入任意 n，如何用同一份代码输出 1~n 的全排列？嵌套循环的层数无法随输入参数变化，这正是暴力枚举（本质是"层数固定的 DFS"）的天花板，直接引出下一章"万能的搜索"（用递归/DFS 处理任意深度的枚举）。

## 关键概念 (Key Concepts)
- **枚举算法（穷举法）**：把问题所有可能的候选解一一列出并代入验证的算法思想，正确性有保证但效率取决于候选空间大小。
- **标记数组（book[] / mark array）**：用 0/1 数组记录某个值是否已出现过，替代大量 `&&` 判断"互不相等"，代码更简洁也更容易扩展到更大规模。
- **枚举范围的上界分析**：动手写循环前先从题目约束（如总资源数）反推每个变量的最大可能取值，避免无意义地枚举过大范围。
- **降维（消元）优化**：当变量之间存在确定性运算关系（如 C=A+B）时，只枚举独立变量，其余变量直接算出，可把复杂度降低一个维度。
- **可达性问题（Reachability）**：候选解在网格/图上是否真的可以从起点到达，纯几何或数值枚举默认不考虑这一约束，需要额外的图搜索验证。
- **全排列（Full Permutation）**：n 个不同元素的所有可能顺序排列，共 n! 种；本章用固定层数的嵌套循环实现，埋下"层数需要随 n 变化"的矛盾。

## 思维模型 (Mental Models)
- 当候选空间离散且有限、每个候选的验证代价可接受时，用暴力枚举先把问题"做出来"，再考虑优化。
- 当发现某个枚举变量可以由其他变量的运算结果得到时，把它从循环中拿掉，用表达式直接计算——这是本章枚举优化的核心思路。
- 把"判断多个变量互不相同"这件事，当作"给取值范围建一个标记数组、数一数标记了几个"来理解，而不是堆叠一串 `&&`。
- 把多层嵌套 for 循环枚举，当作"层数写死的 DFS"来理解：当层数需要随输入变化时，嵌套循环这条路就走到头了，必须换成递归。

## 常见陷阱 (Anti-patterns)
- 用一长串 `a!=b && a!=c && ...` 判断若干变量两两不同：代码冗长、n 越大越不可维护，应改用标记数组统计法。
- 该降维时不降维：三个变量存在运算关系（如 C=A+B）却仍然三层都枚举，导致复杂度多一个数量级，实际运行时间可能相差上千倍（本章火柴棍例子从 1000 多秒降到 1 秒）。
- 枚举前不分析范围上界，直接用题目给定的最大可能值（如"火柴棍最多 24 根"）机械地设一个过大的循环上限，浪费大量无效枚举。
- 在网格/图问题中只按几何或数值条件枚举候选位置，忽略"是否可达"的隐含约束（炸弹人问题：小人根本走不到的位置也被当作有效候选）。
- 用固定层数的嵌套循环实现全排列，只能应对写死的 n，无法参数化，是暴力枚举思路的天花板，须改用递归/DFS。

## 代码示例 (Code Examples)
```c
#include <stdio.h>

/* 0~9 每个数字需要的火柴棍根数 */
int matchCost(int x) {
    int cost[10] = {6, 2, 5, 5, 4, 5, 6, 3, 7, 6};
    int num = 0;
    while (x / 10 != 0) {
        num += cost[x % 10];
        x /= 10;
    }
    num += cost[x];
    return num;
}

int main() {
    int m, sum = 0;
    scanf("%d", &m);
    for (int a = 0; a <= 1111; a++) {
        for (int b = 0; b <= 1111; b++) {
            int c = a + b;               /* 用加法算出C，而不是再枚举一层C */
            if (matchCost(a) + matchCost(b) + matchCost(c) == m - 4) {
                printf("%d+%d=%d\n", a, b, c);
                sum++;
            }
        }
    }
    printf("一共可以拼出%d个不同的等式\n", sum);
    return 0;
}
```
展示本章最核心的优化技巧：把三重枚举（A、B、C 各一层）降为二重枚举（只枚举 A、B，C 由 A+B 直接算出），复杂度从 O(m³) 降到 O(m²)。

## 讲解示例 (Worked Example)
以"火柴棍等式"为例，设小哼手上有 m=18 根火柴棍。

1. 先求上界：除去加号、等号占用的 4 根，剩余 14 根可供 A、B、C 使用；`matchCost` 表为 `[6,2,5,5,4,5,6,3,7,6]`（对应数字 0~9）。
2. 枚举 A、B（0~11111 范围内，但 18 根火柴棍时实际有效值很小），令 `C = A + B`，检查 `matchCost(A)+matchCost(B)+matchCost(C) == m-4 = 14`。
3. 逐一验证部分解：
   - A=0,B=4：`matchCost(0)=6, matchCost(4)=4, C=4, matchCost(4)=4`，总和 `6+4+4=14`，成立 → `0+4=4`。
   - A=2,B=7：`matchCost(2)=5, matchCost(7)=3, C=9, matchCost(9)=6`，总和 `5+3+6=14`，成立 → `2+7=9`。
   - A=1,B=10：`matchCost(1)=2, matchCost(10)=2+6=8, C=11, matchCost(11)=2+2=4`，总和 `2+8+4=14`，成立 → `1+10=11`。
4. 按此方法遍历完整个范围，最终得到 9 个解：`0+4=4、0+11=11、1+10=11、2+2=4、2+7=9、4+0=4、7+2=9、10+1=11、11+0=11`。注意 `A+B=C` 与 `B+A=C`（如 `0+4=4` 与 `4+0=4`）在本题规则下算作两个不同的解，所以这里**不需要**像坑爹的奥数那题一样除以 2。

## 学习要点 (Key Takeaways)
- 写枚举代码前先分析范围上界（例如从"总火柴棍数"反推每个数字不超过 11111），避免盲目扩大循环范围。
- 遇到多变量存在确定运算关系（如 C=A+B）时，减少一层枚举、用表达式推导，是把 O(m^k) 降到 O(m^(k-1)) 的通用技巧，考试中常考"如何优化枚举复杂度"。
- 判断"若干变量互不相同"优先用标记数组（book[]）而非堆叠 `&&`，既是代码整洁问题，也体现了用空间换判断效率的思路。
- 暴力枚举默认不检查物理/图论意义上的可达性约束，遇到网格/路径类题目要留意这个陷阱（炸弹人问题的经典反例）。
- 嵌套循环枚举全排列只能应对固定的 n，是暴力枚举方法论的天花板，为下一章递归/DFS 解法埋下动机。
- 本章四节内容层层递进：从最简单的单变量枚举，到标记法简化判重，到范围分析+降维优化，再到"嵌套循环无法参数化"的极限——核心是教会"暴力也要动脑子"。
