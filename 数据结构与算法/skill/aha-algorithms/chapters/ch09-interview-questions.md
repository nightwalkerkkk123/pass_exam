# 第9章：还能更好吗——微软亚洲研究院面试

## 核心思想 (Core Idea)

本章不讲新的数据结构，而是用一次真实的 MSRA（微软亚洲研究院）面试对话，示范"同一个问题如何被一步步逼近最优解"：面对"找出序列中出现次数超过 50% 的数"这个问题，作者的解法从 O(N²) 暴力比较，经排序、计数数组、离散化、快速选择，最终逼近可以只扫描一遍、O(1) 额外空间的**摩尔投票法（Boyer-Moore Voting Algorithm）**。全章的教学重点是"思考的层层递进"本身，而不是某一个孤立的算法。

## 引入的框架与算法 (Frameworks Introduced)

### 1. 暴力计数法（Brute-force Counting）
对每个数，用双重 for 循环与其余所有数比较，统计每个数出现的次数，取次数最大者。时间复杂度 O(N²)，是面试中给出的第一版答案，用来当作后续优化的参照系。

### 2. 排序法（Sorting-based Scan）
先用堆排序或快速排序将序列排序（O(NlogN)），使相同的数彼此相邻；再从头到尾扫一遍，用变量 `count` 统计当前数连续出现的次数，用 `max` 记录出现次数最大值。由于排序后相同数字必然聚在一起，一次线性扫描即可求出众数。

### 3. 中位数定位法（Position-n/2 Shortcut）
本章给出的一个关键洞察：**如果已知某个数出现次数一定超过 50%，那么排序后它必然占据第 n/2 位**。于是不必扫描整个数组，只需对第 n/2 位置的数再做一次计数确认（该数出现次数是否真的超过总数的 50%）即可，把"扫一遍确认"从整个排序结果收窄到一次计数校验。

### 4. 计数数组 / 类桶排序法（Counting Array，类 Bucket Sort）
开一个数组，下标对应数值，累加每个数值出现的次数，数组中最大值所在的下标就是答案。时间与空间复杂度均为 O(N+M)（M 为数值范围）。缺点在数值跨度很大时（如 1、10000、100000000）会浪费大量空间。

### 5. 离散化（Discretization / Coordinate Compression）
针对上述计数数组的空间浪费问题，先将原始数值按出现顺序（或排序后去重）重新编号，建立一一对应关系（如 1→1，10000→2，100000000→3……），再用小得多的数组做计数，从而把 O(M) 空间压缩为 O(N) 空间。

### 6. 快速选择法（Quickselect，借鉴快速排序的划分思想）
选取一个基准数做类似快速排序的一次划分（partition），把小于基准的移到左边、大于基准的移到右边。划分完成后基准所在下标为 k：若 k>n/2 则在左半部分递归找第 n/2 大的数，若 k<n/2 则在右半部分递归找，若 k=n/2 则该基准数即为答案。期望时间复杂度接近 O(N)（最坏情况仍可能退化，但平均表现优于排序法）。

### 7. 摩尔投票法（Boyer-Moore Voting Algorithm）——本章真正的答案
（因原文OCR质量不足，以下为标准算法讲解，非原书文字复现）原书在结尾只揭示了一条性质作为留白式的启发，并未给出完整代码：**在原序列中任意去掉两个互不相同的数，若某数原本出现次数超过 50%，那么在剩余序列中它出现的比例依然会超过 50%**。这正是摩尔投票法的数学基础。算法执行过程：
1. 维护一个候选数 `candidate` 和一个计数器 `count`，初始 `count = 0`。
2. 依次扫描序列中的每个数 `x`：
   - 若 `count == 0`，令 `candidate = x`，`count = 1`；
   - 否则若 `x == candidate`，`count++`；
   - 否则（`x != candidate`），`count--`（这一步等价于"同时抵消掉一个 candidate 和一个 x，即去掉两个不同的数"）。
3. 扫描结束后 `candidate` 即为多数元素（若题目保证存在多数元素）。

只需一次遍历、O(N) 时间、O(1) 额外空间，是本章及全书对该问题给出的最优解。

## 关键概念 (Key Concepts)

- **多数元素 / 主元素问题（Majority Element Problem）**：在长度为 n 的序列中，找出出现次数严格超过 n/2 的元素（若存在的话），是一类经典的"投票/表决"场景抽象。
- **摩尔投票法（Boyer-Moore Voting Algorithm）**：靠"计数器抵消"思想在一次线性扫描、常数额外空间内找出多数元素的算法。
- **计数抵消（Cancellation）**：把"一个多数元素 vs 一个非多数元素"看成一对相互抵消的票，抵消后剩余票中多数元素的优势比例不会减小，这是摩尔投票法能成立的直觉核心。
- **离散化（Discretization）**：把跨度很大、稀疏分布的数值映射为连续的小整数下标，从而让计数数组等下标索引类算法可用。
- **快速选择（Quickselect）**：借用快速排序的一次划分操作，只递归进入包含目标位置的那一侧，期望线性时间内找出第 k 大/小的元素。
- **投票系统类比**：书中把这一问题类比为选举——只要某候选人得票超过 50%，无需清点全部票数即可判定其当选，呼应位置 n/2 的直觉。

## 思维模型 (Mental Models)

- 当你发现一个问题已有可行解，但复杂度不够理想时，**把"能不能更好"当作追问的默认姿态**：本章的价值不在最终算法，而在于"排序够了吗？计数数组够了吗？还能不能再快"这条不断加压的追问链。
- 当序列中某元素的出现次数已知超过总数一半时，**把这类问题当作"投票+抵消"来理解**：任何一张非多数票都可以和一张多数票"同归于尽"，抵消到最后剩下的必然是多数元素的票。
- 当数值分布跨度极大而计数需要下标索引时，**先离散化再计数**，把"值域大小"问题转化为"元素个数"问题。
- 当只需要"排序结果中某个特定位置的值"而非"完整有序序列"时，**用快速选择代替完整排序**，避免为不需要的全序信息买单。

## 常见陷阱 (Anti-patterns)

- **误以为计数数组一定省时省力**：当数值分布跨度巨大（如书中 1、10000、100000000 的例子）时，直接开数组做计数会造成极大的空间浪费甚至无法申请，必须先离散化。
- **误以为找到位置 n/2 的数就万事大吉**：位置 n/2 的数只是"候选"，仍需要额外一次计数确认其出现次数确实超过总数的 50%——如果题目没有保证一定存在多数元素，这一步不能省略。
- **对摩尔投票法的误用**：若序列中不保证存在出现次数超过 50% 的元素，摩尔投票法扫描后得到的 `candidate` 未必是真正的多数元素，必须再做一次计数校验（这与"位置 n/2 法"要求的校验步骤是同一类陷阱）。

## 代码示例 (Code Examples)

（因原文OCR质量不足，以下为标准算法讲解，非原书文字复现）

```c
/* 摩尔投票法：在保证存在多数元素的前提下，一次遍历求出出现次数超过一半的数 */
int majorityElement(int a[], int n) {
    int candidate = a[0];
    int count = 0;
    for (int i = 0; i < n; i++) {
        if (count == 0) {
            candidate = a[i];
            count = 1;
        } else if (a[i] == candidate) {
            count++;
        } else {
            count--;   /* 相当于同时去掉一个 candidate 和一个不同的数 */
        }
    }
    /* 若题目未保证多数元素一定存在，此处需再扫一遍统计 candidate 的真实出现次数并与 n/2 比较 */
    return candidate;
}
```

该代码展示了"用一个候选值 + 一个计数器抵消"如何把 O(N+M) 的计数数组法压缩为 O(N) 时间、O(1) 空间，是本章追问链条的最终落点。

## 讲解示例 (Worked Example)

以书中给出的例子 `3, 3, 1, 1, 3, 2, 3` 为例（n=7，超过 50% 即需要出现次数 ≥4）：

- **排序法**：排序后为 `1, 1, 2, 3, 3, 3, 3`，扫描统计每个数连续出现次数，`3` 出现 4 次，`4 > 7/2`，命中。
- **位置 n/2 法**：n=7，中间位置（下标从 0 计，取第 3 位，即第 4 个数）排序后是 `3`；再单独统计 `3` 在原序列中出现次数为 4，`4 > 3.5`，确认 `3` 即为答案。
- **摩尔投票法逐步演算**（候选 candidate / 计数 count 的变化）：
  - 读入 3：count=0 → candidate=3, count=1
  - 读入 3：与 candidate 相同 → count=2
  - 读入 1：不同 → count=1
  - 读入 1：不同 → count=0
  - 读入 3：count=0 → candidate=3, count=1
  - 读入 2：不同 → count=0
  - 读入 3：count=0 → candidate=3, count=1
  - 扫描结束，candidate=3，即为出现次数超过 50% 的数。

可以看到，每次 candidate 与新数不同时 count 减一，恰好对应书中揭示的性质——"去掉两个不一样的数，多数元素在剩余序列中的占比依然超过 50%"，最终唯一"扛得住"抵消的候选就是真正的多数元素。

## 学习要点 (Key Takeaways)

- 遇到"某数出现次数超过总数一半"的题型，第一反应应是摩尔投票法：O(N) 时间、O(1) 空间，一次遍历搞定，是这一类题目面试中最理想的答案。
- 如果面试或考试只要求写出思路而非最优解，可以按本章的递进顺序作答：先给暴力解，再给排序解，展示"持续优化"的思考过程，往往比直接甩出最优解更能体现思维过程（这也是本章作者拿到 offer 的关键——面试官看重的是能否持续改进，而非一步到位）。
- 位置 n/2 法与摩尔投票法都必须搭配"确认性计数"：只有在题目保证多数元素存在时才能省略最终校验步骤。
- 离散化是应对数值跨度大、下标类算法（计数排序、计数数组）空间爆炸问题的通用技巧，值得记住并迁移到其他题目（如后续图论章节处理大范围点权/边权时也常用到）。
- 快速选择（Quickselect）的划分思路与后面章节可能出现的快速排序、第 k 大数问题是同一套代码骨架，可以复用记忆。
