# 第4章：万能的搜索

## 核心思想 (Core Idea)
搜索的本质是"穷举所有可能性，同时不重不漏"：深度优先搜索（DFS）沿一条路走到底、走不通就回头重试；广度优先搜索（BFS）像水波纹一样一圈圈向外扩散，谁先扩散到终点谁就是最短路径。

## 引入的框架与算法 (Frameworks Introduced)

### 深度优先搜索（Depth First Search, DFS）
- **用途**：生成全排列/全组合、判断连通性与路径是否存在、洪水填充式的区域染色计数。
- **原理**：写一个递归函数 `dfs(step)`，只解决"当下这一步该怎么办"——用一层 `for` 循环把当前决策点的所有可能分支都尝试一遍；每尝试一个分支就做标记（如 `book[i]=1`），然后递归调用 `dfs(step+1)` 把"下一步怎么办"交给同一套逻辑处理；递归返回后必须撤销刚才的标记（`book[i]=0`），即"回溯"，这样下一次循环才能干净地尝试别的分支。到达终止条件（如 `step==n+1`）时处理一个完整方案并 `return`。

### 广度优先搜索（Breadth First Search, BFS）
- **用途**：求无权图/网格上的最短路径、最少步数问题。
- **原理**：用一个队列模拟"层层扩展"：起点入队并标记为已访问；只要队列非空，就取出队首元素，向四个方向（用方向数组 `next[4][2]` 统一枚举）尝试扩展，凡是未越界、非障碍物且未被访问过的新点，标记为已访问后入队，其步数记为父节点步数+1；处理完队首的所有扩展后将其出队（`head++`）。因为总是先把"近"的点全部扩展完才轮到"远"的点，所以第一次抵达目标点时的步数必然是最短的。

### Flood Fill 漫水填充法（染色法）——DFS/BFS 的应用手法
- **用途**：统计一个连通区域的面积、统计地图中独立连通子图（岛屿）的个数；是 Windows"画图"油漆桶、Photoshop 魔术棒工具背后的算法原型。
- **原理**：以某个格子为"种子点"，用 DFS 或 BFS 向四周扩展，把所有可达、满足颜色/地形条件的格子标记（染）成同一种颜色（例如同一个负数编号）；不同的种子点得到不同的颜色，颜色种类数即为连通分量（岛屿）个数。

## 关键概念 (Key Concepts)
- **状态/决策点（step）**：搜索树上的一个节点，代表目前已经做出的部分选择，例如"正站在第几个盒子面前"。
- **约束判断（剪枝）**：判断某个扩展方向是否合法——是否越界、是否障碍物、是否已访问过，不合法的分支直接跳过不再扩展。
- **回溯（Backtrack）**：DFS 从递归深处返回上一层之前，必须撤销当前这一步做出的标记，使现场恢复到尝试之前的状态，是 DFS 能穷举全部方案而不遗漏的关键。
- **访问标记数组（book 数组）**：记录哪些位置/元素已经被占用或走过，防止重复扩展同一个点。
- **队列（Queue）**：BFS 的核心数据结构，先进先出，保证扩展顺序严格按照"离起点的距离"分层进行。
- **方向数组（next 数组）**：用二维数组把上、下、左、右（或顺时针的右、下、左、上）四个方向的坐标偏移打包起来，配合循环枚举，避免写四段重复代码。
- **连通性（Connectivity）**：网格或图中两个点之间是否存在一条合法路径可以相互到达。

## 思维模型 (Mental Models)
- 当问题要求穷举所有排列/组合/路径方案时，用 DFS+回溯：把"当下该怎么做"和"下一步该怎么做"看作完全相同的子问题，靠递归的自相似结构解决。
- 当问题要求最短路径或最少步数时，用 BFS：因为它按距离分层扩展，第一次到达目标点即为最优解，无需像 DFS 那样穷尽所有路径再比较。
- 把 DFS 当作"一条路走到黑，撞南墙才回头重新选择"来理解；把 BFS 当作"水波纹一圈一圈向外扩散，先扩散到的地方距离一定更近"来理解。
- 把染色法（Flood Fill）当作"给连通的土地刷同一种颜色"来理解：颜色的种类数就是连通分量（岛屿）的个数。

## 常见陷阱 (Anti-patterns)
- **DFS 回溯时忘记撤销标记**（漏写 `book[i]=0`）：会导致后续分支误以为该位置/元素已被占用，从而漏掉本应存在的解。
- **BFS 中忘记 `head++`**：队列永远不出队，扩展逻辑要么死循环、要么提前停滞，得不到正确结果。
- **判断终点与判断越界的顺序颠倒**：应先判断是否已到达终点，再做越界等其他检查；否则终点恰好处于"边界之外一格"的情况（如水管工游戏中终点被定义为 `(n, m+1)`）会被误判为越界而永远无法触发。
- **用 DFS 硬解最短路径问题**：DFS 虽然也能通过穷举所有路径、比较步数求出最短解，但效率远低于 BFS，因为它不具备"第一次到达即最优"的性质，必须遍历完所有可能路径才能确定答案。

## 代码示例 (Code Examples)
```c
// DFS 通用框架（对全排列问题的提炼）
void dfs(int step) {
    if (step == n + 1) {           // 到达边界：n个位置已放满
        for (int i = 1; i <= n; i++) printf("%d", a[i]);
        printf("\n");
        return;                     // 处理完一个方案必须立即返回
    }
    for (int i = 1; i <= n; i++) {
        if (book[i] == 0) {         // 判断这一步是否可行（i号牌是否还在手上）
            a[step] = i;
            book[i] = 1;            // 标记已使用
            dfs(step + 1);          // 递归处理下一步
            book[i] = 0;            // 回溯：撤销标记，才能尝试下一种可能
        }
    }
}
```
这段代码演示了 DFS 的通用结构：先判断边界条件，再用循环枚举当前决策点的所有分支，递归深入后必须回溯撤销标记，这是 DFS 能正确穷举全部情况、不重不漏的关键。

```c
// BFS 通用框架（对迷宫最短路径问题的提炼）
while (head < tail) {
    for (int k = 0; k <= 3; k++) {                 // 枚举4个方向
        int tx = que[head].x + next[k][0];
        int ty = que[head].y + next[k][1];
        if (tx < 1 || tx > n || ty < 1 || ty > m) continue;   // 越界
        if (a[tx][ty] == 0 && book[tx][ty] == 0) {            // 可走且未访问
            book[tx][ty] = 1;
            que[tail].x = tx;
            que[tail].y = ty;
            que[tail].s = que[head].s + 1;         // 步数=父节点步数+1
            tail++;
        }
    }
    head++;   // 当前节点扩展完毕，出队
}
```
这段代码演示了 BFS 用队列实现按层扩展，保证第一次抵达目标点时记录的步数就是最短步数。

## 参考表格 (Reference Tables)
| 维度 | DFS | BFS |
|---|---|---|
| 实现方式 | 递归（或显式栈） | 队列 |
| 典型适用问题 | 穷举全部方案、连通性判断、染色计数 | 最短路径 / 最少步数 |
| 是否需要撤销标记 | 需要（回溯 `book[i]=0`） | 不需要，每个点只入队一次 |
| "第一次到达即最优"性质 | 不具备，需比较所有方案 | 具备 |
| 空间开销特征 | 与递归深度相关 | 与队列中同时存在的状态数相关 |

## 讲解示例 (Worked Example)
**全排列（第1节）**：小哼手持编号 1、2、3 的三张扑克牌，要放进三个盒子。约定每到一个盒子前都先尝试放 1 号、再 2 号、再 3 号。`dfs(1)` 开始：第1个盒子放 1 号牌并标记，递归到第2个盒子；第2个盒子因 1 号已用只能放 2 号，再递归到第3个盒子放 3 号，此时到达 `step==n+1`，输出排列"123"。随后逐层回溯：先取回 3 号牌尝试别的（无别的可放，继续回溯），回到第2个盒子取回 2 号牌，改放 3 号牌，进而产生"132"；如此反复，最终依次生成 `123、132、213、231、312、321` 六种排列。

**解救小哈（第2、3节）**：小哼在迷宫入口 (1,1)，小哈在 (p,q)。若用 DFS，按"右、下、左、上"的顺序依次试探，遇到障碍物或越界就跳过，一旦走到 (p,q) 就用当前步数更新最小值 `min`，然后回溯继续尝试其他分支，穷尽所有路径后 `min` 即为最短步数。若改用 BFS：(1,1) 先入队；依次扩展出 (1,2)、(2,1)；再扩展出 (2,2)（可从两条路径到达，但只入队一次）；如此层层扩展，第一次扩展到 (p,q) 时对应的步数就直接是最短路径，不需要穷举比较。

**再解炸弹人 / 宝岛探险（第4、5节）**：先用 BFS（或 DFS）从小人所在点或飞机降落点开始，向四个方向扩展所有"可达且合法"的格子并计数或统计——炸弹人问题里对每个扩展到的点调用 `getnum` 统计可消灭的敌人数并维护最大值；宝岛探险问题里则单纯统计扩展到的点的总数即为岛屿面积。把这一整套扩展逻辑加上"每发现一个新的正数格子就用不同的负数染色"，就得到了统计地图中独立岛屿个数的 Flood Fill 解法。

## 学习要点 (Key Takeaways)
- DFS 的通用模板：判断终止条件 → 循环枚举分支 → 标记 → 递归 → 撤销标记（回溯）；BFS 的通用模板：起点入队 → 取队首 → 枚举方向 → 合法则标记入队 → 队首出队。
- 求最短路径/最少步数问题优先选 BFS：它"第一次到达即最优"，比 DFS 穷举全部路径再比较更高效。
- 标记数组是控制复杂度、避免重复访问的关键：DFS 用后必须回溯撤销，BFS 用后无需撤销（每点只入队一次）。
- 方向数组（如 `next[4][2]`）是网格搜索题的通用代码技巧，把四个方向的偏移收敛进一个循环。
- 终点判断要放在越界判断之前，否则终点恰好落在网格边界之外一格时会被误判为非法（水管工游戏的终点 `(n, m+1)` 就是典型例子）。
- Flood Fill（染色法）是 DFS/BFS 的直接应用：统计连通区域面积、统计连通子图（岛屿）个数都可归约为它。
- 复杂度直觉：全排列型 DFS 的规模是 O(n!)；网格搜索型 DFS/BFS 配合标记数组通常是 O(n×m)。
