# 第1章：一大波数正在靠近——排序

## 核心思想 (Core Idea)
排序问题可以用完全不同的策略求解，策略之间存在"空间换时间"与"比较次数"的权衡：桶排序用空间换取近乎线性的速度，冒泡排序用最朴素的相邻交换换取零额外空间但代价是 O(N²)，快速排序则用"分治+跳跃式交换"在两者之间找到平衡，是通用场景下最常用的排序算法。

## 引入的框架与算法 (Frameworks Introduced)

### 桶排序（Bucket Sort，本章简化版）
- **适用场景**：待排序数据是范围较小、确定的整数（例如 0~1000），且可以接受用与数据范围等大的额外空间。
- **原理步骤**：
  1. 申请一个大小为"数据范围+1"的数组 `book[]`（书里最初叫 `a[]`，后改名 `book` 表示"记录/标记"），全部初始化为 0。
  2. 依次读入每个数 `t`，执行 `book[t]++`，即把 `t` 出现的次数记录在下标为 `t` 的"桶"里。
  3. 按下标从小到大（或从大到小）遍历 `book[]`，每个下标 `i` 出现 `book[i]` 次就打印 `book[i]` 遍 `i`。
- **本质缺陷**：这其实是"计数排序"的雏形，而非真正的桶排序；它只能对数值本身排序，无法携带关联信息（例如姓名对应分数），且当数据范围极大（如 0~21 亿）时会浪费海量空间，也无法处理小数。

### 冒泡排序（Bubble Sort）
- **适用场景**：数据规模小、对额外空间要求严格（只需原地交换）、代码要求最简单的场合；不追求效率。
- **原理步骤**：
  1. 共进行 n-1 "趟"。
  2. 每一趟从第 1 位开始，依次比较相邻两个数 `a[j]` 与 `a[j+1]`，若顺序错误（如要从大到小排却前小后大）就交换。
  3. 每一趟结束后，会有一个数"归位"（滚到当前未归位区间的末尾，形似气泡上浮/下沉），因此第 `i` 趟只需比较到 `n-i` 位，已归位的数不再参与比较。
- **可携带附加信息**：把数组元素换成结构体（如 `struct student{ name, score }`），比较时只比较 `score` 字段、交换时整体交换结构体，就解决了桶排序"排完序丢身份"的问题。

### 快速排序（Quick Sort）
- **适用场景**：通用场景下追求效率、且不要求排序稳定性时的首选，平均性能远超冒泡排序。
- **原理步骤（一轮"归位"基准数）**：
  1. 从序列中选一个基准数（书中取最左边的数，例如序列 `6 1 2 7 9 3 4 5 10 8` 中的 6）。
  2. 设两个哨兵 `i`（指向最左）、`j`（指向最右）。**必须先移动哨兵 j**：从右往左找第一个小于基准数的数，停下。
  3. 再移动哨兵 i：从左往右找第一个大于基准数的数，停下。
  4. 若 `i<j`，交换 `a[i]` 与 `a[j]`，然后重复步骤 2-3。
  5. 若 `i` 与 `j` 相遇，将基准数与相遇位置的数交换——此时基准数正式"归位"，左边全部 ≤ 基准数，右边全部 ≥ 基准数。
  6. 对左右两个子序列递归执行同样的过程（分治思想），直到子序列长度为 1。
- **思想归属**：快速排序基于"二分"思想——每轮都把问题规模大致折半处理。

## 关键概念 (Key Concepts)
- **归位**：一个数经过排序过程被放到它在最终有序序列中的正确位置，之后不再移动。
- **哨兵 i / 哨兵 j**：快速排序中从序列两端向中间探测的两个下标变量，j 负责找"小于基准数"的数，i 负责找"大于基准数"的数。
- **基准数（pivot）**：快速排序每一轮选定用来做分割参照的数，通常取当前区间最左边的元素。
- **趟（pass）**：冒泡排序中完整走一遍"相邻比较+交换"的过程，每一趟固定归位一个数。
- **时间复杂度的常数忽略**：比较算法优劣时只看阶数最高的项，忽略较小的常数系数与低阶项（如 O(2N+N²) 简化为 O(N²)）。
- **空间换时间**：桶排序用远大于待排序数个数的存储空间换取接近 O(N) 的时间复杂度，是一种经典权衡。
- **稳定性隐含问题**：简化版桶排序把"值"和"值所携带的身份信息"分离，导致排序后无法还原对应关系，这是冒泡/快速排序（对结构体整体交换）能解决而桶排序不能解决的问题。

## 思维模型 (Mental Models)
- 当数据范围小、可预先确定上界时，用桶排序（本质计数排序）先"标记出现次数"再顺序倒出。
- 当需要排序的元素带有额外信息（如姓名+分数）且规模不大时，用冒泡排序对结构体整体交换，保持数据与身份的绑定。
- 当追求通用场景下的效率、数据规模较大时，把快速排序的每一轮当作"用一个基准数把序列劈成两半，再各自递归劈"来理解——本质是分治+二分。
- 把"桶排序"想象成 11 个（或 1001 个）编号排列的桶，每出现一个数就往对应编号的桶里插一面小旗子，最后按编号顺序数旗子即为排序结果。

## 常见陷阱 (Anti-patterns)
- 误以为简化版桶排序是万能排序：当数据范围极大（如 0~21 亿）或含小数时，桶排序完全不可行（无法申请等大的数组，也无法给小数分配整数下标）。
- 快速排序中哨兵顺序颠倒：必须先移动哨兵 j（从右向左找小于基准数的数），若先移动哨兵 i 会导致基准数归位错误，这是书中反复强调"顺序很重要"的地方。
- 混淆"趟数"与"比较次数"：冒泡排序每趟需要比较到 `n-i` 位而非 `n-1` 位，若照抄成每趟都比较到底，会做大量无意义的比较（但不影响结果正确性，只影响效率）。
- 认为快速排序最坏情况也是 O(NlogN)：实际上快速排序最坏时间复杂度会退化到 O(N²)（例如基准数每次都选到最大或最小值），只有平均复杂度是 O(NlogN)。

## 代码示例 (Code Examples)
```c
// 快速排序核心：一轮"哨兵探测 + 基准数归位"，再递归处理左右子序列
void quicksort(int left, int right)
{
    int i, j, t, temp;
    if (left > right) return;

    temp = a[left];   // 以最左边的数作为基准数
    i = left;
    j = right;
    while (i != j) {
        // 顺序很重要：必须先从右往左找
        while (a[j] >= temp && i < j) j--;
        // 再从左往右找
        while (a[i] <= temp && i < j) i++;
        if (i < j) {           // 哨兵未相遇则交换
            t = a[i]; a[i] = a[j]; a[j] = t;
        }
    }
    // 基准数归位
    a[left] = a[i];
    a[i] = temp;

    quicksort(left, i - 1);    // 递归处理左边
    quicksort(i + 1, right);   // 递归处理右边
}
```
这段代码演示了快速排序"选基准—双哨兵探测—交换—基准归位—递归左右子区间"的完整分治过程，是本章最核心也是考试最常考的手写代码。

## 讲解示例 (Worked Example)
**小哼买书**（改编自 NOIP2006 普及组第一题）：n 个同学各写一个自己想读的书的 ISBN 号（1~1000 之间），要求去重后从小到大输出，学校按此列表买书。

示例输入：10 个数 `20 40 32 67 40 20 89 300 400 15`，期望输出：8 个数 `15 20 32 40 67 89 300 400`（40 和 20 各出现了两次，需去重）。

两种解法对照本章三种算法：
1. **桶排序解法**：申请 `a[1001]`，把每个出现过的 ISBN 号对应位置标记为 1（而不是计数），最后从下标 1 遍历到 1000，凡是标记为 1 的下标直接输出——桶排序的"计数"用法天然自带去重效果。时间复杂度 O(N+M)。
2. **冒泡（或快速）排序解法**：先把 10 个数原样排序得到 `15 20 20 32 40 40 67 89 300 400`；因为相同的数排序后必然相邻，所以只需在输出时比较 `a[i]` 与 `a[i-1]`，相同则跳过、不同则输出，即可实现"边输出边去重"。若用冒泡排序整体时间复杂度是 O(N²)；若数据规模变大（如 n≤10万），冒泡排序 1 秒时间限制内跑不完（需约 10 秒），必须换成快速排序（约 170 万次比较，不到 0.002 秒）。

这道题的关键教学点：**同一问题可以用不同算法解决，但当数据规模跨越某个阈值时，算法选择直接决定"能不能在时限内做完"**，这正是本章想传达的"算法的魅力"。

## 学习要点 (Key Takeaways)
- 三种排序的时间复杂度必须记牢：桶排序 O(N+M)，冒泡排序 O(N²)，快速排序平均 O(NlogN)、最坏 O(N²)。
- 选择排序算法前先看数据范围：范围小且是整数优先桶排序；范围大或含小数则不能用桶排序。
- 冒泡排序改造成"结构体排序"（携带 name 等字段）是考试中"排序 + 保留对应信息"这类题目的标准套路。
- 快速排序哨兵顺序（先 j 后 i）、交换条件（`i<j` 才交换）、基准数最终归位方式，是手写代码题最容易失分的细节。
- 时间复杂度估算的经验值：计算机每秒约运行 10 亿次，据此可以反推给定时限下能接受的最高复杂度阶数（例如 1 秒、n=10万时 O(N²) 不可行、O(NlogN) 可行）。
- 桶排序的"计数/标记"变体可以直接用来做"去重"，这是本章向"小哼买书"过渡的关键桥梁。
