# 第8章：更多精彩算法

## 核心思想 (Core Idea)
本章围绕图论四类经典问题展开：如何以最小代价连通全图（最小生成树）、如何找出图中"一炸就散"的脆弱点/脆弱边（割点与割边），以及如何在二分图中配出最多的对子（最大匹配）——手法各异，但内核相通：都是把"局部最优选择"或"局部可达性"逐步累积成"全局最优解"。

## 引入的框架与算法 (Frameworks Introduced)

### 最小生成树 (Minimum Spanning Tree, MST)
问题原型是"镖局运镖"：要选一批道路把所有城镇连通、且总打点费最少。n个顶点的连通图最少需要n-1条边，目标就是从所有边中选出n-1条边，使其不构成回路且权值和最小。书中给出两种解法：

**Kruskal 算法（Kruskal's Algorithm）**
- 何时用：稀疏图（边数M较小）。
- 步骤：①把所有边按权值从小到大排序；②用并查集（Union-Find）记录各顶点所属集合；③依次扫描排好序的边，若该边两端点不在同一集合（加入不成环）则选中并合并两集合，否则跳过；④直到选满n-1条边。判断"两点是否已连通"若用DFS/BFS效率很低，用并查集则接近O(logN)。由 Joseph B. Kruskal 于1956年发表。时间复杂度：排序O(M log M) 加并查集操作，综合为O(M log M)。

**Prim 算法（Prim's Algorithm）**
- 何时用：朴素版适合稠密图；配堆优化后也能很好应付稀疏图。
- 步骤：①任选一个顶点加入生成树，用book[]标记已入树的顶点；②用数组dis[]记录每个非树顶点到"整棵生成树"（注意不是到某个固定源点）的当前最短距离，直连边即边权，无边则为无穷大；③每轮从dis[]中选出最小值对应的顶点纳入生成树，再以它为中间点松弛dis[]（若dis[k] > e[j][k]则更新）；④重复n-1轮直至全部顶点入树。与Dijkstra几乎同构，唯一区别是dis[]的含义从"到源点的距离"变为"到生成树的距离"，松弛时不必再加"起点到中间点"这一段。朴素实现（线性扫描找最小值）为O(N²)；用堆维护dis[]、邻接表存图可降到O(M log M)。历史上先由捷克数学家 Vojtěch Jarník 于1930年提出，1957年 Robert C. Prim 独立再发现，1959年 Dijkstra 又独立发现，故该算法也称DJP算法/Jarník算法。

### 割点 (Cut Vertex / Articulation Point)
问题原型是"重要城市"：轰炸哪座城市能让敌方城市间彻底失联——删除某顶点后图不再连通，该顶点即为割点。朴素解法是逐一删点后跑DFS/BFS验连通性，O(N(N+M))，太慢。高效解法（Tarjan）：①从任意点出发做一次DFS，得到一棵DFS生成树，并给每个顶点按访问顺序标注"时间戳"num[]；②维护low[]数组，low[v]表示v不经过其DFS树父亲、仅靠树边与回边所能回溯到的最早时间戳；③对树上某点x的孩子v，若low[v] >= num[x]，说明v这棵子树绕不回x的祖先，则x是割点；④特例：若x是DFS根，则根是割点当且仅当它在DFS树中有≥2个孩子。由 Robert E. Tarjan 发明，邻接表实现下时间复杂度O(N+M)。

### 割边/桥 (Cut Edge / Bridge)
问题原型是"关键道路"：删除哪条道路会让图断开。解法与割点算法几乎相同，仅改一个符号：判断条件由"low[v] >= num[x]"改为"low[v] > num[x]"（去掉等号）——等号成立时v仍能绕回x本身（父亲），说明该边不是桥；严格大于时v连x都回不去了，该边才是桥。同样由 Robert E. Tarjan 发明，邻接表实现下O(N+M)。

### 二分图最大匹配 (Maximum Bipartite Matching)
问题原型是"我要做月老"：过山车相邻座位安排，女生集合与男生集合，只有互相认识的一对才能配对坐一起，求最多能配对多少人。**二分图（Bipartite Graph）**定义：顶点可分成两个集合X、Y，每条边的两端恰好一个属于X一个属于Y（集合内部无边）。朴素解法是枚举所有匹配方案取最优，指数级不可行。**匈牙利算法（Hungarian Algorithm）**：①对每个尚未匹配的X顶点u，逐条尝试其边(u,v)；②若v尚未匹配，直接配对成功，得到一条增广路（Augmenting Path）；③若v已匹配给某u'，则递归尝试让u'去找替补（对u'的其余边重复本过程），若u'找到替补，则v腾出来配给u，这也算一条增广路（只是路径更长，形成"连锁反应"）；④若u的所有边都尝试失败，则放弃该点；⑤每找到一条增广路配对数+1，直到对所有X顶点都尝试完仍找不到新增广路，当前匹配即为最大匹配。由 Jack Edmonds 给出；目前已知最快的二分图最大匹配算法由 John E. Hopcroft 提出（Hopcroft-Karp算法）。时间复杂度：二分图最多有n/2条增广路，每次搜索最坏遍历所有边O(M)，故总复杂度O(N·M)。

## 关键概念 (Key Concepts)
- **最小生成树 (MST)**：无向连通图中用n-1条边连接全部n个顶点、且边权和最小的子图。
- **并查集 (Union-Find)**：Kruskal判断两点是否已连通的核心数据结构，承自上一章。
- **时间戳 (num[]，DFS discovery time)**：DFS遍历中每个顶点被首次访问的先后顺序编号。
- **low值 (low-link value)**：某顶点不经过其DFS树父亲、仅靠树边+回边所能回溯到的最早时间戳。
- **割点/割顶 (Cut Vertex)**：删除后使连通图变为不连通的顶点。
- **割边/桥 (Bridge)**：删除后使连通图变为不连通的边。
- **二分图 (Bipartite Graph)**：顶点可二染色、同色顶点间无边的图。
- **增广路 (Augmenting Path)**：一条起点和终点都是未匹配点的路径，沿路径翻转匹配可使匹配数+1。
- **匹配 (Matching)**：二分图中一组两两不共享端点的边集合。

## 思维模型 (Mental Models)
- 当需要"以最小代价连通全图"时，用Kruskal（从边的角度贪心，配并查集判环）或Prim（从点的角度扩张，配dis[]松弛）——把MST问题当作"排序+并查集"或"Dijkstra的变形"来理解。
- 当要判断"图的脆弱性"（去掉谁/哪条边会让图断开）时，把DFS遍历树当骨架，用num[]记录"出生顺序"、low[]记录"能跳多远回头"，割点/割边就是那些"回不了头"的临界点。
- 把二分图最大匹配理解成"抢座位的连锁退让"：新来者先占位，占不到就让原主人去找替补，替补链条走通了大家就都挪一位——这正是增广路的直觉。
- Prim和Dijkstra是双胞胎：Dijkstra的dis[]是"到固定源点的距离"，Prim的dis[]是"到当前生成树的距离"，公式几乎一样，只是少加了一段"起点到中间点"的路程。

## 常见陷阱 (Anti-patterns)
- 把Prim的dis[]误当成Dijkstra语义（到1号顶点的最短路径），在松弛时错误地做"dis[j]+边权"的加法——Prim的dis[]只需直接取边权本身，因为目标是离整棵生成树最近，而非离固定源点最近。
- 求割点/割边时用邻接矩阵存图虽代码简洁，却使整体复杂度退化为O(N²)甚至O(N(N+M))，丢掉了Tarjan算法本该有的O(N+M)线性优势——书中为了讲解清晰特意如此，实战必须换邻接表。
- 判断割边时忘记去掉判断条件里的等号（仍写成low[v]>=num[x]），会把"能绕回父亲但绕不回更早祖先"的边也误判为桥。
- 二分图匹配DFS中若忘记设置/重置book[]（防止本轮递归重复访问同一顶点），会导致死循环或"连锁反应"传播错误。

## 代码示例 (Code Examples)
```c
/* Kruskal 算法核心骨架（配合并查集，边已按权值从小到大排序） */
int f[N]; // 并查集数组

int getf(int v) {                 // 找祖先（带路径压缩）
    if (f[v] == v) return v;
    return f[v] = getf(f[v]);
}

int merge(int u, int v) {         // 尝试合并u、v所在集合
    int t1 = getf(u), t2 = getf(v);
    if (t1 != t2) { f[t2] = t1; return 1; }  // 不同集合：合并成功，不成环
    return 0;                                // 同一集合：加入会成环，跳过
}

// 主循环：边已排序，依次尝试
for (i = 1; i <= m && count < n - 1; i++) {
    if (merge(e[i].u, e[i].v)) {
        count++;
        sum += e[i].w;
    }
}
```
这段代码展示了Kruskal算法的核心：排序后的边只需一次线性扫描，配合并查集的合并/查询就能在接近O(M log N)的时间内判断"是否成环"，从而选出最小生成树的n-1条边。

## 参考表格 (Reference Tables)
| 算法 | 适用场景 | 时间复杂度 | 关键数据结构 |
|---|---|---|---|
| Kruskal | 稀疏图 | O(M log M) | 并查集 + 排序 |
| Prim（朴素） | 稠密图 | O(N²) | dis[]数组线性扫描 |
| Prim（堆优化） | 稀疏图 | O(M log M) | 二叉堆 + 邻接表 |
| 割点/割边（Tarjan） | 通用 | O(N+M)（邻接表）/ O(N(N+M))（朴素删点法） | DFS + num[]/low[] |
| 二分图最大匹配（匈牙利算法） | 通用 | O(N·M) | DFS增广路 + match[]/book[] |
| 二分图最大匹配（Hopcroft-Karp） | 通用，更优 | O(√N · M) | BFS分层 + 多路增广 |

## 讲解示例 (Worked Example)
**割点追踪（重要城市问题）**：从1号顶点开始DFS，得到时间戳 num[1]=1、num[2]=3……继续深入到5号顶点时 num[5]=5，此时只剩6号顶点未访问。6号顶点存在一条回边能直接跳回1号顶点，所以 low[6]=1。因为 low[6]=1 < num[5]=5，说明5号顶点的孩子（6号）绕得过5号回到更早的祖先，所以**5号不是割点**。继续回溯到2号顶点：它的孩子5号的low值综合了自己孩子的low和自身的回边，结果 low[5]=3，恰好等于 num[2]=3——意味着5号这整棵子树无论怎么绕，最多只能绕回2号本身，绕不到2号更早的祖先（1号），所以**2号是割点**：炸掉2号，之后4、5、6号城市就再也联系不上1、3号城市了。

**二分图增广路追踪（过山车配对）**：认识关系为 女1-男1、女1-男2、女2-男2、女2-男3、女3-男1。先按顺序尝试：女1配男1（成功），女2配男2（成功），轮到女3，她只认识男1，但男1已配给女1——于是触发"连锁反应"：男1让女1去问问看能否换一个；女1转而去问男2，男2已配给女2；女2再去问男3，男3还空着，女2便与男3配对；男2因此腾出来给女1；男1因此腾出来给女3。最终配对结果：女1-男2、女2-男3、女3-男1，配对数从2对增加到3对——这正是一条完整的增广路径。

## 学习要点 (Key Takeaways)
- MST两大算法的选择依据：图稀疏（M接近N）选Kruskal（排序+并查集）；图稠密（M接近N²）选朴素Prim；若图稀疏又想用Prim，必须配堆优化+邻接表才能达到O(M log M)。
- 一个连通图的最小生成树不一定唯一（存在权值相同的边参与选择时会有多解），但最小生成树的总权值一定唯一。
- 割点、割边算法基于同一棵DFS生成树和num[]/low[]，唯一差别是判断条件的等号是否保留（>=为割点，>为割边）——考试中最容易混淆的地方。
- 割点判断对根节点是特例：非根节点看low[v]>=num[x]，根节点则看DFS树中孩子数是否≥2。
- 二分图最大匹配的正确性依赖：找不到新的增广路即为当前匹配已是最大匹配（可反证）。
- 匈牙利算法每轮为每个未匹配点跑DFS前，必须重置book[]数组，否则会漏掉本应可行的连锁重匹配路径。

## 关联章节 (Connects To)
- 与前一章的并查集（Union-Find）直接衔接：Kruskal算法的成环判断完全复用并查集的合并与查询操作。
- 与图的最短路径算法（如Dijkstra）呼应：Prim算法的dis[]松弛写法与Dijkstra几乎同源，理解一个便能类比理解另一个。
- 割点、割边、二分图最大匹配都建立在图的深度优先遍历（DFS）之上，是DFS在"求连通分量""求路径"之外更进阶的应用，呼应本书前面讲DFS/BFS的章节。
- 二分图匹配是更一般的网络流、指派问题的入门形态，若继续拓展学习图论，这是理解最大流-最小割定理的自然起点。
