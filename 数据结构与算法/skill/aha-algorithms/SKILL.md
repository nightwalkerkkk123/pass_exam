## 核心框架与思维模型 (Core Frameworks & Mental Models)

**枚举（Enumeration）与标记法（book数组）— 最朴素但最不容易写错的武器**
枚举的核心不是技巧，是"不重不漏地列出所有可能，再挑出满足条件的"。关键工具是标记数组 `book[]`，记录哪些位置/数字已被占用，避免重复选取同一元素。数据规模小时（如全排列 n≤10），枚举往往比硬想复杂算法更快写对——先想能不能枚举，再想能不能优化。

**DFS 与回溯（Backtracking）— 一条路走到黑，撞墙就回头**
通用模板四步：判断终止条件 → 循环枚举当前决策点的所有分支 → 标记该分支已使用并递归深入 → 撤销标记（回溯）。回溯是 DFS 能穷举全部方案、不重不漏的关键，漏写撤销标记是最常见的 bug。DFS 也能求最短路径（穷举所有路径取最小），但不具备"第一次到达即最优"的性质，效率远不如 BFS。

**BFS 与 Flood Fill — 水波纹式扩散，先到即最优**
BFS 用队列层层扩展，保证第一次抵达终点时的步数就是最短路径——这是它相对 DFS 求最短路径的决定性优势。Flood Fill（染色法）是 DFS/BFS 的直接应用：以一个种子点向四周扩展染色，统计连通区域面积或连通分量（岛屿）个数，是 Windows 画图"油漆桶"背后的算法原型。

**最短路径四件套 — 按"单源/多源"和"能否有负权边"两个维度选算法**
这是全书图论部分的重头戏。多源（任意两点对）用 **Floyd-Warshall**——三重循环、核心代码只有5行，本质是动态规划"逐步放开可经过的中转顶点"；O(N³)，不能处理负权回路。单源（一个源点到全部点）且边权非负，用 **Dijkstra**——贪心地每轮从"未确定"集合里选 dis 最小的点确定，再用它的边去松弛邻居；O(N²)，堆优化后 O((M+N)logN)。一旦出现负权边，Dijkstra 的贪心假设（已确定的点不会再被更短路径刷新）被打破，必须退回暴力松弛全部边的 **Bellman-Ford**（循环 n-1 轮，O(NM)），或其队列优化版 **SPFA**（只对"估计值刚被更新"的顶点出边做松弛，平均更快，同样可能退化）。四种算法有一个共同的原子操作——"松弛"：`if (dis[v] > dis[u]+w) dis[v] = dis[u]+w;`，几乎所有变体都是在决定"什么时候、对谁做这句话"。负权回路的判定：Bellman-Ford 跑满 n-1 轮后仍能松弛、或 SPFA 中某点入队超过 n 次，都说明存在负权回路（此时最短路径根本不存在）。

**树、堆与并查集 — 同一种"层级结构"驱动两类完全不同的问题**
树是"不包含回路的连通无向图"：n 个结点恰好 n-1 条边，任意两点间路径唯一。完全二叉树可以完全不用指针，直接用数组存储——父 k 的左右儿子是 2k、2k+1，儿子 x 的父是 x/2（整数除法）。**堆**是满足"父 ≤（或 ≥）所有子结点"的完全二叉树，专门加速"反复取最值+插入新值"这类操作（优先队列）：线性扫描 O(N)，堆 O(logN)。两个核心操作 siftdown（自上而下修复，用于删除堆顶/建堆）和 siftup（自下而上修复，用于插入新元素）都是 O(logN)；建堆首选"从最后一个非叶结点倒序做 siftdown"的 O(N) 方法，而不是逐个插入的 O(NlogN) 方法。**并查集**（Union-Find）解决完全不同的问题——"频繁判断两个元素是否同组、并支持合并分组"：用 `f[i]` 记录 i 的"上级"，查找 `getf` 必须递归到底找到根（不能只看直接父结点！），合并时把一个根挂到另一个根下；**路径压缩**（查找时顺手把沿途结点直接挂到根下）能把后续查找摊薄到近似 O(1)。并查集是下一节 Kruskal 判断"加边是否成环"的必备前置知识。

**最小生成树（MST）— 把"连通全图"的代价降到最低**
n 个顶点的连通图最少需要 n-1 条边；目标是从所有边中选 n-1 条，不成环且权值和最小。**Kruskal** 从边的角度贪心：把所有边按权值排序，依次尝试加边，用并查集判断"两端点是否已连通"（成环则跳过），适合稀疏图，O(MlogM)。**Prim** 从点的角度扩张：维护 `dis[]` 记录每个非树顶点"到整棵生成树"（不是到固定源点）的最短距离，每轮选 dis 最小的点纳入生成树并用它松弛其余点，适合稠密图，朴素 O(N²)、堆优化后 O(MlogM)。Prim 和 Dijkstra 几乎是双胞胎，唯一区别是 dis[] 的语义：一个是"到生成树"、一个是"到源点"——松弛时 Prim 不用再加"起点到中间点"那一段。

**割点与割边（Tarjan）— 找出图里"一炸就散"的临界点/临界边**
朴素解法是逐一删点/删边后跑连通性检查，O(N(N+M)) 太慢。Tarjan 的高效解法建立在一次 DFS 生成树上：`num[]` 记录每个顶点被访问的时间戳，`low[]` 记录该顶点不经过其 DFS 树父亲、仅靠树边+回边所能回溯到的最早时间戳。对树上某点 x 的孩子 v，若 `low[v] >= num[x]`，说明 v 这棵子树绕不回 x 的祖先，x 就是割点；判割边时把等号去掉（`low[v] > num[x]`），因为等号成立时 v 仍能绕回 x 本身（父亲），该边不算桥——**这一个等号的差别是考试最容易混淆的地方**。DFS 根是特例：根是割点当且仅当它在 DFS 树中有 ≥2 个孩子。

**二分图最大匹配 — 增广路的"连锁退让"**
二分图的顶点能分成两个集合，边只连接两个集合之间的点。**匈牙利算法**（Hungarian Algorithm）靠增广路：为每个未匹配点尝试它的边，若对方未匹配就直接配对；若对方已匹配，就让"对方现在的搭档"递归去找替补——这条替补链条如果最终走通，大家就都挪一位，配对数 +1。找不到新的增广路，当前匹配即为最大匹配。O(N·M)。递归前必须重置 `book[]`，否则会漏掉本应可行的连锁重匹配。

**面试常考的"降维打击"技巧包（ch09）**
面对"暴力能做但太慢"的题目，书里给出一条通用升级路径：暴力枚举/计数 → 排序后扫描（O(NlogN)）→ 利用问题特殊结构直接定位（如"数组中出现次数超过一半的数"排序后必压中位置 n/2）→ 计数数组/离散化（把值域压缩进可开数组的范围）→ 快速选择 Quickselect（基于快排 partition 只递归一侧，平均 O(N) 求第 k 大/小）→ 摩尔投票法（Boyer-Moore Voting，一个候选值+计数器扫一遍，O(N) 时间 O(1) 空间）。这条"从暴力到最优"的升级路径本身就是应试时的思考框架，而不是单独一个算法。
