## B. 核心决策：最短路 & MST（重点）

| 场景 | 选 | 因为 |
|---|---|---|
| 单源+非负权 | Dijkstra | 贪心可证正确，优于Floyd |
| 单源+可能负权 | Bellman-Ford/SPFA | 负权打破贪心假设 |
| 多源全对N小(≲几百) | Floyd | 五行代码+耐负权 |
| 判负权环 | BF看n-1轮后可否再松弛/SPFA看入队>n次 |
| MST+稀疏(M≈N) | Kruskal | 并查集O(MlogM)<朴素Prim O(N²) |
| MST+稠密(M≈N²) | Prim朴素 | 免排序,每轮O(N)扫描更划算 |

速记：BFS→最短步；DFS→全路径(需撤book)；堆→反复取最值；book[]→存在性判断。

## C. 图算法决策树

```
连通性→Flood Fill
最短路/MST的选择→见B(单源非负/负权/多源；稀疏/稠密)
割点/割边→Tarjan+邻接表(割边条件去等号low[v]>num[x])
二分图匹配→匈牙利(每轮重置book[])
```

## D. 阈值/经验法则

- N≥10⁴+限时1s→快排非冒泡；值域小整数→桶排序。
- Floyd仅N≲300~500，更大用单源算法轮跑。
- 建堆用siftdown自底向上O(N)，不逐个siftup。
- 割点/割边必须邻接表；值域大元素少先离散化。

## E. 易错指纹

| 症状 | 病灶 |
|---|---|
| 快排基准数归位错误 | 应先移j(右→左找小于基准数)后移i |
| DFS/BFS漏分支或死循环 | 回溯忘记book[i]=0／忘记head++ |
| 割点割边全错/割边混进割点 | 存图用邻接矩阵(应邻接表)／条件多写等号(应严格>) |
| 二分图匹配少几对 | 忘记每轮DFS前重置book[] |
| 并查集同组判断时好时坏 | 只比较直接父节点未递归找根 |
