第 17 课:树、堆与并查集 —— 组织架构图,以及"帮派认老大"的最优实现

上一课(第16课)学了 Floyd-Warshall、Bellman-Ford、SPFA 三种最短路径算法,和第7课的 Dijkstra 一起凑成了完整的最短路径工具箱。这一课换个方向:"树"这种层级结构,以及它在两个经典问题里的应用—— 堆(优先队列)和并查集(分组合并)。本课信息量较大,但重点只有一个:并查集;树和堆的部分概念性带过即可。约 20 分钟。
直觉类比:公司组织架构图,以及"想知道你是不是我们这一伙的,先问你老大是谁"。 公司组织架构图、家族族谱,说的是同一种结构:一个"上级"可以带很多"下级",但每个"下级"只认一个直接上级—— 这就是树。这一课真正要练的,是树的一种特殊用法:警察抓到一批嫌疑人,要判断"这几个人是不是一伙的", 靠谱的办法不是问"你俩关系近不近",而是一路往上问:"你的老大是谁?他的老大又是谁?一直问到没有老大 (自己就是老大)为止。" 两个人只要能一路问到同一个"总老大",就认定是同一伙——这正是 并查集(Union-Find)的核心直觉:靠"认祖归宗"来判断归属、合并分组。

一、树与二叉树 —— 30 秒复习

树是任意两个结点之间有且仅有一条路径的无向图,等价于"不包含回路的连通无向图"。由这一条"无环"性质, 可以直接推出:n 个结点的树恰好有 n-1 条边;树里任加一条边,必然构成回路。术语上,没有父结点的叫根结点, 没有子结点的叫叶结点,其余叫内部结点;从根往下数,根算第 1 层,这个层数就是深度。

二叉树是每个结点最多两个子树(左儿子、右儿子)的树。满二叉树要求每个内部结点都有两个儿子; 完全二叉树允许最后一层不满,但必须从左到右连续排列,不能"右边有、左边缺"。这部分遍历算法 第一部分《算法图解》第11章已经讲过,这里不重复,只补一条后面"堆"要用到的关键规则: 完全二叉树可以直接用一维数组存,不需要指针——父结点编号 k 的左儿子是 2k, 右儿子是 2k+1;反过来,儿子编号 x 的父结点是 x/2(C 语言整数除法自动向下取整)。 书中代码统一从下标 1 开始存,这条规则才成立;如果改成从 0 开始存,公式要跟着变成 2k+1/2k+2, 两套下标体系混用是常见出错点。

二、堆(优先队列) —— 不用整体排序,也能秒取最值

堆是一种特殊的完全二叉树:最小堆要求每个父结点的值都不大于子结点,最大堆则相反。 它解决的问题是:如果要反复"取出当前最大/最小值,再塞入新值",普通数组线性扫描找最值单次是 O(N), 做 N 次就退化成 O(N²);换成堆,单次操作能压到 O(log N)。

堆靠两个局部修复动作维持自己的性质,都是 O(log N):向下调整(siftdown)—— 堆顶被替换后不满足性质了,和左右儿子中较小者比较、交换,沿路径一直往下修;向上调整(siftup)—— 插入新元素放到数组末尾,再和父结点比较、交换,一直往上修。建堆时,逐个插入再 siftup 整体是 O(N log N);更优的做法是把 N 个数直接摆进数组,再从最后一个非叶结点(编号 N/2)开始倒着往前, 对每个结点做一次 siftdown,整体只要 O(N)。堆排序就是建好堆后,不断把堆顶和末尾元素交换、 堆缩小一格、再对新堆顶 siftdown,整体 O(N log N)。

一个直观的数字对比:14 个数反复"取最小值+插入新值",如果每次都线性扫描,做 1 亿次操作大约要 1 亿的平方次运算(按每秒 10 亿次估算,约 115 天);换成最小堆,每次取最小只要 O(log N), 1 亿次操作总共约 27 亿次运算,只要 2.7 秒——这就是"用对数据结构"的威力。

三、并查集 —— 本课重点

并查集(Union-Find,也叫不相交集 Disjoint Set)维护的是"分组归属"这件事:一开始每个元素各自成一伙 (各自是一棵只有自己一个结点的树),之后不断根据"这两个元素是一伙的"这类信息,把树合并成更大的树。 它专门解决"频繁判断两个元素是否已在同一分组、并支持合并两个分组"这类问题——判断连通性、分组归类、 后续第18课 Kruskal 最小生成树判环,都要用它。

查找(find):必须一路问到"根",不能只看直接上级

用一个数组 parent[i] 记录 i 的"直接上级",初始时 parent[i] = i(每个人的老大是自己)。 查找 find(v) 就是递归地沿着 parent 往上走,直到走到 parent[x] == x 的那个结点——这就是这个集合的 根(代表元)。判断两个元素是否同一集合,必须比较两者各自的根,而不能只看直接父结点是否相同—— 这是全章最容易踩的坑,后面的常见陷阱和自测题都会反复考这一点。

路径压缩(Path Compression):顺手把沿途结点都挂到根下

如果每次 find 都老老实实沿链条走一遍,树合并多了会越长越"瘦",最坏情况退化成一条链,单次 find 变成 O(N)。路径压缩的做法是:在 find 递归返回的同时,把沿途经过的每个结点的 parent 直接改成指向最终的根, 写法就是 parent[v] = find(parent[v]); return parent[v];。这不会增加这一次查找的开销,却让后面所有 经过这些结点的查找都直接"一步到根",近似摊薄到 O(1)。

合并(union):找到两个根,不同才接起来

合并 union(v, u) 分别对 v、u 做 find,得到两个根 t1、t2; 只有 t1 ≠ t2 时才需要真正合并——把 parent[t2] = t1(书里称"靠左"法则:右边的根归顺到左边的根下)。 比如警方线索里出现"5 号和 2 号是同伙",如果 2 号的老大是 1 号、5 号还是自己的老大,"靠左"就是让 5 号那一支 归顺 1 号那一支,把 parent[1] 改成指向 5——这也提醒我们:判断和合并都必须先找到根, 中间层级(谁是谁的直接上级)会随着合并顺序不断变化,不能只看一层。

操作无路径压缩有路径压缩
初始化(建 n 个各自独立的集合)O(N)O(N)
查找 find / 判断是否同一集合最坏 O(N)(树可能退化成一条链)近似 O(1)(均摊)
合并 union(内部先做两次 find)最坏 O(N)近似 O(1)(均摊)

C 代码实现(带边界条件详解)

注意函数不能叫 union——那是 C 语言的保留关键字(联合体),这里改名 union_set。

#define MAXN 1000
int parent[MAXN];                  // 下标 1~n,存的是"我的直接上级是谁"

void init(int n) {
    for (int i = 1; i <= n; i++)
        parent[i] = i;              // ① 每个人初始时老大是自己,不是 0
}

int find(int v) {                   // 查找 v 所在集合的根(代表元)
    if (parent[v] == v)
        return v;                   // 已经是根,直接返回
    parent[v] = find(parent[v]);    // ② 路径压缩:顺手把沿途结点都挂到根下
    return parent[v];
}

void union_set(int v, int u) {
    int t1 = find(v), t2 = find(u);
    if (t1 != t2) {                 // ③ 必须先判断两个根是否已经相同
        parent[t2] = t1;            // "靠左"法则:右边的根归顺左边的根
    }
}

三处最容易在考试里写错、丢分的地方:

  1. ① init 里 parent[i] = i 不能漏。 如果这个数组是全局数组、又忘了调用 init,C 语言会把它默认清零,所有 parent[i] 都变成 0。这时 find(3) 会一路走到 parent[0] == 0 才停下——在还没做任何 union 之前,所有元素的 find 结果都会收敛到同一个"根" 0, 程序会误判"所有人从一开始就是同一伙",这是比"忘记初始化数组"更隐蔽的一种事故。
  2. ② find 如果不做路径压缩,单次查找可能退化到 O(N)。 反复合并之后,树可能长成一条长链 (比如每次都是新结点挂在旧根下面),这时不压缩路径,每次 find 都要从叶子一路走到根;考试里如果要求 "分析最坏情况复杂度",漏掉这一条会把 O(1) 均摊和 O(N) 最坏混为一谈。
  3. ③ union 前必须分别 find 出两个根,再比较 t1 != t2。 如果图省事写成 parent[v] = u;(不经过 find,直接把原始结点 v 挂到 u 下面),一旦 v、u 已经在同一个集合里 (比如 u 本来就是 v 的子孙),这一步会把 v 原来的父指针连接砍断、还可能连出一个环,导致后续 find 顺着环一直递归下去,程序死循环。"合并"操作的对象永远是两个根,不是两个原始元素。
常见陷阱:
  • 误以为只要两个结点的直接父结点相同就算同一伙——必须一路 find 到根(最终祖先),中间层级会随合并顺序变化,只看一层会算错。
  • 建堆时逐个插入再 siftup,虽然结果正确但是 O(N log N),不是本章推荐的最优做法;应该优先用"从最后一个非叶结点(N/2)倒序 siftdown"的 O(N) 方法建堆。
  • 完全二叉树的数组下标如果从 0 开始存,2k/2k+1 的父子编号公式要相应变成 2k+1/2k+2;书中代码统一从下标 1 开始存,混用两套下标体系是常见出错点。
