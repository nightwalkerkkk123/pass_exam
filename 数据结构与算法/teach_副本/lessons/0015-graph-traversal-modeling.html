第 15 课 · 图的遍历与建模

上一课(第14课)学了 DFS 用递归/栈实现、BFS 用队列实现、以及 Flood Fill 染色法——这节课把同一套"遍历"框架第一次用在"图"这种数据结构上:先学会把图存进矩阵,再用 DFS/BFS 分别解决"求最短路径"和"求最少中转"两类不同问题。约 20 分钟。
从"地图"到"矩阵":图到底在存什么? 想象一张城市地图:每个城市是一个顶点(vertex), 城市之间的道路是边(edge)。地图本身是一堆线条和圆点,计算机没法直接"看懂"它——但只要你能回答 "任意两个城市之间是否有一条直接的路(以及这条路有多长)",这张地图就能被完整翻译成一张二维表格: 邻接矩阵(Adjacency Matrix)。这节课要做的事,就是把图这种看起来很自由的结构, 套进你已经很熟悉的"二维数组"里。
1号城市2号城市3号城市
1号城市05∞
2号城市503
3号城市∞30

这是一张无向图的邻接矩阵(1↔2 距离5,2↔3 距离3,1和3之间没有直接的路记为∞)—— 注意它沿对角线对称,这正是"无向"的数学体现;对角线本身是0,表示"自己到自己不算一段路程"。

一、邻接矩阵 —— 怎么存图

用二维数组 e[i][j] 表示顶点 i 到顶点 j 是否有边(或者边的权值)。 几条约定:

  • 有向图 vs 无向图:边有方向叫有向图(Directed Graph),此时 e[i][j] 和 e[j][i] 可以不同;边没有方向叫无向图(Undirected Graph),意味着"i 能到 j,j 也一定能到 i",此时邻接矩阵沿主对角线对称。
  • 自己到自己(i==j):通常设为 0,表示"不算一段路程"。
  • 不存在的边:设为一个足够大的数(书中用 99999999 代表正无穷),表示"不可达",避免用特殊值 处理带来的额外分支判断。
  • 适用场景:顶点数不太大、需要频繁判断"i 到 j 是否有边"时,邻接矩阵是最直观的入门存法——查询是 O(1),代价是空间恒为 O(n²),图很稀疏时会浪费大量空间(后续章节会讲更省空间的邻接表)。

二、用 DFS 暴力求最短路径

本章"城市地图"一节的做法:从起点出发,用 DFS 递归地尝试所有不重复经过已在路径中的顶点的走法; 每走一步就累加当前路程 dis;一旦到达终点,就用全局变量 min 更新已发现的最小路程; 回溯时一定要把 book[j] 重新置 0(取消标记),否则会漏掉其他分支本该走到的路径。 可以加剪枝:如果 dis 已经超过当前 min,立即返回,不再往下搜。

书中给出的例子:小哼从1号城市走到5号城市,枚举出的全部路径是 1→2→3→4→5(路程14)、1→3→2→5(路程9)、1→5(路程10)—— 三条比较后最短是 1→3→2→5,长度9。把同一张地图改成无向图重新计算,因为边可以双向走,反而找到了 更短的 1→3→5(长度7)——这说明"图是有向还是无向"直接决定了可达路径集合,进而影响最短路径的结果。

常见陷阱:把 DFS 暴力枚举当成"最短路径算法"用在大图上。 这种做法本质是枚举所有路径,时间复杂度 随顶点数指数级增长,只适合本章教学演示的小规模例子;顶点一多,路径条数会爆炸式膨胀, 程序实际跑不完。真正处理大规模最短路径问题,要用下一课的 Floyd / Dijkstra / Bellman-Ford。 另外两个同样容易在考试里被抓包的细节:DFS 忘记回溯(退出某个分支前忘了把 book[j] 改回 0, 会导致其他分支误以为该顶点"已经走过",漏掉本该存在的路径)和有向图/无向图初始化搞混 (无向图要同时给 e[a][b] 和 e[b][a] 赋值,只赋一侧就变成有向图)。

三、用 BFS 求最少中转次数

本章"最少转机"一节把同一套邻接矩阵换成 BFS:所有边权重相同(这里是"一条航线算一次转机"), 只关心最少经过几条边到达终点,不关心具体走了哪条路径。做法:给队列中每个元素额外附带一个字段 (书中用结构体 struct note{x, s},x 是城市编号,s 是到达该城市已经历的转机次数); 每次扩展新顶点时,令其转机次数等于当前顶点转机次数 +1;一旦扩展到目标城市就立刻停止, 此时该顶点携带的次数即为全局最少值。

书中例子:小哼和小哈从1号城市飞到5号城市,BFS 从1号城市入队开始逐层扩展——第一层能到达2、3号城市; 从2号城市能再扩展到3、4号城市(3号已入队,只需新入队4号);从3号城市能扩展到4、5号城市(4号已入队, 只需新入队5号)。5号城市是在"从2号、3号城市继续扩展"这一层第一次被加入队列的——也就是说,从1号出发, 只需要经停恰好一个中间城市就能到达5号,这一步数正是全局最少转机数,不需要再比较其他走法。 这正体现了 BFS"逐层扩散、第一次抵达即最优"的核心性质。

DFS 还是 BFS,怎么选? 边权相同、只求"最少经过几条边/几个中转点"时,优先用 BFS——它按"距起点的层数" 逐层扩展,第一次摸到目标的那一层保证就是最优解,不需要像 DFS 那样枚举完所有路径再比较。 当问题需要枚举完整路径、结合权重做穷举,或者天然是"选择—递归—回溯"结构时,才轮到 DFS。

四、三种做法的适用场景对比

方法解决什么问题时间/空间量级局限
邻接矩阵存图快速查询"i 到 j 是否有边/边权" 空间 O(n²),单次查询 O(1) 图很稀疏时浪费空间(后续讲邻接表)
DFS 暴力求最短路径枚举起点到终点的所有路径,取总路程最小的一条 路径数随顶点数指数级增长 只适合小图教学演示,大图会跑不动
BFS 求最少中转边权相同时,求起点到终点最少经过几条边 每个顶点/边最多入队处理一次,约 O(n+m) 只对"边权相同"的问题成立,边权不同就不能直接用

C 代码:建图(邻接矩阵初始化 + 读入边,带边界条件详解)

#include <stdio.h>
#define INF 99999999
int e[101][101];     // 邻接矩阵,顶点编号按 1~n 使用,数组要开到 n+1 才够
int n, m;             // n 个顶点,m 条边

int main(void) {
    int i, j, a, b, w;
    scanf("%d %d", &n, &m);

    for (i = 1; i /*①*/ <= n; i++) {
        for (j = 1; j <= n; j++) {
            if (i == j) e[i][j] = 0;     // ② 对角线:先按"不算路径"处理成 0
            else e[i][j] = INF;          //    不确定的边先当成"不可达"
        }
    }

    for (i = 1; i <= m; i++) {
        scanf("%d %d %d", &a, &b, &w);
        e[a][b] = w;                     // a 到 b 这条边
        e[b][a] = w;                      // ③ 无向图:漏了这一行,图就变成"单向"的了
    }
    return 0;
}

三处最容易在考试里写错、丢分的地方:

  1. ① 顶点编号从 0 还是 1 开始,决定数组下标怎么用。 如果题目顶点编号是 1~n, 数组要开 e[n+1][n+1] 或以上,循环也要写成 for (i = 1; i <= n; i++); 如果你混用 0-indexed 的循环写法(for (i = 0; i < n; i++))却仍按 1~n 读入顶点编号, 要么漏掉编号为 n 的顶点,要么把不存在的"0号顶点"也纳入计算,读到脏数据。
  2. ② 对角线 e[i][i] 初始化成 0 还是无穷大,要看题目定义,不能想当然。 本章邻接矩阵约定"自己到自己"记为0(不算作路径),但如果题目问的是"是否存在从 i 出发经过若干步骤 回到 i 的环",对角线就不能默认成"可达的0",需要按题意重新判断,直接套用同一种初始化方式容易出错。
  3. ③ 无向图必须对称赋值 e[a][b] 和 e[b][a],漏掉一个方向就变成有向图。 这是本章"常见陷阱"里明确点出的坑:无向图意味着 i 能到 j、j 也能到 i;只写了 e[a][b]=w 却漏了 e[b][a]=w,DFS/BFS 遍历时会出现"能从 a 走到 b,却不能从 b 走回 a"的错误结果, 而且程序不会报错,只会在某些测试用例里悄悄算错答案。

Q1. 一张无向图有4个顶点(编号1~4,顶点编号从1开始),边为:1-2、1-3、2-4、3-4(边权都看作1,没有列出的边记为无穷大)。这张图的邻接矩阵里,e[2][3] 应该填什么?
Q2. 两个任务:①判断从武汉到北京最少需要经过几个城市(每条道路的权重都看成相同);②在一张不大的城市地图上,求武汉到北京总路程最短的一条具体路线(道路长度各不相同)。分别该用什么做法?
Q3. 找茬题:下面这段"DFS 暴力求最短路径"的代码有一处遗漏,选出问题所在。
void dfs(int cur, int dis)
{
    int j;
    if (cur == n) {                 // 到达终点(第 n 号城市)
        if (dis < min) min = dis;   // 更新已知的最短路程
        return;
    }
    for (j = 1; j <= n; j++) {
        if (e[cur][j] < INF && book[j] == 0) {
            book[j] = 1;              // 标记已访问
            dfs(j, dis + e[cur][j]);  // 继续往下走
        }                              // ← 这里少了一行
    }
}
▸ 看这个坑会怎么漏路径

假设 1 号顶点同时能到 2 号和 3 号,2 号也能到 3 号。DFS 先走 1→2,标记 book[2]=1, 再走 2→3,标记 book[3]=1,一路走到终点后回溯。如果回溯时没有把 book[3] 和 book[2] 改回 0,那么当外层 for 循环继续尝试"从1号直接走 1→3"这条分支时, 会发现 book[3] 还是 1(误以为已经访问过),于是直接跳过——这条本该存在的路径 1→3 就这样被漏掉了,而程序不会报错,只是悄悄少枚举了一条路径,可能导致算出的"最短路径"不是真正的最短。
