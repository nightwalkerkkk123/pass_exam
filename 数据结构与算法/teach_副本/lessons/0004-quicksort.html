第 4 课:快速排序 —— 同一个算法,平均情况和最坏情况能差一个数量级

目标:第3课你已经学过递归的基线条件、递归条件和调用栈——这节课把这套框架直接套进排序问题, 搞懂基准值和分区怎么运作,以及为什么"选第一个元素做基准值"在有些输入下会让复杂度整个塌掉。约 25 分钟。
分而治之(D&C)的直觉: 面对一堆乱序的数字不知道怎么下手,就先选一个"基准"(pivot), 把剩下的数字分成比它小的一堆和比它大的一堆,两堆各自再重复同样的动作, 直到每一堆都只剩 0 个或 1 个元素——这时候不用排都已经"有序"了。这就是快速排序的全部思路, 书里用"把土地均匀分割成最大方块"的例子讲过同一套两步法:先看能不能整除(基线条件),不能就切出当前能装下的最大方块, 剩下的地当新问题继续递归——快速排序只是把这套通用思路套用到了排序上。

一、快速排序的三步

快速排序(Quicksort)是 D&C 思路在排序问题上的典范应用,C 标准库的 qsort 就是这么实现的。完整流程:

  1. 基线条件: 数组为空或只有一个元素,直接返回,视为"已经排序"。
  2. 选基准值(pivot): 从数组里选一个元素做参照,初学时最简单的做法是固定选第一个元素。
  3. 分区(partitioning): 把其余元素分成两组——小于等于基准值的 less,大于基准值的 greater。
  4. 递归: 对 less 和 greater 分别递归调用快速排序。
  5. 合并: quicksort(less) + [pivot] + quicksort(greater),拼接得到整体有序数组。

二、基准值与分区 —— 手动追踪一遍

以数组 [33, 15, 10] 为例,固定选第一个元素 33 做基准值:

步骤处理对象pivotlessgreater说明
顶层[33,15,10]33[15,10][]15、10 都 ≤33,进 less;没有元素 >33
第二层[15,10]15[10][]10 ≤15,进 less;greater 为空
第三层[10]———长度=1,触发基线条件,直接返回 [10]
合并回第二层quicksort([10]) + [15] + quicksort([]) = [10] + [15] + [] = [10,15]
合并回顶层quicksort([15,10]) + [33] + quicksort([]) = [10,15] + [33] + [] = [10,15,33]

注意分区只处理"基准值以外"的元素——less、greater 都不包含 pivot 自己,pivot 是合并时单独插进去的那一个。

三、复杂度:平均情况 vs 最坏情况

快速排序的复杂度高度依赖基准值选得好不好,同一个算法在不同输入下能差出一个数量级:

情况大O触发条件调用栈高度
平均 / 最优情况O(n log n) 基准值每次都能把数组大致分成两半(接近中位数),8个元素只需约 log₂8=3 层就能分完 约 log n 层
最坏情况O(n²) 数组本身已经排好序(升序)+ 每次都固定选第一个元素作基准值 → pivot 恰好是当前区间的最小值, 分区后 less 永远为空、greater 拿到几乎全部剩下的元素 退化为约 n 层

对照第2课学过的选择排序,能更清楚地看出快排"平均很快、但没有保证"这个特点:

算法平均情况最糟情况备注
选择排序(第2课)O(n²)O(n²)速度慢,但稳定不波动
合并排序(merge sort)O(n log n)O(n log n)稳定,但常量比快排大
快速排序O(n log n)O(n²)常量小,实践中常比合并排序快;最糟情况取决于基准值选择
一个容易被忽略的角度: 快排和合并排序大O阶数都是 O(n log n),但大O表示法本身会忽略每步操作的固定耗时(常量 c)。 快排的常量比合并排序小,这也是为什么大多数标准库更青睐快速排序——"大O相同"不等于"速度一样"。

C 代码(分区函数 + 递归函数,带边界条件详解)

#define MAXN 100                        // 简化起见,假设数组元素不超过 100 个

// 分区函数:以 arr[0] 为基准值(书中"初学时可简单选第一个元素"的做法)
// 把 arr[1..n-1] 按 <=pivot / >pivot 分别收进 less 和 greater
int partition(int arr[], int n, int less[], int greater[], int *less_n, int *greater_n) {
    int pivot = arr[0];
    int li = 0, gi = 0;
    for (int i = 1; i < n; i++) {        // ① 从下标1开始,不把 pivot 自己也分进去
        if (arr[i] <= pivot) less[li++] = arr[i];
        else                 greater[gi++] = arr[i];
    }
    *less_n = li;
    *greater_n = gi;
    return pivot;
}

// 递归主函数:把排好序的结果写进 result
void quicksort(int arr[], int n, int result[]) {
    if (n < 2) {                          // ② 基线条件:0 个或 1 个元素,直接视为"已排序"
        for (int i = 0; i < n; i++) result[i] = arr[i];
        return;
    }
    int less[MAXN], greater[MAXN];
    int less_n, greater_n;
    int pivot = partition(arr, n, less, greater, &less_n, &greater_n);

    int sorted_less[MAXN], sorted_greater[MAXN];
    quicksort(less, less_n, sorted_less);          // 递归处理 less
    quicksort(greater, greater_n, sorted_greater);  // 递归处理 greater

    int idx = 0;
    for (int i = 0; i < less_n; i++)    result[idx++] = sorted_less[i];
    result[idx++] = pivot;                          // ③ 合并顺序:less + [pivot] + greater
    for (int i = 0; i < greater_n; i++) result[idx++] = sorted_greater[i];
}

三处最容易在考试里写错、丢分的地方:

  1. ① 分区循环必须从下标 1 开始,跳过 pivot 自己。 如果写成 for (i = 0; i < n; i++) 从下标0开始,会把 arr[0](也就是 pivot 自己) 也拿去和自己比较、塞进 less 或 greater,导致分区结果比原数组多算了一个元素, 最终合并出来的数组长度和原数组对不上。
  2. ② 基线条件是 n < 2(即 n==0 或 n==1),不能漏写。 如果忘记写这个 if,递归到子数组长度为0(空数组)时,函数会直接冲进 partition, 而 partition 第一步就是 pivot = arr[0]——这是在读一个根本不存在的下标, 在 C 里是未定义行为(不像 Python 数组越界会直接抛异常),而且函数不会提前结束, 子数组会一直以 n=0 的状态继续递归下去,这是期末最容易因为"忘记设基线条件"写出的无限递归。
  3. ③ 合并顺序必须是 less + [pivot] + greater,顺序不能颠倒或漏掉 pivot。 如果写成 greater 在前、less 在后,或者忘记把 pivot 塞进结果里, 得到的数组整体顺序就是错的,或者少了一个元素——阅卷通常按"结果错误"扣分,不算部分正确。
常见陷阱:对已经排序好的数组,总是固定选第一个元素作基准值。 如果输入数组本身就是升序(比如 [10,15,33,40,55]),每次选 arr[0] 做 pivot—— 它正好是当前区间的最小值,分区后 less 永远是空的、greater 拿走了几乎全部剩下的元素, 相当于每递归一层只筛掉 1 个元素(pivot 自己),调用栈深度退化到 O(n) 层,总时间退化为最糟情况 O(n²)。书中给出的应对方法是:随机选择基准值,让"总是恰好选到最值"这种最坏情况不容易被复现, 平均情况的 O(n log n) 才更可靠地体现出来。

Q3. 找茬题:下面这段快速排序代码丢了一部分,请找出问题。

先想清楚:当子数组长度递归到 0 或 1 时,这段代码会发生什么。

void quicksort(int arr[], int n, int result[]) {
    int less[MAXN], greater[MAXN];
    int less_n, greater_n;
    int pivot = partition(arr, n, less, greater, &less_n, &greater_n);  // ← 直接从这里开始

    int sorted_less[MAXN], sorted_greater[MAXN];
    quicksort(less, less_n, sorted_less);
    quicksort(greater, greater_n, sorted_greater);

    int idx = 0;
    for (int i = 0; i < less_n; i++)    result[idx++] = sorted_less[i];
    result[idx++] = pivot;
    for (int i = 0; i < greater_n; i++) result[idx++] = sorted_greater[i];
}
▸ 看没有基线条件会发生什么

设某次递归调用到 quicksort(sub, 0, result)(sub 是空数组): partition 第一步 pivot = arr[0]——但 sub 长度是0,arr[0] 根本不存在,读到的是内存里的垃圾值(未定义行为);接着 for 循环因为 n=0 不会执行, less_n、greater_n 都返回0,然后又递归调用 quicksort(less,0,...) 和 quicksort(greater,0,...)——也就是用同样的 n=0 又调用了自己两次。 没有任何机制能让 n 发生变化或让函数提前返回,这个 n=0 的递归会不断复制自己, 调用栈只会越叠越深,直到耗尽栈空间、程序崩溃(栈溢出)。
