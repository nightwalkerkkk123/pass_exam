目标:讲透 K 最近邻算法(KNN)的分类/回归两种用法、距离度量的选择、K 值的坑——这是期末最可能考的"机器学习入门"算法; 后半段扫一眼全书收尾提到的十来个算法领域,只求"听说过、认得出",不深挖编码。约 25 分钟。
先接上第9课: 上一课学的动态规划,是用网格把"选或不选"的决策一步步展开,精确求出 0/1 背包问题的最优解—— 本质是一种"确定性推导"的思路。KNN 是完全不同的画风: 它不建模、不推导,靠的是最朴素的直觉—— "物以类聚":想判断一个新样本是什么类别,不用列公式,只看离它最近的 K 个已知样本里,大多数是什么类别, 新样本就归为哪一类。这是《算法图解》从"经典算法"迈进"机器学习"的第一步,也是全书最后一个要精讲的算法。

一、KNN 是什么:一套动作,两种用法

KNN(K-Nearest Neighbors,K最近邻)只有三步:

  1. 特征抽取: 把每个待比较的东西(水果、用户、图像……)翻译成一组可比较的数字,也就是坐标。 没有这一步,"计算相似度"根本无从谈起。
  2. 计算距离: 用距离公式算出新样本和所有已知样本之间的"远近"。
  3. 挑出最近的 K 个邻居,给出结论:
    • 分类(classification): 看这 K 个邻居里哪个类别票数最多,新样本就归为哪一类。 比如判断一个水果是橙子还是柚子,看离它最近的 3 个邻居(K=3)里橙子多还是柚子多。
    • 回归(regression): 把这 K 个邻居的数值取平均,作为预测结果。 比如预测某用户会给一部电影打几分,找出品味最接近的 5 个人(K=5),把他们的打分取平均。

两种用法唯一的区别就在最后一步:分类是"投票选多数",回归是"求平均"——前面特征抽取、算距离、选K个邻居的步骤完全一样。

二、距离怎么算:欧氏距离不是唯一选项

书里给出的默认距离公式是欧几里得距离(把毕达哥拉斯定理推广到多维):

distance = sqrt((x1-y1)² + (x2-y2)² + ... + (xn-yn)²)

这个公式即使特征维度从 2 个(水果的个头、颜色)涨到 5 个(用户对 5 类电影的喜好),依然直接适用,不用改公式。 但它有一个隐藏假设:比较的是两点之间的"绝对距离"。

如果场景是"评分尺度不一致"——比如 A 打分宽松、喜欢的都打 5 分,B 打分严格、只有真爱才打 5 分,两人品味实际很接近, 但用欧氏距离算出来的"距离"会偏大,被误判为不相邻。这时该换成余弦相似度:它比较的不是两点距离,而是两个向量的夹角—— 只关心"喜好的方向是否一致",不在乎打分整体偏高还是偏低。

记忆锚点: 欧氏距离问的是"你们站得有多远",余弦相似度问的是"你们朝向是不是同一个方向"。 题目一旦出现"评分习惯/打分尺度不同"的字眼,大概率是在提示你该用余弦相似度,而不是死套欧氏距离公式。

三、K 怎么选:太小太大都是坑

K 不是固定为 5,可以是 2、10,甚至更大——它是一个需要根据数据规模权衡的旋钮,没有统一公式。 但两头都有明确的风险:

K 的取值典型风险直觉类比
太小(比如 K=1)极易被个别噪声/异常邻居带偏,对局部波动过度敏感只听一个邻居的意见,偏偏这个邻居是个怪人
太大(比如 K 接近样本总数)把大量不相关、甚至不同类别的邻居也拉进来平均,类别边界被模糊/稀释把全班同学的意见都算上,少数关键意见被多数无关意见淹没

书中练习提到:用户量级从几十涨到几百万时,K=5 可能就显得太小了——数据规模变大,通常需要相应调大 K,反之亦然。

常见陷阱(来自 ch10 反面案例):
  • 特征选择不当: 挑来算距离的特征如果和目标不相关(用"是否喜欢猫图片"判断电影推荐)或过于片面 (只让用户给同一系列的三部电影打分,看不出真实的电影品味),距离算得再精确也没有意义。
  • 忽视评分尺度差异: 直接对不同打分习惯的人套用欧氏距离,可能把品味相同的两人判定为"不相邻" ——解决方向是加权或改用余弦相似度(见第二节)。
  • 迷信历史数据能预测一切: 股票市场这类变量繁多、近乎随机的现象,就算精心挑选特征、套用 KNN, 也未必能得到可靠预测——机器学习不是万能药,选对应用场景和算法本身同样重要。

四、C 代码实现:简化版 KNN 分类

下面用"橙子(0)/柚子(1)"两类、"个头/颜色深浅"两个特征的简化场景,实现一次完整的 KNN 分类:

typedef struct {
    double features[2];   // 简化:每个样本只用2个特征(个头、颜色深浅)
    int label;             // 0=橙子, 1=柚子
} Sample;

double euclidean_distance(double *a, double *b, int dim) {
    double sum = 0.0;
    for (int i = 0; i < dim; i++) {
        sum += (a[i] - b[i]) * (a[i] - b[i]);
    }
    return sqrt(sum);      // ① 默认用欧氏距离,不是余弦相似度
}

// 返回值: 0=判定橙子, 1=判定柚子, -1=平票(需要额外规则)
int knn_classify(Sample *samples, int n, double *target, int dim, int k) {
    if (k > n) k = n;          // ② K 不能超过已知样本总数

    double dist[64];
    for (int i = 0; i < n; i++) {
        dist[i] = euclidean_distance(samples[i].features, target, dim);
    }

    // 选出距离最小的k个:k轮"选择排序"式选最小值,交换到数组前面
    for (int i = 0; i < k; i++) {
        int min_idx = i;
        for (int j = i + 1; j < n; j++) {
            if (dist[j] < dist[min_idx]) min_idx = j;
        }
        double td = dist[i]; dist[i] = dist[min_idx]; dist[min_idx] = td;
        Sample ts = samples[i]; samples[i] = samples[min_idx]; samples[min_idx] = ts;
    }

    // 统计前k个邻居里,两类各拿到多少票
    int vote[2] = {0, 0};
    for (int i = 0; i < k; i++) {
        vote[samples[i].label]++;
    }

    if (vote[0] == vote[1]) {
        return -1;             // ③ K为偶数时,两类票数可能相等——平票
    }
    return vote[0] > vote[1] ? 0 : 1;
}

三处最容易在考试里写错、混淆的地方:

  1. ① 距离公式默认用欧氏距离,但要看题目场景。 如果题目提到"打分尺度/评分习惯不一致",就该换成余弦相似度——这是全书特意点出的对比, 考试里容易出"下面哪种情况该用余弦相似度而不是欧氏距离"的辨析题。
  2. ② K > n(K 大于已知样本总数)必须做保护处理。 如果不做 if (k > n) k = n; 这一步,要么后面选最近邻的循环访问越界内存, 要么"K个邻居"其实用光了全部样本,KNN 退化成"看全体样本的多数类",完全失去"看邻近"的意义。
  3. ③ K 为偶数时,最近的 K 个邻居可能出现两类票数相等(平票)。 比如 K=4,2 票橙子、2 票柚子,单纯"多数表决"给不出结果。常见的 tie-break 规则是: 优先选奇数 K、按距离加权投票(离得越近权重越大)、或退回去看距离最近的单个样本。 这是选择题里最爱设的坑——"K=10 时下面哪种结果不可能出现"之类的题,本质就是在考这一点。

概念一句话说明
二叉查找树(BST)/ 红黑树 / B树 / 堆把二分查找的思路搬到"能动态插入删除"的树结构上;红黑树自平衡防止退化,B树适配数据库,堆常用于优先级队列
反向索引(Inverted Index)搜索引擎的核心结构:单词 → 包含它的网页集合的散列表,查询时不用扫全网
傅里叶变换把一段信号(声音/图像/DNA)分解成不同频率成分,MP3压缩、JPG压缩、Shazam识曲背后都是它
并行算法 / MapReduce把任务拆给多核/多机同时算;MapReduce = map(逐元素并行变换) + reduce(归并成一个值),Hadoop 是代表工具
布隆过滤器 / HyperLogLog用"可控的不确定性"换省空间:前者判断"元素是否存在"(只会误报、绝不漏报),后者估算"有多少种不同元素"
SHA(单向散列)能正向算出散列值,但无法从散列值反推原文,用于密码/文件校验(应优先选 SHA-2/SHA-3 或 bcrypt)
Simhash与 SHA 反向而行:追求"相似输入 → 相似散列值",用于查重、反抄袭
Diffie-Hellman / RSA公钥加密、私钥解密,解决了"如何在不安全信道上协商密钥"的老问题
线性规划比图算法更宽泛的最优化框架,最短路、最大流这些图问题都只是它的特例
朴素贝叶斯分类器与 KNN 应用场景相近(如垃圾邮件过滤),但原理是"算概率"而非"算距离"
