第 18 课:最小生成树与图的进阶结构 —— Kruskal、Prim、割点割边、二分图匹配

回顾:第17课学了并查集的 find/union 和路径压缩——这节课 Kruskal 算法会直接用到它。本课信息量大(4个子专题), 每个只讲透最核心的一点,不追求面面俱到。约 25 分钟。
修路问题:最少花多少钱,把所有城市连通? 假设你是镖局老板,要在 n 座城镇之间选一批道路(每条路都有一个打点费/权值), 要求所有城镇都能互相到达,且总打点费最少。n 个顶点的连通图最少需要 n-1 条边才能连通—— 从所有边里选出这 n-1 条边、且边权和最小,这棵树就叫最小生成树(Minimum Spanning Tree,MST)。 书里给出两种贪心解法:Kruskal(按边贪心)和 Prim(按点贪心)。

一、Kruskal 算法 —— 按边贪心,复用并查集判环

思路很直接:把所有边按权值从小到大排序,从最小的边开始,依次尝试加入。每条边要不要选,只看一件事—— 加入它会不会让图出现回路(环)。判断"会不会成环"正是并查集的拿手好戏: 如果这条边的两个端点已经在同一个集合(已经连通),加入就会成环,跳过; 如果不在同一个集合,加入不会成环,选中这条边并合并两个集合。选满 n-1 条边就结束。

这一步"判断连通、合并集合",和第17课的 find / union 完全是同一套逻辑——Kruskal 本质上就是"排序 + 并查集"。

二、Prim 算法 —— 按点贪心,逐步扩大生成树

思路换了一个角度:从一个顶点开始,用 dis[] 记录每个还没入树的顶点"到整棵生成树"的当前最短距离 (直连边就是边权,没有边就是无穷大)。每一轮从 dis[] 里挑出最小值对应的顶点纳入生成树, 再用这个新顶点去更新(松弛)其余顶点的 dis[]。重复 n-1 轮,直到所有顶点都入树。

Prim 和 Dijkstra 几乎是双胞胎,唯一区别是 dis[] 的含义:Dijkstra 的 dis[] 是"到固定源点的距离", Prim 的 dis[] 是"到当前整棵生成树的距离"——松弛时不需要再加"起点到中间点"那一段,直接取边权本身。

对比表:Kruskal vs Prim,该选哪个?

维度KruskalPrim
贪心角度按边贪心:全局排序后逐条扫描按点贪心:从一点出发逐步扩张

三、割点与割边(Tarjan算法)—— 找图中的"命门"

问题原型是"炸哪座城市能让敌方彻底失联"——删掉某个顶点后图不再连通,这个顶点就是割点; 删掉某条边后图不再连通,这条边就是割边(桥)。逐一删点/删边再跑一次 DFS 验证连通性太慢, Tarjan 给出线性做法:做一次 DFS,记录每个顶点的时间戳 num[](第几个被访问到), 再维护 low 值 low[](某点不经过其DFS树父亲、仅靠树边+回边能回溯到的最早时间戳)。

对树上某点 x 的孩子 v:

  • 割点判断条件:low[v] >= num[x](带等号)——说明 v 这棵子树绕不回 x 的祖先,x 是割点。 DFS 根节点是特例:根是割点当且仅当它在 DFS 树中有 ≥2 个孩子。
  • 割边判断条件:low[v] > num[x](去掉等号)——等号成立时 v 还能绕回 x 本身(父亲), 说明这条边不是桥;严格大于时 v 连 x 都回不去,这条边才是桥。

割点、割边用的是同一棵 DFS 树、同一套 num[]/low[],唯一差别就是判断条件的等号要不要留——这是期末最容易混淆的地方。

四、二分图最大匹配 —— 匈牙利算法的"连锁反应"

二分图:顶点能分成两个集合 X、Y,每条边的两端恰好一个属于 X、一个属于 Y(集合内部无边)。 问题原型是"过山车配对":女生集合与男生集合,只有互相认识的一对才能配对,求最多能配成多少对。

匈牙利算法思路:对每个还没配对的 X 顶点 u,逐条尝试它的边 (u,v)—— 若 v 还没配对,直接配上;若 v 已经配给了 u',就让 u' 去尝试换一个别的对象(递归重复这个过程), 如果 u' 真的换成功了,v 就腾出来配给 u。这条"连锁反应"路径就叫增广路。 每找到一条增广路,配对数就 +1;直到所有 X 顶点都试过仍找不到新的增广路,当前匹配就是最大匹配。

五、C 代码:Kruskal 算法核心逻辑(含易错点标注)

typedef struct { int u, v, w; } Edge;   // 边:两端点 + 权值

int f[MAXN];                      // 并查集数组,复用第17课的写法

int getf(int v) {                 // 查找根节点(带路径压缩)
    if (f[v] == v) return v;
    return f[v] = getf(f[v]);
}

int cmp(const void *a, const void *b) {
    return ((Edge *)a)->w - ((Edge *)b)->w;  // ① 按权值从小到大排序
}

int kruskal(Edge e[], int m, int n) {
    int i, count = 0, sum = 0;
    for (i = 1; i <= n; i++) f[i] = i;      // 初始化:每个点各自为根
    qsort(e + 1, m, sizeof(Edge), cmp);      // 先排序,再扫描

    for (i = 1; i <= m && count < n - 1; i++) {
        int t1 = getf(e[i].u), t2 = getf(e[i].v);
        if (t1 != t2) {                       // ② 比较 find 结果,不是比较 u、v 本身
            f[t2] = t1;                        // 合并两个集合
            count++;
            sum += e[i].w;
        }
        // t1==t2 说明 u、v 已经连通,这条边加入会成环,直接跳过
    }
    return (count == n - 1) ? sum : -1;      // ③ 选出的边数必须恰好是 n-1
}

三处最容易在考试里写错、丢分的地方:

  1. ① 排序必须是权值从小到大(升序)。 cmp 写反(变成 e2->w - e1->w)会让算法先选大权值的边, 选出来的边集合总权值不再是最小的,整个 Kruskal 就失去了意义。
  2. ② 判断"会不会成环"要比较 find(u) 和 find(v) 的结果,不能直接比较 u 和 v 这两个端点编号。 两个端点编号不同,不代表它们不在同一个连通分量里——很可能之前几步已经把它们合并到同一棵树下了。
  3. ③ 最终选出的边数应该正好是 n-1,可以作为检验。 如果扫完所有边 count 还不到 n-1, 说明原图本来就不连通,根本没有生成树,不能强行返回一个"半成品"当作答案。
常见陷阱(容易混着上一课或相邻算法的写法出错):
  • Prim 的 dis[] 不是 Dijkstra 的 dis[]。 松弛时如果照搬 Dijkstra 写成"dis[j] + 边权"就错了—— Prim 的 dis[] 只需直接取边权本身,因为目标是"离整棵生成树最近",而不是"离固定源点最近"。
  • 判断割边时忘记去掉等号。 如果照抄割点的条件 low[v] >= num[x],会把"能绕回父亲、但绕不回更早祖先"的边也误判成桥—— 割边必须是严格的 low[v] > num[x]。
  • 二分图匹配递归找增广路时,忘记重置 book[] 数组。 book[] 是防止本轮递归重复访问同一个顶点的标记, 一旦忘记在每轮开始前清零,轻则漏掉本该可行的连锁重匹配,重则直接死循环。
