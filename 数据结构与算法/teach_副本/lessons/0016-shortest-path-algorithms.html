第 16 课:最短路径三算法(Floyd-Warshall / Bellman-Ford / SPFA) —— 与 Dijkstra 的收尾对照

目标:从"只求一个起点到所有点"的 Dijkstra(第7课)升级到"求所有点两两之间"的 Floyd-Warshall, 再学会用 Bellman-Ford / SPFA 应对负权边——这是全课程图论最短路径专题的收尾课,重点是把四种算法的 选择标准彻底理清。约 25 分钟。
先接上第15课: 第15课学了用邻接矩阵存图,以及直接用 DFS/BFS 求最短路径的局限——DFS/BFS 只在 "边数最少即最短"(所有边权重相同或压根没有权重)时才对,一旦边有不同的权重,就必须换专门的最短路径算法。 第7课的 Dijkstra 已经解决了"只求一个起点到所有点"这个问题,但它有两个从没被戳破的假设: 一是只关心单个源点,二是默认所有边权都是正的。这节课要同时升级这两个假设: ①如果我要的是所有点两两之间的最短路径,不只是一个起点出发的呢?②如果图里出现了负权边 (比如"倒贴钱也要走"的边),Dijkstra 的贪心策略还成立吗?这两个"维度"分别对应 Floyd-Warshall 和 Bellman-Ford/SPFA。

一、核心思想:都是"松弛",只是顺序不同

这一章所有算法,本质上都在做同一件事——松弛(relax): 用一条新路径(经由某个中转点或中转边)去尝试更新"从 A 到 B 目前已知的最短路程",如果更短就替换。 四个算法的区别只在于:按什么顺序、对哪些点/边做这件事。

1. Floyd-Warshall —— 枚举"中转点",求所有点对

用二维矩阵 e[i][j] 存图(e[i][i]=0,不可达设为极大值 INF)。 核心思想是"逐步允许经过更多的中转点":第一轮只允许经过 1 号顶点中转,判断 e[i][j] 是否比 e[i][1]+e[1][j] 大;第二轮允许经过 1、2 号顶点中转;依次类推,直到允许经过全部 n 个顶点。这本质上是一种动态规划思想:"从 i 到 j 只经过前 k 号点的最短路程"。

书里(ch06)给了一个具体的 4 城市例子,很好地展示了"每多放开一个中转点,路程可能进一步变短"这件事:

允许经过的中转点e[4][3](4号到3号的路程)为什么变化
无(只用直达公路)124→3 直达公路长度是 12
允许经 1 号中转11e[4][1]+e[1][3]=5+6=11,比12小,松弛成功
允许经 1、2 号中转10e[1][3] 先被 2 号中转缩短为5(e[1][2]+e[2][3]=2+3=5), 于是 e[4][1]+e[1][3]=5+5=10

2. Bellman-Ford —— 不贪心,对所有边暴力松弛

Dijkstra 的贪心"每轮选当前最小的点确定下来,不再回头"依赖一个前提:没有负权边。一旦某条边是 负数,"已经确定的点"完全可能被后面一条经过负权边的路径反超,但 Dijkstra 不会回头检查。 Bellman-Ford 索性放弃贪心,改成对全部 m 条边反复做松弛:外层循环 n-1 次, 内层循环遍历全部边 u[i]→v[i](权值 w[i]),判断 dis[v[i]] > dis[u[i]]+w[i] 是否成立并松弛。之所以只需 n-1 轮,是因为最短路径必是不含回路的简单路径,最多包含 n-1 条边;所以第 k 轮松弛后得到的是"最多经过 k 条边"的最短路径,n-1 轮后必然收敛。

3. SPFA —— Bellman-Ford 的队列优化

Bellman-Ford 每轮都要扫描全部边,太浪费——大部分点的估计值早就不再变化了。SPFA 用一个队列只维护 "估计值刚被更新过、需要重新去松弛别人"的顶点:源点先入队;每次取出队首顶点,对它的所有出边尝试松弛, 若某条边成功松弛且被更新的终点当前不在队列中,就把它入队(同一顶点不重复入队,用 book 数组判重);处理完这个顶点的出边后把它出队;重复直到队列为空。它形式上很像 BFS,但 BFS 的点出队后不会 再入队,SPFA 的点可能被反复刷新、反复入队。

判负权回路的两种手感: Bellman-Ford 中,如果 n-1 轮跑完之后还有边能继续松弛, 说明图里存在负权回路(此时最短路径根本不存在,绕一圈就能无限变小,不能继续松弛下去当作答案)。 SPFA 中,如果某个顶点入队次数超过 n 次,同理说明存在负权回路。

三、C 代码实现 Floyd-Warshall(带边界条件详解)

先初始化矩阵,再跑三重循环:

// 第一步:初始化——这一步的两个坑最容易在期末丢分
for (i = 1; i <= n; i++)
    for (j = 1; j <= n; j++)
        dist[i][j] = (i == j) ? 0 : INF;   // ① 自己到自己是0,不直接相连是"无穷大",不是0

for (t = 0; t < m; t++) {
    scanf("%d %d %d", &u, &v, &w);
    if (w < dist[u][v]) dist[u][v] = w;    // 读入已知边,重边取更小的一条覆盖上去
}

// 第二步:三重循环核心——只有五行,但顺序不能错
for (k = 1; k <= n; k++) {                 // ② k(允许经过的中转点)必须是最外层
    for (i = 1; i <= n; i++) {
        for (j = 1; j <= n; j++) {
            if (dist[i][k] < INF && dist[k][j] < INF &&      // ③ 防止 INF+INF 溢出
                dist[i][j] > dist[i][k] + dist[k][j]) {
                dist[i][j] = dist[i][k] + dist[k][j];
            }
        }
    }
}

三处最容易在考试里写错、丢分的地方:

  1. ① 初始化时,自己到自己(i==j)是 0,不直接相连的两点是 INF,不能都写成 0。 如果不直接相连的两点也被初始化成 0,Floyd 会误以为任意两个原本不通的点之间存在一条长度为 0 的 "免费"路径,后续所有松弛结果全部错误——这是最隐蔽的一类初始化 bug,因为代码能跑、不会崩溃, 只是答案全错。
  2. ② 三重循环里,k 必须是最外层,不能把 i 或 j 放在最外层。 k 这层循环的意义是"逐步放开允许经过的中转点"——这是一个动态规划式的状态转移顺序: 只有当"允许经过前 k-1 个点做中转"的结果都已经算完整,才能正确判断"现在再多允许经过 第 k 个点,能不能把 e[i][j] 缩短"。如果 k 不在最外层, e[i][k]、e[k][j] 用到的可能是还没被"放开"更新完的旧数据, 有些本该被多个中转点接力发现的更短路径就永远发现不了,得到偏大、错误的结果。
  3. ③ 判断 dist[i][k]+dist[k][j] 之前,要加上 dist[i][k]<INF && dist[k][j]<INF 的保护条件。 如果 i、k 之间(或 k、j 之间)本来不通, 两个"无穷大"直接相加,在 int 范围内可能整数溢出,变成一个错误的、甚至是负数的 溢出值,反而被误判成"更短的路径",把整张 dist 表污染掉。
常见陷阱:
① 对含负权边的图错误使用 Dijkstra。 Dijkstra 的贪心"选出当前最小、确定它不会再变短"这个判断, 完全依赖"不存在更短的路径经过尚未确定的点"这个前提——一旦某条边是负数,后面完全可能有一条经过负权边的 路径反超已经"确定"的点,但 Dijkstra 不会回头检查,直接给出错误答案而不报错。这和第7课"换黑胶唱片" 例子说的是同一件事。
② Floyd-Warshall 三重循环顺序颠倒(k 没放最外层)。 代码依然能跑、不会报错, 但破坏了"逐步放开中转点"的 DP 状态转移顺序,算出来的 dist 矩阵可能不是全局最短路径, 是这门课最容易被当作"改错题"考的一个点。

四、通关前先自测

Q1. 一张有5个顶点的图,边权中有负数,但已确认没有负权回路,现在只需要求 "1号顶点到其余4个顶点"的最短路径。应该选哪个算法?
Q2. 如果把 Floyd-Warshall 的三重循环写成 for(i) for(j) for(k) (即把 k 挪到最内层),会发生什么?
Q3. 找茬题:下面这段 Floyd-Warshall 的初始化代码有 bug,选出问题所在。
for (i = 1; i <= n; i++)
    for (j = 1; j <= n; j++)
        if (i == j) dist[i][j] = 0;
        else dist[i][j] = 0;        // ← 这一行
// 之后再读入已知的边,把 dist[u][v] = w 覆盖进去
▸ 看错在哪、怎么改

正确写法应该是:

for (i = 1; i <= n; i++)
    for (j = 1; j <= n; j++)
        dist[i][j] = (i == j) ? 0 : INF;   // 不直接相连是 INF,不是 0

举个具体的错误后果:假设 3 号和 5 号顶点原本毫无关系,错误代码里 dist[3][5]=0。 三重循环跑起来后,任何顶点 i 都会发现"经过 3 号再到 5 号"这条"路径"长度是 dist[i][3]+0,凭空比真实路程小很多,把整张矩阵污染成一堆偏小的错误值。

Q4. Floyd-Warshall 代码里,为什么要在比较 dist[i][k]+dist[k][j] 之前, 加上 dist[i][k]<INF && dist[k][j]<INF 这样的保护条件?
