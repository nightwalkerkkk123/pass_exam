# 设计文档：操作系统考前小题+问答题速记页

## 1. 项目背景

- 仓库：`E:/CODE/pass_exam`
- 这是一个《操作系统》期末冲刺学习工作区，不是传统软件工程仓库。
- 已有 87 节 HTML 微课在 `operating_systems/lessons/`，以及 `0013-exam-cheatsheet-map.html` 全书速查地图。
- 用户需要一个新的网页：**考前小题知识点速记** + **问答题速记**，用于考前一晚/考前 30 分钟快速过一遍。

## 2. 目标

交付一个独立的、与现有课程风格一致的 HTML 页面：

- 把选择/填空/判断的**小题辨析点**和简答题的**标准答法**抽出来。
- 采用可折叠卡片，先想后看，适合自测。
- 支持「只看高频」过滤、全部展开/收起、打印优化。
- 覆盖 Ch1–Ch6 全部考点，按高频→中频排序。

## 3. 范围

### In Scope

- 新建 `operating_systems/lessons/0088-exam-cram-cards.html`。
- 在 `operating_systems/lessons/index.html` 中新增一条指向该页的链接。
- 可选：在 `operating_systems/learning-records/` 新增一条使用记录。

### Out of Scope

- 不改现有 0013 速查地图的内容和结构。
- 不引入外部框架（React/Vue/构建工具）。
- 不添加后端或数据持久化。
- 不覆盖 Ch7–Ch12（项目已确认跳过）。

## 4. 信息架构

页面从上到下：

1. **Hero 区**
   - 标题：操作系统期末 · 小题 + 问答题速记
   - 一句话说明
   - 题型分值条（选择/填空/判断/简答/计算）
   - 控制按钮：只看高频 / 全部 / 全部展开 / 全部收起

2. **小题速记区**（id="small"）
   - 按 Ch1–Ch6 分节
   - 每张卡片：正面 = 题干/考点关键词；背面 = 结论/辨析
   - 标签：高 / 中 / 低

3. **问答题速记区**（id="qa"）
   - 同样按 Ch1–Ch6 分节
   - 每张卡片：正面 = 问题；背面 = 考试写法要点（分点、关键词）
   - 标签：高 / 中 / 低

4. **易错雷区**
   - 底部固定 10–12 条最容易翻车的点
   - 不折叠，直接展开

## 5. 视觉与交互

### 5.1 视觉风格

沿用现有 lessons/ 设计系统：

- 颜色变量：`--ink #1a2230`、`--muted #667`、`--line #e3e6ec`、`--bg #f7f8fb`
- 强调色：`--blue #2b6cb0`、`--green #2f855a`、`--red #c53030`、`--amber #b7791f`
- 字体：`-apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif`
- 卡片：白底、细边框、圆角、左侧色条区分高频（红）/ 中频（橙）/ 低频（灰）

### 5.2 交互

- 点击卡片标题展开/收起答案。
- 「只看高频」按钮：隐藏 `.freq-mid` 和 `.freq-low`，只保留 `.freq-hi`。
- 「全部展开/收起」按钮：切换所有卡片的展开状态。
- Sticky 顶部导航：跳到小题区、问答区、易错雷区、各章。
- `@media print`：隐藏按钮和导航、强制展开所有卡片、背景变白、适配 A4。

## 7. 技术方案

### 7.1 方案选择

采用**单文件内联版**：HTML + CSS + JS 全部写在一个 `.html` 文件中。

原因：

- 与现有 87 节课风格一致。
- 打开即用，离线/打印友好。
- 项目没有复杂构建流程，单文件维护成本最低。

### 7.2 未选方案

- JSON 数据驱动版：多一个文件，与项目惯例不一致。
- 在 0013 上扩展：会稀释考前速记的聚焦感，用户已明确要独立页面。

## 10. 风险与注意事项

- **老师口径优先**：成组链接法、位示图 0/1 含义、混合索引 D 值等敏感点，必须按老师课件/范例写，不照搬王道。
- **避免 408 化**：不引入超出学校自命题范围的考点。
- **保持零基础友好**：每张卡片答案不要堆太多术语，先给结论再补关键词。
