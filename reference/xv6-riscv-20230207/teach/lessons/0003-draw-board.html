课 3 · 白板画一张图

目标:把 bread() 7 行 + 你的类比,画成 1 张白板能徒手画出的图。

前提:读完 课 2,挑好类比。

导航 · 1. 板书图 3 个原则 · 2. 通用模板 · 3. 范例(食堂版) · 4. 你画一遍 · 5. 计时 60 秒

1. 板书图 3 个原则

原则 1:60 秒画完
讲台时间宝贵,板书超过 90 秒就拖了节奏。你的图必须 60 秒内能徒手画完。
原则 2:只画 3–5 个元素
多 1 个元素 = 多 10 秒 + 同学多 1 个要记的。
**推荐上限 5 个** (例:1 个起点 + 1 个判断 + 2 个终点 + 1 个返回箭头)
原则 3:画 1 个判断,不是 1 个过程
不要画"详细 IO 时序图",画"1 个菱形判断 + 2 个出口"。
同学看到图能秒懂"哦有个判断",然后听你讲。

2. 通用模板(适用于所有"先查后取"型代码)

┌──────────┐ │ 调用方 │ (你) └────┬─────┘ │ "请给我 dev 磁盘上第 blockno 块" ▼ ┌──────────────────┐ │ 缓冲池里有这块? │ ◄──── 1 个菱形判断 └────┬─────────┬───┘ │ 是 │ 否 ▼ ▼ ┌────────┐ ┌──────────────┐ │ 拿走 │ │ 真去磁盘读 │ │ 返回 │ │ 读完返回 │ └────┬───┘ └────┬─────────┘ │ │ └─────┬──────┘ ▼ ┌──────────┐ │ 返回结果 │ (档案袋) └──────────┘

这张图能用 60 秒画完吗? — 不能,因为太多方框。我们精简成 4 元素:

简化版(4 元素): 调用方 ─► 缓冲池(里面有?) ── 是 ─► 直接返回 │ │ 否 ▼ 磁盘(真去读) ──► 返回 重点: 1 个判断 + 2 条路径

3. 范例(食堂打饭版)

假设你在课 2 选了 食堂打饭 类比,板书图长这样:

食堂打饭版板书图(60 秒可画): 同学打饭: ┌────────────┐ │ 餐盘架 │ (buffer cache) │ 有这道菜? │ └─────┬──────┘ 有 │ 没有 ┌────┴────┐ ▼ ▼ 端走 叫师傅真炒 │ │ └────┬────┘ ▼ 端着菜走

画法步骤(60 秒拆解):

  1. 0:00–0:20 画 1 个大方框,里面写"餐盘架"和"有这道菜?"(20 秒)
  2. 0:20–0:35 从方框画 2 条分叉,一条写"有"、一条写"没有"(15 秒)
  3. 0:35–0:50 在 2 条线终点各写 1 个词("端走"和"叫师傅真炒")(15 秒)
  4. 0:50–1:00 画 1 条汇聚线 + "端着菜走"收尾(10 秒)

4. 你画一遍(必做)

🎯 任务(15 分钟,必做):
  1. 找一张白纸(或者 iPad / 任何能画画的工具)
  2. 拿出你课 2 挑的类比
  3. 用 4 元素模板,画 1 张图(目标 60 秒)
  4. 画完拍张照保存,讲台可以用

画的 3 个检查点:

  • 看图的同学能不能 5 秒内说"哦就是先看有没有"?(懂不懂)
  • 图里有没有出现任何 C 字段(没有就对了,只用生活词)(简不简)
  • 你能不能一边指着图一边讲完整个 2 步流程(顺不顺)
🎯 自检(必做):把图拿给 1 个不懂 FS 的同学看,只说"看 3 秒,然后告诉我这张图在讲什么"。如果对方能说"先看有没有、没有就真去取",就过了。

5. 计时练习(关键!)

🎯 任务(10 分钟,必做):拿手机计时,从你拿起笔开始到画完 1 张图,看用了多少秒。

目标:

  • 60 秒以内 — 合格,讲台够用
  • 40 秒以内 — 优秀,留时间讲
  • 超过 90 秒 — 元素太多或没练熟,继续画到 60 秒内

画 5 遍的进步曲线(预期):

  • 第 1 遍:90+ 秒(还在想位置)
  • 第 2 遍:70 秒(开始顺手)
  • 第 3 遍:60 秒(找到节奏)
  • 第 4 遍:50 秒(可以了)
  • 第 5 遍:40 秒(讲台稳了)
