2. 范例(食堂打饭版)

把模板填好,长这样(供你参考):

[0:00–0:30] 开场

"今天讲 xv6 怎么读一个磁盘块,主战场 bio.c 的 bread() 函数,7 行 C 代码。"

[0:30–1:30] 类比 + 画图

"它做的事,就像你去食堂打饭。"

← 画图(60 秒)

"图里'餐盘架'就是缓冲池(装已经读过的数据),'叫师傅真炒'就是真去磁盘读。"

[1:30–4:00] 看代码边讲

"看这 7 行代码。"

"第 1 行 b = bget(dev, blockno) — 这一步是让系统给我们一个档案袋,先问'餐盘架上有没有'。"

"第 2 行 if(!b->valid) — 这一步就是判断'档案袋里是空的吗'。"

"如果档案袋里有菜(valid 是 1),就直接返回 — 像餐盘架上已经有这道菜,直接端走。"

"如果没有(valid 是 0),就走 virtio_disk_rw(b, 0) — 让师傅真炒一盘。"

"炒完标 b->valid = 1 — 标记'这道菜已经做好了,下次不用再炒'。"

[4:00–4:30] 金句

"总结一下 — 它先查餐盘架,没菜再叫师傅真炒 — 因为 99% 的菜都在架上,这样又快又省。"

[4:30–5:00] 收尾

"这就是 xv6 怎么读一个磁盘块。讲完了,谢谢。"

3. 你写 1 份(必做)

🎯 任务(20 分钟,必做):
  1. 把模板复制到纸上(或者打字到电脑)
  2. 把"[]"里的内容替换成你课 2–5 准备的内容
  3. 大声念 1 遍,听通顺不通顺
  4. 念不顺的地方改词,改到顺为止
🎯 合格的脚本标志:
  • 5 段(开场 / 类比图 / 代码 / 金句 / 收尾)齐全
  • 每一段都有"用类比讲清"的具体词
  • 念 1 遍不需要中途查稿
  • 5 段总和念完 4:30–5:30

4. 真讲 1 遍给同学(关键!)

🎯 任务(30 分钟,必做):
  1. 找一个不懂 FS 的同学 / 家人
  2. 拿白纸 + 笔 + 你的脚本
  3. 看着同学讲 5 分钟(可以看着脚本,但尽量少看)
  4. 讲完问同学 3 个问题:
    • "我刚才在讲什么?"
    • "为什么 xv6 要先查再读?"
    • "你觉得我哪里讲得不清楚?"
  5. 把同学的回答记下来,这就是讲台反馈

对同学的要求:

  • 不打断你,认真听
  • 听完给 1 句"我觉得哪里不太懂"
  • 如果你卡住,不要提醒,让你自己救场(这是真讲台会发生的)

5. 讲完验收(必做)

🎯 任务(10 分钟,必做):用 5 个问题自检,全部答"是"才算过。
  1. ☐ 我能不看稿讲完 5 分钟吗?(看稿 ≤ 2 次算过)
  2. ☐ 我能边画图边讲吗?(画图时间 ≤ 90 秒)
  3. ☐ 同学能说"听懂了"吗?(不需要完全懂,能说出"先查后取"就算懂)
  4. ☐ 老师追问 3 个问题,我能答出 ≥ 1 个吗?(诚实答"不知道"也算过 1 个)
  5. ☐ 我对讲台不焦虑吗?(身体不发抖 / 声音不发颤 / 不想着"赶紧讲完")
全部是:恭喜,你已经会讲 5 分钟了。
1–2 个否:回对应课复习,再讲 1 遍。
3+ 个否:找家人朋友再听 1 遍,把"哪里不行"列出来,针对性补。

🎯 写学习记录(超重要!)

讲完 1 遍后,写 学习记录 0001,这是你最重要的输出。

记录这 4 件事:

  1. 实际讲了多少分钟
  2. 哪个环节最卡(开场/类比/画图/代码/金句/收尾)
  3. 同学答对几个追问
  4. 下次要补什么
