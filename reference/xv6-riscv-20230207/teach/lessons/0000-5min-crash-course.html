🧠 概念 1 · 操作系统是什么

一句话:操作系统 = 电脑的大管家。
你(用户)想让电脑干活,但电脑硬件不会听人话 — 需要大管家翻译。

电脑的硬件其实很笨:CPU 只懂"加减乘除",磁盘只懂"读写磁道",网卡只懂"收发信号"。你要让电脑"打开一个 Word 文档",背后是:大管家让 CPU 跑程序、让磁盘把 Word 文件读出来、让屏幕显示字。这个大管家就是操作系统(英文: Operating System,简称 OS)。

大管家管 4 件事:

  1. CPU — 让多个程序轮流用 CPU(你一边听歌一边写 Word,其实 CPU 是在两个程序之间飞速切换的)
  2. 内存 — 让每个程序有自己独立的"工作台",互不打扰
  3. 磁盘 — 让你的文件存下来,关机也不丢(这就是文件系统)
  4. 设备 — 键盘、屏幕、网卡、打印机等等
不重要的:Windows / macOS / Linux / Android 都是大管家,只是不同的"管家风格"。我们这次讲 xv6,是教学用的极简版大管家。

🧠 概念 2 · 文件系统是什么

一句话:文件系统 = 磁盘的档案柜。
磁盘就是一块能存数据的硬盘(像个大仓库),文件系统告诉你"哪个文件放在仓库的哪个位置"。

你有一个 Word 文件"期末报告.docx",它不是一整块存在磁盘上的 — 它被切成很多小块(每块 1 KB),撒在磁盘各个角落。文件系统就是记着这些小块都撒在哪、文件名叫什么、是谁的、什么时候建的。

当你说"打开期末报告.docx",系统做的事:

  1. 查"档案柜总账本"(叫 superblock)— 看仓库有几格,档案柜怎么分
  2. 查"档案卡"(叫 inode = index node)— 看"期末报告.docx"分成几块、每块在仓库第几格里
  3. 按档案卡的指示,把那些块读出来拼成完整文件
必须记住的 3 个词:
• 磁盘块(block) = 1 KB 的小块,文件系统操作的最小单位
• inode = 档案卡,记着文件的所有信息(大小、位置、权限)
• 目录 = 一张表,把"文件名"映射到"inode 号"

🧠 概念 3 · xv6 是什么

一句话:xv6 = 麻省理工(MIT)写的教学版大管家。
真大管家(Linux)有几千万行代码,看 1 年也看不完;xv6 只有 1 万行,几周就能看完。

xv6 长这样:

  • 真能用 — 你能在 qemu 模拟器里启动它,跑 shell 命令
  • 代码极简 — 进程管理、内存管理、文件系统、设备驱动都齐全,但每个模块只有几百行
  • 和真 Linux 很像 — 接口和 Linux 几乎一样,学了能迁移到 Linux
  • 教学配套好 — MIT 有配套书、配套视频、配套实验

我们这次要讲 xv6 的文件系统部分,对应 3 个文件:

文件干嘛的类比
bio.c把磁盘块读到内存里(缓冲层)档案柜的"取件窗口"
log.c保证改文件时不会"改一半就崩溃"改文件前先在草稿本上改,改完再正式入账
fs.c管 inode、目录、文件操作档案柜本体 + 档案卡
不重要的:RISC-V 是 xv6 跑在的 CPU 架构(类似 x86、ARM,但更简单)。你不需要懂 RISC-V,只要知道"这是另一种 CPU 就行"。

🔍 C 代码长啥样(给你 30 秒打个底)

打开 bio.c 你会看到这种代码:

// 这个函数叫 bread,从磁盘读一个块
struct buf*
bread(uint dev, uint blockno)
{
  struct buf *b;

  b = bget(dev, blockno);          // 1. 先找个"取件窗口"放这块
  if(!b->valid) {                  // 2. 如果窗口里没这块
    virtio_disk_rw(b, 0);          //    真去磁盘读
    b->valid = 1;                  //    标记"现在窗口里有这块了"
  }
  return b;                        // 3. 把窗口交给调用方
}

看不懂没关系。**课 1 会一句一句带你读**。这里只告诉你:这就是 C 代码的样子,不用怕。

3 个你会看到的 C 符号(记住意思就行):
• struct buf* — "一个指向 buf 结构体的指针"(你可以理解为"档案袋的编号")
• b->field — "b 这个档案袋里的 field 这一栏"(相当于 b.field)
• // 开头的灰字 — 注释,人话解释,不是代码

🎯 5 分钟讲台你到底要做什么

一句话:讲 5 分钟 = 用 1 个类比 + 1 张图 + 1 句话讲完一段代码,让同学听懂。
不是:把 100 行代码逐行念给同学听。

很多新手以为"讲源码 = 把每行念一遍"。**这是错的**。念 100 行 5 分钟根本念不完,同学也听不懂。

正确做法是 3 件套:

3 件套是什么为什么
1 句类比用生活场景讲代码在干什么同学能秒懂,能记住
1 张图白板徒手画,关键流程图看着图讲话不紧张
1 句金句"它把短操作和长操作分成两种锁"这就是"为什么这样写"

3 件套齐了,5 分钟就讲完了。**剩下 2 分钟**留给老师追问。

5 分钟的"配比":
• 0:00–0:30 开场 1 句("今天讲 xv6 怎么读磁盘块")
• 0:30–1:30 类比 + 画图
• 1:30–4:00 边看代码边讲(只讲 5–10 行)
• 4:00–4:30 1 句金句收尾
• 4:30–5:00 "讲完了,谢谢"
