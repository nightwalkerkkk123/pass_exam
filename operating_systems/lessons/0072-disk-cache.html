磁盘 cache:OS 在内存里给磁盘开的"影子盘"

次重点,常出选择题 2–3 分。最易混淆点:cache 不在磁盘上,而在内存里。教材出处:5_设备管理/ppt文本.md 幻灯片 226(5.6 节)+ textbook/ch08_磁盘存储器管理.md §8.3。

0·全局线 1·一句话 2·是什么 3·为什么在内存 4·三种替换 5·读写策略 6·vs CPU cache 7·易错 8·自测
考场必背两句话:
① 磁盘 cache(磁盘高速缓存 / Disk Cache / Page Cache)是 OS 在内存中开辟的一块区域,缓存磁盘扇区/物理页数据,目的是消除磁盘和内存的速度差。
② 两种写策略:写回法 (Write Back,延迟写) 是先改缓存,块被换出时才写磁盘,性能高;写直达法 (Write Through) 是同步写缓存和磁盘,数据安全。

🗺 0. 这道题在考什么

🧸 一句话: 磁盘慢得想砸电脑,OS 就在内存里偷偷开个"假盘子"——最近用过的盘块先放这里,下次读就快了 10 万倍。

常见考点:① 辨析 cache 的位置(内存还是磁盘上);② 三种替换算法名称;③ 写回法 vs 写直达;④ 预读与延迟写的目的。

📖 1. 是什么:OS 在内存里开的一块缓冲

定义: 磁盘高速缓存(Disk Cache)是操作系统在主存(内存)中开辟的一块缓冲区,用于缓存最近访问过的磁盘块,以减少磁盘 I/O 次数、提高访问速度。

教材原话(textbook §8.3):"磁盘的速度比内存慢几个数量级,为了提高磁盘的访问速度,减少磁盘 IO 次数,操作系统会在内存中开辟一块缓冲区,缓存最近访问过的磁盘块,叫做磁盘高速缓存(Disk Cache)或者页高速缓存(Page Cache)。"

另一个名字叫页高速缓存(Page Cache)——Linux 内核里 Page Cache 就是 disk cache 的实际实现。

⚠ 常错点:很多同学以为 cache 是磁盘上的一块"高速盘",错!高速缓存一定在内存里,只是替磁盘服务而已,这是名字带"磁盘"二字误导的地方。

📍 2. 为什么不放在磁盘,而放在内存

存储介质访问延迟量级
内存 DRAM~ 100 ns0.0000001 秒
SSD~ 100 μs0.0001 秒(慢 1000 倍)
机械磁盘 HDD~ 10 ms0.01 秒(慢 10 万倍)

差约 105 倍,如果 cache 放在磁盘上,等于"慢车给快车当备用胎",毫无意义。唯一合理的介质是内存(DRAM)。

记忆口诀:"cache 永远跟着速度快的走"——CPU 与内存间有 CPU cache,内存与磁盘间有 disk cache。位置总在靠近 CPU 那一侧。

🔄 3. 三种替换算法(置换算法)

算法思想优点缺点实际使用
FIFO
(先进先出)		淘汰最早进入缓存的块实现最简单可能出现 Belady 异常,性能差几乎不用
LRU
(最近最久未用)		淘汰最长时间没被访问的块性能接近最优需要记录访问时间,开销大近似实现
CLOCK 算法最常用
LFU
(最少使用)		淘汰访问次数最少的块性能接近 LRU需要计数器,开销大;冷启动时新块易被淘汰少见
记忆口诀:"F 最早、L 久、L 次"——FIFO 看进入早晚、LRU 看访问久暂、LFU 看访问次数。

教材 §8.3 还提到 CLOCK 算法,是 LRU 的"近似但便宜"的版本——给每个缓存块加一个引用位,环形扫描,已被使用的位清 0 后跳过,找到第一个引用位为 0 的就淘汰。Linux Page Cache 用的就是 CLOCK 的变种。

✍️ 4. 预读(read-ahead)与延迟写(write-back)

预预读(Read Ahead)
读一个块时,提前把后面相邻的几个块一起读到 cache。依据局部性原理,程序多半会顺序访问。比如 Linux 通常预读 128 KB 或 256 KB。
延延迟写(Write Back / Write Behind)
写时只改 cache 并标脏位,不立即写磁盘;等该块要被换出或后台定期刷新时,才真正写回磁盘。
直写直达(Write Through)
写时同时改 cache 和磁盘,磁盘写完才返回。一致性好,性能差。
写回 Write Back写直达 Write Through
性能高(多次写可合并)低(每次都要 I/O)
安全低(崩溃可能丢脏数据)高(磁盘永远最新)
现代 OS✅ 默认采用仅在关键元数据场景(文件系统日志)
📐 考场默写模板: 
提高磁盘 I/O 速度的主要途径:
1. 选性能好的磁盘 + 好的调度算法
2. 设置磁盘高速缓冲区(磁盘 cache)  ← 在内存里!
3. 提前读 + 延迟写(依据局部性原理)
4. 优化物理块分布(同柱面聚簇)
5. 虚拟盘 / RAID

🆚 5. 与 CPU cache 的区别

CPU Cache(L1/L2/L3)Disk Cache(Page Cache)
位置CPU 芯片内部 / 主板上内存(主存)中
服务加速 CPU ↔ 内存加速 内存 ↔ 磁盘
容量几 KB ~ 几十 MB占空闲内存的相当比例(GB 级)
硬件/软件硬件实现OS 软件管理(也称 page cache)

⚠️ 6. 易错点

坑 1 · 以为 cache 在磁盘上: 名字虽然叫"磁盘"cache,但物理位置在内存。题型最爱这么绕:"以下哪个是 OS 在磁盘上分配的缓冲区?"答案:无,disk cache 在内存。
坑 2 · 替换算法记混: LRU = 时间 L 久(最近最久未用);LFU = 次数(最少使用)。看到"按访问次数淘汰"就是 LFU,不是 LRU。
坑 3 · 写回 vs 写直达选反: 想"性能优先"→ 选写回;想"数据安全"→ 选写直达。现代 OS 默认是写回。教材 §8.3 提的"延迟写"就是写回法的另一种说法(课件幻灯片 292)。

✅ 7. 自测

Q1 · 单选

关于"磁盘高速缓存(Disk Cache)",下列说法正确的是:

A. Disk Cache 是磁盘上的一块保留区域,作为高速备份
B. Disk Cache 的引入是为了弥补内存速度比磁盘慢
C. 写回法在写操作时立即将数据写入磁盘,以保证一致性
D. Disk Cache 位于内存,常用 CLOCK 算法近似实现 LRU 置换

点我看答案

答案:D。
A 错:Disk Cache 在内存,不是磁盘上。
B 错:是弥补磁盘比内存慢(105 倍),主谓反了。
C 错:写回法是不立即写磁盘、延迟到换出/刷新。
D 对:Disk Cache 的实现常用 CLOCK 算法,这是 LRU 的近似版。

Q2 · 简答骨架

请列出"提高磁盘 I/O 速度"的 3 种主要方法,并说明磁盘 cache 在其中扮演什么角色。

点我看参考答案
  1. 选择性能好的磁盘 + 好的磁盘调度算法(FCFS / SSTF / SCAN / CSCAN),减少寻道时间和旋转延迟。
  2. 设置磁盘高速缓存(Disk Cache):在内存中开辟一块缓冲区,缓存最近访问过的磁盘块;配合提前读(依据局部性把邻近块预读到 cache)和延迟写(写时只改 cache、块换出或定期刷新时才写回磁盘)。
  3. 优化物理块分布(同柱面聚簇)+ 虚拟盘(RAM 盘) + RAID 廉价磁盘冗余阵列。

Disk cache 的角色:充当磁盘的"备份仓库",命中时省下磁盘 I/O(微秒级 → 纳秒级),典型命中率可达 90%+(教材 §8.3)。
