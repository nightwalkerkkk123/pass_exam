RR 时间片轮转：剩多少跑多少

目标：把 RR 的 4 步走法、「剩多少跑多少」、完成时间的读法、q 大小的影响，一次讲透。配套数据：A=3 / B=6 / C=4 / D=2 / E=5，全部时刻 0 到达，q=2。

1. 一句话：RR 在干嘛？

RR（Round-Robin，时间片轮转） = 把 CPU 时间切成一段一段（每段叫一个「时间片」q），每个进程轮流上 CPU 跑 最多 q 分钟，时间一到就强制打断送回队尾。

类比：老师给 5 个学生每人发 1 张发言卡（最多讲 2 分钟）。讲不完也掐断，把麦克风递给下一个人。
特点：公平、响应快，但周转时间不是最短（因为反复打断，进程要等很多轮）。

2. 4 步走法（背下来）

  1. 就绪队列按 FCFS 排：[A, B, C, D, E]
  2. 调度队首上 CPU
  3. 让它跑 min(剩余, q) 分钟
  4. 没跑完 → 送回队尾；跑完 → 撤出
关键公式：每次上 CPU 的实际时间 = min(剩余时间, q)
• 剩 ≤ q → 一次跑完（剩多少跑多少，主动让出，不是被时间片打断）
• 剩 > q → 跑满 q 分钟（被时间片打断，送回队尾）

5. 完成时间：怎么读？

完成时间 = 进程最后一次跑完的那个时间格的右边界。
不是「跑完时看钟表是几点」——直接看甘特图，最后一次占用格子的右边界数字。

金句（必背）：
• SJF 的平均周转时间最短（批处理系统首选）
• RR 的响应时间最快（分时系统首选，比如 Windows / Linux 桌面）
• FCFS 最简单但最不公平

8. 时间片 q 的影响

q 大小后果类比
q 太大（> 最长服务时间） 每个进程一次跑完 → 退化成 FCFS 老师让每人讲 1 小时，结果只有 3 个人能讲完
q 太小（如 q=1ms） 切换太频繁 → 切换开销爆炸，CPU 都花在切上下文，真正干活的时间反而少 老师每秒切一次学生，结果一堂课 50 分钟只讲了 3 分钟内容
q 合适（≈ 10-100 ms） 响应快 + 开销小 老师让每人讲 2 分钟，大家都能发言，整体推进顺利
考试陷阱：题里如果说「q 设得非常大」→ 答「退化为 FCFS」，不是「退化为 SJF」！RR 和 SJF 没关系。

9. 自测（做再看答案）

Q1（口述）RR 的 4 步走法是什么？
▸ 答案
1) 就绪队列按 FCFS 排；2) 取队首上 CPU；3) 跑 min(剩, q)；4) 没完送队尾，完则撤出。
Q2（口述）「剩多少跑多少」什么时候触发？触发了会怎样？
▸ 答案
进程剩余时间 ≤ q 时触发。只跑剩余的那些分钟就主动让出 CPU，不会被时间片打断，也不送回队尾，而是直接撤出。
Q3（看图）看上面甘特图，C 的完成时间是多少？怎么读出来的？
▸ 答案
C 完成时间 = 15。C 最后一次占用是 13-15 这一段（正好 2 分钟跑满），右边界是 15。
Q4（选择）RR 时间片 q 设得非常大，会退化成？
A. FCFS   B. SJF   C. 优先级调度   D. 多级反馈队列
▸ 答案
A. FCFS。q 太大 → 每个进程一次跑完 → 没人被切换 → 按到达顺序 = FCFS。注意不是 SJF，RR 永远按到达顺序排队，不会主动挑短作业。
Q5（计算）如果 RR 改成 q=4，重新跑同样的数据（A=3/B=6/C=4/D=2/E=5），D 的完成时间变成多少？
▸ 答案
q=4 时：
• 0-3 A 跑完（剩 0 ≤ 4，完成！时刻 = 3）
• 3-7 B 跑 4（剩 2）→ 送队尾
• 7-9 C 跑 2（剩 2 ≤ 4，跑完！时刻 = 9）
• 9-11 D 跑 2（剩 0，完成！时刻 = 11）
• 11-15 E 跑 4（剩 1）
• 15-17 B 跑 2（剩 0，完成！时刻 = 17）
• 17-18 E 跑 1（剩 0，完成）

D 完成时间 = 11。q 变大 → 每个人能跑更久 → D 不用排那么多次。
Q6（口述）RR 和 SJF 相比，最大的 trade-off 是什么？什么场景下选 RR？
▸ 答案
RR 平均周转时间比 SJF 长（因为反复打断、等待多），但响应时间最快、最公平。
场景：分时系统（如 Windows / Linux 桌面）—— 用户多、每个用户都要快速看到反馈，宁可牺牲一点周转也要保证响应。
