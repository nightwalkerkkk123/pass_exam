第 22 课 · Ch6 文件管理 · 零基础可视化

看得见的「索引分配 → 混合索引 → 容量计算」

这一课全部可以动手点。目标:不靠背,而是看明白。配合 第 20 课讲稿 一起用。

一句话先记住: 文件被切成很多数据块散落在磁盘上。「怎么记住这些块都在哪」,就是分配方式要解决的事。索引分配的办法是:用一张“目录清单”集中记下所有块号。

1. 索引分配 = 一张“快递柜目录单”

🧸 5 岁也懂的比喻: 你有很多包裹散放在快递柜各处。你不用记每个柜子,只要有一张目录单: “第 0 个包裹在 #52 柜,第 1 个在 #9 柜……”。要拿第 I 个包裹?看单子第 I 行,直接走到那个柜子。 这张目录单,就是索引块;每个柜子,就是一个数据块。
👇 点目录单的任意一行,看它“一步”跳到哪个数据块:
索引块(目录单)
磁盘上的数据块(绿色=这个文件的)
← 点左边目录单的某一行试试
为什么索引分配厉害: 想读第 1000 块,不用从头一个个找(那是链接分配的慢), 查目录单第 1000 行就一步拿到块号 → 这叫随机访问一步到位;块散放各处也行 → 没有外部碎片。

2. 难题:一张目录单写不下怎么办?

🧸 比喻: 目录单本身也是一张纸(一个盘块),纸上能写的行数有限。文件太大、块太多,一张单子写不下。 硬撑着给每个文件都配一张超大单子?小文件根本用不满,浪费。

于是聪明的办法出现了:大部分文件很小,给它几行就够;只有少数大文件,才让它“单子套单子”。这就是下一节的混合索引。

3. 混合索引(UNIX inode)= 单子里分四种行

🧸 比喻: 文件的“身份证”叫 inode,上面有几行地址,分四种:
• 直接:这行直接写柜号 → 一步拿到(小文件够用)。
• 一次间接:这行写“详细清单在另一张纸上” → 多翻 1 张纸。
• 二次间接:写“清单的清单” → 多翻 2 张纸。
• 三次间接:多翻 3 张纸(留给超大文件)。
👇 逐个“点亮”各级地址,看结构怎么展开、容量怎么累加(设定:盘块 4KB、块号 4B → 每张清单可写 1024 个块号):

设计精髓(讲给老师的金句): 绝大多数文件都很小,只用直接地址,读一次就到、零额外开销; 只有极少数大文件,才一层层展开间接清单。同一个结构,既让多数小文件快,又让少数大文件大。

4. 计算的根:为什么是 4KB ÷ 4B = 1024?

🧸 比喻: 一张清单纸 = 一个盘块 = 4KB = 4096 字节。纸上写一个块号要占 4 字节。 那这张纸能写几个块号?4096 ÷ 4 = 1024 个。就像“一个能装 4096 颗豆子的罐子,每个块号占 4 颗,能放 1024 个块号”。
一个 4KB 盘块,被切成 1024 个“格子”,每格放一个 4B 块号(下面真的画了 1024 格):

所以一张清单能指向 1024 个块。 记这个数为 P = 1024。下面所有容量,都是 P 在做乘法。

容量怎么一级级乘上去

地址类型怎么算能多管多少块容量
直接 ×1010 块1010×4KB = 40KB
一次间接1 张清单 = P 块1024P×4KB = 4MB
二次间接清单套清单 = P×P 块1,048,576P²×4KB = 4GB
三次间接套三层 = P×P×P 块1,073,741,824P³×4KB = 4TB
抓住这一个规律就够了:每多套一层清单,能管的块数就 × P(×1024)。
4KB → 4MB → 4GB → 4TB,每一步都是 ×1024。这就是“指数级放大”——不用背那串数,理解“每级 ×1024”即可。

试试:把盘块从 4KB 改成 1KB,看 P 变成 256、三次间接从 4TB 缩到 16GB——块越小,能管的文件越小。这就是 xv6(1KB 块)单文件才 268KB 的原因(它还只做到一级间接)。
