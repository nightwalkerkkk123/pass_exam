IO 系统 · CPU 怎么和"慢家伙"打交道

目标：15 分钟装下"中断是啥"+"4 种 IO 控制方式"+"SPOOLing 是啥"——本章全是选择题考点，没有大题

1. 一句话 + 餐厅类比

CPU 很快，IO 设备很慢——OS 要想办法让 CPU 不傻等 IO，但又能在 IO 完事时及时知道。 核心机制 = 中断（Interrupt）。
🚶‍♂️
程序查询（轮询）
服务员一直跑去厨房问"好了吗？"
🔔
中断驱动
菜好了按铃，服务员才来
📦
DMA
厨房自己用推车把菜送到桌
👨‍🍳
通道
整支厨师团队独立作业，全做完才通知
OS 概念餐厅类比
CPU服务员（很快很忙）
IO 设备厨房（很慢）
忙等（轮询）服务员一直跑去看菜好没好
中断（Interrupt）厨房按铃，叫服务员来
DMA 控制器推车（自动送菜到桌）
通道独立厨师长（带一队人做完所有菜）

2. 4 种 IO 控制方式对照表（必背）

方式CPU 怎么"等"设备CPU 干预频率适用
程序查询
（轮询）		 CPU 不停问"好了吗？" 全程忙等 极简单嵌入式
中断驱动 设备按铃叫 CPU 每个字中断 1 次 中低速字符设备
（键盘、鼠标）
DMA DMA 控制器自己搬整块数据 每块中断 1 次 高速块设备
（磁盘、网卡）
通道 通道（专门处理器）独立干完 整序列结束才中断 1 次 大型机/服务器
易记口诀：
• 轮询 = CPU 累死（一直问）
• 中断 = 设备主动按铃（每个字都按）
• DMA = 整块一起搬（每块按一次）
• 通道 = 一队人独立干（整件事按一次）
→ CPU 干预越来越少，并行度越来越高。

3. 缓冲（Buffer）4 种

缓冲 = 在内存里开一块"等候区"，让 CPU 和设备不用互相等。

类型有几块特点
单缓冲1 块设备写 → CPU 读，轮流用
双缓冲2 块设备和 CPU 同时用，效率翻倍
循环缓冲N 块组成环3 个指针，生产者/消费者轮转
缓冲池系统统一管理所有进程共享，最常用
关键 1 秒：缓冲池是现代 OS 的标配（Linux 的页缓存就是缓冲池思路）。
考试会问"哪种缓冲最灵活 / 共享性好"——答：缓冲池。

4. 设备分配的 4 张表

OS 用 4 张表记录"谁有设备、谁有控制器、谁有通道"。

DCT
Device Control Table · 设备控制表
  • 每设备 1 张
  • 记录：设备状态、指向 COCT、等待队列
COCT
Controller Control Table · 控制器表
  • 每控制器 1 张
  • 记录：控制器状态、指向 CHCT
CHCT
Channel Control Table · 通道表
  • 每通道 1 张
  • 记录：通道状态、等待队列
SDT
System Device Table · 系统设备表
  • 整个系统 1 张
  • 记录：所有设备 + 驱动程序入口
易记 1 句话：设备→控制器→通道，从下到上 = DCT→COCT→CHCT，最上面挂一张 SDT 做总目录。

5. SPOOLing（假脱机）—— 重点

问题：打印机这种"独占设备"1 次只能 1 个人用，其他人都在等。

SPOOLing 解法：把数据先存到磁盘"输出井"，让用户进程感觉"我立刻打完了"。后台再慢慢真打。

SPOOLing 三大件：
  1. 输入井 / 输出井：磁盘上的两块"等待区"
  2. 输入/输出缓冲区：内存里的小缓冲
  3. 输入/输出进程：后台两个"搬运工"
典型应用：假脱机打印——多个用户都按"打印"，数据先进输出井排队，用户感觉立即完成；打印机后台慢慢打。
→ 打印机这种"独占"被改造成了"共享"。

6. 必背 4 句话（考试原话）

① 中断是 CPU 和设备"打招呼"的核心机制（程序查询除外）。
② 4 种 IO 控制：轮询（最累）→ 中断（每字）→ DMA（每块）→ 通道（整序列），并行度递增。
③ 4 张表：DCT（每设备）→ COCT（每控制器）→ CHCT（每通道）→ SDT（全系统 1 张）。
④ SPOOLing = 假脱机，用输出井把独占设备改造成共享设备（典型：打印机共享）。

7. 自检（先做再翻答案）

Q1（口述） 什么是"中断"？为什么说"程序查询"不算用中断？
Q2（口述） DMA 和中断驱动有什么区别？为什么磁盘常用 DMA？
Q3（选择） 下面哪个说法正确？
A. 程序查询方式下 CPU 利用率最高
B. 通道方式下 CPU 全程参与数据传输
C. DMA 方式下，数据传输由 DMA 控制器完成，CPU 不参与
D. 中断驱动方式适合高速块设备
Q4（口述） 4 张设备表（DCT / COCT / CHCT / SDT）的关系是？SDT 为什么是"全系统 1 张"？
Q5（口述） SPOOLing 解决什么问题？典型应用是什么？
答案对照
A1：中断 = 设备完成后"按铃"叫 CPU。程序查询是 CPU 主动一直问，不是设备通知，所以不算中断。
A2：
• 中断驱动：每个字都要中断一次，让 CPU 传，CPU 累
• DMA：整块数据由 DMA 控制器自己搬，CPU 不参与，CPU 轻松
磁盘要传的数据量大，每字中断一次CPU 受不了，所以用 DMA。
A3：C。A 错（轮询 CPU 累死）；B 错（通道独立于 CPU）；C 对；D 错（高速块设备用 DMA，字符设备才用中断）。
A4：从设备到通道的层级是 DCT → COCT → CHCT。SDT 是总目录，记录所有设备和它们的 DCT 地址，全系统 1 张就能索引所有设备。
A5：把"独占设备"（如打印机）改造成"共享设备"——用磁盘的"输出井"做中转。典型应用：假脱机打印。
