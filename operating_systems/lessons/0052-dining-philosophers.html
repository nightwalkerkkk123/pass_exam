哲学家进餐问题:一个圆桌讲透"5 人抢 5 根筷子"的死锁与三种解法

这是 0051 读者写者的兄弟问题——一个考死锁与防死锁。你已经在 第 11 课 看过速写,这一节把它拆到每行代码 + 让你亲眼看到死锁现场。教材出处:2_进程管理/ppt文本.md 幻灯片 288–292(2.4.2 节)。

0·全局线 1·画面 2·死锁现场 3·方法 ①限 4 人 4·方法 ②AND 5·方法 ③奇偶 6·三种方法对照 7·自测
一句话先记住: 5 个哲学家围圆桌,每人左右各 1 根筷子(共 5 根),必须拿 2 根才能吃。最朴素算法会死锁——5 人同时各拿左筷、然后都在等右筷。防死锁 3 种思路:①限最多 4 人同时抢 ②AND 信号量(拿两根要拿都拿)③奇偶颠倒顺序。

🗺 0. 全局故事线:这道题在考什么

学校自命题里,经典同步三连(生产者-消费者 / 读者-写者 / 哲学家)选一考,哲学家最常考"3 种防死锁方法"。本质是让你设计打破死锁 4 必要条件(互斥/占有等待/不剥夺/循环等待)的方案。

🧸 一句话: 5 个书呆子围坐圆桌,每两人之间 1 根筷子,吃面必须用 2 根。如果所有人同时伸手去拿自己左手那根,就全卡住了——每人手里都攥着 1 根,都在等右手那根,右手那根在别人手里也攥着——谁也吃不上。

这一节我们做 4 件事:①看圆桌画面 ②亲眼看死锁怎么发生 ③逐个学 3 种防死锁方法 ④做 5 道自测(默写 + 选择 + 改错)。

🎬 1. 画面:5 哲学家的圆桌

设定(教材原话,幻灯片 288-289)

  • 5 位哲学家(编号 P0~P4)围坐一张圆桌。
  • 5 根筷子(编号 C0~C4),每两人之间 1 根——Pi 的左筷是 Ci、右筷是 C(i+1) mod 5。
  • 哲学家思考 → 饿了 → 拿起左右筷 → 吃面 → 放下 → 思考 循环。
  • 筷子是临界资源,同一时刻只能 1 人拿 1 根。
编号映射(默写时不要写反):
· 哲学家 Pi 的左筷 = chopstick[i]
· 哲学家 Pi 的右筷 = chopstick[(i+1) mod 5]
· 例:P3 的左筷是 C3、右筷是 C4;P4 的左筷是 C4、右筷是 C0(因为 5 mod 5 = 0,圆桌回到起点)。

💀 2. 死锁现场:5 人同时各拿左筷

这是教材幻灯片 289 的朴素算法。会死锁。点"▶ 走一遍"看 5 个哲学家同时伸手拿左筷、然后全卡住的现场。

朴素(会死锁)算法

// 5 根筷子,每根初值 1 semaphore chopstick[5] = {1,1,1,1,1}; philosopher_i() { // i = 0,1,2,3,4 repeat think(); // 思考 P(chopstick[i]); // 拿左筷 P(chopstick[(i+1) mod 5]); // 拿右筷 eat(); // 吃面 V(chopstick[i]); // 放左筷 V(chopstick[(i+1) mod 5]); // 放右筷 until false; }
场景所有人伸手拿左筷
每根筷子状态[1] [1] [1] [1] [1]
死锁?否
演示 5 个哲学家同时拿左筷 → 全部卡住 → 谁也吃不上
💀 死锁形成过程(每一步都是同时发生):
1️⃣ 5 个人同时饿了,同时伸手拿左筷 → 5 根筷子都被拿走,每人手里攥着 1 根;
2️⃣ 每个人都试着拿右筷——但右筷都在隔壁的人手里;
3️⃣ 5 人都阻塞在 P(chopstick[(i+1) mod 5]) 上,谁都吃不上,谁都不放手;
4️⃣ 循环等待 + 占有等待 → 死锁(满足死锁四必要条件中的两条)。
教材原话(幻灯片 291):"可采取以下几种解决方法..."

🛡 3. 防死锁方法 ①:至多 4 人同时进餐

思路(教材原话)

"至多只允许有四位哲学家同时去拿左边的筷子,最终能保证至少有一位哲学家能够进餐,并在用毕时能释放出他用过的两只筷子,从而使更多的哲学家能够进餐。"

出处:2_进程管理/ppt文本.md 幻灯片 291 解决方法 (1)

// 多设一个信号量,限制最多 4 人同时争筷子 semaphore chopstick[5] = {1,1,1,1,1}; semaphore count = 4; // 同时进餐人数上限 philosopher_i() { repeat think(); P(count); ① 先抢"进入许可"(最多 4 人) P(chopstick[i]); P(chopstick[(i+1) mod 5]); eat(); V(chopstick[i]); V(chopstick[(i+1) mod 5]); V(count); ② 吃完放许可 until false; }
为什么有效: 最多只允许 4 个人同时伸手,5 根筷子至少剩 1 根没人抢——打破"循环等待"。剩下的那个人能拿到 2 根筷子里至少 1 根,然后引发连锁释放,所有人陆续能吃上。

🛡 4. 防死锁方法 ②:AND 信号量(两根筷子要么都拿、要么都不拿)

思路(教材原话)

"仅当哲学家的左、右两只筷子均可用时,才允许他拿起筷子进餐。"

出处:2_进程管理/ppt文本.md 幻灯片 291 解决方法 (2) + 292 AND 信号量机制

// 用 AND 信号量 Swait:两根筷子"全有才能拿,一根没有就都不拿" semaphore chopstick[5] = {1,1,1,1,1}; philosopher_i() { repeat think(); // Swait 原子申请:两根筷子任意一根不可用 → 整组都不拿 Swait(chopstick[(i+1) mod 5], chopstick[i]); eat(); Ssignal(chopstick[(i+1) mod 5], chopstick[i]); until false; }
为什么有效: Swait 是原子操作——要么把两根筷子都拿到、要么都不拿(不会发生"手里攥 1 根等另 1 根")。直接打破"占有并等待"条件。代价:Swait 不是所有语言都内置,需要硬件/系统支持。

🛡 5. 防死锁方法 ③:奇偶号哲学家拿筷顺序相反

思路(教材原话)

"规定奇数号哲学家先拿他左边的筷子,然后再去拿右边的筷子;而偶数号哲学家则相反。按此规定,将是 1、2 号哲学家竞争 1 号筷子;3、4 号哲学家竞争 3 号筷子。即五位哲学家都先竞争奇数号筷子,获得后,再去竞争偶数号筷子,最后总会有一位哲学家能获得两只筷子而进餐。"

出处:2_进程管理/ppt文本.md 幻灯片 291 解决方法 (3)

// 奇偶颠倒:奇数号先左后右,偶数号先右后左 semaphore chopstick[5] = {1,1,1,1,1}; philosopher_i() { repeat think(); if (i % 2 == 0) { // 偶数号 P0/P2/P4:先右后左 P(chopstick[(i+1) mod 5]); // 先拿右 P(chopstick[i]); // 再拿左 } else { // 奇数号 P1/P3:先左后右 P(chopstick[i]); // 先拿左 P(chopstick[(i+1) mod 5]); // 再拿右 } eat(); V(chopstick[i]); V(chopstick[(i+1) mod 5]); until false; }
为什么有效: 相邻两人一奇一偶,他们的第一根筷子是同一根(奇数号的"先拿左"恰好等于他右边那位偶数号的"先拿右"——因为偶数号 Pi 的右筷是 chopstick[i+1] = 奇数号 Pi+1 的左筷 chopstick[i+1])。必有一人先抢到、另一人拿不到——5 人不可能同时各持 1 根筷子,打破"循环等待"。

✅ 7. 自测:先自己想,再点开答案

Q1 · 为什么朴素算法会死锁?

用一段话描述朴素算法为什么必然死锁。要说清"什么时候卡死、为什么没人能脱身"。
点我看答案

朴素算法死锁的过程:

  1. 5 个哲学家同时饿了,几乎同时执行 P(chopstick[i]) 拿左筷;
  2. 5 根筷子都被拿走,每人手里攥 1 根,没人能吃;
  3. 每个人都接着执行 P(chopstick[(i+1) mod 5]) 想拿右筷,但右筷都在隔壁那位手里(隔壁的人也在等自己的右筷……);
  4. 所有人都阻塞,都在等别人放手;
  5. 没人能吃 → 没人能放下筷子 → 没人能脱身。

对应死锁四条件: 互斥(筷子同一时刻只许 1 人拿) ✓ · 占有并等待(每人手里 1 根等另 1 根) ✓ · 不剥夺(不能从别人手里抢) ✓ · 循环等待(P0 等 P1 的右筷 → P1 等 P2 的右筷 → … → P4 等 P0 的右筷) ✓ ——四条全满足,所以必死锁。

Q2 · 编号映射(默写易错)

哲学家 P3 的左筷和右筷分别是哪一根?
哲学家 P4 的右筷又是哪一根?(提示:5 mod 5 = ?)
点我看答案

· P3 的左筷 = chopstick[3]

· P3 的右筷 = chopstick[(3+1) mod 5] = chopstick[4]

· P4 的右筷 = chopstick[(4+1) mod 5] = chopstick[5 mod 5] = chopstick[0]——圆桌绕回起点!

这正是"圆桌"的关键——P4 的右手边其实是 P0 的左手边,筷子 C0 是 P4 和 P0 共用。

Q3 · 方法 ① 的 count 信号量初值为什么是 4 不是 5?

① 限最多 4 人同时进餐方案里,新加的 count 信号量初值是 4。如果改成 5,会怎样?
点我看答案

初值改成 5 = 没有任何限制。5 人还是能同时各拿 1 根左筷,死锁还会发生。

限 4 人的意义在于:5 根筷子中至少有 1 根没人抢。也就是说至少有 1 个人能拿到他需要的 2 根筷子里的至少 1 根,然后引发连锁释放。所以初值 4 正好,改成 3 或 2 也行,但代价更大(最多 3 人或 2 人同时进餐,效率低)。

Q4 · 默写挑战:奇偶分拿

不看屏幕,在纸上默写方法 ③ 奇偶分拿的完整代码。包括:
  1. 信号量声明
  2. philosopher_i() 函数完整 P/V 序列(用 if (i % 2 == 0) 分支)
点我对答案
semaphore chopstick[5] = {1,1,1,1,1}; philosopher_i() { repeat think(); if (i % 2 == 0) { P(chopstick[(i+1) mod 5]); P(chopstick[i]); } else { P(chopstick[i]); P(chopstick[(i+1) mod 5]); } eat(); V(chopstick[i]); V(chopstick[(i+1) mod 5]); until false; }

能一次写对不漏分支 = 哲学家这题你稳拿。卡壳就在编号映射——回去看 Q2。

Q5 · 打乱判断(易错辨析)

下面哪条说法是正确的?(多选 + 单选都有陷阱,选错了回去读对应章节)
← 选完再点"提交判分"
点我看完整答案

正确答案:C、D(多选)。A 和 B 错。

A 错:"全部人都改成先拿右"——5 人还是同时伸手、同时拿到右筷,然后都等左筷——照样死锁。要打破死锁必须奇偶颠倒(一半人先左、一半人先右),不能所有人都改。

B 错:限 4 人方案的 count 信号量是在门口限人数,本质是让 5 根筷子中至少剩 1 根没人抢,从而打破"循环等待"。它没有改变"占有并等待"——里面 4 个人仍可能各拿 1 根等另 1 根,只是不可能 5 人同时这样,所以不会形成死锁环。

C 对:奇偶分拿的核心巧妙就在这里——相邻两人一奇一偶,他们的"先拿"恰好是同一根筷子。必然一人抢到、另一人没抢到→ 没法 5 人同时各持 1 根 → 循环等待不成立。

D 对:AND 信号量(Swait) 的本质就是"要么都拿、要么都不拿"的原子动作,直接消除了"占有 1 根等另 1 根"的中间状态,打破"占有并等待"。

Q6 · 改错题

下面这段代码号称是"奇偶分拿",但实际会死锁。指出所有错误并修正。
philosopher_i() { repeat think(); if (i % 2 == 0) { P(chopstick[i]); // 偶数号:先左 P(chopstick[(i+1) mod 5]); } else { P(chopstick[i]); // 奇数号:先左 P(chopstick[(i+1) mod 5]); } eat(); V(chopstick[i]); V(chopstick[(i+1) mod 5]); until false; }
点我看答案

错误: if 和 else 两个分支写得一模一样——都先拿左筷再拿右筷,等于所有人都是"先左后右",没有"奇偶颠倒",死锁照旧发生。

正确写法(教材原话:偶数号相反):

if (i % 2 == 0) { // 偶数号 P0/P2/P4:先右后左 P(chopstick[(i+1) mod 5]); P(chopstick[i]); } else { // 奇数号 P1/P3:先左后右 P(chopstick[i]); P(chopstick[(i+1) mod 5]); }

记忆锚: 教材原话"奇数号先左后右,偶数号相反"——"相反"就是把两个 P 的顺序倒过来。如果你写完发现 if/else 长得一样,就是忘了"相反"。
