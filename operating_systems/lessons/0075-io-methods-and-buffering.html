I/O 控制四级 + 缓冲:CPU 越闲越聪明

考场必背两句话:
① I/O 控制 4 级(CPU 介入递减):程序查询 → 中断驱动 → DMA → I/O 通道。
② 缓冲 4 种(适配场景递增):单缓冲 → 双缓冲 → 循环缓冲 → 缓冲池。

🗺 0. 这两节在考什么

🧸 一句话: CPU 是班里最贵的"老师",I/O 是最慢的"差事"。4 级控制 = 老师陪着干的程度;缓冲 = 让差事排队不堵门口。

出题方式:选择题填"哪种方式 CPU 介入最少"或"双缓冲适用于";简答题 5 分让对比 4 级。

📖 1. 是什么:4 级 I/O 控制 = CPU 让位的 4 个阶段

核心矛盾: CPU 速度 ns 级,设备速度 ms 级——CPU 等设备等于"大学教授等小学生写作业"。4 级方式就是逐步把 CPU 从等待中解放出来。

教材原话(幻灯片 197):"在 I/O 设备输入每个数据的过程中,由于无须 CPU 干预,因而可使 CPU 与 I/O 设备并行工作"。

⚡ 2. 4 级控制方式详解

① 程序查询方式(轮询,Programmed I/O)

CPU 一直死循环读设备状态寄存器,直到"就绪"才搬运 1 个字节。100% 占用 CPU。

类比:老师站在差生旁边盯着写作业,啥也干不了。

② 中断驱动方式(Interrupt-Driven I/O)

CPU 发起 I/O 后去干别的;设备每准备好1 个字节就发中断,CPU 来搬一次。CPU 利用率 ~99%(幻灯片 207 给出 100ms 输入 vs 0.1ms 处理的对比)。

类比:差生写完一个字就举手,老师过来收一次,中途可以改其他作业。

③ DMA 方式(直接存储器访问)

DMA 控制器接管总线,以数据块(整块)为单位在设备和内存之间直传;只在开始/结束各中断一次 CPU。CPU 不参与搬运,只参与"开门/关门"。

类比:差生写完一整本作业自己放进老师抽屉,中间不用举手。

DMA 控制器 4 类寄存器(幻灯片 225):命令/状态寄存器 CR、内存地址寄存器 MAR、数据寄存器 DR、数据计数器 DC。

④ I/O 通道方式(Channel)

通道 = 专门跑 I/O 程序的小型处理机。CPU 只发一条"启动 I/O"指令给通道,通道自己跑通道程序,完成一组数据块的传输。CPU、通道、设备三者并行。

类比:差生把作业交给班长(通道),班长自己收齐一摞交给老师。

📊 3. 4 级对比表(必背)

方式传输单位CPU 介入时机CPU 利用率适用设备
程序查询1 字节全程轮询极低(等设备)极慢设备(键盘早期)
中断驱动1 字节每字节中断 1 次高(~99%)低速字符设备
DMA数据块块开始 + 块结束很高高速块设备(磁盘)
I/O 通道一组数据块整组开始 + 整组结束最高大型机外设群
记忆口诀:"字字字块块组"——传输单位从"字(节)"一路升级到"组(块集合)",CPU 介入频次一路降低。

🪣 4. 4 种缓冲(缓冲管理,5.3 节)

缓冲的目的:解决 CPU 和 I/O 设备的速度差——让 CPU 不必等设备,设备不必等 CPU。

缓冲类型结构适用场景关键数字
单缓冲1 个缓冲区CPU 与设备串行每块处理时间 ≈ max(C, T) + M
双缓冲2 个缓冲区交替流水线型 I/O处理时间 ≈ max(C, T)
循环缓冲n 个缓冲组成环突发数据流生产者-消费者同步
缓冲池多缓冲 + 4 类队列系统级,多进程共享3 类缓冲(空/满/装)+ 4 个队列(空/装入/提取/输出)

注:C = CPU 处理一块时间,T = 设备输入一块时间,M = 内存搬运时间。双缓冲能近似"流水线",但若 C ≪ T 仍要等。

记忆口诀:"单双环池"——从简到繁,从专用到共享。缓冲池最强(系统统一管理多进程),其它都是单进程私用。

🖨 5. SPOOLing 简化版(链 0057)

SPOOLing(外围设备联机并行操作)是缓冲思想的高级应用:用磁盘模拟多个"虚拟"独占设备,把独占设备改造成可共享。

经典例子:打印机本来一次只能打一份;SPOOLing 在磁盘开输出井,所有任务排队写入,打印机按队列慢慢打。

三件套:输入井/输出井(磁盘缓冲) + 输入缓冲/输出缓冲(内存缓冲) + 输入进程/输出进程(搬运)。

详细题解见 0057 SPOOLing——本节不展开。

⚠️ 6. 易错点 + 必背 4 句话

必背 4 句话:
① 4 级方式按 CPU 介入度递减排:程序查询 > 中断 > DMA > 通道。
② DMA 的传输单位是"数据块",中断驱动是"字节"——别混。
③ 缓冲 4 种:单、双、循环、缓冲池,适用场景由简到繁。
④ SPOOLing 用磁盘模拟独占设备,实现"假共享",本质是缓冲的高级用法。
坑 1 · 顺序写反: 不少同学把 4 级顺序写成"通道→DMA→中断→程序查询"。记住是"CPU 介入越来越多"的方向反推。
坑 2 · DMA 与中断驱动混: 中断驱动每字节中断 1 次,DMA 每数据块只中断 1~2 次——中断次数差几个数量级。
坑 3 · 缓冲池当成循环缓冲: 循环缓冲是单进程私有环;缓冲池是系统级共享池(含空队列、输入队列、输出队列、缓冲队列 4 个),别混。

✅ 7. 自测

Q1 · 4 级排序

请按 "CPU 介入度从高到低" 排出 4 级 I/O 控制方式,并说出每级传输单位。

点我对答案

答案:程序查询(字节,全程轮询)→ 中断驱动(字节,每字节中断)→ DMA(数据块,首尾各中断 1 次)→ I/O 通道(数据块组,启动 + 结束共 2 次)。

记忆:介入多→介入少,传输单位由细到粗。

Q2 · 易错辨析

下面哪个说法是错的?

A. 程序查询方式下 CPU 利用率最低
B. DMA 方式的数据传输单位是字节
C. 缓冲池用于多进程共享,循环缓冲用于单进程
D. SPOOLing 用磁盘模拟独占设备,实现共享

点我看答案

答案:B。DMA 的传输单位是"数据块",不是字节——字节是中断驱动的单位。C 项混淆点:缓冲池确实是多进程共享,循环缓冲单进程。
