磁盘调度算法：FCFS/SSTF/SCAN/C-SCAN 磁头移动计算

大题 10 分核心考点——计算磁头移动总磁道数和平均寻道长度。教材出处：汤小丹第8章。

考场必背公式:
① 磁头移动总距离 = |终点 - 起点| 的累加
② 平均寻道长度 = 总移动磁道数 / 访问请求数
③ SCAN(电梯): 先往一个方向走到头，再反向
④ C-SCAN(循环扫描): 往一个方向走到头，然后直接跳回起点继续同向

🗺 0. 这道大题在考什么

🧸 一句话: 硬盘有很多同心圆磁道，磁头移动很费时间。不同"先去哪边"的策略，总移动距离差很多——考你手算对比！

📖 1. 四种调度算法对比

算法中文名策略特点
FCFS先来先服务按请求顺序访问最公平，但磁头可能来回跳，效率低
SSTF最短寻道优先找离当前磁头最近的请求效率高，但可能饥饿(远处请求永远等)
SCAN电梯算法向一个方向移动，顺路服务像电梯，到顶再下来，较均衡
C-SCAN循环扫描单向移动，到头直接跳回比SCAN更公平，跳回时不服务

📖 2. 例题详解

题目

磁盘请求队列: 55, 58, 39, 18, 90, 160, 150, 38, 184

磁头当前位置: 100，方向: 向大磁道号移动。磁道范围: 0-199

计算 FCFS、SSTF、SCAN、C-SCAN 的总移动磁道数。

FCFS (先来先服务)

访问顺序: 100 → 55 → 58 → 39 → 18 → 90 → 160 → 150 → 38 → 184 移动: |55-100| + |58-55| + |39-58| + |18-39| + |90-18| + |160-90| + |150-160| + |38-150| + |184-38| = 45 + 3 + 19 + 21 + 72 + 70 + 10 + 112 + 146 = 498 磁道

SSTF (最短寻道优先)

当前100，最近的是90(差10)，然后58(差32)，然后55(差3)... 访问顺序: 100 → 90 → 58 → 55 → 39 → 38 → 18 → 150 → 160 → 184 总移动: 10+32+3+16+1+20+132+10+24 = 268 磁道

SCAN (电梯，向大号方向)

向大号走到头(199)，顺路服务: 100 → 150 → 160 → 184 → 199(到头) 然后反向: 199 → 90 → 58 → 55 → 39 → 38 → 18 注意: SCAN通常走到最远请求(184)或磁道末尾(199)，不同教材定义不同！ 按"走到最远请求184再反向": 100 → 150 → 160 → 184 → 90 → 58 → 55 → 39 → 38 → 18 总移动: 50+10+24+94+32+3+16+1+20 = 270 磁道 (或走到199是282)

C-SCAN (循环扫描)

向大号走到头(199)，然后直接跳回0，再从0往大号走服务剩余请求 100 → 150 → 160 → 184 → 199 → 跳回0 → 18 → 38 → 39 → 55 → 58 → 90 总移动: 50+10+24+15 + 199 + 18+20+1+16+3+32 = 408 磁道

⚠️ 3. 易错点

坑 1 · SCAN的终点: 有的教材走到最远请求就反向，有的走到磁道末端(199)。看题目给的范围！
坑 2 · C-SCAN的跳回不服务: 从199跳回0的过程中不服务任何请求，这点与SCAN不同。
坑 3 · 绝对值别漏: |a-b| 是距离，别算成 a-b 导致负数！

📝 4. 真题演练

真题

磁头当前在50，请求: 95, 180, 34, 119, 11, 123, 62, 64。用SSTF计算总移动磁道数。

答案

50最近的是62(差12)→64(差2)→34(差30)→11(差23)→95(差84)→119(差24)→123(差4)→180(差57)

总移动: 12+2+30+23+84+24+4+57 = 236 磁道
