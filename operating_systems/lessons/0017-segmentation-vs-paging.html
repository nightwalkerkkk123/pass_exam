分段 vs 分页 · 段页式(一维 vs 二维)

这一课只回答一件事:分页和分段到底差在哪?先动手拆一个地址,再上术语。配合 速查地图 使用。

一句话先记住: 分页是“物理切分、一个数就能定位”——地址是一维的;分段是“逻辑切分、必须给两个数”——地址是二维的;段页式就是把两者叠起来:先分段,每段再分页,兼得逻辑清晰与高利用率。

看懂这一件事,本节就懂一半: 分页只要你给一个数(因为页大小固定,系统自己就能除出页号和偏移)→ 一维;分段必须你给两个数(段长不固定,不告诉它段号根本不知道你落在哪个段)→ 二维。

🧸 2. 具体例子(ELI5):一维 vs 二维

🧸 分页 = 一本页码连续的书。 每页字数一样多(比如每页 4096 个字)。你说“我要第 8196 个字符”,系统一除就知道:8196 ÷ 4096 = 第 2 页、余 4 → 第 2 页的第 4 个字。一个数就够了,不用你再报“第几页”。
🧸 分段 = 一栋楼里长短不一的房间。 每个房间大小都不同,有的 600 步长、有的 8000 步长。你只说“第 3000 号位置”没用——因为不知道到底落在哪个房间。你必须说清两个数:“几号房 + 房内第几步”。这就是二维。
🧸 段页式 = 先分部门、每部门再分格子间。 先按逻辑分“部门(段)”,每个部门内部再切成一样大的“格子间(页)”。找东西要说三样:哪个部门(段号)→ 部门里第几个格子(页号)→ 格子里第几步(页内偏移)。

⚖️ 3. 对照表:分段 vs 分页(照教材口径)

比什么分页(Paging)分段(Segmentation)
划分角度 信息的物理单位;为提高内存利用率、满足OS 管理需要 信息的逻辑单位;一段是一组完整信息;满足用户编程需要
大小是否固定 固定,由系统决定(如 4KB) 不固定,由段的实际内容 / 用户决定
地址维度 一维(给一个数即可) 二维(必须给段号 + 段内偏移)
碎片 有内部碎片,无外部碎片 有外部碎片,无内部碎片
共享与保护 不便按逻辑模块共享 / 保护 便于段级共享 / 保护,适合动态链接
访问效率 / 定位 变换快,适合底层管理 更灵活,适合用户,稍慢

📖 4. 是什么:精确定义(对齐教材)

分页: 逻辑地址 = [页号 | 页内偏移];页号 = 地址 ÷ 页大小(整除),偏移 = 地址 mod 页大小;页表:页号 → 物理块号;物理地址 = 块号 × 页大小 + 偏移。消灭外部碎片,但有内部碎片。
项目分段
逻辑地址[段号 | 段内偏移]
段表记录(段号、段长 limit、段基址 base)
物理地址段基址 + 段内偏移(先检查 偏移 < 段长,否则越界中断)
碎片 / 特点有外部碎片、无内部碎片;便于共享、保护、动态链接
⚠️ 最容易错的陷阱: 段表项存的是「段基址」(段的起始物理地址),不是块号;物理地址是「基址 + 偏移」相加,不是「块号 × 大小」相乘。相乘是分页的算法,别张冠李戴。
段页式: 先分段、每段再分页;逻辑地址三部分 = [段号 | 页号 | 页内偏移];每个进程一个段表、每个段一个页表;段表项记录(页表起始地址、页表长度);地址变换:段表 → 页表 → 物理块号,物理地址 = 块号 × 页大小 + 偏移。不用快表需 3 次访存(段表→页表→数据)。
优点 = 结合两者(逻辑清晰、便于共享保护 + 内存利用率高、无外部碎片);缺点 = 复杂、开销大、需要更多硬件支持。

❓ 5. 为什么:一层层往下挖

第 1 层:为什么要分页?

连续分配会产生外部碎片(内存被切得七零八落,总量够却拼不出一整块),想消除又得靠紧凑(搬来搬去),开销很大。于是改用离散分配:把内存切成固定大小的页框,程序也切成同样大的页,哪里空放哪里。页大小固定 → 彻底消灭外部碎片,大幅提高内存利用率。

第 2 层:分页够好了,为什么还要分段?

分页是为了 OS 管理方便的物理切分,它不符合用户的逻辑视角:一个函数 / 模块可能正好被页边界劈成两半,横跨两页。这样很不利于按逻辑模块去共享和保护(你想共享“这个函数”,却发现它散在页里)。分段按逻辑意义切——一段就是一个完整模块,于是便于共享、保护、动态链接。

第 3 层:为什么现代系统用段页式?

因为我们两样都想要:既要分段的逻辑视角与共享保护,又要分页的高内存利用率、无外部碎片 → 那就结合:先分段满足逻辑,再对每段分页满足利用率。代价是结构复杂 + 多次访存(段表→页表→数据),所以用快表(TLB)来补偿访存开销。

✅ 6. 自测:先自己想,再点开答案

选择题 1

分页系统的逻辑地址是几维的?

A. 二维
B. 三维
C. 一维
D. 由页大小决定,不固定

点我看答案

答案:C。 分页只需给一个线性地址,系统靠固定页大小自己就能除出页号和偏移,所以是一维。A 错:二维是分段的特征(要给段号+偏移);B 错:三维是段页式(段号+页号+偏移);D 错:维度和页大小无关,分页永远是一维。

选择题 2

“有内部碎片、无外部碎片”说的是分页还是分段?

A. 分页
B. 分段
C. 两者都有内部碎片
D. 两者都没有任何碎片

点我看答案

答案:A。 分页:页大小固定,最后一页可能装不满 → 有内部碎片;但页框大小一致、哪里空放哪里 → 无外部碎片。分段正好相反:段长按需分配、装得满 → 无内部碎片,但段间空隙拼不成整块 → 有外部碎片。所以 B 说反了;C、D 与事实不符。

选择题 3

段页式的逻辑地址三部分,正确的顺序是?

A. 页号 → 段号 → 页内偏移
B. 页内偏移 → 页号 → 段号
C. 段号 → 页内偏移 → 页号
D. 段号 → 页号 → 页内偏移

点我看答案

答案:D。 段页式是“先分段、每段再分页”,所以从大到小依次是 段号 → 页号 → 页内偏移。A 错:段号必须在最前(先定段才能找该段的页表);B 是完全倒过来;C 把页内偏移和页号次序颠倒了。

选择题 4

分段系统里,段表项存的是什么、物理地址怎么算?

A. 存物理块号;物理地址 = 块号 × 页大小 + 偏移
B. 存段基址和段长;物理地址 = 段基址 + 段内偏移(相加)
C. 存段基址;物理地址 = 段基址 × 段长 + 偏移
D. 存页号;物理地址 = 页号 × 段长 + 偏移

点我看答案

答案:B。 段表项记录(段长 limit、段基址 base),定位是相加:先查 偏移 < 段长,再 物理地址 = 段基址 + 段内偏移。A 是最毒的陷阱——那是分页的算法(块号 × 页大小 + 偏移,相乘),别把分段和分页搞混:分段是相加、分页才是相乘。C 的“基址 × 段长”纯属乱凑;D 段表里根本不存页号。
