1先亮 inode：混合索引的"本体"

数据结构kernel/fs.h
#define NDIRECT 12                  // 直接地址项:12 个
#define NINDIRECT (BSIZE / sizeof(uint)) // 一个间接块能装的块号数 = 1024/4 = 256
#define MAXFILE (NDIRECT + NINDIRECT)    // 单文件最大块数 = 12 + 256 = 268

struct dinode {
  short type;            // 文件类型(目录/文件/设备)
  ...
  uint size;             // 文件大小(字节)
  uint addrs[NDIRECT+1];  // ★13 个地址:前 12 直接,第 13 个=一次间接
};
🎤 怎么说

"这是文件在磁盘上的'身份证'——inode。看最后一行 addrs[13]:前 12 格直接装数据块号，小文件够用了；第 13 格不装数据，装一个'间接块'的块号，那个间接块里再装 256 个数据块号，留给大文件。课本画的'直接地址 + 一次间接'，在 C 里就是这一个数组。"

2直接块分支：小块号，一步到位

直接kernel/fs.c · bmap()
static uint
bmap(struct inode *ip, uint bn)
{
  uint addr, *a;
  struct buf *bp;

    if(bn < NDIRECT){            // ① 块号 < 12 → 走直接
    if((addr = ip->addrs[bn]) == 0){ // 这一格还没分配?
      addr = balloc(ip->dev);       // 找一个空闲磁盘块
      if(addr == 0) return 0;       // 磁盘满
      ip->addrs[bn] = addr;          // 记进 inode
    }
    return addr;                    // 直接返回:跳 1 次
  }
  bn -= NDIRECT;                     // ② 不是小块号? 减掉 12 继续看间接
🎤 怎么说

"先看块号小不小。如果 bn < 12，就是直接块:去 addrs[bn] 取门牌号。如果这格是 0（还没分配过），就 balloc 找一个空闲块、记下来，否则直接返回。注意这里只跳了一次就拿到了地址。"

👉 指着 bn -= NDIRECT 说:"块号 ≥ 12 的，先减掉 12，剩下的去间接块里数。"

3间接块分支：大块号，跳两次（重点）

间接kernel/fs.c · bmap()
  if(bn < NINDIRECT){
    // 取间接块(没有就分配)
    if((addr = ip->addrs[NDIRECT]) == 0){
      addr = balloc(ip->dev);
      ip->addrs[NDIRECT] = addr;
    }
        bp = bread(ip->dev, addr);       // ★第1跳:把间接块读进内存
    a = (uint*)bp->data;            // 它内容是一串块号 a[0..255]
        if((addr = a[bn]) == 0){          // ★第2跳:在清单里挑第 bn 个
      addr = balloc(ip->dev);
      if(addr){ a[bn] = addr; log_write(bp); }
    }
    brelse(bp);
    return addr;
  }
  panic("bmap: out of range");
}
🎤 怎么说（这段最容易被追问，慢讲）

"块号 ≥ 12，走间接。先拿到那个间接块的门牌号（没有就分配一个）。关键是这两行:bread 把间接块读进内存——这是第 1 跳；读进来发现它不是数据，是一串块号 a[0..255]，我再用 a[bn] 在清单里挑第 bn 个——这是第 2 跳，才拿到真正的数据块号。"

👉 一句收口:"所以直接块跳 1 次、间接块跳 2 次，多的这一跳就是大文件的代价。"

4板书：1 图 + 1 类比 + 1 句设计意图

✍️ 上台就画这一张
  • 图:画一个 13 格的方块（前 12 标"直接"，第 13 标"间接"）→ 第 13 格画箭头指向另一个长条块（标 [200,201,…256 个]）→ 长条块里某格再画箭头指向"数据"。直接格直接连"数据"。
  • 类比:"addrs[5]=#9 像'你包裹在 9 号柜'，直接去拿；addrs[12]=#300 像'你的包裹清单贴在 300 号柜门上'，先去 300 号看清单，再按清单去拿——清单本身也占一个柜子。"
  • 设计意图（1 句）:"为常见情况优化——绝大多数文件很小，直接块让它们一跳到位；间接块只在文件变大时才动用，省空间又能撑大。"

5答辩 Q&A（老师最可能问的 5 个）

Q1一个文件最大能多大?怎么算的?
答法: "最大块数 = 直接 12 + 一次间接 256 = 268 块；每块 1KB，所以约 268KB。256 是因为一个 1KB 的间接块能装 1024÷4 = 256 个 4 字节块号。" 真实 UNIX 还有二次/三次间接，能到 GB~TB。
Q2addrs[12] 里存的是数据还是别的?
答法: "不是数据，是一个索引块的块号。读进来是一串 256 个块号，要再挑一个、再读一次才到数据。"
Q3为什么不全用间接、把直接块也省掉?
答法: "因为绝大多数文件小。保留直接块让小文件少一次磁盘 I/O——这是'为常见情况优化'的取舍。"
Q4balloc 是干嘛的?空闲块怎么管理?
答法: "balloc 找一个空闲磁盘块。xv6 用位示图（bitmap）记每个块空/占——对应课本'空闲块管理'里的位示图法。" （若学过成组链接法可补:"课本另一种是成组链接法。"）
Q5这段代码和教材哪个知识点对应?
答法: "对应 Ch6 文件的物理结构/外存分配方式里的索引分配，更准确说是混合索引（直接+间接），也是本学期实验⑤ 文件管理的核心。"
