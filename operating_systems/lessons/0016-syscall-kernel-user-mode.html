穿墙记:用户态 / 核心态 与 系统调用(trap)

你的程序想碰硬件,得先敲门。配合 第 14 课「OS 是什么」 与 第 15 课「四大特征」 一起看。

一句话先记住: CPU 有两个「档位」——用户态(目态)跑你的应用、够不到硬件;核心态(管态)跑操作系统内核、能碰一切硬件。中间隔着一堵墙,墙上只有一道门,叫系统调用(trap)。想让内核帮忙,必须走这道门。

可视化的题眼: 用户态那一侧永远够不到右边的硬件。想碰硬件,只能让小圆点走过这道门、变成红色(进核心态)去干活,干完再变回蓝色回来。这趟路线就是 用户态 → 核心态 → 用户态。

🧸 2. 具体例子:住客不能自己进后厨

5 岁也懂的比喻: 你的程序想读一个文件,可它不能自己跑去拨磁盘——就像餐厅住客不能自己冲进后厨炒菜。
它只能按铃(trap)喊服务员(内核):「帮我 read 一下!」服务员进后厨(核心态)替你把菜端出来(操作磁盘、拿到数据),再递回到你桌上(返回用户态)。
后厨的刀火电(硬件)只有服务员能碰;你只能隔着门请求。这道「按铃请服务员」的机制,就是系统调用。

📖 4. 是什么(逐字对齐教材口径)

五大功能的精确表述

① 处理机管理:以进程为单位——进程控制、进程同步、进程通信、调度。
② 存储器管理:内存分配、内存保护、地址映射、内存扩充。
③ 设备管理:缓冲管理、设备分配、设备处理、设备独立性。
④ 文件管理:存储空间管理、目录管理、文件读写、共享与保护——实现按名存取。
⑤ 用户接口:命令接口、程序接口(即系统调用)、图形接口。

两种运行态

核心态(Kernel Mode,又称管态、特权态): CPU 可执行所有指令(含特权指令),可访问所有硬件资源,OS 内核运行于此。
用户态(User Mode,又称目态): CPU 只能执行非特权指令,不能直接访问硬件,只能访问自己的内存空间,应用程序运行于此。

特权指令 & 模式切换 & 系统调用

特权指令: 只能在核心态执行的指令(如 I/O 指令、关中断、修改页表 / 系统寄存器);用户态执行特权指令会触发异常并被 OS 终止。
模式切换: 用户态切到核心态的唯一途径是中断、异常或系统调用;核心态可直接切回用户态。
系统调用: 用户态程序请求 OS 服务的接口,是应用程序获取 OS 服务的唯一途径。
系统调用执行过程: 用户程序调用库函数 → 产生陷入指令 trap → CPU 从用户态切到核心态 → 进入系统调用处理程序执行内核服务 → 返回用户态。
(即路线为「用户态 → 核心态 → 用户态」;而系统调用代码本身运行在核心态。)

❓ 5. 为什么(往下深挖三层)

第 1 层 · 为什么要分用户态 / 核心态两种模式?

为了保护。如果谁都能直接执行 I/O、关中断、改页表这类危险操作,一个写错的应用就能把整个系统搞崩、甚至偷看别的进程内存。于是把「能碰硬件的钥匙」全部锁进核心态,应用只能在用户态跑受限指令。

第 2 层 · 为什么系统调用是获取 OS 服务的「唯一途径」?

因为应用在用户态够不到硬件和内核数据,它想读盘、开进程,只能通过这道受控的门请求内核代劳。而内核借这道门做权限检查、参数检查——门是关卡,不是随便进出的过道。既然墙外做不了,门又是唯一入口,系统调用自然成了唯一途径。

第 3 层 · 为什么切核心态只能靠中断 / 异常 / 系统调用,不能想跳就跳?

如果程序能自己一句指令就跳进核心态,那「保护」就彻底失效了——任何程序都能随意进内核。所以只留这三个入口,而且它们都会把控制权交给内核预先设定好的处理程序(受控入口),程序无法指定「跳到内核的哪一行」。这就是为什么只有内核能把自己降回用户态,反方向的门却上了锁。

一句话收束: 双态 + 特权指令 + 唯一受控入口,三件事合起来只做一件事——把危险操作关进笼子,再只开一扇带门卫的门。这就是操作系统「运行机制」的地基。

✅ 6. 自测(先想再翻答案)

第 1 题(高频)· 系统调用的执行过程(即处理程序代码)运行在什么态?

  1. 核心态
  2. 始终在用户态
  3. 先核心态,执行到一半切回用户态执行
  4. 与调用它的程序同一个态,不发生切换
点我看答案

答案:A(核心态)。 注意区分两件事:整趟路线是「用户态 → 核心态 → 用户态」;但系统调用处理程序这段代码本身是在核心态执行的——它要碰硬件,只能在核心态跑。陷阱选项 C/D 就是把「路线」和「代码在哪跑」混为一谈。

第 2 题(高频)· 下列哪一组,全部都是「用户态 → 核心态」的合法途径?

  1. 调用普通函数、做加法、赋值语句
  2. 系统调用、跳转指令 jmp、函数返回
  3. 关中断指令、进程调度、时间片用完
  4. 中断、异常、系统调用
点我看答案

答案:D(中断、异常、系统调用)。 用户态切核心态只有这三条路,而且都不是程序自己主动跳的——普通函数调用、加法、jmp 都留在用户态;关中断本身是特权指令,用户态根本执行不了(会触发异常被终止)。

第 3 题 · 下列哪一条是特权指令(只能在核心态执行)?

  1. 两个寄存器做加法
  2. 关中断
  3. 把常数写入一个普通变量
  4. 比较两个数的大小
点我看答案

答案:B(关中断)。 关中断、I/O 指令、修改页表 / 系统寄存器都属于特权指令,用户态执行会触发异常并被 OS 终止;而加法、赋值、比较都是非特权指令,用户态就能跑。

第 4 题 · 应用程序获取操作系统服务的唯一途径是?

  1. 直接执行 I/O 指令去操作硬件
  2. 把自己切换到核心态后随意调用内核函数
  3. 系统调用(程序接口)
  4. 修改页表把内核内存映射进来读
点我看答案

答案:C(系统调用 / 程序接口)。 A、D 是特权操作,用户态做不到;B 里「应用自己切核心态」根本不存在——切换只能靠中断 / 异常 / 系统调用,而且入口由内核掌控。所以唯一途径就是系统调用。
