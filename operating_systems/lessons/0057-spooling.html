SPOOLing = 用磁盘做缓冲,把独占设备改造成可共享的虚拟设备

这是简答题 5 分大题的标准答案骨架——全称 + 四大组成 + 打印流程 + 虚拟设备。教材出处:5_设备管理/ppt文本.md 幻灯片 203-211(5.4.5 节)。背景见 0008 I/O 系统、0047 I/O 控制方式、0048 缓冲与 SPOOLing 专题。

考场必背两句话:
① SPOOLing(Simultaneous Peripheral Operations On-Line) = 假脱机,在磁盘上开井当缓冲、用输入/输出进程模拟外围机,把独占设备改造为可共享的虚拟设备。
② SPOOLing 系统由 4 部分组成:输入井和输出井(磁盘上)、输入缓冲区和输出缓冲区(内存中)、输入进程 SPi 和输出进程 SPo、请求打印队列。

🗺 0. 这道简答题在考什么

🧸 一句话: 打印机太慢、用户太多——你提交打印任务不用等打印完,系统把你的任务"先存起来,排到队里,打印机一好就取出来打"。这就是 SPOOLing = 排队取号 + 仓库缓冲。

5 分题的标准答法:全称与作用(1 分)+ 四大组成(2 分)+ 打印机工作流程(1 分)+ 核心作用是"独占→共享的虚拟设备"(1 分)。

📖 1. 是什么:SPOOLing = 假脱机

全称: Simultaneous Peripheral Operations On-Line(联机/在线外围设备同时操作)。

中文: 假脱机操作。"脱机"指"脱离主机的控制用专门的外围机操作";"假"指这种外围机是用进程/磁盘模拟的,而不是真的专用硬件。

教材原话(幻灯片 203): "SPOOLing 指假脱机操作——由系统中的两个专门负责 I/O 的进程,模拟 I/O 外围机的功能,实现(假)脱机输入/输出。"

核心思想: 在主机和独占的慢速 I/O 设备之间插一层磁盘缓冲,让主机把"数据送往 / 取自设备"变成"数据送到 / 取自磁盘",设备再异步慢慢读 / 写。

🧩 2. SPOOLing 系统四大组成

组成部分所在位置作用对应教材
① 输入井 / 输出井磁盘上的两个存储区· 输入井 = 模拟脱机输入的磁盘,收容 I/O 设备输入的数据
· 输出井 = 模拟脱机输出的磁盘,收容用户进程的输出数据		幻灯片 206
② 输入缓冲区 / 输出缓冲区内存中的两个缓冲区· 输入缓冲区:暂存输入设备送来的数据,再传到输入井
· 输出缓冲区:暂存从输出井送来的数据,再送输出设备		幻灯片 207
③ 输入进程 SPi / 输出进程 SPo系统中的两个专用进程· SPi 模拟脱机输入外围控制机:从输入设备→输入缓冲区→输入井
· SPo 模拟脱机输出外围控制机:从输出井→输出缓冲区→输出设备		幻灯片 208
④ 请求打印队列内核 / 共享表登记用户进程的打印 / 输出请求(包含请求号、文件位置等)幻灯片 211
记忆口诀: "井在盘、缓在存、SP 进程在 CPU、队列在内核"——按"从慢到快"四层就背下来了。

🖨 3. 以打印机为例的工作流程(以输出为主)

// 用 SPOOLing 把"打印"分两步走 // ① 用户进程"提交打印"——立刻返回,不真等打印完成 用户进程: request_print(data) → 向输出井申请空闲盘块 → 把 data 写入输出井 → 在"请求打印队列"登记一张请求表 → 立即 return (用户进程继续跑) // ② SPo 输出进程异步慢慢打印 SPo 进程: while (打印机空闲 && 请求队列非空) { 取队首的请求表; 从输出井读出该请求的数据 → 到输出缓冲区(内存); 启动打印机打印缓冲区内容; 打印完成 → 释放输出井盘块 → 从队列摘下请求; }

要点: 用户进程几乎瞬间得到返回(只写盘);真正的 I/O 由 SPo 在后台串行完成。这就是"把同步变异步、把独占变共享"的关键设计。

✨ 4. 核心作用:把独占设备改造成可共享的虚拟设备

🧸 类比: 一台打印机(物理) 通过 SPOOLing 看起来像 N 台独立打印机(虚拟)——N 个用户同时发打印请求,都不用等,N 份数据都乖乖在输出井排队,一台物理机一台台地打。

教材原话(幻灯片 209): "系统特点——提高 I/O 速度;将独占设备改造为共享设备;实现了虚拟设备功能。"

定义分层:

  • 独占设备: 一段时间只允许一个用户使用(如早期的打印机、磁带机);
  • 共享设备: 可同时允许多个用户使用(如磁盘);
  • 虚拟设备: 采用虚拟技术把一台物理设备变换为多个逻辑上独立的设备(SPOOLing 是经典实现手段)。
📐 考场默写模板: 
SPOOLing(假脱机)系统由 4 部分组成:
① 输入井和输出井:磁盘上的两个缓冲区
② 输入缓冲区和输出缓冲区:内存中的两个缓冲区
③ 输入进程 SPi 和输出进程 SPo:模拟脱机外围控制机
④ 请求打印队列:用户打印请求登记表

核心作用:把独占设备改造为可共享的虚拟设备(以打印机为例,
一台物理打印机通过 SPOOLing 看起来像 N 台虚拟打印机,
各用户请求进入输出井排队,由 SPo 进程异步串行输出)。

📝 5. 必背 4 句话

① SPOOLing = Simultaneous Peripheral Operations On-Line = 假脱机操作。
② 系统由 4 部分组成:输入/输出井(磁盘)+ 输入/输出缓冲区(内存)+ SPi/SPo 进程 + 请求队列。
③ 核心作用:把独占设备改造为可共享的虚拟设备。
④ 用户请求"瞬间完成"(写盘),真正 I/O 由 SPo 进程异步完成,把同步变异步。

⚠️ 6. 易错点

坑 1 · "假脱机=真的脱机": "假"指用磁盘和进程模拟的脱机——不是真的硬件外围机。注意区分,老师爱考"SPOOLing 是否需要专用硬件"。
坑 2 · "虚拟设备 = 物理设备": 虚拟设备是逻辑上的"看起来像 N 台",不是物理上变成 N 台;物理依然只有 1 台打印机。虚拟设备通过 SPOOLing 队列+缓冲区实现。
坑 3 · 漏写四大组成: 答"四大组成"必须把 4 块写全——只写"井和缓冲区"会被扣分(还差 SPi/SPo 进程 和 请求队列)。记忆口诀 "井、缓、SP、队"。
坑 4 · 把"缓冲区"和"井"混: 缓冲区在内存(小、快、临时);井在磁盘(大、慢、中长期)。这也是常考点。

📝 7. 真题样例(简答 5 分)

样题

什么是 SPOOLing 技术?它由哪几部分组成?为什么说 SPOOLing 技术可以实现虚拟设备?以打印机为例说明。(5 分)

点我看参考答案

答:

(1) 定义: SPOOLing(Simultaneous Peripheral Operations On-Line,中文译为假脱机操作),是一种用磁盘做缓冲、用专门 I/O 进程模拟外围机的技术,可以在联机条件下实现脱机 I/O 的效果。

(2) 组成(4 部分):

  1. 输入井和输出井:在磁盘上开辟的两个存储区,用于收容输入/输出的数据;
  2. 输入缓冲区和输出缓冲区:在内存中开辟的两个缓冲区,作为磁盘和设备间中转;
  3. 输入进程 SPi 和输出进程 SPo:模拟脱机外围控制机的两个专用 I/O 进程;
  4. 请求打印队列:用于登记用户进程的打印请求。

(3) 虚拟设备(以打印机为例): 打印机是独占设备——若直接用,用户进程必须阻塞到打印完成。通过 SPOOLing,用户的打印数据先写入输出井(磁盘)、请求进入打印队列,用户进程立刻返回做别的事;SPo 输出进程在打印机空闲时从输出井取数据送输出缓冲区并打印。这样,N 个用户进程"同时"提交打印,系统把它们都排队到输出井,由 SPo 异步串行处理;一台物理打印机对用户来说表现为 N 台虚拟打印机供 N 个用户并发使用,即将独占设备改造成了可共享的虚拟设备。

出处:5_设备管理/ppt文本.md 幻灯片 203、206-211。关键词:4 部分 + 虚拟设备。

Q2 · 易错辨析

下列说法哪个是错的?

A. SPOOLing 的"假脱机"指用磁盘和进程模拟的外围机,不是真的硬件外围设备
B. SPOOLing 输出井和输出缓冲区都在内存中,用于暂存待打印的数据
C. SPOOLing 系统的核心作用是把独占设备改造为可共享的虚拟设备
D. SPOOLing 的请求打印队列登记的是用户进程的打印请求

点我看答案

答案:B。B 错在"都在内存中"——只有输出缓冲区在内存,输出井在磁盘。其它三项完全正确。

Q3 · 打印全过程

描述用 SPOOLing 进行打印的完整流程(从用户调用 request_print() 到打印机真正吐出内容)。

点我看答案

1. 用户进程调用 request_print(data);

2. 系统从输出井申请空闲盘块,把 data 写入输出井;

3. 系统在请求打印队列末尾登记一张请求表(请求号、输出井位置等),request_print() 立即返回(用户进程继续);

4. SPo 输出进程检测到打印机空闲 / 有新请求,取队首请求表;

5. SPo 根据请求表定位输出井盘块,把数据读到输出缓冲区(内存);

6. 启动打印机打印缓冲区内容;

7. 打印完成 → 释放输出井盘块 → 从请求队列摘下请求表。

其中第 2-3 步对用户 "瞬间完成";4-7 是 SPo 在后台异步进行——这就是"同步变异步"。
