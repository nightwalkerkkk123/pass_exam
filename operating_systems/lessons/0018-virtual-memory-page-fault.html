虚拟存储器 · 缺页中断 · 抖动与工作集

这一课回答:内存装不下大程序时,系统靠什么把它跑起来?配合 速查地图 使用。

一句话先记住: 内存装不下整个大程序,靠程序局部性只把当前用得着的页装进来——这就是虚拟存储器;访问到一个没装进来的页,就触发缺页中断去外存把它调进来;如果装得太少,就会不停地换入换出(抖动);用工作集来估计一个进程至少要装多少页才不会抖。

🖼 1. 直觉图:「缺页中断处理流程模拟器」

引用串 [0, 5, 1, 5, 9, 0] 按顺序访问,只有 3 个物理块,满了用 FIFO(先进先出)置换。页 5 是会被写过的脏页。点【访问下一页】一步步看缺页中断怎么处理:

📋 页表(逻辑页)
页号状态位物理块号修改位
💾 内存物理块(3 块)
命中 0 次 · 缺页 0 次
← 点【访问下一页】开始:引用串 [0, 5, 1, 5, 9, 0],3 块,FIFO 置换。
看懂这台模拟器,本节就懂一半: 每访问一页,先看状态位——是 1 就命中直接用;是 0 就缺页中断去调页。调页时若没空闲块要置换,置换前看牺牲页的修改位:脏页先写回外存,干净页直接丢弃省一次 IO。最后都要重新执行被打断的那条指令。

🧸 2. 具体例子(ELI5):小书桌与大书架

🧸 5 岁也懂的比喻: 你的书桌很小,只能同时摊开 3 本书(这是内存);旁边的书架很大,装得下所有书(这是外存/磁盘)。
你正在做的题只翻那几本书,摊桌上就够用了——这就是局部性,不用把所有书都搬上桌。
要用一本没在桌上的书时,你得停下来走到书架去取——这一停就是缺页中断。
桌子已经满了,得先收走一本(置换):如果那本你写过批注(脏页),得先把批注抄回书架再收;如果没写过(干净页),直接放回去就行,省事。
要是桌子实在太小,你不停地取书、收书,几乎没时间真正做题——这就是 抖动。

⚖️ 3. 对照表(照教材口径)

对照一:常规存储器 vs 虚拟存储器

比什么常规存储器虚拟存储器
装入方式 一次性:运行前把整个作业全部装入内存 多次性:分成多次、用到才调入(最重要的特征)
驻留情况 驻留性:装入后一直占着内存直到运行结束 对换性:暂时不用的页换出外存,需要时再换入
本质效果 受内存容量限制,程序不能大于内存 虚拟性:从逻辑上扩充了内存(是前两者带来的结果)

对照二:请求分页页表里的各个控制位

控制位别名作用
状态位存在位0 = 该页不在内存;1 = 在内存(访问它是命中还是缺页,看这一位)
访问位引用位记录该页近期是否被访问过,供置换算法(如 LRU/CLOCK)挑牺牲页用
修改位脏位该页调入后有没有被改过。改过 → 换出时要写回外存;没改过 → 可直接丢弃
保护位—该页的读 / 写 / 执行访问权限
外存地址—该页在磁盘上的位置,缺页时按它去外存找这一页

📖 4. 是什么:精确定义(对齐教材)

虚拟存储器定义: 是一种具有请求调入功能和置换功能、能从逻辑上对内存容量加以扩充的存储器系统。它的逻辑容量 = 内存容量 + 外存容量之和,运行速度接近内存,而成本接近外存。

实现基础: 程序局部性原理——程序在执行时,在一小段时间内只会用到程序的一小部分(代码/数据),所以没必要把整个程序全装进内存。

虚存的四个特征:

特征含义
多次性(最重要)作业分成多次调入内存,而不是一次全装入
对换性暂不用的页可以换出外存,需要时再换入
虚拟性从逻辑上扩充了内存容量(前三者的最终目的)
离散性程序在内存中是离散(不连续)存放的,这是前三者的基础

实现方式: 请求分页(最常用)/ 请求分段 / 请求段页式;都必须采用离散分配。

★ 缺页中断处理过程(5 步,简答高频):
① 保护 CPU 现场(保存被中断指令的状态);
② 按页表中的外存地址,在外存中找到该页;
③ 若内存有空闲块 → 调入并修改状态位;若无空闲块 → 调用页面置换算法选牺牲页(牺牲页是脏页要先写回外存,干净页可直接淘汰);
④ 把该页调入内存、修改页表(填物理块号、状态位置 1);
⑤ 恢复现场,重新执行刚才被中断的那条指令。

缺页中断 vs 普通中断(区别是考点):

比什么普通中断缺页中断
产生时机一条指令执行完之后才响应在一条指令执行期间产生
一条指令产生次数至多一次可能产生多次(一条指令跨多页)
类型可为外部中断是内部中断(故障 fault)
处理完做什么执行下一条指令重新执行当前(被中断的)指令
缺页率: f = 缺页次数 F ÷ 总访问次数 A。
影响因素: 页面越大 → 缺页率越低;分配的物理块数越多 → 越低;置换算法好坏;以及程序本身的局部性好坏。

❓ 5. 为什么:逐层深挖三层

第 1 层:为什么要虚拟存储器?

因为大程序可能超过内存总容量,按常规"一次性全装入"的办法,它根本跑不起来。
而且常规方式的"一次性 + 驻留性"很浪费:一个进程大部分时间在等 I/O、在阻塞,却把整个身子一直占着内存不放;更亏的是,程序里有大量分支、异常处理代码这次运行根本用不到,却也被装了进来。凭什么把用不到的部分也塞进宝贵的内存?

第 2 层:为什么"只装一部分"还能正常跑?

靠程序局部性原理:
• 时间局部性——刚访问过的东西,很快还会再访问(比如循环体);
• 空间局部性——访问了某个地址,附近的地址也很可能马上被访问(比如顺序执行、数组遍历)。
所以在一小段时间内,程序只在一小片页上打转。只要这一小片"当前用的页"在内存里就够跑;偶尔缺了一页,再临时调进来就是了。

第 3 层:为什么会抖动?怎么防?

"只装一部分"是有代价的:如果装得太少 → 频繁缺页 → 频繁换入换出 → CPU 大部分时间都花在处理缺页和置换上,真正执行的有效指令极少 → 这就是抖动(颠簸),CPU 利用率趋近于 0。
根本原因是:分配给进程的物理块数 < 它的工作集。
用工作集模型来救:它统计进程在最近一段时间窗口内实际访问过的页集合,以此估计进程至少需要多少页。只要分配的物理块数 ≥ 工作集大小,就能防抖;若所有进程的工作集之和 > 总物理块数,就说明多道程序度太高,要挂起部分进程 / 降低多道程序度。

注:置换算法(OPT / FIFO / LRU / CLOCK、以及 Belady 异常)的具体计算不在本课,本课只需知道"块满了会置换""算法不当会加剧抖动"。算法细算见 → 第 6 课(页面置换算法)。

✅ 6. 自测:先自己想,再点开答案

选择题 1

虚拟存储器能够实现的基础是下面哪一项?

A. 内存容量足够大
B. 程序局部性原理
C. CPU 主频足够高
D. 磁盘速度接近内存

点我看答案

答案:B。 正因为程序执行有时间/空间局部性——一小段时间只用一小片页——才能"只装一部分"还跑得动。A 错:内存要是真够大就不需要虚存了,这是它要解决的问题不是基础;C 错:主频快只影响速度,和"能不能只装一部分"无关;D 错:磁盘再快也是外存,不是虚存成立的前提。

选择题 2

请求分页页表里的"修改位(脏位)"主要用来?

A. 表示该页当前是否在内存
B. 供置换算法参考该页近期是否被访问过
C. 表示该页的读/写/执行权限
D. 决定该页换出时要不要写回外存

点我看答案

答案:D。 修改位记录该页调入后有没有被改过:改过就得先写回外存,没改过可直接丢弃(省一次 IO)。A 是状态位(存在位)的作用;B 是访问位(引用位)的作用;C 是保护位的作用——四个位各司其职,别混。

选择题 3

缺页中断与普通中断相比,关键区别是?

A. 缺页在一条指令执行期间产生,处理完要重新执行当前指令
B. 缺页只会在一条指令执行完之后才产生
C. 缺页处理完后执行的是下一条指令
D. 缺页属于外部中断,普通中断才是内部中断

点我看答案

答案:A。 缺页是内部中断(故障),在指令执行中途发现所需页不在内存时产生,调页后必须重新执行这条指令;一条指令还可能缺页多次。B、C 说的都是普通中断的特点(执行完才响应、处理完执行下一条),正好说反;D 把内/外部中断也弄反了。

选择题 4

系统频繁换入换出页面、CPU 利用率趋近于 0,这是什么现象?根因是?

A. 死锁;根因是进程间循环等待资源
B. 缺页中断;根因是页表太大放不下
C. 抖动(颠簸);根因是分配的物理块数 < 进程工作集(进程太多、每个分到的块太少)
D. Belady 异常;根因是用了 FIFO 置换算法

点我看答案

答案:C。 这就是抖动/颠簸:块给得太少 → 频繁缺页置换 → 有效指令极少。A 死锁是"互相等资源卡住",不是频繁换页;B 单次缺页是正常现象,不是"利用率趋 0"的现象名;D Belady 异常指"块变多缺页反而变多"的反常计数现象,不是这里描述的整体崩溃。
