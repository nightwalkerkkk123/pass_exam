经典同步问题 · 哲学家进餐 + 读者-写者

目标：15-20 分钟装下 2 个经典问题 + 3 种防死锁方法 + 读者优先伪代码

1. 一句话 + 圆桌类比

同步问题 = "N 个人抢 N 个东西，谁抢不到就阻塞"。 哲学家问题考死锁；读者-写者问题考互斥和优先权。
共通思路（和 L10 同源）：
① 找资源 → 设信号量 S
② 申请顺序：先资源后互斥
③ P/V 配对，方向对

2. 哲学家进餐问题

2.1 设定

场景：5 个哲学家围圆桌，5 根筷子（每两人之间 1 根）。每人左右各 1 根，必须拿 2 根才能吃。

🧑
哲学 0
🧑
哲学 1
🧑
哲学 2
↑ 上方 3 人
↓ 下方 2 人
🧑
哲学 4
🧑
哲学 3
-

死锁场景（5 人同时饿）：

  • 哲学 0 拿左筷 0，等右筷 1
  • 哲学 1 拿左筷 1，等右筷 2
  • ……
  • 哲学 4 拿左筷 4，等右筷 0

⛔ 5 人都在等"右筷"，永远等不到 → 死锁

2.2 3 种防死锁方法（必背）

方法核心思路代价
① 最多 4 人同时进餐 设信号量 count=4，P(count) 进门。保证至少 1 人能同时拿 2 根。 最多浪费 1 个位置
② 奇偶分拿 奇数号哲学家先拿左再拿右；偶数号反过来。 相邻 2 人必有一人先拿右，必有一根筷子留空
③ 同时拿左右 用一个 mutex 锁住"拿 2 根"这个动作，要么都拿，要么都不拿。 拿筷子是"原子"操作，实现稍复杂
考试答法（任选一种）：
方案 ② 最常考——把"先拿左再拿右"按奇偶颠倒，相邻 2 人中必有一人先拿右，必有一根筷子留空，必然不会 5 人同时持 1 根。

2.3 方案 ② 伪代码（奇偶分拿）

semaphore chopstick[5] = {1,1,1,1,1};  // 5 根筷子

philosopher_i() {  // i = 0,1,2,3,4
    while(true) {
        think();
        if (i % 2 == 0) {     // 偶数号：先拿左再拿右
            P(chopstick[i]);
            P(chopstick[(i+1) % 5]);
        } else {              // 奇数号：先拿右再拿左
            P(chopstick[(i+1) % 5]);
            P(chopstick[i]);
        }
        eat();
        V(chopstick[i]);
        V(chopstick[(i+1) % 5]);
    }
}

3. 读者-写者问题

3.1 设定

  • 读者：只读文件，多个读者可同时读（互不冲突）
  • 写者：改文件，必须独占（写时不能有别人）
  • 关键：读者之间不互斥，写者与任何人互斥
👀
读者（>=1 个）
图书馆借书：
多个人同时看一本书 OK
✍️
写者（只能 1 个）
图书馆改书：
改的时候不能有别人在场

3.2 2 个信号量（必背）

信号量初值作用
rmutex1保护读者计数 readcount
wmutex1写者与"第一个/最后一个读者"互斥

3.3 读者优先伪代码（标准答案）

semaphore rmutex = 1, wmutex = 1;
int readcount = 0;

reader() {
    while(true) {
        P(rmutex);              // 锁住"改 readcount"的动作
        if (readcount == 0)
            P(wmutex);           // 第 1 个读者：把写者挡门外
        readcount++;
        V(rmutex);

        read_file();                  // 真正读

        P(rmutex);
        readcount--;
        if (readcount == 0)
            V(wmutex);           // 最后一个读者：放写者进来
        V(rmutex);
    }
}

writer() {
    while(true) {
        P(wmutex);                // 门外排队等
        write_file();                  // 真正写
        V(wmutex);
    }
}
关键 1 秒：
• 第 1 个读者进门时 P(wmutex)——告诉写者"我在读"
• 最后 1 个读者出门时 V(wmutex)——告诉写者"没人了"
• 中间的读者只改 readcount，不碰 wmutex（实现"读者之间不互斥"）
→ 写者一直要等"readcount 变 0"，可能饿死（读者优先的代价）

4. 必背 4 句话（考试原话）

① 哲学家问题核心是防死锁，3 种方法：最多 4 人 / 奇偶分拿 / 同时拿左右。
② 读者-写者核心是多读单写，用 readcount 计数 + 2 个 mutex。
③ 读者优先 → 写者可能饿死；写者优先 → 读者可能饿死。
④ P/V 配对，方向对，先资源后互斥。

5. 自检（先做再翻答案）

Q1（口述） 哲学家问题为什么会死锁？5 人同时饿的具体过程是什么？
Q2（口述） 哲学家问题"奇偶分拿"为什么能防死锁？
Q3（口述） 读者-写者问题中"rmutex"和"wmutex"各自保护什么？
Q4（默写） 默写读者优先的核心 4 句：
① 第 1 个读者进门时 ________
② 最后一个读者出门时 ________
③ 中间读者只改 ________
④ 写者只 ________
Q5（选择） 关于读者-写者，下面哪个说法正确？
A. 多个读者之间需要互斥
B. 读者优先策略下，写者可能饿死
C. 写者可以和读者同时操作
D. rmutex 初值应为 0
答案对照
A1：5 人同时饿 → 每人拿左筷 → 每人在等右筷 → 右筷都在别人手里 → 永远等不到 → 死锁。
A2：奇数号先拿右，偶数号先拿左。相邻 2 人（一奇一偶）必有一人先拿右，这根筷子被先抢走，另一人拿不到这根，从而无法"5 人各持 1 根"形成死锁环。
A3：
• rmutex：保护 readcount 这个共享变量（多个读者同时改 count 会乱）
• wmutex：让写者与"正在读的人"互斥
A4：
① P(wmutex) 把写者挡门外
② V(wmutex) 放写者进来
③ readcount（不改 wmutex）
④ P(wmutex) 等、V(wmutex) 放
A5：B。A 错（读者之间不互斥）；B 对（读者优先的代价）；C 错（写者必须独占）；D 错（rmutex 初值 1）。

6. 完整流程速记（手写 PV 时按这 4 步走）

已知：N 哲学家 / 读者写者

① 找资源 → 信号量（S 初值 = 资源数）
② 区分互斥/资源
   互斥：mutex 初值 1
   资源：S 初值 = 资源数
③ 申请顺序：先资源后互斥（除非有特殊安排，如读者优先的 rmutex 套 wmutex）
④ P/V 配对
