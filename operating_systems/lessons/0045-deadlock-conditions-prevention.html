第 45 课 · Ch3 处理机调度(死锁) · 零基础可视化

死锁四必要条件 + 死锁预防:破坏任一条件,路口就通了

这一课只回答一件事:死锁靠哪四个条件同时撑住,以及砍掉其中任意一个怎样把它掐死在源头。配合 银行家算法(避免) 一起看。

0·全局线 1·先看画面 2·ELI5 3·预防vs避免vs检测 4·定义+四条件 5·必要≠充分 6·自测
一句话先记住: 死锁 = 互斥 + 占有并等待 + 不可剥夺 + 循环等待 四个条件同时成立;死锁预防 = 提前砍掉其中任意一个,让死锁根本无法成型。

🗺 0. 全局故事线:这些点怎么串起来

别把下面这些当成孤立知识点——它们是"资源分配不当→可能卡死"这一个问题一路逼出来的。

竞争资源多个进程抢有限的不可剥夺资源
→
推进顺序不当请求/释放次序错开
→
死锁(僵局)四条件同时成立,无外力谁也走不了
→
预防事先破坏四条件之一
→
避免/检测放宽限制,运行时判安全 / 事后再查

🧸 2. ELI5:死锁到底是什么(附反例)

🧸 像给 5 岁小孩讲: 两个小朋友做手工,小明拿着剪刀想要胶水,小红拿着胶水想要剪刀。两人都攥着手里的不放、又都伸手等对方的——就这么一直僵着,手工永远做不完。这就是死锁:互相等着对方手里的东西,谁也不先松手。(教材定义见下节)
⚠️ 反例:如果没有"循环等待"会怎样? 假设规定谁都必须先拿剪刀、再拿胶水。那么小明拿到剪刀后小红根本拿不到剪刀,只能等——但小明能顺利拿胶水做完、再把剪刀还出来。等待关系变成一条直线而不是一个环,僵局就散了。抽掉四条件里的任何一根柱子,死锁这座桥就塌了。

⚖️ 3. 对照表:预防 vs 避免 vs 检测,别混成一锅

🧸 一句话: 预防=事前立死规矩(砍条件);避免=事中每次分配前先算一卦(银行家/安全状态);检测=事后允许出事再派人排查。三者时机不同、松紧不同。

⚖️ 三种处理方法对照(照教材口径)

方法时机思路代表手段限制/代价
死锁预防事前破坏四必要条件之一,从根本杜绝一次性申请 / 剥夺 / 按序申请限制最严,资源利用率最低
死锁避免事中(每次分配前)不破坏条件,但拒绝进入不安全状态银行家算法限制较宽,利用率较高;要预知最大需求
死锁检测事后允许死锁发生,再检测并解除资源分配图化简/死锁定理限制最宽;要额外检测+解除(剥夺/撤销)开销
结论(考试口径一句话): 预防"事先破坏条件",避免"运行中判断安全状态",检测"事后发现再解除"——银行家算法属于"避免",不属于"预防",这是最爱考的坑。

📖 4. 是什么:死锁定义 + 四个必要条件

死锁的定义(教材原文): 死锁是指多个进程在运行过程中因争夺资源而造成的一种僵局,当进程处于这种僵持状态时,若无外力作用,它们都将无法再向前推进。

课本把"产生死锁的原因"归为三类:竞争不可抢占性资源、竞争可消耗资源(如消息)、进程推进顺序不当(请求/释放次序错)。而下面四条,是死锁成立的四个必要条件——缺一不可:

四个必要条件(口诀:互 · 占 · 不 · 环):
  1. 互斥条件:进程对分到的资源排他性使用,一段时间内某资源只由一个进程占用。
  2. 请求和保持条件(占有并等待):进程已保持至少一个资源,又提出新请求,而新资源被别人占用,于是被阻塞,但对已占资源保持不放。
  3. 不可剥夺条件(不可抢占):进程已获得的资源,未用完前不能被别人抢走,只能由它自己用完后释放。
  4. 循环等待条件:存在一条进程-资源的循环链,链中每个进程占有的资源正被链中下一个进程请求(等待成环)。
⚠️ 易错: 别把"死锁的三个原因"和"四个必要条件"搞混——原因是"为什么会发生",条件是"发生时一定同时满足的四个特征"。题目问哪个,就答哪个。

❓ 5. 为什么:为什么是"必要"不是"充分"?破坏它们各要付什么代价?

第 1 层:什么叫"必要条件"?为什么四条全中还只是"可能"死锁?

"必要条件"的逻辑含义是:死锁一旦发生 ⇒ 这四条一定都成立(有死锁 → 有这四条)。反过来不一定成立:四条全满足,只是具备了死锁的土壤,系统可能死锁,也可能因为进程恰好错开时序而顺利跑完。所以它是必要不充分条件。

⚠️ 你两次答错的点: "四条件全满足 → 一定死锁" 是错的。正确说法:"四条件全满足是死锁的必要条件,死锁则四条件必全满足;但四条件全满足未必立刻死锁"。判定"是否真的死锁"要靠资源分配图能否完全化简(死锁定理)——那才是充要条件。

第 2 层:既然是"必要"条件,为什么砍掉一个就能防住死锁?

因为"必要"意味着死锁 → 四条件全成立。取它的逆否命题:只要有一条不成立 → 一定不死锁。这就是死锁预防的全部数学依据——不用四条全砍,砍一根柱子桥就塌。上面路口交互件里"关一个开关就通车",演示的正是这个逆否逻辑。

第 3 层:那为什么现实里通常不砍"互斥",而砍另外三条?各有什么代价?

因为互斥往往是资源的固有属性——打印机、磁带机天生不能两个进程同时写,硬把它变共享会出错。教材明说:互斥一般不能破坏,预防主要破坏其余三条。每种破坏都有代价(下表就是路口交互件解锁时弹出的那几句):

📐 四种破坏策略 · 摒弃哪条 · 代价(考试高频)

摒弃的条件预防策略优点缺点/代价
互斥把独占资源改造成可共享(SPOOLing/虚拟化/缓冲)—多数资源天生独占,一般无法破坏
请求和保持
(占有并等待)		一次性申请运行所需全部资源,拿不齐就一个都不占简单、安全、易实现资源利用率低、作业需等资源齐才启动,延迟运行/易饥饿
不可剥夺申请新资源得不到满足时,立即释放已占的全部资源,日后重新申请可避免死锁实现复杂、前段工作失效、反复申请、性能下降
循环等待资源统一编号,进程必须按序号递增申请(线性排序)利用率/吞吐量比前两种好序号需稳定、限制新设备加入、限制编程自由
📐 遇到"如何预防死锁"大题的答题模板(4 步):

  2. 说明互斥一般不破坏(资源固有属性)。
  3. 逐一给出破坏另外三条的具体做法:一次性申请 / 得不到就释放已占 / 按序号递增申请。
  4. 每条补一句代价(利用率低 / 前段失效 / 限制编程),体现"预防限制严、利用率低"。

✅ 6. 自测:先自己想,再点开答案

选择题 1 · 必要 vs 充分(你的老坑)

关于死锁的四个必要条件,下列说法正确的是?

A. 四个条件只要满足其中一个,系统就一定发生死锁
B. 四个条件全部满足,系统就一定立刻发生死锁
C. 四个条件全部满足只是可能死锁,它们是死锁的必要不充分条件
D. 破坏其中任一条件,系统吞吐量一定随之提高

点我看答案

答案:C。 "必要条件"= 死锁发生则四条必全中(死锁→四条),但反向不成立,四条全中只是有死锁风险,故为必要不充分。
A 错:满足一个远不够,四条必须同时成立;
B 错:全满足只是"可能",能否真死锁要看资源分配图能否化简(死锁定理),不是"一定立刻";
D 错:破坏条件是为防死锁,吞吐量往往下降(如一次性申请利用率低),不会"一定提高"。

选择题 2 · 策略配对

"进程在运行前必须一次性申请它需要的全部资源,拿不齐就一个都不分配"——这种死锁预防策略破坏的是哪个条件?

A. 循环等待条件
B. 请求和保持条件(占有并等待)
C. 不可剥夺条件
D. 互斥条件

点我看答案

答案:B。 "一次性申请全部、否则不占"让进程不会出现"已占一部分又等另一部分"的局面,正是摒弃"请求和保持(占有并等待)"。
A 错:破坏循环等待是"按资源序号递增申请";
C 错:破坏不可剥夺是"申请不到就释放已占资源";
D 错:破坏互斥是把独占资源改共享,且一般无法做到。

选择题 3 · 预防/避免/检测分工

关于处理死锁的三种方法,下列说法正确的是?

A. 银行家算法属于死锁预防
B. 死锁检测是在分配资源之前就杜绝死锁
C. 死锁预防和死锁避免本质是同一件事
D. 预防事先破坏必要条件杜绝死锁;避免(银行家算法)在运行时判断安全状态;检测允许死锁发生后再查并解除

点我看答案

答案:D。 三者时机不同:预防(事前砍条件)、避免(事中每次分配前判安全)、检测(事后发现再解除)。
A 错:银行家算法是避免,不是预防(最爱考的坑);
B 错:检测是允许死锁发生后再查,不是事前杜绝;
C 错:预防破坏必要条件、限制严;避免不破坏条件、只拒绝进入不安全状态,二者不同。

选择题 4 · 代价配对

破坏"不可剥夺"条件(申请新资源得不到满足时,释放已占有的全部资源)的主要代价是?

A. 前段工作失效、需反复申请、系统性能下降
B. 资源利用率低、进程需等资源齐才启动而延迟运行
C. 资源序号需稳定,限制新设备加入、限制用户编程自由
D. 每次分配都要运行安全性算法,系统开销大

点我看答案

答案:A。 反复"释放已占→重新申请"会让之前做过的工作作废、反复申请、性能下降。
B 错:那是破坏"占有并等待(一次性申请)"的代价;
C 错:那是破坏"循环等待(按序申请)"的代价;
D 错:那是银行家算法(避免)的代价,不属于预防。
