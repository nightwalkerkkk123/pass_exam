分页与分段地址计算：页号·页内偏移·段号·段内地址

大题 10 分核心考点——逻辑地址 → 物理地址的转换计算。教材出处：汤小丹《计算机操作系统》第4章。

考场必背公式:
① 分页: 逻辑地址 = 页号 × 页大小 + 页内偏移 ; 物理地址 = 块号 × 页大小 + 页内偏移
② 分段: 逻辑地址 = 段号 + 段内地址 ; 物理地址 = 段基址 + 段内地址 (段内地址 < 段长)

🗺 0. 这道大题在考什么

🧸 一句话: 程序用的是逻辑地址(从0开始编)，但内存里实际存在物理地址。分页/分段就是两种"地址翻译"机制——大题考你怎么算！

📖 1. 分页地址计算

1.1 地址结构

逻辑地址(32位示例)组成计算
页号 P高20位P = 逻辑地址 / 页大小
页内偏移 W低12位W = 逻辑地址 % 页大小 (即取模)

页大小通常 4KB = 2^12，所以页内偏移占12位，可表示 0~4095。

1.2 地址转换公式

// 已知: 逻辑地址 LA, 页大小 PS, 页表(页号→块号) 页号 P = LA / PS // 整数除法 页内偏移 W = LA % PS // 取模 查页表得块号 B 物理地址 PA = B × PS + W

1.3 例题

题: 页大小 1KB，逻辑地址 2170，页表：0→2, 1→3, 2→1。求物理地址。

点我看解答

页大小 1KB = 1024

页号 P = 2170 / 1024 = 2

页内偏移 W = 2170 % 1024 = 122

查页表：页号2 → 块号1

物理地址 = 1 × 1024 + 122 = 1146

📖 2. 分段地址计算

2.1 地址结构

逻辑地址组成说明
段号 S高位部分指示用哪个段
段内地址 d低位部分必须 < 段长，否则越界中断

2.2 地址转换

// 查段表得: 段基址 base, 段长 limit if (段内地址 d >= 段长 limit) 越界中断; 物理地址 PA = 段基址 base + 段内地址 d;

2.3 例题

题: 段表：0号段(基址1000, 长度500), 1号段(基址2500, 长度1000)。逻辑地址[1, 300]的物理地址？

点我看解答

段号=1，段内地址=300

检查：300 < 1000(段长)，合法

物理地址 = 2500 + 300 = 2800

⚠️ 3. 易错点

坑 1 · 混淆页号和块号: 页号是逻辑地址拆出来的，块号是页表里查出来的。页号从0开始连续，块号可以不连续。
坑 2 · 忘记检查段长: 分段必须检查段内地址 < 段长，否则越界。分页没有越界检查，因为页内偏移天然 < 页大小。
坑 3 · 单位换算错误: 1KB = 1024B，不是1000！计算时用 1024。

📝 4. 真题演练

真题 1（分页）

某系统页大小 4KB，逻辑地址 0x12345678，页表：0x12345 → 块号 0x54321。求物理地址。

点我看解答

4KB = 2^12，页内偏移取低12位 = 0x678

页号 = 0x12345（高20位）

查页表得块号 0x54321

物理地址 = 0x54321 × 4096 + 0x678 = 0x54321678

真题 2（分段）

段表：0(基址2000,长600), 1(基址3500,长800), 2(基址5000,长300)。判断下列地址是否越界：

A) [0, 500] B) [1, 800] C) [2, 299]

点我看解答

A: 500 < 600，合法，物理地址=2500

B: 800 >= 800，越界！（注意是>=）

C: 299 < 300，合法，物理地址=5299

✅ 5. 自测

Q1

页大小 2KB，逻辑地址 5000，页表 0→5, 1→2, 2→7。求物理地址。

答案

2KB=2048，P=5000/2048=2，W=5000%2048=904

块号=7，PA=7×2048+904=15240
