I/O 四级控制方式:一步步把 CPU 从「搬运工」里解放出来

这一课只回答一件事:程序查询 → 中断驱动 → DMA → 通道,每一级到底把 CPU 从 I/O 苦力里解放了多少?配合 I/O 系统层次 一起看。

0·全局线 1·可视化 2·ELI5 3·对照表 4·是什么 5·为什么 6·自测
一句话先记住: 四种 I/O 控制方式是同一条主线——把「谁来盯着数据搬运」这件苦差,一步步从 CPU 手里拿走。传输单位越来越大(字 → 字 → 块 → 一批块),CPU 被打扰的次数越来越少(全程忙等 → 每字一次 → 每块一次 → 整批一次)。

🗺 0. 全局故事线:四种方式是被同一个问题逼出来的

别把这四个当成孤立名词——它们是「CPU 太宝贵,别让它干搬运杂活」这一个念头,一路逼出来的四代方案。

①程序查询(忙等)CPU 亲自搬每个字,还得一直问「好了没」,全程走不开
→
②中断驱动设备传完一个字就中断通知,CPU 平时能干别的,每字仍打扰一次
→
③DMA数据直接进内存不经 CPU,以「块」为单位,只在块首块尾打扰
→
④通道一段通道程序管一批块,整个 I/O 序列只打扰 CPU 一次

来源:textbook/0_总框架.md §22.2–22.5;textbook/ch06_IO系统.md「I/O 控制方式」。

🧸 2. 用「烧水」讲清四代方案

🧸 一句话: 你要烧一壶水(设备在慢慢干活),问题是「烧水时你能不能去干别的」。
方式烧水时你在干嘛被打扰几次
①程序查询你死盯着水壶,每隔一秒看一眼「开了没」,寸步不离——啥也干不了全程(忙等)
②中断驱动你去看书,但水每冒一个泡就响一声,你都得抬头看一眼再低头——能看书但老被打断每个泡一次(每字一次)
③DMA换成会自动装杯的机器,你只在「开机」和「装满一杯」时管一下,杯子中途不烦你每杯一次(每块一次)
④通道雇了个助理,你只说一句「烧十壶、装满、摆好」,他全包,弄完喊你一声整个任务一次
⚠️ 反例(没有它会怎样): 如果一直停留在①程序查询——从终端敲一个字符约需 100 ms,而 CPU 把字符搬进缓冲区不到 0.1 ms。那 CPU 就有 99.9 ms 白白忙等。光是升级到②中断驱动,就能把 CPU 利用率成百倍提高。这就是为什么必须一代代往下走。
来源:5_设备管理/ppt文本.md §5.2.2(100 ms / 0.1 ms 例子)。

⚠️ 最高频易错:中断驱动 vs DMA(红框务必分清)
① 传输单位不同:中断驱动以「字/字节」为单位,DMA 以「数据块」为单位。
② 打扰频率不同:中断驱动每传完一个字就中断一次(N 个字 = N 次中断);DMA 只在一个块的开始和结束时打扰 CPU,块内不打扰(B 个块 ≈ B 次中断)。
③ 数据走向不同:中断驱动的数据还得经 CPU 从设备寄存器搬到内存;DMA 的数据直接由设备送入内存(或反向),整块传送在 DMA 控制器控制下完成,不经过 CPU。
一句话记牢:DMA 较中断驱动「又成百倍地减少了 CPU 对 I/O 的干预」。
来源:5_设备管理/ppt文本.md §5.2.3(「基本单位是数据块」「直接送入内存」「仅在开始和结束时需 CPU 干预」)。

📖 4. 是什么:四条精确定义(对齐教材口径)

①程序查询(程序直接控制 / 轮询 Polling): CPU 启动设备后不断轮询查询设备状态寄存器,直到设备就绪,再由 CPU 从/向数据寄存器搬一个字(节);CPU 全程忙等,与设备完全串行,不支持并行。来源:ch06 §5.2.1;0_总框架.md §22.3。
②中断驱动(Interrupt-Driven): CPU 启动设备后转去执行别的进程;设备每传完一个字/字节就发一次中断,CPU 暂停手头工作、用极短时间处理中断(搬这个字)再返回。CPU 与设备可并行,但每字都要打扰一次。来源:ch06 §5.2.2;5_设备管理/ppt文本.md §5.2.2。
③DMA(Direct Memory Access,直接存储器访问): 由DMA 控制器接管总线,在设备与内存之间成块直接传输数据。三大特点:传输基本单位是数据块;数据直接在设备与内存间流动、不经 CPU;仅在一个(或多个)数据块传送的开始与结束时才需 CPU 干预。来源:5_设备管理/ppt文本.md §5.2.3;0_总框架.md §22.5。
④通道(I/O Channel): 一种专门的 I/O 处理机,有自己的指令(通道指令)。CPU 只需发一条「启动 I/O」并给出通道程序首址,通道就独立执行这段通道程序,控制一批数据块的传输;把 DMA「对一个数据块的干预」进一步降为对一组数据块的干预,实现 CPU、通道、设备三者并行,完成后才发一次中断。来源:5_设备管理/ppt文本.md §5.1.3、§5.2.4;ch06 §5.2.4。

❓ 5. 为什么:层层追问这条演进主线

第 1 层:为什么不能一直用程序查询?
因为 I/O 设备比 CPU 慢成千上万倍。程序查询让 CPU 全程忙等——一次终端输入 CPU 要空转 99.9 ms(见 §2 反例)。CPU 是全机最贵的资源,让它盯水壶是巨大浪费。矛盾:CPU 快、设备慢,却要 CPU 陪着设备一起慢。

第 2 层:中断驱动解决了忙等,为什么还不够?
中断驱动让 CPU 在等待期间能干别的,已经成百倍提高利用率。但它仍以字为单位、每字中断一次:面对磁盘这种一次要传几 KB 的高速块设备,会产生海量中断,光是「进/出中断」的上下文开销就把 CPU 淹没。矛盾:解决了「等」,没解决「频繁被打断」。

第 3 层:DMA 已经很好,为什么还要通道?边界在哪?
DMA 以块为单位、数据不经 CPU,已把干预降到「每块一次」。但一次 I/O 任务常是一串块加控制逻辑(如读多个不连续磁盘块),每块仍要 CPU 设一次 DMA。通道用一段通道程序把「这一批块怎么读、读到哪」全描述好,CPU 发一条指令就撒手,整批只中断一次,还能三者并行。
边界/代价:越往后 CPU 越省,但硬件越贵越复杂(中断控制器 → DMA 控制器 → 专门通道处理机)。所以普通 PC 用中断+DMA 足矣,通道多见于大型机/服务器。这是一条「用硬件成本换 CPU 解放」的取舍曲线。来源:5_设备管理/ppt文本.md §5.2.4「通道是 DMA 的发展……减少为对一组数据块……的干预」。

📐 判题模板(遇到「这是哪种方式」3 步):
  1. 看传输单位:字/字节 → ①或② ; 块 → ③ ; 一批块/有通道程序 → ④。
  2. 看CPU 干预:全程忙等 → ① ; 每字中断 → ② ; 每块中断 → ③ ; 整批一次 → ④。
  3. 看数据走不走 CPU:经 CPU 搬 → ①② ; 直入内存 → ③④。

✅ 6. 自测:先自己想,再点开答案

选择题 1

关于「中断驱动」与「DMA」两种方式的区别,下列说法正确的是:

A. 两者数据传输的基本单位都是数据块
B. 中断驱动的数据不经过 CPU,DMA 的数据要经过 CPU
C. 中断驱动以字/字节为单位、每传一个字中断一次;DMA 以数据块为单位、只在块的开始和结束时才打扰 CPU
D. DMA 每传一个字节就要中断 CPU 一次

点我看答案

答案:C。 这正是本课红框反复强调的分界线:传输单位(字 vs 块)+ 打扰频率(每字 vs 每块首尾)。
A 错:中断驱动是字/字节,不是块。
B 错:说反了——是 DMA 数据直入内存不经 CPU,中断驱动仍要 CPU 搬。
D 错:每字节中断一次是「中断驱动」的特征,DMA 是每块才中断一次。

选择题 2

四种方式中,CPU 与 I/O 设备完全串行、CPU 全程忙等无法做其他工作的是:

A. 程序查询(程序直接控制)方式
B. 中断驱动方式
C. DMA 方式
D. 通道方式

点我看答案

答案:A。 程序查询靠 CPU 不停轮询状态寄存器,CPU 被死死拴住,与设备串行。
B/C/D 都让 CPU 在等待期间去干别的(可并行),只是「被打扰的频率」不同:每字 / 每块 / 整批一次。

选择题 3

相比 DMA,I/O「通道」方式最核心的进步是:

A. 通道让数据第一次做到不经过 CPU 直接进内存
B. 通道把传输单位从「块」变回「字节」以提高精度
C. 通道不需要任何额外硬件,成本比 DMA 低
D. 通道用一段通道程序管理一批数据块,把「对一个块的干预」降为「对一组块的干预」,整个序列只打扰 CPU 一次

点我看答案

答案:D。 教材原话:通道是 DMA 的发展,把「以一个数据块为单位的干预,减少为对一组数据块……的干预」。
A 错:「数据不经 CPU 直入内存」是 DMA 就已实现的,不是通道首创。
B 错:通道传输单位更大(一批块),不是退回字节。
C 错:通道需要专门的 I/O 处理机,硬件成本最高、最复杂。

选择题 4

某高速磁盘一次要连续传输若干个数据块,希望 CPU 尽量少被打扰、且数据不经过 CPU。单看这次传输,最合适的是:

A. 程序查询方式,简单可靠
B. DMA 方式(以块为单位、直入内存、块首尾才干预)
C. 中断驱动方式,每个字节及时响应
D. 无论哪种方式,CPU 被打扰次数都一样

点我看答案

答案:B。 磁盘是高速块设备,DMA 以块为单位、数据直入内存、只在块首尾干预,正好对症(通道更强但题干只问单次块传输且未提通道硬件,DMA 是标准答案)。
A 错:程序查询忙等,高速设备下 CPU 浪费最严重。
C 错:中断驱动每字节中断一次,面对大块数据会产生海量中断。
D 错:打扰次数恰恰不同——全程 / N / B / 1,这正是四方式的分水岭。

口述题(别打字,张嘴说)

  1. 用一句话说出四种方式的主线(一步步把 CPU 从什么里解放出来)。
  2. 不看屏幕,说出四方式各自的传输单位和打扰 CPU 的次数(字/字/块/一批块;全程/N/B/1)。
  3. 用自己的话讲清中断驱动和 DMA 到底差在哪(单位、频率、数据走不走 CPU 三点)。

✅ 通过条件:这三条都能顺口说出、且第 3 条能一口气讲全三个差异点 → 你是真懂了。哪条卡壳,回 §1 拖一遍滑块、回 §3 红框再看一眼。
