📄 1. 分页地址变换 + 快表 EAT ⚡(计算大题一)

🧸 一句话: 逻辑地址就是“第几页 + 页里第几个字节”。因为每页一样大(页大小固定),系统自己就能把你给的一个数除出“页号+偏移”;再拿页号去页表查到它住进了内存第几个物理块,把块号和偏移拼起来,就是真实物理地址。

🖼 交互件一:分页地址变换 + EAT 计算器

👇 改页大小和逻辑地址,点【变换】看三步怎么走。下半区是快表 EAT小算器。本例页表固定为 [0→8, 1→5, 2→9, 3→3, 4→7]。

页号 P
页内偏移 W
逻辑地址拆成这两段;查页表把 P 换成物理块号 F,W 原样不动 → 物理地址 = F × L + W
← 点【变换】看 2500 怎么一步步变成物理地址。
快表 EAT(有效访问时间):快表命中直接得块号只花 ε+t;不命中要再访问内存里的页表,共 2t+ε。
← 点【算 EAT】。公式 EAT = p·(t+ε) + (1−p)·(2t+ε) = 2t+ε−p·t。
📐 解题模板(分页地址变换,三步):
  1. 拆地址:页号 P = A ÷ L(整除);页内偏移 W = A mod L(取余)。先把单位看清:4KB=4096,不是 4000!
  2. 查页表:F = 页表[P](页号 → 物理块号)。若 P ≥ 页表长度 → 越界中断。
  3. 拼地址:物理地址 = F × L + W。偏移 W 前后一字不变,只换块号。
快表 EAT(汤小丹口径): 设内存访问时间 t、快表访问时间 ε、命中率 p,则
EAT = p×(t+ε) + (1−p)×(2t+ε) = 2t + ε − p×t。
不用快表(基本分页)= 2t(一次查页表 + 一次取数据)。命中率越高,EAT 越接近 t + ε。

🅿️ 2. 动态分区四挑法 FF / NF / BF / WF

🧸 一句话: 内存里有一堆大小不一的空闲块,来一个作业要挑一块给它。FF=从头找第一个够大的;NF=接着上次位置往后找;BF=挑刚好够、最小的;WF=挑最大的。同一个作业、同一堆空闲块,四种挑法可能选中完全不同的块。

🖼 交互件二:FF / BF / WF 三算法“同题不同选”对比

👇 空闲块固定为 100K / 500K / 200K / 300K / 600K(按地址排)。改一个作业大小,看 FF、BF、WF 各自选中哪一块——它们经常选出三个不同的块。(NF 与 FF 的区别是“接着上次位置找”,这里不动指针时与 FF 一致。)

← 点【对比三种挑法】看同一个 300K 作业被三种算法送进了三个不同的块。

⚖️ 四挑法 + 碎片对照(照教材口径)

算法空闲链按什么排怎么挑毛病
FF 首次适应地址递增从低地址找第一个够用的低地址越切越碎
NF 循环首次地址递增(成环)从上次结束处往后找第一个够用的会切开高地址大块
BF 最佳适应容量递增挑够用里最小的留一堆没用的小碎渣
WF 最坏适应容量递减挑最大的大块很快被用光
碎片必考辨析: 内部碎片=分给作业的分区里面没用满的空间(固定分区、分页有);外部碎片=分区之间那些太小、谁也用不上的空隙(动态分区、连续分配有)。外碎片攒多了 → 紧凑(搬进程拼成大块),前提是动态重定位(重定位寄存器)。想彻底躲开外碎片 → 干脆离散分配(分页),这就接到了第 1 节。

🧩 3. 分段 vs 分页 · 段页式(一维 vs 二维 · 相加 vs 相乘)

🧸 一句话: 分页是给系统管内存用的“物理切分”,每页一样大 → 给一个数就能定位(一维);分段是给程序员用的“逻辑切分”,每段长短不一 → 必须给段号+段内偏移两个数(二维)。段页式把两者叠起来:先分段、每段再分页。

⚖️ 分段 vs 分页 对照(照教材口径)

比什么分页 Paging分段 Segmentation
为什么分信息的物理单位,为提高内存利用率、便于 OS 管理信息的逻辑单位,一段是一组完整信息,便于用户编程
大小固定,由系统定(如 4KB)不固定,由内容/用户定
地址维度一维(给一个数)二维(段号 + 段内偏移)
碎片有内部碎片,无外部碎片有外部碎片,无内部碎片
表里存什么页表:页号 → 物理块号段表:段号 → (段长 limit、段基址 base)
物理地址怎么算块号 × 页大小 + 偏移(相乘)段基址 + 段内偏移(相加)
共享/保护不便按逻辑模块共享便于段级共享、保护、动态链接
⚠️ 最毒的陷阱(几乎年年考): 分页是“相乘”(块号 × 页大小 + 偏移),分段是“相加”(段基址 + 段内偏移)。段表里存的是段基址(起始物理地址),不是块号。谁把分段也写成“× 段长”,直接扣分。

🔢 同一组数,分页(相乘)vs 分段(相加)对照走一遍

分页:页大小 L=1KB=1024,页表 [0→8,1→5,2→9],逻辑地址 2500。

P = 2500 ÷ 1024 = 2,W = 2500 mod 1024 = 452 → 查页表 F = 9 → 物理地址 = 9 × 1024 + 452 = 9668(相乘)。

分段:段表 段2 → (base=120000, limit=8000),段号 s=2、段内偏移 d=100。

先查越界:100 < 8000 ✓ → 物理地址 = base + d = 120000 + 100 = 120100(相加)。

段页式: 逻辑地址三部分 = [段号 s | 页号 p | 页内偏移 d];每进程一张段表、每段一张页表。变换:查段表得该段页表起址 → 查页表得物理块号 → 物理地址 = 块号 × 页大小 + d。不用快表要访存 3 次(段表 → 页表 → 数据),所以配快表(TLB)提速。想看这三种模式的可点拆解器,去 第 17 课。

🔁 4. 页面置换 OPT / FIFO / LRU / CLOCK ⚡(计算大题二)

🧸 一句话: 物理块满了又要装新页,就得按规则淘汰一个老页。OPT 淘汰“将来最久不用”的(理论最优、不可实现);FIFO 淘汰“最早进来”的(最简单、可能 Belady);LRU 淘汰“过去最久没用”的(接近最优、无 Belady);CLOCK 用一个访问位近似 LRU(便宜)。

⚖️ 四种算法判据(必背)

算法淘汰谁看的是特点
OPT 最佳将来最久不再用的未来缺页率最低;不可实现,只用来打分
FIFO 先进先出最早进内存的进入顺序最简单;可能 Belady 异常
LRU 最近最久未用过去最久没用过的过去接近 OPT;无 Belady;要硬件记时间
CLOCK 时钟/NRU扫到访问位=0的第一个访问位LRU 的低成本近似:环形链表+指针,遇1清0后移,遇0淘汰
📐 解题模板(页面置换缺页率,手算 4 步):
  1. 画 K 行的表(K=物理块数),列 = 引用串每一次访问。
  2. 逐个访问:已在内存 → 命中(FIFO 不动;LRU 把它记为“刚用过”);不在 → 缺页+1,内存没满直接装,满了按算法选牺牲页换掉。
  3. OPT 看后面:对内存里每一页,找它下一次出现的位置,淘汰最靠后/永不再现的那个。
  4. 缺页率 = 缺页次数 F ÷ 总访问次数 N(是比率,不是次数本身!)。

📉 5. Belady 异常 · 抖动 · 工作集

🧸 一句话: Belady 异常 = “多给一块内存,缺页反而变多”的反常现象——只有 FIFO 会(OPT/LRU 这类“堆栈类算法”不会)。抖动 = 块给得太少,系统不停换入换出、几乎不干正事,CPU 利用率趋近 0。工作集用来估“一个进程至少要几块才不抖”。

🖼 交互件四:Belady 异常现场(同串,3 块 vs 4 块,都用 FIFO)

👇 引用串 1,2,3,4,1,2,5,1,2,3,4,5 用 FIFO 各跑一遍:左边 3 块、右边 4 块。看总缺页数——多给一块,反而从 9 涨到 10。这就是 Belady。

🅰 3 块 · FIFO
🅱 4 块 · FIFO
← 点【跑一遍对比】。
为什么 FIFO 会 Belady、别的不会? FIFO 只看“谁最早进来”,不看“谁最近有用”——块变多、队列变长,它可能把“马上要用”的页当成“老页”踢掉。而 OPT/LRU 是堆栈类算法:块数变多时,小内存里的页集合始终是大内存的子集,所以缺页只会不增。结论:块越多缺页越少?对 FIFO 不成立。

❓ 抖动(颠簸)与工作集

抖动是什么: 分配给进程的物理块数 < 它的工作集 → 频繁缺页 → 频繁换入换出 → CPU 几乎全花在处理缺页上,有效工作趋近 0。

工作集怎么救: 工作集 = 进程在最近一段时间窗口内实际访问过的页集合,用它估计进程至少要几块。只要分配块数 ≥ 工作集就不抖;若所有进程工作集之和 > 总块数,说明多道程序度太高,要挂起部分进程 / 降低多道程序度。

缺页率的影响因素: 页面越大、分配块数越多、置换算法越好、程序局部性越好 → 缺页率越低。

大题一 · 分页地址变换 + EAT

某系统采用分页存储管理,页大小 = 1KB。某进程页表如下:

页号0123
物理块号2468

(1) 求逻辑地址 1500 和 3200 对应的物理地址。
(2) 若逻辑地址 5000,会发生什么?
(3) 内存访问 t=100ns、快表访问 ε=10ns、命中率 p=80%,求有效访问时间 EAT。

点我看逐步解答

(1) 1KB = 1024 字节。

• A=1500:P = 1500÷1024 = 1,W = 1500 mod 1024 = 476;查页表 F = 4 → 物理 = 4×1024 + 476 = 4572。

• A=3200:P = 3200÷1024 = 3,W = 3200 mod 1024 = 3200−3072 = 128;查页表 F = 8 → 物理 = 8×1024 + 128 = 8320。

(2) A=5000:P = 5000÷1024 = 4,而页表只有 0~3 页 → 页号越界,产生越界中断,访问被拒绝。

(3) EAT = p×(t+ε) + (1−p)×(2t+ε) = 0.8×(100+10) + 0.2×(200+10) = 0.8×110 + 0.2×210 = 88 + 42 = 130ns。(核对:2t+ε−p×t = 210 − 80 = 130 ✓)

易错:偏移 476/128 要原样带进物理地址;单位是 1024 不是 1000;越界要判 P 是否 ≥ 页表长度。

大题二 · 页面置换缺页率 + Belady

引用串 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5(共 12 次访问)。

(1) 物理块 = 3,分别用 OPT、FIFO、LRU 求缺页次数与缺页率。
(2) 用 FIFO,把物理块从 3 增到 4,缺页次数如何变化?说明这是什么现象。

点我看逐步解答

(1) 物理块 = 3(N=12):

• OPT:缺页 7 次(步 1,2,3,4,7,10,11)→ 缺页率 = 7/12 ≈ 58.3%。

• FIFO:缺页 9 次(步 1,2,3,4,5,6,7,10,11)→ 缺页率 = 9/12 = 75%。

• LRU:缺页 10 次(步 1,2,3,4,5,6,7,10,11,12)→ 缺页率 = 10/12 ≈ 83.3%。

注意:这串恰好 FIFO 比 LRU 还少——是这条特定串的巧合,不代表 FIFO 一般更好。考试问的是“会不会算”,别慌。用上面的交互件三切换三种算法,可现场核对这三个数。

(2) FIFO,物理块 3 → 4:

3 块 FIFO 缺页 9 次;4 块 FIFO 缺页 10 次。多给一块,缺页反而多了一次 → 这就是 Belady 异常,只有 FIFO 会出现(OPT/LRU 是堆栈类算法,不会)。用交互件四可现场看这 9→10。

选择题 1

分页系统页大小 1KB,页表 [0→3, 1→5, 2→1],逻辑地址 2100 的物理地址是?

A. 52(= 0×1024 + 52)
B. 1076(= 1×1024 + 52)
C. 2100(不变)
D. 3124(= 3×1024 + 52)

点我看答案

答案:B。 P = 2100÷1024 = 2,W = 2100 mod 1024 = 2100−2048 = 52;查页表[2] = 1 → 物理 = 1×1024 + 52 = 1076。A 错:块号漏查页表当成 0;C 错:忘了做变换(或误把页号 2 直接当块号,2×1024+52 也恰是 2100);D 错:页号取成 3(应查页表[2]=1)。

选择题 2

下列关于 Belady 异常 的说法,正确的是?

A. 任何置换算法都可能出现
B. 物理块越多,缺页率一定越低
C. 它是 LRU 的专属缺陷
D. 只有 FIFO 可能出现,OPT/LRU 不会

点我看答案

答案:D。 Belady 异常(块变多缺页反而变多)只在 FIFO 这类非堆栈算法出现;OPT、LRU 是堆栈类算法,块多时页集合是块少时的超集,缺页只会不增。A、C 错;B 恰恰被 Belady 反驳。

选择题 3

分段系统段表 段1 → (base=40000, limit=6000),访问段号 1、段内偏移 100,物理地址是?

A. 40000 + 100 = 40100
B. 1 × 6000 + 100 = 6100
C. 40000 × 6000 + 100
D. 越界,拒绝访问

点我看答案

答案:A。 先越界检查:100 < 6000 ✓ 合法;分段是相加:物理 = 段基址 + 段内偏移 = 40000 + 100 = 40100。C 是把分段错当“相乘”(最毒陷阱);B 用段号乘,纯属乱凑;D 错,没越界。

选择题 4

“最佳适应算法 BF”的空闲分区链按什么排序、分配时选哪块?

A. 按地址递增,选第一个够用的
B. 按容量递减,选最大块
C. 按容量递增,选能满足的最小块
D. 按地址递减,选最后一个够用的

点我看答案

答案:C。 BF 空闲链按容量从小到大排,从头找到第一个够用的就是“最小够用块”。A 是 FF(按地址);B 是 WF(容量递减、选最大);D 生造。记牢:BF 按大小、不是按地址——最容易记反。
