分页地址翻译——把虚拟地址翻成物理地址

明天目标:对着真实 C 代码逐行讲 8–10 分钟,答得住追问 · 源码:xv6-riscv/kernel/vm.c

🎯 这份稿子可以独立上场。 它把课本第 4 章「基本分页存储管理 · 地址变换机构」落到一段真实代码:walk() 就是课本那张「逻辑地址 → 页表 → 物理地址」框图的软件实现。逐行讲代码 + 随时指回课本概念(页号 P、页内地址 d、地址变换、多级页表、快表 TLB),就是「讲解 + 答辩」的满分形态。

0. 先记这句主线(讲偏了就拉回来)

「CPU 给出一个虚拟地址,页表负责把它翻译成物理地址。walk() 就是这个翻译过程的软件实现——把虚拟地址拆成『几级页号 + 页内偏移』,一级级查页表,最后拿到物理页。」
对应课本一句话:页号 P 查页表得物理块号,块号 × 页大小 + 页内地址 d = 物理地址。xv6 只是把「一级页号」换成了「三级页号」(多级页表)。

1. 背景:一个虚拟地址是怎么拆的?

课本说:逻辑地址 = 页号 P + 页内地址 d。xv6 跑在 RISC-V 的 Sv39 方案上,页大小 4KB,页表分三级。所以「页号」不是一段,而是被切成三段 9 位的索引,再加上 12 位页内偏移。源码注释(vm.c 第 79–84 行)讲得很清楚:

79// A 64-bit virtual address is split into five fields:
80//   39..63 -- must be zero.        (高位必须为 0)
81//   30..38 -- 9 bits of level-2 index.  (第 2 级页号)
82//   21..29 -- 9 bits of level-1 index.  (第 1 级页号)
83//   12..20 -- 9 bits of level-0 index.  (第 0 级页号)
84//    0..11 -- 12 bits of byte offset within the page. (页内偏移 d)
63..39必须为 0
L2 索引38..30 · 9位
L1 索引29..21 · 9位
L0 索引20..12 · 9位
页内偏移 d11..0 · 12位
对上课本:课本里逻辑地址 = 页号 P | 页内地址 d。这里只是把「页号 P」这一大段,再切成 L2 / L1 / L0 三个 9 位小段——这就是多级(三级)页表的地址结构。最右边 12 位永远是页内偏移 d,翻译过程中原封不动带到物理地址里。

为什么 4KB 页 → 偏移正好 12 位?

页内偏移要能表示「一页里的每一个字节」。一页 = 4KB = 4096 字节 = 212,要区分 4096 个字节地址,正好需要 12 位(212=4096)。所以 PGSHIFT = 12。同理每级 9 位,因为一张页表正好装 29 = 512 个页表项(riscv.h: pagetable_t // 512 PTEs)。

为什么用多级页表?(对照课本「两级页表」)

课本讲两级页表的理由就是省内存。如果用一张单级页表覆盖整个虚拟空间,光页表本身就要占巨大的连续内存(且大多是空洞)。多级页表的妙处:只有真正用到的那一小块地址,才需要为它分配下一级页表。没映射的区域,上一级页表项是空的,根本不占下一级页表的空间。代码里这点体现在 walk() 的 按需分配(后面第 96–99 行)。

2. 逐行讲 walk()(来自 kernel/vm.c)

背景注释先念:「返回页表 pagetable 中与虚拟地址 va 对应的那个最低级 PTE 的地址;若 alloc≠0,沿途缺哪级页表就新建哪级。」

85pte_t *
86walk(pagetable_t pagetable, uint64 va, int alloc)
87{
88  if(va >= MAXVA)
89    panic("walk");            // ① 越界检查:va 超过最大虚拟地址就报错(对应课本"越界中断")
90
91  for(int level = 2; level > 0; level--) {  // ② 从最高级 L2 逐级往下走到 L1
92    pte_t *pte = &pagetable[PX(level, va)];  // ③ 取本级 9 位页号当下标 → 定位 PTE
93    if(*pte & PTE_V) {                  // ④ 这个 PTE 有效吗?(V=有效位)
94      pagetable = (pagetable_t)PTE2PA(*pte); // ⑤ 有效→取出下一级页表的物理地址,进入下一级
95    } else {                            // ⑥ 无效→这一级还没建
96      if(!alloc || (pagetable = (pde_t*)kalloc()) == 0)
97        return 0;                      // 不让分配 / 分配失败 → 翻译失败
98      memset(pagetable, 0, PGSIZE);     // 新页表清零(全部 PTE 设为无效)
99      *pte = PA2PTE(pagetable) | PTE_V; // 把新页表登记进上一级 PTE,并置有效位
100    }
101  }
102  return &pagetable[PX(0, va)];  // ⑦ 循环结束已到 L0 页表,返回最低级 PTE 的地址
103}

三个关键宏(讲代码时一定会被指着问)

这几个宏来自 kernel/riscv.h,是「拆地址、读 PTE」的核心工具:

355#define PXMASK          0x1FF // 9 位掩码,二进制 9 个 1 = 511,正好取 9 位
356#define PXSHIFT(level)  (PGSHIFT+(9*(level))) // 该级索引在地址里要右移多少位
357#define PX(level, va) ((((uint64) (va)) >> PXSHIFT(level)) & PXMASK) // 取出第 level 级的 9 位页号
348#define PA2PTE(pa)  ((((uint64)pa) >> 12) << 10) // 物理地址 → 装进 PTE(去掉低 12 位,腾出低 10 位放标志)
350#define PTE2PA(pte) (((pte) >> 10) << 12) // PTE → 还原出物理页地址(扔掉低 10 位标志,补回 12 位)
341#define PTE_V (1L << 0) // 有效位:第 0 位,1=该 PTE 指向真实页/页表
把 PX(level, va) 讲透(这是全场最该会的一行):它就是课本「从逻辑地址里切出页号 P」的代码版。va >> PXSHIFT(level) 把要的那 9 位移到最低位,再 & 0x1FF 把高位全抹掉,只留 9 位。比如取 L2:右移 30 位(12+9×2),再 &511,得到的就是 38..30 那 9 位。移位 + 掩码 = 切地址,这是分页地址变换最核心的位运算。

把代码切成「一句话讲法」表(这是你讲解的主结构)

行代码一句话讲法对应课本

91for level 2→1「从最高级 L2 往下逐级走。三级页表就走两次循环 + 最后一级。」多级页表

注意一个常见误解:walk() 返回的是最低级 PTE 的指针,不是物理地址本身。要真正拿到物理地址,得再 PTE2PA(*pte) 取出物理页框号,再加上页内偏移 d。这一步在 walkaddr() 第 124 行:pa = PTE2PA(*pte)。把这点说清楚,显示你真读懂了。

3. 一张图:walk 怎么走(PPT 核心页)

虚拟地址 va = [ L2(9) | L1(9) | L0(9) | offset(12) ] │ satp 寄存器 ──► L2 页表(根) │ 下标 = PX(2,va) ▼ L2[PX2] ──*pte&PTE_V?─是─► PTE2PA ──► L1 页表 │ │ 下标 = PX(1,va) ▼ ▼ L1[PX1] ──*pte&PTE_V?─是─► PTE2PA ──► L0 页表 │ │ 下标 = PX(0,va) ▼ ▼ L0[PX0] ───── PTE2PA ─────► 物理页框号(物理页) │ └──► 物理地址 = 物理页框号 × 4KB + offset(那 12 位原样带过来) 口诀:三级页号一级级查页表，最后一级 PTE 给出物理页；偏移 12 位全程不变。

4. 计算:页号 / 偏移 / 物理地址(算给老师看)

老师在第 4 章必考这种计算。先用课本最经典的「一级页表」算法热身,再说明 xv6 三级只是多查几层。

例题(课本式,页大小 L = 4KB = 4096B):给逻辑地址 LA = 9000。
① 页号 P = INT(LA / L) = INT(9000 / 4096) = 2
② 页内偏移 d = LA mod L = 9000 mod 4096 = 9000 − 2×4096 = 808
③ 查页表:设页号 2 对应物理块号(页框号) = 5
④ 物理地址 PA = 物理块号 × 页大小 + 页内偏移 = 5 × 4096 + 808 = 20480 + 808 = 21288
换成 xv6 的视角讲一句:「课本这里页号 P 是一段,直接查一次页表得块号。xv6 的页号被切成 L2 | L1 | L0 三段,要查三层页表才得到最终物理页框号;但第 ④ 步『块号 × 页大小 + 偏移』完全一样——偏移那 12 位永远不参与翻译,原样拼到物理地址末尾。」

验证「为什么偏移是 12 位」(顺手算给老师)

d 的范围是 0 ~ 4095(一页 4096 个字节)。4096 = 212,所以表示偏移恰好要 12 个二进制位 → PGSHIFT = 12。上例 d = 808,二进制 001100101000 正好 12 位以内。这也解释了 PTE2PA 为什么右移再左移 12 位:页框号天然「页对齐」,低 12 位永远是 0,留给偏移用。

6. 答辩:老师可能逐行问的 10 题

逐行讲源码,老师最爱指着某一行问「这行干嘛」。下面是高危行的标准答法。

Q1. 虚拟地址是怎么拆成页号和偏移的?必答
答:低 12 位是页内偏移 d(因为页 4KB=2¹²);剩下的高位是页号。在 xv6 Sv39 里页号又被切成三段各 9 位(L2/L1/L0)。代码里用 PX(level,va) 完成:va 右移 (12+9×level) 位,再 & 0x1FF,就取出某一级的 9 位页号。移位定位、掩码截断。
Q2. 指着第 92 行 PX(level, va) 问:这行到底干嘛?高危
答:它从虚拟地址里切出第 level 级的 9 位页号,作为本级页表的数组下标。>> PXSHIFT(level) 把那 9 位移到最低位,& PXMASK(0x1FF) 只保留 9 位。这就是课本「用页号 P 去页表里定位表项」的代码实现。
Q3. PTE_V 是什么?第 93 行为什么要判断它?高危
答:PTE_V 是页表项的有效位(Valid),在 PTE 的第 0 位。第 93 行 *pte & PTE_V 就是问「这一级的映射建好了吗」。有效→顺着它进下一级;无效→要么(允许时)新建一级页表,要么翻译失败返回 0。它对应课本里页表项的存在位/有效位,也是缺页判断的基础。
Q4. 一次地址翻译要查几次页表?为什么用多级,不怕慢吗?必答
答:Sv39 三级,一次翻译最多查 3 层页表(再加 1 次访问数据本身)。用多级是为了省内存——只给真正用到的地址分配下一级页表,空洞区域不占空间。怕慢的问题靠快表 TLB 解决:翻译结果缓存起来,绝大多数访问直接命中 TLB,不用再走三层。这正是课本「具有快表的地址变换机构」。
Q5. 页大小 4KB,为什么页内偏移正好是 12 位?高危
答:偏移要能寻址一页里的每个字节。4KB = 4096 = 212,要区分 4096 个地址正好需要 12 位。所以 PGSHIFT=12。若页改成 8KB(2¹³),偏移就要 13 位。
Q6. 第 94 行 PTE2PA(*pte) 干嘛?它和 PA2PTE 什么关系?高危
答:PTE 的低 10 位是标志位(V/R/W/X/U 等),高位才是物理页号。PTE2PA 把 PTE 右移 10 位扔掉标志、再左移 12 位,还原成下一级页表(或物理页)的物理地址。PA2PTE 是反操作:把物理地址装进 PTE。两者一对正反变换。
Q7. walk() 返回的是物理地址吗?易错
答:不是。它返回最低级 PTE 的指针。要得到物理地址还要两步:PTE2PA(*pte) 取出物理页框号,再加上页内偏移 d。这一步在 walkaddr 第 124 行。对应课本公式 物理地址 = 块号 × 页大小 + 页内地址。
Q8. 第 96–99 行的「按需分配」在课本里对应什么?进阶
答:对应课本讲多级(两级)页表为什么省内存。如果某段虚拟地址从没用过,它对应的上级 PTE 一直无效,就永远不会为它分配下一级页表。只有真正建立映射(mappages)时,walk(...,alloc=1) 才一路把缺的页表新建出来。
Q9. 这套机制是软件做的还是硬件做的?那 walk() 算什么?易问
答:真实 CPU 运行时,地址翻译由 MMU 硬件自动完成(读 satp 指向的页表,逐级查)。walk() 是内核软件用同样的算法来「读/改页表」——比如建立映射(mappages)、查物理地址(walkaddr)。它和硬件 MMU 走的是同一张页表、同一套拆分规则,所以读 walk() 就等于读懂了硬件翻译过程。
Q10. 第 88 行 va >= MAXVA 的检查对应课本哪个概念?易答
答:对应课本地址变换里的越界检查(页号 ≥ 页表长度则越界中断)。Sv39 只用 39 位虚拟地址,超过 MAXVA 的地址非法,直接 panic,防止访问到不该访问的内存。
