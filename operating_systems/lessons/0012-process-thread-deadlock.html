进程·线程·死锁 · 概念收尾

目标：15 分钟装下"进程 vs 程序"、"进程 vs 线程"、"死锁 4 条件"——全是选择题考点

1. 进程 vs 程序（4 行对照）

程序 = 死的，一份代码躺在硬盘上
进程 = 活的，程序的一次执行（有生命周期、有状态）
维度程序进程
本质一段静态代码程序的一次执行
存在位置磁盘上内存里（PCB + 代码 + 数据）
动态性静态动态（创建 → 运行 → 撤销）
并发性不能并发可以并发（多进程同时跑）
考试最爱问："程序和进程的最本质区别是什么？"——答：动态性。程序是死的，进程是活的。

2. PCB（进程控制块）

PCB = 进程的"身份证"，OS 用 PCB 描述和管理进程。PCB 是进程存在的唯一标志。

分类内容
标识信息进程 ID (PID)、用户 ID、父进程 ID
状态信息就绪/运行/阻塞、各种寄存器值（PC、PSW）
控制信息优先级、信号、文件描述符、资源清单
组织信息PCB 链表 / 索引（OS 用这些结构管理所有进程）

3. 进程 3 状态（必背图）

🏃
运行 (Running)
正在 CPU 上跑（单核 CPU 同时只有 1 个）
⏳
就绪 (Ready)
万事俱备，只等 CPU
😴
阻塞 (Blocked)
等 IO / 等资源，没法跑

就绪 → 运行（调度）→ 阻塞（等 IO）→ 就绪（IO 完成）→ 运行……

易记 1 句话：就绪是"等 CPU"，阻塞是"等别的"（IO/资源）。就绪 ↔ 运行 互相转，运行 → 阻塞 → 就绪。

4. 进程 vs 线程（4 行对照）

进程 = 一家公司（一栋楼 + 一套制度）
线程 = 公司里的员工（共享公司的资源，但各干各的活）
维度进程线程
资源拥有独立内存空间共享所属进程的内存
切换切换慢（要换内存）切换快（只换寄存器和栈）
通信难（要 IPC：管道/消息/共享内存）易（直接读共享变量）
并发并发粗（一公司干一件事）并发细（一公司多人各干各的）
考试最爱问："线程和进程的最本质区别是什么？"——答：是否共享地址空间。同进程线程共享；不同进程不共享。

5. 死锁的 4 个必要条件（必背）

死锁 = 一组进程互相等对方手里的资源，谁也跑不动。
发生死锁，4 个条件必须同时成立——破坏任意 1 个就不会死锁。
1
互斥条件
资源一次只能 1 个进程用
（打印机、临界区）
2
请求和保持
手里有 A 还想要 B
（拿到不释放）
3
不可抢占
别人手里的资源，
OS 也不能抢走
4
循环等待
A 等 B，B 等 C，C 等 A
（形成环）
易记口诀："独、占、等、环"
• 独占（互斥）
• 占着不释放（请求和保持）
• 等着不能抢（不可抢占）
• 环形等待（循环等待）

5.1 对应"破坏"方法（了解即可）

死锁 4 条件如何破坏
互斥把独占资源改成共享（但有些资源天生独占）
请求和保持一次性申请所有资源（运行前全拿）
不可抢占OS 可以强行抢走（优先级高的能抢低的）
循环等待给资源编号，按编号顺序申请（L3 银行家算法的思路）

6. 必背 4 句话（考试原话）

① 进程 = 程序的执行实例（动态）；程序 = 静态代码。最本质区别是动态性。
② PCB 是进程存在的唯一标志，OS 用 PCB 管理进程。
③ 进程 3 状态：就绪（等 CPU）/ 运行（在跑）/ 阻塞（等 IO）。
④ 死锁 4 条件：互斥、请求和保持、不可抢占、循环等待（"独占等候环"）——破坏任意 1 个就解决。

7. 自检（先做再翻答案）

Q1（口述） 程序和进程的最本质区别是什么？
Q2（口述） 进程 3 状态之间的转换关系？就绪和阻塞的关键区别？
Q3（口述） 进程和线程的最本质区别？切换谁更快？
Q4（默写） 死锁的 4 个必要条件（按"独、占、等、环"口诀默写）。
Q5（选择） 下面哪种情况一定不会发生死锁？
A. 4 个条件都满足
B. 破坏"循环等待"（资源按编号申请）
C. 2 个进程都不可抢占
D. 所有资源都互斥
答案对照
A1：动态性。程序是死的（一段代码），进程是活的（一次执行，有生命周期）。
A2：
• 就绪 → 运行：被调度
• 运行 → 就绪：时间片用完
• 运行 → 阻塞：等 IO / 等资源
• 阻塞 → 就绪：IO 完成 / 资源到位
• 就绪 = 等 CPU；阻塞 = 等别的东西（没法跑）
A3：是否共享地址空间。同进程线程共享；不同进程不共享。线程切换快（不换内存）。
A4：独（互斥）、占（请求和保持）、等（不可抢占）、环（循环等待）。
A5：B。破坏任意 1 个条件就不会死锁——B 破坏了"循环等待"。A、C、D 都是死锁必要条件。
