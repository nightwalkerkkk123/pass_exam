🔢 1. 位与十六进制：二进制、十六进制、按位拆

🧸 一句话：计算机的地址就是一长串 0 和 1。人读「0010111101101010」会瞎，所以约定：每 4 个二进制位写成 1 个十六进制位（4 比特 = 1 个 16 进制字符）。这就是为什么地址里全是 0–F 而不是 0–9。

① 二进制 ↔ 十六进制对照（4 位一截）

十六进制每位 = 二进制 4 位的缩写：

每格左下是十进制，右下是二进制 4 位。死记几张就会了：A=10、B=11、C=12、D=13、E=14、F=15。最常用：A=1010、B=1011、C=1100、D=1101、E=1110、F=1111。

📍 2. 地址与分页：为什么需要页表

🧸 一句话：程序在写代码时用的是「逻辑地址」（从 0 开始编号，每个进程自己觉得独占了一整块内存）。但内存硬件只有「物理地址」（真实 RAM 的每个字节有一个编号）。需要一张「翻译表」把逻辑地址变成物理地址——这张表就是页表。整本分页管理学，本质就是这张表的 设计 与 优化。

① 不分页会怎样？

如果把进程「整个」放进内存：

内存（4 KB 总大小，假设）: 
┌────────────────────┐  0x0000
│                    │
│   进程 A（占 1.2KB）│
│                    │
├────────────────────┤  0x04CC
│  没装 A 前这里是空洞  │  ← 外部碎片
├────────────────────┤  0x0500
│   进程 B（占 0.8KB）│
│                    │
├────────────────────┤  0x0833
│  又一个空洞          │  ← 外部碎片
├────────────────────┤  0x0900
│   进程 C（占 1.6KB）│
│                    │
├────────────────────┤  0x0F40
│                    │
└────────────────────┘  0x1000
问题：A、B、C 大小都不一样，进程之间会留外部碎片，小洞塞不进大进程。要么紧凑（移动进程，开销大）、要么用不完。

② 分页的解法：固定大小切块

「分页」的核心思想很简单：不管进程多大，都切成同样大小的块（典型 4KB），叫「页」；内存也切成同样大小的「物理块」/「页框」。任意页可以装进任意物理块。

分页后（页大小 = 256B，4 KB 总内存 = 16 块）: 
┌────┐ 0x0000  块 0
├────┤ 0x0100  块 1   ← A 的页 0
├────┤ 0x0200  块 2   ← B 的页 1
├────┤ 0x0300  块 3   ← A 的页 1
├────┤ 0x0400  块 4   ← B 的页 0
├────┤ 0x0500  块 5
├────┤ 0x0600  块 6   ← C 的页 0
├────┤ 0x0700  块 7   ← C 的页 1
├────┤ 0x0800  块 8   ← C 的页 2
├────┤ 0x0900  块 9
├────┤ 0x0A00  块 10  ← C 的页 3
├────┤ 0x0B00  块 11
├────┤ 0x0C00  块 12  ← A 的页 2
├────┤ 0x0D00  块 13
├────┤ 0x0E00  块 14  ← B 的页 2
├────┤ 0x0F00  块 15
└────┘ 0x1000
好处：没有外部碎片——空块就是空块，谁都能装。要么装满、要么是空闲块。
关键事实（必背）：
  • 页大小 = 物理块大小（同一个值，从两个角度说）
  • 逻辑地址空间分页、物理内存分块——页装进块
  • 页号→块号 的映射关系存在「页表」里，每进程一份
  • 同一逻辑页号每次装到哪个物理块是不定的（这就是分页的最大自由）

③ 翻译一次的成本

CPU 每次访问内存，都要问：「逻辑地址 X 是哪个页、这个页装到哪个物理块里？」

  1. 从逻辑地址拆出页号 P 和页内地址 d
  2. 查「页表」第 P 项 → 拿到物理块号 B
  3. 算物理地址 = B × 页大小 + d

📋 3. 页表本身：每行装什么、查表时硬件做什么

🧸 一句话：页表就是个「页号 → 物理块号」的查表。每行叫一项，最小一项通常几十字节。一项里除了「块号」还得装点别的（有效位、保护位...）。本节先把页表项的结构拆开，再讲怎么把它组织成树。

① 页表项结构（简化版）

字段占多少位作用
有效位 V1 位该页是否在内存（1=在、0=被换出或在硬盘）
物理块号 B主要字段该页装在哪个物理块
保护位2–3 位读/写/执行权限
使用位 U1 位是否被访问过（辅助置换）
脏位 D1 位是否被修改过（要写回外存）

一道计算题里通常只关心「物理块号」这一字段，其他位对 Q31–Q33 不影响。

📜 公式 ① · 逻辑地址怎么拆：页号（高位）+ 页内（低位）

🧸 一句话：逻辑地址就像个 房号+床号：高几位告诉你「哪间房」（页号），低几位告诉你「房里的哪张床」（页内）。房越大占的高位越多、床越多页内越大。

① 推导

  • 页内地址要覆盖页大小：页大小 = 4096B → 需要 log₂(4096) = 12 位才能编址到每一字节
  • 页号要覆盖总页数：剩余的高位 = 逻辑地址总位 − 页内位
  • 所以：「页大小 = 2N」→「页内位数 = N」（最常考的反向：一看到页大小 = 4KB 立刻反应页内 = 12 位）

② 视觉：逻辑地址 = 页号(高 N 位) + 页内(低 M 位)

典型 16 位逻辑地址、页大小 4096B： 高 4 位 页号 低 12 位 页内地址 ┌────────┬───────────────┐ │ 4 │ 12 │ └────────┴───────────────┘ ↓ 共 16 位 = 4 + 12

④ 反过来推

给一个逻辑地址总位数和页大小，求「页号能编址多少页」：

总逻辑地址位数 = 16 页大小 = 4096B → 页内位 = 12 页号位数 = 16 − 12 = 4 → 总页数 = 2⁴ = 16 页
⚠️ 翻车点：「总页数」和「页内地址数」是两回事。16 页 ≠ 12 位页内=16。B 是地址可编址的范围，2¹² = 4096 字节；页数是页表项编号的总数，2⁴ = 16。混淆会算错 1 个 2 的幂。
公式 ①（默写）
页号位 = log₂(总页数)；页内位 = log₂(页大小)
两者之和 = 逻辑地址总位数

📜 公式 ② · 物理地址 = 物理块 × 页大小 + 页内地址

🧸 一句话：物理地址 = 物理块的起点 + 在块内的偏移。块的起点 = 块号 × 块大小（也就是页大小）。所以「物理块 × 页大小 + 页内」= 「用块号定位起点、再加偏移」。

② Q31 实例：物理块 14，页大小 4096，页内 0xF6A

物理块 14 的「起点」是什么？
  块起点 = 块号 × 块大小 = 14 × 0x1000 = 0xE000

在这个起点上加偏移 0xF6A：
  物理地址 = 0xE000 + 0xF6A = 0xEF6A

③ 通用形式

物理地址 = 物理块号 × 页大小 + 页内地址 易记：块号定起点 + 偏移定细节。乘加顺序固定，物理块号先乘再加。
⚠️ 三大翻车点：
  • 十进制 14 ≠ 0x14。14 转 16 进制 = 0xE（不是 0x14=20）。题目给「块 14」默认是十进制。
  • 4096B = 0x1000（不是 0x4000，不是 0x4096）。
  • 加法时统一十六进制/十进制，否则会算错。

④ 块号 × 页大小的 2 次幂速算

块号 × 页大小，常用对齐到 2 的幂：

块号× 4096B× 1024B× 2048B
块 10x10000x4000x800
块 70x70000x1C000x3800
块 140xE0000x38000x7000
块 160x100000x40000x8000

规律：块号 N × 0x1000 = 把 N 转成十六进制后面补 3 个 0（每补 1 个 0 = 乘 16）。

公式 ②（默写）
物理地址 = 物理块 × 页大小 + 页内
乘加顺序不能反；4096 = 0x1000；十进制 14 = 0xE

📜 公式 ③ · 拼接视角：两边共享页内地址

🧸 一句话：逻辑地址和物理地址其实是「同构」的——都是「高位 + 低位」。翻译只把高位换掉（页号 → 物理块号），低位（页内地址）保持原样。这就是为什么页大小相等时，翻译就是「查表换前缀」。

② 为什么页内地址不变？

底层原因：页大小 = 物理块大小（同一个值）。页内的字节数 = 块内的字节数。所以「一页的第 d 字节」和「一块的第 d 字节」是同一个偏移位置。翻译时不需要重新算偏移——直接把页内地址当成块内偏移。

③ 等价性证明（看一眼就过）

公式 ②：物理地址 = B × S + d 公式 ①：d 的位数 = log₂(S) 把公式 ②拆位： 高位 = log₂(S) 位 = floor(物理地址 / S) = B ← bk 低位 = log₂(S) 位 = 物理地址 mod S = d ← off 所以 [物理地址] 的高 log₂(S) 位就是 B，低 log₂(S) 位就是 d。✓

④ 和 Q31 连上

Q31 的答案是 0xEF6A，展开看：

1110 1111 0110 1010

▲ 物理块号（高位） = 1110B = 14，正是 Q31 查表得来的那个 14
▲ 页内地址（低位） = 1111 0110 1010B = 0xF6A，和原逻辑地址的低 12 位完全相同

这就是公式 ③「拼接视角」的实际样子——翻译只换高 4 位，低 12 位保持原样。

⚠️ 注意：「拼接」和「加法」（公式 ②）是同一件事的两种看法：加法看数值、拼接看位。算 0xE000 + 0xF6A 就是在把 0xE 和 0xF6A 拼成 0xEF6A。看到任何一种说法都该本能反应到另一种。
公式 ③（拼接视角）
[逻辑] = [页号][页内]　→　查表　→　[物理] = [块号][页内]
页内位 = 偏移位 = log₂(页大小)；翻译只换前缀

📜 公式 ④ · 内存多大：块数 × 页大小

② 关键区分：物理 vs 逻辑

量意义由什么决定
物理块数 N（题目里写「内存 8 块」）硬件装得下多少物理内存大小 / 页大小
逻辑页数 M（题目里写「逻辑地址空间最大 16 页」）进程能编址多少逻辑地址大小 / 页大小
页大小 S两者通用设计者定
物理内存大小硬件= N × S
逻辑地址空间大小进程视角= M × S

考试最容易在这里翻车：题干说「内存 8 块」→ 这是 N=8，不是 M；说「逻辑地址空间 16 页」→ 这是 M=16，不是 N。两个数可以完全不同。

③ 桥接 Q32

Q32 题面：

题面：页式管理，逻辑地址空间最大 16 页（这是 M=16），每页 2048B（这是 S），内存 8 块（这是 N=8）。求：① 逻辑地址至少多少位？② 内存空间多大？
  • ① 位数：M=16 → log₂(16) = 4 位页号；S=2048 → log₂(2048) = 11 位页内；总 = 15 位
  • ② 内存：N × S = 8 × 2048 = 16384B = 16KB
公式 ④
内存大小 = 物理块数 × 页大小
物理块数 ≠ 逻辑页数（前者硬件、后者进程视角）

📜 公式 ⑤ · 多级页表寻址：项数倍叠加

🧸 一句话：一级页表就一页大小，每一项指向一个物理块 → 最多能寻址「一页能装多少项 × 一项多大内存」。想寻址更大？把单级拆成多级。每多一级寻址翻 N 倍（N=一页能装多少项）。

① 一级页表的瓶颈

页大小 = 4KB，页表项 = 4B：

一页页表能装多少项： 4KB / 4B = 4096 B / 4 B = 1024 项 = 2¹⁰ 项 一级寻址上限： 1024 项 × 4KB/项 = 4 MB = 2²² B

也就是说，一级页表只能寻址 4 MB。要寻址 4GB 必须上多级。

② 二级页表的翻倍效果

级数总项数寻址上限逻辑地址位查表次数
一级10242²² B = 4 MB10 + 12 = 22 位1
二级1024 × 1024 = 1M2³² B = 4 GB10 + 10 + 12 = 32 位2

④ 为什么不是「多级 = 二倍」？

核心：不是「多级 = 2 倍寻址」。是「多级 = 项数倍叠加」——每多一级，寻址翻 N 倍，N = 一页能装多少项 = 页大小 / 项大小。
每级 N 项、级数 k： 总项数 = N^k 寻址上限 = N^k × 页大小 逻辑地址位数 = k × log₂(N) + log₂(页大小)

⑤ 桥接 Q33

Q33：页大小 4KB、项 4B。① 一级寻址？② 二级？

  • ① 1024 × 4KB = 4MB
  • ② 1024 × 1024 × 4KB = 1024² × 4KB = 1M × 4KB = 4GB
⚠️ 三大易错：
  • 一级寻址 ≠ 4GB。一级受限于「一页能装多少项」，最大 4MB（1024 项 × 4KB）。要 4GB 必须上二级。
  • 「多级 = 二倍」是幻觉。是「项数次方倍」，不查清 N = 1024 就会算错。
  • 项大小 ≠ 物理块大小。一项是页表里的元数据（4B、8B 常考），一块是物理内存里的数据块（4KB 最常考）。
公式 ⑤
页表项数/页 = 页大小 / 项大小；寻址 = 项数级数 × 页大小
1024 项/页 × 4KB/项 → 1 级 4MB，2 级 4GB

自测 1. 16 位逻辑地址，页大小 1KB，第 5 页存在物理块 9。求逻辑地址 0x3A21 的物理地址。
答案
页大小 1KB = 2¹⁰ → 页内 10 位 → 页号 6 位
0x3A21 = 0011 1010 0010 0001
高 6 位 = 001110 = 14（注意题面只给了 5 页的映射，所以默认未映射的页算 0 块）
低 10 位 = 10 0010 0001 = 0x221
物理地址 = 9 × 0x400 + 0x221 = 0x2400 + 0x221 = 0x2621
自测 2. 页式管理，逻辑地址空间 32 页，每页 512B，内存 16 块。① 逻辑地址至少多少位？② 内存多大？
答案
① 32 = 2⁵ → 页号 5 位；512 = 2⁹ → 页内 9 位；总 = 14 位
② 内存 = 16 × 512B = 8192B = 8 KB
自测 3. 页大小 8KB，页表项 8B。① 一级页表最大寻址？② 二级？
答案
项数/页 = 8KB / 8B = 1024 项 = 2¹⁰
① 一级 = 1024 × 8KB = 8 MB
② 二级 = 1024² × 8KB = 1M × 8KB = 8 GB
自测 4. 为什么分页管理下，「物理块数」和「逻辑页数」可以不一样大？
答案
物理块数 = 硬件内存 / 页大小（硬件装得下多少）。
逻辑页数 = 逻辑地址空间 / 页大小（每个进程以为它独占多少）。
前者由机器内存条决定，后者由 CPU 地址线位数 / 操作系统设计决定。二者没有固定对应关系，可以差很大。虚拟内存就是利用了这个差异（逻辑地址空间可以远大于物理内存，靠置换扩展）。
自测 5. 题目说「块号 14」，要算物理地址，最容易写错的点是什么？
答案
最容易写错的点（也是 Q31 的重灾区）：
1. 把块号 14 当十六进制 → 0x14 = 20 → 算成 20 × 4096 = 0x14000 + 页内 → 错！
块号默认是十进制，14 转十六进制是 0xE，正确起点 14 × 4096 = 0xE000。
2. 4096 当十六进制算 → 0x4096 = 16534 B，错的离谱。
4096B 默认是十进制 = 0x1000 十六进制。
3. 加法左边十进制右边十六进制 → 必然算错。
先统一到十六进制再加。
自测 6. 二级页表最大寻址 = N² × 页大小，为什么不是 N² + 页大小？
答案
寻址能力 = 能定位多少个独立字节。
一级寻址：N 项 × 每项指 1 块 × 每块 S 字节 = N × S 字节。
二级寻址：顶级 N 项 × 二级 N 项 × 每项指 1 块 × 每块 S 字节 = N × N × S = N² × S 字节。
这是相乘，不是相加——每一级都在把「可寻址数」放大 N 倍，所以是 Nk。把「相乘」写成「相加」是初学者最容易犯的算式错。
