三级调度:作业进门、内存腾挪、CPU 分配 —— 三层各管一段

这是简答题 5 分大题的标准答案骨架——高级/中级/低级 = 作业/内存/CPU 三层。教材出处:3_处理机调度/ppt文本.md 幻灯片 26-41(第一节调度层次)。背景见 0002 调度算法、0044 多级反馈队列。

考场必背两句话:
① 三级调度 = 高级调度(作业调度)+ 中级调度(内存调度)+ 低级调度(CPU 调度);频率从低到高、对象从粗到细。
② 高级调度决定"作业能不能进系统",中级调度决定"内存中的要不要换出",低级调度决定"谁上 CPU"。

🗺 0. 这道简答题在考什么

🧸 一句话: 把系统想成 "排队办事大厅"——高级是门口放谁进、中级是里面人太多把人请出去歇会、低级是窗口前叫谁过来办。

5 分题的标准答法:三级各 2 句话定义(3 分)+ 区别对照表(1 分)+ 联系一句话收束(1 分)。

📖 1. 是什么:处理机的三级调度

调度层次: 在多道程序环境下,处理机的调度并不是一次完成的,而是按调度对象分为三个层次——从外存到内存、再到 CPU——层层递进。

教材原话(幻灯片 41): "具有三级调度的调度队列模型"——高级、中级、低级共同协调,使系统在宏观(吞吐量)和微观(单进程响应)两方面都取得较好表现。

🚪 2. 高级调度(作业调度)

🧸 类比: 公司门口的 HR——决定外面排队的求职者里今天放哪几个进公司。
  • 对象: 处于后备状态(外存)的作业。
  • 工作: 按算法从后备队列挑作业 →分配内存和外设 → 建立进程 → 装入内存进入就绪。
  • 频率: 最低(批处理 / 多道程度决定)。
  • 目的: 为多道程序服务,决定系统能"装多少作业"。
  • 教材原话(幻灯片 27):"高级调度即作业调度,它决定允许哪些作业参与竞争 CPU 和其它系统资源,将一个或一批作业从后备状态变为运行状态;高级调度是为作业分配虚拟处理机,也称宏观调度。"

🔄 3. 中级调度(内存调度)

🧸 类比: 办公椅不够时把短期不用的 人请去休息室"挂起"(腾出工位)、忙的时候再叫回来"激活"。
  • 对象: 已经在内存中的进程。
  • 工作: 决定哪些内存进程暂时换出到外存(挂起→静止就绪/静止阻塞),哪些再换回内存(激活→活动就绪/活动阻塞)。
  • 频率: 中等(内存吃紧时触发)。
  • 目的: 提高内存利用率和系统吞吐量,处理"内存不足 → 挂起部分进程"的局面。
  • 教材原话(幻灯片 32):"中级调度决定哪些进程可参与竞争 CPU;中级调度将进程从活动态(活动就绪、活动阻塞)变为静止的挂起态(静止就绪、静止阻塞),或相反;中级调度实际上是实现'挂起'和'激活'操作;中级调度也称为进程交换调度,通常仅用于分时系统。"
关键术语: 中级调度 = "挂起 / 解挂"调度,也叫对换(Swapping)调度。

⚡ 4. 低级调度(进程调度 / CPU 调度)

🧸 类比: 窗口前的 柜员叫号——就绪队列里"下一个是谁"。
  • 对象: 活动就绪队列中的进程。
  • 工作: 按算法从中选一个 → 分配 CPU → 状态切换为"运行"。
  • 频率: 最高(毫秒级,几十 ms 一调)。
  • 目的: 让 CPU 不停歇、让用户响应及时。
  • 教材原话(幻灯片 34):"低级调度即进程调度,低级调度决定哪个进程可获得 CPU;低级调度从活动就绪队列中挑选一个进程,将它变为运行态,同时启动 CPU 执行该进程;低级调度也称微观调度。"

📐 考场默写模板: 
三级调度的联系与区别:
· 高级调度(作业调度):对象=后备作业,发生在"外存→内存",
  决定多道程序度,频率最低,又称长程调度/宏观调度;
· 中级调度(内存调度):对象=内存中进程,发生在"内存↔外存",
  实现挂起/激活,提高内存利用率,频率中等,又称交换调度/中程调度;
· 低级调度(进程调度):对象=活动就绪进程,发生在"就绪→运行",
  分配 CPU,频率最高,又称微观调度/短程调度。

联系:三级调度层层递进——高级调度决定"作业能不能进",
中级调度决定"在内存的要不要出去",低级调度决定"谁上 CPU"。

🔗 6. 联系:三级调度层层配合

三级调度不是孤立的——它们共同决定"一个作业从外存到完成"的整个生命周期:

  1. 高级调度 先放作业进内存(没有它=作业永远进不了系统);
  2. 低级调度 让它们一个个上 CPU 跑(没有它=有 CPU 也用不上);
  3. 中级调度 在内存吃紧时把短期不用的进程挂起腾地方,需要时再激活(没有它=内存满了就死锁)。

三者的分工粒度从粗到细、频率从低到高:高级(粗/低频) → 中级(中/中频) → 低级(细/高频)。

📝 7. 必背 4 句话

① 三级调度 = 作业调度(高级)+ 内存调度(中级)+ CPU 调度(低级)。
② 高级决定"作业能否进系统",中级决定"内存中的要不要换出",低级决定"谁上 CPU"。
③ 频率从低到高、粒度从粗到细:高级(分钟级,宏观)→中级(内存紧张时,中观)→低级(毫秒级,微观)。
④ 中级调度的本质是"挂起 / 激活"(对换),不是把所有进程倒来倒去的"交换"。

⚠️ 8. 易错点

坑 1 · 把"中级调度"和"交换"画等号: 中级调度是"挂起 / 激活"决策,真正动手搬运页 / 进程是 OS 的对换程序;不是同一件事。答题不要把"中级调度"说成"交换调度"而不提"挂起/激活"。
坑 2 · 把"高级调度"和"作业调度"当两件事: 是同一件事两个名字(宏观调度 / 长程调度 / 作业调度都一样)。同样,"低级调度 = 进程调度 = CPU 调度"。
坑 3 · 频率记反: 高级(最低) → 中级(中等) → 低级(最高)。口诀 "高少低多"(高级调度次数少,低级调度次数多)。
坑 4 · "中级调度只在分时系统": 教材明确说中级调度"通常仅用于分时系统"(幻灯片 32)——批处理系统通常不需要,因为批处理是"作业全部装入再调"。

📝 9. 真题样例(简答 5 分)

样题

简述处理机的三级调度,并比较高级调度、中级调度和低级调度的主要区别。(5 分)

点我看参考答案

答:

(1) 三级调度: 在多道程序环境下,处理机的调度按对象和粒度分为三层:高级调度(作业调度)、中级调度(内存调度)和低级调度(进程调度/CPU 调度)。

(2) 三者的主要区别:

  • 高级调度:对象是作业,发生在外存到内存之间,频率最低,目的是为多道程序服务,决定多道程序度,也称宏观调度/长程调度。
  • 中级调度:对象是内存进程,发生在内存与外存之间(挂起/激活),频率中等,目的是提高内存利用率和系统吞吐量,也称交换调度/中程调度,通常仅用于分时系统。
  • 低级调度:对象是活动就绪进程,发生在就绪到运行之间,频率最高,目的是将 CPU 分配给选中的进程,也称微观调度/短程调度。

(3) 联系: 三级调度层层递进、互相配合——高级调度决定"作业能否进系统",中级调度决定"已进系统的要不要换出",低级调度决定"谁上 CPU"。

出处:3_处理机调度/ppt文本.md 幻灯片 26-34、41。关键词:三级调度名 + 频率从低到高。

✅ 10. 自测

Q1 · 默写挑战

不看屏幕,默写:三级调度名称、对象、发生位置、频率高低。

点我对答案

· 高级调度:对象=作业,外存→内存,频率最低;

· 中级调度:对象=内存进程,内存↔外存(挂起/激活),频率中等;

· 低级调度:对象=活动就绪进程,就绪→运行,频率最高。

口诀 "高级→中级→低级,频率从低到高"。

Q2 · 易错辨析

下列说法哪个是错的?

A. 高级调度又称作业调度、宏观调度、长程调度,对象是处于后备状态的作业
B. 中级调度又称内存调度、交换调度,实现进程的挂起和激活
C. 低级调度频率最低,但目标是把 CPU 分给合适的进程
D. 三级调度频率从高到低依次是低级、中级、高级

点我看答案

答案:C。低级调度"频率最高、粒度最细",C 说反了。其它三项正确。D 是正确的——频率从高到低正好就是 低级(高)→中级→高级(低)。

Q3 · 联系三态

中级调度把"活动就绪"挂起会变成什么状态?再激活回什么状态?为什么中级调度常被称为"对换"调度?(提示:见 0042 进程状态转换)

点我对答案

· 活动就绪(在内存、就绪)被挂起 → 静止就绪(在外存、就绪);

· 活动阻塞被挂起 → 静止阻塞;

· 激活则是反向过程(静止就绪→活动就绪,静止阻塞→活动阻塞)。

· 因为它来回搬运的是整个进程的内存映像——从内存写回外存 / 从外存读回内存,所以叫"对换调度"(swapping)。与分页级别的"页面换出/换入"是不同粒度的对换。
