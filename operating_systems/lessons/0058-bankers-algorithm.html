考场必背两句话:
① 4 个数据结构:Available(可用资源向量)、Max(最大需求矩阵)、Allocation(已分配矩阵)、Need(尚需矩阵 = Max - Allocation)。

🗺 0. 这道大题在考什么

🧸 一句话: 银行家算法就是"先假设你要的,我会先算账——算完确认不会死锁,再真给"。死锁判断 = 找一个安全序列;死锁处理 = 算一次试分配 + 再判安全。

10 分题的标准拿法:写 4 个数据结构(2 分)+ 写 5 步操作流程(2 分)+ 算 T0 时刻安全序列(4 分)+ 处理进程请求(2 分)。流程分和计算分各占一半,写不出来时哪怕只默写流程也能保底 4 分。

📖 1. 是什么:银行家算法 = 死锁避免的代表算法

定义: 银行家算法由 Dijkstra 提出,是死锁避免的经典算法——在每次分配资源前,先试探分配,再用安全性算法检查是否会进入不安全状态;若会进入不安全状态则拒绝分配。

教材原话(幻灯片 320):"最著名的死锁避免算法是 Dijkstra 提出的银行家算法。这是由于该算法能用于银行系统现金贷款的发放而得名。"

背景:出自教材 3.6.3 节(利用银行家算法避免死锁)。本章和 0059 调度算法都是 10 分大题。

🧱 2. 4 个数据结构(必默写)

名字类型含义关键关系
Available向量(长度 m)系统中每类资源的当前可用数Available[j] = K → 现有 Rj 类资源 K 个
Maxn×m 矩阵每个进程对每类资源的最大需求Max[i][j] = K → 进程 i 最多需要 Rj 类 K 个
Allocationn×m 矩阵每个进程已经分到的资源数Allocation[i][j] = K → 进程 i 已分得 K 个
Needn×m 矩阵每个进程尚需的资源数Need[i][j] = Max[i][j] − Allocation[i][j]
记忆口诀:"有最大、已分配、还差多少"——Max(我要最多多少)、Allocation(已经拿到多少)、Need(还差多少 = Max - Allocation)、Available(系统仓库还剩多少)。四个东西、三类角色、一条减法。

🪜 3. 5 步操作流程(必默写)

设进程 Pi 发出请求向量 Requesti(长度为 m):

1检 Request ≤ Need
若 Requesti[j] > Needi[j] 则报错(进程要的比它声称的最大还多,异常)。
2检 Request ≤ Available
若 Requesti[j] > Available[j] 则让 Pi 等待(系统没货)。
3试探分配
修改 4 个量:Available −= Request, Allocationi += Request, Needi −= Request。暂时分配,未生效。
4安全性算法检查
用试分配后的状态跑安全性算法,看能否找到安全序列。
5分支
若能找到安全序列 → 真分配(把第 3 步的修改落定);否则 → 回滚(恢复试分配前的状态,让 Pi 等待)。

🔍 4. 安全性算法(子流程)

输入: 试分配后的 Available / Need / Allocation。

输出: 安全序列 <Pi1, Pi2, ..., Pin> 或 "不安全"。

步骤

  1. 设工作向量 Work = Available,Finish[i] = false(所有进程)。
  2. 找满足 Finish[i] = false 且 Needi[j] ≤ Work[j] 的进程 Pi。找不到转第 4 步。
  3. 假装 Pi 完成:Work += Allocationi,Finish[i] = true。回到第 2 步。
  4. 若所有 Finish[i] = true → 安全,序列就是找到的顺序;否则 → 不安全。
口诀:"找得到就假装完成释放"——找到 Need ≤ Work 的进程,假装它跑完,把 Allocation 加回 Work,继续找下一个。这就是模拟"一个一个进程跑完,系统能否清仓"。

✍ 5. 手算例题(教材幻灯片 335-339 改编)

题目: 系统有 5 个进程 P0~P4,3 类资源 A/B/C,数量分别为 10/5/7。T0 时刻分配如下:

进程Allocation(已分)Max(最大需求)
ABCABC
P0010753
P1200322
P2302902
P3211222
P4002433

问(1): 计算 Need = Max − Allocation,写出 T0 时刻的 Need 矩阵和 Available 向量。

问(2): 用安全性算法判断 T0 是否安全,若安全给出一个安全序列。

问(3): 此时 P1 发出请求 Request1=(1, 0, 2),系统能否分配?

5.1 第一步:算 Need 和 Available

Need = Max − Allocation(对应位置相减):

进程Need ANeed BNeed C
P07−0=75−1=43−0=3
P13−2=12−0=22−0=2
P29−3=60−0=02−2=0
P32−2=02−1=12−1=1
P44−0=43−0=33−2=1

Available = 总资源 − Σ Allocation = (10,5,7) − (0+2+3+2+0, 1+0+0+1+0, 0+0+2+1+2) = (10,5,7) − (7, 2, 5) = (3, 3, 2)

5.2 第二步:安全性算法(找安全序列)

Work = Available = (3, 3, 2),Finish[i] 初始全部 false。

轮次找到的 PiNeed ≤ Work?Work 累加Finish
1P1(1,2,2) ≤ (3,3,2) ✓Work += Allocation1 = (3,3,2)+(2,0,0) = (5, 3, 2)F[P1]=true
2P3(0,1,1) ≤ (5,3,2) ✓Work += Allocation3 = (5,3,2)+(2,1,1) = (7, 4, 3)F[P3]=true
3P0(7,4,3) ≤ (7,4,3) ✓Work += Allocation0 = (7,4,3)+(0,1,0) = (7, 5, 3)F[P0]=true
4P2(6,0,0) ≤ (7,5,3) ✓Work += Allocation2 = (7,5,3)+(3,0,2) = (10, 5, 5)F[P2]=true
5P4(4,3,1) ≤ (10,5,5) ✓Work += Allocation4 = (10,5,5)+(0,0,2) = (10, 5, 7)F[P4]=true

结论:T0 安全,安全序列 <P1, P3, P0, P2, P4>(注意:每次从满足条件的进程中任选一个即可,本答案选了 P1 起手)。

5.3 第三步:处理 P1 请求 Request1=(1, 0, 2)

①检 Request ≤ Need
(1,0,2) ≤ (1,2,2) ✓ 通过
②检 Request ≤ Available
(1,0,2) ≤ (3,3,2) ✓ 通过
③试探分配
Available=(2,3,0);Allocation1=(3,0,2);Need1=(0,2,0)
④安全性算法
Work=(2,3,0);P1 Need=(0,2,0)≤(2,3,0)✓ → Work+=(3,0,2)=(5,3,2);然后 P3→(7,4,3)→P0→(7,5,3)→P2→(10,5,5)→P4→(10,5,7)全部 Finish=true
⑤结论
存在安全序列 <P1,P3,P0,P2,P4>,可以分配

⚠️ 7. 易错点

坑 1 · 把 Need = Max + Allocation: 必考易错点!Need = Max − Allocation(还差多少),不是相加。Allocation 是"已分到的",Need 是"还需要的",两者相减。
坑 2 · Available 直接抄题: 有些题目不直接给 Available,需要自己算:Available = 总资源 − Σ Allocation(按列求和)。
坑 3 · 安全性算法 Work 累加忘改: 找到满足 Need ≤ Work 的进程 Pi 后,Work 要累加 Allocationi(不是 Needi)——因为进程完成会释放它已分配的全部资源。
坑 4 · "可以分配"漏写试分配后的数据: 哪怕判可以分配,也必须写出试分配后的 Available/Allocation/Need 三个新值——这是 2 分的实操分。

样题(改编自教材幻灯片 368)

系统 5 进程 P0~P4,4 类资源 A/B/C/D,Available = (1, 6, 2, 2)。Allocation / Need 如下:

进程AllocationNeed
ABCDABCD
P000320012

P303320652
P400140656

问(1): 该系统是否安全?问(2): 若 P2 提出请求 (1, 2, 2, 2),系统能否分配?

点我看参考答案

答(1):用安全性算法。Work = (1, 6, 2, 2)。

  • P0: Need=(0,0,1,2) ≤ (1,6,2,2) ✓ → Work += Alloc0=(0,0,3,2) → Work=(1,6,5,4);Finish[P0]=true
  • P2: Need=(2,3,5,6) ≤ (1,6,5,4)?否(A 资源 2>1)。
  • P3: Need=(0,6,5,2) ≤ (1,6,5,4) ✓ → Work+=(0,3,3,2)=(1,9,8,6);Finish[P3]=true
  • P2: Need=(2,3,5,6) ≤ (1,9,8,6)?否。P1/P4 Need 也都 ≥ 当前 Work。
  • 剩余进程均无法满足 → 系统不安全。

答(2):P2 请求 (1,2,2,2)。

  • ① Request=(1,2,2,2) ≤ Need=(2,3,5,6) ✓
  • ② Request=(1,2,2,2) ≤ Available=(1,6,2,2) ✓
  • ③ 试分配:Available=(0,4,0,0);Alloc2=(2,5,7,6);Need2=(1,1,3,4)
  • ④ Work=(0,4,0,0)。所有进程 Need 第一分量均 ≥ 1(P2=P0 的 1, P0=0)——P0 Need=(0,0,1,2) ≤ (0,4,0,0)?否。找不到满足的进程 → 不安全。
  • ⑤ 不能分配,让 P2 等待。

✅ 9. 自测

Q1 · 数据结构默写

不看书,默写银行家算法的4 个数据结构 + 它们之间的关系。

点我对答案

4 个数据结构:Available(可用资源向量,长度 m)、Max(最大需求矩阵,n×m)、Allocation(已分配矩阵,n×m)、Need(尚需矩阵,n×m)。

关键关系:Need[i][j] = Max[i][j] − Allocation[i][j](记一个减号,别记成加号)。

卡壳了回到 §2 重新读一遍。

Q2 · 算法流程默写

不看书,默写进程 Pi 请求资源后的5 步操作流程。

点我对答案

①检 Requesti ≤ Needi(否则报错)→ ②检 Requesti ≤ Available(否则等待)→ ③试探分配(Available−=Request,Allocation+=Request,Need−=Request)→ ④执行安全性算法→ ⑤安全才真分配,不安全则回滚并等待。

Q3 · 易错辨析

下面哪个说法是错的?

A. Need = Max − Allocation,等于 0 表示进程已分配到最大需求
B. 安全性算法中,找到满足 Need ≤ Work 的进程后,Work 要 += Allocation(不是 += Need)
C. 银行家算法是死锁检测算法,允许分配后再检测死锁
D. 试探分配后若安全性算法检查不安全,系统会让该进程等待并回滚试分配

点我看答案

答案:C。银行家算法是死锁避免算法(在分配前试探),不是死锁检测算法(在分配后检测)。A/B/D 都是教材原话。
