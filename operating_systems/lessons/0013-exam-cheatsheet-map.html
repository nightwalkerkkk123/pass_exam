★必考Ch2

PV 操作 / 信号量

P(S)：S-- ；若 S<0 则本进程阻塞（申请资源） V(S)：S++ ；若 S≤0 则唤醒一个阻塞进程（释放资源） S>0 = 可用资源数 ； S<0 = |S| 个进程正在等待
  • 互斥信号量初值 =1；同步/前趋信号量初值 =0；P、V 必成对。
  • 生产者-消费者：mutex=1, empty=n, full=0；生产 P(empty)→P(mutex)…V(mutex)→V(full)。
  • 读者-写者：读读并发、读写/写写互斥；第一个读者 P(rw)、最后一个读者 V(rw)。
⚠ 两个 P 不能颠倒：必须「先资源 P(empty) 后互斥 P(mutex)」，颠倒会死锁。
→ 0010 同步 · 0011 经典同步
★必考Ch3

进程/作业调度计算

周转时间 = 完成时间 − 到达时间 带权周转 = 周转时间 ÷ 服务时间（分母是运行时间！） 平均(带权)周转 = 各进程之和 ÷ 进程数 响应比(HRRN) = 1 + 等待时间 ÷ 服务时间
  • FCFS 按到达序；SJF 选最短作业；RR 时间片轮转；HRRN 选响应比最高。
  • SJF / HRRN 非抢占：上了 CPU 就跑到完，中途更短的不打断。
⚠ 带权周转的分母是「服务时间(运行时间)」，不是周转时间；相等时按到达先后。
→ 0002 调度算法
★必考Ch3

银行家算法 + 安全序列

Need = Max − Allocation 收到 Request 时： ① Request ≤ Need ② Request ≤ Available ③ 试分配 ④ 跑安全性算法找安全序列 安全性：Work=Available；反复找 Need[i]≤Work 的进程， 令其完成 Work += Allocation[i]，全部完成 = 安全
  • 口诀：Request ≤ Need ≤ (原)Available。找到安全序列才真分配，否则回退等待。
  • 安全态一定不死锁；不安全态未必死锁但有风险。
→ 0003 银行家算法（含 P0–P4 例题）
Ch4

分页地址变换 + 快表 EAT

页号 = 逻辑地址 ÷ 页大小（整除） 页内偏移 = 逻辑地址 % 页大小（取余） 物理地址 = 物理块号 × 页大小 + 页内偏移 EAT = (访存+查快表)×命中率 + (2×访存+查快表)×(1−命中率)
  • 页大小是 2 的幂 → 高位=页号、低位=偏移，直接位截取，不用真做除法。
⚠ 看题面：若说「快表与访存并行」或「不计快表时间」，EAT 公式相应删项。
→ 0005 分页基础
★必考Ch4–5

页面置换缺页率

缺页率 = 缺页(调入)次数 ÷ 总访问次数 OPT 淘汰「将来最久不用」｜ FIFO 淘汰「最先进入」 LRU 淘汰「最近最久未用」｜ CLOCK 扫描淘汰访问位=0
  • 做题：画物理块表逐个访问，不在内存就缺页并按判据淘汰；首次装入也算缺页。
  • Belady 异常（块变多、缺页反增）只出现在 FIFO；LRU/OPT 是堆栈类，绝不出现。
→ 0006 页面置换 + Belady
Ch5

磁盘调度 + 访问时间

磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间 旋转延迟(平均) = 半圈 = 1 ÷ (2 × 转速) 传输时间 = 读取字节数 ÷ 传输速率 FCFS｜SSTF 最近优先｜SCAN 电梯｜C-SCAN 单向到端跳回
  • 移动总距离：按算法定顺序，累加相邻磁道差的绝对值。
⚠ 看题面：SCAN/C-SCAN 是否走到物理端点再折返、C-SCAN 回跳是否计距离，题目说了算；SSTF 会饿死两端。
→ 0007 磁盘调度
Ch6

混合索引 · 最大文件大小

N = 块大小 B ÷ 地址项大小 A （每块能存的块号数） 最大文件 = D×B + I1·N·B + I2·N²·B + I3·N³·B UNIX 例：B=4KB,A=4B,D=10,N=1024 = 40KB + 4MB + 4GB + 4TB
  • 直接地址给小文件（一步到）；一/二/三级间接给大文件（多翻几张清单）。
⚠ 别套记忆数：本课速查表用 D=10（不是王道常见的 13）。一律按老师给的 D / N 代入。
→ 0009 混合索引 · 0023 间接块
★必考Ch6

成组链接法（空闲块）

分配1块：栈顶 count>1 → 弹栈顶块号, count-- count==1（该块记着下一组）→ 先读入下一组到栈, 再分配该块 回收1块：栈未满 → 压栈顶, count++ 栈满 → 当前整组写入被回收块, 该块成新组首(count=1)
  • 栈里第一个数是「块数」不是块号；本仓库例题栈顶在右。
⚠ 对齐老师课件：速查表/总框架均未写此法，上面按汤4版标准。老师每年换数重考，务必用老师范例 1–9 的写法。
→ 0004 成组链接法
Ch4

动态分区 FF / NF / BF / WF

FF 首次适应：空闲链按地址递增，取第一个够用的 NF 循环首次：从上次结束处接着找 BF 最佳适应：选最接近需求的最小块 → 留大量小碎片 WF 最坏适应：选最大块切 → 很快无大块可用
  • 内碎片＝分配单位内的零头（分页/固定分区）；外碎片＝散碎凑不出大块（连续/分段），可用紧凑消除。
→ 见 0005 对照分页；连续分配为老师习题常客。

选择/填空/判断的弹药库。⚠ 辨析 行是最容易设坑的地方。点上方「只看高频」可把中低频折起来，考前一晚只扫红标 高。

第 1 章 · 操作系统引论

  • 中OS 定义：管理/控制软硬件资源、合理组织流程、方便用户使用的系统软件（关键词：有效·合理·方便）。
  • 高四大特征：并发（宏观同时·微观交替）、共享、虚拟（时分/空分复用）、异步。并发是核心。
  • 并发 vs 并行：并发＝同一时间间隔内交替（单 CPU 可）；并行＝同一时刻真正同时（需多核）。
  • 共享分两类：互斥共享（临界资源，如打印机）与同时访问（如磁盘/只读代码）。并发与共享互为存在条件。
  • 中五大功能：处理机、存储器、设备、文件管理 + 用户接口。接口三类：命令 / 程序(系统调用) / 图形。
  • 系统调用是用户程序取得 OS 服务的唯一途径（库函数 ≠ 系统调用）。
  • 用户态 vs 核心态：特权指令（启动 I/O、置 PSW、开关中断）只能核心态执行；经 trap（访管指令）陷入切换。
  • 低发展史：单道批处理→多道批处理（利用率高、无交互）→分时（交互及时）→实时（及时+可靠）。
  • 设计首要目标：批处理→吞吐量/周转；分时→响应时间；实时→及时性/可靠性。

第 2 章 · 进程管理

  • 高进程＝程序的一次执行，是资源分配与调度的基本单位；有动态/并发/独立/异步性。
  • 程序 vs 进程：程序静态永久，进程动态暂时；一个程序可对应多个进程。
  • 高PCB 是进程存在的唯一标志，创建进程即创建 PCB。
  • 高三态：就绪(除 CPU 外资源齐)、运行(占 CPU)、阻塞(等事件，给 CPU 也不能跑)。
  • 两条不可能转换：阻塞→运行（必先经就绪）、就绪→阻塞（就绪不占 CPU，不会去等 I/O）。
  • 阻塞是进程自己调 block（主动）；唤醒由别的进程调 wakeup（被动）。
  • 进程 vs 线程：进程＝资源分配单位；线程＝CPU 调度单位，切换开销小、共享进程资源。
  • 中临界资源：一次只允许一个进程用；临界区：访问它的那段代码。
  • 同步四准则：空闲让进、忙则等待、有限等待(防饥饿)、让权等待(等待时放 CPU、防忙等)。
  • 低进程通信三类：共享存储、消息传递(send/receive)、管道(半双工字节流)。

第 3 章 · 处理机调度 + 死锁

  • 中三级调度：作业(高级，频率最低)、中级(内外存对换/挂起)、进程(低级，最频繁)。
  • 中指标：CPU 利用率、吞吐量、周转、带权周转、等待、响应时间。
  • 周转(批处理关心) vs 响应时间(分时关心)；带权周转反映短作业被长作业拖累程度（≥1，越接近 1 越好）。
  • 中RR 时间片：过大退化为 FCFS，过小切换开销大；多级反馈队列公认较优。
  • 高死锁四必要条件：互斥、请求与保持、不可剥夺、循环等待。破坏任一即可预防。
  • 必要 ≠ 充分：死锁 ⇒ 四条件全成立；四条件成立未必死锁。
  • 循环等待 ≠ 死锁：资源每类仅 1 个实例时「有环 ⟺ 死锁」；多实例时有环未必死锁。
  • 预防 vs 避免：预防＝事先破坏条件(限制强)；避免＝不破坏条件、每次分配前判断是否进入不安全态(银行家)。
  • 中死锁处理四策略：预防 / 避免 / 检测与解除 / 鸵鸟不处理。

第 4 章 · 存储器管理（含虚存）

  • 静态 vs 动态重定位：静态＝装入时改地址、程序不可移；动态＝运行时由重定位寄存器转换、可移可换出。虚存必须用动态。
  • 高分页：逻辑分页、内存分页框，两者等大、离散装入，页表做映射。
  • 内碎片 vs 外碎片：分页只有内碎片；连续分配/分段只有外碎片（可用紧凑消除）。
  • 分页 vs 分段：页＝物理单位/定长/对用户透明/一维；段＝逻辑单位/变长/用户可见/二维(段号,段内偏移)；分段便于共享保护。
  • 中段页式：先分段再分页，地址(段号,页号,偏移)，查段表+页表共两次访存，配快表。
  • 高虚存基于局部性：时间局部性(刚访问的再访问)+空间局部性(邻近将被访问)。
  • 请求分页页表字段（填空高频）：页号、物理块号、状态位 P、访问字段 A、修改位 M/脏位、外存地址。
  • 缺页中断是内部中断(异常)，处理完重执行原指令（不是下一条），一条指令可能触发多次。
  • 抖动(Thrashing)：进程太多、分到块太少→频繁换入换出；用工作集控制并发度来防。

第 5 章 · 设备管理（磁盘 + I/O）

  • 高I/O 四控制方式（CPU 越管越少）：程序直接控制(轮询忙等)→中断驱动→DMA→通道。
  • 中断 vs DMA vs 通道：中断以字为单位；DMA 以块为单位(一块中断一次)；通道以一组块/通道程序为单位(一批才中断一次)。
  • 中缓冲：缓和 CPU 与 I/O 速度差；单缓冲不能全并行，双缓冲可并行，缓冲池供多设备共享。
  • SPOOLing(假脱机)：用磁盘输入井/输出井把独占设备虚拟成共享设备（典型：打印机排队）。
  • 中磁盘访问时间＝寻道+旋转延迟+传输；寻道(移臂)才是优化重点。
  • SSTF 平均寻道短但两端饥饿；SCAN 到端才折返、C-SCAN 到端跳回不反向服务。

第 6 章 · 文件管理

  • 中逻辑结构(用户看)：有结构(记录式)/无结构(流式)；物理结构(磁盘看)：连续/链接/索引。
  • 三种物理结构：连续＝随机访问快但有外碎片、难增长；隐式链接＝易增长但只能顺序访问、断一处全丢；FAT 是「显式链接」不是索引。
  • 高索引分配：每文件一个索引块集中存所有块号，随机访问一步到、无外碎片；小文件浪费索引块。
  • 中目录实现按名存取；FCB 一个＝一个目录项，FCB 集合＝目录。
  • inode 内不含文件名！ 文件名只在目录项里；一个 inode 可被多个目录项指向(硬链接)。
  • 位示图（老师口径 0=空闲、1=已分配）：块号 n=i×8+j；回收 i=n÷8, j=n mod 8。
  • 硬链接 vs 软链接：硬链接共享 inode、count=0 才真删、不能跨文件系统；软链接存路径、可跨系统、原文件删则悬空失效。

④ 易错雷区 · 踩过的坑

这些是最容易在选择/判断题上翻车的辨析点（含你历史记录里两次答错的）。考到就多停一拍。

死锁：必要 ≠ 充分
四条件是死锁的必要条件；四条件都成立未必死锁。（曾两次答错）
FIFO vs LRU / Belady
FIFO 按进入先后淘汰、可能 Belady；LRU 按最久未用淘汰、绝无 Belady。
进程状态两条不可能转换
阻塞→运行、就绪→阻塞不存在。就绪=等 CPU，阻塞=等其他事件。
分页 vs 分段
分页=定长/一维/透明/只内碎片；分段=变长/二维/可见/只外碎片、利于共享。
请求分页页表字段
状态位 P、访问字段 A、修改位 M、外存地址——填空常漏。
inode 不含文件名
文件名只在目录项；inode 只存属性+地址，可被多名指向。
中断 / DMA / 通道 的单位
中断=字，DMA=块，通道=一组块；越往后 CPU 干预越少。
位示图 0/1 含义
本课老师口径 0=空闲、1=已分配（有的教材相反，认老师的）。
硬链接 vs 软链接
硬链接 count=0 才删、不跨系统；软链接存路径、原文件删则失效。
两个 P 不能颠倒
先 P(资源) 后 P(mutex)；颠倒（先互斥后资源）会死锁。
SSTF 会饥饿 / SCAN 折返
SSTF 两端易饿死；SCAN 到端才折返、C-SCAN 到端跳回不反向服务。
系统调用 / 特权指令
系统调用是取得 OS 服务唯一途径；特权指令只能核心态执行。
