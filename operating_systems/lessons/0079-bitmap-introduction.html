位示图（Bitmap）：用一位记录一个盘块的状态

独立专题——位示图是磁盘空闲块管理的核心方法。字号位号必背，常出计算题。教材出处：汤小丹第6章。

0·全局线 1·是什么 2·结构 3·核心公式 4·可视化 5·分配流程 6·回收流程 7·真题 8·自测
考场必背两个公式:
① 块号 → 字号/位号: 字号 = 块号 / 字长（取整） ; 位号 = 块号 % 字长
② 字号/位号 → 块号: 块号 = 字号 × 字长 + 位号
位示图大小: (盘块总数 / 8) 字节 = 盘块总数 / (字长×8) 个字

🗺 0. 这节课解决什么问题

🧸 一句话: 磁盘有几万个盘块，怎么记录"哪些是空的、哪些被用了"？位示图就是最简单直接的办法——一块占一位，0=空，1=占。

位示图要回答 3 件事：

  1. 块号 ↔ 字号/位号 怎么互相换算？
  2. 分配时 怎么找一个空闲块？
  3. 回收时 怎么把块标记为空？

📖 1. 是什么：位示图的本质

定义

位示图(Bitmap)：用若干连续字组成的位数组，每个字有 n 位，每个位对应磁盘上的一个物理块。

  • 位值 0：对应盘块空闲
  • 位值 1：对应盘块已分配

为什么不直接用列表？

方案100万块硬盘的开销特点
空闲链表链表节点很多，遍历慢分配 O(n) 查找
空闲表连续区段数较小时省空间适合连续分配
位示图100万/8 = 125KB分配 O(n/字长) 位运算极快
成组链接Unix 经典方案实现稍复杂

📖 2. 结构：位示图长什么样

假设：盘块总数 N=64，每个字长=16位(2字节)，则位示图共有 64/16 = 4个字。

位示图布局（4字 × 16位）:

绿色=0空闲，红色=1占用。下图展示了块 0、2、5、10、20 已分配的样例。

字、位、块号的关系

  • 字号 i：表示在第几个字（行），从 0 开始
  • 位号 j：表示在字的第几位（列），从 0 开始
  • 块号 b：全局盘块编号，从 0 开始

📖 3. 核心公式：块号 ↔ 字号位号

公式（字长 = L 位）

已知块号 b，求字号 i 和位号 j： i = b / L // 整除 j = b % L // 取模 已知字号 i 和位号 j，求块号 b： b = i × L + j 位示图大小（盘块总数 = N）： 位数 = N 字节数 = N / 8 字数 = N / L (L为字长位数)

推导来源

把位示图想象成一个二维数组 B[i][j]：

  • 第 0 个字（i=0）存的是块号 0~L-1 的状态
  • 第 1 个字（i=1）存的是块号 L~2L-1 的状态
  • 第 i 个字（i）存的是块号 i·L ~ (i+1)·L-1 的状态

所以块号 b 在第 b/L 个字、第 b%L 位。

📖 4. 可视化：动手算字号位号

示例 1：字长=16，求块号 35 的字号位号

字号 i = 35 / 16 = 2 // 第3个字 位号 j = 35 % 16 = 3 // 第4位 验证: 2 × 16 + 3 = 35 ✓

示例 2：字号 5、位号 8（字长 16），求块号

块号 b = 5 × 16 + 8 = 88

示例 3：盘块 N=4096，字长=16，位示图大小？

字数 = 4096 / 16 = 256 个字 字节数 = 4096 / 8 = 512 字节

📖 5. 分配流程：找一个空闲块

伪代码

procedure get_free_block():
    for i in 0..bitmap_size:           // 顺序扫描每个字
        for j in 0..word_length:        // 检查每个位
            if B[i][j] == 0:            // 找到空闲位
                B[i][j] = 1             // 标记占用
                block_no = i * L + j    // 算出块号
                return block_no
    return NONE                         // 全部已分配

改进：位运算加速

现代 OS 不用一位一位查，而是用 CPU 的位扫描指令(BSF/BSR) 硬件加速，一次找出 word 中最低/最高的 0 位。

📖 6. 回收流程：把块标记为空

伪代码

procedure free_block(b):
    i = b / L           // 算出字号
    j = b % L           // 算出位号
    if B[i][j] == 1:    // 校验：原本是占用状态
        B[i][j] = 0     // 置 0 表示回收
    else:
        error("double free!")   // 异常：重复释放

注意

回收前必须校验该位是不是 1。如果回收一个已经是空闲的块(位为0)，位示图状态会错乱。

📝 7. 真题样例

真题（10 分）

某磁盘有 4096 块，字长 32 位。请回答：

  1. 位示图占用多少字节？
  2. 分配一个盘块后，盘块号为 1200，对应位示图中的哪个字号哪个位？
  3. 字号 25、位号 10 对应的盘块号是多少？
点我看答案
(1) 位示图字节数 = 4096 / 8 = 512 字节 字数 = 4096 / 32 = 128 个字 (2) 块号 1200: 字号 = 1200 / 32 = 37 位号 = 1200 % 32 = 16 → 第 38 个字的第 17 位 (3) 字号 25、位号 10: 块号 = 25 × 32 + 10 = 810

真题变种（5 分）

位示图中第 i 字第 j 位对应盘块号为？

答案

块号 = i × L + j（L 为字长）

✅ 8. 自测

Q1 · 公式默写

不看屏幕默写两个公式：块号 → 字号位号；字号位号 → 块号。

答案

块号 → 字号位号：i = b / L，j = b % L

字号位号 → 块号：b = i × L + j

Q2 · 算一算

字长=8 位，求块号 17 的字号和位号。

答案

字号 = 17 / 8 = 2（第3个字）

位号 = 17 % 8 = 1（第2位）

验证：2 × 8 + 1 = 17 ✓

Q3 · 大小计算

某磁盘 16384 块，字长 32 位，位示图占用多少字节？多少字？

答案

字节数 = 16384 / 8 = 2048 字节 (2KB)

字数 = 16384 / 32 = 512 个字

Q4 · 易错辨析

回收一个盘块时，下面哪个说法是错的？

A. 回收前要计算字号和位号
B. 回收前要校验该位原本是 1（占用）
C. 回收时直接把对应位置 0
D. 重复回收一个块（位已是0）系统会崩溃

答案

D。重复回收一般不会"崩溃"，但位示图状态会错乱（实际空闲但位示图显示占用，或反之）。所以要校验后置 0。ABC 都是正确操作。

📊 9. 位示图 vs 其他空闲块管理方法

方法空间开销分配速度回收速度适用场景
空闲表法极省（只记连续区段）中中连续分配
空闲链表法每块一个指针慢 O(n)快小块数场景
位示图N/8 字节快（位运算）极快通用首选
成组链接法每组一个超级块中中Unix 类系统

位示图最大的优点是位运算极其高效——分配时扫一个字比扫描一条链快几个数量级。
