0 · 一句话先说清

位示图的每一位对应磁盘上的一块——0=空闲、1=已用。问题来了:C 语言的最小寻址单位是字节(8 位),不能直接「读第 17 位」。所以每次访问任何一位,都必须分两步:

第几个字节 = bi / 8   |   字节里第几位 = bi % 8

这就是课本公式 n = i × 8 + j 的反推:已知块号 bi,反求 (i, j)。4 行代码就是把这一步玩 4 遍。

1 · 4 行核心源码(kernel/fs.c:75–81)

75      m = 1 << (bi % 8);                  // ① 算掩码:第 j 位置 1
76      if((bp->data[bi/8] & m) == 0){       // ② 取第 i 字节,& 掩码 → 测位
77        bp->data[bi/8] |= m;                  // ③ 把那一位置 1(标记已用)
78        log_write(bp);                       // 改动写日志
79        brelse(bp);                          // 释放缓冲
80        bzero(dev, b + bi);                  // 清零这块新分配的盘块
81        return b + bi;                       // ④ 返回全局块号 = b + bi
四步本质: ① 准星 → ② 检查 → ③ 占用 → ④ 给号。下面让你输入一个块号亲手走一遍。

1算掩码

m = 1 << (bi % 8)
把单独的 1 位「左移」到要查的位置——准星瞄准。

2判空闲

(data[bi/8] & m) == 0?
& 只留掩码那一位,其他位强制清 0。结果 0 = 那一位是 0 = 空闲。

3标已用

data[bi/8] |= m
|= 把那一位强行变 1,其他位不动。这就是「占座」。

4给号

return b + bi
b 是当前位图块覆盖的起始块号,加上局部偏移 bi = 全局磁盘块号。

3 · 课本公式 n = i × 8 + j 在代码里的样子

课本说法含义代码里就是例子(bi = 19)
n全局位号(=哪一块)bi19
i第几个字节(从 0)bi / 819 / 8 = 2(第 2 字节)
j字节里第几位(从 0)bi % 819 % 8 = 3(第 3 位)
n = i×8 + j反向:由 (i,j) 算 ni*8 + j2×8 + 3 = 19 ✓
记忆口诀:/8 找字节,%8 找位。一个除一个余,正好把「全局 1D 索引」拆成「2D 坐标 (字节, 位)」。

4 · 三个位运算,挨个拆给老师看

① m = 1 << (bi % 8) —— 造一个「准星」

从最右边的 1 出发,左移 bi%8 位:

// bi % 8 = 0: 1 << 0 = 0b00000001  ← 准星瞄第 0 位
// bi % 8 = 1: 1 << 1 = 0b00000010  ← 准星瞄第 1 位
// bi % 8 = 3: 1 << 3 = 0b00001000  ← 准星瞄第 3 位
// bi % 8 = 7: 1 << 7 = 0b10000000  ← 准星瞄第 7 位

本质:把单独的一颗子弹,左移到要打的位置。其他位永远是 0,所以 m 同时充当「位选择器」。

② (data[i] & m) == 0 —— 用准星「测一下」

&(按位与):两位都 1 才 1,否则 0。用 m 当模板,把不关心的位全过滤成 0,只剩下我们要测的那一位:

  data[2] = 0b10110110     // ← 第 3 位是 0(空闲)
&       m = 0b00001000
─────────────────────
       结果 = 0b00000000 = 0  →  空闲!可以分配

  data[2] = 0b10111110     // ← 第 3 位是 1(已被别人占)
&       m = 0b00001000
─────────────────────
       结果 = 0b00001000 ≠ 0 → 已用,跳过

关键洞察:结果只可能是 0 或 m,不可能是别的——因为 m 只有那一位是 1。

③ data[i] |= m —— 把那一位「强制点亮」

|(按位或):有一个 1 就是 1。m 只有目标位是 1,所以 |= m 就是「目标位强制变 1,其他位原样保留」:

  data[2] = 0b10110110     // 之前第 3 位 = 0
|       m = 0b00001000
─────────────────────
   新 data = 0b10111110     // ★ 只有第 3 位变 1,其他 7 位完全没动
为什么不能写成 data[i] = m? 那样会把整个字节直接覆盖成 0b00001000,抹掉同字节里其他 7 块的占用记录——丢盘块!这就是「|= 不是 =」必须考的原因。

(对称) bfree 里:data[i] &= ~m —— 「擦除」那一位

回收时反过来:~m 是 m 取反(目标位是 0,其他位全是 1),&= 让目标位被强制清 0、其他位保持原样:

  data[2] = 0b10111110
&     ~m = 0b11110111      // ~m:目标位变 0,其他位变 1
─────────────────────
   新 data = 0b10110110     // ★ 只把目标位清 0

分配 ↔ 回收的对称:|= m(占)对偶 &= ~m(还)。讲到这一步,基本可以打满分。

5 · 答辩高频问

Q1. 为什么不直接写 data[bi] = 1 ?高危
答:data 是 字节数组,data[bi] 会把整个字节当成一个数(8 块的占用情况),根本不是「第 bi 位」。所以必须 /8 找字节、%8 找字节内偏移,再用位运算单独操作那 1 位。这就是 C 里「按位寻址」的标准套路。
Q2. 为什么是 1 << (bi % 8),不是 1 << bi?送分
答:每个字节最多只有 8 位,左移超过 7 就溢出了(C 标准里属于未定义行为)。bi%8 就是把「全局位号」压到「字节内偏移 0–7」,确保位移量永远合法。
Q3. 整个 balloc 复杂度是?高危
答:最坏 O(磁盘总块数 / 位图块大小 × 位图块大小) = O(总块数)——因为外层按位图块顺序扫,内层在一个位图块内顺序扫位。位运算本身是 O(1),但「找空闲」是从头扫,所以位示图法的分配开销和磁盘大小成正比。这就是大磁盘要用「成组链接法」的原因。
Q4. 多核情况下两个进程同时 balloc,会不会都拿到同一块?高危
答:不会。bread 拿到的 buf 是带睡眠锁的(bread 内部 acquiresleep),同一时刻只有一个 CPU 能持有这个位图块的 buf,其他 CPU 必须等。所以 75–77 这三行天然是临界区,不会出现「读到 0、对面也读到 0、都置 1、都返回同一个 bi」。这是位示图法在 xv6 里能正确工作的关键。
Q5. 如果这块位图字节是 0xFF,代码会怎么走?送分
答:0xFF = 全 1,data[bi/8] & m 永远 ≠ 0,if 不进。bi 自增到 8 的倍数后,外层 for 让 bp = bread(下一个位图块),继续找。如果整张盘都满,最后 printf("balloc: out of blocks") 返回 0。
