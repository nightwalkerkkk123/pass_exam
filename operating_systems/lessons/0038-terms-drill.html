Ch1 · 操作系统引论

操作系统Operating System, OS
基础

白话：夹在程序和硬件之间、主动管 4 样硬件资源的店长（系统软件）。

为什么重要：

整个学科的对象。考试第一题就问「什么是 OS」。
4 大硬件资源Hardware Resources
基础

白话：CPU（计算）、内存（临时工作台）、磁盘（永久仓库）、I/O 设备（键盘/鼠标/网卡）—— OS 要主动管的对象。

为什么重要：

整本书的 4 大模块就是这 4 个的对应管理（进程/存储/文件/IO）。
4 大抽象Abstractions
基础

白话：OS 给程序员的 4 个接口：进程（CPU 抽象）、虚拟地址空间（内存抽象）、文件（磁盘抽象）、read/write/open/close（IO 抽象）。

为什么重要：

填空/简答「OS 给用户提供什么接口」= 答这 4 个。
硬件 ≠ 抽象：硬件是 OS「管」的对象；抽象是 OS「给」程序员的接口。
并发Concurrency
Ch1

白话：宏观同时、微观交替。单核 CPU 快速切换进程让多任务看起来同时跑。

为什么重要：

操作系统四大基本特征之首。
vs 并行：并行是多核真正同时；单核 OS 只有并发，没有并行。
并行Parallelism
Ch1

白话：多个 CPU/多核真的同时执行不同任务。

为什么重要：

硬件能力，不是 OS 特征。
很多人把「并发」「并行」写反。判断：核心数 > 1 才可能并行。
共享Sharing
Ch1

白话：多个进程共用一套资源（CPU、内存、打印机）。

为什么重要：

OS 四大特征之一，需要互斥/同步机制保护。
vs 并发：并发讲「同时推进」，共享讲「同时占用」。
虚拟Virtual
Ch1

白话：1 台电脑看起来像 N 台（虚拟内存、虚拟机）。

为什么重要：

OS 四大特征之一。虚拟存储是 Ch5 整章主题。
异步Asynchronism
Ch1

白话：程序不知道下一步何时发生（走走停停）。

为什么重要：

OS 四大特征之一。进程走走停停但结果应该一样。
系统调用System Call
Ch1 / Ch7

白话：用户程序请求 OS 服务的唯一合法通道（open/read/write/fork/exec）。

为什么重要：

用户态 ↔ 核心态切换的入口。考试必考。
vs 库函数：库函数（如 printf）内部会触发系统调用，但本身不是。
用户态 / 核心态User Mode / Kernel Mode
Ch1 / Ch7

白话：用户态 = 普通程序，受限不能碰硬件；核心态 = OS 内核，可访问一切。

为什么重要：

系统调用会触发「用户态 → 核心态 → 用户态」切换。
中断Interrupt
Ch1 / IO

白话：设备「敲 CPU 脑袋」说「我干完了」。

为什么重要：

IO 控制方式里「中断驱动」是高频考点。
vs 系统调用：中断是异步（设备主动），系统调用是同步（程序主动）。
特权指令Privileged Instruction
Ch7

白话：只能在核心态执行的指令（修改 PSW、启动 IO）。

为什么重要：

解释「为什么必须有用户态/核心态区分」的标准答案。

Ch2 · 进程与线程

进程Process
核心

白话：程序的一次执行（菜谱 vs 正在做的菜）。

为什么重要：

资源分配的基本单位（必背）。
vs 程序：程序是静态代码；进程是动态执行。开 2 个 QQ = 1 程序 2 进程。
线程Thread
核心

白话：进程里的一条执行流。1 进程可有多线程（浏览器 1 个 UI + 1 个下载 + ...）。

为什么重要：

CPU 调度的基本单位（必背金句）。
vs 进程：线程共享进程的地址空间，所以切换开销小、通信方便，但一个挂可能全挂。
PCBProcess Control Block
核心

白话：进程的学生档案（PID/状态/PC/寄存器/内存/IO）。

为什么重要：

「进程存在的唯一标志」必背句。
进程三状态Ready / Running / Blocked
核心

白话：就绪（差 CPU）、运行（在 CPU 上）、阻塞（等 IO 或信号量）。

为什么重要：

状态转换图必画必考。
挂起Suspend
Ch2

白话：进程被 swap 到磁盘，内存里没有。带「就绪挂起」「阻塞挂起」两个子状态。

为什么重要：

七状态模型 = 三状态 + 挂起。中级调度就是管挂起的。
vs 阻塞：阻塞还在内存里（保留 PCB），挂起连 PCB 都被踢到磁盘。
临界资源Critical Resource
Ch2

白话：一段时间只许一个进程用的资源（打印机、共享变量）。

为什么重要：

PV 互斥问题的源头。
vs 临界区：临界资源是「东西」；临界区是「访问它的代码段」。
临界区Critical Section
Ch2

白话：访问临界资源的那段代码。

为什么重要：

PV 写法的核心：「把临界区夹在 P(mutex) 和 V(mutex) 之间」。
互斥Mutual Exclusion
Ch2

白话：「不能同时做」—— 一段时间只一个进程进临界区。

为什么重要：

PV 互斥信号量初值 = 1。
vs 同步：互斥是「不能同时」，同步是「必须先后」。
同步Synchronization
Ch2

白话：「必须按顺序做」—— 进程 B 等 A 完成才跑。

为什么重要：

同步信号量初值 = 0，由前置进程 V 触发。
信号量Semaphore
核心

白话：一个整数计数器，表示可用资源数。P 减、V 加。

为什么重要：

进程同步的核心工具，PV 是必会必写的操作。
P 操作wait / P / down
核心

白话：S=S-1；若 S<0 则进程阻塞排队。申请资源。

为什么重要：

口诀「P 减 V 加，P 进 V 出」必背。
V 操作signal / V / up
核心

白话：S=S+1；若 S≤0 则唤醒一个等待者。释放资源。

为什么重要：

和 P 成对出现。
P/V 不是单纯加减：关键在「<0 排队睡、唤醒等的人」。
互斥信号量 mutexMutex
核心

白话：初值 = 1 的信号量（一把钥匙），用于互斥访问临界区。

为什么重要：

P/V 必须成对、夹住临界区。
生产者-消费者Producer-Consumer
核心

白话：三信号量模板：mutex=1, empty=n, full=0。

为什么重要：

PV 经典题之首。口诀：「P 顺序要命（资源在前 mutex 在后）、V 顺序随意」。
先 P(mutex) 再 P(empty) = 死锁（攥钥匙等空位）。
读者-写者Readers-Writers
Ch2

白话：多个读者可同时读，但写者必须独占。读者优先/写者优先/公平 3 种变体。

为什么重要：

PV 第二经典题。
哲学家就餐Dining Philosophers
Ch2

白话：5 个哲学家 5 根筷子，每人要两把才吃。演示死锁与解决（限制 4 人、奇偶拿法、资源分级）。

为什么重要：

死锁章节经典案例。

Ch3 · 处理机调度与死锁

处理机调度CPU Scheduling
Ch3

白话：就绪队列里很多进程，决定「让谁先用 CPU」。

为什么重要：

Ch3 整章主题。
三级调度3-Level Scheduling
Ch3

白话：高级（作业调度）→ 中级（内存调度，挂起/激活）→ 低级（进程调度，分配 CPU）。

为什么重要：

简答题「调度分哪几层」的标准答案。
常考的是低级调度（= 进程调度）。
周转时间Turnaround Time, T
核心公式

白话：T = 完成时间 − 到达时间。进程从进系统到离开总耗时。

为什么重要：

调度题 80% 都要算这个。
带权周转时间Weighted Turnaround, W
核心公式

白话：W = T / 服务时间。>1 越亏，越接近 1 越爽。

为什么重要：

SJF 的目标就是让 W 最小（平均 W 最小）。
W 不是「加权」，是「带权」= 除以服务时间。
响应时间Response Time
Ch3

白话：首次得到 CPU 的时刻 − 到达时间（不是完成！）。

为什么重要：

分时系统的关键指标。
vs 周转时间：响应只看「首次」，周转看「整个完成」。
FCFSFirst-Come First-Served
先来先服务

白话：按到达顺序排。简单公平，但短作业会等长作业。

为什么重要：

调度算法 baseline。
非抢占。一旦开始就要跑完或阻塞。
SJF / SJF（非抢占）Shortest Job First
短作业优先

白话：谁的服务时间短，谁先跑。

为什么重要：

金句「SJF 是使系统平均周转时间最短的算法」。
vs SRTN：SJF 非抢占（一旦跑就不打断）；SRTN 抢占（来了更短的就把当前踢下去）。
SRTNShortest Remaining Time Next
最短剩余时间优先

白话：SJF 的抢占版——有新进程剩余时间更短就踢掉当前。

为什么重要：

抢占式调度常考。
HRRNHighest Response Ratio Next
高响应比优先

白话：响应比 = 1 + 等待/服务。最高者优先，兼顾长短。

为什么重要：

不会饥饿（SJF 的长作业饥饿的解药）。
响应比 = (等待 + 服务) / 服务 = 1 + 等待/服务。
RRRound-Robin
时间片轮转

白话：每个进程跑一个时间片 q，时间到就送队尾。

为什么重要：

分时系统标配。响应快但平均周转不是最短。
q 太大 → 退化成 FCFS（不是 SJF！）；q 太小 → 切换开销爆炸。
时间片 qTime Quantum
Ch3

白话：RR 里每个进程最多连续跑 q 分钟。

为什么重要：

q 的取值是选择题陷阱（退化方向、切换开销）。
抢占式调度Preemptive
Ch3

白话：OS 能强行打断当前进程，把 CPU 让给别人。

为什么重要：

RR/SRTN 是抢占；FCFS/SJF/HRRN 是非抢占。
vs 非抢占：非抢占意味着一旦开始就要跑完或阻塞。
优先级调度Priority Scheduling
Ch3

白话：谁优先级高谁先跑。

为什么重要：

实时系统常用。问题：低优先级饥饿。
解法 = Aging：等越久优先级越高。
多级反馈队列 MFQMultilevel Feedback Queue
Ch3

白话：多个优先级队列，时间片大小不同；进程用完时间片就降级。

为什么重要：

通用调度算法首选（兼顾长短作业、不用预知服务时间）。
饥饿Starvation
Ch3

白话：某个进程长期得不到服务。

为什么重要：

SJF（长作业）、SSTF（远处磁道）、优先级（低优先级）都会饥饿。
vs 死锁：饥饿是「永远等不到但还在等」；死锁是「互相等、永远不动」。
死锁Deadlock
核心

白话：进程互相等对方手里的资源，谁都不动。

为什么重要：

Ch3 后半整章 + 银行家算法。
vs 饥饿：死锁是「环形等待」，饥饿是「单方面等不到」。
死锁 4 必要条件4 Necessary Conditions
必背

白话：互斥 + 请求和保持 + 不可剥夺 + 循环等待。四个同时满足才死锁。

为什么重要：

死锁预防就是破坏其中之一。
银行家算法Banker's Algorithm
核心

白话：分配资源前假装分配，检查是否存在安全序列。有则分，无则拒。

为什么重要：

死锁避免的标志算法，15-20 分计算大题。
安全序列 / 安全状态Safe Sequence / State
核心

白话：存在一个进程完成顺序让所有进程都能完成 = 安全；否则 = 不安全（不一定死锁但有风险）。

为什么重要：

银行家算法每一步都要找安全序列。
Available / Max / Allocation / NeedBanker Data Structures
核心

白话：银行家四件套：Available=当前可用，Max=最大需求，Allocation=已分配，Need=还需要的（=Max−Allocation）。

为什么重要：

做题先把这 4 个矩阵抄出来。

Ch4 · 存储器管理

逻辑地址 vs 物理地址Logical / Physical Address
Ch4 基础

白话：逻辑地址 = 程序「以为自己」用的地址（从 0 开始）；物理地址 = 内存里真实的位置。

为什么重要：

地址转换是分页/分段的前提。
地址重定位Address Relocation
Ch4

白话：把逻辑地址「翻译」成物理地址。

为什么重要：

静态重定位（装入时一次定）/ 动态重定位（运行时换）。
连续分配Contiguous Allocation
Ch4

白话：每个进程占一段连续的内存空间。

为什么重要：

产生外部碎片。
内部碎片 vs 外部碎片Internal / External Fragmentation
Ch4

白话：内部碎片 = 分给你的但没用完；外部碎片 = 没分出去但太零碎用不了。

为什么重要：

分页有内部碎片、连续分配有外部碎片。考试必背对照。
首次适应 FFFirst Fit
Ch4

白话：从低地址找第一个够大的空闲块分配。

为什么重要：

分配算法基线。优点：低地址优先，保留大块。
最佳适应 BFBest Fit
Ch4

白话：找最小但够用的空闲块。保留大块，但碎片更小更零碎。

为什么重要：

名字陷阱——「最佳」其实最差（产生大量小碎片）。
最坏适应 WFWorst Fit
Ch4

白话：挑最大的空闲块切出去用。碎片较大但易复用。

为什么重要：

和 BF 对照。
分页Paging
核心

白话：把程序和内存都切成同样大小的块。程序叫页，内存叫页框（块）。

为什么重要：

现代 OS 的内存管理基础。地址结构 = 页号 + 页内偏移。
页框 / 物理块Frame
Ch4

白话：物理内存里和页同样大小的一块。

为什么重要：

页 → 页框 的映射是分页的核心。
页表Page Table
核心

白话：记录「页号 → 页框号」对应关系的表。

为什么重要：

查页表 = 地址转换。
页内偏移 WOffset within Page
核心

白话：页里的位置（低几位地址直接就是偏移）。

为什么重要：

逻辑地址 = 页号 × 页大小 + 偏移 = 页号 + 偏移。
快表 TLBTranslation Lookaside Buffer
Ch4

白话：页表的 cache，存最近用过的页号→页框号。

为什么重要：

有效访问时间 EAT 的算式里有它。
快表命中时 EAT ≈ 访存 + TLB 访问（1 次访存 + 1 次查 TLB）。
分段Segmentation
Ch4

白话：按程序逻辑结构（代码段、数据段、栈段）划分，每段长度不等。

为什么重要：

和分页对比的常考对比表。
vs 分页：分页固定大小（一维）、分段按逻辑（二维）；分页内部碎片、分段外部碎片。
段页式Segment-Paging
Ch4

白话：先分段、再在段内分页。既有段的逻辑性、又有页的灵活性。

为什么重要：

理论上最完整。

Ch5 · 虚拟存储器

虚拟存储器Virtual Memory
核心

白话：程序感觉内存很大，实际只有部分在内存、其余在磁盘。

为什么重要：

Ch5 整章主题。三个机制：部分装入 + 请求调入 + 页面置换。
局部性原理Locality
Ch5

白话：程序倾向于用「刚才用过的」和「附近的」数据。

为什么重要：

虚拟存储能 work 的根本原因。
时间局部性 vs 空间局部性Temporal / Spatial
Ch5

白话：时间 = 刚访问的很快再用（循环变量 i）；空间 = 附近地址很可能也用（数组）。

为什么重要：

简答题必背。
缺页中断 / 缺页Page Fault
核心

白话：要访问的页不在内存，要从磁盘调入（慢，一次 IO）。

为什么重要：

缺页率 = 缺页次数 / 总访问次数。所有置换算法都在优化这个数。
页面置换Page Replacement
Ch5

白话：内存满了、又要调新页，淘汰一个旧页。

为什么重要：

虚拟存储的关键环节。
OPTOptimal (Belady)
最佳置换

白话：淘汰「将来最久用不到」的页。理论最优。

为什么重要：

所有算法的基准（天花板），但需预知未来 → 不可实现。
FIFOFirst-In First-Out
先进先出

白话：淘汰「最早进入内存」的页（看出生）。

为什么重要：

最简单。唯一会犯 Belady 异常的常用算法。
vs LRU：FIFO 看「进入时刻」，LRU 看「上次使用时刻」。
LRULeast Recently Used
最近最久未用

白话：淘汰「最久没访问」的页（看露面）。

为什么重要：

效果好，无 Belady 异常。堆栈类算法。
vs FIFO：判据是「上次露面」，不是「出生顺序」。
Clock 算法Clock / NRU
Ch5

白话：LRU 的近似——每页有访问位（0/1），扫到 0 就淘汰，扫到 1 就清 0 继续找。

为什么重要：

实现简单、效果接近 LRU，OS 实际在用。
Belady 异常Belady's Anomaly
Ch5

白话：物理块增多、缺页率反而升高。

为什么重要：

只有 FIFO 会犯，LRU/OPT 是堆栈类不会犯。
抖动Thrashing
Ch5

白话：缺页太频繁，大把时间花在换页、真正执行时间很少。

为什么重要：

CPU 利用率下降、磁盘 IO 爆满。
原因：物理块太少 / 进程太多 / 工作集装不下。
工作集Working Set
Ch5

白话：进程在某段时间窗口内实际访问的页面集合。

为什么重要：

分配的物理块 ≥ 工作集大小 → 不抖动。

Ch6 · 设备与文件管理

DMADirect Memory Access
Ch6

白话：让 DMA 控制器代替 CPU 在设备和内存间搬数据。

为什么重要：

CPU 不再一字节一字节搬，释放 CPU 给别的活。
vs 中断驱动：中断驱动每个字节都要 CPU 介入；DMA 整块数据 CPU 只介入 1 次。
通道I/O Channel
Ch6

白话：专门负责 IO 的「小 CPU」，能执行 IO 指令。DMA 的升级版。

为什么重要：

大型机/主机用。
缓冲区 / 缓冲Buffer
Ch6

白话：内存里临时存放 IO 数据的区域。

为什么重要：

缓和 CPU（快）和 IO 设备（慢）速度差。
SPOOLingSimultaneous Peripheral Operations On-Line
Ch6

白话：假脱机。把独占设备（打印机）改造成逻辑共享（先存磁盘输出井、由 spool 进程一个个打）。

为什么重要：

把「独占」伪装成「共享」。
文件控制块 FCBFile Control Block
Ch6

白话：文件的档案（名字/大小/位置/权限/时间）。

为什么重要：

目录项的实质。
inodeIndex Node
Ch6

白话：UNIX/Linux 里的文件元数据（属性 + 数据块位置）。

为什么重要：

文件名不在 inode 里（目录项才存名字 → inode 号）。
vs 目录项：目录项是「名字 → inode 号」的映射；inode 是「inode 号 → 文件本身」的映射。
连续分配Contiguous Allocation (File)
Ch6

白话：文件占连续磁盘块（顺序访问快、外部碎片、扩展难）。

为什么重要：

三种文件分配方式之一。
链接分配Linked Allocation
Ch6

白话：文件块用指针串起来（无外部碎片、扩展容易，但只能顺序访问）。

为什么重要：

三种之一。FAT 是它的改进版（指针集中到一张表）。
索引分配Indexed Allocation
Ch6

白话：建索引块记录所有磁盘块号（支持随机访问、扩展易，但小文件浪费索引块）。

为什么重要：

UNIX inode 用这个。
混合索引Multi-level Indexed
Ch6

白话：inode 里前 N 个是直接地址，再后面挂一级、二级、三级间接索引。

为什么重要：

UNIX 用，小文件省索引块、大文件能扩展。
最大文件 = D×B + N×B + N²×B + N³×B（N=B/A）。
空闲链表法Free Linked List
Ch6

白话：把所有空闲块用指针串成一条链。

为什么重要：

简单但分配/回收都要遍历整条链。
位示图 / 位图Bitmap
Ch6

白话：用一串 0/1 表示磁盘块空闲与否（1 位 = 1 块）。

为什么重要：

位运算找空闲块快、紧凑。
计算公式：块号 b → 字号 j = b/n，字内位 k = b%n（n 是字长）。
成组链接法Group Linking
Ch6

白话：空闲块每 N 个一组，组间链接；S.free 栈只存当前一组，其余藏在每组最后一块的肚子里。

为什么重要：

UNIX/Linux 实际用法。两条边界规则必背。
寻道 / 寻道时间Seek Time
Ch6

白话：磁头从当前磁道移到目标磁道的时间。

为什么重要：

磁盘访问 3 部分时间中最大头。
磁盘访问时间 = 寻道 + 旋转延迟 + 传输。
旋转延迟Rotational Latency
Ch6

白话：磁头到了磁道后，等盘片转到正确扇区的时间。平均 = 半圈。

为什么重要：

7200 RPM → 平均旋转延迟 ≈ 4.17 ms。
FCFS（磁盘）FCFS Disk
Ch6

白话：按请求到达顺序处理磁盘请求。

为什么重要：

磁盘调度基线。公平但磁头来回折腾。
SSTFShortest Seek Time First
Ch6

白话：每次挑离磁头当前位置最近的请求。

为什么重要：

寻道短，但两端请求会饥饿。
不是「挑数字最小的」，是「离当前最近的」。
SCAN（电梯算法）SCAN / Elevator
Ch6

白话：磁头沿一个方向走，处理沿途请求，走到最后一个请求就掉头（汤小丹口径：不走到磁盘边缘）。

为什么重要：

避免饥饿，常见题。
vs CSCAN：CSCAN 单向扫、到头直接跳回另一端（跳回那段距离要算）。
CSCANCircular SCAN
Ch6

白话：单向扫描，到一端后直接跳回另一端从头扫。等待时间均匀。

为什么重要：

跳回的距离必须算进总寻道（教材例题常漏）。
磁臂粘着 / 磁头粘着Arm Stickiness
Ch6

白话：磁头粘在一个磁道附近反复服务，新请求来了但磁头不挪。

为什么重要：

N-SCAN / FSCAN 解法。

Ch7 · 系统接口与保护

陷入 / trapTrap
Ch7

白话：用户程序主动触发的「我需要 OS 服务」信号，进入核心态。

为什么重要：

系统调用的机制核心。
vs 中断：中断是设备异步触发；trap 是程序同步触发。
系统调用号Syscall Number
Ch7

白话：每个系统调用有个编号（如 read=3, write=4），OS 用它分发到对应服务。

为什么重要：

系统调用过程「传号 → 跳表 → 执行」的中段。
访问控制矩阵Access Control Matrix
Ch7

白话：行 = 用户，列 = 资源，每格 = 该用户对该资源的权限。

为什么重要：

理论上完整，但稀疏矩阵浪费空间。
访问控制表 ACLAccess Control List
Ch7

白话：每个资源附带一张表，列出谁能干啥（按列存）。

为什么重要：

Windows NTFS 用。
vs 能力表：能力表是按行存（每用户一张表列他能干啥）。
能力表 CapabilityCapability List
Ch7

白话：每个用户持一张表，列出他能访问哪些资源 + 权限。

为什么重要：

UNIX 风格。
保护域Protection Domain
Ch7

白话：一组「能访问的资源 + 权限」的集合（用户态、核心态是两个不同域）。

为什么重要：

解释用户态/核心态的另一种视角。

📝 排障自测（6 题）

用下面 6 题检验你扫盲后的真实掌握度。能立刻答出来的 → 绿；想 10 秒以上 → 橙；想不起来 → 红，回去翻对应章节。

Q1（口述）「周转时间 T」「带权周转时间 W」「响应时间」三个公式分别是什么？W 越大说明什么？
▸ 答案
T = 完成时间 − 到达时间；W = T ÷ 服务时间；响应时间 = 首次得到 CPU 时刻 − 到达时间。
W 越大说明进程「等得久 / 跑得短」，越亏。SJF 就是要让平均 W 最小。
Q2（选择）FIFO 和 LRU 都淘汰「最老的页」，对吗？
A. 对，都是最老的   B. 错，FIFO 看「进入内存最早」，LRU 看「最久没访问」
▸ 答案
B。金句：「FIFO 看出生，LRU 看露面」。LRU 淘汰的是「最久没被访问」，命中会续命（重置时间）；FIFO 不管你用没用，进入最早就踢。
Q3（口述）死锁的 4 个必要条件是什么？举一个会破坏「循环等待」的具体做法。
▸ 答案
互斥 + 请求和保持 + 不可剥夺 + 循环等待。
破坏循环等待：给资源排编号，所有进程只能按编号从小到大申请（资源有序分配法）—— 这样不可能形成环。
Q4（计算）RR 算法 q=4 跑 A=3/B=6/C=4/D=2/E=5，D 的完成时间是多少？
▸ 答案
q=4 → A 0-3 跑完（剩 0，时刻=3）→ B 3-7 跑 4（剩 2，送队尾）→ C 7-9 跑 2（剩 0，时刻=9）→ D 9-11 跑 2（剩 0，时刻=11）。
D 完成时间 = 11。q 变大 → 每人能跑更久 → D 不用排那么多次。
Q5（口述）分页有内部碎片还是外部碎片？分段呢？连续分配呢？
▸ 答案
分页 → 内部碎片（最后一块常装不满）；分段 → 外部碎片（段长不等）；连续分配 → 外部碎片。
Q6（口述）DMA 和「中断驱动 IO」的本质区别是什么？
▸ 答案
中断驱动每个字节都要 CPU 介入（设备每完成一字节就中断一次）；
DMA让 DMA 控制器整块搬数据，CPU 只在开始和结束时介入两次 → 解放 CPU。适合磁盘/网卡等高速设备。
