四大特征 · 并发 vs 并行(高频辨析)

操作系统的四大基本特征里,并发是地基。这一课先让你看见并发和并行到底差在哪,再上术语。承接 第 14 课「什么是操作系统」。

一句话先记住: OS 有四大基本特征 —— 并发、共享、虚拟、异步。其中并发是最重要的,其余三个都建立在并发之上。而“并发”最容易和“并行”搞混:并发是“看起来同时”(靠飞快切换),并行是“真的同时”(靠多个 CPU)。

2. 🧸 具体例子:一个厨师 vs 三个厨师

🧸 5 岁也懂的比喻: 你用一台单核电脑,一边听歌、一边下载、一边打字,感觉“三件事同时进行”。 其实 CPU 在毫秒级飞快地轮流做:切给听歌一小会 → 切给下载一小会 → 切给打字一小会…… 切得太快,你眼睛根本察觉不到停顿,以为它们“同时”。这就是并发。

换个画面:一个厨师轮流炒 3 道菜(炒两下这锅、再炒两下那锅)= 并发; 3 个厨师各站一个灶、各炒一道菜 = 并行。菜的道数一样,差别在有几个“干活的人(CPU)”。

4. 📖 是什么:精确定义(对齐教材口径)

并发: 两个或多个事件在同一时间间隔内发生 —— 宏观上同时、微观上交替。为实现并发,OS 引入了进程(独立运行、独立分配资源的基本单位),后来又引入线程来进一步提高并发度。
并行: 两个或多个事件在同一时刻同时发生,需要多个 CPU 支持。
共享: 系统资源可被多个并发进程共同使用。分两种:互斥共享——一段时间内只允许一个进程访问,这种资源称临界资源(如打印机);同时访问——允许多个进程同时访问(如磁盘上的只读文件、可重入代码)。并发与共享互为存在条件。
虚拟: 通过某种技术把一个物理实体变为若干个逻辑上的对应物。两类手段:时分复用(如虚拟处理机)与空分复用(如虚拟存储器)。
异步: 在多道程序环境下,进程以不可预知的速度向前推进(走走停停)。但只要运行环境相同,多次运行的结果应完全相同 —— 异步性本身不破坏正确性,靠同步机制来保证。

5. ❓ 为什么:一层层深挖

为什么说并发是“最重要”的特征?

因为没有并发,就谈不上多个程序共用资源。共享是为了让并发的进程能同时用系统资源;虚拟是为了让有限的物理资源“变多”去支撑更多并发进程;异步则是并发执行必然带来的副产物。抽掉并发,其余三个都失去了存在的理由 —— 所以并发是地基。

再深一层:为什么“并发和共享互为存在条件”?

两个方向都成立:① 如果没有并发(系统里只有一个程序在跑),那根本没有“多个进程抢同一份资源”的场景,共享就无从谈起。② 如果资源不能被共享(一份资源死死锁给一个进程,别人碰不得),那多个程序也无法真正地并发推进。所以两者缺一不可、互为前提。

再深一层:异步从哪来?为什么它不破坏正确性?

异步的根源:进程会因为抢不到 CPU、或要等待 I/O 完成而被迫暂停,过一会儿再继续 —— 所以它前进的速度不可预知,呈现“走走停停”。为什么仍然正确:只要用同步 / 互斥机制保证关键操作的先后次序(该谁先做的就先做),那么相同的输入就一定得到相同的结果。异步说的是“过程的快慢不确定”,不是“结果不确定”。

🧸 一句话收束: 并发是“为什么要有 OS 这么复杂”的答案 —— 正因为要让很多程序看起来同时跑在可能只有一个的 CPU 上,才需要进程、调度、共享、同步这一整套机制。

6. ✅ 自测:先想再翻答案

Q1(最高频)一台单 CPU(单核)计算机,能实现“并行”吗?

点我看答案 正确:B。并行的定义是“同一时刻多个事件真的同时发生”,物理上需要多个 CPU/核同时干活。单核任一时刻只有一个任务在跑,只能靠时间片切换制造“同时”的错觉 —— 那是并发,不是并行。

Q2 “宏观上同时发生、微观上交替执行”描述的是?

点我看答案 正确:C。“宏观同时、微观交替”正是并发的标准描述。并行是“宏观、微观都同时”。

Q3 打印机属于哪种共享?

点我看答案 正确:B。打印机若被两个进程同时写,输出就会串行错乱,所以它是互斥共享的资源,这类资源称为临界资源。像磁盘上的只读文件才属于“同时访问”。

Q4 四大特征中,哪个是其它特征的基础 / 最重要?

点我看答案 正确:B。并发是地基:共享是为支撑并发而生,虚拟是为放大资源以支撑更多并发,异步是并发执行的必然结果。
