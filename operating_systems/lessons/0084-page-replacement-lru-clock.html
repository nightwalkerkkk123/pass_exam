📚 1. LRU = 最近最久未用（Least Recently Used）

🧸 ELI5 · 衣架比喻：你家有 3 个衣架，挂了 [红 T 恤, 蓝衬衫, 绿外套]。今天你又穿了「黄 T 恤」，但没位置了 —— 你会淘汰哪件？LRU 会说：「淘汰你最久没穿的那件」—— 也就是衣架最里面、压得最深的那件。
📌 形式定义：缺页时，淘汰最久没被访问过的那个物理块。访问一次就把该页挪到「最近」位置。

三个关键事实

事实含义为什么这样
「最近」= 栈顶最近访问的页压在栈顶淘汰时找栈底（最久没用）
命中也要挪命中也要把该页挪到栈顶不挪就记不住「上次何时用过」
缺页要淘汰缺页时弹栈底页，把新页压栈顶这就是 LRU 的核心动作
⚠️ 维护成本：每次访问（不论命中还是缺页）都要动数据结构 —— 经典做法有「栈」或「计数器+双向链表」。这就是为啥说「LRU 性能好但开销较大」。
LRU ≈ OPT（近似最优） 因为「过去一段时间没用到的页，未来一段时间大概率还用不到」——局部性原理。

⏰ 3. CLOCK = 时钟置换（Clock / Second Chance）

🧸 ELI5 · 时钟指针比喻：把物理块围成一个圆盘，上面有一根指针不停转。每块贴一个访问位（用过 = 1，没用 = 0）。缺页时指针开始扫：
  • 扫到 访问位 = 1：给它第二次机会（把 1 改成 0），指针继续往下走
  • 扫到 访问位 = 0：就踢它！把新页放这里，指针指到下一格
📌 形式定义：循环扫描访问位，淘汰第一个访问位=0的页。所有访问位为 1 的页都被「降级」为 0 后放过，相当于「再给你一次机会」。

为什么 CLOCK ≈ LRU

信号LRU 怎么记录CLOCK 怎么记录
「最近用过」在栈顶 / 时间戳最大访问位 = 1
「很久没用」在栈底 / 时间戳最小访问位 = 0（且还没被降级）
访问时维护挪到栈顶（要改链表）只置 1（不改位置）
CLOCK ≈ LRU，但每次访问只动 1 个 bit（硬件置位）， LRU 要动一个链表节点 —— 所以 CLOCK 开销小。
⚠️ 关键细节：
  • 命中时只把访问位置 1，指针不动
  • 缺页时才启动扫描，扫描过程中可能绕一整圈（把所有 1 都降为 0）
  • 指针在缺页置换完成后指向被淘汰块的下一格

⚠️ 易错提醒：
  • 命中不动指针：CLOCK 命中时只置访问位=1，指针原地不动（很多人误以为命中也要挪指针）
  • 指针绕过一圈：CLOCK 一轮扫描会把所有 bit=1 都降为 0，再回到起点淘汰那个 bit=0 的
  • 置换后指针前进 1 格：淘汰发生在扫描起点，新页装这里后 ptr→下一格（不是从头）
  • CLOCK 不会 Belady：因为它本质是「带二次机会的 FIFO」，扫描顺序稳定；但改用改进型 CLOCK时会考虑修改位，可能差异

🎯 6. 改进型 CLOCK：访问位 + 修改位（final-test 重点）

🧸 ELI5 · 双重判断：基础 CLOCK 只看「用过没」（访问位）。但如果一个页又被用过又被改过，淘汰它就得写回磁盘，开销大。改进型 CLOCK 再加一个修改位（脏位），优先淘汰干净且没用的页 —— 不用写回磁盘。

四类页面（按淘汰优先级）

① (访问=0, 修改=1)
没用过但脏了
② (访问=1, 修改=0)
用过但干净
③ (访问=1, 修改=0)
用过且干净
④ (访问=0, 修改=0)
没用过且干净 ← 最该淘汰
淘汰优先级（高 → 低）： ④(0,0)没用且干净 → ①(0,1)没用但脏 → ②(1,0)用过且干净 → ③(1,1)用过且脏
⚠️ 关键口诀（背这一句）：「干净没用 → 最该淘汰」。访问位=0 表示「没用过」（给机会也不需要），修改位=0 表示「干净」（淘汰不用写回磁盘）。两个都是 0 成本最低，最该踢。
📝 对应 final-test 第 18 题：「改进型 CLOCK 优先级最高」 = (访问=0, 修改=0)。该题答案即此。

✅ 7. 自测 4 题

自测 1. LRU 命中页时，是否需要把该页挪到「最近使用」位置？为什么？
答案
必须挪。不挪就不知道「最近何时用过」。如果命中不挪，老页会一直停在原位置，下次淘汰可能误伤它。LRU 的核心就是用「位置」记录「上次访问的顺序」，命中不动等于不记录。
自测 2. CLOCK 命中页时，指针会动吗？为什么？
答案
不动。命中时只把该块的访问位置 1，指针原地不动。CLOCK 维护的是「最近被访问过」这个事实（用 1 个 bit），不是「访问的顺序」，所以命中不需要挪位置。只有缺页时指针才启动扫描。
自测 3. 改进型 CLOCK 算法中，淘汰优先级最高（即最先被淘汰）的是哪一类页？为什么？
答案
(访问位=0, 修改位=0)，即「没用过且干净」的页。
原因：访问位=0 表示该页最近没用过，淘汰它对后续访问影响最小；修改位=0 表示该页没被修改（与磁盘一致），淘汰它不需要写回磁盘，成本最低。两个 0 叠加就是「最便宜的选择」。
自测 4. 序列 3, 4, 5, 6, 3, 4, 7, 3, 4, 5, 6, 7，3 块物理块。LRU 和 CLOCK 各多少缺页？
答案
LRU 推演：
3,4,5 → 3 miss；6 淘汰 3 → 4 miss；3 淘汰 4 → 5 miss；4 淘汰 5 → 6 miss；7 淘汰 6 → 7 miss；3 命中；4 命中；5 淘汰 3 → 8 miss；6 淘汰 4 → 9 miss；7 命中
LRU 缺页 9 次（缺页率 9/12 = 75%）

CLOCK 在这个序列上与 LRU 几乎相同：CLOCK 缺页 9-10 次（取决于降级时机，本题也按 9-10 次答都对）。

共同规律：这种「循环访问 + 偶尔穿插新页」的序列是 LRU/CLOCK 的舒适区，比 FIFO 好得多（FIFO 这种序列会 Belady）。
