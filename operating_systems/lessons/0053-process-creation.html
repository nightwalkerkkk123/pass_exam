进程的创建:4 个事件触发、4 步完成

这是简答题 5 分大题的标准答案骨架——默写 4 个事件 + 4 个步骤 = 满分。教材出处:2_进程管理/ppt文本.md 幻灯片 136-143(2.2.1 节)。背景知识见 0042 进程状态。

0·全局线 1·一句话 2·4 个事件 3·4 步流程 4·伪代码 5·易错 6·真题 7·自测
考场必背两句话:
① 引起进程创建的事件 4 类:用户登录、作业调度、提供服务、应用请求。
② 创建进程的过程 4 步:申请空白 PCB → 分配资源 → 初始化 PCB → 插入就绪队列。

🗺 0. 这道简答题在考什么

🧸 一句话: 进程不是凭空出现的——必须有"理由"(事件) + "动作"(创建原语)。简答题考的就是把这两条说清楚。

5 分题的标准答法:事件列 4 条 + 步骤列 4 条 + 一句话总结,每条 1 分左右,剩下 1 分给表达清晰度。

📖 1. 是什么:进程的创建 = 父进程调用创建原语生成子进程

定义: 进程创建指操作系统根据某种触发事件(用户登录、作业调度等),为新任务建立进程控制块 PCB、分配资源、初始化并插入就绪队列的完整过程。

教材原话(幻灯片 141):"3. 进程的创建 (Creation of Progress)"——创建是 OS 提供给用户/上层进程的基本能力。

在 Unix/Linux 里对应 fork() + exec():父进程 fork 出子进程(拷贝 PCB),子进程 exec 加载新程序;在 Windows 里对应 CreateProcess()。

⚡ 2. 触发事件:4 类

事件类型具体场景例子
① 用户登录分时系统中,用户在终端登录成功SSH 登录一台 Linux 服务器
② 作业调度批处理系统中,作业调度程序按算法选中一个新作业凌晨 1 点 cron 跑备份脚本
③ 提供服务运行中的用户程序提出服务请求,系统专门创建新进程提供服务打印服务:用户进程请求打印时,系统创建打印进程
④ 应用请求应用程序自己请求创建新进程完成并行任务浏览器每开一个 tab 就是一个新进程
记忆口诀:"登调请请"(登录/调度/请求服务/请求应用)。或者想画面:登录=开门、调度=叫号、服务=客人要酒、应用=自己开店。

🪜 3. 创建过程:4 步流程

1申请空白 PCB
从 PCB 集合中找一块空闲的 PCB。任何进程都要先有自己的"户口本"。
2分配资源
为新进程分配内存、打开文件、初始化数据段。分配不成功则释放 PCB回退。
3初始化 PCB
填入 PID、优先级、父进程、状态("ready")、程序计数器等关键字段。
4插入就绪队列
把新 PCB 挂到 RQ 队尾,等待被调度器选中,获得 CPU。
📐 考场默写模板: 
进程的创建过程分为 4 步:
(1) 申请空白 PCB
(2) 为新进程分配资源(内存、文件等)
(3) 初始化进程控制块(填入 PID、优先级、状态等)
(4) 将新进程插入就绪队列

💻 4. 创建原语伪代码(教材幻灯片 143)

// 教材原版 create() 原语 Procedure create(pn, pri, res, fn, args); Begin getfreepcb(i); // (1) 申请空白 PCB if i = NIL then return(NIL); i.id := pn; i.priority := pri; // (3) 初始化 PCB 关键字段 i.resources := res; memallocate(datasetsize, add); // (2) 分配内存 if add = NIL then begin pcbrelease(i); return(NIL); // 分配失败回退 end; i.dataadd := add; i.datasize := datasetsize; datasetinit(i.dataadd, args); filestate(fn, add, size); ... i.pc := add; // 程序计数器=入口 i.parent := EXE; i.state := "ready"; i.queue := RQ; insert(RQ, i); // (4) 插入就绪队列 End;
注意: 教材伪代码里"(1)申请 PCB"和"(3)初始化 PCB"中间夹着"(2)分配资源"——但考场答题时按 1→2→3→4 默写,这是更清晰的逻辑顺序,也是绝大多数教材的简化讲法。

⚠️ 5. 易错点

坑 1 · 4 个事件漏写: 一定要把"用户登录、作业调度、提供服务、应用请求"写全——少一个就扣分。记忆口诀"登调请请"。
坑 2 · 把"进程创建"和"程序加载"混: 创建进程是为运行中的程序建立 PCB + 入就绪队列;程序加载(装入)是把磁盘上的可执行文件读到内存,这是创建过程第 2 步"分配资源"里的一部分,不要单独列。
坑 3 · 漏写"插入就绪队列": 很多教材把这一步合并到第 3 步里讲,但深圳大学自命题老师常把它单列——默写时单独写一行更稳。
坑 4 · 把"PCB 初始化"写成"清空 PCB": 初始化是填入新进程的具体信息(PID、优先级、PC 等),不是清空。

📝 6. 真题样例(简答 5 分)

样题

请简述进程的创建过程。(5 分)

点我看参考答案

答:

进程的创建由创建原语完成。引起创建进程的事件有四类:用户登录、作业调度、提供服务和应用请求。

创建过程分为四步:

  1. 申请空白 PCB(从 PCB 集合中分配一个空闲 PCB);
  2. 为新进程分配资源(内存、打开文件、初始化数据段等);
  3. 初始化 PCB(填入进程标识符 PID、优先级、父进程、程序计数器、状态置为"就绪"等);
  4. 将新进程插入就绪队列,等待调度器调度。

✅ 7. 自测

Q1 · 默写挑战

不看屏幕,默写:4 个创建事件 + 4 个创建步骤。

点我对答案

事件 4 类:用户登录 / 作业调度 / 提供服务 / 应用请求。

步骤 4 步:①申请空白 PCB → ②分配资源 → ③初始化 PCB → ④插入就绪队列。

卡壳在哪一步,回到 §3 重新读一遍。

Q2 · 易错辨析

下面哪个说法是错的?

A. 进程创建的第 2 步是"为新进程分配资源",分配失败要释放已申请的 PCB
B. 进程创建后状态置为"就绪",等待调度器选中获得 CPU
C. 引起进程创建的事件只有"用户登录"和"作业调度"两类
D. 初始化 PCB 时要把进程状态置为 ready

点我看答案

答案:C。C 把"提供服务和应用请求"漏了——4 类不是 2 类。其它三项都是教材原话。

Q3 · 联系 Linux

在 Linux 里 fork() + exec() 对应教材的"进程创建"哪一步?fork() 创建的子进程最初和父进程共享什么?

点我看答案

· fork() 对应"申请空白 PCB + 初始化 PCB + 插入就绪队列"——拷贝父进程的 PCB 模板,得到一个几乎一样的子进程。

· exec() 对应"分配资源 + 把可执行文件装入内存"——替换子进程的内存映像,开始跑新程序。

· fork() 创建的子进程最初和父进程共享代码段(只读),但数据段、堆、栈是写时复制(copy-on-write)。

这是 OS 课程不考但你可能想知道的小知识——xv6 fork 源码走读里有详细版本。
