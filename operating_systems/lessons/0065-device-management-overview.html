🗺 0. 这章考什么

🧸 一句话: 设备管理就是"让 CPU 别干等键盘和打印机"——把 I/O 设备、控制器、通道组织好,让程序用"逻辑名字"就能操作,不用关心背后是哪个物理设备。

常考题型:① 默写设备管理 4 大功能(5 分);② 解释"设备独立性"的含义及好处;③ 单/双/循环/缓冲池 4 种缓冲区对比;④ DMA 和通道的区别。

📖 1. 是什么:设备管理 = 4 件事

定义: 设备管理是 OS 对除 CPU 和内存以外的所有 I/O 设备(磁盘、打印机、键盘、控制器、DMA、通道等)进行的统一管理。

教材原话(幻灯片 5):"设备管理是指计算机系统对除 CPU 和内存以外的所有输入、输出设备的管理。设备管理不但要管理实际 I/O 操作的设备(如磁盘机、打印机),还要管理诸如设备控制器、DMA 控制器、中断控制器、I/O 处理机(通道)等支持设备。"

⚡ 2. 4 大功能(幻灯片 12)

1监视所有设备状态
随时知道设备是否可用、忙闲、故障。
2制定设备分配策略
实现设备的合理分配和回收。
3设备的驱动和处理
按设备特性发出控制命令、完成数据传输。
4数据缓冲
解决 CPU 高速和设备低速的矛盾。
记忆口诀:"监分驱缓"——监视 / 分配 / 驱动 / 缓冲(简答 4 个词写满就拿分)。

📂 3. 设备分类(共享属性,幻灯片 19)

类别含义例子分配方式
独占设备一段时间只允许一个用户使用打印机、磁带机独占分配(SPOOLing 可改造)
共享设备可并发被多个用户使用磁盘可并发访问
虚拟设备通过 SPOOLing 把独占设备变成多个逻辑设备SPOOLing 打印宏观共享、微观独占

🔌 4. 设备控制器(幻灯片 30-45)

位置: CPU 和 I/O 设备之间的接口(中介)。

组成(4 类寄存器):

  • 命令/状态寄存器 CR:接收 CPU 命令或报告设备状态
  • 数据寄存器 DR:暂存要传送的数据
  • 地址寄存器:数据送往内存的地址
  • 数据计数器 DC:本次要读/写的字节数

基本功能:接收和识别命令、标识和报告设备状态、数据交换、地址识别、数据缓冲、差错控制。

🎯 5. 设备独立性(幻灯片 154-155)

📐 简答默写模板: 
设备独立性指应用程序独立于具体使用的物理设备。
用户采用逻辑设备名称来使用设备,由系统在实际使用时,
实现逻辑设备到物理设备的映射。
带来两方面好处:
(1) 设备分配时的灵活性
(2) 易于实现 I/O 重定向

🔁 6. I/O 四级控制(概述,详见 0047)

方式CPU 干预数据单位并行度
程序 I/O全程轮询,极大浪费字符无
中断驱动每字符中断一次字符提高 ~100 倍
DMA仅开始/结束干预数据块提高 ~100 倍
I/O 通道通道程序自主完成一组数据块最高

📦 7. 缓冲区四种

类型缓冲数处理一块时间特点
单缓冲1max(C,T)+M输入/提取串行,简单
双缓冲2max(C,T)可并行,接近理想
循环缓冲N近 1环形队列,Getbuf/Releasebuf
缓冲池N 个分组管理—4 个工作缓冲区:hin/sin/hout/sout
符号说明: C = 计算时间,T = 传输时间,M = 传送到用户区时间。单缓冲处理一块时间 = max(C,T)+M(必考)。
易错: 单缓冲公式不是 (T+C+M),而是 max(C,T)+M——因为输入缓冲和数据计算是并行做的,取慢的那个。

⚠️ 8. 易错点

坑 1 · 漏写"数据缓冲": 4 大功能很多同学漏掉"数据缓冲"——它是解决 CPU 与 I/O 速度差的核心机制,必写。
坑 2 · 把"独占设备"和"虚拟设备"混: 独占设备 = 物理上一段时间只给一个人用;虚拟设备 = 用 SPOOLing 技术把独占设备改造为多个逻辑设备(宏观共享)。
坑 3 · 单缓冲公式写错: 必考的是 max(C,T)+M,不是 T+C+M——因为输入到缓冲(下一步)与 CPU 计算(上一步)可并行。

💻 9. 设备分配的补充要点

设备分配的 3 个考虑因素(幻灯片 147):

  • 设备类型:独占 / 共享 / 虚拟设备;
  • 设备分配算法:FCFS / 优先级等;
  • 设备分配中的安全性:分配后是否会导致死锁(类似银行家算法的思路)。

SPOOLing(幻灯片 209)三大特点:① 提高 I/O 速度;② 将独占设备改造为共享设备;③ 实现了虚拟设备功能。

📊 10. 4 种设备数据传输方式对比

方式数据单位CPU 干预频率适合场景
程序 I/O字符每秒上千次极简单嵌入式
中断驱动字符/字节每字符一次键盘、鼠标
DMA数据块每块开始/结束磁盘、网卡
I/O 通道一组数据块启动 + 结束大型机/巨型机
必背: CPU 干预频率从"每字符"到"每块"再到"每组块",越来越低,系统效率越来越高。

🧠 12. 通道 vs DMA(易混淆)

DMA: 只能控制一台或少数几台同类设备,数据传送过程由 DMA 控制器硬件完成,CPU 仅在开始/结束干预。

I/O 通道: 通道是一个专门的处理器,有自己的通道程序(存在主机内存),可以控制多台不同类型设备,独立完成 I/O。

教材原话(幻灯片 48):"I/O 通道是一种特殊的处理机。它具有执行 I/O 指令的能力,并通过执行通道(I/O)程序来控制 I/O 操作。但通道没有自己的内存,通道所执行的通道程序是放在主机的内存中的。"

📝 13. 缓冲区大小选择速算

单缓冲: 1 个缓冲区,处理 n 块数据总时间 ≈ n · (max(C,T)+M);

双缓冲: 2 个缓冲区,处理 n 块数据总时间 ≈ n · max(C,T);

循环缓冲: N 个缓冲区(N ≥ 3 时近理想流水)。

考试速算: 给 C/T/M 数值,问处理 100 块数据总时间——选哪种缓冲最快?答案通常是双缓冲或循环缓冲。

Q2 · 易错辨析

下面哪个说法正确?

A. 独占设备可以同时被多个进程使用
B. 设备独立性指应用程序直接使用物理设备名
C. 单缓冲处理一块数据的时间是 T + C + M
D. 缓冲池有 4 个工作缓冲区:hin/sin/hout/sout

点我看答案

Q3 · 联系实际

为什么打印机这种"独占设备"能被多个用户"同时"使用?这背后用的是什么技术?

点我对答案

通过 SPOOLing(假脱机) 技术:
· 输出井在磁盘上暂存打印任务;
· 用户进程"打印"实际只是把数据写到磁盘输出井,立即返回;
· 宏观上看所有用户都能"同时"打印,微观上打印机还是一个一个处理;
· 这就是"虚拟设备"——把独占设备改造成可共享的逻辑设备。

详见 0048 SPOOLing。
