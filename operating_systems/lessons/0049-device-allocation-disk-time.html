设备怎么分出去、用户怎么"不认物理设备"、磁盘一次访问要多久

这一课只回答一件事：系统靠哪几张表把设备分给进程（DCT/COCT/CHCT/SDT）、为什么用户只写"逻辑设备名"（设备独立性 + LUT）、以及磁盘访问时间 Ta = Ts + Tr + Tt 怎么一步步算。计算部分是必考大题。

一句话先记住： 平均旋转延迟 = 转一圈时间的一半；磁盘访问时间 = 寻道 Ts + 旋转延迟 Tr + 传输 Tt，三段一段都不能漏；设备独立性 = 用户只写"逻辑设备名"，靠 LUT 这层间接映射到真正的物理设备。

🗺 0. 全局故事线：设备管理这几件事怎么串起来

别把下面这些当成孤立知识点——它们回答的是同一件事：进程要用设备，OS 怎么把设备安全、灵活、高效地交出去，以及交出去后一次磁盘 I/O 到底花多少时间。

② 设备分配OS 查 SDT→DCT→COCT→CHCT 四张表，依次分设备、控制器、通道

③ 别写死物理设备进程若认准某台设备，坏一台就跑不了
→

→

🧸 2. 讲人话：地址簿 与 取书三步

🧸 设备独立性像"通讯录昵称"： 你手机里存的是"妈妈"，不是一串会变的电话号码。妈妈换号了，你只改通讯录里"妈妈"对应的号码，所有拨号动作一个字都不用改。这里"妈妈"= 逻辑设备名，真号码 = 物理设备名，通讯录 = LUT。
⚠️ 反例（没有它会怎样）： 假如程序里写死"往 3 号打印机打印"。今天 3 号打印机卡纸报废，整段程序就直接崩——你得回去改代码、重新编译、重新部署。有了逻辑设备名 + LUT，运维只需把 /dev/lp 的映射从打印机3 改到打印机5，程序毫不知情照常跑。这就是"设备分配的灵活性"和"I/O 重定向"两大好处。（据 5_设备管理/ppt文本.md：设备独立性两大好处）
🧸 磁盘访问像"图书馆取书三步"： ①走到对的书架排（寻道 Ts，最慢、占大头）；②站定后等旋转书架把你要的那格转到面前（旋转延迟 Tr，平均等半圈）；③伸手把书抽出来（传输 Tt，读多少拿多少）。少算任何一步，答案就错。

⚖️ 3. 对照表：四张分配表 & 逻辑 vs 物理设备名

🧸 一句话： 分配一台设备，要一路查四张表，因为设备挂在控制器上、控制器挂在通道上，环环都得空闲才能真正分成功。

⚖️ 设备分配的四张数据结构（照 PPT 口径）

表全称每系统几张记什么 / 作用
SDT系统设备表 System Device Table全系统仅 1 张登记所有设备，每个表目指向该设备的 DCT——分配的总入口
DCT设备控制表 Device Control Table每台设备 1 张记设备状态（忙/闲）、等待队列，并指向它所连的 COCT
COCT控制器控制表 COntroller Control Table每个控制器 1 张记控制器状态，并指向它所连的 CHCT
CHCT通道控制表 CHannel Control Table每个通道 1 张记通道状态——三级里最后一道
📐 分配流程（据 5_设备管理/ppt文本.md 幻灯片 163–164）：
  1. 据（逻辑名映射出的）物理设备名，从 SDT 找到该设备的 DCT → 查设备忙否
  2. 由 DCT 找到所连的 COCT → 查控制器忙否
  3. 由 COCT 找到所连的 CHCT → 查通道忙否
  4. 三者都空闲，才真正分配"设备 + 控制器 + 通道"；任一忙则挂等待队列

⚖️ 逻辑设备名 vs 物理设备名

比什么逻辑设备名物理设备名
谁在用用户 / 应用程序写的（如 /dev/lp）OS 内部真正的硬件编号（如 打印机1）
会不会变不变——程序一直用同一个名字可能变——设备坏了就换一台
靠什么连起来LUT 逻辑设备表：逻辑名 → 物理名 → 驱动入口
结论： "设备独立性"这个能力 = 用户只认逻辑名 + OS 靠 LUT 做逻辑→物理的间接映射。多一层间接，就换来了灵活性和容错。

📖 4. 是什么：精确定义（对齐教材口径）

设备独立性（Device Independence）： 应用程序独立于具体使用的物理设备——用户采用逻辑设备名来使用设备，由系统在实际使用时，实现逻辑设备到物理设备的映射。好处两条：① 设备分配时的灵活性；② 易于实现 I/O 重定向。（据 5_设备管理/ppt文本.md 幻灯片 153–155）
磁盘访问时间三段（据 textbook/算法速查表.md、ch08）：
① 寻道时间 Ts：磁头从当前磁道移到目标磁道的时间 = 启动磁臂 s + 移动 n 个磁道 = s + m·n（占总时间 70% 以上）。
② 旋转延迟 Tr：目标扇区转到磁头下的时间，平均 = 磁盘转半圈 = 1/(2r)，其中 r 是转/秒（r/min ÷ 60）。
③ 传输时间 Tt：把数据读/写完的时间 = 数据量 / 传输速率 = b/(r·N)，其中 N 是每道字节数、b 是要读字节数（因为传输速率 = 每秒转 r 圈 × 每圈 N 字节）。
总时间 Ta = Ts + Tr + Tt。
📐 磁盘访问时间解题模板（4 步）：
  1. 转速换算：r（转/秒）= 转速(r/min) ÷ 60。
  2. 算 Tr = 1/(2r)（半圈时间），别用整圈。
  3. 算 Tt = b/(r·N)，即 要读字节数 ÷（每秒 r 圈 × 每圈 N 字节）。
  4. 相加：Ta = Ts + Tr + Tt，注意单位统一到 ms。

❓ 5. 为什么：层层追问动机与边界

第 1 层 · 为什么要设备独立性？
因为物理设备会变、会坏、会被别的进程占。如果程序把物理设备名写死，任何一次硬件变动都逼着改代码。加一层逻辑名，就把"程序"和"硬件"解耦——这正是 OS "无关性"设计目标（用户程序独立于具体设备）的落地。（5_设备管理/ppt文本.md：设备管理目标"无关性"）

第 2 层 · 为什么分配要查 SDT→DCT→COCT→CHCT 四张表，而不是一张？
因为一次真正的数据通路是"设备—控制器—通道"三级串联，缺一级都传不了数据。四张表把三级资源各自的忙闲、队列、连接关系拆开记录，分配时逐级检查、逐级预留，才能保证整条通路都空闲。任一级忙 → 进程挂到对应等待队列。

第 3 层（边界）· 为什么旋转延迟取"半圈"而不是整圈或随机？
目标扇区在盘面上的位置是随机的：最好赶上刚到磁头下（约 0），最坏刚好错过要等整整一圈（1/r）。在均匀分布假设下，平均等待 = 半圈 = 1/(2r)。这是"平均"旋转延迟——考题不特别说明就用它。若题目问"最坏情况"才用整圈 1/r。同理传输时间：读满一整道 b=N 时，Tt=N/(rN)=1/r 恰好等于转一圈，可作自检。

⚠️ 三大失分点： ① 忘记 r/min ÷60 换成 r/s；② 旋转延迟用了整圈（应是半圈 1/(2r)）；③ 漏掉 Tt 或漏掉 Ts——三段必须全加。

✅ 6. 自测：先自己想，再点开答案

选择题 1 · 平均旋转延迟

磁盘的"平均旋转延迟时间"等于：

A. 磁头移动一个磁道的时间
B. 磁盘转整整一圈的时间
C. 磁盘转一圈时间的一半
D. 数据量除以传输速率

点我看答案

答案：C。 目标扇区位置随机，平均要等半圈，Tr=1/(2r)。 A 是寻道里"移动一个磁道"的量，属 Ts；B 是最坏情况（整圈），不是"平均"；D 是传输时间 Tt 的定义，不是旋转延迟。

选择题 2 · 设备独立性靠谁实现

操作系统实现"设备独立性"（逻辑设备名 → 物理设备名映射）依靠的数据结构是：

A. SDT 系统设备表
B. LUT 逻辑设备表
C. DCT 设备控制表
D. CHCT 通道控制表

点我看答案

答案：B。 LUT（逻辑设备表）逐行记录"逻辑设备名→物理设备名→驱动程序入口"，正是这层间接映射带来设备独立性。 A 的 SDT 登记全系统所有物理设备、是分配的总入口，不做逻辑名映射；C 的 DCT 记单台设备状态；D 的 CHCT 记通道状态——都不负责逻辑↔物理的翻译。

选择题 3 · 磁盘访问时间计算

某磁盘转速 6000 r/min，每道 10000 字节，平均寻道时间 8 ms。现要读 5000 字节，则平均访问时间 Ta 约为：

A. 8 ms
B. 13 ms
C. 16 ms
D. 18 ms

点我看答案

答案：D。 r=6000/60=100 转/秒。Tr=1/(2×100)=0.005 s=5 ms；传输速率=100×10000=106 字节/s，Tt=5000/106=0.005 s=5 ms；Ta=8+5+5=18 ms。 A 只算了寻道、漏了后两段；B 用了 7200 转那题的旋转延迟凑数、且漏 Tt；C 少算了旋转延迟或传输之一。可用上面的计算器输入这组数验证。

选择题 4 · 哪张表全系统唯一

下列设备分配数据结构中，整个系统只有一张、作为设备分配总入口的是：

A. SDT 系统设备表
B. DCT 设备控制表
C. COCT 控制器控制表
D. CHCT 通道控制表

点我看答案

答案：A。 SDT 全系统仅一张，登记所有设备、每个表目指向对应 DCT，分配就从它查起。 B 的 DCT 每台设备一张；C 的 COCT 每个控制器一张；D 的 CHCT 每个通道一张——都不是全局唯一。
