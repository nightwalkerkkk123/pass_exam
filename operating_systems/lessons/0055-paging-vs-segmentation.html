第 55 课 · Ch4 内存管理 · 简答 · 分页与分段的区别

分页 = 物理切、固定大小;分段 = 逻辑切、大小不固定

这是简答题 5 分大题的标准答案骨架——一段话区别 + 对照表 + 段页式组合。教材出处:4_存储器管理/ppt文本.md 幻灯片 326-327(分页 vs 分段)、333-334(段页式)。背景见 0005 分页基础、0017 分页 vs 分段专题。

0·全局线 1·一句话 2·对照表 3·地址公式 4·段页式 5·必背 6·易错 7·真题 8·自测
考场必背两句话:
① 分页按物理视角切、大小固定,地址一维;分段按逻辑视角切、大小不固定,地址二维。
② 分页:无外碎片、有内碎片;分段:有外碎片、无内碎片。段页式 = 分段 + 分页:先按段分,再按页切,段表 + 页表两张表,查表两次。

🗺 0. 这道简答题在考什么

🧸 一句话: 分页像"按固定尺寸切肉片"(切多少由刀决定),分段像"按部位切"(切多大由料理决定)。两种切法的核心差别 = 谁定的尺寸 / 对谁有意义。

5 分题的标准答法:一句话区别(1 分)+ 对照表 6-8 行(2 分)+ 地址公式 / 段页式简介(2 分)。

📖 1. 是什么:分页与分段 = 两种离散内存分配方式

分页(4.3 节): 把进程(作业)的逻辑地址空间划分为大小相等的"页"(由系统决定大小,常 4KB),把物理地址空间划分为大小相等的"块 / 页框";以页为单位离散分配。

分段(4.4 节): 把进程按逻辑意义划分成多个"段"(主程序段、数据段、堆栈段、子程序段……),段大小由用户/程序决定,每段独立从 0 编址;以段为单位离散分配。

教材原话(幻灯片 327): "页是信息的物理单位(满足系统管理的需要),段是信息的逻辑单位(满足用户的需要);页的大小固定(由系统决定),段的大小不固定(由用户决定);分页的作业(进程)地址是一维的,分段的作业(进程)地址是二维的。"

📊 2. 对照表(分页 vs 分段,考场默写版)

比较维度分页(Paging)分段(Segmentation)
划分单位页(物理单位,大小固定)段(逻辑单位,大小不固定)
划分决定者系统(硬件/管理员)用户/程序员(按程序模块)
地址结构一维:页号 P + 页内偏移 W二维:段号 S + 段内偏移 W
对用户可见性不可见,系统自动切可见,程序员按段编程
地址空间线性连续(一维)多个独立段地址空间(二维)
碎片情况无外碎片,有内碎片
(页尾装不满浪费)		无内碎片,有外碎片
(段间留空洞)
共享困难(共享粒度=页,通常不可读)容易(整段共享,如共享 editor)
地址变换查页表一次查段表一次
典型代表大多数 OS(虚拟内存)x86 保护模式、Intel 段机制
记忆口诀: "页固段变、页一维段二维、页物理段逻辑"——背这三句,对比基础就稳了。

🧮 3. 地址变换公式

① 分页的地址变换

// 逻辑地址 = (页号 P, 页内偏移 W),页大小 = PAGE_SIZE P = LA / PAGE_SIZE; // 页号 = 逻辑地址 / 页大小 W = LA % PAGE_SIZE; // 页内偏移 = 逻辑地址 mod 页大小 // 查页表得到物理块号 F F = 页表[P]; PA = F × PAGE_SIZE + W // 物理地址 = 块号×页大小 + 页内偏移

例:页大小 4KB,逻辑地址 4832(=4×1024 + 896),P=4,W=896 → 查页表得 F=15 → PA = 15×4096+896。

② 分段的地址变换

// 逻辑地址 = (段号 S, 段内偏移 W),从段表查段基址 BASE[S] 查段表得 BASE[S] 与 LIMIT[S]; // 段基址 + 段长 if (W >= LIMIT[S]) 越界中断; PA = BASE[S] + W // 物理地址 = 段基址 + 段内偏移

例:段号 2 段长 1000 段基址 5000,段内偏移 895,W<LIMIT 合法 → PA = 5000+895 = 5895。

③ 分页与分段的口诀对照

📐 考场默写模板: 
分页地址变换: 物理地址 = 页表[P] × 页大小 + 页内偏移
              (P = LA / 页大小, W = LA mod 页大小)
分段地址变换: 物理地址 = 段表[S].BASE + 段内偏移
              (先查段表得段基址和段长,越界则中断)

🧬 4. 段页式存储管理(分页 + 分段的合体)

思路: 同时吸收分段的"按用户视角"和分页的"无外碎片",先按段划分,再在段内按页划分。

教材原话(幻灯片 334): "段页式系统是分段和分页原理的组合——先将用户作业(进程)分成若干具有一定逻辑意义的段,再将段分成大小固定的若干个页;段页式系统需要同时设置段表和页表;"

① 段表
每段一条表项,登记段长和对应页表起始地址。
② 页表
每段一张页表,登记段内每页对应的物理块号。
③ 地址变换
查段表得页表,再查页表得块号 → 两次查表(快表可减为 1 次)。

口诀: "段表查段、页表查页、两次访问";段页式 继承分页的无外碎片 + 分段的用户可见性。

📝 5. 必背 4 句话

① 分页按物理单位切、分段按逻辑单位切;分页大小固定、分段大小不固定。
② 分页地址一维(P+W)、分段地址二维(S+W);分页对用户不可见、分段对用户可见。
③ 分页无外碎片、有内碎片;分段无内碎片、有外碎片。
④ 段页式 = 分段 + 分页:段表 + 页表,三次访问(段表→页表→物理块)。

⚠️ 6. 易错点

坑 1 · "分页没有碎片": 分页没有外碎片(磁盘级碎片),但有内碎片(页尾装不满)。老师最爱考的"内/外碎片 vs 分页/分段"对应关系必须记清。
坑 2 · 把"段页式"说成"既无内也无外碎片": 段页式继承分页特点 = 无外碎片、有内碎片;不要"以为两者合并就两全其美"。
坑 3 · 分页地址"一维"写反: 分页的逻辑地址用一维线性编号(P+W 只用给一个 LA 数即可),分段必须二维(S+W)。反过来说"分页二维"就错了。
坑 4 · 把"段表基址"和"页表项"搞混: 分段地址变换只用段表;分页只用页表;段页式先查段表(得到页表起始地址)再查页表(得到物理块号),这是两张不同的表。

📝 7. 真题样例(简答 5 分)

样题

请比较分页存储管理与分段存储管理的主要区别,并简述段页式存储管理的基本思想。(5 分)

点我看参考答案

答:

(1) 分页与分段的主要区别:

  1. 页是信息的物理单位(满足系统管理需要),段是信息的逻辑单位(满足用户需要);
  2. 页大小固定(由系统决定),段大小不固定(由用户决定);
  3. 分页作业地址是一维的,分段作业地址是二维的;
  4. 分页对用户不可见、由系统自动切,分段对用户可见、按程序模块划分;
  5. 分页无外碎片但有内碎片,分段无内碎片但有外碎片。

(2) 段页式存储管理的基本思想: 段页式是分段和分页的组合——先按逻辑意义把进程划分为若干段,再把每段划分为大小固定的若干页;同时设置段表和页表,段表项记录段长和对应页表的起始地址,地址变换时先查段表得到页表,再查页表得到物理块号。

出处:4_存储器管理/ppt文本.md 幻灯片 326-327、333-334。关键词:物理/逻辑、固定/不固定、一维/二维、内/外碎片。

Q2 · 易错辨析

下列说法哪个是错的?

A. 分页按物理单位划分,大小固定;分段按逻辑单位划分,大小不固定
B. 分页无外碎片但有内碎片,分段无内碎片但有外碎片
C. 段页式存储管理中,地址变换需要先查页表得到段表,再查段表得到物理块号
D. 分页地址一维,分段地址二维

点我看答案

答案:C。C 把"段表"和"页表"的次序说反了。段页式地址变换的顺序是"先查段表得到页表起始地址,再查页表得到物理块号"。其它三项完全正确。

Q3 · 算一算

某分页系统页大小 4KB,逻辑地址 4832 = 4×1024 + 896,进程页表中 P=4 对应物理块号 F=15。求物理地址?分段系统中,段号 2 段基址 5000 段长 1000 段内偏移 895,合法吗?物理地址?

点我对答案

分页: P=4, W=896 → PA = F × 4096 + W = 15 × 4096 + 896 = 61440 + 896 = 62336。

分段: 段内偏移 895 < 段长 1000,合法 → PA = BASE + W = 5000 + 895 = 5895。

更多变式题见 0040 存储计算冲刺 和 0005 分页基础。
