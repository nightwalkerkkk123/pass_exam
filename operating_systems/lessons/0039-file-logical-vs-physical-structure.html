第 39 课 · Ch7 文件管理 · 零基础可视化 · Ch6 地基

文件的逻辑结构 vs 物理结构

用户视角 ⊥ 磁盘视角:两条互相独立的轴。先动手把它们随便组合一遍,再上术语。配合 速查地图 使用。

一句话先记住: 逻辑结构是“你(用户)怎么看这个文件”——它是一串字节的流式文件,还是一条条记录的记录式文件;物理结构是“OS 怎么把它摆到磁盘上”——连续、链接,还是索引。这是两条互相独立的轴:一个流式文件既可以连续存、也可以索引存。别把它俩混成一件事。

🖼 1. 直觉图:「逻辑 ⊥ 物理」双轴选择器

👇 左边选逻辑结构(用户怎么看),右边选物理结构(OS 怎么摆到磁盘)。两栏互不影响,随便组合都成立(2×3 = 6 种)。每改一次,下面的结论和示意图都会跟着变。

📖 逻辑结构(用户怎么看)
你把文件当成什么来读写
🌊 流式文件
🗂️ 记录式文件
⊥
💽 物理结构(OS 怎么摆到磁盘)
OS 把它的数据块放在盘的哪里
📦 连续
🔗 链接
📇 索引
📖 用户看到的样子
💽 磁盘上的摆法
亲手验证到的事: 无论左边选哪个,右边三种都能选;无论右边选哪个,左边两种都能选——6 种组合全部合法。这正说明:“怎么读”(逻辑)和“怎么摆”(物理)是两条正交的轴,互不决定。

🧸 2. 具体例子:一本书(ELI5)

🧸 5 岁也懂的比喻:一本书。
逻辑结构 = 你怎么读它: 是“从头到尾一句接一句的散文”(🌊 流式,像小说),还是“一条条词条、每条各自独立”(🗂️ 记录式,像字典)?这是你阅读的方式。
物理结构 = 这本书的纸页怎么装订在书架上: 是“一整本连着放”(📦 连续)?是“每页夹一张纸条写着下一页在哪”(🔗 链接)?还是“前面有一张目录页写着每章在第几个抽屉”(📇 索引)?这是书被摆放的方式。
关键: 同一本散文,可以整本连放,也可以拆页用目录索引——“怎么读”和“怎么摆”是两回事,谁都不决定谁。

⚖️ 3. 对照表(照教材口径)

对照一:逻辑结构的两大类

类别是什么特点 / 访问方式
无结构 · 流式文件 一串无结构的字符流,不划分记录,整个文件就是一长条字节序列。 简单、灵活;按字节 / 偏移量访问。UNIX / Linux / Windows 的普通文件大多是流式。
有结构 · 记录式文件 由若干记录组成,每条记录是一组数据项的集合。 分两种:
• 定长记录:每条一样长 → 可随机访问,直接算出第 i 条的位置;缺点是短记录会浪费空间。
• 变长记录:每条长度不一 → 省空间;但只能顺序查找,随机访问难。

对照二:物理结构的三种(总览对照)

方式怎么摆优点缺点
📦 连续分配 占一组连续的块;目录只记起始块号 + 长度。 顺序和随机访问都快(算一下就知道第 i 块在哪)。 外部碎片;文件动态增长困难(旁边没空位就得整体搬家)。
🔗 链接分配 用指针把离散的块串成一条链。
隐式链接:指针藏在块内;显式链接:把指针集中到 FAT 表。		 无外部碎片;易于动态增长(找个空块挂上去就行)。 随机访问差(要顺着链一个个走)。
📇 索引分配 每个文件配一个索引块,里面存下它所有数据块的地址。 随机访问快(查索引表直接定位);无外部碎片。 小文件索引块利用率低(为一两块也得配一整块索引);大文件要用多级 / 混合索引。
范围说明: 物理结构里“混合索引最大文件大小”的详细计算不在本课,本课只做总览对照。要算的去看 → 第 22 课(索引分配),以及第 9 课 / 第 23 课。

📖 4. 是什么:精确定义(对齐教材)

文件定义: 文件是具有文件名的一组相关信息的集合,存放在外存上,是操作系统存储信息的基本单位。

文件属性(填空可能考):

属性含义
文件名用户按名存取用的标识。
类型普通文件 / 目录 / 设备文件等。
大小当前长度(字节数或块数)。
物理位置文件在外存上的存放地址。
时间创建 / 修改 / 访问时间。
保护权限谁可读 / 写 / 执行。
其他标志位只读、隐藏、系统等。

文件系统的功能(简答):

功能做什么
文件存储空间管理管理外存空闲区,给文件分配 / 回收物理块。
目录管理实现按名存取(靠名字找到文件)。
文件读写管理完成对文件内容的读写操作。
文件共享与保护让多用户能共享,同时防止越权访问。
操作接口提供 open / read / write / close 等给用户 / 程序用。
可靠性与安全性防丢失、防破坏。

两种结构的精确定义:

逻辑结构物理结构
从谁的观点看从用户观点出发看到的文件组织形式。文件在外存上的存储组织形式。
和什么有关与存储介质的物理特性无关。与存储介质特性、外存分配方式相关。
用户能否操作用户可直接操作(按自己方便的方式读写)。对用户透明(用户感觉不到)。
补一个地基概念: 外存分配的基本单位是物理块(也叫簇),常见大小 512B / 1KB / 4KB。物理结构讨论的“连续 / 链接 / 索引”,摆放的正是这些物理块。

❓ 5. 为什么:逐层深挖 3 层

第 1 层:为什么要把“逻辑结构”和“物理结构”分开?

因为用户只想按自己方便的方式读写文件(逻辑视角),他根本不该操心这个文件在磁盘上是连着放还是散着放(物理视角)。OS 把物理细节藏起来,让用户只面对“文件内容”这个逻辑对象——这正是“资源抽象”的又一次体现(和第 14 课把磁盘块抽象成文件是同一件事)。

第 2 层:这两条轴为什么是独立(正交)的?

因为它俩管的是两件不相干的事:逻辑结构决定“用户怎么访问内容”,物理结构决定“OS 怎么在盘上存放”。一个流式文件可以连续存、也可以索引存;改变物理摆法,完全不影响用户按字节读——所以两条轴能自由组合,这就是第 1 节你亲手验证的 6 种组合都成立。

第 3 层:物理结构的选择在权衡什么?

核心权衡是“随机访问速度 vs 碎片 / 动态增长”:
• 连续——访问最快,却有外部碎片、难增长;
• 链接——无碎片、易增长,却随机访问差;
• 索引——兼顾随机访问和增长,却对小文件有额外开销。
没有万能解,选哪种要看文件实际怎么被使用——这也是为什么三种方式都活着、教材要你全都掌握。

✅ 6. 自测:先自己想,再点开答案

选择题 1

“从用户观点出发、和存储介质的物理特性无关”,说的是文件的哪种结构?

A. 物理结构
B. 目录结构
C. 逻辑结构
D. 索引结构

点我看答案

答案:C。 逻辑结构 = 用户视角、与物理特性无关。A(物理结构)恰恰相反,它和存储介质、分配方式相关、对用户透明;B、D 是干扰项——目录结构讲的是文件之间怎么组织,索引结构是物理结构的一种,都不是“用户视角看单个文件”这层意思。

选择题 2

UNIX / Linux 的普通文件,大多属于哪种逻辑结构?

A. 无结构的流式文件
B. 定长记录的记录式文件
C. 变长记录的记录式文件
D. 索引式文件

点我看答案

答案:A。 UNIX / Linux / Windows 的普通文件大多是无结构的流式文件——一串字节流、不划分记录,按字节 / 偏移访问。B、C 是“有结构的记录式文件”的两个子类,不是普通文件的常态;D 混淆了轴——“索引”是物理结构,不是逻辑结构。

选择题 3

“无外部碎片、易于动态增长,但随机访问差”,说的是哪种物理结构?

A. 连续分配
B. 索引分配
C. 混合索引
D. 链接分配

点我看答案

答案:D。 链接分配用指针把离散块串起来,所以无外部碎片、易增长,但要顺着链走,随机访问差。A(连续)正相反:随机访问快却有外部碎片、难增长;B、C(索引 / 混合索引)随机访问是快的,不符合“随机访问差”。

选择题 4

关于逻辑结构和物理结构的关系,下面哪种说法正确?

A. 逻辑结构决定物理结构,选了流式就必须连续存
B. 两者相互独立,同一逻辑结构可搭配不同物理结构
C. 流式文件必须连续存,记录式文件必须索引存
D. 物理结构决定逻辑结构,连续存的一定是流式文件

点我看答案

答案:B。 逻辑 ⊥ 物理,互不决定——一个流式文件既能连续存也能索引存(第 1 节你已亲手验证 6 种组合全成立)。A、D 都是“一个决定另一个”的经典陷阱,方向相反但都错;C 更是硬把某种逻辑结构绑死到某种物理结构上,同样是把两条独立的轴混成了一件事。
