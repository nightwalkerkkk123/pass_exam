位示图法:一串比特位,标记所有盘块"占 / 空"

次重点题型,常出选择题或填空(2–3 分)。核心是字号位号公式与位运算互转。教材出处:6_文件管理/ppt文本.md 幻灯片 247–251(6.5 节)。

0·全局线 1·一句话 2·基本思想 3·字号位号 4·分配回收 5·手算例 6·易错 7·自测
考场必背两句话:
① 位示图 = 用一串二进制位标记磁盘块状态(0 = 空闲,1 = 已占;或反过来,题目会明说)。
② 块号 ↔ 位置换算:n = 字号 × 字长 + 位号(已知位置求块号),字号 = ⌊(块号-1) / 字长⌋、位号 = (块号-1) mod 字长(已知块号求位置,适合0 起编号);若从 1 起编号,公式写作 i = (n-1) / 字长、j = (n-1) % 字长。

🗺 0. 这道题在考什么

🧸 一句话: 磁盘有几万几亿个盘块,OS 要追踪"哪些已分 / 哪些空闲"。位示图就是 OS 拿一张很小的比特位小本本,每个盘块对应一位,翻一翻就知道状态。

常见题型: ① 概念辨析(位示图 vs 空闲链表 vs 成组链接);② 给块号求字号位号;③ 给字号位号求块号;④ 简述分配 / 回收流程。前两类是选择题,后两类是填空或简答。

📖 1. 是什么:位示图法

定义: 位示图法利用二进制的一位来表示磁盘中一个盘块的使用情况。

教材原话(幻灯片 248):"位示图指利用二进制的一位来表示磁盘中一个盘块的使用情况。如当值为 0 时,表示空闲盘块;当值为 1 时,表示已分配盘块。"

0/1 的含义可能在不同教材里反过来——答题时一定要按题目"0 表示____,1 表示____"的口径写。MS-DOS、Windows FAT 系列都内置位示图。

🧠 2. 基本思想:一串比特位串成一张表

要素说明
单位每个磁盘块对应1 bit
取值(默认)0 = 空闲,1 = 已分配
组织若干 bit 组成一个字(常用 16/32 位);一张位示图 = 多个字排成行列
大小盘块数 N,位示图占 N bits = N/8 字节(可全部装入内存)
优点(教材原话)① 占用空间较小,可以调入内存;② 分配和回收操作简单、有效
适用场景微型机、小型机,如 MS OS 中(幻灯片 251)
记忆口诀:"一个盘块一位管,0 空 1 占互可换;整张表小可放内存,扫一扫就分配完"。

🧮 3. 字号位号公式(核心考点)

先约定符号,避免答案写错被扣分

  • 字长 L:一个字含多少 bit(常见 16、32,字节式则 8)。
  • 盘块编号 n:从 0 或从 1 开始(看题目;默认教材幻灯片 250 的写法 i=n/8、j=n%8,意味着 n 从 0 起)。
  • 字号 i、位号 j:位示图矩阵中的行列号,i 从 0 起,j 从 0 起。
方向公式(从 0 起编号)公式(从 1 起编号)
块号 → 位置
(知道块号求字号位号)		i = n / L
j = n mod L
(整数除法,商 / 余数)		i = (n-1) / L
j = (n-1) mod L
位置 → 块号
(知道字号位号求块号)		n = i × L + jn = i × L + j + 1
速记技巧:"位置→块号:直接 i × 字长 + j(从 0 起);块号→位置:先减 1 再除余(从 1 起),用 0 起时直接除余。"教材幻灯片 250 用的就是从 0 起的版本(因为结果是 i×8+j,正好是位偏移)。
📐 考场默写模板: 
分配:① 扫描位示图找一组 0(对应空闲块)
     ② 由 (字号 i, 位号 j) 算出块号 n = i×L + j   (从 0 起)
     ③ 把该位置改为 1
回收:① 由块号 n 算出 (i, j):  i = n/L (取整),  j = n mod L
     ② 把该位置改为 0

🔁 4. 分配 / 回收流程(简答题骨架)

分1扫描位示图
按行扫描,找一个(或连续多个)值为 0 的位。可用"找第一个 0"或"找最长 0 串"两种策略。
分2位置 → 块号
由 (字号 i, 位号 j) 用 n = i × L + j(从 0 起)算出盘块号。
分3置位标识
将位示图中这几位从 0 改为 1。内存里改完即生效(位示图常驻内存)。
回1块号 → 位置
回收时反推: i = n / L, j = n mod L。
回2清位标识
把对应位从 1 改回 0。

✏️ 5. 手算例题(必会)

例题(从 0 起编号,字长 L = 8)

已知:某磁盘位示图字号 1 的位号 5 已被占用。

问:它对应的盘块号 n是多少?

点我看解

解:公式 n = i × L + j = 1 × 8 + 5 = 13。

所以第 1 字第 5 位对应的盘块号是 13 号。

例题(从 1 起编号,字长 L = 16)

已知:磁盘共有 16384 块,系统用位示图管理。某文件申请得到第 200 号盘块。

问:它在位示图里的字号 i 和位号 j各是多少?

点我看解

解:块号从 1 起,先减 1:
i = (200 − 1) / 16 = 199 / 16 = 12(整数除,12×16=192,余 7)
j = (200 − 1) mod 16 = 199 mod 16 = 7

所以第 200 块对应字号 12、位号 7(从 0 起)。验证:12×16+7+1=200 ✓

每张位示图占 16384 bits = 2 KB,常驻内存没问题。

例题(综合:扫描找空闲)

已知:位示图共 3 字,每字 8 位,从 0 起编号。第 0 字 = 1111 0100,第 1 字 = 0011 1111,第 2 字 = 0000 0000。

问:① 现在磁盘上已分配几个块?② 若系统要给某文件分配连续 5 块,会从哪个块号开始分配?分配后位示图怎么变?

点我看解

① 数 1 的个数:第 0 字 4 个 1(1111 0100),第 1 字 6 个 1(0011 1111),第 2 字 0 个 1 → 已分配 10 块。

② 找连续 5 个 0:逐位扫:
· 第 0 字 (0,0)~(0,2) = 1 1 1;(0,3)(0,4) = 0 1 不是连续 5 个 0。
· 第 1 字 (1,0)~(1,5) = 0 0 1 1 1 1,只开头 2 个 0。
· 第 2 字 (2,0)~(2,7) = 0 0 0 0 0 0 0 0,有 8 个连续 0,够 5 个,就从这里开始。

起点块号 n = 2×8 + 0 = 16。分配 5 块,n ∈ [16, 20]。

分配后第 2 字变为 0000 1111(第 0~4 位变 1),第 3~7 位仍为 0。

⚠️ 6. 易错点

坑 1 · 编号从 0 还是从 1 写反: 教材幻灯片 250 给的公式是 i = n/8、j = n%8、n = i×8+j,这默认块号从 0 起。多数教材题(如期末考)用从 1 起(块号 1 ~ N),需要先减 1 再除余。公式答反直接扣光。
坑 2 · 字长 L 用错: L 是多少 bit对应一个字。如果题目说"位示图按字节(8 位)组织",L = 8;如果说"按 32 位字组织",L = 32。数字错就全错。
坑 3 · 0/1 含义写反: 教材默认 0 空闲、1 已占,但有些教材/题目会反过来。答题时先抄题目口径("由题意,0 表示____,1 表示____"),别直接默写。

✅ 7. 自测

Q1 · 公式速算

某系统磁盘共 8192 块,位示图按"32 bit 字"组织。某文件申请得到第 5000 号盘块(从 1 起编号)。求字号 i 和位号 j。

点我对答案

i = (5000−1) / 32 = 4999 / 32 = 156(整数除,156×32=4992,余 7)

j = (5000−1) % 32 = 4999 % 32 = 7

验证:156 × 32 + 7 + 1 = 4999 + 1 = 5000 ✓

位示图大小:8192 bits = 1024 B = 1 KB 常驻内存。

Q2 · 概念辨析(单选)

下面关于位示图法的说法,错误的是?

A. 位示图占用空间小,可以装入内存
B. 位示图适合微型机和小型机,如 MS OS
C. 分配过程要顺序扫描整张位示图,因此分配速度慢
D. 回收过程要先由块号算出位置信息

点我看答案

答案:C。位示图常驻内存、逐位扫描位运算极快(一句 find_first_zero 几个 CPU 周期级),速度并不慢;真正慢的是成组链接要维护空闲盘块栈。A、B、D 都是教材幻灯片 251 的原话。
