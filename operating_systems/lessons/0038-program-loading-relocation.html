程序的装入 · 链接 · 重定位

这一课只回答一件事:程序里写的地址,和它在内存的真实位置,怎么对上号?先动手拖一个滑块,再上术语。配合 速查地图 使用。

一句话先记住: 程序里写的地址(逻辑地址 / 相对地址)是从 0 算起的,和它最终待在内存的真实位置(物理地址)对不上;把逻辑地址换算成物理地址的过程叫重定位。换算发生在“装入时一次性”就是静态重定位;发生在“运行时每次访问、由硬件做”就是动态重定位。

🖼 1. 直觉图:「地址重定位演示器」

程序里有一条指令 LOAD,它想访问相对地址 100。这条指令最终跑在内存的哪里,取决于程序被装到哪个基址。选一种模式,拖滑块,再点按钮体验两种模式的天壤之别。

📦 你的程序(装入模块)
指令 LOAD 要访问:100
程序里现在存的是逻辑地址 100(从 0 开始编址)。
🎚 装入起始地址(基址)
基址 = 1000
物理地址 = 基址 + 逻辑地址 = 1000 + 100 = 1100
← 先选一种模式,再点「计算物理地址」。
看懂这台演示器,本节就懂一半了: 无论哪种模式,换算规则都是 物理地址 = 基址 + 逻辑地址。区别只在两件事——什么时候换算(装入时一次 vs 运行时每次)、程序能不能搬家(静态搬了就出错 vs 动态改个寄存器照跑)。

🧸 2. 具体例子:请柬上的「从大门第 100 步」

🧸 5 岁也懂的比喻: 你写请柬,只会写「进大门后往里第 100 步」——这是逻辑地址,因为你还不知道会场会摆在城市哪个角落。
① 静态重定位:布置会场时,工作人员一次性把它改成「绝对位置:东经…北纬…」(装入时就定死)。可一旦会场整体挪个地方,那张绝对坐标就全错了,只能重印请柬重新算一遍。
② 动态重定位:请柬上永远写「从大门第 100 步」。门开在哪不管,进门的那一刻再由门口的人(重定位寄存器)按「当前大门位置 + 100 步」把你带过去。会场随便挪,请柬一个字都不用改——挪门口牌子就行。

⚖️ 3. 对照表:三种装入 + 三种链接

对照一:三种装入方式(重点在中间两种)

装入方式地址怎么来 / 何时转换特点与代价
绝对装入 编译时就直接产生绝对地址,装入时不改。 最简单;但要求编译时就知道装到哪 → 仅适用单道程序环境。
可重定位装入
(=静态重定位)		 装入时一次性把相对地址 + 起始地址 → 绝对地址,以后不再改。 简单、无需硬件;但装入后不能移动、须占连续空间、难共享。
动态运行时装入
(=动态重定位)		 装入后内存里仍是相对地址,转换推迟到执行时由重定位寄存器做。 可移动、不需连续空间、支持对换和虚存、便于共享;但需硬件支持。

对照二:三种链接方式

链接方式什么时候链特点与代价
静态链接 运行前就把所有目标模块 + 库链成一个完整装入模块。 运行快;但文件大、库更新要重新链接、多程序调同一库有多份副本浪费内存。
装入时动态链接 装入内存时边装边链。 便于修改更新、可共享、省内存;装入稍慢。
运行时动态链接 执行到真正用某模块时才调入并链接。 省内存、启动快、便于更新共享;有运行时链接开销。典型:Windows 的 DLL、Linux 的 .so。

📖 4. 是什么:精确定义(对齐教材口径)

程序运行前的三步: 编译(源代码 → 目标模块)→ 链接(目标模块 + 库 → 完整的装入模块)→ 装入(装入模块 → 内存运行)。重定位就发生在最后一步「装入 / 运行」环节。
术语定义
逻辑地址(相对地址)程序中使用的地址,从 0 开始编址(编译 / 用户视角)。
物理地址(绝对地址)内存单元的真实地址。
重定位把逻辑地址转换成物理地址的过程。

静态重定位 vs 动态重定位(三点对照,考试就考这三行):

比什么静态重定位动态重定位
① 何时转换装入时一次性全部改完运行时每次访问由硬件现算
② 程序能否移动不能(地址已写死)能(逻辑地址不变)
③ 硬件要求不需要需要重定位寄存器

❓ 5. 为什么:一层层往下挖

第 1 层:为什么需要重定位?

因为程序在编译时并不知道自己将来会被装到内存哪个位置,只能先按「从 0 开始」编相对地址。真正装入时才知道落在哪个基址,于是必须把相对地址换算成真实物理地址——不换算,程序访问 100 就会跑到内存真正的 100 号单元去,那根本不是它的地盘。

第 2 层:为什么还要区分静态和动态?

静态图省事——装入时改一次就不管了,连硬件都不用。但代价是地址被写死、程序钉在原地不能动。一旦要对换(把进程换出再换回)、要紧凑(移动进程腾出连续空间)、要多个程序共享同一段代码,那些写死的绝对地址就全乱套了。区分,是为了在「省事」和「灵活」之间做选择。

第 3 层:为什么动态重定位这么关键?

它把地址转换推迟到「每次访问的运行时」,程序里永远只存逻辑地址,真实位置由重定位寄存器现算——于是搬动一个进程,只需改一个寄存器的值,程序一个字都不用动。
再往下问一句「所以呢?」 → 正是它,才让对换、紧凑、虚拟存储、代码共享统统成为可能。它是现代内存管理的地基。

✅ 6. 自测:先自己想,再点开答案

选择题 1

下列哪种方式把「地址转换推迟到程序执行时、每次访问由重定位寄存器完成」?

A. 动态运行时装入(动态重定位)
B. 可重定位装入(静态重定位)
C. 绝对装入
D. 静态链接

点我看答案

答案:A。 只有动态重定位把转换推迟到运行时、由重定位寄存器现算。陷阱 B:静态重定位是装入时一次性转换、把地址写死,不是运行时;C 绝对装入编译时就是绝对地址、根本不转换;D 是链接方式,不是装入 / 重定位。

选择题 2

静态重定位(可重定位装入)的关键缺点是?

A. 需要专门的重定位寄存器硬件
B. 转换发生在每次访问、运行开销大
C. 程序装入后不能再移动、必须占用连续内存空间
D. 编译时就必须知道程序的绝对地址

点我看答案

答案:C。 静态重定位把地址一次性写死,痛点正是装入后不能移动、须占连续空间、难共享。陷阱 A、B 说反了——需要硬件、每次访问才转换的是动态重定位;D 是绝对装入的特点,不是静态重定位。

选择题 3

「逻辑地址(相对地址)」指的是?

A. 内存单元的真实物理地址
B. 重定位寄存器里存的基址
C. 磁盘上文件的存放位置
D. 程序中使用、从 0 开始编址的地址

点我看答案

答案:D。 逻辑地址是编译 / 用户视角、从 0 起算的相对地址,要经重定位才变成物理地址。陷阱 A:那是它的对立面——物理地址;B 是基址不是逻辑地址;C 与地址转换无关。

选择题 4

Windows 的 DLL、Linux 的 .so 属于哪种链接方式?

A. 静态链接
B. 运行时动态链接
C. 装入时动态链接
D. 绝对装入

点我看答案

答案:B。 DLL/.so 是「执行到真正用某模块时才调入并链接」,即运行时动态链接。陷阱 C:装入时动态链接是「装入内存那一刻就边装边链」,时机更早;A 静态链接会在运行前把库并进可执行文件;D 是装入方式,不是链接方式。

口述题(别打字,张嘴说)

  1. 说出程序运行前的三步(编译 → 链接 → 装入),并讲清每一步的输入 → 输出是什么。
  2. 用公式「物理地址 = 基址 + 逻辑地址」讲一遍:静态和动态各在什么时候做这步换算?各由谁来做?
  3. 为什么动态重定位能支持「对换 / 紧凑 / 虚存 / 共享」,而静态却不能移动?(要点:动态只存逻辑地址、搬家只改一个寄存器;静态地址已被写死。)

三题都能顺口说出来 → 你是真懂了,不只是「听过」。哪题卡壳,回到第 1 节再拖一遍那台演示器。
