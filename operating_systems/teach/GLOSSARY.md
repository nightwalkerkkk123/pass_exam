# 操作系统 Glossary

> 用户所在工作区的标准术语表。**所有 lesson 和 learning record 都必须使用本表里的词**，避免同义词混乱。
> 只有"用户已经理解"的术语才进入本表 —— 这是压缩出来的知识，不是字典。

## 通用约定

- **OS**：本表中"操作系统"统一用 **OS**
- **进程 vs 线程**：见各章条目
- 「调度」无前缀时默认指 **处理机调度**；磁盘调度要写全
- 「硬件」= 4 样硬件资源：CPU / 内存 / 磁盘 / I/O 设备
- 「抽象」= OS 给程序员的 4 个接口：进程 / 虚拟地址空间 / 文件 / read-write-open-close

## 基础概念（已掌握）

**OS（操作系统）**：
夹在程序和硬件之间、主动管 4 样硬件资源的店长（系统软件）。
_Avoid_: "桥"（太静态，没有"管"的动作）

**4 大硬件资源**：
CPU（计算）、内存（临时工作台）、磁盘（永久仓库）、I/O 设备（键盘/鼠标/显示器/网卡）—— OS 要主动管的对象。
_Avoid_: 应用程序（不是硬件资源）

**4 大抽象**：
进程（CPU 抽象）、虚拟地址空间（内存抽象）、文件（磁盘抽象）、read/write/open/close（IO 抽象）—— OS 给程序员的接口。
_Avoid_: 硬件本身（硬件是 OS 要"管"的对象；抽象是 OS"给"程序员的接口）

**程序 vs 进程**：
程序 = 静态代码（菜谱）；进程 = 程序的一次执行（正在做的菜）。1 个程序可以跑出 N 个进程（双开 QQ = 1 程序 2 进程）。
_Avoid_: 把"程序"和"进程"混用

**4 大特征**：
并发（多个程序"同时"在跑）、共享（多程序共用资源）、异步（程序不知下一步何时发生）、虚拟（1 台电脑看起来像 N 台）。
口诀：「**并共异虚**」。
_Avoid_: "实时"（是 OS 类型，不是特征）；"并行"（是硬件能力，不是 OS 特征）

**并发 vs 并行**：
并发 = 1 个 CPU 通过快速切换让多任务"看起来同时"；并行 = 多核 CPU 真的同时执行。
金句：「**单核 OS 只有并发，没有并行**」。
_Avoid_: 把"并发"和"并行"互换（很多人写反）

## 磁盘调度（Lesson 1-2，已掌握计算）

**寻道 / 总寻道距离**：
磁头从当前磁道移动到目标磁道叫"寻道"。一道题的**总寻道距离 = 每一段移动量之和 = Σ |相邻两磁道之差|**（一段一段加）。
_Avoid_: 「起点到终点的差」——磁头来回折返时 `|起点−终点|` 会漏掉折返路程，必须**逐段**加。她在 L1 踩过这个措辞坑。

**FCFS（先来先服务）**：
按请求**到达顺序**处理，不挑。简单公平，但来回折腾、平均寻道长。

**SSTF（最短寻道优先）**：
每步挑离**当前位置最近**的请求。寻道短，但两端请求会**饥饿**（老服务中间）。
_Avoid_: 把 SSTF 说成"挑数字最小的"——是挑"离当前最近的"。

**SCAN（电梯算法）**：
磁头沿**一个方向**走，处理沿途请求，**走到最后一个请求就掉头**（汤小丹口径：到最后请求折返，不是到磁盘边缘），再反向扫回。避免饥饿。

**CSCAN（循环扫描）**：
单向扫描：一个方向走到最后请求后，**直接跳回另一端的第一个请求**（这一跳的距离要算进总寻道！），继续同方向扫。等待更均匀。
_Avoid_: 漏算"跳回"那一段距离（教材例题该段 =169，常被漏）。

## 页面置换（Lesson 4-5，已掌握计算）

**缺页 vs 命中**：
缺页 = 要访问的页**不在内存**，要从磁盘调入（慢，一次 IO）；命中 = 页**已在内存**，直接用。**缺页率 = 缺页次数 / 总访问次数**。

**FIFO vs LRU（判据必须分清，她在 L5 串过）**：
两者看的**时间不同**——**FIFO 看「出生顺序」**（进入内存最早 → 踢它，不管最近用没用）；**LRU 看「上次露面」**（最久没被访问 → 踢它，命中会续命）。
金句：「**FIFO 看出生，LRU 看露面**」。
_Avoid_: 把 FIFO 说成"淘汰最久没访问的"——那是 LRU。

**OPT（最佳置换）**：
踢「**将来**最久用不到/再也不用」的页（向未来看）。缺页最少，但需预知未来 → **不可实现**，只当性能基准（天花板）。

**Belady 异常**：
物理块增多、缺页反而增多。**只有 FIFO 会犯**；LRU/OPT 是堆栈类算法，块越多缺页单调不增，不会犯。

## 文件管理（Lesson 7-8，已掌握计算）

**N（混合索引）**：
**一个磁盘块能装的地址「个数」= 块大小 B ÷ 地址项大小 A**。是「几个」不是「多大」。
_Avoid_: 说成"地址大小"——那是 A（一个地址占几字节，是分母）。她在 L7 串过。

**混合索引最大文件**：
**D×B + N×B + N²×B + N³×B**（直接 + 一级 + 二级 + 三级，全加）。间接层是**乘 N**不是加。标准题 B=4KB/A=4B → N=1024 → 40KB+4MB+4GB+4TB（4TB 来自三级）。

**成组链接法**：
空闲块每组 N 个，组间链接。**S.free 栈只存当前一组**（[数量]+块号，栈顶在右），其余组藏在"每组最后一块"肚子里。
两条边界规则：**分配剩 1 块** → 先读出该块肚子里的下一组到 S.free，再分配该块（防断链）；**回收遇满组** → 把当前清单写进新回收块，新块单独成组。
_Avoid_: 分配最后一块时直接弹走（会丢整条链）；回收满组时还往栈里硬压。

## 进程同步 / PV（Lesson 9-10，已掌握）

**临界资源 / 临界区**：
临界资源 = 一段时间只许一个进程访问的资源；临界区 = 进程中访问它的那段代码。

**信号量 S + P/V**：
S = 可用资源计数。**P(wait) = 申请 = S−1**，若 S<0 则进程阻塞排队；**V(signal) = 释放 = S+1**，若 S≤0 则唤醒一个等待者。
口诀「**P 减 V 加，P 进 V 出**」。**S<0 时 |S| = 等待进程数**。
_Avoid_: 把 P/V 只说成加减——关键在"减到<0要排队睡、加完有人等要唤醒"。

**互斥信号量 mutex**：
初值 **= 1**（一把钥匙）。临界区夹在 `P(mutex) … V(mutex)` 之间。P、V 必须成对。

**生产者-消费者**：
三信号量：**mutex=1，empty=n，full=0**。生产者 `P(empty)→P(mutex)→放→V(mutex)→V(full)`；消费者把 empty/full 对调。
金句：「**P 顺序要命（资源在前、mutex 在后），V 顺序随意**」。反了会死锁（攥着钥匙等资源，对方拿不到钥匙没法释放）。
_Avoid_: 先 P(mutex) 再 P(empty/full)——死锁。
