16. 银行家算法 计算

系统有 A、B、C 三类资源，总量为 (10, 5, 7)。T0 时刻各进程情况如下：

进程AllocationMax
ABCABC
P0010753
P1200322
P2302902
P3211222
P4002433

(1) 求 Available 和各进程 Need；(2) 系统是否安全？给出一个安全序列；(3) 若 P1 发出请求 Request₁=(1,0,2)，能否分配？为什么？

▸ 显示参考答案

参考答案

(1) 已分配合计 = (7,2,5)，故 Available = (10,5,7) − (7,2,5) = (3,3,2)。
Need = Max − Allocation：P0(7,4,3)、P1(1,2,2)、P2(6,0,0)、P3(0,1,1)、P4(4,3,1)。

(2) 安全。 跑安全性算法（Work 从 (3,3,2) 起）：
P1 Need(1,2,2)≤(3,3,2) ✅ → Work=(3,3,2)+(2,0,0)=(5,3,2)
P3 Need(0,1,1)≤(5,3,2) ✅ → +(2,1,1)=(7,4,3)
P4 Need(4,3,1)≤(7,4,3) ✅ → +(0,0,2)=(7,4,5)
P0 Need(7,4,3)≤(7,4,5) ✅ → +(0,1,0)=(7,5,5)
P2 Need(6,0,0)≤(7,5,5) ✅ → +(3,0,2)=(10,5,7)
全部完成 → 安全序列 <P1, P3, P4, P0, P2>（不唯一）。

(3) 能分配。 ① Request₁(1,0,2) ≤ Need₁(1,2,2) ✅；② ≤ Available(3,3,2) ✅。试探分配后：Available=(2,3,0)，Alloc₁=(3,0,2)，Need₁=(0,2,0)。再跑安全性算法仍能找到安全序列（如 <P1,P3,P4,P0,P2>）→ 安全 → 可以分配。

深挖： 第 (3) 问必须“试探分配 + 重跑安全性检查”，不能只看 Request≤Available 就给——那只是必要条件，不是充分条件。

17. 页面置换：FIFO vs LRU 计算

分配 3 个物理块，页面访问串为：7 0 1 2 0 3 0 4 2 3 0 3 2（开始物理块为空，每次调入算一次缺页）。
分别用 FIFO 和 LRU，求缺页次数。

▸ 显示参考答案

参考答案

FIFO = 10 次缺页。 命中的只有第 5 个(0)、第 12 个(3)、第 13 个(2) 三次，其余 10 次均缺页。
淘汰永远赶走“最早进来”的页：…到第 11 个访问 0 时，把 4 换出得 {2,3,0}，最后 3、2 命中。

LRU = 9 次缺页。 淘汰“最久未被访问”的页。关键差异在序列后段：LRU 因为 0 被反复访问而把它保住，比 FIFO 少缺一次。

深挖： 同一串、同样 3 块，LRU(9) 比 FIFO(10) 少——印证 LRU 利用了“最近用过的很可能再用”（局部性），通常优于 FIFO。但注意：这并不是绝对的，关键看访问串。

18. 混合索引：文件最大长度 计算

某文件系统 inode 含 10 个直接块、1 个一级间接、1 个二级间接、1 个三级间接。盘块大小 1KB，每个盘块号占 4 字节。求单个文件最大可寻址的块数与大小。

▸ 显示参考答案

参考答案

每块能放 1KB ÷ 4B = 256 个块号。

  • 直接：10 块
  • 一级间接：256 块
  • 二级间接：256 × 256 = 65 536 块
  • 三级间接：256 × 256 × 256 = 16 777 216 块

合计 ≈ 10 + 256 + 65 536 + 16 777 216 = 16 843 018 块，每块 1KB → 最大文件 ≈ 16 843 018 KB ≈ 16 GB。

深挖： 容量几乎全部来自三级间接（≈16GB），直接块只占 10KB。混合索引的精妙：小文件用直接块（一跳就到，快）；大文件靠多级间接（慢一点但能寻址巨大空间）——兼顾两端。

四、简答深挖（4 题，写出“为什么”）

19. 为什么进程“阻塞→运行”不可能，只能“阻塞→就绪”？
▸ 显示参考答案

参考答案

阻塞解除只意味着进程等待的事件发生了（重新具备运行条件），但它还没拿到 CPU。CPU 此刻可能正被别的进程占用，是否分配 CPU 由进程调度程序决定。所以被唤醒的进程只能先进入就绪队列排队，等到被调度才转入运行。
一句话：得到资源 ≠ 得到 CPU，必须经“就绪”中转。

20. 银行家算法为什么属于“避免”而不是“预防”？二者根本区别是什么？
▸ 显示参考答案

参考答案

预防：事先破坏四个必要条件之一（如一次性分配全部资源、按序申请），从规则上断绝死锁可能，但代价是资源利用率低。
避免：不破坏任何必要条件，资源仍可动态申请；只是在每次分配前用银行家算法预演，判断分配后是否进入不安全状态，安全才分配。
根本区别：预防＝改规则断根；避免＝规则不变、每步预演绕开死路。银行家算法是后者。

21. 为什么 DMA 方式比中断驱动方式能“成百倍”减少 CPU 干预？
▸ 显示参考答案

参考答案

中断驱动方式下，每传输一个字（字节）设备就中断 CPU 一次，由 CPU 把这个字从控制器搬到内存——传一个数据块要中断成百上千次。
DMA 方式下，数据由 DMA 控制器直接在设备与内存之间成块传送，CPU 只在一个数据块的开始（启动）和结束（收尾中断）时各干预一次。于是一个块只打扰 CPU 1～2 次，而不是几百上千次。
关键：干预的“单位”从“字”升级到“块”，且数据通路绕开了 CPU。

22. Belady 异常说明了什么？为什么 LRU、OPT 不会出现？
▸ 显示参考答案

参考答案

Belady 异常说明：“分配更多内存（页框）”并不总能减少缺页——在 FIFO 下，增加页框反而可能使缺页率升高。这打破了“内存越大越好”的直觉。
原因：FIFO 只按“进入时间”淘汰，与“是否还要用”无关，缺乏单调性。而 LRU、OPT 属于栈式算法：满足“堆栈性质”——n+1 个页框时驻留的页集合，必然包含 n 个页框时的页集合，所以增加页框只会让缺页不增（单调），不会出现异常。
一句话：FIFO 无单调性会反常；栈式算法（LRU/OPT）有单调性，绝不会 Belady。
