第 16 课 · 银行家算法 （动手可视化版）

这一版不靠你硬记。你亲手点“下一步”，看金库怎么“滚雪球”，再看“乱借”怎么把所有人卡死。约 15 分钟。
一句话先记住： 银行家算法是死锁的「避免」——每次借钱前先在脑子里预演一遍：借了之后，还能不能找到一个顺序让所有客户都凑够、干完、还钱？能→「安全」→借；找不到→「不安全」→让他等。目的不是赚利息，是不让大家互相卡死。

一、先看懂一张“进程卡片”

每个客户（=进程）就是一张卡片。卡片上的横条 = 他总共需要的钱（Max）。横条分两段：

  • 蓝色 = 已经借走的（Allocation）
  • 斜纹 = 还差多少才够开工（Need = Max − Allocation）
开工规则（关键设定）： 客户必须把斜纹部分也借满，才能开工赚钱；干完工，会把借的钱一分不少全还回金库。没借满 → 攥着钱干等，不开工也不还钱。

同样的初始局面（金库 3 台）。这次银行家不预演，看见 P3 张口就把 3 台全给了它。点下面的红按钮，看会发生什么：

🏦 金库手头 3 台磁带机可用
点红按钮，模拟“不预演、乱借”……
看明白了吗？ 金库被掏空成 0，可 P1 还需 5、P2 还需 2、P3 还需 4——谁都喂不饱、谁都不开工、谁都不还钱，永远互相干等。这就是死锁。银行家算法存在的全部意义，就是在借出去之前预演一遍，提前发现这条死路并绕开它。

四、把演示①写成“安全性算法”（考试写法）

  1. 令 Work = Available，所有进程 Finish = 假。
  2. 找一个 Finish=假 且 Need ≤ Work 的进程。
  3. 假设它跑完：Work = Work + Allocation（它把占用的全还回来），Finish=真。回到第 2 步。
  4. 全部 Finish=真 → 安全（记下顺序=安全序列）；中途找不到 → 不安全。
步Work（手头）挑中谁（Need≤Work）它还钱后 Work
13P2（还需2 ≤ 3）✅3 + 2 = 5
25P1（还需5 ≤ 5）✅5 + 5 = 10
310P3（还需7 ≤ 10）✅10 + 2 = 12

全部完成 → 安全，安全序列 P2 → P1 → P3。和你刚才点出来的一模一样。

五、必背三句

① Need = Max − Allocation（还差的 = 最多要的 − 已拿的）。

② 安全性算法：找 Need≤Work 的进程 → 它跑完 Work += Allocation → 重复，全完成则安全。

③ 银行家算法 = 死锁避免（不是预防）：每次分配前预演，绝不踏入不安全状态。
