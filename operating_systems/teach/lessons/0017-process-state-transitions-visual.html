第 17 课 · 进程的三种状态与转换 （动手版）

三个厨师抢一个灶台（CPU）。亲手点事件，看“进程小盘子”怎么在 就绪 / 运行 / 阻塞 之间跳。约 12 分钟。
一句话先记： 运行＝正占着 CPU 炒菜；就绪＝料备好了只差灶台；阻塞＝缺酱油在等送货，给灶台也没用。

二、四条合法转换（课本原因）

转换触发原因灶台画面
就绪→运行调度程序把 CPU 分给它灶台空了，轮到你上
运行→就绪时间片用完 / 被抢占到点被赶下灶台，回去排队（料还在）
运行→阻塞运行中要等事件（如发 I/O）炒一半缺酱油，主动让出灶台去等
阻塞→就绪等的事件发生了（I/O 完成）酱油到了，先回去排队（灶台可能有人）

三、两条“绝不可能”——考试必考

① 阻塞 → 运行：不可能。 酱油刚到，你不能直接跳上灶台——灶台可能正被别人占着，必须先回就绪队列排队。
本质：得到资源 ≠ 得到 CPU，所以阻塞解除后只能进“就绪”，要 CPU 还得再被调度。

② 就绪 → 阻塞：不可能。 就绪态没在运行，它发不出 I/O 请求，不缺任何“事件”，只缺 CPU，自然不会进入“等事件”的阻塞。
本质：没运行就不会主动去等事件。
