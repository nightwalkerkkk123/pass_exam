二、问答题(5 × 4 = 20 分)

1. 简述磁盘调度中 SCAN(电梯)算法与 CSCAN(循环扫描)算法的区别。
参考答案
SCAN:磁头沿一个方向移动,处理沿途请求,到达该方向最后一个请求后掉头,反向扫回处理另一侧请求。CSCAN:磁头沿一个方向扫到最后请求后,不掉头,直接跳回另一端的第一个请求,再继续同方向扫描(单向服务)。区别:SCAN 双向往返;CSCAN 单向+跳回,各请求等待时间更均匀,但跳回有额外开销。
2. 为什么 FIFO 可能出现 Belady 异常,而 LRU 不会?
参考答案
FIFO 仅按"进入内存的先后"淘汰,与页面是否常用无关,增加物理块后淘汰序列可能变化,导致缺页反而增多(Belady 异常)。LRU 属于堆栈类算法:物理块多时,内存中保留的页面集合是块少时的"超集",缺页次数随块数单调不增,故不会出现 Belady 异常。
3. 写出产生死锁的四个必要条件。
参考答案
① 互斥条件 ② 请求和保持条件 ③ 不可抢占(不可剥夺)条件 ④ 循环等待条件。(四者同时成立才会死锁,破坏其一即可预防。)
4. 简述分页和分段的三点主要区别。
参考答案
① 划分依据:分页是物理单位(系统按固定大小划分),分段是逻辑单位(按程序逻辑、大小可变)。② 地址结构:分页一维,分段二维(段号+段内偏移)。③ 碎片:分页有内部碎片无外部,分段有外部碎片无内部。(另:分段更便于共享和保护。)
5. 什么是虚拟存储器?它的实现基于什么原理?
参考答案
虚拟存储器:基于请求调页(段)和置换功能,把部分程序装入内存即可运行,使用户感觉到的内存容量远大于实际物理内存——逻辑上扩充了内存。其实现基于程序的局部性原理(时间局部性 + 空间局部性):程序在一段时间内只访问局部的少数页面,故无需全部装入。

三、计算题(5 × 8 = 40 分)

1.【磁盘调度】磁头当前在 53 号磁道,请求队列为 98, 183, 37, 122, 14, 124, 65, 67,先向磁道号增大方向移动。分别求 FCFS、SSTF、SCAN、CSCAN 的总寻道距离(SCAN/CSCAN 到该方向最后一个请求即折返/跳回)。
参考答案
FCFS:53→98→183→37→122→14→124→65→67,=45+85+146+85+108+110+59+2 = 640。
SSTF:53→65→67→37→14→98→122→124→183 = 236。
SCAN(向大,到183折返):53→65→67→98→122→124→183→37→14 = (183−53)+(183−14)=130+169 = 299。
CSCAN:53→65→67→98→122→124→183→(跳回)14→37 = (183−53)+(183−14)+(37−14)=130+169+23 = 322。
2.【页面置换】物理块数 3,页面访问串 7, 0, 1, 2, 0, 3, 0, 4。分别求 FIFO 和 LRU 的缺页次数。
参考答案
FIFO:依次缺页 7,0,1,2,(0命中),3,0,4 → 共 7 次缺页(命中 1 次)。
 过程:[7][7,0][7,0,1]→换7[0,1,2]→0命中→换0[1,2,3]→换1[2,3,0]→换2[3,0,4]。
LRU:7,0,1,2,(0命中),3,(0命中),4 → 共 6 次缺页(命中 2 次)。
 过程:[7][7,0][7,0,1]→换7[0,1,2]→0命中→换1[2,0,3]→0命中→换2[0,3,4]。
LRU(6)优于 FIFO(7),因为 0 被频繁访问,LRU 把它保留下来。
3.【混合索引】某文件系统采用 UNIX 混合索引:10 个直接地址项 + 1 个一级间接 + 1 个二级间接 + 1 个三级间接。磁盘块大小 1KB,每个地址项 4B。求一个磁盘块能存放的地址项数 N,以及该文件系统的最大文件大小。
参考答案
N = 块大小 ÷ 地址项大小 = 1024 ÷ 4 = 256。
直接:10 × 1KB = 10KB;一级:256 × 1KB = 256KB;二级:256² × 1KB = 65536KB = 64MB;三级:256³ × 1KB = 16M×1KB = 16GB。
最大文件 = 10KB + 256KB + 64MB + 16GB ≈ 16GB。

5.【成组链接法】空闲块管理用成组链接法,每 3 块为一组。当前 S.free = [2], 5, 6(2 个空闲块 5、6,栈顶在右)。依次:① 回收第 7 块 ② 回收第 8 块 ③ 分配 2 块。写出每步后的 S.free 和空闲总数。
参考答案
① 回收7:栈未满(2<3),压入 → [3]5,6,7,总数 3(此时满)。
② 回收8:栈已满(3)!把 5,6,7 写入第8块,S.free 只剩 → [1]8,总数 4。
③ 分配第1块:只剩1块,第8块存着下一组→先读出 5,6,7 到 S.free,再分配8 → [3]5,6,7,总数 3。
   分配第2块:弹栈顶7 → [2]5,6,总数 2。
最终 S.free = [2]5,6,空闲总数 2;依次分配出 8、7。

四、设计题(10 分)

1. 有一个可存放 n 个产品的缓冲池,多个生产者进程不断生产产品放入缓冲池,多个消费者进程不断从缓冲池取产品消费。要求:① 定义所需信号量及其初值;② 用 P/V 操作写出生产者和消费者的同步互斥算法;③ 说明两个 P 操作的顺序为什么不能颠倒。
参考答案
① 信号量:mutex=1(缓冲池互斥)、empty=n(空缓冲区数)、full=0(满缓冲区/产品数)。 
producer(){              consumer(){
  生产一个产品;            P(full);    // 申请产品
  P(empty); // 申请空位     P(mutex);   // 进缓冲池
  P(mutex); // 进缓冲池     从缓冲池取产品;
  放入缓冲池;              V(mutex);
  V(mutex);               V(empty);   // 多一个空位
  V(full);  // 多一个产品  }
}
③ 必须先 P(资源信号量 empty/full),再 P(mutex)。若颠倒(先 P(mutex) 再 P(empty/full)):例如缓冲池空时消费者先拿到 mutex(钥匙),再 P(full) 因无产品而阻塞,但它仍占着 mutex;此时生产者想放产品却 P(mutex) 拿不到钥匙也阻塞 → 双方互相等待,死锁。而两个 V 操作顺序可以颠倒,不会死锁。
