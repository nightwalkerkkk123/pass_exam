第 18 课 · 虚拟内存的坑概念 （可视化）

局部性 → 请求分页 → 缺页 → Belady 异常 → 抖动。你会算页面置换了，这一课补齐所有“为什么”。约 18 分钟。
一句话主线： 程序有局部性（一段时间只用一小撮页）→ 所以不必全部装入，用请求分页按需调页 → 页不在内存就缺页去外存取 → 但置换没做好会踩两个坑：Belady 异常和抖动。

一、局部性原理：为什么不必全装入 📖

下面是一个程序在时间上的“页访问串”。点播放，看那个蓝色窗口滑过去——任一时刻，窗口里只有少数几个不同的页在用。这就是局部性。

当前窗口内不同页数：– 个（窗口宽 4）
结论： 既然一段时间只用到几页，内存里只放当前用到的页即可，其余留在外存（磁盘）。于是 20MB 内存能跑 100MB 程序——这就是虚拟存储器的理论基础（Denning 1968 局部性规律）。

二、请求分页：按需调页 + 页表项

请求分页 = 基本分页 ＋ 请求调页 ＋ 页面置换。页表项里多了几个位（填空高频）：

字段作用
状态位 P该页在不在内存。P=0 → 缺页，触发缺页中断去外存调入
访问字段 A记录最近被访问情况，供置换算法（如 LRU）参考
修改位 M调入后有没有被改过。改过→换出时要写回外存；没改→直接丢弃
外存地址该页在磁盘的位置，缺页时按它去取
缺页的代价极大： 有效访问时间 EAT = (1−p)·访问内存 + p·缺页处理。缺页处理要读磁盘（毫秒级），比访存（纳秒级）慢几万倍。所以缺页率 p 哪怕只有 0.1%，性能也会暴跌——这正是抖动可怕的原因（见第四节）。

三、Belady 异常：多给内存，缺页反而更多 😱

同一个访问串 1 2 3 4 1 2 5 1 2 3 4 5，都用 FIFO 置换。左边给 3 个页框，右边给 4 个页框。点“下一步”，两边同步跑，盯住缺页计数。

🟩 4 个页框

当前访问：–
缺页次数：0
进度 0/12
Belady 异常（课本叫“Belady 奇异”）： 采用 FIFO 时，给进程增加页框数，缺页率反而可能升高的反常现象。
注意：只有 FIFO 会。LRU、OPT 这类“栈式算法”不会出现 Belady 异常——这是常考的对比点。

四、抖动（Thrashing）：把时间全花在换页上 🌀

拖动滑块，调整分配给进程的页框数，看缺页率怎么变：

页框很少页框充足
缺页率：很高 · 状态：🌀 抖动

抖动（课本知识清单原话）： 页面频繁换入换出，导致系统效率急剧下降。
根因：分配给进程的页框太少，装不下它的“常用页集合”，于是刚换出的页马上又要用 → 反复换 → CPU 大部分时间在搬页、几乎不干正事。
（解决思路：按程序“工作集”——即一段时间真正在用的页集合——分配足够页框。工作集老师未重点讲，了解即可。）
