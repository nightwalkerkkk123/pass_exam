第 9 课:信号量与 PV 操作(先搞懂互斥)

目标:看懂信号量是什么、P/V 各干什么、怎么用它实现"互斥"。约 18 分钟。这是 PV 大题的地基。
问题从哪来: 两个进程同时改一个共享变量(比如都给余额 +1),可能互相覆盖,结果出错(叫"竞态")。 解决办法:让访问共享资源的那段代码一次只许一个进程进——这就是互斥。信号量就是实现它的工具。

一、两个名词(先认门)

名词意思大白话
临界资源一段时间内只允许一个进程访问的资源单间厕所(一次一人)
临界区进程中访问临界资源的那段代码"上厕所"那段动作

二、信号量 = 一个计数器 + 两个动作

信号量 S 就是一个整数,代表"还剩多少个资源可用"。它只能被两个原子操作(不可被打断)碰:

P(S) ·又叫 wait(S)·"申请"
P(S):
  S = S - 1;        // 先减 1(拿走一个)
  if (S < 0)
    阻塞,排队等;   // 没了→睡觉,让出CPU

类比:拿厕所钥匙。没钥匙就在门口排队。

V(S) ·又叫 signal(S)·"释放"
V(S):
  S = S + 1;        // 先加 1(还回一个)
  if (S <= 0)
    唤醒一个等待者;  // 有人在等→叫醒他

类比:还厕所钥匙,并喊一声"下一位"。

🔑 一句话记死: P = 申请 = 减 1(不够就排队睡);V = 释放 = 加 1(有人等就叫醒)。
口诀:"P 进 V 出","P 减 V 加"。
S 的值代表什么(常考填空):
  • S > 0:还有 S 个资源可用(几把空钥匙)。
  • S = 0:资源刚好用完,但没人在等。
  • S < 0:资源没了,而且 |S| = 正在排队等待的进程数。(S=−2 → 2 个进程在等)

三、用信号量实现"互斥"(最基本的模板)

只要一把"钥匙"——互斥信号量 mutex,初值 = 1(一开始钥匙挂在墙上,可用)。把临界区夹在 P 和 V 中间:

semaphore mutex = 1;   // 一把钥匙

进程() {
  P(mutex);        // 拿钥匙(拿不到就等)——进门
  // 临界区:访问共享资源
  V(mutex);        // 还钥匙,喊下一位——出门
}
为什么这样就互斥了?mutex 初值 1。第一个进程 P(mutex)→ mutex 变 0,进去了。 第二个进程 P(mutex)→ mutex 变 −1(<0),排队睡。等第一个 V(mutex)→ mutex 回 0,叫醒第二个。 任何时刻最多 1 个进程在临界区。
⚠ 两个必丢分的坑:
  • P 和 V 必须成对。只 P 不 V → 钥匙永远不还,后面全卡死(死锁)。
  • 互斥信号量初值必须是 1。设成 0 → 第一个进程都进不去;设成 2 → 两个人同时进,失去互斥。

四、必背(三句)

① P = 申请 = S−1(<0 则阻塞排队);V = 释放 = S+1(≤0 则唤醒一个)。

② 互斥:mutex 初值 =1,临界区夹在 P(mutex) … V(mutex) 之间。

③ S<0 时,|S| = 等待的进程数。

五、轮到你了

Q1. P(S) 操作(wait)做的是?
Q2. 用信号量实现互斥,mutex 的初值应该设为?
Q3. 现在 mutex = −2,这说明?
▸ 解释

S<0 时 |S| = 等待进程数。mutex=−2 → 1 个进程在临界区里,另外 2 个在门口排队。
