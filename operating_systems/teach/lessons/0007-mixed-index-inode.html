第 7 课:混合索引 —— 算文件能有多大

目标:给你 inode 的指针结构 + 块大小,你能算出文件最大容量。约 18 分钟。纯算术。
背景: 文件存在磁盘上,被切成一块块(block)散落各处。inode(索引节点)是一张"登记表",记录"这个文件的数据块都在哪"。 问题来了:大文件有成千上万块,inode 这张小表放不下那么多地址 → 于是发明了间接指针(套娃)。这就是本课的核心。

一、套娃:一个指针怎么管住很多块

UNIX inode 里有 13 个指针,分四种(把指针想成"寻宝图上的标记"):

直接指针 ×10 → 每个直接指 1 个数据块(地图上直接标了宝藏)
一级间接 ×1 → 指向 1 个"地址块" → 里面装 N 个地址 → N 个数据块
二级间接 ×1 → 指向地址块 → 它指向 N 个地址块 → 每个再指 N 个 → N×N 个数据块
三级间接 ×1 → 再套一层 → N×N×N 个数据块
🔑 灵魂一句: 间接指针不直接指数据,而是指向"一整块全是地址的块"。一块能装 N 个地址(N = 块大小 ÷ 一个地址的大小)。所以每多套一层,容量就 × N。

二、关键数字 N + 公式

N = 每个磁盘块能装的地址个数 = 块大小 B ÷ 地址项大小 A

四部分各能装的数据(单位:字节),全部相加 = 文件最大大小:

部分能寻址的数据块数容量(×块大小 B)
直接(D 个)DD × B
一级间接NN × B
二级间接N²N² × B
三级间接N³N³ × B

三、标准例题(UNIX,必背)

块大小 B = 4KB,地址项 A = 4B,直接 10 个,一/二/三级各 1 个。求最大文件大小。

先算 N: N = 4KB ÷ 4B = 4096 ÷ 4 = 1024(一块能装 1024 个地址)。

部分算式= 容量
直接10 × 4KB40 KB
一级间接1024 × 4KB4 MB
二级间接1024² × 4KB = 1024×1024×4KB4 GB
三级间接1024³ × 4KB4 TB
最大文件 ≈40KB + 4MB + 4GB + 4TB ≈ 4TB
看出规律没?每多一级,× 1024:4KB→(直接10个≈40KB)…一级 4MB → 二级 4GB → 三级 4TB。4TB 几乎就是答案(前三项加起来还不到它的零头)。这就是"套娃"的威力。

四、必背 + 两个坑

① N = 块大小 ÷ 地址大小(一块装几个地址)。

② 最大 = D×B + N×B + N²×B + N³×B(直接 + 一级 + 二级 + 三级,全加)。

⚠ 两个坑:
  • 间接层是乘 N(N、N²、N³),不是加 N。套娃是翻倍式放大。
  • N 是 块大小 ÷ 地址大小,别拿块大小当 N。B=4KB、A=4B 时 N=1024,不是 4096。

五、轮到你了

Q1. 块大小 1KB,每个地址 4B。一个"地址块"能装多少个地址(即 N)?
▸ 提示

N = 块大小 ÷ 地址大小 = 1024B ÷ 4B = 256。

Q2. "一级间接指针"指向的是什么?
Q3. 块大小 1KB,地址 4B(所以 N=256)。二级间接这一项能寻址多大的数据?

二级 = N² × 块大小。自己算完再点。

▸ 步骤

二级 = N² × B = 256 × 256 × 1KB = 65536 × 1KB = 65536 KB = 64 MB。
(顺带:一级=256×1KB=256KB,三级=256³×1KB=16GB。)
