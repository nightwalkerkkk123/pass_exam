第 20 课 · I/O 与设备管理 （可视化）

一条主线：让 CPU 越来越少为慢设备操心。四种 I/O 控制方式 + 缓冲 + SPOOLing。约 16 分钟。
核心矛盾： CPU 极快、I/O 设备极慢。设备管理的一切技术，都是为了别让 CPU 干等慢设备、提高两者的并行。

一、四种 I/O 控制方式：点一下看 CPU 被解放多少

点上面四个标签切换，看 CPU 干预程度（越短越好）和 并行程度（越长越好）怎么变。这是一条进化线。

① 程序直接控制轮询 / 忙等
② 中断驱动完成才打扰
③ DMA整块自己搬
④ 通道一组块自己跑

怎么工作： CPU 不停地查询设备状态位“好了没？好了没？”，在设备完成前一直死等（忙等 busy-waiting），什么别的都干不了。

课本例子： 从终端输入一个字符约 100ms，CPU 约有 99.9ms 在忙等中空转。

CPU 干预程度（越短越好）
CPU 与 I/O 并行程度（越长越好）

数据单位：字

怎么工作： CPU 启动 I/O 后去干别的；设备完成时发一个中断，CPU 才停下来处理一下（取走数据），然后接着干别的。

关键： 每传输一个字就中断 CPU 一次。比起忙等，CPU 利用率“成百倍”提高。

CPU 干预程度（越短越好）
CPU 与 I/O 并行程度（越长越好）

数据单位：字

怎么工作： 加一个 DMA 控制器。数据直接在设备↔内存之间成块传送，全程不经过 CPU；只在一个数据块的开头和结尾才需要 CPU 干预（启动、收尾中断）。

关键： 数据单位是块，不再是字。DMA 传输期间“窃取”总线周期，CPU 可并行做别的。

CPU 干预程度（越短越好）
CPU 与 I/O 并行程度（越长越好）

数据单位：数据块

怎么工作： 通道是一种专门的 I/O 处理机。CPU 只需发一条 I/O 指令（给出通道程序首址），通道就自己执行“通道程序”，完成一组数据块的传输与控制。

关键： 是 DMA 的发展，干预单位从“一个块”升级到“一组块”。可实现 CPU、通道、设备三者并行。

CPU 干预程度（越短越好）
CPU 与 I/O 并行程度（越长越好）

数据单位：一组数据块

一句话记牢这条线： CPU 干预 每个字 → 每次完成 → 每个块 → 每组块，越来越少；并行度越来越高。
最常考的分界：中断（单位=字） vs DMA（单位=块）。

二、缓冲技术：为什么要在中间放个“缓冲区”

引入缓冲的三个目的（课本原话）：
① 缓和 CPU 与 I/O 设备速度不匹配的矛盾；
② 减少对 CPU 的中断频率，放宽中断响应时间限制；
③ 提高 CPU 和 I/O 设备之间的并行性。

就像水池：水龙头（快）和下水道（慢）速度不一致，中间放个水池（缓冲区）缓冲一下，两边就不用互相干等。

类型特点
单缓冲一个缓冲区，输入和提取串行，处理一个数据时间 = max(C,T)+M
双缓冲两个缓冲区轮流用（缓冲对换），输入与“传送+计算”可并行，更快
循环缓冲多个缓冲区组成循环队列
缓冲池公用缓冲区池，多进程共享（空/输入/输出三种队列）

三、设备分类 + SPOOLing（假脱机）

设备按共享属性分三类： 独占设备（一段时间只许一个用户用，如打印机）、共享设备（可并发多用户用，如磁盘）、虚拟设备（用虚拟技术把一个物理设备变成多个逻辑设备）。

SPOOLing（假脱机）就是制造“虚拟设备”的技术：打印机本是独占设备，多个进程抢着打会乱。SPOOLing 让每个进程先把要打印的内容写到磁盘的输出井（排队），再由专门的程序一个个真正送去打印。

效果： 在用户看来，好像人人都有一台打印机（虚拟设备），其实底层只有一台被排队共享。SPOOLing 把独占设备改造成了可共享的虚拟设备。
