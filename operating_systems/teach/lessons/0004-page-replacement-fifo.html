第 4 课:页面置换 —— 缺页 + 画表法 + FIFO

目标:给你一个访问串 + 几个物理块,你能画表数出 FIFO 的缺页次数。约 15 分钟。
先接上第 3 课: 内存装不下一个进程的所有页,所以只把一部分页放进内存(这就是"虚拟存储")。 当程序要访问的页不在内存时 → 叫 缺页(page fault),得从磁盘把它调进来。 如果内存的物理块满了,就必须踢一页出去给新页腾位置——踢哪一页?这就是"页面置换算法"。

一、比喻:一张只放得下 3 本书的小书桌

你的书桌只能同时摊开 3 本书(= 3 个物理块)。你按一个顺序要用一本本书(= 访问串)。规则:

  • 要用的书已经在桌上 → 命中(不缺页),直接用。
  • 要用的书不在桌上 → 缺页,得去书架拿。桌子没满就直接放上;桌子满了,得先收走一本。
🔑 FIFO 规则(先进先出):桌子满了要收书时,收走"在桌上待得最久"的那本(最早放上去的)。跟它最近用没用无关,只看"谁先来的"。

二、画表法(所有页面置换题都用它)

例题:物理块 = 3 个,访问串 = 1 2 3 1 4 2 5。用 FIFO,求缺页次数。

画一张表:每一列是一次访问,三行是三个物理块。每次访问就问一句:这页在桌上吗?

访问1231425
块11111144
块2222225
块333333
结果缺缺缺命中缺命中缺

逐列读一遍:

  1. 访问 1:桌空,放入 → 缺
  2. 访问 2:不在,放入 → 缺
  3. 访问 3:不在,放入(桌满了) → 缺
  4. 访问 1:已在桌上 → 命中!(FIFO 命中时啥也不动)
  5. 访问 4:不在,桌满 → 收走待最久的 1(它最早放的),放入 4 → 缺
  6. 访问 2:已在桌上 → 命中!
  7. 访问 5:不在,桌满 → 收走现在待最久的 2,放入 5 → 缺
数一下"缺"的列:第 1、2、3、5、7 次 → 缺页 5 次,命中 2 次。
缺页率 = 缺页次数 ÷ 总访问次数 = 5 ÷ 7 ≈ 71%。
⚠ FIFO 的"待最久"怎么判断? 记住每个页是第几次被放进来的(进入顺序),收书时收"进入最早"的那个。 命中不改变进入顺序——第 4 步访问 1 命中,但 1 的"资历"不变,所以第 5 步它仍是最老的、被收走。这是 FIFO 和后面 LRU 最大的区别。

三、必背(两句)

① 缺页 = 要访问的页不在内存;命中 = 已在内存。缺页率 = 缺页次数 / 总访问次数。

② FIFO = 桌满时收走"进入内存最早"的页;命中不改变资历。
