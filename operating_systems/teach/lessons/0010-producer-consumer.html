第 10 课:生产者-消费者问题

目标:会写生产者/消费者的 PV 代码,讲清两个 P 的顺序为什么不能反。约 20 分钟。
场景: 一个能放 n 件货的货架(缓冲区)。生产者不停往里放货,消费者不停取货。三条约束:
① 货架满了,生产者得等(没空位);② 货架空了,消费者得等(没货);③ 同一时刻只能一个人动货架(互斥)。

一、三个信号量,各管一件事

信号量初值管什么类比
mutex1货架互斥(一次一人动)货架的一把钥匙
emptyn还有几个空位(生产者关心)"空位"牌,初始 n 个
full0已有几件货(消费者关心)"有货"牌,初始 0 个
记忆:empty 初值 = n(一开始全是空位),full 初值 = 0(一开始没货),mutex = 1(一把钥匙)。 生产者消耗 empty、产出 full;消费者消耗 full、产出 empty。一减一加,刚好对称。

二、两段代码(对称着记)

生产者(放货)

producer() {
  生产一件产品;
  P(empty);   // 申请1个空位(满了就等)
  P(mutex);   // 拿钥匙,进货架
  把产品放入缓冲区;
  V(mutex);   // 还钥匙
  V(full);    // 货物 +1,通知消费者
}

消费者(取货)

consumer() {
  P(full);    // 申请1件货(没货就等)
  P(mutex);   // 拿钥匙,进货架
  从缓冲区取出产品;
  V(mutex);   // 还钥匙
  V(empty);   // 空位 +1,通知生产者
}
对称规律(背这个就不会写错):
  • 生产者:P(empty) → P(mutex) → 放 → V(mutex) → V(full)
  • 消费者:P(full) → P(mutex) → 取 → V(mutex) → V(empty)
  • 两者只是把 empty↔full 对调。先 P 资源、再 P 钥匙;先还钥匙、再 V 资源。

三、💀 全题唯一的死亡陷阱:两个 P 的顺序

P(资源) 必须在 P(mutex) 之前!绝不能反!
也就是:必须 先申请空位/货(empty/full),再拿钥匙(mutex)。反过来会死锁。

看反过来会怎样——假设消费者写成 P(mutex); P(full);(先拿钥匙再申请货),而此刻货架是空的:

步消费者生产者结果
1P(mutex):拿到钥匙(mutex=0)—消费者进了货架
2P(full):没货(full=0→−1)→睡—消费者攥着钥匙睡着了
3(睡)P(empty):有空位,OK生产者想放货
4(睡)P(mutex):钥匙被攥着(mutex=0→−1)→睡生产者也睡了
死锁! 消费者攥着钥匙等"货",生产者等"钥匙"才能放货送来"货"——互相死等,谁也动不了。
正确写法(先 P(full) 再 P(mutex)):货架空时消费者卡在 P(full),但它还没拿钥匙,生产者照样能拿钥匙放货 → 不死锁。
⚠ 记牢: 两个 P 的顺序要命(资源在前、mutex 在后);两个 V 的顺序无所谓(还钥匙/通知,先后都不会死锁)。

四、必背(三句)

① 三信号量:mutex=1,empty=n,full=0。

② 生产者 P(empty)→P(mutex)→放→V(mutex)→V(full);消费者把 empty/full 对调。

③ P(资源) 一定在 P(mutex) 前,反了会死锁;V 顺序随意。

Q3. 如果把消费者写成 P(mutex); P(full); …(先拿钥匙再申请货),可能发生什么?
▸ 回顾死锁推演

货架空时:消费者 P(mutex) 拿钥匙 → P(full) 没货睡着(还攥着钥匙)→ 生产者 P(mutex) 拿不到钥匙也睡 → 互相死等。所以 P(full) 必须在 P(mutex) 前。
