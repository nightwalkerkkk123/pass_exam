第 24 课 · 哲学家就餐问题

5 个哲学家、5 根筷子、为什么"全拿左边"必死锁、3 种解法对比、PV 代码、动手模拟。20 分钟装完。
一句话：5 个哲学家围坐圆桌，每人两边各 1 根筷子，吃面要 2 根。每人重复「想 → 拿左边 → 拿右边 → 吃 → 放下」——
如果所有人同时先拿左边，每人手里 1 根、再等右边的，而右边也在别人手里 → 所有人永远等下去（死锁）。

二、为什么「全拿左边」必死锁？

死锁有 4 个必要条件（背过默写过吗？）。我们逐条对号入座：

死锁条件在本题如何成立
互斥一根筷子同一时刻只能被 1 个哲学家使用
请求和保持每个哲学家拿到左筷子后，不放下，继续请求右边
不剥夺哲学家 A 不能强行从 B 手里夺走筷子
环路等待P0 等 P1 的右筷子 → P1 等 P2 的右筷子 → … → P4 等 P0 的右筷子 → 绕回
4 个条件齐活 → 死锁一定发生。所以解法的核心思路 = 破坏其中至少一个条件（最常破坏的是环路等待和请求和保持）。

三、3 种解法对比（考试最爱考）

解法一 · 限制最多 4 人同时入座

设置资源信号量 room = 4，进餐前 P(room)、离开后 V(room)。最多 4 人抢 5 根筷子，至少 1 人能拿到 2 根。

破的是：间接削弱环路等待（5 人变 ≤4 人，等不到完整的环）。

解法二 · 原子地拿 2 根

设置互斥信号量 mutex = 1，把「拿左+拿右+检查」整体锁住，要么 2 根都拿到，要么 1 根都拿不到。

破的是：请求和保持（不会发生「拿 1 根等 1 根」的状态）。

解法三 · 奇偶分策略（最常考）

奇数号哲学家先拿右边、再拿左边；偶数号相反：先拿左边、再拿右边。这样不可能形成环路等待。

破的是：环路等待（最干净，资源利用率最高）。

三种解法的取舍（考试简答题模板）

解法破哪个条件并发度实现难度评价
限 4 入座削弱环路等待中简单牺牲 1 人换不死锁
原子拿 2 根请求和保持低中临界区大,影响效率
奇偶分 ✅环路等待高中最常推荐,资源利用率最高

四、PV 代码实现（解法三 · 奇偶分）

5 根筷子用 5 个信号量，初值都是 1：

// 信号量定义
semaphore chopstick[5] = {1, 1, 1, 1, 1};   // 5 根筷子

// 哲学家 i 的行为（i = 0..4）
void philosopher(int i) {
    while (true) {
        think();                                // 思考
        if (i % 2 == 0) {
            // 偶数：先左后右
            P(chopstick[i]);                    // 拿左边
            P(chopstick[(i+1) % 5]);         // 拿右边
        } else {
            // 奇数：先右后左（顺序相反！）
            P(chopstick[(i+1) % 5]);         // 拿右边
            P(chopstick[i]);                    // 拿左边
        }
        eat();                                  // 吃面
        V(chopstick[i]);                        // 放下左边
        V(chopstick[(i+1) % 5]);             // 放下右边
    }
}
为什么奇偶分能破死锁？设想奇数号哲学家都在等右筷子，偶数号都在等左筷子——
→ 偶数号已经拿着左筷子（也就是奇数号要等的右筷子的左邻居）……总之，不可能形成 P0→P1→…→P4→P0 的闭环，因为相同奇偶的哲学家拿筷顺序相反。
破坏环路等待 = 死锁 4 条件少一个 → 死锁不可能发生。

解法一（限 4 入座）的 PV 速写

semaphore room = 4;                    // 最多 4 人入座
semaphore chopstick[5] = {1,1,1,1,1};
void philosopher(int i) {
    while (true) {
        think();
        P(room);                                // 申请入座
        P(chopstick[i]);
        P(chopstick[(i+1) % 5]);
        eat();
        V(chopstick[i]);
        V(chopstick[(i+1) % 5]);
        V(room);                                // 离开
    }
}

五、亲手看死锁怎么发生（点上面红按钮）

圆桌上每个哲学家用一个圆点表示，颜色 = 状态：蓝(思考) / 黄(饥饿等) / 绿(吃) / 红(死锁卡死)。每根筷子是短线段，绿=空闲、橙=被占。

预期实验结果：点 「全先拿左边」 → 5 位哲学家几乎同时左手拿到一根筷子，进入「等待右边」状态，所有人都在等、手里都握着 → 圆桌变红色 = 死锁。
然后点 「奇偶分策略」（重置后）→ 偶数先拿左、奇数先拿右 → 总有一位能同时拿到 2 根 → 吃面成功，无死锁。
