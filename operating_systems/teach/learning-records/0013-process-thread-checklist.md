## A. 进程三态 + 转换(第一拍,可视化中)
- [ ] A1 三态:运行(占CPU)/就绪(只差CPU)/阻塞(等事件,给CPU也没用)
- [ ] A2 四条合法转换:就绪→运行(调度)、运行→就绪(时间片)、运行→阻塞(等事件)、阻塞→就绪(事件到)
- [ ] A3 **两条不可能:阻塞→运行、就绪→阻塞**(期末选择高频)
- [ ] A4 阻塞→就绪 而非 阻塞→运行 的原因:得到资源≠得到CPU,须经就绪中转
- [ ] A5 就绪 vs 阻塞 根本区别:就绪只缺CPU(可立即跑)/阻塞缺事件(给CPU也跑不了)

## B. 进程 vs 线程(第二拍,公司/员工比喻)
- [x] B1 进程=资源分配的基本单位;线程=CPU调度/执行的基本单位(用户独立选出最完整选项,掌握)
- [x] B2 同一进程的多个线程共享地址空间/资源,各有自己的栈+PC
- [x] B3 线程切换比进程切换开销小(不换地址空间)
- [x] B4 为什么引入线程:进程切换贵(换地址空间)、创建进程贵、并发粒度粗
