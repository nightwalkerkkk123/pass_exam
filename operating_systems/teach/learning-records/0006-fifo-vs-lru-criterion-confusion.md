# 高价值纠错(已解决):学习者把 FIFO 的淘汰标准说成了 LRU 的(进入时间 vs 访问时间)

> **2026-06-24 已纠正确认**:出"FIFO踢哪/LRU踢哪"定向题(最老页=刚用过、最旧访问页=另一页),她答 FIFO→A、LRU→B,两者踢不同页 → 判据已焊开。FIFO/LRU 混淆解决。


L5 通关②,学习者解释"为何第6次访问1时FIFO缺页"时说「FIFO 无脑淘汰**最久没有访问的**」——这把 FIFO 说成了 LRU。FIFO 标准是「进入内存最早」(出生顺序),LRU 才是「最久没被访问」(上次露面)。

**为什么高价值**：这是页面置换最经典的混淆,直接决定计算题对错。她 ①(LRU 定义)③(Belady=FIFO)都对,唯独 FIFO 的判据和 LRU 串了——说明她记住了"LRU 看访问",却没把"FIFO 看进入"立成对立面。
**纠正口径**：FIFO=看「出生顺序」(进入内存时间);LRU=看「上次露面」(最后访问时间)。那道题页1出生最早(FIFO嫌老→踢)但刚露过面(LRU护活跃→留)。
**Implications**：
- 已当场用对比表 + "出生顺序 vs 上次露面"助记纠正,并出一道定向确认题(FIFO 踢哪个)。需她复述确认后再进下一题型。
- 以后凡涉及 FIFO/LRU 同题对比,要她先口述两者"看的时间不同"再动手。
