## A. 死锁是什么 + 四个必要条件
- [x] A1 死锁 = 一组进程互相等待对方资源、永远推进不了(十字路口四车成圈)
- [x] A2 四个必要条件:互斥 / 请求和保持 / 不剥夺 / 环路等待

## B. 四种处理策略(第二拍)
- [x] B1 预防:事前破坏四条件之一(一次全分配/得不到就释放/按序申请),严格、利用率低
- [x] B2 避免:不破坏条件,运行时动态判断"是否进入不安全状态" = 银行家算法
- [x] B3 **预防 vs 避免 的区别**(用户自述正确:预防破坏条件,避免动态预演绕开不安全状态)
- [x] B4 检测:允许死锁发生,用资源分配图检测(死锁定理=图不可完全简化)
- [x] B5 解除:剥夺资源 / 撤消进程
- [x] B6 银行家算法在全景里的位置 = "避免"(不是预防)— 用户独立答对
