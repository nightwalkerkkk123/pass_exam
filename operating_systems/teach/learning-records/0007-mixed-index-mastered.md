# L7 通过:掌握混合索引文件最大大小计算(第 3 个计算题型)

学习者通过 L7:理解套娃式间接索引(一级/二级/三级 = ×N / ×N² / ×N³),③答对"4TB 来自三级间接",并在纠正后用对 N 的定义。

**纠正点(中价值)**：她初说"N 是一块能存放的地址**大小**"——"大小"措辞危险(易与地址项大小 A 混)。已澄清 **N=个数(B÷A)，A=字节大小**,并出定向题(B=2KB,A=4B → N=?),她答 512,确认 N 的"个数"含义清楚、N≠A。
**Implications**：
- 三个计算题型(磁盘调度/页面置换/混合索引)已掌握。仅剩 **PV 操作**。
- PV 是难度断层:需先搭 进程同步/临界区/信号量/P-V 语义(她有"程序vs进程",无"临界区")。须拆 2 课:L8=信号量+互斥基础,L9=生产者-消费者。不能像前三个那样一课速通。
- 她"个数 vs 大小""出生 vs 露面"这类措辞精度问题反复出现 → 教 PV 时对"P=申请/减、V=释放/加""谁在临界区内/外"等用词要逐字校准。
