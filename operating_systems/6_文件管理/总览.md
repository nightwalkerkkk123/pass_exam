# 一、文件和文件系统

## 1. 什么是文件？

文件是：

> **具有文件名的一组相关信息的集合。**

比如：

```text
一篇文章
一个程序
一张表格
一组数据
一个图片
一个视频
```

只要它是一组有独立意义的信息，并且可以被命名、保存、访问，就可以组织成文件。

文件有两个重要特点：

```text
可保存性：信息可以长期保存在磁盘上
按名存取：用户只需要知道文件名，不需要知道文件具体在哪个磁盘块
```

------

## 2. 文件的两种类型

### 有结构文件

有结构文件由一条条**记录**组成。

关系是：

```text
文件 → 记录 → 数据项
```

例如一个班级学生信息文件：

```text
整个班学生资料 = 文件
每个学生的信息 = 一条记录
姓名、性别、学号、出生年月 = 数据项
```

也就是：

```text
学生文件
├── 记录1：张三，男，2004年，001
├── 记录2：李四，女，2005年，002
└── 记录3：王五，男，2004年，003
```

------

### 无结构文件

无结构文件也叫**流式文件**。

它直接由字符或字节组成。

例如：

```text
文本文件
图片文件
音频文件
视频文件
```

UNIX、DOS、Windows 大多采用流式文件思想。

简单理解：

```text
有结构文件：一条一条记录
无结构文件：一串连续字节流
```

------

## 3. 什么是文件系统？

文件系统是操作系统中负责文件管理的部分。

它的主要功能有：

```text
分配和回收磁盘空间
实现文件名到磁盘空间的映射
提供文件共享
提供文件保护和保密
实现创建、删除、读、写、打开、关闭等文件操作
```

文件系统提供两类接口：

| 接口     | 作用                 | 例子                             |
| -------- | -------------------- | -------------------------------- |
| 命令接口 | 用户直接使用         | `dir`、`ls`、`cd`                |
| 程序接口 | 程序通过系统调用使用 | `open`、`read`、`write`、`close` |

------

# 二、文件的逻辑结构

文件结构分成两种：

```text
逻辑结构：用户看到的文件内部组织方式
物理结构：文件在磁盘上的实际存储方式
```

这一节主要讲逻辑结构。

------

## 1. 顺序文件

顺序文件就是记录按顺序排列。

有两种组织方式：

```text
串结构：记录顺序和关键字无关
顺序结构：记录按关键字排列
```

例如按学号排序的学生表，就是顺序结构文件。

### 定长记录

每条记录长度一样，可以直接定位。

例如每条学生记录固定 100 字节，要找第 10 条：

```text
起始位置 + 10 × 100
```

优点是查找方便。

------

### 变长记录

每条记录长度不一样。

例如：

```text
姓名长度不同
地址长度不同
备注长度不同
```

这时不能直接跳到某条记录，通常要顺序扫描。

------

### 顺序文件的缺点

```text
查找单个记录比较慢
插入记录困难
删除记录困难
变长记录处理麻烦
```

------

## 2. 索引文件

索引文件的思想是：

> 给文件建立一张索引表，通过索引快速找到记录。

索引表中一般保存：

```text
关键字
记录地址
```

比如：

```text
学号 001 → 记录地址 A
学号 002 → 记录地址 B
学号 003 → 记录地址 C
```

优点：

```text
查找速度快
适合随机访问
```

缺点：

```text
索引表需要额外空间
维护索引有开销
```

------

## 3. 索引顺序文件

索引顺序文件是顺序文件和索引文件的折中。

它不是给每一条记录建索引，而是：

> 把记录分组，只给每组第一个记录建立索引。

例如 100 万条记录：

```text
顺序文件平均查找：约 50 万次
索引文件：查找快，但要 100 万个索引项
索引顺序文件：分组后索引更少，查找也比较快
```

课件里的例子是：

```text
100 万条记录
每 100 条为一组 → 10000 个组
每 100 个组为一群 → 100 个群
查找过程：群索引 → 组索引 → 组内记录
大约 50 + 50 + 50 = 150 次
```

这个例子很重要，用来说明索引顺序文件兼顾了**查找速度**和**索引空间开销**。

------

# 三、外存分配方式

外存分配方式就是：

> 一个文件的数据块在磁盘上怎么存放。

主要有三种：

```text
连续分配
链接分配
索引分配
```

------

# 1. 连续分配

连续分配是指：

> 给文件分配一组连续的磁盘块。

例如文件 A 占用：

```text
0, 1, 2, 3, 4
```

目录中只需要记录：

```text
文件名
起始块号
文件长度
```

例如：

| 文件名 | 起始块 | 长度 |
| ------ | ------ | ---- |
| A      | 0      | 2    |
| B      | 2      | 9    |
| C      | 11     | 5    |

------

## 连续分配的优点

```text
顺序访问快
随机访问方便
实现简单
```

比如文件起始块是 `ADDR`，要访问第 `I` 个逻辑块：

```text
物理块号 = ADDR + I
```

------

## 连续分配的缺点

```text
需要连续磁盘空间
会产生外部碎片
文件长度需要事先知道
文件扩展困难
```

所以连续分配适合：

```text
长度固定
顺序访问多
不经常增长的文件
```

------

# 2. 链接分配

链接分配是指：

> 文件的各个磁盘块可以分散存放，每个块指向下一个块。

例如：

```text
文件 A：0 → 10 → 3 → 结束
```

------

## 2.1 隐式链接

隐式链接是把指针直接放在数据块里。

目录中记录：

```text
文件名
开始块
结束块
```

每个数据块中保存下一个块号。

------

### 隐式链接优点

```text
没有外部碎片
不需要连续空间
不需要事先知道文件长度
文件容易扩展
```

------

### 隐式链接缺点

```text
只适合顺序访问
随机访问效率低
可靠性差，一个指针坏了后面全断
指针占用数据区空间
```

比如要访问第 1000 块，必须从第一块一路找过去。

------

## 2.2 显式链接：FAT

显式链接是把所有链接指针集中放在一张表里。

这张表叫：

```text
FAT：File Allocation Table，文件分配表
```

FAT 表中每个表项对应一个磁盘块，表项内容表示下一个块号。

例如：

```text
FAT[0] = 10
FAT[10] = 3
FAT[3] = -1
```

说明文件链是：

```text
0 → 10 → 3 → 结束
```

------

### FAT 中常见标记

```text
-1：文件结束
-2：坏块
0：空闲块
其他数字：下一块块号
```

------

### FAT 的优点

```text
查找比隐式链接快
FAT 可调入内存
没有外部碎片
不需要连续空间
文件扩展方便
```

------

### FAT 的缺点

```text
FAT 表占用外存空间
大磁盘时 FAT 很大
FAT 也会占用较多内存
直接访问仍不如索引分配高效
```

课件中举例：

```text
1GB 硬盘
块大小 512B
约 2M 个盘块
每个盘块号约 3B
FAT 约需要 6MB 空间
```

如果用簇，比如 1 簇 = 8KB，则 FAT 空间可以减少。

------

# 3. 索引分配

索引分配的思想是：

> 把文件所有数据块号集中放在索引块中。

目录中记录文件对应的索引块号。

索引块中记录：

```text
文件第 0 块在哪
文件第 1 块在哪
文件第 2 块在哪
……
```

------

## 单级索引

每个文件分配一个索引块。

优点：

```text
支持直接访问
没有外部碎片
文件不需要连续空间
```

缺点：

```text
索引块可能装不满，浪费空间
大文件时一个索引块不够
```

------

## 二级索引

二级索引是：

```text
主索引块 → 二级索引块 → 数据块
```

如果：

```text
盘块大小 = 1KB
每个盘块号 = 4B
一个索引块可存 256 个块号
```

则二级索引最大支持：

```text
256 × 256 = 64K 个盘块
64K × 1KB = 64MB
```

课件中专门强调了这个计算。

------

## 混合索引

混合索引把多种索引方式结合起来：

```text
直接地址
一次间接地址
二次间接地址
三次间接地址
```

这就是 UNIX inode 常用思想。

课件中的设定：

```text
每块 4KB
每个盘块地址 4B
所以一个索引块可存 4KB / 4B = 1K 个盘块号
```

------

### 直接地址

如果有 10 个直接地址项：

```text
最大文件 = 10 × 4KB = 40KB
```

适合小文件。

------

### 一次间接地址

一次间接地址相当于单级索引。

```text
最大增加 = 1K × 4KB = 4MB
```

所以：

```text
直接地址 + 一次间接地址 ≈ 40KB + 4MB = 4.04MB
```

------

### 二次间接地址

二次间接地址：

```text
一级索引块 → 二级索引块 → 数据块
```

可管理：

```text
1K × 1K × 4KB = 4GB
```

------

### 三次间接地址

三次间接地址可管理：

```text
1K × 1K × 1K × 4KB = 4TB
```

所以混合索引最大文件大约为：

```text
40KB + 4MB + 4GB + 4TB
```

------

## 三种外存分配方式对比

| 分配方式 | 优点                       | 缺点                 | 适合              |
| -------- | -------------------------- | -------------------- | ----------------- |
| 连续分配 | 顺序访问快，随机访问方便   | 外部碎片，扩展困难   | 长度固定文件      |
| 链接分配 | 无外部碎片，扩展方便       | 随机访问差，可靠性差 | 顺序访问文件      |
| FAT      | 管理方便，查找较快         | FAT 占空间           | 中小型文件系统    |
| 索引分配 | 随机访问好，扩展较方便     | 索引块占空间         | 大文件、随机访问  |
| 混合索引 | 小文件省空间，大文件也支持 | 实现复杂             | UNIX/Linux 类系统 |

------

# 四、目录管理

目录管理是文件系统的核心之一。

目录的作用是：

> **实现按名存取。**

用户输入文件名，文件系统通过目录找到文件的 FCB 或 inode，再找到文件数据所在磁盘块。

------

## 1. 文件控制块 FCB

FCB 是描述和控制文件的数据结构。

它和文件一一对应。

FCB 中通常包含三类信息。

------

### 基本信息

```text
文件名
文件物理位置
文件逻辑结构
文件物理结构
文件大小
```

------

### 存取控制信息

```text
文件主权限
核准用户权限
一般用户权限
```

例如：

```text
只读
读写
执行
禁止访问
```

------

### 使用信息

```text
创建时间
最后修改时间
最后访问时间
当前使用状态
```

------

## 2. FCB 和目录项的关系

课件里强调：

```text
FCB 是一个目录项
FCB 的集合组成目录
```

也就是说，目录本质上就是很多 FCB 组成的表。

------

## 3. 为什么引入索引结点 inode？

如果目录项里直接放完整 FCB，目录项会很大。

比如：

```text
FCB = 64B
盘块 = 1KB
一个盘块只能放 16 个 FCB
目录中有 640 个文件
需要 40 个盘块
平均查找要访问 20 个盘块
```

这样查找文件名很慢。

------

## 4. inode 思想

把文件名和文件描述信息分开。

目录项只保存：

```text
文件名
inode 编号 / inode 指针
```

文件详细信息放在 inode 中。

这样目录项变小，目录查找更快。

课件例子：

```text
目录项 = 16B
其中 14B 文件名，2B 指针
1KB 盘块可放 64 个目录项
640 个文件只需要 10 个盘块
平均查找 5 个盘块
```

比 FCB 直接放目录里快很多。

------

## 5. 单级目录

整个系统只有一张目录表。

优点：

```text
简单
```

缺点：

```text
查找慢
不允许重名
不便于共享
```

------

## 6. 两级目录

两级目录结构：

```text
主文件目录 MFD
用户文件目录 UFD
```

每个用户有自己的目录。

优点：

```text
提高检索速度
不同用户可以有同名文件
方便不同用户共享文件
```

缺点：

```text
用户不能继续建立子目录
扩展性不够
```

------

## 7. 树型目录

树型目录是最常用的目录结构。

特点：

```text
允许多级子目录
结构清晰
扩展性好
支持路径名
```

例如：

```text
/home/user/data/a.txt
```

------

## 8. 路径

路径分两类：

```text
绝对路径
相对路径
```

### 绝对路径

从根目录开始。

例如：

```text
/usr/local/bin
```

### 相对路径

从当前目录开始。

例如：

```text
../data/a.txt
```

------

## 9. 当前目录

如果每次访问文件都从根目录开始，会很麻烦。

所以系统设置当前目录。

例如当前目录是：

```text
/home/artur
```

访问：

```text
a.txt
```

默认就是：

```text
/home/artur/a.txt
```

------

## 10. 文件头数组

课件里有一页讲路径解析效率，提到：

> 目录文件中不应该存放完整文件头，而应该存放指向文件头的指针；文件头可以连续存放在磁盘固定位置，形成“文件头数组”。

可以这样理解：

```text
目录项：文件名 → 文件头编号
文件头数组：编号 → 文件详细信息
数据盘块：真正的数据
```

例如：

```text
<my, 82>
```

表示目录项 `my` 指向文件头数组中第 82 项。

这样做的好处是：

```text
目录项更短
路径解析更快
查找效率更高
```

------

# 五、文件存储空间管理

这一节讲：

> 磁盘上哪些块空闲？如何分配？如何回收？

主要方法：

```text
空闲表法
空闲链表法
位示图法
成组链接法
```

------

# 1. 空闲表法

为空闲区建立一张表。

表项包括：

```text
第一空闲盘块号
空闲盘块数
```

例如：

| 序号 | 第一空闲盘块号 | 空闲盘块数 |
| ---- | -------------- | ---------- |
| 1    | 2              | 4          |
| 2    | 9              | 3          |
| 3    | 15             | 5          |

------

## 分配算法

可以用动态分区类似算法：

```text
首次适应
循环首次适应
最佳适应
最坏适应
```

------

## 回收算法

如果释放的盘区和已有空闲盘区相邻，需要合并。

例如：

```text
原空闲区：10~19
释放区：20~25
合并为：10~25
```

空闲表法适合连续分配，也常用于对换区。

------

# 2. 空闲链表法

空闲链表法分两种：

```text
空闲盘块链表
空闲盘区链表
```

------

## 空闲盘块链表

把所有空闲盘块一个一个串起来。

分配时：

```text
从链表头部分配盘块
```

回收时：

```text
把释放的盘块挂到链表尾部
```

优点：

```text
分配和回收简单
```

缺点：

```text
链表可能很长
中间指针坏了，后面的空闲块可能丢失
```

------

## 空闲盘区链表

把连续空闲盘区作为单位链接。

优点：

```text
链表较短
```

缺点：

```text
分配和回收更复杂
需要合并相邻空闲区
指针损坏仍然危险
```

------

# 3. 位示图法

位示图用一位表示一个盘块状态。

课件中规定：

```text
0：空闲
1：已分配
```

------

## 分配过程

```text
1. 扫描位示图，找到值为 0 的位
2. 把位置信息转换成盘块号
3. 把对应位改成 1
```

如果是第 `i` 字节第 `j` 位：

```text
盘块号 n = i × 8 + j
```

------

## 回收过程

给定盘块号 `n`：

```text
i = n / 8
j = n mod 8
```

然后把第 `i` 字节第 `j` 位改为 0。

------

## 位示图法优点

```text
占用空间小
可以调入内存
分配和回收简单
效率较高
```

常用于微型机和小型机系统。

------

# 4. 成组链接法

这是你照片里那道作业的核心。

成组链接法思想：

```text
把空闲盘块每 N 个分成一组
组与组之间链接
超级块中保存当前可用的一组空闲盘块号
```

其中 `s.free` 是空闲盘块号栈。

------

## 分配规则

如果：

```text
s.free = 1
```

说明栈里只剩一个盘块号，这个盘块里保存下一组空闲盘块信息。

此时：

```text
先把该盘块内容读入空闲盘块号栈
再把该盘块分配出去
```

否则：

```text
直接分配栈中的一个盘块
s.free 减 1
```

------

## 回收规则

如果：

```text
s.free = N
```

说明当前栈满了。

此时：

```text
把当前栈内容写入新释放的盘块
然后 s.free = 1
把新释放盘块作为当前栈中的盘块
```

否则：

```text
s.free 加 1
把释放盘块号放入栈
```

------

## 成组链接法的核心优点

```text
不用把所有空闲块链在一起
减少链表过长的问题
分配和回收效率较高
适合 UNIX 类文件系统思想
```

# 六、文件共享与文件保护

文件共享就是多个用户或多个目录项访问同一个文件。

主要方式：

```text
基于索引结点的共享
符号链接共享
```

------

## 1. 基于索引结点的共享

多个目录项可以指向同一个 inode。

例如：

```text
Zhao 目录中的 A
Qian 目录中的 B
```

都指向同一个索引结点。

系统在 inode 中设置：

```text
链接计数 count
```

表示有多少个目录项共享该文件。

------

### 删除时怎么处理？

如果 Zhao 删除 A，但 Qian 还在共享 B：

```text
只删除 Zhao 目录项
count 从 2 变成 1
文件本体不删除
```

只有当：

```text
count = 0
```

文件才真正删除。

------

### 优点

```text
多个用户看到的是同一份文件
任意用户修改，其他用户可以看到
节省磁盘空间
```

------

### 缺点

```text
原文件主删除自己的目录项后，文件属主可能仍不变
可能出现计费或权限管理问题
```

这类似 Linux 的**硬链接**。

------

## 2. 符号链接

符号链接保存的是目标文件的路径名。

类似 Windows 快捷方式、Linux 软链接。

例如：

```text
link.txt → /home/user/a.txt
```

------

### 优点

```text
可以跨文件系统
可以共享网络上的文件
实现灵活
```

------

### 缺点

```text
访问速度较慢，需要沿路径查找
如果原文件被删除，符号链接会失效
```

------

# 七、磁盘容错技术

这一部分关注：

> 磁盘出错时，如何避免数据丢失。

------

## 1. 第一级容错技术 SFT-I

SFT-I 主要防止磁盘表面错误导致数据丢失。

包括：

```text
双份目录和双份 FAT
热修复重定向
写后读校验
```

------

### 双份目录和双份 FAT

目录和 FAT 是文件系统最重要的数据。

所以可以保存两份：

```text
主目录 / 主 FAT
备用目录 / 备用 FAT
```

当主目录或主 FAT 损坏时，启用备份。

------

### 热修复重定向

磁盘中设置热修复区。

如果某个盘块坏了：

```text
把数据写到热修复区
以后访问坏块时自动重定向到热修复区
```

------

### 写后读校验

写完数据后立即读出来检查。

过程：

```text
内存缓冲区 A 写入磁盘
再从磁盘读入缓冲区 B
比较 A 和 B
如果不一致，说明可能写入失败
```

如果多次失败，就认为盘块损坏，写入热修复区。

------

## 2. 第二级容错技术 SFT-II

SFT-II 主要解决整个磁盘驱动器故障。

包括：

```text
磁盘镜像
磁盘双工
```

------

### 磁盘镜像

两个磁盘连接在同一个控制器上。

每次写主盘，也写备份盘。

```text
主磁盘 = 备份磁盘
```

主盘坏了，启用备份盘。

------

### 磁盘双工

两个磁盘分别连接在两个控制器上。

比磁盘镜像更可靠，因为控制器也有备份。

------

### SFT-II 优缺点

优点：

```text
容错能力强
可靠性高
```

缺点：

```text
空间利用率只有 50%
成本高
```

------

# 八、数据一致性控制

数据一致性控制解决的问题是：

> 系统崩溃、并发访问、重复写入时，如何保证文件系统数据不乱。

------

## 1. 事务

事务是访问和修改数据的一个程序单位。

事务具有**原子性**：

```text
要么全部完成
要么全部不做
```

例如银行转账：

```text
A 扣 100
B 加 100
```

不能只扣 A，不加 B。

------

## 2. 事务记录

事务记录保存修改信息：

```text
事务名
数据项名
旧值
新值
```

事务记录必须保存在稳定存储器中。

------

## 3. Undo 和 Redo

### Undo

撤销事务，把数据恢复到旧值。

```text
Undo(Ti)：恢复事务 Ti 修改过的数据为旧值
```

### Redo

重做事务，把数据设置为新值。

```text
Redo(Ti)：把事务 Ti 应该修改的数据重新置为新值
```

------

## 4. 并发控制

如果多个进程同时访问同一个对象，需要控制顺序。

方法：

```text
互斥锁
信号量
```

互斥锁思想：

```text
访问对象前先加锁
访问完成后释放锁
```

------

## 5. 磁盘块号一致性

正常情况下，每个盘块只能处于两种状态之一：

```text
空闲盘块
数据盘块
```

两者应该互补。

课件中使用两组计数器：

```text
空闲盘块计数器
数据盘块计数器
```

------

### 正常情况

某盘块要么：

```text
空闲计数器 = 1，数据计数器 = 0
```

要么：

```text
空闲计数器 = 0，数据计数器 = 1
```

------

### 丢失盘块

如果：

```text
空闲计数器 = 0
数据计数器 = 0
```

说明这个块既不在空闲表，也不属于任何文件。

它变成了“丢失盘块”。

处理：

```text
把它加入空闲盘块管理结构
```

------

### 空闲盘块重复

如果：

```text
空闲计数器 = 2
数据计数器 = 0
```

说明同一个盘块在空闲表中出现了两次。

处理：

```text
从空闲盘块管理机制中删除一个重复项
```

------

### 数据盘块重复

如果：

```text
数据计数器 = 2
```

说明同一个盘块被两个文件同时占用。

这是严重错误。

处理：

```text
必须立即报告并修复
```

因为这可能导致两个文件互相覆盖数据。

# 十一、最容易考的计算题

## 1. 连续分配地址计算

如果文件起始块号为 `ADDR`，要访问第 `I` 个逻辑块：

```text
物理块号 = ADDR + I
```

如果给的是逻辑地址 `LA`：

```text
逻辑块号 I = LA / 块大小
块内位移 d = LA mod 块大小
```

------

## 2. 索引块容量计算

如果：

```text
盘块大小 = 4KB
盘块号大小 = 4B
```

则一个索引块能存：

```text
4KB / 4B = 1024 = 1K 个盘块号
```

------

## 3. 一级索引最大文件

```text
1K × 4KB = 4MB
```

------

## 4. 二级索引最大文件

```text
1K × 1K × 4KB = 4GB
```

------

## 5. 三级索引最大文件

```text
1K × 1K × 1K × 4KB = 4TB
```

------

## 6. 混合索引最大文件

如果有：

```text
10 个直接地址
1 个一级索引
1 个二级索引
1 个三级索引
```

则最大文件约为：

```text
10 × 4KB + 1K × 4KB + 1K × 1K × 4KB + 1K × 1K × 1K × 4KB
= 40KB + 4MB + 4GB + 4TB
```
