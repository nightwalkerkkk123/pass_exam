# 第六章 IO系统

## 6.1 IO系统概述
### IO系统的功能
IO系统的主要功能是管理输入输出设备，完成用户的IO请求，提高IO设备的利用率和IO速度，方便用户使用设备：
1. 隐藏设备的物理特性和实现细节，向上层提供统一的、抽象的设备接口
2. 提供设备独立性，用户程序不需要关心具体使用的物理设备，只需要通过设备号或者设备名访问
3. 实现设备的并发共享，协调多个进程对设备的访问，避免冲突
4. 提高IO设备和CPU的利用率，尽量使IO和CPU并行工作
5. 进行错误处理，对IO过程中出现的错误进行处理，尽可能对上层透明

### IO设备的分类
IO设备种类繁多，可以从不同角度分类：
1. **按使用特性分类**：
   - 存储设备：如磁盘、磁带、光盘等，用来存储信息，速度较慢，容量大
   - 输入输出设备：如键盘、鼠标、显示器、打印机等，用来和用户交互
   - 通信设备：如网卡、调制解调器等，用来和其他计算机通信
2. **按传输速率分类**：
   - 低速设备：传输速率在每秒几字节到几百字节，如键盘、鼠标
   - 中速设备：传输速率在每秒几千字节到几兆字节，如打印机、扫描仪
   - 高速设备：传输速率在每秒几十兆字节到几十GB，如磁盘、网卡、显卡
3. **按信息交换的单位分类**：
   - 字符设备：以字符为单位进行信息交换，一次传输一个字符，无结构，速度慢，如键盘、鼠标、串口设备等，通常采用中断驱动方式
   - 块设备：以数据块为单位进行信息交换，一次传输一个或多个数据块（通常一个块大小为512B~4KB），有结构，速度快，如磁盘、固态硬盘等，通常采用DMA方式

### IO系统的层次结构
IO系统采用分层结构设计，每一层利用下层提供的服务，向上层提供服务，隐藏下层的实现细节：
| 层次 | 主要功能 |
|------|----------|
| 用户层IO软件 | 为用户程序提供调用IO操作的接口，如库函数、系统调用的封装，处理用户的IO请求，进行参数校验，调用下层服务 |
| 设备独立性软件 | 实现设备无关性（设备独立性），负责设备命名、设备保护、设备分配与回收、块设备的缓冲管理、提供统一的逻辑块大小，屏蔽不同硬件设备的差异 |
| 设备驱动程序 | 是和具体硬件设备直接交互的软件，负责设置设备的寄存器、控制设备的工作方式、启动IO操作、处理中断、将抽象的IO请求转换为具体的硬件操作，不同的设备有不同的驱动程序 |
| 中断处理程序 | 当IO操作完成或者发生错误时，硬件产生中断，中断处理程序响应中断，唤醒等待的驱动程序，处理IO完成事件，进行错误处理 |

**IO请求的处理流程**：用户程序发起IO请求 → 用户层软件处理 → 设备独立性软件处理 → 设备驱动程序启动硬件 → 硬件完成IO操作 → 产生中断 → 中断处理程序处理 → 唤醒等待的进程 → 返回结果给用户程序

## 6.2 IO控制方式
IO控制方式是指CPU和设备之间的数据传输控制方式，随着硬件技术的发展，IO控制方式从简单到复杂，CPU的干预越来越少，并行度越来越高。

### 1. 程序IO方式（轮询方式）
- **原理**：CPU启动IO设备后，不断轮询查询设备的状态寄存器，直到设备准备好传输数据，然后CPU从设备数据寄存器读取数据或者写入数据，一次传输一个字节或者字
- **实现流程**：
  1. CPU向设备控制器发送IO命令，启动设备
  2. 设备控制器开始工作，CPU进入轮询循环，不断读取设备的状态寄存器，检查是否准备好
  3. 设备准备好后，CPU读取数据寄存器中的数据，或者将数据写入数据寄存器
  4. 设备完成操作，修改状态寄存器，CPU继续轮询下一个操作
- **优点**：实现非常简单，不需要额外的硬件支持，适合简单的嵌入式系统或者低速设备
- **缺点**：CPU利用率极低，大部分时间都在忙等，CPU和设备完全串行工作，不支持并行操作
- **适用场景**：简单的低速设备，对性能要求不高的场景

### 2. 中断驱动IO方式
- **原理**：CPU启动IO设备后，不需要等待，继续执行其他任务，当设备准备好传输数据时，向CPU发送中断请求，CPU响应中断，暂停当前任务，执行中断处理程序，完成数据传输，传输完成后回到被中断的任务继续执行
- **实现流程**：
  1. CPU向设备控制器发送IO命令，启动设备，然后继续执行其他任务
  2. 设备控制器开始工作，准备好数据后，向CPU发送中断请求
  3. CPU收到中断请求后，保存当前上下文，执行中断处理程序
  4. 中断处理程序读取设备数据寄存器中的数据，或者将数据写入数据寄存器，通知设备继续下一个操作
  5. 中断处理完成后，CPU恢复上下文，回到之前的任务继续执行
- **优点**：CPU和设备可以并行工作，CPU利用率比程序IO方式高很多，不需要忙等
- **缺点**：
  1. 每次传输一个字或者字节都需要产生中断，中断处理有一定开销，对于高速设备来说，会产生大量中断，导致CPU开销过大
  2. 中断响应和处理需要时间，对于实时性要求极高的场景可能不够快
- **适用场景**：中低速字符设备，如键盘、鼠标、串口等，一次传输的数据量小，中断次数不多，CPU开销可接受

### 3. 直接存储器访问方式（DMA, Direct Memory Access）
- **原理**：专门的DMA控制器接管总线控制权，在设备和内存之间直接进行批量数据传输，不需要CPU干预，传输过程完全由DMA控制器控制，传输完成后DMA控制器向CPU发送中断通知
- **DMA控制器的组成**：
  - 命令/状态寄存器：存放CPU发送的命令和设备状态
  - 内存地址寄存器：存放数据在内存中的起始地址，传输过程中自动递增
  - 数据寄存器：存放暂存的数据
  - 传输字节计数器：存放需要传输的总字节数，传输过程中自动递减
- **实现流程**：
  1. CPU向DMA控制器发送IO命令，包含操作类型（读/写）、内存起始地址、传输字节数、设备地址，然后CPU继续执行其他任务
  2. DMA控制器接管总线控制权，根据命令向设备控制器发送请求，启动设备
  3. 设备准备好数据后，DMA控制器将设备的数据直接写入内存指定地址，或者将内存的数据直接写入设备，不需要CPU参与，每次传输一个数据块
  4. 传输字节计数器递减到0时，DMA控制器向CPU发送中断，通知传输完成
  5. CPU响应中断，处理后续工作
- **优点**：
  1. 数据传输不需要CPU干预，完全由DMA控制器控制，CPU可以和设备完全并行工作，CPU利用率极高
  2. 一次传输整个数据块，只需要在传输开始和结束时产生一次中断，大大减少了中断次数，降低了CPU开销
  3. 传输速度快，适合高速块设备的数据传输
- **缺点**：
  1. 需要额外的DMA控制器硬件支持，增加硬件成本
  2. DMA控制器的数量有限，多个设备同时使用DMA时需要竞争DMA控制器
  3. 只能实现简单的数据传输，复杂的IO操作（如校验、错误处理）仍然需要CPU干预
  4. 大传输时占用总线时间长，可能影响其他设备的总线访问
- **适用场景**：高速块设备，如磁盘、固态硬盘、网卡等，一次传输大量数据的场景

### 4. 通道方式（Channel）
- **原理**：通道是一种专门的IO处理机，有自己的指令系统（通道指令）和程序，能够独立执行通道程序，控制多个设备和内存之间的数据传输，比DMA功能更强，可以同时控制多个不同的设备，执行复杂的IO操作序列
- **通道的类型**：
  1. 字节多路通道：按字节交叉方式工作，同时为多个低速字符设备服务，适合大量低速设备
  2. 数组选择通道：按数组块方式工作，一段时间内只能为一个高速设备服务，传输速率高，利用率低，适合高速块设备
  3. 数组多路通道：结合前两者的优点，按数组块方式工作，可以同时为多个高速设备服务，兼顾速度和利用率，是最常用的通道类型
- **实现流程**：
  1. CPU将IO请求和对应的通道程序放在内存中，然后向通道发送启动命令，通知通道执行通道程序，CPU继续执行其他任务
  2. 通道独立执行内存中的通道程序，控制多个设备完成一系列复杂的IO操作，不需要CPU干预
  3. 通道程序执行完成后，通道向CPU发送中断，通知操作完成
  4. CPU响应中断，处理后续工作
- **优点**：
  1. 通道是专门的IO处理机，IO操作完全独立于CPU，CPU和通道以及多个设备之间可以完全并行工作，CPU利用率极高，大大减轻了CPU的负担
  2. 一个通道可以控制多个设备，同时处理多个IO请求，IO效率极高
  3. 通道可以执行复杂的IO操作序列，如多块数据传输、错误处理、校验等，不需要CPU干预
- **缺点**：需要专门的通道硬件，成本很高，复杂度高
- **适用场景**：大型计算机、服务器、大型数据中心等IO操作非常频繁的场景，普通PC和移动设备一般不使用通道

### 四种IO控制方式对比
| 对比维度 | 程序IO | 中断驱动IO | DMA | 通道 |
|----------|--------|------------|-----|------|
| CPU干预程度 | 全程干预，忙等 | 每个字/字节干预一次（中断） | 每个数据块干预一次 | 整个IO序列干预一次 |
| 并行性 | 无，CPU和设备串行 | 有一定并行性 | 高并行性 | 极高并行性 |
| 数据传输单位 | 字/字节 | 字/字节 | 数据块 | 多个数据块序列 |
| 硬件成本 | 无额外硬件 | 需要中断控制器 | 需要DMA控制器 | 需要专门的通道处理机 |
| 适用场景 | 低速简单设备 | 中低速字符设备 | 高速块设备 | 大型系统，高IO负载 |

## 6.3 缓冲管理
### 缓冲的基本概念
- **定义**：缓冲是在内存中开辟的一块存储区域，用来暂存IO设备和CPU之间传输的数据，缓解设备和CPU之间速度不匹配的矛盾
- **引入缓冲的原因**：
  1. **缓和CPU和IO设备速度不匹配的矛盾**：CPU速度极快，而IO设备速度慢，没有缓冲的话，CPU需要等待设备传输，利用率低
  2. **减少对CPU的中断频率，放宽对中断响应时间的限制**：比如没有缓冲的话，每来一个字节都要中断一次，有了缓冲的话，可以攒够一个缓冲块再中断一次
  3. **提高CPU和IO设备之间的并行性**：有了缓冲后，CPU可以继续处理其他任务，设备慢慢传输数据，不需要等待
  4. **解决数据粒度不匹配的问题**：比如生产者每次生产小块数据，消费者每次消费大块数据，通过缓冲可以攒够大块再消费
- **缓冲的实现方式**：
  1. **硬件缓冲**：用专门的硬件寄存器作为缓冲，如设备控制器中的数据寄存器，速度快，容量小，成本高
  2. **软件缓冲**：在内存中开辟一块区域作为缓冲，容量大，成本低，速度比硬件缓冲慢，是最常用的缓冲实现方式

### 单缓冲和双缓冲
#### 1. 单缓冲
- **原理**：在设备和CPU之间设置一个缓冲区，当进行输入时，设备先将数据写入缓冲区，写满后CPU再从缓冲区读取数据进行处理；输出时，CPU先将数据写入缓冲区，写满后设备再从缓冲区读取数据输出
- **性能分析**：假设输入一个数据块的时间为T，CPU处理一个数据块的时间为C，数据在缓冲区和用户进程之间复制的时间为M（通常M很小可以忽略），则处理一个数据块的平均时间为max(T, C) + M
- **优点**：实现简单，提高了一定的并行性，比没有缓冲好
- **缺点**：设备和CPU仍然需要等待缓冲区空/满，并行性不够高，对于大量数据传输效率不高

#### 2. 双缓冲（缓冲对换）
- **原理**：设置两个缓冲区A和B，输入时，设备先向A写入，写满后，CPU开始读取A中的数据处理，同时设备向B写入；当B写满时，A已经被读取完，设备又开始向A写入，CPU读取B中的数据，两个缓冲区交替使用
- **性能分析**：处理一个数据块的平均时间为max(T, C)，如果C ≤ T，设备可以一直传输，不需要等待；如果T ≤ C，CPU可以一直处理，不需要等待，实现了设备和CPU的完全并行
- **优点**：并行性高，设备和CPU可以完全并行工作，效率比单缓冲高很多
- **缺点**：需要两个缓冲区，内存开销稍大，当数据的生产和消费速度波动大时，双缓冲可能不够用

### 循环缓冲
- **原理**：多个大小相同的缓冲区组成一个环形队列，有三个指针：
  1. nextg：指向消费者下一个要取的满缓冲区
  2. nextp：指向生产者下一个要放数据的空缓冲区
  3. current：指向当前正在处理的缓冲区
- 缓冲区分为三类：空缓冲区、满缓冲区、正在使用的缓冲区，生产者将数据写入空缓冲区，写满后变成满缓冲区，消费者从满缓冲区读取数据，读完后变成空缓冲区，循环使用
- **优点**：可以平滑数据生产和消费的速度波动，适合生产者和消费者速度不匹配，或者数据量波动大的场景，缓冲区利用率高
- **缺点**：实现复杂，需要指针管理和同步机制，避免生产者和消费者冲突

### 缓冲池（Buffer Pool）
- **原理**：将系统中所有的缓冲区统一管理，组成一个缓冲池，缓冲区可以供多个设备和进程共享使用，根据缓冲区的状态分为三个队列：
  1. 空缓冲队列：所有空闲的缓冲区组成的队列
  2. 输入队列：装满了输入数据的缓冲区组成的队列
  3. 输出队列：装满了输出数据的缓冲区组成的队列
- 四种操作缓冲区的过程：
  1. Getbuf：从队列中获取一个缓冲区
  2. Putbuf：将缓冲区放回对应的队列
- **优点**：缓冲区可以共享，利用率极高，灵活，可以根据需求动态分配缓冲区，适合多进程多设备的系统
- **缺点**：管理复杂，需要同步机制避免冲突，有一定的管理开销
- **适用场景**：现代操作系统广泛使用，如Linux的页高速缓存、磁盘缓冲等都是基于缓冲池实现的

## 6.4 设备分配
设备分配的任务是根据用户的IO请求，为其分配所需的设备、设备控制器、通道（如果有通道的话），保证设备能够正常工作，避免死锁。

### 设备分配的数据结构
操作系统通过以下四种数据结构记录设备的状态和信息：
1. **设备控制表（DCT, Device Control Table）**：每个设备一张，记录设备的基本信息：
   - 设备标识符、设备类型、设备状态（忙/闲）
   - 指向控制器控制表的指针
   - 等待该设备的进程队列的队首指针
   - 设备属性、参数等
2. **控制器控制表（COCT, Controller Control Table）**：每个设备控制器一张，记录控制器的信息：
   - 控制器标识符、状态
   - 指向通道控制表的指针（如果有通道）
   - 等待该控制器的进程队列指针
3. **通道控制表（CHCT, Channel Control Table）**：每个通道一张，记录通道的信息：
   - 通道标识符、状态
   - 等待该通道的进程队列指针
4. **系统设备表（SDT, System Device Table）**：整个系统一张，记录所有设备的信息，每个设备对应一个表项，包含设备标识符、设备类型、DCT地址、设备驱动程序入口地址等

### 设备分配的原则
1. **充分发挥设备的使用效率**：尽量让设备保持忙状态，提高设备利用率
2. **避免死锁**：设备分配时要避免出现循环等待的情况，防止死锁
3. **方便用户使用**：对用户屏蔽设备的物理细节，提供设备独立性，用户可以用逻辑设备名访问设备，不需要关心具体的物理设备
4. **公平性**：对所有进程公平对待，避免出现饥饿现象，优先级高的进程可以优先分配设备

### 设备分配方式
#### 按设备的固有属性分类
1. **独占分配**：设备一旦分配给某个进程，就一直由该进程独占使用，直到进程释放，适合临界资源类设备，如打印机、磁带机等
   - 优点：实现简单，不会出现冲突
   - 缺点：设备利用率低，因为设备大部分时间可能处于空闲状态
2. **共享分配**：设备可以同时分配给多个进程使用，多个进程可以并发访问设备，适合可共享的块设备，如磁盘、固态硬盘等
   - 优点：设备利用率高，多个进程可以同时使用
   - 缺点：需要复杂的调度和同步机制，避免访问冲突
3. **虚拟分配**：通过SPOOLing技术将独占设备虚拟为多个逻辑设备，供多个进程同时使用，将独占设备改造为共享设备，如打印机的假脱机打印
   - 优点：提高了独占设备的利用率，让多个进程可以同时使用独占设备
   - 缺点：需要额外的内存和磁盘空间缓冲，有一定的开销

#### 按分配时机分类
1. **静态分配**：在进程运行前就将所需的设备全部分配给进程，进程运行结束后才释放设备
   - 优点：不会出现死锁，实现简单
   - 缺点：设备利用率低，进程运行期间即使不用设备也会一直占用
2. **动态分配**：进程运行过程中需要使用设备时才提出申请，系统分配给它，使用完后立即释放
   - 优点：设备利用率高，灵活
   - 缺点：分配算法复杂，如果分配不当可能会出现死锁

### 设备分配算法
1. **先来先服务（FCFS）**：按照进程提出IO请求的先后顺序排队，先到的请求先分配设备
   - 优点：公平，实现简单
   - 缺点：没有考虑优先级，紧急任务可能需要等待很久
2. **优先级高者优先**：优先级高的进程的IO请求优先分配设备
   - 优点：可以保证紧急任务优先处理，适合实时系统
   - 缺点：低优先级的请求可能会出现饥饿现象，需要定期提升等待时间长的请求的优先级

### 设备独立性（Device Independence）
也叫设备无关性，是指用户程序独立于具体使用的物理设备：
- **实现方式**：用户程序使用逻辑设备名请求设备，而不是直接使用物理设备号，操作系统负责将逻辑设备名映射到具体的物理设备
- **优点**：
  1. 设备分配更灵活，同一个逻辑设备名可以映射到不同的物理设备，设备坏了可以切换到其他设备，不需要修改用户程序
  2. 易于实现IO重定向，输入输出的设备可以方便地改变，不需要修改程序，比如将输出从显示器改为文件只需要改逻辑设备映射
  3. 用户编程不需要关心具体的物理设备，简化了编程
- **实现需要的支持**：逻辑设备表（LUT），记录逻辑设备名到物理设备名、驱动程序入口的映射关系

## 6.5 SPOOLing技术（假脱机技术）
### 基本概念
SPOOLing是Simultaneous Peripheral Operation On-Line的缩写，即同时外围联机操作，也叫假脱机技术，是利用外存来模拟脱机输入输出，将独占设备改造为共享设备的技术。
- 脱机技术是早期的输入输出方式，用专门的外围机将输入数据从低速设备输入到磁带，然后再将磁带的数据读入主机内存，输出时先将数据写到磁带，再用外围机输出到低速设备，不需要主机干预，提高主机利用率
- SPOOLing技术是在联机情况下实现的脱机功能，不需要专门的外围机，用操作系统来模拟

### SPOOLing系统的组成
SPOOLing系统由三部分组成：
1. **输入井和输出井**：在磁盘上开辟的两个存储区域，输入井模拟脱机输入时的磁带，用来存放输入设备输入的数据；输出井模拟脱机输出时的磁带，用来存放用户进程要输出的数据
2. **输入缓冲区和输出缓冲区**：在内存中开辟的两个缓冲区，用来缓和CPU和磁盘之间的速度不匹配，输入缓冲区用来暂存从输入设备来的数据，然后再写到输入井；输出缓冲区用来暂存从输出井来的数据，然后再写到输出设备
3. **输入进程和输出进程**：两个后台进程，输入进程模拟脱机输入时的外围机，负责将输入设备的数据通过输入缓冲区写到输入井，当用户进程需要输入数据时直接从输入井读取；输出进程模拟脱机输出时的外围机，负责将用户进程要输出的数据先写到输出井，当输出设备空闲时，再将输出井的数据通过输出缓冲区写到输出设备

### SPOOLing技术的特点
1. **提高了IO速度**：用户进程的输入输出直接和磁盘上的输入井输出井交互，不需要和低速设备直接交互，速度大大提高，将对低速设备的IO转换为对磁盘的IO
2. **将独占设备改造为共享设备**：多个用户进程可以同时访问输入井和输出井，感觉自己独占了设备，实际上独占设备只由SPOOLing进程管理，按顺序处理请求，实现了共享
3. **实现了虚拟设备功能**：每个用户进程都感觉自己独占了一个设备，实际上是物理设备被虚拟为多个逻辑设备，每个用户一个

### 典型应用：假脱机打印
- 当用户进程请求打印时，SPOOLing系统并不直接将打印机分配给用户进程，而是为其在输出井中申请一块空闲磁盘块，将要打印的数据写入输出井
- 同时为用户申请一个打印请求，加入到打印队列中
- 当打印机空闲时，输出进程从打印队列中取出一个打印请求，将输出井中的数据打印出来
- 对于用户来说，感觉打印请求立即完成了，不需要等待打印机，多个用户可以同时请求打印，不会冲突

### SPOOLing的优缺点
- **优点**：大大提高了独占设备的利用率和IO速度，实现了设备共享，用户体验好
- **缺点**：需要占用大量的磁盘空间作为输入井输出井，内存中需要开辟缓冲区，有一定的系统开销，可能会出现磁盘空间不足的情况

## 6.6 设备驱动程序
### 设备驱动程序的功能
设备驱动程序是上层软件和硬件设备之间的接口，是直接和硬件交互的软件，主要功能：
1. 接收上层软件发来的抽象IO请求，如read、write命令，将其转换为具体的硬件能够理解的命令，发送给设备控制器，启动设备执行IO操作
2. 检查用户IO请求的合法性，检查设备的状态是否正常
3. 设置设备的工作参数，控制设备的工作方式，如设置串口的波特率、校验位等
4. 响应设备控制器发来的中断请求，根据中断类型进行相应的处理，如IO完成处理、错误处理等
5. 向高层软件返回IO操作的结果和状态信息

### 设备驱动程序的特点
1. **与具体硬件设备紧密相关**：不同的设备有不同的驱动程序，甚至同一型号的不同版本的设备驱动程序都可能不同
2. **与硬件的特性紧密相关**：驱动程序需要了解设备的寄存器、工作方式、通信协议等硬件细节
3. **大部分驱动程序运行在核心态**：需要直接访问硬件寄存器和内存，所以必须运行在核心态，避免用户程序随意访问硬件导致系统崩溃
4. **可以动态加载卸载**：现代操作系统支持驱动程序的动态加载，设备插入时加载驱动，拔出时卸载，不需要重启系统
5. **采用异步中断驱动方式**：驱动程序启动设备后就进入等待状态，IO完成后设备产生中断，驱动程序响应中断处理后续工作

### 设备驱动程序的处理过程
以读请求为例：
1. 接收上层发来的读请求，检查请求的合法性，如偏移量是否超过设备容量，权限是否合法等
2. 检查设备当前是否可用，如果忙就将请求挂到设备的等待队列中，睡眠等待
3. 如果设备空闲，根据请求的参数设置设备控制器的寄存器，如要读取的磁盘块号、内存地址、传输长度等，发送读命令启动设备
4. 驱动程序进入睡眠状态，等待IO完成
5. 设备完成读操作后，产生中断，中断处理程序被调用，唤醒睡眠的驱动程序
6. 驱动程序检查IO操作是否成功，有没有错误，处理错误情况
7. 将读取到的数据复制到用户进程的地址空间，返回操作结果给上层软件
