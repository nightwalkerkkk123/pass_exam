# 第一章 操作系统引论

## 1.1 操作系统的基本概念
### 操作系统的定义
操作系统是管理计算机硬件与软件资源的计算机程序，是直接运行在裸机上的最基本的系统软件，是系统软件的核心，任何其他软件都必须在操作系统的支持下才能运行。
操作系统的目标：
1. **方便性**：让计算机更容易使用，用户不需要直接操作硬件，通过操作系统提供的接口使用计算机
2. **有效性**：提高系统资源的利用率，提高系统的吞吐量，让硬件资源得到充分利用
3. **可扩充性**：方便添加新的功能和模块，支持硬件和软件的升级扩展
4. **开放性**：遵循国际标准，支持不同厂商的软硬件兼容和互操作

### 操作系统的作用
1. **用户与计算机硬件系统之间的接口**：用户不需要直接操作硬件，通过命令接口、程序接口、图形接口来使用计算机
2. **计算机系统资源的管理者**：管理四大类资源：
   - 处理机：分配和调度CPU
   - 存储器：分配和回收内存空间
   - IO设备：分配和管理输入输出设备
   - 文件：管理文件的存储、共享、保护
3. **实现对计算机资源的抽象**：隐藏硬件的具体实现细节，向上层提供统一的、方便使用的逻辑接口，比如将磁盘抽象为文件系统，将IO设备抽象为设备文件，用户不需要关心硬件的具体工作原理

### 操作系统的地位
操作系统是介于硬件和应用程序之间的一层软件，向下管理硬件资源，向上为应用程序提供运行环境和服务，是计算机系统中最核心、最基础的系统软件。

## 1.2 操作系统的发展历程
操作系统的发展是随着计算机硬件技术的发展而发展的，经历了以下几个阶段：

### 1. 手工操作阶段（无操作系统阶段）
- 时间：1946-1955年，电子管计算机时代
- 特点：
  - 没有操作系统，用户直接操作硬件
  - 用户独占整台计算机，所有资源都归当前用户使用
  - 程序输入需要人工操作纸带或者卡片，CPU需要等待人工操作，利用率极低，大量时间空闲
- 缺点：资源利用率极低，用户使用非常不方便，只有专业人员才能操作计算机

### 2. 单道批处理系统
- 时间：20世纪50年代末-60年代初，晶体管计算机时代
- 原理：将一批作业预先输入到外存，由监督程序（操作系统的雏形）顺序将作业装入内存运行，一个作业运行完成后再装入下一个，内存中始终只有一个作业在运行
- 特征：
  - **自动性**：作业自动顺序执行，不需要人工干预
  - **顺序性**：作业按照装入的顺序执行，先进先出
  - **单道性**：内存中只有一个作业在运行
- 优点：减少了人工操作的等待时间，提高了CPU利用率，比手工操作阶段效率高很多
- 缺点：
  - 内存中只有一个作业，IO设备和CPU还是串行工作，CPU和IO设备忙闲不均，资源利用率仍然不高
  - 作业需要排队等待，平均周转时间长

### 3. 多道批处理系统
- 时间：20世纪60年代中期，集成电路计算机时代
- 原理：内存中同时装入多个作业，这些作业可以交替在CPU上运行，当一个作业因为IO等原因阻塞时，CPU可以立刻切换到另一个作业运行，让CPU和IO设备并行工作
- 特点：
  - **多道性**：内存中同时存在多个作业
  - **宏观上并行**：所有作业都在运行过程中，都开始了执行但是还没有结束
  - **微观上串行**：在单CPU环境下，每个时刻只有一个作业在CPU上执行，多个作业交替执行
- 优点：
  - 极大提高了CPU利用率，CPU几乎不会空闲
  - 提高了内存和IO设备的利用率
  - 增加了系统吞吐量，单位时间内完成的作业数大大增加
- 缺点：
  - 作业平均周转时间长，短作业需要等待前面的长作业执行
  - 没有交互能力，作业一旦提交给系统，用户就不能再干预作业的运行，不适合需要交互的场景
  - 需要解决复杂的资源管理问题：处理机调度、内存分配、设备分配、死锁避免等

### 4. 分时系统
- 背景：20世纪60年代，计算机开始普及，需要支持多个用户同时交互使用计算机
- 原理：将CPU的时间划分成很短的时间片，轮流分配给各个终端用户使用，每个用户的请求都能得到快速响应，用户感觉自己独占计算机一样
- 特征：
  - **多路性**：多个用户同时使用一台计算机，系统按时间片轮流为每个用户服务
  - **独立性**：每个用户感觉自己独占计算机，互不干扰
  - **及时性**：用户的请求能在很短的时间内得到响应，通常是几秒钟以内，用户感觉不到等待
  - **交互性**：用户可以和系统进行人机交互，输入命令，查看结果，调试程序等
- 优点：
  - 支持多用户交互使用，大大提高了计算机的利用率
  - 响应速度快，用户体验好
- 缺点：
  - 不适合对实时性要求非常高的场景
  - 平均周转时间比批处理系统长
- 典型系统：UNIX、Linux、Windows等现代通用操作系统都是分时系统

### 5. 实时系统
- 背景：需要对外部事件及时响应的场景，如工业控制、航空航天、军事、多媒体等
- 原理：系统能够及时响应外部事件的请求，在规定的截止时间内完成对该事件的处理，并控制所有实时任务协调一致地运行
- 类型：
  - **硬实时系统**：必须严格满足任务的截止时间要求，一旦超时就会产生严重的后果，甚至灾难性结果，如航空航天控制系统、工业控制、汽车安全系统等
  - **软实时系统**：允许偶尔超时，不会产生严重后果，如视频播放系统、信息查询系统等，错过截止时间只会影响用户体验
- 特征：
  - **及时性**：响应时间要求极高，通常是毫秒级甚至微秒级，必须在截止时间内完成处理
  - **高可靠性**：系统必须高度可靠，不能出现故障，否则会导致严重损失，通常有冗余备份机制
  - **交互性弱**：通常不需要大量的人机交互，主要是自动处理外部事件
  - **多路性**：可以同时处理多个外部事件和多路IO
- 和分时系统的区别：
  - 实时系统的及时性要求更高，分时系统的响应时间通常是秒级，实时系统可以是毫秒级甚至微秒级
  - 实时系统的可靠性要求更高，通常是7*24小时不间断运行，容错能力强
  - 分时系统的交互性强，支持大量用户交互，实时系统交互性弱
  - 分时系统追求公平性，所有用户平等分配时间片，实时系统追求优先级，高优先级任务优先处理

### 6. 操作系统的进一步发展
- **微机操作系统**：个人计算机上的操作系统，如DOS、Windows、macOS等
- **多处理机操作系统**：支持多个CPU的操作系统，提高系统性能
- **分布式操作系统**：管理分布式系统中的多台计算机，提供统一的视图，支持分布式任务调度和资源共享
- **嵌入式操作系统**：运行在嵌入式设备上的操作系统，体积小，功耗低，实时性强，如Android、iOS、嵌入式Linux、VxWorks等
- **网络操作系统**：支持网络功能，提供网络通信、资源共享、网络服务的操作系统

## 1.3 操作系统的基本特征
现代操作系统都具有四个基本特征，其中并发性是最重要的特征，其他特征都是基于并发性的：

### 1. 并发性（Concurrence）
- **定义**：两个或多个事件在同一时间间隔内发生，这些事件在宏观上是同时发生的，微观上是交替发生的
- **并发和并行的区别**：
  - 并行（Parallelism）：两个或多个事件在同一时刻同时发生，需要多个CPU支持
  - 并发：多个事件在同一时间间隔内交替发生，单CPU也可以支持并发
- **进程**：为了实现并发，操作系统引入了进程的概念，进程是系统中可以独立运行、独立分配资源的基本单位，多个进程可以并发执行
- **线程**：为了进一步提高并发性，引入了线程，线程是独立调度的基本单位，一个进程可以有多个线程，进一步提高了并发度

### 2. 共享性（Sharing）
- **定义**：系统中的资源可以被多个并发执行的进程共同使用，而不是被某个进程独占
- **共享方式**：
  - **互斥共享**：一段时间内资源只能被一个进程使用，其他进程需要等待，这类资源叫做临界资源，如打印机、共享变量等
  - **同时共享**：一段时间内可以有多个进程同时访问资源，宏观上是同时的，微观上可能是交替访问的，如磁盘文件、可重入代码等
- 并发和共享是操作系统的两个最基本的特征，它们互为存在条件：没有并发就没有共享的必要，没有共享并发也就无法实现，多个进程无法访问资源。

### 3. 虚拟性（Virtual）
- **定义**：通过某种技术，将一个物理实体变为若干个逻辑上的对应物，物理实体是实际存在的，逻辑实体是用户感觉到的，虚拟的
- **虚拟技术分类**：
  - **时分复用技术**：让多个用户分时使用一个物理资源，提高资源利用率：
    - 虚拟处理机：多道程序设计让每个用户感觉自己独占一个CPU，实际是多个用户分时使用同一个CPU
    - 虚拟设备：SPOOLing技术将一台物理打印机虚拟为多台逻辑打印机，让多个用户同时使用
  - **空分复用技术**：将物理资源划分成多个小的部分，供多个用户同时使用：
    - 虚拟磁盘：将一个物理磁盘划分成多个逻辑分区，每个分区看起来像独立的磁盘
    - 虚拟存储器：将内存和外存结合，从逻辑上扩充内存容量，用户感觉到的内存比实际物理内存大很多
    - 虚拟网络：将一个物理网卡虚拟为多个逻辑网卡，支持多个网络连接
- 虚拟技术可以极大提高资源的利用率，同时方便用户使用。

### 4. 异步性（Asynchronism）
- **定义**：进程以不可预知的速度向前推进，进程什么时候执行，什么时候暂停，什么时候完成，都是不可预知的
- 原因：进程的执行受到其他进程和系统状态的影响，多个进程并发执行，互相制约，导致进程的执行速度是异步的
- 特点：只要运行环境相同，作业经过多次运行，都会得到完全相同的结果，异步性是允许的，不会影响程序的正确性，操作系统需要提供同步机制保证结果的正确性。

## 1.4 操作系统的主要功能
操作系统作为资源管理者，提供五大核心功能：

### 1. 处理机管理功能
主要任务是对CPU的分配和运行进行管理，以进程为基本单位：
- 进程控制：创建、撤销进程，控制进程的状态转换
- 进程同步：协调多个进程的并发执行，避免竞态条件，保证正确共享资源
- 进程通信：实现进程之间的信息交换，支持进程协作
- 调度：包括作业调度（高级调度）、中级调度、进程调度（低级调度），选择合适的进程分配CPU运行

### 2. 存储器管理功能
主要任务是管理内存资源，为多个进程的运行提供良好的环境：
- 内存分配：为每个进程分配内存空间，支持静态分配和动态分配
- 内存保护：保证每个进程只在自己的内存空间内运行，互不干扰，防止地址越界和非法访问
- 地址映射：将程序使用的逻辑地址转换为内存中的物理地址，对用户透明
- 内存扩充：通过虚拟存储技术，从逻辑上扩充内存容量，让更大的程序可以在较小的内存上运行

### 3. 设备管理功能
主要任务是管理IO设备，方便用户使用，提高设备利用率：
- 缓冲管理：提供缓冲区机制，缓和CPU和IO设备速度不匹配的矛盾，提高并行度
- 设备分配：根据用户的IO请求，分配相应的设备、控制器、通道等资源
- 设备处理：也叫设备驱动程序，负责和硬件直接交互，控制设备完成IO操作，响应中断
- 设备独立性：用户程序独立于具体的物理设备，只需要使用逻辑设备名访问设备，提高灵活性

### 4. 文件管理功能
主要任务是管理外存上的文件，实现按名存取，方便用户使用：
- 文件存储空间管理：为文件分配外存空间，回收释放的空间，提高外存利用率，减少碎片
- 目录管理：管理文件目录，实现文件名到物理地址的映射，提高文件检索速度，允许文件重名
- 文件的读写管理：根据用户的请求，从外存读取文件或者将数据写入文件
- 文件的共享和保护：支持多个用户共享文件，同时提供保护机制，防止文件被非法访问和篡改

### 5. 用户接口功能
操作系统为用户提供使用系统的接口，分为三类：
- **命令接口**：用户通过输入命令来操作计算机，分为联机命令接口（交互式）和脱机命令接口（批处理）
- **程序接口**：也叫系统调用，是提供给应用程序使用的接口，应用程序通过系统调用请求操作系统提供服务，是应用程序获取操作系统服务的唯一途径
- **图形用户接口（GUI）**：通过图形化的界面元素（窗口、图标、菜单、按钮），让用户通过鼠标点击等方式操作系统，使用更直观方便，适合普通用户使用

## 1.5 操作系统的结构设计
操作系统的结构设计经历了几个发展阶段，不同的结构有不同的优缺点：

### 1. 无结构操作系统（整体式结构）
- 时间：早期操作系统
- 特点：操作系统是一堆过程的集合，每个过程可以任意调用其他过程，没有明确的结构和模块划分
- 优点：实现简单，执行效率高
- 缺点：结构混乱，可读性差，难以维护和扩展，调试困难，安全性差，一个模块的错误会影响整个系统

### 2. 模块化结构操作系统
- 原理：将操作系统按照功能划分为若干个独立的模块，每个模块有明确的功能和接口，模块之间通过接口互相调用
- 优点：
  - 提高了操作系统的可维护性，修改一个模块不会影响其他模块
  - 提高了可扩充性，添加新功能只需要添加新模块
  - 结构清晰，可读性好，便于调试和开发
- 缺点：
  - 模块之间的依赖关系复杂，接口设计困难
  - 各模块之间没有特权级区别，一个模块的错误可能导致整个系统崩溃
  - 难以对模块进行安全隔离

### 3. 分层式结构操作系统
- 原理：将操作系统划分为若干个层次，每层只能调用底层提供的服务，依赖底层，不依赖高层，各层之间有清晰的接口
- 优点：
  - 结构清晰，层次之间的依赖关系明确，便于调试和维护
  - 提高了系统的可靠性，底层错误只会影响上层，上层错误不会影响底层
  - 易于扩展，修改某一层不会影响其他层
- 缺点：
  - 层次划分困难，很难找到一个完美的分层方案
  - 执行效率低，调用需要逐层穿透，有一定开销
  - 不够灵活，很难跨层调用优化性能

### 4. 微内核结构操作系统
- 原理：将操作系统分成两部分：微内核和服务器进程：
  - 微内核：运行在核心态，只提供最核心最基础的功能，包括进程管理、内存管理、中断处理、进程间通信、基本IO等，尽量小
  - 其他操作系统功能，如文件系统、设备驱动、网络协议等，都作为独立的服务器进程运行在用户态，通过消息传递和微内核通信
- 优点：
  - 提高了系统的可扩展性，添加新功能只需要添加用户态的服务进程，不需要修改内核
  - 提高了系统的可靠性，各个服务进程独立运行，一个服务崩溃不会影响其他服务和内核
  - 可移植性好，硬件相关的代码都在微内核中，移植到新平台只需要修改少量微内核代码
  - 天然支持分布式系统，微内核的消息传递机制可以很容易扩展到分布式环境
  - 融入了面向对象技术，采用对象和消息机制，结构清晰
- 缺点：
  - 运行效率低，所有服务请求都需要经过微内核的消息传递，需要多次用户态和核心态的切换，以及消息拷贝开销，比整体式操作系统性能低
  - 微内核的设计难度大，需要划分清楚哪些功能放在内核，哪些放在用户态
- 典型系统：QNX、L4、Minix、Windows NT内核、鸿蒙OS等，是现代操作系统结构的发展方向

### 5. 外核结构（Exokernel）
- 原理：内核只负责资源的隔离和保护，不提供传统操作系统的抽象，将硬件资源直接暴露给应用程序，应用程序可以根据自己的需要定制资源管理策略
- 优点：灵活性极高，应用程序可以最大化性能，适合特殊需求的场景
- 缺点：开发难度大，应用程序需要自己管理资源，复杂
- 适用场景：专用系统、高性能计算、容器等

## 1.6 操作系统的运行环境
### 1. 核心态和用户态
为了保护操作系统内核的安全，CPU提供了两种运行模式：
- **核心态（Kernel Mode）**：也叫管态、特权态，CPU可以执行所有指令，包括特权指令，可以访问所有硬件资源，操作系统内核运行在核心态
- **用户态（User Mode）**：也叫目态，CPU只能执行非特权指令，不能直接访问硬件资源，只能访问自己的内存空间，应用程序运行在用户态
- **特权指令**：只能在核心态执行的指令，如IO指令、关中断、修改系统寄存器、修改页表等，如果用户态执行特权指令会触发异常，操作系统会终止程序
- **模式切换**：用户态切换到核心态的唯一途径是通过中断、异常或者系统调用，核心态可以直接切换回用户态，通过修改程序状态字实现
- **系统调用**：用户态程序请求操作系统服务的接口，是用户态主动进入核心态的方式，比如读写文件、分配内存、创建进程等操作都需要通过系统调用实现

### 2. 中断和异常
- **中断（Interrupt）**：由外部硬件设备发送给CPU的信号，通知CPU有事件需要处理，比如IO完成中断、时钟中断、键盘中断等，中断是异步的，和当前执行的程序无关
- **异常（Exception）**：也叫内中断、陷入，是CPU执行指令时出现的错误或者特殊情况，比如除零错误、地址越界、缺页、系统调用指令等，异常是同步的，和当前执行的指令相关
- **中断处理过程**：
  1. 中断发生，CPU暂停当前执行的程序，保存上下文
  2. 根据中断号，查找中断向量表，找到对应的中断处理程序入口
  3. 切换到核心态，执行中断处理程序
  4. 中断处理完成，恢复被中断程序的上下文，返回用户态继续执行
- 中断和异常是操作系统实现并发和管理硬件的基础，所有硬件和操作系统的交互都是通过中断实现的。

## 本章核心考点
### 选择题高频考点（408必考2-4分）
1. 操作系统的四大基本特征（并发、共享、虚拟、异步）的含义，并发和并行的区别，并发是最重要的特征
2. 操作系统的发展历程：单道批处理、多道批处理、分时、实时系统的特点和区别，分时和实时的区别
3. 多道程序设计的优点，提高CPU利用率，让CPU和IO并行工作
4. 核心态和用户态的区别，哪些指令是特权指令，哪些操作必须在核心态执行
5. 中断和异常的区别，系统调用的概念和执行过程
6. 操作系统的五大功能

### 考点误区
- 并发≠并行，并行需要多CPU，单CPU只能并发不能并行
- 多道批处理系统的宏观并行是指都在运行过程中，不是真的同时运行
- 系统调用的过程是用户态→核心态→用户态，系统调用本身是内核代码，运行在核心态
- 微内核不是内核越小越好，需要平衡性能和可扩展性，微内核性能比整体式低
