# 第四章 存储器管理

## 4.1 存储器的层次结构
### 存储器层次结构设计
计算机系统的存储器采用多层级结构，从上到下依次为：
1. **寄存器**：CPU内部的高速存储单元，速度最快，容量极小（KB级别），价格最高
2. **高速缓存（Cache）**：CPU和主存之间的高速缓冲，速度接近CPU，容量较小（MB级别），利用程序的局部性原理缓存常用指令和数据
3. **主存（内存）**：计算机的主要工作存储器，速度中等，容量较大（GB级别），CPU只能直接访问主存和寄存器
4. **磁盘（外存）**：永久性存储设备，速度较慢，容量大（TB级别），价格低，用于存放系统和用户数据
5. **磁带/光盘**： archival存储，速度极慢，容量极大，价格极低，用于备份和归档

**层次设计原则**：上层存储器速度是下层的数倍，容量远小于下层，价格远高于下层，上层是下层的缓存，数据在各层之间自动调度，使得整个存储系统的速度接近最快层，容量接近最底层，价格接近最底层。

### 程序的局部性原理
虚拟存储和高速缓存的实现基础，分为两种：
1. **时间局部性**：程序中的某条指令或数据一旦被访问，不久之后可能会被再次访问（如循环、重复调用的函数）
2. **空间局部性**：程序访问了某个存储单元后，不久之后其附近的存储单元也可能会被访问（如顺序执行的指令、数组遍历）

## 4.2 程序的装入和链接
### 程序的执行过程
用户程序要在系统中运行，需要经过以下步骤：
1. **编译**：由编译程序将用户源代码编译成若干个目标模块
2. **链接**：由链接程序将编译后的目标模块以及它们所需要的库函数链接在一起，形成一个完整的装入模块
3. **装入**：由装入程序将装入模块装入内存运行

### 程序的装入方式
1. **绝对装入方式**：
   - 原理：编译程序直接产生绝对地址的目标代码，装入程序直接按照代码中的地址将程序装入内存，不需要修改
   - 适用场景：只能用于单道程序环境，程序的内存地址在编译时就已经确定
2. **可重定位装入方式（静态重定位）**：
   - 原理：目标模块使用相对地址，装入时根据内存的起始地址对程序中的地址进行一次性修改，把相对地址转换为绝对地址
   - 优点：实现简单，不需要硬件支持
   - 缺点：程序装入内存后就不能再移动，必须占用连续的内存空间，难以实现共享
3. **动态运行时装入方式（动态重定位）**：
   - 原理：装入程序直接将程序的相对地址装入内存，地址转换推迟到程序真正执行时进行，需要重定位寄存器的支持
   - 优点：程序可以在内存中移动，不需要连续的内存空间，可以实现对换和虚拟存储，便于代码共享
   - 缺点：需要额外的硬件支持，实现稍复杂

### 程序的链接方式
1. **静态链接**：
   - 原理：程序运行前，就将所有目标模块和所需的库函数链接成一个完整的装入模块，运行时不再分开
   - 优点：运行速度快，不需要运行时支持
   - 缺点：生成的可执行文件大，库函数更新需要重新链接，多个程序调用同一库函数时会在内存中存在多份副本，浪费内存
2. **装入时动态链接**：
   - 原理：目标模块装入内存时，边装入边链接，需要哪个模块就链接哪个模块
   - 优点：便于模块修改和更新，可以实现模块共享，节省内存
   - 缺点：装入速度稍慢，需要操作系统支持
3. **运行时动态链接**：
   - 原理：程序执行过程中需要用到某个模块时，才将该模块调入内存并链接，是最灵活的链接方式
   - 优点：节省内存，加快程序启动速度，便于模块更新和共享
   - 缺点：运行时会有一定的链接开销，需要操作系统支持
   - 典型应用：Windows的DLL（动态链接库）、Linux的.so（共享对象）

## 4.3 连续分配存储管理方式
连续分配是指为一个用户程序分配一个连续的内存空间，是最早的存储管理方式。

### 1. 单一连续分配
- 原理：内存分为系统区和用户区，系统区仅供操作系统使用，用户区全部给当前运行的用户程序使用
- 优点：实现简单，不需要复杂的硬件支持
- 缺点：只能用于单道程序系统，内存利用率极低，资源浪费严重
- 适用场景：早期的单用户单任务操作系统（如MS-DOS）

### 2. 固定分区分配
- 原理：将内存用户空间预先划分为若干个固定大小的分区，每个分区可以装入一道作业，分区大小可以相同也可以不同
- 数据结构：分区说明表，记录每个分区的大小、起始地址、使用状态
- 优点：实现简单，支持多道程序，没有外部碎片
- 缺点：有内部碎片（分区内未被使用的空间），分区大小固定，难以适应作业大小的变化，内存利用率低
- 适用场景：早期的批处理系统，现在已经很少使用

### 3. 动态分区分配（可变分区分配）
- 原理：不预先划分分区，而是在作业装入内存时，根据作业的大小动态地建立分区，分区的大小正好等于作业的大小，分区的数目也是可变的
- 数据结构：空闲分区表、空闲分区链，记录空闲分区的大小和起始地址
- 优点：没有内部碎片，内存利用率比固定分区高，灵活性好
- 缺点：会产生外部碎片（分区之间无法利用的小空闲块），需要紧凑技术来合并碎片

#### 动态分区分配算法
1. **首次适应算法（FF, First Fit）**：
   - 原理：空闲分区链按地址递增顺序排列，分配时从链首开始查找，找到第一个大小能满足要求的空闲分区，分配出去
   - 优点：优先使用低地址空间，高地址会保留较大的空闲分区，分配速度快
   - 缺点：低地址会产生很多小的空闲碎片，查找效率逐渐降低
2. **循环首次适应算法（NF, Next Fit）**：
   - 原理：和首次适应类似，但是每次分配从上次查找结束的位置开始继续查找，而不是从链首开始
   - 优点：空闲分区分布更均匀，减少查找开销
   - 缺点：会导致高地址的大空闲分区被分割，缺乏大的连续空闲区
3. **最佳适应算法（BF, Best Fit）**：
   - 原理：空闲分区按大小递增顺序排列，分配时选择能满足要求的最小的空闲分区
   - 优点：会保留大的空闲分区，产生的碎片相对较小
   - 缺点：会产生很多难以利用的小碎片，每次分配需要遍历整个空闲链表，查找开销大
4. **最坏适应算法（WF, Worst Fit）**：
   - 原理：空闲分区按大小递减顺序排列，分配时选择最大的空闲分区
   - 优点：产生的碎片相对较大，更容易被后续作业利用
   - 缺点：会很快耗尽大的空闲分区，不适合有大内存需求的系统

#### 紧凑（拼接）技术
动态分区分配会产生很多外部碎片，当内存中的空闲分区总和大于作业需要的空间，但没有单个连续的分区能满足需求时，可以采用紧凑技术：
- 原理：移动内存中所有正在运行的进程，使它们相邻，把原来分散的多个小空闲分区合并成一个大的连续空闲分区
- 代价：需要重定位寄存器的支持，移动进程需要消耗大量CPU时间，会导致所有进程暂停运行，系统开销很大
- 时机：通常在内存分配失败时进行，或者定期进行

## 4.4 对换（Swapping）
### 对换的基本概念
- 定义：把内存中暂时不能运行的进程或者暂时不用的程序和数据换出到外存的对换区，腾出足够的内存空间，把已经具备运行条件的进程或者进程需要的程序和数据换入内存
- 作用：提高内存利用率和系统吞吐量，是中级调度的实现方式
- 分类：
  1. 整体对换：以整个进程为单位进行对换，用于多道程序系统
  2. 页面/分段对换：以页/段为单位进行对换，是虚拟存储的基础

### 对换空间的管理
外存分为文件区和对换区：
- 文件区：用于存放文件，追求存储空间利用率，采用离散分配方式
- 对换区：用于存放换出的进程数据，追求换入换出速度，通常采用连续分配方式，IO速度比文件区快

### 对换的过程
1. **换出过程**：选择要换出的进程（优先级低、驻留时间长、阻塞状态的进程），将进程的内存数据写到对换区，释放进程占用的内存空间，修改进程状态为挂起状态
2. **换入过程**：当内存有空闲空间时，选择要换入的挂起进程，在内存中为其分配空间，将进程数据从对换区读到内存，修改进程状态为就绪状态，挂到就绪队列

## 4.5 分页存储管理方式
连续分配会产生碎片问题，紧凑技术开销大，分页存储管理采用离散分配方式，避免了外部碎片，提高了内存利用率。

### 分页的基本原理
1. **页面和物理块**：
   - 页面（页）：将进程的逻辑地址空间划分为若干个大小相等的块，称为页面，每个页面有一个编号，称为页号
   - 物理块（页框/帧）：将内存的物理地址空间划分为和页面大小相同的块，称为物理块，每个物理块有一个编号，称为物理块号
   - 页面大小：通常是2的幂，一般为4KB，页面大小过小会导致页表过大，占用内存多；页面大小过大会导致页内碎片增大，内存利用率降低
2. **逻辑地址结构**：分页系统的逻辑地址是一维的，分为两部分：
   ```
   +----------------+----------------+
   |      页号      |    页内偏移     |
   +----------------+----------------+
   ```
   - 页号：逻辑地址除以页面大小的整数部分
   - 页内偏移：逻辑地址除以页面大小的余数
3. **页表**：也叫页面映射表，是进程逻辑页到物理块的映射表，每个进程有一个独立的页表，存放在内存中，页表的基地址存放在进程PCB的页表寄存器中
   - 页表项结构：每个页表项记录了对应逻辑页的物理块号，以及一些控制位（有效位、修改位、访问位等）
4. **地址变换过程**：
   1. CPU给出逻辑地址，硬件自动将逻辑地址分为页号和页内偏移
   2. 检查页号是否超过进程的地址空间长度，若超过则产生地址越界中断
   3. 根据页表寄存器中的页表基地址和页号，找到对应的页表项，得到物理块号
   4. 物理地址 = 物理块号 × 页面大小 + 页内偏移
   5. 访问物理地址对应的内存单元

### 具有快表的地址变换机构
由于页表存放在内存中，普通的分页地址变换需要两次访问内存（第一次查页表，第二次访问数据），访存效率降低了一半，为了提高地址变换速度，引入了快表：
1. **快表（TLB, Translation Lookaside Buffer）**：是一个高速缓冲存储器，放在CPU内部，用来缓存最近使用的页表项，访问速度比内存快一个数量级
2. **地址变换过程**：
   1. CPU给出逻辑地址，分解为页号和页内偏移
   2. 首先在快表中查找该页号，若找到（命中），直接得到物理块号，不需要访问内存中的页表
   3. 若快表中没有找到（不命中），访问内存中的页表，得到物理块号，同时将该页表项写入快表，若快表已满则采用置换算法淘汰一个旧的页表项
   4. 用物理块号和页内偏移拼接成物理地址，访问内存
3. **有效访问时间计算**：
   假设内存访问时间为t，快表访问时间为ε，快表命中率为p，则有效访问时间EAT为：
   $$ EAT = p \times (t + \varepsilon) + (1 - p) \times (2t + \varepsilon) = 2t + \varepsilon - p \times t $$
   例：t=100ns，ε=20ns，p=90%，则EAT=0.9×120 + 0.1×220 = 108 + 22 = 130ns，比不使用快表的200ns快很多

### 两级和多级页表
当进程的地址空间很大时（如32位系统有4GB地址空间，4KB页面的话页表有1M个页表项，每个页表项4B的话页表占4MB内存），页表本身会占用大量连续的内存空间，为了解决这个问题，引入了多级页表：
1. **两级页表**：将页表再进行分页，把页表的每个页面离散存放在不同的物理块中，再建立一个页目录表（外层页表），记录每个页表页面的物理块号
   - 逻辑地址结构分为三部分：页目录号 → 页号 → 页内偏移
   - 地址变换过程：首先查页目录表，得到页表页面的物理块号，然后查该页表页面，得到物理块号，最后拼接页内偏移得到物理地址，需要三次访问内存
2. **多级页表**：对于更大的地址空间（如64位系统），可以采用三级、四级甚至更多级的页表，逻辑地址分为更多层，每一层对应一级页表
3. **优点**：不需要为页表分配连续的内存空间，不需要把全部页表都放在内存中，只需要把当前需要的页表页面调入内存，节省内存空间
4. **缺点**：地址变换需要多次访问内存，访存开销更大，需要快表来提高效率

### 反置页表
传统的页表是每个进程一个，按逻辑页号索引，反置页表是整个系统一个，按物理块号索引：
1. **结构**：每个物理块对应一个页表项，记录该物理块中存放的是哪个进程的哪个逻辑页
2. **优点**：整个系统只需要一个页表，占用内存空间小，适合大地址空间系统
3. **缺点**：按逻辑页号查找反置页表需要遍历整个表，查找效率低，通常需要结合哈希表提高查找速度
4. **适用场景**：PowerPC、SPARC、IA-64等架构，以及一些大型系统

## 4.6 分段存储管理方式
分页是从物理角度划分内存，提高内存利用率，而分段是从用户的逻辑角度划分地址空间，更符合用户的编程习惯。

### 分段的基本原理
1. **分段的概念**：将进程的地址空间按照逻辑意义划分为若干个段，每个段有自己的段名和段号，段长由段的实际长度决定，每个段从0开始编址，采用连续地址空间
   - 常见的段：代码段、数据段、栈段、堆段、共享库段等
2. **逻辑地址结构**：分段系统的逻辑地址是二维的，由段号和段内偏移两部分组成：
   ```
   +----------------+----------------+
   |      段号      |    段内偏移     |
   +----------------+----------------+
   ```
   用户编程时使用段名+段内偏移的方式访问地址，编译时转换为段号+段内偏移的二维地址
3. **段表**：每个进程有一个段表，记录每个段的段号、段长、在内存中的起始物理地址（基址），段表存放在内存中，段表基地址存放在段表寄存器中
4. **地址变换过程**：
   1. CPU给出逻辑地址，分为段号和段内偏移
   2. 检查段号是否超过段表长度，若超过则产生地址越界中断
   3. 根据段表基地址和段号找到对应的段表项，检查段内偏移是否超过段长，若超过则产生地址越界中断
   4. 物理地址 = 段基址 + 段内偏移
   5. 访问物理地址对应的内存单元

### 分段和分页的区别
| 对比维度 | 分页 | 分段 |
|----------|------|------|
| 划分角度 | 信息的物理单位，为了提高内存利用率，操作系统管理的需要 | 信息的逻辑单位，包含一组意义相对完整的信息，用户编程的需要 |
| 大小是否固定 | 页面大小固定，由系统决定 | 段长不固定，由段的实际内容决定，用户决定 |
| 地址空间 | 一维地址空间，只需要一个地址就能确定位置 | 二维地址空间，需要段号和段内偏移两个部分才能确定位置 |
| 碎片 | 没有外部碎片，有内部碎片 | 没有内部碎片，有外部碎片 |
| 共享和保护 | 不方便实现以逻辑模块为单位的共享和保护 | 方便实现段级的共享和保护，更适合动态链接 |
| 访问效率 | 地址变换速度快，适合系统底层管理 | 更灵活，适合用户使用，访问效率稍低 |

### 段的共享与保护
1. **段的共享**：多个进程的段表可以指向同一个物理内存段，实现代码和数据的共享，只有可重入代码（纯代码）才能被共享，共享数据段需要同步机制保护
2. **段的保护**：
   - 地址越界保护：检查段号是否合法，段内偏移是否超过段长
   - 存取权限保护：每个段表项中增加存取权限位（读、写、执行），访问时检查权限是否合法
   - 环保护机制：系统分为多个特权级，高特权级可以访问低特权级的段，低特权级不能随意访问高特权级的段

## 4.7 段页式存储管理方式
段页式存储管理结合了分段和分页的优点，既具有分段系统的逻辑清晰、便于共享保护的优点，又具有分页系统的内存利用率高、无外部碎片的优点，是现在主流操作系统采用的存储管理方式。

### 基本原理
1. **地址空间划分**：首先将进程的地址空间按逻辑意义划分为若干个段，每个段再划分为若干个大小固定的页面
2. **逻辑地址结构**：分为三部分：
   ```
   +----------------+----------------+----------------+
   |      段号      |      页号      |    页内偏移     |
   +----------------+----------------+----------------+
   ```
3. **数据结构**：每个进程有一个段表，每个段对应一个页表
   - 段表项：记录该段的页表起始地址和页表长度
   - 页表项：记录该页对应的物理块号和控制位

### 地址变换过程
1. CPU给出逻辑地址，分解为段号、页号、页内偏移
2. 检查段号是否合法，若超过段表长度则产生越界中断
3. 根据段表基地址和段号找到对应的段表项，得到该段的页表基地址
4. 检查页号是否超过页表长度，若超过则产生越界中断
5. 根据页表基地址和页号找到对应的页表项，得到物理块号
6. 物理地址 = 物理块号 × 页面大小 + 页内偏移
7. 访问物理地址对应的内存单元

**注意**：不使用快表的情况下，段页式地址变换需要三次访问内存（第一次查段表，第二次查页表，第三次访问数据），通常需要配合快表使用，快表中同时缓存段号和页号，命中时只需要一次访问内存。

### 优缺点
- 优点：结合了分段和分页的优点，灵活性好，内存利用率高，便于共享和保护
- 缺点：系统复杂度高，管理开销大，地址变换速度慢，需要更多的硬件支持
