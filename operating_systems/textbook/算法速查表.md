# 操作系统常用算法速查表

## 一、调度算法
| 算法名称 | 英文缩写 | 核心思想 | 优点 | 缺点 | 适用场景 |
|----------|----------|----------|------|------|----------|
| 先来先服务 | FCFS | 按到达顺序调度 | 公平、实现简单 | 短作业等待时间长，不利于IO繁忙型作业 | 批处理系统，低级调度 |
| 短作业优先 | SJF | 优先调度估计运行时间最短的作业 | 平均周转时间短，吞吐量高 | 长作业饥饿，估计时间不准确，未考虑优先级 | 批处理系统 |
| 高响应比优先 | HRRN | 优先调度响应比最高的作业：<br>响应比 = 1 + 等待时间/服务时间 | 兼顾长短作业，避免饥饿 | 每次调度需要计算响应比，开销大 | 批处理系统 |
| 时间片轮转 | RR | 按时间片轮流执行就绪队列中的进程 | 响应时间快，公平，交互性好 | 时间片太小切换开销大，太大退化为FCFS | 分时系统 |
| 优先级调度 | PSA | 优先调度优先级最高的进程 | 可以保证紧急任务优先处理 | 低优先级进程饥饿 | 实时系统、批处理系统 |
| 多级反馈队列 | MFQ | 多个优先级队列，不同队列时间片不同，进程用完时间片降级 | 兼顾长短作业，交互性好，无需估计运行时间 | 实现稍复杂 | 通用调度算法，适用各种系统 |
| 最早截止时间优先 | EDF | 优先调度截止时间最早的任务 | 调度灵活，可调度性高 | 实现复杂，需要知道任务截止时间 | 实时系统 |
| 最低松弛度优先 | LLF | 优先调度松弛度最低的任务：<br>松弛度 = 截止时间 - 剩余运行时间 - 当前时间 | 能保证紧急任务及时处理 | 松弛度需要动态更新，开销大 | 强实时系统 |

## 二、页面置换算法
| 算法名称 | 英文缩写 | 核心思想 | 优点 | 缺点 | 特殊性质 |
|----------|----------|----------|------|------|----------|
| 最佳置换算法 | OPT | 淘汰以后永不使用或最长时间不使用的页面 | 缺页率最低，性能最好 | 无法实现，只能作为评价基准 | - |
| 先进先出置换算法 | FIFO | 淘汰最先进入内存的页面 | 实现最简单 | 性能差，可能出现Belady异常（物理块增加缺页率升高） | 唯一可能产生Belady异常的常用算法 |
| 最近最久未使用 | LRU | 淘汰最近最久没有被访问的页面 | 性能接近OPT，无Belady异常 | 需要硬件支持（栈/移位寄存器），实现开销大 | 堆栈类算法 |
| 时钟置换算法 | CLOCK/NRU | 循环扫描，淘汰访问位为0的页面 | 实现简单，开销小，性能接近LRU | 没有考虑修改位，可能淘汰需要写回的页面 | 最近未使用算法 |
| 改进型时钟算法 | - | 优先淘汰(访问位=0,修改位=0)的页面，其次(0,1)，其次(1,0)，其次(1,1) | 减少磁盘IO次数，性能更好 | 需要额外扫描 | 兼顾修改位，更实用 |
| 最少使用算法 | LFU | 淘汰最近一段时间访问次数最少的页面 | 考虑了页面使用频率 | 实现复杂，不能区分近期使用情况 | - |

## 三、磁盘调度算法
| 算法名称 | 英文缩写 | 核心思想 | 优点 | 缺点 |
|----------|----------|----------|------|------|
| 先来先服务 | FCFS | 按请求到达顺序处理 | 公平，实现简单 | 平均寻道时间长，寻道距离波动大 |
| 最短寻道时间优先 | SSTF | 优先处理离当前磁道最近的请求 | 平均寻道时间短，性能较好 | 可能导致饥饿，中间磁道优先，两端请求等待时间长 |
| 扫描算法（电梯算法） | SCAN | 磁头沿一个方向移动，处理完所有该方向请求后返回 | 避免饥饿，寻道性能稳定 | 两端请求等待时间长，刚越过的请求需要等待很长时间 |
| 循环扫描算法 | CSCAN | 磁头沿一个方向移动，处理完所有请求后直接跳回另一端重新开始 | 所有请求的等待时间更均匀，避免两端请求等待过长 | 磁头空跳会产生额外寻道开销 |
| N步扫描算法 | N-SCAN | 将请求分为多个长度为N的子队列，子队列按FCFS处理，子队列内用SCAN | 避免磁臂粘着现象，适合高负荷系统 | 实现稍复杂 |
| FSCAN算法 | - | 将请求分为两个队列，一个是当前处理队列，一个是新请求队列，处理完当前队列后处理新队列 | 避免磁臂粘着，实现简单 | 新请求需要等待当前队列处理完 |

## 四、动态分区分配算法
| 算法名称 | 英文缩写 | 核心思想 | 优点 | 缺点 |
|----------|----------|----------|------|------|
| 首次适应算法 | FF | 从空闲分区链首开始查找，找到第一个大小满足的分区分配 | 优先使用低地址空间，高地址保留大分区，分配速度快 | 低地址产生很多小碎片，查找开销逐渐增大 |
| 循环首次适应算法 | NF | 从上次查找结束的位置开始继续查找 | 分区分布更均匀，减少查找开销 | 会导致缺乏大的连续空闲分区 |
| 最佳适应算法 | BF | 优先选择大小最接近请求的空闲分区分配 | 保留大分区，碎片较小 | 产生很多难以利用的小碎片，查找需要遍历整个链表 |
| 最坏适应算法 | WF | 优先选择最大的空闲分区分配 | 产生的碎片较大，更容易利用 | 缺乏大分区，不适合有大内存需求的系统 |
| 快速适应算法 | - | 按分区大小分类，相同大小的分区组成链表，直接查找对应大小的链表 | 分配速度极快，不会产生碎片 | 合并分区复杂，内存利用率低 |

## 五、重要公式汇总
### 1. 调度相关
- 周转时间 = 完成时间 - 到达时间
- 带权周转时间 = 周转时间 / 服务时间
- 平均周转时间 = 所有进程周转时间之和 / 进程数
- 平均带权周转时间 = 所有进程带权周转时间之和 / 进程数

### 2. 分页相关
- 逻辑地址结构：逻辑地址 = 页号 × 页大小 + 页内偏移
- 物理地址结构：物理地址 = 物理块号 × 页大小 + 页内偏移
- 页号 = 逻辑地址 / 页大小 （整除）
- 页内偏移 = 逻辑地址 % 页大小 （取余）
- 有快表的有效访问时间 EAT = (访存时间 + 快表访问时间) × 快表命中率 + (2 × 访存时间 + 快表访问时间) × (1 - 快表命中率)
- 缺页的有效访问时间 EAT = 访存时间 × (1 - 缺页率) + 缺页处理时间 × 缺页率

### 3. 磁盘相关
- 磁盘访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间
- 寻道时间 = 启动磁臂时间 + 移动n个磁道的时间 = t_s + n × t_t
- 旋转延迟时间：平均为磁盘旋转半周的时间 = 1/(2×转速)，如7200转/分的硬盘平均旋转延迟为1/(2×120) = 4.17ms
- 传输时间 = 读取字节数 / 传输速率 = 扇区大小 × 读取扇区数 / 传输速率

### 4. 银行家算法相关
- 需求矩阵 Need[i][j] = 最大需求 Max[i][j] - 已分配 Allocation[i][j]
- 安全性检查：寻找安全序列，保证所有进程的Need[i] ≤ Work，执行Work += Allocation[i]，直到所有进程都完成

### 5. 混合索引文件大小计算
假设：
- 磁盘块大小为 B 字节
- 每个地址项大小为 A 字节
- 直接地址项有 D 个
- 一级间接地址项有 I1 个
- 二级间接地址项有 I2 个
- 三级间接地址项有 I3 个
- 每个磁盘块可存放的地址项数 N = B / A

则：
- 直接地址可表示的文件大小：D × B
- 一级间接可表示的文件大小：I1 × N × B
- 二级间接可表示的文件大小：I2 × N² × B
- 三级间接可表示的文件大小：I3 × N³ × B
- 文件最大总长度 = 直接 + 一级 + 二级 + 三级

例：UNIX文件系统，B=4KB，A=4B，D=10，I1=1，I2=1，I3=1
N = 4KB / 4B = 1024
最大文件长度 = 10×4KB + 1×1024×4KB + 1×1024²×4KB + 1×1024³×4KB = 40KB + 4MB + 4GB + 4TB

## 六、信号量经典模板
### 1. 基本互斥
```c
semaphore mutex = 1; // 互斥信号量
process() {
  wait(mutex);  // 进入临界区
  // 临界区代码
  signal(mutex); // 退出临界区
}
```

### 2. 基本同步（前趋关系 P1→P2）
```c
semaphore S = 0; // 同步信号量
P1() {
  // 执行P1的代码
  signal(S); // P1执行完发信号
}
P2() {
  wait(S); // 等待P1完成
  // 执行P2的代码
}
```

### 3. 生产者消费者（n个缓冲区）
```c
semaphore mutex = 1;  // 缓冲区互斥访问
semaphore empty = n;  // 空缓冲区数量
semaphore full = 0;   // 满缓冲区数量

producer() {
  while(true) {
    produce_item();
    wait(empty);
    wait(mutex);
    put_item_to_buffer();
    signal(mutex);
    signal(full);
  }
}

consumer() {
  while(true) {
    wait(full);
    wait(mutex);
    get_item_from_buffer();
    signal(mutex);
    signal(empty);
    consume_item();
  }
}
```

### 4. 读者写者（读者优先）
```c
semaphore rmutex = 1;  // 保护读者计数
semaphore wmutex = 1;  // 写互斥
int readcount = 0;

reader() {
  while(true) {
    wait(rmutex);
    if(readcount == 0) wait(wmutex); // 第一个读者加写锁
    readcount++;
    signal(rmutex);
    
    read_file();
    
    wait(rmutex);
    readcount--;
    if(readcount == 0) signal(wmutex); // 最后一个读者释放写锁
    signal(rmutex);
  }
}

writer() {
  while(true) {
    wait(wmutex);
    write_file();
    signal(wmutex);
  }
}
```
