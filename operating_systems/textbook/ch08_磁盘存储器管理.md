# 第八章 磁盘存储器管理

## 8.1 磁盘概述
磁盘是目前计算机系统中最主要的外存储设备，具有容量大、价格低、非易失性、可随机访问等特点，是文件系统的主要存储介质。

### 磁盘的分类
1. **机械硬盘（HDD, Hard Disk Drive）**：
   - 原理：采用磁性存储介质，通过电机驱动磁头在高速旋转的盘片上移动，读取或写入数据
   - 特点：容量大（可达几十TB），价格低，顺序访问速度快，随机访问速度慢（因为需要寻道和旋转延迟），抗震性差，功耗大，噪音大
   - 适用场景：大容量存储，冷数据存储，归档存储，对成本敏感的场景
2. **固态硬盘（SSD, Solid State Drive）**：
   - 原理：采用闪存芯片作为存储介质，没有机械运动部件，完全是电子结构
   - 特点：随机访问速度极快，没有寻道和旋转延迟，抗震性好，功耗低，噪音小，重量轻，价格比HDD高，写寿命有限，有擦除次数限制
   - 适用场景：操作系统安装盘，常用程序存储，对性能要求高的场景，笔记本、移动设备等
3. **混合硬盘（SSHD）**：结合了HDD的大容量和SSD的高速，内部集成了小容量的SSD作为缓存，自动将常用数据缓存到SSD中，性能接近SSD，容量和HDD差不多，价格介于两者之间

### 机械硬盘的结构和性能参数
#### 物理结构
1. **盘片**：圆形的磁性存储介质，一个硬盘有多个盘片，叠在一起同轴旋转，每个盘片有两个表面，都可以存储数据，每个表面对应一个磁头
2. **磁道**：盘片表面划分成很多同心圆，每个同心圆叫做一个磁道，从外到内编号，最外面是0磁道
3. **扇区**：每个磁道划分成若干个扇形区域，叫做扇区，每个扇区大小通常是512字节或者4KB，是磁盘读写的最小物理单位，每个扇区有自己的地址标识
4. **柱面**：所有盘片上相同编号的磁道组成一个柱面，柱面号等于磁道号
5. **磁头**：每个盘面有一个磁头，用来读取和写入数据，磁头可以沿着半径方向移动，寻道时移动到目标磁道
6. **主轴**：带动所有盘片高速旋转，常见转速有5400转/分、7200转/分、10000转/分、15000转/分

#### 磁盘地址表示
磁盘的物理地址用三维地址表示：柱面号（磁道号） → 磁头号（盘面号） → 扇区号，操作系统通过这三个地址定位磁盘上的任意扇区。

#### 磁盘的访问时间
磁盘访问一个扇区的时间由三部分组成：
1. **寻道时间Ts**：
   - 定义：磁头从当前位置移动到目标磁道所需要的时间
   - 组成：寻道启动时间 + 磁头移动n个磁道的时间 = Ts = m * n + s，其中m是移动一个磁道的时间，s是启动时间
   - 典型值：普通HDD的平均寻道时间是3~15ms，7200转硬盘通常是8~9ms，15000转的企业级硬盘是3~5ms
   - 占比：寻道时间是磁盘访问时间的主要组成部分，通常占总访问时间的70%以上
2. **旋转延迟Tr**：
   - 定义：目标扇区旋转到磁头下方所需要的时间，平均是磁盘旋转半周的时间
   - 计算：Tr = 1/(2*转速)，转速的单位是转/秒
   - 典型值：7200转/分的硬盘，转速是120转/秒，平均旋转延迟是1/(2*120) ≈ 4.17ms；15000转/分的硬盘，平均旋转延迟是2ms
3. **传输时间Tt**：
   - 定义：将扇区的数据从磁盘传输到内存的时间，或者将内存的数据写入磁盘扇区的时间
   - 计算：Tt = 读取的字节数 / 传输速率，传输速率是磁盘的持续传输速度，通常是几十到几百MB/s
   - 典型值：读取一个4KB的扇区，传输速率100MB/s的话，Tt = 4KB / 100MB/s = 0.04ms，比寻道和旋转延迟小很多，可以忽略不计

#### 平均访问时间
平均访问时间 = 平均寻道时间 + 平均旋转延迟 + 传输时间，因为传输时间很小，通常可以近似为平均寻道时间 + 平均旋转延迟，比如7200转的硬盘平均访问时间约为9ms + 4.17ms = 13ms左右。

### 固态硬盘的结构和性能参数
#### 结构
- 存储介质：NAND闪存芯片，分为SLC、MLC、TLC、QLC等，寿命依次降低，容量依次增大，价格依次降低
- 控制器：SSD的核心处理器，负责闪存的读写、纠错、磨损均衡、坏块管理、垃圾回收等
- DRAM缓存：可选，用来存放映射表和缓存数据，提高性能
- 接口：SATA、PCIe、NVMe等，NVMe接口的SSD速度最快，可达几GB/s

#### 性能参数
- 随机读写IOPS：每秒可以完成的随机读写次数，是SSD最重要的性能指标，高端NVMe SSD的随机读IOPS可达几十万甚至上百万，远高于HDD的几百IOPS
- 顺序读写速度：连续读写的速度，高端NVMe SSD可达7GB/s以上，SATA SSD约500MB/s
- 访问延迟：SSD的访问延迟只有几十到几百微秒，远低于HDD的十几毫秒
- 寿命：用TBW（Total Bytes Written，总写入字节数）表示，消费级SSD通常是几百到几千TBW

## 8.2 磁盘调度算法
磁盘是计算机系统中最繁忙的IO设备之一，多个进程会同时发出磁盘IO请求，磁盘调度算法的目标是合理安排这些请求的处理顺序，减少平均寻道时间，提高磁盘的IO吞吐量，减少平均响应时间。

磁盘调度的核心优化目标是减少平均寻道时间，因为寻道时间占总访问时间的大部分。

### 1. 先来先服务算法（FCFS, First Come First Served）
- **原理**：按照进程发出IO请求的先后顺序进行调度，先到的请求先处理，不考虑请求访问的磁道位置
- **例子**：当前磁头在53号磁道，请求序列是98, 183, 37, 122, 14, 124, 65, 67，那么寻道顺序就是53→98→183→37→122→14→124→65→67，总寻道长度是|98-53| + |183-98| + |37-183| + |122-37| + |14-122| + |124-14| + |65-124| + |67-65| = 45 + 85 + 146 + 85 + 108 + 110 + 59 + 2 = 640，平均寻道长度是640/8=80
- **优点**：公平，实现最简单，不会出现饥饿现象，所有请求都会被处理
- **缺点**：平均寻道长度大，寻道性能差，没有优化，对随机请求的处理效率极低
- **适用场景**：IO请求很少的场景，或者对公平性要求极高的场景，一般不使用

### 2. 最短寻道时间优先算法（SSTF, Shortest Seek Time First）
- **原理**：每次选择当前磁头位置最近的请求进行处理，使得每次的寻道时间最短
- **例子**：当前磁头在53号磁道，请求序列是98, 183, 37, 122, 14, 124, 65, 67，最近的是65，然后67，然后37，然后14，然后98，然后122，然后124，然后183，总寻道长度是|65-53| + |67-65| + |37-67| + |14-37| + |98-14| + |122-98| + |124-122| + |183-124| = 12 + 2 + 30 + 23 + 84 + 24 + 2 + 59 = 236，平均寻道长度236/8=29.5，比FCFS好很多
- **优点**：平均寻道长度小，寻道性能好，比FCFS效率高很多，实现也比较简单
- **缺点**：会出现饥饿现象，如果一直有新的离当前磁头近的请求到来，远离磁头的请求会一直得不到处理，出现饥饿
- **适用场景**：IO请求比较均匀，没有大量突发请求的场景，现在很少单独使用，通常作为其他算法的基础

### 3. 扫描算法（SCAN, 电梯算法）
- **原理**：磁头沿着一个方向移动，比如从低磁道向高磁道移动，在移动过程中处理所有当前方向上的请求，直到到达最边上的请求，然后改变方向，继续处理反方向的请求，类似电梯的运行方式，所以也叫电梯算法
- **例子**：当前磁头在53号磁道，方向是向高磁道移动，请求序列是98, 183, 37, 122, 14, 124, 65, 67，寻道顺序是53→65→67→98→122→124→183→37→14，总寻道长度是12+2+31+24+2+59+146+23 = 299，平均寻道长度37.375，比SSTF稍差，但是没有饥饿
- **优点**：
  1. 寻道性能好，平均寻道长度比较短，接近SSTF
  2. 不会出现饥饿现象，所有请求最终都会被处理，因为磁头会双向扫描，照顾到所有磁道的请求
  3. 公平性比较好
- **缺点**：
  1. 两端的请求的等待时间会比较长，刚越过的请求需要等磁头走到头再回来才能处理，对于两端的请求不公平
  2. 磁头方向切换时，如果没有新的请求，会空转，有一定浪费
- **适用场景**：大多数通用系统，是应用最广泛的磁盘调度算法之一

### 4. 循环扫描算法（CSCAN, Circular SCAN）
- **原理**：磁头只沿着一个方向移动，比如从低到高，处理完所有当前方向的请求后，直接跳回到另一端的第一个请求，不处理反方向的请求，然后继续向同一个方向移动，循环扫描
- **例子**：当前磁头在53号磁道，方向向高，请求序列是98, 183, 37, 122, 14, 124, 65, 67，寻道顺序是53→65→67→98→122→124→183→14→37，总寻道长度是12+2+31+24+2+59+169+23 = 322，平均寻道长度40.25
- **优点**：
  1. 所有请求的平均等待时间更均匀，不会出现SCAN算法中两端请求等待时间过长的问题，新请求的最大等待时间是扫描一次磁盘的时间
  2. 寻道性能稳定，波动小
- **缺点**：
  1. 磁头从一端跳回另一端时的寻道是无效寻道，会浪费一定的寻道时间
  2. 平均寻道长度比SCAN稍长
- **适用场景**：负载比较重的系统，请求很多的场景，所有请求的响应时间比较均匀

### 5. N步扫描算法（N-SCAN）
- **原理**：将磁盘请求队列分成多个长度为N的子队列，每次处理一个子队列，子队列内部采用SCAN算法处理，处理完一个子队列再处理下一个子队列，新到来的请求加入后面的子队列
- **优点**：
  1. 避免了磁臂粘着现象，在SCAN算法中，如果一个磁道一直有新的请求到来，磁头会一直停留在这个磁道，影响其他请求，N步扫描中，新请求加入后面的队列，不会影响当前队列的处理
  2. 结合了SCAN的优点，又避免了磁臂粘着
- **缺点**：实现相对复杂，N的大小需要合理配置，太小性能接近FCFS，太大接近SCAN，还是可能出现粘着

### 6. FSCAN算法
- **原理**：是N步扫描的简化版本，将请求队列分成两个队列，一个是当前正在处理的队列，一个是新到来的请求队列，当前队列用SCAN算法处理，处理完后，将新队列和剩下的请求合并为新的当前队列，新的请求不会影响当前队列的处理
- **优点**：实现简单，有效避免了磁臂粘着现象，性能接近SCAN
- **适用场景**：主流的操作系统很多采用这种算法，或者类似的变种

### 磁盘调度算法的选择
- SSD固态硬盘不需要磁盘调度算法，因为SSD没有机械运动，没有寻道时间，访问任何位置的速度都一样，调度算法对SSD没有意义，反而会增加开销
- 对于HDD机械硬盘，SCAN和CSCAN以及它们的变种是最常用的，性能好，公平性好，没有饥饿问题
- 实际系统中的调度算法通常比这些基础算法更复杂，会考虑请求的优先级、IO的类型（读/写）、相邻请求的合并等，比如Linux的 Deadline、NOOP、CFQ调度器等

## 8.3 磁盘高速缓存
磁盘的速度比内存慢几个数量级，为了提高磁盘的访问速度，减少磁盘IO次数，操作系统会在内存中开辟一块缓冲区，缓存最近访问过的磁盘块，叫做磁盘高速缓存（Disk Cache）或者页高速缓存（Page Cache）。

### 基本原理
- 程序的局部性原理：最近访问过的磁盘块，不久之后很可能会再次访问，将这些块缓存在内存中，下次访问时直接从内存读取，不需要访问磁盘，大大提高访问速度
- 磁盘高速缓存利用的是内存的高速特性，弥补磁盘的速度不足
- 缓存的单位是物理页或者磁盘块，通常是4KB大小，和内存页大小一致

### 缓存的命中和失效
- 命中：要访问的磁盘块已经在缓存中，直接从内存读取，速度很快，只需要几十纳秒
- 失效：要访问的磁盘块不在缓存中，需要从磁盘读取，然后放到缓存中，速度慢，需要十几毫秒
- 命中率：命中的次数占总访问次数的比例，命中率越高，性能越好，操作系统的磁盘高速缓存命中率通常可以达到90%以上，大大提高了系统性能

### 缓存的置换算法
当缓存满了，需要新调入一个块时，需要选择一个块置换出去，常用的置换算法：
1. **最近最久未使用算法（LRU）**：置换最长时间没有被访问过的块，性能最好，是最常用的算法
2. **先进先出算法（FIFO）**：置换最早进入缓存的块，实现简单，但是性能差，可能出现Belady异常
3. **最少使用算法（LFU）**：置换访问次数最少的块，性能接近LRU，但是需要记录访问次数，开销大
4. **时钟算法（CLOCK）**：LRU的近似实现，性能接近LRU，实现简单，开销小，是实际系统最常用的算法

### 缓存的写策略
对于写操作，有两种写策略：
1. **写回法（Write Back）**：
   - 原理：写操作只修改缓存中的块，不立即写入磁盘，只给块做一个脏标记，等到块被置换出缓存时，才将脏块写回磁盘
   - 优点：写操作速度快，减少磁盘写入次数，多个对同一个块的写操作可以合并为一次磁盘写入，大大提高性能
   - 缺点：如果系统崩溃，缓存中还没有写回磁盘的脏数据会丢失，导致数据不一致，需要日志或者其他机制保证数据安全
   - 适用场景：现代操作系统几乎都采用写回法，性能极高
2. **写直达法（Write Through）**：
   - 原理：写操作同时修改缓存和磁盘，写入磁盘完成后才返回
   - 优点：数据安全性高，不会丢失数据，一致性好
   - 缺点：写操作速度慢，每次写都需要访问磁盘，性能差
   - 适用场景：对数据安全性要求极高，不能容忍数据丢失的场景，或者缓存很小的场景

### 周期性写回
为了避免脏数据在缓存中停留太久，丢失风险高，操作系统会有一个后台进程，周期性地将缓存中的脏数据写回磁盘，通常是30秒或者5分钟一次，将所有脏块刷新到磁盘，减少系统崩溃时的数据丢失量。Linux中的`pdflush`或者`flush`进程就是干这个的。

### 预读（Read Ahead）
根据局部性原理，当访问一个磁盘块时，操作系统会将后面相邻的几个块也提前读到缓存中，因为程序很可能会顺序访问后面的块，这样下次访问后面的块时就命中缓存，不需要访问磁盘，大大提高顺序访问的速度。比如读取一个文件，操作系统通常会预读后面的128KB或者256KB的数据到缓存。

## 8.4 提高磁盘IO速度的其他方法
### 1. 提前读（预读）
上面已经讲过，利用顺序访问的局部性，提前读取相邻的块，提高命中率。

### 2. 延迟写
就是写回法的另一种说法，将写操作延迟到块被置换或者定期刷新时才写入磁盘，合并多个写操作，提高性能。

### 3. 优化物理块分布
将同一个文件的物理块尽量放在连续的位置，或者放在同一个柱面、相邻的磁道，减少访问时的寻道时间和旋转延迟，提高顺序访问速度。文件系统在分配块时会尽量分配连续的块，或者相邻的块。

### 4. 虚拟盘（RAM盘）
将一部分内存虚拟成磁盘，数据存在内存中，访问速度极快，和内存速度一样，但是断电数据会丢失，用来存放临时文件或者经常访问的只读数据，可以极大提高速度，比如Linux的tmpfs文件系统就是虚拟盘，/tmp目录默认挂载在tmpfs上。

### 5. 磁盘阵列（RAID, Redundant Array of Independent Disks）
将多个独立的物理磁盘组合成一个逻辑磁盘，提供更高的存储容量、更高的IO性能、更高的可靠性和冗余性，是现在服务器和存储系统的标配。
- **RAID 0**：条带化，将数据分成条带，分布在多个磁盘上，并行读写，性能最高，容量是所有磁盘之和，但是没有冗余，任何一个磁盘损坏，所有数据都会丢失，适合对性能要求高，对可靠性要求低的场景
- **RAID 1**：镜像，两个磁盘互为镜像，写入数据时同时写入两个磁盘，读可以从任意一个磁盘读，可靠性极高，一个磁盘损坏不会丢失数据，容量是总容量的一半，写性能稍差，读性能好，适合对可靠性要求极高的场景
- **RAID 5**：分布式奇偶校验，数据和奇偶校验信息分布在多个磁盘上，允许一个磁盘损坏，损坏后可以通过奇偶校验恢复数据，容量是n-1块磁盘的总和，读性能好，写性能稍差，兼顾性能、容量和可靠性，是最常用的RAID级别
- **RAID 6**：双奇偶校验，允许两个磁盘同时损坏，比RAID5更可靠，但是写性能更差，适合大容量存储，可靠性要求高的场景
- **RAID 10**：RAID1+RAID0，先做镜像，再做条带化，兼顾RAID0的高性能和RAID1的高可靠性，容量是一半，性能高，可靠性高，适合对性能和可靠性都要求高的场景

### 6. 设备独立性和IO调度
前面已经讲过，合理的IO调度算法可以减少寻道时间，提高IO速度。

### 7. 异步IO和DMA/通道
采用DMA或者通道方式，IO操作不需要CPU干预，CPU和IO并行工作，提高系统吞吐量。采用异步IO，进程发起IO请求后不需要阻塞等待，可以继续执行其他任务，IO完成后通知进程，提高系统效率。

## 8.5 磁盘可靠性和坏块管理
磁盘在使用过程中会出现坏块，也就是无法正常读写的扇区，需要对坏块进行管理，避免影响系统正常运行。
### 坏块的类型
1. **硬坏块**：物理损坏，磁性介质失效，无法修复，必须标记出来不再使用
2. **软坏块**：逻辑错误，比如ECC校验错误，可能是因为突然断电或者写入干扰，可以通过擦除或者重新写入修复
### 坏块管理方法
1. **硬盘控制器内部管理**：现代硬盘都支持坏块自动重映射，硬盘控制器会自动检测坏块，将坏块重定向到磁盘上预留的备用扇区，对操作系统透明，不需要操作系统处理
2. **操作系统管理**：如果硬盘不支持自动重映射，操作系统的文件系统会扫描磁盘坏块，将坏块标记为不可用，不会分配给文件使用，比如Linux的badblocks命令可以扫描坏块，mkfs可以将坏块加入坏块列表
3. **RAID冗余**：RAID系统可以在磁盘出现坏块或者损坏时，通过冗余信息恢复数据，不会丢失数据，提高可靠性
