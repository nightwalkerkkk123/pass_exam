# 操作系统考研高频考点汇总（408统考适用）

## 核心考点分布
操作系统在408统考中占35分，题型：
- 选择题：10题 × 2分 = 20分
- 综合题：1题 × 15分 = 15分

高频考点按分值占比排序：
1. 进程管理（约35%）：进程同步、调度、死锁
2. 存储管理（约25%）：分页管理、虚拟存储、页面置换
3. IO与文件管理（约25%）：磁盘调度、文件系统、成组链接法
4. 操作系统基础（约15%）：OS特征、接口、结构

---

## 第一章 操作系统引论（选择题考点）
### 高频考点
1. **操作系统的特征**：并发性（最基本特征，区分程序和进程的关键）、共享性、虚拟性、异步性，四大特征的含义和关系
2. **操作系统的功能**：处理机管理、存储器管理、设备管理、文件管理、用户接口，各功能的作用
3. **操作系统的发展历程**：
   - 多道批处理系统的特点和优点（提高资源利用率、吞吐量）
   - 分时系统的特征（多路性、独立性、及时性、交互性），时间片概念
   - 实时系统的特点和与分时系统的区别（及时性要求更高、可靠性更高、交互性弱）
4. **操作系统的运行环境**：用户态和核心态的区别，特权指令和非特权指令，系统调用的概念和过程
5. **操作系统的结构**：微内核结构的特点和优缺点（灵活性高、可靠性高、可移植性好，但性能开销大）

### 常考题型
- 选择题：判断哪些是操作系统的特征/功能，区分分时和实时系统，判断特权指令，微内核特点
- 选择题常见误区：并发≠并行，并行是同一时刻，并发是同一时间间隔；系统调用是用户态到核心态的切换

---

## 第二章 进程的描述与控制（选择题+综合题考点）
### 高频考点
1. **进程基本概念**：
   - 进程的定义、特征（动态性、并发性、独立性、异步性），进程与程序的区别
   - 进程的三种基本状态（就绪、执行、阻塞）及转换条件，PCB的作用（进程存在的唯一标志）
2. **进程同步**：
   - 同步机制的四大准则（空闲让进、忙则等待、有限等待、让权等待）
   - 整型信号量和记录型信号量的区别，记录型信号量的P/V操作含义：
     - S.value > 0：可用资源数
     - S.value = 0：无可用资源，无等待进程
     - S.value < 0：绝对值是等待该资源的进程数
   - 用信号量实现互斥、同步、前趋关系的方法
3. **经典同步问题**（综合题必考）：
   - **生产者-消费者问题**：空/满缓冲区信号量、互斥信号量的应用，P操作顺序不能颠倒
   - **哲学家进餐问题**：死锁的避免方法（最多4人同时进餐、按奇偶顺序拿筷子、同时拿左右筷子）
   - **读者-写者问题**：读者优先和写者优先的实现原理，读者计数的互斥访问
4. **进程通信**：三种通信方式（共享存储器、消息传递、管道）的原理和区别，直接通信和间接通信的区别，管道的特点（半双工、先进先出、只能在父子/兄弟进程间使用）
5. **线程**：线程的概念、与进程的区别，线程的实现方式（用户级线程、内核级线程、组合方式）的优缺点，多线程模型（一对一、多对一、多对多）的特点

### 常考题型
- 选择题：进程状态转换判断、信号量值的含义、线程与进程的区别、通信方式的区分
- 综合题：信号量编程，解决同步互斥问题（如生产者消费者变形、读者写者变形、理发师问题等），10分左右大题

---

## 第三章 处理机调度与死锁（选择题+综合题考点）
### 高频考点
1. **调度层次**：高级调度（作业调度）、中级调度（对换）、低级调度（进程调度）的区别和作用，调度的时机和切换过程
2. **调度算法**：
   - 算法原理：FCFS、SJF（抢占/非抢占）、优先级调度、RR轮转、高响应比优先、多级反馈队列
   - 算法评价：平均周转时间、平均带权周转时间的计算
   - 算法适用场景：RR用于分时系统，多级反馈队列是通用调度算法
   - 周转时间 = 完成时间 - 到达时间，带权周转时间 = 周转时间 / 服务时间
3. **实时调度**：EDF（最早截止时间优先）和LLF（最低松弛度优先）的原理和调度过程，松弛度计算
4. **死锁**：
   - 死锁产生的四个必要条件（互斥、请求保持、不可抢占、循环等待）
   - 死锁的处理方法：
     - 预防死锁：破坏四个必要条件中的一个或几个，具体方法和优缺点
     - 避免死锁：安全状态和不安全状态的概念，银行家算法（综合题必考）
     - 检测死锁：资源分配图的化简，死锁定理
     - 解除死锁：资源抢占、撤销进程的方法

### 常考题型
- 选择题：调度算法的特点比较、死锁必要条件判断、银行家算法安全性判断、资源分配图死锁判断
- 综合题：
  1. 调度算法计算：给出进程到达时间和服务时间，计算不同调度算法下的周转时间、带权周转时间
  2. 银行家算法：给出资源分配情况，判断是否安全、能否满足进程的资源请求、找安全序列，15分左右大题

## 第四章 存储器管理（选择题+综合题考点）
### 高频考点
1. **存储器层次结构**：寄存器→高速缓存→主存→磁盘→磁带，各级的速度和容量对比，程序的局部性原理（时间局部性、空间局部性）
2. **程序的装入和链接**：三种装入方式（绝对装入、可重定位装入、动态运行时装入）的区别，静态重定位和动态重定位的区别，三种链接方式（静态链接、装入时动态链接、运行时动态链接）的区别
3. **连续分配存储管理方式**：
   - 单一连续分配、固定分区分配、动态分区分配的原理
   - 动态分区分配算法：首次适应（FF）、循环首次适应（NF）、最佳适应（BF）、最坏适应（WF）的原理和优缺点
   - 紧凑（拼接）技术的作用，内部碎片和外部碎片的区别
4. **分页存储管理方式**（重点）：
   - 基本原理：将内存分为固定大小的页框，程序分为同样大小的页面，通过页表实现地址变换
   - 地址变换过程：逻辑地址=页号+页内偏移，物理地址=页框号×页大小+页内偏移
   - 快表（TLB）的作用和地址变换过程，具有快表的地址变换有效访问时间计算
   - 两级和多级页表的原理，反置页表的特点
5. **分段存储管理方式**：
   - 基本原理：将程序按逻辑意义分为多个段，每个段有段名和段长，通过段表实现地址变换
   - 分段与分页的区别：分页是信息的物理单位，分段是信息的逻辑单位；分页大小固定，分段大小不固定；分页的地址空间是一维的，分段的地址空间是二维的
6. **段页式存储管理方式**：基本原理，地址变换过程（段号→段表→页号→页表→物理地址）

### 常考题型
- 选择题：碎片的区分、分配算法的特点、分页分段的区别、地址变换过程
- 综合题：分页存储管理的地址变换计算，逻辑地址转物理地址，具有快表的有效访问时间计算，页表项的设计

---

## 第五章 虚拟存储器（选择题+综合题考点）
### 高频考点
1. **虚拟存储器基本概念**：
   - 定义：具有请求调入功能和置换功能，能从逻辑上对内存容量加以扩充的存储器系统
   - 特征：多次性、对换性、虚拟性，实现基础是程序的局部性原理
   - 实现方式：请求分页、请求分段、请求段页式
2. **请求分页存储管理方式**：
   - 页表的扩充：状态位（是否在内存）、访问位、修改位、外存地址
   - 缺页中断：当访问的页面不在内存时，产生缺页中断，将所需页面调入内存，缺页中断是特殊的中断，在指令执行期间产生和处理
   - 页面分配策略：固定分配局部置换、可变分配全局置换、可变分配局部置换
   - 页面调入策略：请求调页、预调页，从何处调入页面（文件区、对换区）
3. **页面置换算法**（重点）：
   - 最佳置换算法（OPT）：淘汰以后永不使用或最长时间不使用的页面，性能最好但无法实现，用于评价其他算法
   - 先进先出置换算法（FIFO）：淘汰最先进入内存的页面，实现简单，可能产生Belady异常（分配的物理块数增加，缺页率反而升高）
   - 最近最久未使用置换算法（LRU）：淘汰最近最久未使用的页面，性能接近OPT，需要硬件支持（移位寄存器、栈），不会产生Belady异常
   - 时钟置换算法（CLOCK）：最近未使用算法，每页设置访问位，循环扫描，淘汰访问位为0的页面，实现简单，性能接近LRU
   - 改进型时钟算法：同时考虑访问位和修改位，优先淘汰未访问且未修改的页面，减少磁盘IO次数
   - 最少使用算法（LFU）：淘汰最近一段时间使用次数最少的页面
4. **抖动与工作集**：
   - 抖动的概念：页面频繁换入换出，导致系统效率急剧下降的现象，原因是同时运行的进程太多，分配给每个进程的物理块太少，不能满足进程正常运行的基本需求
   - 工作集：某段时间间隔内，进程要访问的页面集合，工作集大小随时间变化，具有局部性，工作集模型可以预防抖动，基于局部性原理，给进程分配的物理块数不小于工作集大小

### 常考题型
- 选择题：虚拟存储器的特征、缺页中断的特点、置换算法的特点、Belady异常、工作集原理
- 综合题：
  1. 页面置换算法模拟：给出页面访问序列和物理块数，计算不同置换算法下的缺页次数和缺页率，OPT、FIFO、LRU、CLOCK算法的模拟过程
  2. 有效访问时间计算：给出缺页率、内存访问时间、磁盘IO时间，计算平均有效访问时间

## 第六章 IO系统（选择题考点）
### 高频考点
1. **IO系统的功能和层次结构**：用户层IO软件、设备独立性软件、设备驱动程序、中断处理程序，各层的作用
2. **IO控制方式**：
   - 程序IO方式（轮询方式）：CPU不断查询IO设备状态，CPU利用率低，简单
   - 中断驱动IO方式：IO设备完成操作后向CPU发中断，CPU与IO设备可以并行工作，适合字符设备
   - 直接存储器访问（DMA）方式：IO设备和内存之间直接成块传输数据，不需要CPU干预，只在传输开始和结束时需要CPU，适合块设备
   - 通道控制方式：专门的IO处理器执行通道程序控制IO操作，一个通道可以控制多个设备，进一步减轻CPU负担，适合大型系统
3. **缓冲管理**：
   - 缓冲的作用：缓和CPU和IO设备速度不匹配的矛盾、减少对CPU的中断频率、提高CPU和IO设备之间的并行性
   - 缓冲的类型：单缓冲、双缓冲、循环缓冲、缓冲池，单/双缓冲下的系统处理时间计算
4. **设备分配**：
   - 设备分配的数据结构：设备控制表（DCT）、控制器控制表（COCT）、通道控制表（CHCT）、系统设备表（SDT）
   - 设备分配的原则：充分发挥设备效率，避免死锁，方便用户
   - 设备独立性（无关性）：用户程序独立于具体使用的物理设备，好处是提高设备分配的灵活性，易于实现IO重定向
   - SPOOLing技术（假脱机技术）：利用外存模拟脱机输入输出，将独占设备改造为共享设备，由输入井、输出井、输入缓冲区、输出缓冲区、输入进程、输出进程组成，典型应用是打印机共享
5. **磁盘调度**：
   - 磁盘的结构和性能参数：寻道时间（移动磁头到指定磁道的时间，占磁盘访问时间的大部分）、旋转延迟（等待扇区转到磁头下的时间）、传输时间
   - 磁盘调度算法（重点）：
     1. 先来先服务（FCFS）：公平，简单，平均寻道时间长，适合请求少的情况
     2. 最短寻道时间优先（SSTF）：优先选择访问磁道与当前磁道距离最近的请求，平均寻道时间短，但可能出现饥饿
     3. 扫描算法（SCAN/电梯算法）：磁头沿一个方向移动，处理完该方向所有请求后再反向，避免饥饿，寻道性能好，对两端磁道的请求不利
     4. 循环扫描算法（CSCAN）：磁头沿一个方向移动，处理完所有请求后直接回到另一端再开始，所有请求的等待时间更均匀，避免两端请求等待时间过长
     5. N步SCAN算法：将请求队列分为多个长度为N的子队列，按FCFS处理每个子队列，子队列内用SCAN，避免磁臂粘着
     6. FSCAN算法：将请求队列分为两个，一个是当前处理的请求队列，一个是新到达的请求队列，处理完当前队列后处理新队列，避免磁臂粘着

### 常考题型
- 选择题：IO控制方式的区别、缓冲的作用、SPOOLing技术的特点、磁盘调度算法的模拟和寻道时间计算
- 综合题：磁盘调度算法的应用，给出磁道访问序列和当前磁道位置，计算不同调度算法下的总寻道长度和平均寻道长度

## 第七章 文件管理（选择题+综合题考点）
### 高频考点
1. **文件和文件系统**：
   - 文件的定义：具有文件名的一组相关元素的集合，文件属性（类型、长度、创建时间、所有者、权限等）
   - 文件的逻辑结构：
     - 无结构文件（流式文件）：由字符流组成，如文本文件、二进制文件
     - 有结构文件（记录式文件）：由记录组成，分为顺序文件、索引文件、索引顺序文件（如ISAM、VSAM），不同结构的检索效率对比
   - 文件的物理结构（文件在外存的存放方式）：
     - 连续分配：文件存放在连续的磁盘块中，顺序访问速度快，支持随机访问，会产生外部碎片，文件动态增长困难
     - 链接分配：文件存放在不连续的磁盘块中，通过指针链接，分为隐式链接和显式链接（FAT文件分配表），不会产生外部碎片，文件动态增长方便，只能顺序访问，不能随机访问
     - 索引分配：为每个文件建立索引表，存放文件逻辑块到物理块的映射，支持随机访问，不会产生外部碎片，文件动态增长方便，索引表占用额外存储空间
     - 混合索引分配：结合直接地址、一级间接地址、二级间接地址、三级间接地址，如UNIX/Linux的inode结构，兼顾小文件的访问效率和大文件的存储需求
2. **目录管理**：
   - 文件控制块（FCB）：存放文件的描述信息和控制信息，是目录的基本组成单元，inode是UNIX/Linux中FCB的改进，将文件名和文件描述信息分开，提高目录检索效率
   - 目录结构：
     - 单级目录：所有文件在一个目录中，实现简单，存在重名问题，查找速度慢
     - 两级目录：分为主文件目录和用户文件目录，解决了重名问题，可实现文件共享
     - 多级目录（树形目录）：层次清晰，便于分类管理和权限控制，查找速度快，路径名分为绝对路径和相对路径
   - 目录查询技术：线性检索、Hash检索
3. **文件存储空间的管理**：
   - 空闲表法：记录空闲磁盘块的连续区间，适合连续分配，回收和分配算法简单
   - 空闲链表法：将空闲磁盘块链成链表，分配时从链头取，回收时加入链，适合离散分配，遍历链表开销大
   - 位示图法：用二进制位表示磁盘块的使用情况，0表示空闲，1表示已分配，分配和回收效率高，占用空间小，容易保存在内存中，常用
   - 成组链接法（重点）：结合空闲表和空闲链表的优点，适合大型文件系统（如UNIX/Linux），将空闲磁盘块分组，组内用链表链接，组的信息存放在栈中，分配和回收效率高，不需要将整个空闲链表放在内存中
4. **文件共享与保护**：
   - 文件共享：基于索引结点的共享方式（硬链接）、利用符号链实现共享（软链接），两种方式的区别
   - 文件保护：访问控制矩阵、访问控制列表、用户权限表，UNIX/Linux的rwx权限机制

### 常考题型
- 选择题：文件逻辑/物理结构的特点、目录结构的特点、存储空间管理方法的特点、硬链接和软链接的区别
- 综合题：
  1. 混合索引的文件大小计算：给出直接地址项、一级/二级/三级间接地址项的数量，磁盘块大小，地址项大小，计算文件的最大长度
  2. 成组链接法的分配和回收过程：给出成组链接的结构，模拟分配和回收磁盘块的过程，10分左右大题

## 复习建议
