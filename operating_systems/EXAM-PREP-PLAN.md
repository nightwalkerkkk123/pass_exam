**高频考法：** 生产者-消费者问题，填空空的 P/V 操作

重点资源：

- exams/ch02-test.html：第 20-25 题（大题部分）

**操作步骤：**

1. 背诵"信号量三步法"
   ```
   信号量初始化：S = 初始资源数
   进入临界区：P(S)  [原子操作：S-1，若S<0则等待]
   离开临界区：V(S)  [原子操作：S+1，唤醒一个等待]
   ```

2. 做标准例题
   - Lesson 10 的 3 个例题：缓冲区大小 3 的生产者-消费者
   - 手写每一步的 P/V 操作

3. 变式题训练
   - 缓冲区大小改成 5（从讲义改数字）
   - 加入"优先级"限制（读者-写者问题变式）
   - 计时做 3 遍，每遍都应该 < 5 分钟

4. 检验标准
   - 能否解释每一个 P 操作为什么在这里？
   - 能否指出漏写一个 P 会导致什么问题？

### 任务 3.2：页面置换算法（Lesson 6）

**高频考法：** 给定页面访问序列，计算缺页次数 / 使用 LRU 或 FIFO 置换

重点资源：
- Lesson 6 讲义：OPT / FIFO / LRU 的完整例题
- exams/ch05-test.html：第 15-20 题
- 教科书的 Belady 异常示例

**操作步骤：**

1. 每个算法 3 分钟速记
   ```
   OPT（最优）：找未来最晚使用的页，置换它
   FIFO：找最早进入的页，置换它（用队列）
   LRU：找最近最少使用的页，置换它（用时间戳）
   ```

2. 标准例题：4 个物理块，页面访问序列 `1,2,3,4,1,2,5,1,2,3,4,5`
   - 用 FIFO 做一遍：画 4 行（4 个块）× 12 列（12 步）的表格
   - 用 LRU 做一遍：每个块旁写时间戳
   - 对比答案：FIFO 缺页 8 次，LRU 缺页 7 次

3. Belady 异常
   - 讲义中提到：增加块数反而增加缺页（FIFO 的特性）
   - 做一个 3 块 vs. 4 块的对比题
   - 理解为什么 LRU 不会有这个问题

4. 变式题 × 3
   - 改序列：`1,2,1,3,4,5,4,3,2,1`
   - 改块数：从 4 改成 3
   - 计时做，每题 < 4 分钟

### 任务 3.3：磁盘调度算法（Lesson 7）

**高频考法：** 给定柱面访问序列和磁头当前位置，计算 SCAN / CSCAN 的总移动距离

重点资源：
- Lesson 7 讲义：SCAN（电梯）和 CSCAN 的逐步演示
- exams/ch06-test.html（注：实际在课程讲义中，因为 Ch8 分散在 L7-L8）

**操作步骤：**

1. 算法速记（核心是"方向"和"到头就折返"）
   ```
   FCFS：按请求顺序
   SSTF：每次找最近的柱面（贪心）
   SCAN：磁头从 0→max，一边走一边处理请求
        走到头后折返，再从 max→0
   CSCAN：SCAN 变体，只从 0→max 处理，然后快速回到 0（更公平）
   ```

2. 标准例题：磁头在 50，请求序列 `95, 180, 34, 119, 11, 123, 62, 64`，盘片 0-199
   - SCAN（从 50 向 199 方向）：
     - 上升：50 → 62 → 64 → 95 → 119 → 123 → 180 → 199
     - 下降：199 → 34 → 11 → 0
     - 总距离 = (199-50) + (199-11) = 149 + 188 = 337
   - 画图：横轴是柱面号，纵轴是时间，箭头显示磁头运动

3. CSCAN 对比
   - 同样例题用 CSCAN：走完上升段后直接跳到 0，不扫回程
   - 区别：避免"刚回到 0 又要冲到 199"的问题

4. 计时变式 × 3
   - 改起点、改盘片范围、改序列
   - 每题要求 < 5 分钟

### 任务 3.4：混合索引与文件大小（Lesson 9）

**高频考法：** UNIX inode 混合索引，计算文件最大容量（关键数值：10 直接指针 + 1 一级间接 + 1 二级间接）

重点资源：
- Lesson 9 讲义：inode 结构图 + 计算公式推导
- exams/ch06-test.html：第 25 题左右

**操作步骤：**

1. inode 结构硬背（会经常出现）
   ```
   UNIX inode 有 13 个指针：
   - 10 个直接指针：指向 10 个数据块
   - 1 个一级间接指针：指向一个指针块（含 block_size/4 个指针）
   - 1 个二级间接指针：指向指针的指针块
   - 1 个三级间接指针：同上，能寻址更多

   假设 block_size = 1KB = 1024 字节，指针 4 字节：
   - 1 个指针块 = 1024/4 = 256 个指针
   ```

2. 文件最大大小计算（以 block_size=1KB、3 级间接为例）
   ```
   直接部分：10 × 1KB = 10KB
   一级间接：256 × 1KB = 256KB
   二级间接：256 × 256 × 1KB = 64MB
   三级间接：256 × 256 × 256 × 1KB = 16GB
   总计：10KB + 256KB + 64MB + 16GB ≈ 16GB
   ```

3. 讲义例题：block_size=2KB，计算最大文件大小
   - 1 个指针块 = 2048/4 = 512 个指针
   - 逐项相加

4. 变式题 × 2
   - 改 block_size = 4KB
   - 只算到二级间接的情况
   - 每题 < 3 分钟

#### 4.2.2 "常见追问"预案

**问题 1：磁盘调度的 SCAN 和 CSCAN 有什么区别？**
```
回答框架（< 1 分钟）：
- SCAN：磁头从 0 到 max，边走边处理，走到头折返
- CSCAN：只从 0 到 max，然后快速跳回 0（不扫回程）
- 优势：CSCAN 更公平，消除了"在 max 附近的请求" 的饥饿现象
- 例子：如果请求都堆在 180-200，SCAN 要等很久才能再处理，CSCAN 下一轮立即处理
```

**问题 2：为什么 LRU 页面置换比 FIFO 更优？**
```
回答框架（< 1 分钟）：
- FIFO：淘汰最老的页（与使用规律无关）
- LRU：淘汰最不常用的页（符合时间局部性）
- 反例：Belady 异常只在 FIFO 中发生，LRU 不会
- 原因：LRU 根据"最近使用"预测"近期需要"，符合实际程序行为
```

**问题 3：PV 操作中，为什么要在访问临界资源前做 P 操作？**
```
回答框架（< 1 分钟）：
- P(S) 的原子性保证：S 减 1，如果 S < 0 则阻塞
- 作用：限制同时进入临界区的进程数
- 例子：缓冲区大小 3，第 4 个生产者做 P(buffer)  会阻塞，直到有消费者做 V(buffer)
```

**问题 4：成组链接法的"链接"是什么意思？**
```
回答框架（< 1 分钟）：
- 磁盘空闲块被分组存储：第一组记录在超级块，最后一个块指向下一组的位置
- 优势：不需要维护整个空闲表，只需要在内存中保留当前组
- 例子：假设每组 20 个块，共 1000 个空闲块，只需内存中存 20 个指针
```
