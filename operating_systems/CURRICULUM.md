## 定位（一句话）

为**深零起点**的她，在 **1–2 周**内，按**考试权重**排序、用**图文并茂 + 例子 + 对照 + 深挖是什么/为什么**的统一方式，把学校自命题《操作系统》Ch1–6 讲到「既懂原理又能拿分」。

- 时间/目标：**1–2 周冲刺，但要真懂**（用户 2026-07-09 确认）。
- 边界：学校自命题、非 408；覆盖 Ch1–6；不碰第 9–12 章、不做玩具内核、xv6/lab 系列仅作参考。

## 一、统一课程模板（每一节都长这样）

每个概念固定**六步**，把教学要求焊进结构，保证每个概念都被同样深地挖过一遍：

| 步 | 内容 | 目的 |
|---|---|---|
| 🖼 直觉图 | 可视化/可点的图，先看见 | 图文并茂，不靠纯文字 |
| 🧸 具体例子 | 一个最小例子讲一遍（ELI5） | 举例子 |
| ⚖️ 对照表 | 和最易混概念并排比较 | 做比较（如 并发 vs 并行） |
| 📖 是什么 | 精确定义，对齐老师口径 | 是什么 |
| ❓ 为什么 | 逐层深挖：为何这么设计？不这样会怎样？边界在哪？ | 为什么（深挖） |
| ✅ 自测 | 打乱选项的选择题 + 复述，过了才推进 | 验证理解 |

**通过标准（铁律）**：她能用**自己的话复述** + 自测答对，才进下一节。一节只装 **1 个**窄概念。

## 三、每节直播流程

1. 我先给你打开这节的可视化课（🖼 图）。
2. 用六步法带你走：例子 → 对照 → 是什么 → 为什么深挖。
3. 关键处停下，**让你用自己的话复述**。
4. 用 **AskUserQuestion** 出打乱选项的题验证。
5. 复述通过 + 答对 → 打勾、进下一节；答错 → 回炉，并在 `learning-records/` 记一条。
6. 每段结束 → 做对应 `exams/chNN-test.html`；全程结束 → 仿真期末卷。
