# 一、操作系统引论

## 1. 操作系统是什么

**操作系统 OS** 是配置在计算机硬件上的第一层系统软件。

它的核心作用是：

```text
管理计算机硬件和软件资源
控制程序运行
改善人机交互
为应用程序提供服务接口
```

一句话：

> 操作系统是计算机硬件和用户/应用程序之间的中间层。

------

## 2. 操作系统的目标

教材里常见目标有四个：

```text
方便性
有效性
可扩充性
开放性
```

### 方便性

让用户不用直接操作硬件。

例如用户只需要：

```c
printf("hello");
```

不用关心显示器、显卡、缓冲区、驱动程序如何工作。

### 有效性

提高资源利用率和系统吞吐量。

主要体现在：

```text
提高 CPU 利用率
提高内存利用率
提高 I/O 设备利用率
提高系统整体吞吐量
```

### 可扩充性

系统以后能方便增加新功能。

例如：

```text
增加新设备驱动
增加新文件系统
支持新硬件
支持新网络协议
```

### 开放性

系统遵循标准，能与其他系统兼容和互联。

例如：

```text
POSIX 标准
TCP/IP 协议
统一 API
开放文件格式
```

------

## 3. 操作系统的作用

### 作为用户和硬件之间的接口

用户不能直接控制硬件，而是通过 OS 提供的接口操作计算机。

接口包括：

```text
命令接口
图形界面接口
系统调用接口
```

### 作为资源管理者

OS 管理四类资源：

```text
处理机 CPU
存储器 Memory
I/O 设备
文件和数据
```

### 实现资源抽象

操作系统把复杂硬件抽象成简单对象。

例如：

```text
磁盘块 → 文件
物理内存 → 虚拟地址空间
CPU → 进程/线程
I/O 设备 → read/write 接口
```

------

## 4. 操作系统的发展过程

### 手工操作阶段

特点：

```text
无操作系统
用户独占整台机器
人工装入程序
CPU 经常等待人工操作
资源利用率低
```

问题：

```text
人机矛盾
CPU 和 I/O 速度不匹配
```

------

### 脱机输入/输出

也叫 **Off-line I/O**。

思想：

```text
用外围机提前把输入数据送入磁带/磁盘
主机运行时直接从高速设备读取
```

优点：

```text
减少 CPU 等待慢速输入设备的时间
提高 I/O 速度
```

------

### 单道批处理系统

特点：

```text
一次只在内存中放一道作业
作业自动连续执行
系统中有监督程序 Monitor
```

优点：

```text
减少人工干预
提高系统效率
```

缺点：

```text
CPU 仍会因 I/O 等待而空闲
内存中只有一个程序
资源利用率仍然不高
```

------

### 多道批处理系统

核心思想：

```text
内存中同时放多个作业
当一个作业等待 I/O 时，CPU 去执行另一个作业
```

优点：

```text
CPU 利用率高
内存利用率高
I/O 设备利用率高
系统吞吐量大
```

缺点：

```text
平均周转时间较长
无交互能力
系统复杂度增加
```

需要解决的问题：

```text
处理机调度
内存分配与保护
I/O 设备分配
文件管理
作业管理
用户与 OS 接口
```

------

### 分时系统

核心思想：

```text
多个用户通过终端同时使用一台主机
系统按时间片轮流为每个用户服务
```

关键技术：

```text
多路性
独立性
及时性
交互性
```

### 分时系统特征

| 特征   | 含义                       |
| ------ | -------------------------- |
| 多路性 | 多个用户同时使用系统       |
| 独立性 | 每个用户感觉自己独占机器   |
| 及时性 | 用户请求能在较短时间响应   |
| 交互性 | 用户可以边运行边修改、调试 |

------

### 实时系统

实时系统强调：

```text
必须在规定时间内完成任务
```

典型场景：

```text
工业控制
导弹控制
航空航天
多媒体播放
嵌入式系统
```

### 实时任务分类

```text
周期性实时任务
非周期性实时任务
硬实时任务
软实时任务
```

### 硬实时

必须严格按时完成。

例如：

```text
导弹控制
飞机自动驾驶
核电站控制
```

超时可能造成严重后果。

### 软实时

允许偶尔超时，但性能会下降。

例如：

```text
视频播放

在线会议
```

## 5. 操作系统的基本特征

这是第一章核心考点：

```text
并发
共享

```

------

### 并发 Concurrence

并发是指多个程序在同一时间段内执行。

注意区别：

```text
并发：宏观上同时，微观上交替
并行：真正同时执行，需要多个 CPU 或多核
```

例如单核 CPU 上：

```text
0~10ms：运行 A
10~20ms：运行 B
20~30ms：运行 C
```

看起来 A、B、C 同时运行，其实 CPU 在快速切换。

------

### 共享 Sharing

系统资源可以被多个进程共同使用。

共享方式：

```text
互斥共享
同时访问
```

#### 互斥共享

同一时刻只允许一个进程访问。

例如：

```text
打印机
磁带机
临界资源
```

#### 同时访问

多个进程可以“同时”访问。

例如：

```text
只读文件
磁盘文件
共享代码段
```

------

### 虚拟 Virtual

把一个物理实体变成多个逻辑实体。

常见虚拟技术：

```text
虚拟处理机
虚拟内存
虚拟设备
虚拟文件系统
```

例子：

```text
一台 CPU → 多个进程都感觉自己有 CPU
小内存 → 程序感觉自己有很大地址空间
一台打印机 → 多个进程感觉可以同时打印
```

------

### 异步 Asynchronism

程序执行不是一气呵成，而是走走停停。

原因：

```text
进程调度
I/O 等待
中断
资源竞争
```

但只要运行环境相同，程序多次运行结果应该一致。

# 二、进程的描述与控制

## 1. 前趋图

前趋图用于描述程序或任务之间的先后关系。

例如：

```text
S1 → S2 → S3
```

表示：

```text
S2 必须在 S1 完成后才能执行
S3 必须在 S2 完成后才能执行
```

前趋关系常用于：

```text
并发程序分析
进程同步
任务调度
```

------

## 2. 程序顺序执行

程序顺序执行的特征：

```text
顺序性
封闭性
可再现性
```

### 顺序性

程序按照固定顺序执行。

### 封闭性

程序独占资源，不受外界影响。

### 可再现性

只要输入相同，多次运行结果相同。

------

## 3. 程序并发执行

并发执行的特征：

```text
间断性
失去封闭性
不可再现性
```

### 间断性

程序执行过程中可能被暂停。

### 失去封闭性

多个程序共享资源，互相影响。

### 不可再现性

并发执行顺序不同，结果可能不同。

例如：

```c
count++;
```

多个线程同时执行，结果可能出错。

------

## 4. 进程的定义

**进程 = 程序在某个数据集合上的一次运行活动。**

更常见说法：

> 进程是资源分配和独立运行的基本单位。

进程包括：

```text
程序段
数据段
进程控制块 PCB
```

其中 PCB 是进程存在的唯一标志。

------

## 5. 进程与程序的区别

| 项目           | 程序             | 进程               |
| -------------- | ---------------- | ------------------ |
| 性质           | 静态             | 动态               |
| 是否执行       | 存放在磁盘上     | 正在运行           |
| 是否有生命周期 | 无明显生命周期   | 有创建、运行、终止 |
| 是否拥有资源   | 不拥有           | 拥有资源           |
| 是否可并发     | 程序本身不可并发 | 进程可并发         |

一句话：

```text
程序是静态代码，进程是运行中的程序。
```

------

## 6. 进程的基本状态

三态模型：

```text
就绪 Ready
运行 Running
阻塞 Blocked
```

### 就绪

进程已经具备运行条件，只差 CPU。

### 运行

进程正在 CPU 上执行。

### 阻塞

进程等待某事件，暂时不能运行。

例如：

```text
等待 I/O 完成
等待信号量
等待资源
等待用户输入
```

------

## 7. 五态模型

```text
新建 New
就绪 Ready
运行 Running
阻塞 Blocked
终止 Terminated
```

状态转换：

```text
新建 → 就绪
就绪 → 运行
运行 → 就绪
运行 → 阻塞
阻塞 → 就绪
运行 → 终止
```

------

## 12. 临界资源

一次只允许一个进程访问的资源。

例如：

```text

共享变量
共享缓冲区
某些表项
```

------

## 13. 临界区

访问临界资源的代码段叫临界区。

典型结构：

```c
entry section;     // 进入区
critical section;  // 临界区
exit section;      // 退出区
remainder section; // 剩余区
```

------

## 14. 临界区管理原则

```text
空闲让进
忙则等待
有限等待
让权等待
```

### 空闲让进

临界区空闲时，允许一个请求进程进入。

### 忙则等待

已有进程在临界区时，其他进程必须等待。

### 有限等待

不能让某进程无限期等待。

### 让权等待

等待时应释放 CPU，不能一直忙等。

------

## 16. P、V 操作

### P 操作

```text

信号量减 1
若资源不足则阻塞
```

### V 操作

```text
释放资源
信号量加 1
若有等待进程则唤醒
```

## 19. 生产者—消费者问题

```text

```

# 三、处理机调度与死锁

## 1. 调度层次

```text
高级调度：作业调度
中级调度：内存调度/交换调度
低级调度：进程调度
```

------

## 2. 高级调度

从外存后备队列中选择作业调入内存。

又叫：

```text
作业调度
长程调度
```

------

## 3. 中级调度

决定哪些进程留在内存，哪些进程换出到外存。

又叫：

```text
交换调度
内存调度
```

------

## 4. 低级调度

从就绪队列中选择一个进程分配 CPU。

又叫：

```text
进程调度
短程调度
```

这是最频繁的调度。

------

## 5. 调度目标

```text
CPU 利用率高
系统吞吐量大
周转时间短
等待时间短
响应时间短
公平性好
截止时间保证
```

------

## 6. 作业调度算法

### 先来先服务 FCFS

按到达顺序调度。

优点：

```text
简单
公平
```

缺点：

```text
短作业可能等待很久
平均周转时间可能较长
```

------

### 短作业优先 SJF

优先调度运行时间短的作业。

优点：

```text
平均等待时间短
平均周转时间短
```

缺点：

```text
长作业可能饥饿
需要预知运行时间
```

------

### 优先级调度

优先级高的先运行。

问题：

```text
低优先级进程可能饥饿
```

解决：

```text
老化 Aging
```

------

### 高响应比优先 HRRN

响应比公式：

```text
响应比 = (等待时间 + 服务时间) / 服务时间
```

等价于：

```text
响应比 = 1 + 等待时间 / 服务时间
```

优点：

```text
兼顾短作业和长作业
等待越久优先级越高
```

------

## 7. 进程调度算法

### 时间片轮转 RR

每个进程运行一个时间片。

适合：

```text
分时系统
交互式系统
```

时间片过大：

```text
退化为 FCFS
```

时间片过小：

```text
上下文切换频繁，开销大
```

------

### 多级队列调度

把就绪队列分成多个队列。

例如：

```text
系统进程队列
交互进程队列
批处理进程队列
```

不同队列使用不同调度算法。

------

### 多级反馈队列调度

进程可以在多个队列之间移动。

特点：

```text
优先照顾短作业
照顾 I/O 型进程
防止长作业长期得不到运行
```

这是现代系统常见思想。

------

## 8. 实时调度

实时调度关注任务截止时间。

常见算法：

```text
最早截止时间优先 EDF
最低松弛度优先 LLF
速率单调调度 RMS
```

------

## 9. 死锁

死锁是多个进程因竞争资源而互相等待，导致都无法继续执行。

例子：

```text
P1 占有 R1，等待 R2
P2 占有 R2，等待 R1
```

------

## 10. 死锁四个必要条件

必须背：

```text
互斥条件
请求和保持条件
不可剥夺条件
循环等待条件
```

### 互斥

资源一次只能被一个进程使用。

### 请求和保持

进程已经占有资源，同时又请求新资源。

### 不可剥夺

资源不能被强制抢走，只能由进程主动释放。

### 循环等待

存在进程资源等待环。

------

## 11. 处理死锁的方法

```text
预防死锁
避免死锁
检测死锁
解除死锁
鸵鸟策略
```

------

## 12. 死锁预防

破坏四个必要条件之一。

### 破坏互斥

通常不现实，因为很多资源天然互斥。

### 破坏请求和保持

方法：

```text
一次性申请所有资源
申请新资源前释放已有资源
```

缺点：

```text
资源利用率低
可能饥饿
```

### 破坏不可剥夺

允许系统强制剥夺资源。

适用于：

```text
CPU
内存
```

不适合：

```text
打印机
磁带机
```

### 破坏循环等待

给资源编号，进程按编号递增顺序申请。

------

## 13. 死锁避免

核心思想：

```text
资源分配前先判断是否安全
只有安全才分配
```

典型算法：

```text
银行家算法
```

------

## 14. 安全状态

如果系统能找到一个安全序列，使所有进程最终都能完成，则系统处于安全状态。

```text
安全状态一定无死锁
不安全状态不一定死锁
死锁状态一定不安全
```

------

## 15. 银行家算法

涉及数据结构：

```text
Available：可用资源
Max：最大需求
Allocation：已分配资源
Need：还需要资源
```

关系：

```text
Need = Max - Allocation
```

资源请求能否满足：

```text
Request <= Need
Request <= Available
分配后仍处于安全状态
```

------

## 16. 死锁检测

允许死锁发生，然后检测。

常用：

```text
资源分配图
死锁检测算法
```

------

## 17. 死锁解除

方法：

```text
剥夺资源
撤销进程
进程回退
```

选择撤销进程时考虑：

```text
进程优先级
已运行时间
还需资源数
已占资源数
撤销代价
```

# 四、存储器管理

## 2. 程序装入


## 3. 程序链接

链接方式：

```text
静态链接
装入时动态链接
运行时动态链接
```

------

## 4. 地址概念

```text
逻辑地址
物理地址
地址空间
存储空间
```

### 逻辑地址

程序中使用的地址。

### 物理地址

内存中的真实地址。

### 地址重定位

把逻辑地址转换成物理地址。

------

## 5. 连续分配管理

连续分配是给进程分配一块连续内存。

方式：

```text
单一连续分配
固定分区分配
动态分区分配
可重定位分区分配
```

------

## 6. 单一连续分配

内存分成：

```text
操作系统区
用户区
```

一次只能运行一个用户程序。

特点：

```text
简单
利用率低
无并发
```

------

## 7. 固定分区分配

内存预先分成若干固定分区。

问题：

```text
内部碎片
分区大小固定
灵活性差
```

------

## 8. 动态分区分配

根据进程需要动态划分内存。

常见算法：

```text
首次适应 FF
循环首次适应 NF
最佳适应 BF
最坏适应 WF
```

------

## 9. 首次适应 FF

从空闲分区表开头找第一个足够大的分区。

优点：

```text
速度快
保留高地址大分区
```

缺点：

```text
低地址碎片多
```

------

## 10. 循环首次适应 NF

从上次查找位置继续找。

优点：

```text
分布较均匀
```

缺点：

```text
可能缺少大分区
```

------

## 11. 最佳适应 BF

选择最小且足够大的分区。

优点：

```text
大分区保留较好
```

缺点：

```text
产生很多小碎片
```

------

## 12. 最坏适应 WF

选择最大的空闲分区。

优点：

```text
剩余分区仍较大
```

缺点：

```text
大分区很快被切碎
```

------

## 13. 碎片

```text
内部碎片
外部碎片
```

### 内部碎片

分配给进程的内存内部没用完。

常见于：

```text
固定分区
分页
```

### 外部碎片

空闲内存总量够，但不连续。

常见于：

```text
动态分区
分段
```

------

## 14. 紧凑

通过移动进程，把分散空闲区合并成大空闲区。

缺点：

```text
开销大
需要动态重定位支持
```

------

## 15. 对换 Swapping

把暂时不用的进程换出到外存，需要时再换入内存。

作用：

```text
提高内存利用率
支持更多进程并发
```

------

## 16. 分页存储管理

把地址空间分成固定大小的页。

```text
逻辑地址空间 → 页 page
物理内存 → 块/页框 frame
```

页和页框大小相同。

------

## 17. 页表

页表记录：

```text
页号 → 块号
```

逻辑地址结构：

```text
页号 P + 页内位移 W
```

物理地址：

```text
块号 F + 页内位移 W
```

------

## 18. 基本地址变换

过程：

```text
根据页号查页表
找到对应物理块号
块号 + 页内位移形成物理地址
```

------

## 19. 快表 TLB

TLB 是页表缓存。

作用：

```text
加快地址转换
减少访问内存次数
```

TLB 命中：

```text
直接得到物理块号
```

TLB 未命中：

```text
访问内存中的页表
```

------

## 20. 多级页表

目的：

```text
减少页表占用的连续内存空间
```

常见：

```text
二级页表
三级页表
多级页表
```

------

## 21. 反置页表

反置页表按物理块建立表项。

特点：

```text
每个物理块一个表项
节省页表空间
查找较复杂
```

------

## 22. 分段存储管理

按程序逻辑单位分段。

例如：

```text
代码段
数据段
栈段
子程序段
共享段
```

逻辑地址：

```text
段号 S + 段内位移 W
```

段表内容：

```text
段基址
段长
保护位
```

------

## 23. 分页和分段区别

| 项目         | 分页           | 分段             |
| ------------ | -------------- | ---------------- |
| 划分依据     | 固定大小       | 逻辑单位         |
| 大小         | 页大小固定     | 段大小不固定     |
| 地址空间     | 一维           | 二维             |
| 碎片         | 内部碎片       | 外部碎片         |
| 用户是否可见 | 通常不可见     | 用户可见         |
| 目的         | 提高内存利用率 | 满足程序逻辑结构 |

------

## 24. 段页式管理

先分段，再分页。

地址结构：

```text
段号
段内页号
页内位移
```

优点：

```text
兼有分段的逻辑清晰
兼有分页的内存利用率高
```

缺点：

```text
地址转换复杂
需要段表和页表
```

# 五、虚拟存储器

## 1. 虚拟存储器概念

虚拟存储器使程序感觉拥有比实际内存更大的空间。

核心思想：

```text
部分装入
按需调入
页面置换
```

------

## 2. 虚拟存储器特征

```text
多次性
对换性
虚拟性
```

### 多次性

程序不必一次全部装入内存。

### 对换性

程序运行过程中可以换入换出。

### 虚拟性

逻辑上扩充内存容量。

------

## 3. 局部性原理

程序访问呈现局部性。

包括：

```text
时间局部性
空间局部性
```

### 时间局部性

刚访问过的数据，近期可能再次访问。

### 空间局部性

访问某地址后，附近地址可能也会被访问。

------

## 4. 请求分页

页面需要时才调入内存。

页表项通常包括：

```text
页号
物理块号
状态位/存在位
访问字段
修改位
外存地址
```

------

## 5. 缺页中断

访问的页面不在内存时发生缺页中断。

处理过程：

```text
CPU 访问页面
发现页不在内存
产生缺页中断
操作系统查找页面在外存的位置
寻找空闲物理块
若无空闲块则页面置换
把页面调入内存
修改页表
重新执行被中断指令
```

------

## 6. 页面置换算法

常见算法：

```text
最佳置换 OPT
先进先出 FIFO
最近最久未使用 LRU
Clock 算法
改进 Clock 算法
最少使用 LFU
页面缓冲算法 PBA
```

------

## 7. OPT

淘汰未来最长时间不会被访问的页面。

优点：

```text
缺页率最低
```

缺点：

```text
无法实际实现
只能作为理论标准
```

------

## 8. FIFO

淘汰最早进入内存的页面。

优点：

```text
简单
```

缺点：

```text
可能出现 Belady 异常
```

------

## 9. Belady 异常

正常直觉：

```text
内存块数增加，缺页次数应该减少
```

但 FIFO 中可能出现：

```text
物理块数增加，缺页次数反而增加
```

这就是 Belady 异常。

------

## 10. LRU

淘汰最近最长时间未被访问的页面。

依据：

```text
时间局部性
```

优点：

```text
效果接近 OPT
```

缺点：

```text
实现开销大
```

------

## 11. Clock 算法

又称最近未使用算法 NRU 的近似实现。

使用访问位 A。

过程：

```text
指针循环扫描
若 A=0，淘汰
若 A=1，置 0，继续扫描
```

------

## 12. 改进 Clock

同时考虑：

```text
访问位 A
修改位 M
```

优先淘汰：

```text
未访问且未修改的页面
```

因为修改过的页面换出时要写回磁盘。

------

## 13. 抖动

进程频繁缺页，大量时间用于页面换入换出，真正执行时间很少。

原因：

```text
分配物理块太少
多道程序度过高
工作集无法装入内存
```

解决：

```text
降低多道程序度
增加物理块
使用工作集模型
调整页面置换策略
```

------

## 14. 工作集

某段时间窗口内进程实际访问的页面集合。

作用：

```text
判断进程需要多少物理块
防止抖动
```

------

## 15. 请求分段

按段进行请求调入。

特点：

```text
支持动态链接
支持共享
支持保护
```

------

## 11. 设备无关性

## 12. 缓冲区管理

```text
双缓冲

缓冲池
```

## 13. SPOOLing

```text

```

```text
把独占设备改造成虚拟共享设备
```

------

## 14. 磁盘调度算法

```text
FCFS
SSTF
SCAN
C-SCAN
LOOK
C-LOOK
```

### FCFS

按请求到达顺序服务。

优点：

```text
公平
简单
```

缺点：

```text
寻道距离可能大
```

### SSTF

最短寻道时间优先。

优点：

```text
平均寻道时间短
```

缺点：

```text
可能饥饿
```

### SCAN

电梯算法。

磁头按一个方向移动，沿途服务请求，到头后反向。

### C-SCAN

循环扫描。

只在一个方向服务，到头后快速返回另一端。

### LOOK

SCAN 的改进。

不一定走到磁盘端点，只走到最远请求处。

### C-LOOK

C-SCAN 的改进。

只在请求范围内循环。

# 十、多处理机操作系统

## 1. 多处理机系统

多个 CPU 共同完成任务。

目的：

```text

```

------

## 2. 多处理机分类

```text

对称多处理 SMP

```

------

## 3. SMP

所有处理器地位相同。

特点：

```text
共享内存
共享 I/O
同一个操作系统管理所有 CPU
任何 CPU 都可运行内核代码
```

------

## 4. AMP

主处理器负责系统控制，从处理器执行分配任务。

特点：

```text
结构简单
主处理器可能成为瓶颈
可靠性较差
```

------

## 5. 多处理机调度

问题：

```text

负载均衡
处理器亲和性
并行程序调度
同步开销
```

------

## 6. 处理器亲和性

尽量让进程继续在原 CPU 上运行。

原因：

```text
缓存中可能已有该进程数据
减少 Cache 失效
提高性能
```

------

## 7. 负载均衡

让多个 CPU 工作量尽量均匀。

方式：

```text
推迁移 push migration
拉迁移 pull migration
```

## 最容易混淆的概念

| 概念 1   | 概念 2   | 区别                           |
| -------- | -------- | ------------------------------ |
| 并发     | 并行     | 并发是交替，并行是真同时       |
| 程序     | 进程     | 程序静态，进程动态             |
| 进程     | 线程     | 进程分配资源，线程调度执行     |
| 同步     | 互斥     | 同步强调顺序，互斥强调独占     |
| 死锁预防 | 死锁避免 | 预防破坏条件，避免判断安全     |
| 逻辑地址 | 物理地址 | 程序看到的地址 vs 内存真实地址 |
| 分页     | 分段     | 固定大小 vs 逻辑单位           |
| 内部碎片 | 外部碎片 | 分区内部浪费 vs 分区之间零散   |
| 缓冲     | 缓存     | 缓和速度差 vs 保存常用数据     |
| 文件目录 | inode    | 名字映射 vs 文件元数据         |
