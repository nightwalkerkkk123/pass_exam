# 第一章 操作系统引论总览

本章结构是：

```text
1.1 操作系统的目标和作用
```

# 一、为什么学习操作系统？

课件一开始强调：这门课不是教你“怎么使用 Windows / Linux”，而是让你理解：

```text
操作系统内部如何工作
操作系统用什么数据结构和算法
操作系统如何在性能、方便性、安全性之间权衡
```

比如你运行一个简单程序：

```c
printf("hello world");
```

表面上只是输出一句话，但操作系统背后要做很多事：

```text
找到程序文件
创建进程
装入程序代码
建立地址空间
处理缺页中断
分配内存
执行系统调用
找到输出设备
把字符转换成屏幕像素
最终显示 hello world
```

所以操作系统课程的重点是：

> **理解计算机系统如何协调 CPU、内存、磁盘和设备。**

------

# 二、操作系统是什么？

## 1. 操作系统定义

课件中的定义是：

> 操作系统是一组有效控制和管理计算机系统的硬件和软件资源、合理组织计算机工作流程、方便用户的程序集合。

关键词有三个：

```text
有效：提高系统效率和资源利用率
合理：公平组织资源，避免死锁和饥饿
方便：为用户提供良好接口
```

------

## 2. 操作系统做什么？

操作系统主要完成所有：

```text
硬件相关
应用无关
```

的工作。

比如：

```text
CPU 调度
内存分配
磁盘读写
设备控制
文件管理
系统调用
权限保护
```

它不直接做具体应用问题。

例如：

```text
操作系统不做天气预报
不做房屋设计
不是编译程序
不是办公软件
```

它提供的是底层支撑。

------

# 三、操作系统的四种理解角度

## 1. 从系统组成看：OS 是系统软件

计算机系统可以分为：

```text
硬件
系统软件
应用软件
```

操作系统属于系统软件，是应用软件和硬件之间的基础层。

------

## 2. 从资源管理看：OS 是资源管理者

计算机资源主要有四类：

```text
处理器 CPU
存储器 Memory
I/O 设备
信息：程序和数据
```

对应操作系统功能：

| 资源       | OS 功能               |
| ---------- | --------------------- |
| CPU        | 处理机管理 / 进程管理 |
| 内存       | 存储器管理            |
| I/O 设备   | 设备管理              |
| 数据和程序 | 文件管理              |

------

## 3. 从软件分层看：OS 是扩充机器

裸机很难直接使用。

操作系统把裸机扩充成更好用的虚拟机。

例如：

```text
真实 CPU      → 多个进程好像各自拥有 CPU
真实内存     → 虚拟内存
真实磁盘块   → 文件
真实设备     → 统一 I/O 接口
```

也就是说：

> 操作系统把复杂硬件包装成更容易使用的抽象机器。

------

## 4. 从用户角度看：OS 是用户和裸机之间的接口

用户通过这些方式使用操作系统：

```text
命令方式
系统调用方式
图形窗口方式
```

例如：

```text
命令接口：ls、cd、dir、copy
系统调用：open、read、write、fork
图形接口：窗口、图标、菜单
```

------

# 四、操作系统的目标

常考四个目标：

```text
方便性
有效性
可扩充性
开放性
```

## 1. 方便性

让用户不必直接操作硬件。

例如：

```c
read(fd, buf, 100);
```

用户只需要调用 `read`，不用关心磁盘控制器、扇区、缓冲区、中断等细节。

------

## 2. 有效性

提高资源利用率和系统吞吐量。

包括：

```text
提高 CPU 利用率
提高内存利用率
提高 I/O 设备利用率
提高系统吞吐量
```

------

## 3. 可扩充性

操作系统应便于添加新功能。

例如：

```text
新增设备驱动
新增文件系统
支持新硬件
增加网络功能
```

------

## 4. 开放性

操作系统应遵循标准，便于互联互通。

例如：

```text
兼容标准接口
支持网络协议
支持跨平台软件
```

------

# 五、操作系统的发展过程

这一节非常重要，容易考选择题和简答题。

------

## 1. 无操作系统阶段

早期计算机没有操作系统，用户人工操作。

特点：

```text
用户独占全机
人工装入程序
CPU 等待人工操作
资源利用率低
```

缺点：

```text
人机矛盾
CPU 和 I/O 速度不匹配
```

------

## 2. 脱机输入/输出

英文：

```text
Off-Line I/O
```

思想：

> 用外围机先把数据从慢速设备转移到磁带/磁盘，再由主机高速读取。

优点：

```text
减少 CPU 空闲时间
提高 I/O 速度
```

------

## 3. 单道批处理系统

单道批处理系统一次只让一道作业进入内存运行。

特点：

```text
自动性
顺序性
单道性
```

优点：

```text
减少人工干预
作业可以自动连续处理
```

缺点：

```text
内存中只有一道程序
程序等待 I/O 时 CPU 仍然空闲
资源利用率不高
```

------

## 4. 多道批处理系统

核心思想：

> 内存中同时装入多道程序，一个程序等待 I/O 时，CPU 去运行另一个程序。

优点：

```text
CPU 利用率高
内存利用率高
I/O 设备利用率高
系统吞吐量大
```

缺点：

```text
平均周转时间长
无交互能力
系统复杂
```

需要解决的问题：

```text
处理机管理
内存管理
I/O 设备管理
文件管理
作业管理
```

这是现代操作系统的基础。

------

## 5. 分时系统

分时系统是为了满足用户交互需求产生的。

核心思想：

> 多个用户通过终端同时使用一台主机，系统按时间片轮流为每个用户服务。

分时系统特征：

```text
多路性
独立性
交互性
及时性
```

### 多路性

多个用户同时连接主机。

### 独立性

每个用户感觉自己独占机器。

### 交互性

用户可以输入命令、修改程序、查看结果。

### 及时性

系统能在较短时间内响应用户请求。

课件中提到响应时间公式：

```text
T = n × q
```

其中：

```text
n：用户数
q：时间片
T：响应时间
```

时间片选择很关键：

```text
时间片太大 → 响应慢
时间片太小 → 上下文切换开销大
```

------

## 6. 实时系统

实时系统强调：

> 系统必须在规定时间内响应外部事件。

应用：

```text
实时控制
实时信息处理
飞机控制
导弹发射
卫星控制
订票系统
```

实时任务分类：

```text
周期性实时任务
非周期性实时任务
硬实时任务
软实时任务
```

### 硬实时任务

必须严格满足截止时间。

例如：

```text
导弹控制
飞机飞行控制
工业控制
```

错过时间可能造成严重后果。

### 软实时任务

偶尔错过截止时间影响不大。

例如：

```text
视频播放
在线音频
信息查询
```

------

# 六、操作系统的基本特性

四个字必须背：

```text
并发
共享
虚拟
异步
```

------

## 1. 并发

并发是指：

> 两个或多个事件在同一时间间隔内发生。

注意和并行区别：

```text
并发：同一时间段内发生，单 CPU 上宏观同时、微观交替
并行：同一时刻真正同时发生，需要多个 CPU 或多核
```

例子：

```text
0~10ms    运行 A
10~20ms   运行 B
20~30ms   运行 C
```

宏观上 A、B、C 同时运行，微观上 CPU 每一刻只执行一个。

------

## 2. 共享

共享是指：

> 系统资源可供多个并发进程共同使用。

共享方式有两种。

### 互斥共享

一段时间内只允许一个进程访问。

例如：

```text
打印机
磁带机
共享变量
某些表格
```

这种资源叫：

```text
临界资源
独占资源
```

### 同时访问

允许多个进程在一段时间内同时访问。

例如：

```text
磁盘文件
只读代码
可重入程序
```

------

## 3. 虚拟

虚拟是指：

> 通过某种技术，把一个物理实体变成多个逻辑上的对应物。

常见虚拟：

```text
虚拟处理机
虚拟存储器
虚拟设备
虚拟信道
```

例子：

```text
一个 CPU → 多个用户感觉自己都有 CPU
小内存 → 用户感觉内存很大
一台打印机 → 多个用户感觉都能打印
```

注意：

> 虚拟设备的平均速度通常会低于真实设备速度。

如果一个物理设备虚拟成 n 个逻辑设备，平均速度大约是原来的：

```text
1 / n
```

------

## 4. 异步

异步是指：

> 进程不是一气呵成地执行，而是走走停停，以不可预知的速度推进。

原因：

```text
等待 CPU
等待 I/O
等待资源
等待锁
被中断
被调度
```

但只要运行环境相同，程序多次运行仍应得到相同结果。

------

# 七、操作系统的主要功能

本章列出五大功能：

```text
处理机管理
存储器管理
设备管理
文件管理
用户接口
```

------

# 1. 处理机管理

处理机管理主要围绕进程展开。

包括：

```text
进程控制
进程同步
进程通信
调度
```

------

## 进程控制

负责：

```text
创建进程
撤销进程
控制进程状态转换
创建线程
终止线程
```

------

## 进程同步

解决多个进程有序运行的问题。

包括两类：

```text
互斥：访问临界资源时不能同时进入
同步：合作进程之间要按一定顺序执行
```

例如：

```text
先输入数据
再计算
最后打印
```

------

## 进程通信

进程之间需要交换信息。

例如：

```text
输入进程 → 计算进程 → 打印进程
```

常见方式：

```text
消息传递
共享内存
管道
信号
```

------

## 调度

调度分为：

```text
作业调度
进程调度
线程调度
```

核心问题：

> 就绪队列里有多个进程，CPU 应该分给谁？

------

# 2. 存储器管理

存储器管理包括：

```text
内存分配
内存保护
地址映射
内存扩充
```

------

## 内存分配

分为：

```text
静态分配
动态分配
```

静态分配：

```text
作业装入时确定内存空间
运行期间不能再申请额外空间
不能移动
```

动态分配：

```text
运行过程中可以申请新空间
允许作业在内存中移动
```

------

## 内存保护

目的：

> 确保每个程序只访问自己的内存空间。

常用机制：

```text
上下界寄存器
越界检查
越界中断
```

如果程序访问非法地址，系统发出中断，停止该程序。

------

## 地址映射

程序中的地址叫：

```text
逻辑地址
相对地址
```

内存真实地址叫：

```text
物理地址
```

地址映射就是：

```text
逻辑地址 → 物理地址
```

这个功能需要硬件支持。

------

## 内存扩充

不是增加真实内存，而是通过虚拟存储技术：

```text
从逻辑上扩充内存容量
让用户感觉内存比实际更大
让更多程序并发运行
```

主要依赖：

```text
请求调入
置换
```

------

# 3. 设备管理

设备管理的任务：

```text
完成用户 I/O 请求
分配 I/O 设备
提高 CPU 和 I/O 设备利用率
提高 I/O 速度
方便用户使用设备
```

主要功能：

```text
缓冲管理
设备分配
设备处理
虚拟设备
```

------

## 缓冲管理

CPU 快，I/O 设备慢，所以需要缓冲区。

作用：

```text
缓和速度不匹配
提高 CPU 利用率
提高系统吞吐量
```

常见缓冲机制：

```text
单缓冲
双缓冲
公用缓冲池
```

------

## 设备分配

根据用户请求和系统资源情况，为进程分配设备。

涉及：

```text
设备控制表
控制器控制表
通道控制表
```

对于独占设备，还要考虑系统是否安全。

------

## 设备处理

设备处理程序也叫：

```text
设备驱动程序
```

它负责：

```text
检查 I/O 请求合法性
查看设备状态
设置设备工作方式
向设备控制器发送命令
响应中断请求
```

------

# 4. 文件管理

文件管理包括：

```text
文件存储空间管理
目录管理
文件读写管理
文件保护
```

------

## 文件存储空间管理

负责：

```text
为文件分配外存空间
回收文件释放的空间
提高外存利用率
提高文件系统速度
```

常以盘块为基本单位，盘块大小通常为：

```text
512B ~ 8KB
```

------

## 目录管理

目录管理实现：

```text
按名存取
文件共享
快速目录查询
```

目录项通常包含：

```text
文件名
文件属性
文件物理位置
```

------

## 文件读写管理和保护

文件读写过程：

```text
根据文件名查目录
找到文件外存位置
根据读写指针访问文件
读写后修改指针
```

文件保护目标：

```text
防止未授权访问
防止冒名访问
防止错误方式使用文件
```

------

# 5. 用户接口

用户接口分三类：

```text
命令接口
程序接口
图形接口
```

------

## 命令接口

分为：

```text
联机用户接口
脱机用户接口
```

联机接口：

```text
用户输入一条命令
系统解释并执行
再等待下一条命令
```

例如：

```text
ls
cd
dir
copy
del
```

脱机接口：

```text
批处理用户用 JCL 写好作业说明书
系统按说明书自动执行
```

------

## 程序接口

程序接口就是系统调用。

它是：

> 用户程序取得操作系统服务的唯一途径。

例如：

```text
open
read
write
close
fork
exec
wait
```

------

## 图形接口

图形接口通过：

```text
窗口
图标
菜单
对话框
鼠标操作
```

让用户不用记复杂命令。

------

# 八、操作系统结构设计

本章最后讲 OS 结构。

主要包括：

```text
无结构操作系统
模块化结构
层次结构
微内核结构
```

------

## 1. 无结构操作系统

早期 OS 只追求功能和效率，没有清晰结构。

特点：

```text
过程之间随意调用
结构混乱
调试困难
维护困难
容易出错
```

------

## 2. 模块化结构

把 OS 按功能划分成模块：

```text
进程管理模块
存储器管理模块
I/O 设备管理模块
文件管理模块
```

模块还可以继续分成子模块。

优点：

```text
提高正确性
提高可理解性
提高可维护性
增强适应性
加快开发过程
```

缺点：

```text
模块划分不一定准确
模块之间依赖关系复杂
结构仍可能不够清晰
```

------

## 3. 层次结构

层次结构把 OS 分成若干层。

原则：

```text
上层只能调用相邻下层
调用关系有序
避免循环依赖
```

优点：

```text
结构清晰
容易调试
容易验证
```

缺点：

```text
层次划分困难
效率可能降低
```

课件中提到 Dijkstra 的 THE 系统，是早期层次结构 OS 的代表。

------

## 4. 微内核结构

微内核只保留最核心功能。

通常包括：

```text
进程管理
存储器管理
进程间通信
低级 I/O
```

其他功能放到用户态服务器中，例如：

```text
文件服务器
网络服务器
作业服务器
设备服务器
```

------

## 微内核优点

```text
扩充性好
可靠性好
适合分布式处理
适合嵌入式和实时系统
```

------

## 微内核缺点

```text
服务之间通过消息传递
系统调用开销较大
效率可能低于宏内核
```

------

```text
批处理系统：周转时间和系统吞吐量
```

## 作业题 2

题干关键词对应：

```text
多个程序同时放入主存，宏观并行运行 → 多道程序设计
一个程序划分成若干同时执行模块 → 并发程序设计
多个用户在终端交互输入、排错、控制程序 → 分时 OS
多个计算机组成系统，无主次，共享资源，协同执行 → 分布式 OS
响应时间重要性超过资源利用率，用于导弹、飞机、订票 → 实时 OS
```

所以：

```text
A = ⑧ 多道程序设计
B = ⑨ 并发程序设计
C = ① 分时 OS
D = ⑤ 分布式 OS
E = ② 实时 OS
```
